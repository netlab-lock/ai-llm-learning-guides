🔄 02 · 位置编码外推

位置编码外推是长上下文训练的核心技术挑战。RoPE 是当前主流的位置编码，但它在训练长度之外的位置上性能急剧下降。

1. RoPE 回顾

通俗类比：RoPE 编码位置，就像用钟表上的指针位置来表示时间。时针、分针、秒针的角度共同确定一个时刻，类似于模型通过频率叠加来“感知”序列中词的位置。
常见误区：容易误以为 RoPE 的‘频率’是原始的数学傅里叶变换频率。注意，这里每对维度对应一个频率，该频率是为编码序列中的**相对位置**而设计的，与输入语义无关。
深入探讨：RoPE 核心是旋转矩阵。对于 Query (q) 和 Key (k) 向量中第 `i` 个二维子空间 `(q_2i, q_2i+1)`，施加的旋转角度为 `θ_i = 10000^{-2i/d}`，其中 `d` 是维度。位置 `m` 的旋转矩阵为 `R(m) = [cos(mθ_i), -sin(mθ_i); sin(mθ_i), cos(mθ_i)]`。这使得点积 `q·k` 仅依赖相对位置 `m-n`。
q_m = R(m) * q
k_n = R(n) * k

点积 q_m · k_n = (R(m)q)^T (R(n)k) = q^T R(n-m) k
特性RoPE绝对位置编码 (如 Sinusoidal)
编码方式相对位置，通过旋转绝对位置，通过向量相加
与语义交互在注意力计算时（Q, K）在输入嵌入层
外推潜力原生较差，需改进同样较差

1.1 基本原理

RoPE (Rotary Position Embedding, Su et al. 2021) 通过旋转矩阵将位置信息编码到 Query 和 Key 向量中：

RoPE 公式

f(x, m) = R(m) · x

其中 R(m) 是旋转矩阵:
R(m) = [[cos(mθ₁), -sin(mθ₁), 0, 0, ...],
[sin(mθ₁), cos(mθ₁), 0, 0, ...],
[0, 0, cos(mθ₂), -sin(mθ₂), ...],
[0, 0, sin(mθ₂), cos(mθ₂), ...],
...]

θ_i = 10000^(-2i/d) ← 不同维度有不同的旋转频率

1.2 复数形式

RoPE 可以用复数形式更简洁地表达：

f(x, m) = x · e^(imθ)

其中:
x = [x₁+ix₂, x₃+ix₄, ...] ← 将相邻两个维度视为复数
m = 位置索引
θ = 旋转频率

1.3 关键性质

  • 相对位置编码：Q·K 的内积只依赖于相对位置 m-n
  • 高效实现：只需逐元素乘法，无需构造矩阵
  • 高频与低频：不同维度有不同的旋转频率，高频维度编码局部位置，低频维度编码全局位置

2. 为什么 RoPE 不能直接外推

通俗类比：位置编码就像给教室里的学生编号。训练时模型只见过 1-100 号，推理时突然遇到 101 号，模型就像遇到超出编号系统的新学生，会感到困惑和混乱，导致性能崩溃。
常见误区：一个常见误区是认为只要增加最大位置值 `L` 就能解决。盲目增加 `L` 不仅效果差，还可能因 `cos(mθ)` 在 `m` 过大时出现剧烈震荡，破坏注意力分数的数值稳定性。
深入探讨：直接外推失效的核心原因是**频率外推**。RoPE 的基础频率 `θ = 10000^{-2i/d}` 决定了每个维度对旋转的敏感度。当位置 `m` 超过训练最大长度 `L` 时，`mθ` 会远大于训练时见过的范围。例如，一个维度的波长 `λ = 2π/θ`，当 `m` 超过 `λ/2`，`cos(mθ)` 就进入了完全不同的周期，其分布与训练分布相去甚远。

  • Q·K 的内积分布与训练时差异巨大

3. Position Interpolation (PI)

通俗类比：PI 就像把一个 100 厘米的尺子，刻度加密成 200 个小格来测量 200 厘米。尺子长度没变，但每个小格代表的“位置”缩小了，使得模型能在熟悉的“刻度密度”下处理更长的序列。
常见误区：最易犯的错是忽略 PI 要求对**整个序列**进行线性缩放。如果只缩放部分位置，或错误地使用非均匀缩放，会破坏相对位置关系，导致注意力混乱。另外，过度压缩（如 8k->128k）会损失高频位置信息。
深入探讨：PI 的核心操作非常简单：将输入位置 `m` 替换为 `m' = m * (L/L')`，其中 `L` 是训练长度，`L'` 是目标外推长度。这等价于将每个维度的有效波长 `λ` 缩小 `L'/L` 倍，即 `θ' = θ * (L/L')`。例如，从 4k 扩展到 32k，`L/L' = 4k/32k = 1/8`，所有位置值除以 8。
原始位置: [0, 1, 2, ..., L]

插值后:   [0, 1/8, 2/8, ..., L/8] (当 L'=8L)

等效于: 让模型认为总序列长度仍是 L，
但每个“步长”的信息量被压缩。
步骤操作等效理解
位置缩放`m' = m * (L / L')`将长序列“压缩”回训练时的短序列视角
频率缩放`θ' = θ * (L' / L)`增加所有维度的基频，缩短波长
微调在插值后位置编码上微调模型让模型适应新的、被压缩的位置表示

3.1 原理

Position Interpolation (Chen et al. 2023) 的思想非常简单：将外推问题转化为内插问题。

Position Interpolation

原始: θ_m = m · θ_i (位置 m 可能超出训练范围)

PI: θ'_m = (m · L_train / L_target) · θ_i

等价于: 将位置索引从 [0, L_target] 缩放到 [0, L_train]

例: 训练 4K, 外推到 16K
位置 8000 → 8000 × 4096/16384 = 2000
(位置 8000 被映射到训练范围内的位置 2000)

3.2 优缺点

维度评估
优点极其简单，只需修改位置索引；不需要重新训练
优点少量微调（1000 步）即可恢复性能
缺点分辨率损失：高频信息被压缩
缺点缩放因子越大，分辨率损失越严重
适用2x-8x 外推效果好；16x+ 性能下降明显

4. NTK-aware Scaling

通俗类比：NTK-aware 就像调整音乐播放的采样率。它不是简单地把所有音调都降低（像 PI 那样），而是智能地判断：对高频音符做更多降调以防止走音，对低频音符做较少调整以保持音色，从而在更长的序列上“演奏”出清晰的乐章。
常见误区：易误解为 NTK-aware 只是一种缩放 `θ` 的简单方法。实际上，它的关键在于**非均匀缩放**。误用均匀缩放因子会丢失其保护高频信息的核心优势，效果可能退化为普通的 PI 或直接外推。
深入探讨：NTK-aware 的核心是修改基频 `θ` 的公式：`θ' = θ * (L'/L)^{-2i/(d-2)}`。这个公式的巧妙之处在于，它对不同频率 `i` 施加不同的缩放强度。维度索引 `i` 越大（对应越高频），缩放比例 `L'/L` 的负指数越接近 `-1`，即缩放越接近 PI；`i` 越小（低频），指数越接近 `0`，即几乎不缩放。这通过调整 `d`（RoPE 的维度）来控制曲线。

4.1 原理

NTK-aware Scaling (Reddit "bloc97") 的核心洞察：不应该均匀缩放所有维度，应该保持高频维度不变，只缩放低频维度。

NTK-aware Scaling

原始: θ_i = base^(-2i/d) (base = 10000)

NTK: θ'_i = (base · α)^(-2i/d)

等价于: 将 base 从 10000 改为 10000 · α

效果:
- 高频维度 (i 小): θ 几乎不变 → 保持局部分辨率
- 低频维度 (i 大): θ 显著增大 → 周期变短，适应更长范围

4.2 直觉理解

NTK-aware 的直觉：

  • 高频维度负责编码"近距离"位置关系，不需要外推
  • 低频维度负责编码"远距离"位置关系，需要适应更长范围
  • 通过修改 base frequency，可以让低频维度"更慢地旋转"，覆盖更长范围

5. YaRN (Yet another RoPE extensioN)

通俗类比：YaRN 就像一套智能压缩系统。它把数据（位置信号）分成三部分：高价值文件（低频信息）保持原样，中等文件（中频）进行适度压缩，普通文件（高频）进行高强度压缩。最终打包成一个更小但能完整解压的包，用于超长序列。
常见误区：使用 YaRN 时，一个关键且易被忽略的步骤是**必须配合调整注意力的温度**。原始论文推荐温度 `t` 为 `0.1 * ln(2) / ln(base)`。忽略这一步会导致注意力权重过于尖锐或平坦，影响生成质量。
深入探讨：YaRN 的核心是“分而治之”。它将 RoPE 的维度按波长 `λ = 2π/θ` 分为三组： 1. **No change (低频)**: `λ > L`，直接外推，不做缩放。 2. **NTK-aware (中频)**: `L > λ > L'`，进行如 NTK-aware 的非均匀缩放。 3. **PI (高频)**: `λ < L'`，应用严格的 Position Interpolation 线性缩放。 此外，它引入一个可学习的温度缩放因子 `γ`，并将其融入注意力分数计算。

5.1 三种方案

YaRN (Peng et al. 2023) 综合了 PI 和 NTK 的优点，提出三种方案：

方案 1: NTK-by-parts

将维度分为三组：

  • 高频维度（周期 < 训练长度）：不缩放
  • 中频维度（周期 ≈ 训练长度）：线性插值缩放
  • 低频维度（周期 > 训练长度）：PI 缩放

方案 2: Dynamic NTK

在推理时根据实际输入长度动态调整缩放因子：

α = max(1, L_input / L_train)

方案 3: YaRN (完整版)

在 NTK-by-parts 的基础上，额外添加注意力缩放：

attention(Q, K) = Q·K^T / √d × (1 + 0.1 × ln(α)) ← 温度因子

6. 其他外推方法

通俗类比：除了改造 RoPE 本身，还有像“旋转位置编码混合”（RoPE-H）和“加性旋转位置编码”（ARPE）等思路。它们就像给钟表（RoPE）换上新机芯（H）或增加游丝（A），试图从根本上提升其走时（编码）的精度和范围。
常见误区：切勿认为存在一种“万能”外推方法。许多方法（如 ALiBi）根本不使用 RoPE，而是采用完全不同的原理（如线性偏置）。直接套用为 RoPE 设计的外推技巧（如 PI）到这些方法上，通常无效且会引入错误。
深入探讨：**ALiBi (Attention with Linear Biases)**: 不学习或使用任何位置编码。它直接在注意力分数上加一个与距离 `i-j` 成正比的线性偏置：`score_{ij} = q_i · k_j - m * |i-j|`，`m` 是头相关的斜率。优势是零额外参数，外推性好，但对相对位置的建模能力较弱。 **长度外推微调 (LEF)**: 在预训练后，使用一个比预训练更长的序列对位置编码部分（或全部模型）进行微调。这是最暴力但有时有效的后处理方法。

7. 各方法综合对比

通俗类比：选择外推方法就像选择交通工具。PI 像公交车（通用、简单），NTK-aware 像混合动力车（平衡、节能），YaRN 像全自动驾驶（智能、昂贵）。ALiBi 则像自行车（结构简单，但爬长坡吃力）。没有最好，只有最适合你的目的地（序列长度）和预算（计算资源）。
常见误区：最常见的错误是脱离实际应用场景进行盲目对比。必须考虑：1) 目标长度与训练长度的比值；2) 是否允许额外微调；3) 对模型原有性能的保持要求。一个在 2x 长度上优秀的方法，在 8x 长度上可能失效。
深入探讨：综合选择建议： - **小幅外推 (≤2x L)**: PI 或 NTK-aware 是性价比最高的选择，只需极少或无需微调。 - **中等外推 (2x-8x L)**: YaRN 或动态 NTK-aware 是更稳妥的选择，通常需要少量微调。 - **大幅外推 (>8x L)**: 必须采用 YaRN 等高级方法，并配合在长文本上的微调。同时应评估 ALiBi 等替代方案。 - **全新预训练**: 从一开始就设计好外推策略，如采用 ALiBi 或配置好 RoPE 的基频，是最根本的解决方案。

RoPE旋转角度计算：高频维度θ=0.1：位置4096时旋转角度=0.1×4096=409.6 rad，相当于409.6/(2π)≈65.2个完整周期；位置8192时=819.2 rad≈130.4个周期。低频维度θ=0.0001：位置4096时=0.4096 rad≈0.065个周期；位置8192时=0.8192 rad≈0.13个周期。高频维度在外推时更容易出问题的原因：①高频维度的旋转角度在外推时快速增大，超出训练时见过的范围——训练时可能只见过0-2π的旋转，外推到409.6 rad时模型从未见过如此大的角度；②高频维度的正弦/余弦值在外推时会剧烈振荡——sin(409.6)的值与训练时的sin(0-2π)分布完全不同；③低频维度的旋转角度增长慢，外推时仍在训练范围内。
    关键要点：①RoPE的高频维度在外推时最脆弱；②YaRN/NTK等外推方法通过调整θ来缓解这个问题；③低频维度天然具有更好的外推性。
  2. PI(Position Interpolation)映射计算：PI将位置线性缩放：pos' = pos × (L_train/L_target) = pos × (4096/32768) = pos/8。位置16000映射到：16000/8 = 2000。信息损失分析：①位置16000和位置2000在PI后映射到相同的旋转角度——模型无法区分这两个位置的token；②位置15000和16000在PI后分别映射到1875和2000，间距从1000缩小到125——相对位置信息被压缩；③PI的本质是"牺牲位置分辨率换外推范围"——在32K范围内，模型只能区分约4096个不同的位置模式。
    关键要点：①PI通过线性缩放实现外推但损失位置精度；②位置分辨率从4K降低到4K/8=512个有效位置；③YaRN通过非线性缩放减少了这种损失。
