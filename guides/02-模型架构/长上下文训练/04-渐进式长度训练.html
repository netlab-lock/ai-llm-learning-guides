📈 04 · 渐进式长度训练

渐进式长度训练是长上下文训练的核心工程策略。直接用长上下文训练不仅效率低，还可能导致训练不稳定。

1. 为什么需要渐进式训练

1. 为什么需要渐进式训练：核心技术原理与实现细节

从系统设计角度看，1. 为什么需要渐进式训练的核心在于平衡计算效率与模型质量。具体来说，需要在三个维度上做出权衡：(1)计算复杂度——算法的理论时间复杂度决定了在大规模数据上的可行性，O(n²)的算法在序列长度超过8K时会成为瓶颈；(2)内存占用——模型参数、激活值和优化器状态的显存需求决定了所需的硬件配置，70B模型在FP16下需要约140GB显存；(3)通信开销——分布式训练中的集合通信(AllReduce/AllGather)可能占据30-50%的总训练时间。

工程实现中，1. 为什么需要渐进式训练通常采用分层架构：底层是硬件抽象层(CUDA/Triton kernels)，负责高效的矩阵运算和内存管理；中间层是算法核心层，实现主要的计算逻辑；上层是服务接口层(OpenAI兼容API)，提供标准化的调用方式。这种分层设计使得每一层都可以独立优化——例如底层可以针对不同GPU架构(Ampere/Hopper)编译不同的kernel，而上层接口保持不变。

从最新进展来看，2025-2026年1. 为什么需要渐进式训练领域出现了几个重要趋势：混合专家(MoE)架构成为主流、推理时计算(Test-Time Compute)的价值被充分认识、以及端到端的训练-推理协同优化取代了单独的推理优化。这些趋势正在重塑LLM系统的设计范式。

1. 为什么需要渐进式训练的训练流程与优化策略

1. 为什么需要渐进式训练的训练流程通常分为三个阶段：预训练(Pre-training)在大规模无标注语料上学习语言表示，微调(Fine-tuning)在任务特定数据上适配下游任务，对齐(Alignment)通过人类偏好反馈优化输出质量。每个阶段的数据配比、学习率调度和正则化策略都需要精心设计。

1.1 直接长上下文训练的问题

  • 训练不稳定：长序列的梯度方差大，容易出现 loss spike
  • 效率低下：短上下文阶段用长序列训练是浪费计算
  • 位置编码适应：模型需要时间适应新的位置范围
深入探讨： RoPE的位置编码为θ_m = m·10000^{-2i/d}，其中m是位置，i是维度。将上下文从L扩展到L'时，NTK-aware插值修改基频：θ'_m = m·(α·10000)^{-2i/d}，其中α=L'/L。这相当于在频率域进行缩放——低频维度（位置信息）被压缩，高频维度（局部信息）保持不变。YaRN进一步引入温度缩放因子t=0.1·ln(α)+1来校正注意力分布。

# DPO训练核心实现
import torch
from transformers import AutoModelForCausalLM

def compute_logprobs(model, input_ids, labels):
    """计算序列的对数概率"""
    logits = model(input_ids).logits
    log_probs = torch.nn.functional.log_softmax(logits, dim=-1)
    # 只计算response部分的log prob
    token_logps = log_probs.gather(2, labels.unsqueeze(-1)).squeeze(-1)
    return token_logps.sum(dim=-1)

def dpo_loss(policy, ref, chosen, rejected, beta=0.1):
    """DPO损失：无需奖励模型的偏好优化"""
    pi_chosen = compute_logprobs(policy, chosen['input_ids'], chosen['labels'])
    pi_rejected = compute_logprobs(policy, rejected['input_ids'], rejected['labels'])
    with torch.no_grad():
        ref_chosen = compute_logprobs(ref, chosen['input_ids'], chosen['labels'])
        ref_rejected = compute_logprobs(ref, rejected['input_ids'], rejected['labels'])

    # 核心公式：隐式奖励差的sigmoid
    log_ratio_chosen = pi_chosen - ref_chosen
    log_ratio_rejected = pi_rejected - ref_rejected
    loss = -torch.nn.functional.logsigmoid(beta * (log_ratio_chosen - log_ratio_rejected))
    return loss.mean()

import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

# Step 1: 加载策略模型和参考模型
policy = AutoModelForCausalLM.from_pretrained("model-name")
ref_model = AutoModelForCausalLM.from_pretrained("model-name")
tokenizer = AutoTokenizer.from_pretrained("model-name")

# Step 2: DPO损失函数
def dpo_loss(policy_chosen_logps, policy_rejected_logps,
             ref_chosen_logps, ref_rejected_logps, beta=0.1):
    """DPO核心损失: 基于偏好对的直接优化"""
    chosen_rewards = beta * (policy_chosen_logps - ref_chosen_logps)
    rejected_rewards = beta * (policy_rejected_logps - ref_rejected_logps)
    loss = -torch.nn.functional.logsigmoid(chosen_rewards - rejected_rewards)
    return loss.mean()

# Step 3: 训练循环

2. Llama 3.1 的渐进式训练的训练流程通常分为三个阶段：预训练(Pre-training)在大规模无标注语料上学习语言表示，微调(Fine-tuning)在任务特定数据上适配下游任务，对齐(Alignment)通过人类偏好反馈优化输出质量。每个阶段的数据配比、学习率调度和正则化策略都需要精心设计。

通俗类比：渐进式训练像"体育训练"——不会第一天跑马拉松。先5公里（8K），再10公里（16K），逐步增到42公里（128K）。每步让身体（模型）适应新强度。
常见误区：不要跳过中间阶段直接8K→128K。渐进式训练不仅更稳定，最终效果也更好——模型有时间"消化"每个长度的知识。
# 为什么需要渐进式训练 性能基准测试
import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
    elapsed = (time.perf_counter() - start) / iters
    print(f"Latency: {elapsed*1000:.2f}ms")
    return elapsed

✅ 参考答案

核心原理：Llama 3.1的渐进式策略：起始长度8K→16K→32K→64K→128K，每阶段约2B tokens，学习率从1e-5逐步降至5e-6。特点是"小步快跑"——每步长度翻倍，数据量适中。Qwen2.5的策略：起始长度4K→32K→128K，每阶段约10B tokens，学习率从2e-5降至1e-5。特点是"大步跳跃"——步长更大但数据量更多。效率对比：Llama 3.1的策略更"精细"——5个阶段确保每个长度区间都有充分训练，但总数据量约10B tokens。Qwen2.5的策略更"粗犷"——3个阶段但每阶段数据量大(10B)，总数据量约30B tokens。在最终性能上两者接近(128K上下文)，但Llama 3.1的数据效率更高(用更少数据达到相似效果)。
关键要点：①渐进式训练的核心是"逐步扩展序列长度"；②步长和数据量需要平衡——步长太大可能导致性能退化；③学习率需要随长度增加而降低。
实际案例：Llama 3.1在128K的NIAH测试上达到95%+的准确率。
关联知识：与RoPE的外推调整(PI/YaRN)配合使用——渐进式训练+位置编码调整是长上下文的标准方案。

深入探讨：灾难性遗忘是核心挑战。除降低学习率外，可使用"数据混合"——长上下文数据中混入一定比例短上下文数据，帮助保持短上下文能力。
