背景与动机：为什么需要了解芯片制造？

造？

原因很简单：集合通信的本质是数据在芯片之间流动。数据从GPU的显存出发，经过NVLink/PCIe/InfiniBand等链路，到达另一个GPU。要理解这个过程的瓶颈在哪里，你必须先理解数据在物理上是怎么被"搬运"的——而这取决于芯片的物理结构。

举个例子：当你听到"H100的NVLink带宽是900 GB/s"时，你需要知道：

  • 这个NVLink是焊在芯片上的物理线路，不是软件定义的
  • 它的带宽受限于封装中能放多少根线
  • H100之所以能做到900 GB/s，是因为它用了先进封装技术把更多线路塞进去了

所以，理解芯片封装是理解所有后续通信技术的物理基础。

1.2 名字由来：Die、Wafer、Chiplet从哪来？

Die（裸片）

"Die"这个词的来源有争议，但最被接受的说法是：它来自德语/荷兰语中的"Dädel"或"die"，意思是"模具"或"小块"。在半导体制造中，Die指的是从晶圆上切割下来的一小块正方形硅片——每一块Die就是一颗芯片的"本体"。

中文翻译"裸片"很形象：在被封装之前，它就是一片薄薄的、裸露的硅片，上面有精密的电路，但没有任何保护外壳。

Wafer（晶圆）

"Wafer"在英语中是"薄片"的意思（比如威化饼干也叫wafer）。在半导体中，它指的是从单晶硅锭上切下来的圆形薄片——直径通常是300mm（12英寸），厚度约0.7mm。所有的芯片都是在晶圆上"印"出来的。

Chiplet（小芯片）

"Chiplet"是"Chip"（芯片）加上后缀"-let"（小），字面意思就是"小芯片"。这个词大约在2015年开始流行，当时AMD和Intel开始探索把一个大芯片拆成多个小芯片的设计方案。

封装（Packaging）

"封装"这个词直译自英文"Packaging"——就是"打包"。把裸露的Die"打包"起来，加上保护外壳、引出引脚/焊球，让它能安装到电路板上。但现代先进封装远不止"打包"那么简单——它涉及到Die之间的高速互连、散热、供电等一系列复杂工程。

1.3 核心原理：从晶圆到芯片

Step 1：制造晶圆

硅（Si）从石英砂（SiO₂）中提炼，经过提纯得到多晶硅，再通过Czochralski法（直拉法）生长成单晶硅锭——一根直径300mm、长约1米的圆柱体。然后用金刚石锯切成厚度约0.7mm的薄片，就是晶圆。

Step 2：在晶圆上"印"电路

这个过程叫光刻（Photolithography），是芯片制造的核心。简化来说：

  1. 在晶圆表面涂一层光刻胶（Photoresist）——一种对光敏感的化学物质
  2. 用掩模版（Mask）遮住不需要曝光的部分，用紫外光照射
  3. 被光照到的光刻胶发生化学变化，可以被溶剂洗掉
  4. 留下的光刻胶形成"模板"，接下来可以蚀刻（刻出沟槽）或离子注入（改变硅的导电性）
  5. 重复几十到上百次，层层叠加，形成完整的晶体管和互连线

Step 3：切割成Die

一块300mm晶圆上可以制造几百到几千颗相同的Die（取决于Die的大小）。制造完成后，用金刚石锯或激光沿着切割道（Scribe Line）把晶圆切成一颗颗独立的Die。

Step 4：封装

裸片（Die）太脆弱了——薄到可以透光、一碰就碎。需要封装来：保护Die、散热、引出电气连接（引脚/焊球）。这一步是本章的重点，下面详细讲。

1.4 技术细节：封装技术的演进

传统封装：Wire Bonding（引线键合）

最古老的方式：把Die放在一个塑料/陶瓷外壳里，用金线/铜线把Die上的焊盘（Pad）和外壳上的引脚连起来。

缺点：引线又长又细，带宽有限（几百MHz），不适合高速通信。

Flip-Chip（倒装芯片）

把Die翻过来，电路面朝下，通过Die底部的焊球（Solder Bump）直接和基板连接。

优点：焊球阵列密度远高于引线，带宽大幅提升。现代GPU几乎都用Flip-Chip。

2.5D封装：硅中介层（Silicon Interposer）

当两个Die需要高密度互连时（比如GPU Die和HBM），在它们下面放一块"硅中介层"——上面有高密度的金属走线。

关键参数：

参数硅中介层有机基板差距
走线间距0.4-2 μm20-50 μm10-100倍
走线密度~1000 线/mm~20-50 线/mm20-50倍
带宽TB/s级GB/s级1000倍

这就是为什么HBM必须通过硅中介层和GPU连接——普通的有机基板根本放不下那么多线。

3D封装：TSV（硅穿孔）

2.5D是水平排列，3D是垂直堆叠。TSV（Through-Silicon Via，硅穿孔）是在硅片上钻出微小的通孔，填入金属（通常是铜），让上下层Die直接导通。

TSV的关键参数：

  • 直径：5-10 μm（比头发丝还细）
  • 间距：20-50 μm
  • 每颗HBM有数千个TSV
  • 通过TSV的带宽：可达TB/s级

TSMC CoWoS：业界主流的2.5D封装

CoWoS（Chip-on-Wafer-on-Substrate）是台积电（TSMC）的2.5D封装技术，也是目前AI GPU的主流封装方案。

名字的含义：

  • Chip-on-Wafer：把Chip（GPU Die、HBM Die）放到Wafer（硅中介层）上
  • on-Substrate：再把整个硅中介层放到有机基板上

具体来说：

  1. HBM通过3D封装堆叠 → HBM带宽达到3-5 TB/s → GPU可以快速读写模型参数和梯度

1.8.2 内部工作机制：从硅片到可通信芯片

前面我们讲了封装技术"是什么"和"有什么用"。现在让我们深入一层，看看这些技术在物理上到底是怎么工作的。

(a) 光刻：如何在硅片上"印刷"电路？

光刻是芯片制造的核心工艺，本质上是一种纳米级的照相术。

现代光刻使用EUV（极紫外光），波长只有13.5nm（普通紫外光是193nm）。为什么需要这么短的波长？因为光的分辨率和波长成正比——要"画"出3nm/5nm的电路线条，必须用比线条更细的光。

具体过程：

  1. 涂胶：在晶圆表面旋涂一层光刻胶（Photoresist），厚度约100nm。光刻胶分两种：正胶被光照射后变"软"（可溶解），负胶被光照射后变"硬"。
  2. 曝光：EUV光源发出极紫外光，经过一系列反射镜（EUV不能用透镜，因为所有材料都会吸收EUV光）聚焦到掩模版上。掩模版上有电路图案——透光部分对应要保留的结构。
  3. 显影：用显影液冲洗，溶解掉被曝光的部分（正胶情况），留下的光刻胶就形成了"模板"。
  4. 蚀刻/注入：以光刻胶为掩护，用等离子体刻蚀掉裸露的硅/金属层（形成沟槽），或者用离子注入改变硅的导电类型（形成晶体管的源极/漏极）。
  5. 去胶：去除剩余光刻胶，准备下一层图案的曝光。

一颗先进制程芯片需要重复这个过程60-80次（对应60-80层金属互连线），每一层都需要独立的掩模版。这就是为什么芯片制造成本极高——一套EUV掩模版的价格高达数千万美元。

💡 为什么EUV如此关键： 在EUV之前，业界用193nm的DUV光刻+多重曝光技术（把一次曝光拆成多次）来实现更小的线宽。但多重曝光成本极高、良率下降。EUV的出现让单次曝光就能实现7nm/5nm/3nm节点，是摩尔定律延续的关键支撑。ASML的EUV光刻机单台售价超过3亿美元，是人类制造过的最复杂的机器之一。

(b) TSV：如何在硅片上"钻孔"实现垂直互连？

TSV（Through-Silicon Via，硅穿孔）是3D封装的核心技术。它的本质是：在硅片上钻出贯穿的微小通孔，填入金属导体，让上下层芯片直接电学连通。

TSV的制造过程：

  1. 深反应离子蚀刻（DRIE）：用等离子体在硅片上刻蚀出直径5-10μm、深度50-100μm的通孔。这不是"钻"出来的——而是用化学反应"吃"出来的。DRIE工艺被称为"Bosch工艺"，通过交替进行蚀刻和钝化，在孔壁上形成特征性的"扇贝纹"。
  2. 绝缘层沉积：在孔壁上沉积一层氧化硅（SiO₂），厚度约1μm。这层绝缘层防止金属导体和硅基底短路。
  3. 阻挡层+种子层：先沉积一层TaN/TiN（阻挡层，防止铜扩散到硅中），再沉积一层薄铜（种子层，为后续电镀提供导电基底）。
  4. 铜电镀填充：把硅片放入电镀液中，通过电化学反应在通孔中填充铜。这一步是最有挑战的——要保证直径只有5μm的通孔完全填满铜，不能有空洞（Void），否则会导致电阻增大甚至断路。
  5. 化学机械抛光（CMP）：把表面多余的铜磨平，使晶圆表面恢复平坦。

完成后，上层Die和下层Die就通过这根直径5μm的铜柱实现了垂直电气连通。HBM的8-12层DRAM层就是通过数千个TSV紧密排列实现的——每个TSV传输1bit或几bit的信号，数千个TSV并行工作，总带宽达到TB/s级。

⚠️ TSV的良率挑战： TSV是贯穿硅片的金属柱，制造过程中可能出现空洞、裂纹、与硅基底短路等问题。一个TSV失效就可能导致整颗HBM报废。因此TSV的良率是3D封装的核心挑战之一——这也是HBM价格居高不下的重要原因之一。

(c) 硅中介层：CoWoS中的"无源PCB"

硅中介层（Silicon Interposer）是CoWoS封装的核心组件。它的本质其实很简单：它就是一块用硅制造的超精密"印刷电路板"（PCB），只不过上面的走线比普通PCB精细100倍以上。

关键特性：

  • 它是"无源"的：硅中介层上没有晶体管、没有逻辑电路，只有金属走线和通孔（Via）。它唯一的作用就是在GPU Die和HBM之间、或者多个Chiplet之间搬运电信号。
  • 走线间距0.4-2μm：相比有机基板的20-50μm，精细了10-100倍。这意味着同样的面积可以放下10-100倍的互连线。
  • 制造工艺：和制造芯片一样，硅中介层也是在晶圆上通过光刻、蚀刻、金属沉积等工艺制造出来的。它不需要先进制程（通常用65nm或更成熟的节点），但需要高精度的走线工艺。
  • 面积越来越大：从A100的~830mm²到B200的~1700mm²，再到B300 Ultra的~2500mm²+——已经超过了很多芯片Die本身的面积。这也是CoWoS产能紧张的根本原因：硅中介层太大，每片晶圆只能做几个。

打个比方：如果GPU Die是一台高性能CPU，HBM是一堆高速内存条，那么硅中介层就是把它们连接在一起的"超微型主板"——只不过这个"主板"是用硅和铜制造的，上面的"电线"只有头发丝的百分之一粗细。

(d) HBM如何通过微凸块连接到GPU Die？

HBM和GPU Die之间的连接，靠的是硅中介层上密集排列的微凸块（Micro-bump）。

具体连接方式：

  1. 硅中介层顶部：有两组焊盘阵列——一组对应GPU Die，一组对应HBM Stack。
  2. 微凸块（μbump）：在焊盘上制作直径约25-40μm、间距约40-55μm的微小焊球。这些焊球的尺寸大约是普通BGA焊球的1/10。
  3. 回流焊接：把GPU Die和HBM精确对准放置到硅中介层上（对准精度需要达到±2μm），然后加热到焊球熔化（约200-250°C），冷却后形成固态焊接连接。
  4. 硅中介层内部：微凸块下方的金属走线在中介层内部"路由"，把GPU的信号引到HBM对应的位置。一颗HBM Stack可能需要数千根互连线（数据线+地址线+控制线+电源），全部通过硅中介层内部的走线实现。
  5. 底部连接：硅中介层通过更大的C4焊球（直径~100μm，间距~150-200μm）连接到有机基板，再通过BGA焊球连接到PCB主板。
💡 物理直觉： 整个CoWoS封装的核心思想就是：用硅中介层的高密度走线解决GPU和HBM之间的"最后一毫米"互连问题。GPU Die内部的走线间距是nm级（几十nm），PCB上的走线间距是mm级——中间有一个巨大的"带宽鸿沟"。硅中介层的μm级走线正好填补了这个鸿沟。这就是为什么没有硅中介层（或类似技术），HBM根本无法发挥其理论带宽。

1.8.3 向前串联：封装如何决定通信天花板

这一章我们学习了芯片封装的基础知识。你可能会问：这些和后面的"集合通信"有什么关系？

答案是：封装技术是一条物理约束链的起点，它决定了整个通信系统性能的上限。让我们把这条链拉出来看看。

约束链 ①：HBM堆叠数 → 显存带宽 → NVLink数据供给

CoWoS中介层的面积决定了能放多少个HBM Stack：

  • A100（~830mm²中介层）→ 6个HBM2e → 总显存带宽 2 TB/s
  • H100（~900mm²中介层）→ 6个HBM3 → 总显存带宽 3.35 TB/s
  • B200（~1700mm²中介层）→ 8个HBM3e → 总显存带宽 8 TB/s

为什么显存带宽重要？因为在AllReduce等集合通信中，GPU需要先从HBM读出数据，再通过NVLink发送给其他GPU。如果HBM读取速度跟不上NVLink的发送速度，NVLink就会"饿死"——这就是模块04（NVLink）中会详细讨论的"计算-通信重叠"问题的物理根源。

约束链 ②：CoWoS中介层面积 → 可容纳的Chiplet数 → GPU计算密度

更大的中介层面积不仅意味着更多的HBM，还意味着可以集成更多的Compute Die（计算核心）。Blackwell B200之所以把两个Compute Die放在一块中介层上，是因为TSMC的CoWoS-L技术提供了足够大的面积（~1700mm²+）。

单芯片的计算密度直接决定了模块02（GPU内部架构）中讨论的SM数量、Tensor Core数量，进而决定了单卡算力。

约束链 ③：TSV密度 → Die间带宽 → NVLink-C2C

Blackwell B200的两个Compute Die之间通过NVLink-C2C直连——这不是通过外部NVLink Switch，而是在封装内部通过硅中介层实现的Die到Die互连。

NVLink-C2C的带宽（1.8 TB/s双向）之所以能达到这么高，是因为：

  • 两个Die之间的走线在硅中介层上，间距仅0.4-2μm
  • 可以放下数万根高速差分线
  • 信号传输距离极短（mm级），延迟极低

这正是模块04（NVLink与NVSwitch）中会深入讲解的NVLink-C2C技术的物理基础。

约束链 ④：3D封装 → HBM直接堆叠在GPU上方 → 消除PCIe跳转

在传统架构中，GPU通过PCIe访问"外部"显存（如早期的GDDR方案）。而在HBM方案中，HBM通过硅中介层直接和GPU连接，数据路径是：

GPU Die → μbump（几μm）→ 硅中介层 → μbump → HBM

整个路径不到1毫米，延迟是纳秒级。相比之下，通过PCIe访问外部内存的路径是厘米级，延迟是微秒级——差了1000倍。

这个物理特性使得模块07（GPUDirect RDMA）成为可能：GPU可以直接把HBM中的数据通过NVLink→NVSwitch→InfiniBand发送到远端GPU的HBM中，整个过程完全绕过CPU和系统内存。如果HBM不是通过先进封装直接连接到GPU的，这种零拷贝的高效通信是不可能实现的。

🔬 深入：为什么说"封装即命运"？

在AI集群中，集合通信（如AllReduce）的性能瓶颈往往不在算法本身，而在于物理带宽的层级结构：

  • 片上带宽（GPU Die内部）：~数十 TB/s — 取决于芯片内部走线
  • 封装带宽（GPU↔HBM）：3-12 TB/s — 取决于CoWoS+TSV ← 本章
  • 节点内带宽（GPU↔GPU）：900-1800 GB/s — 取决于NVLink封装走线
  • 节点间带宽（服务器↔服务器）：400-800 Gb/s — 取决于网卡和交换机

每一层大约比上一层低一个数量级。这个"漏斗"形状的带宽层级，是由物理封装从内到外逐级递减决定的。算法工程师的工作，就是在这些物理约束下找到最优的通信策略——这就是后续模块08-10的主题。

import time
import torch

def profile_inference(model, input_data, warmup=10, iterations=100):
    """推理性能分析：测量延迟、吞吐和显存"""
    # Warmup
    for _ in range(warmup):
        with torch.no_grad():
            model(input_data)
    torch.cuda.synchronize()
    
    # Measure latency
    latencies = []
    for _ in range(iterations):
        start = time.perf_counter()
        with torch.no_grad():
            output = model(input_data)
        torch.cuda.synchronize()
        latencies.append(time.perf_counter() - start)
    
    # Statistics
    avg_ms = sum(latencies) / len(latencies) * 1000
    p50 = sorted(latencies)[len(latencies)//2] * 1000
    p99 = sorted(latencies)[int(len(latencies)*0.99)] * 1000
    mem_mb = torch.cuda.max_memory_allocated() / 1024 / 1024
    
    print(f"Latency: avg={avg_ms:.2f}ms, p50={p50:.2f}ms, p99={p99:.2f}ms")
    print(f"Peak Memory: {mem_mb:.0f}MB")
    print(f"Throughput: {1000/avg_ms:.0f} req/s")
    return {"avg_ms": avg_ms, "p50": p50, "p99": p99, "mem_mb": mem_mb}
