一、背景与动机

一、背景与动机

on 的计算复杂度是 O(n²)，内存复杂度也是 O(n²)——需要存储完整的 n×n 注意力矩阵。对于 4K 上下文，这个矩阵是 4096×4096×2 bytes = 32MB；对于 128K 上下文，是 32GB！

FlashAttention 的核心思想：不存储完整的注意力矩阵，而是通过 tiling(分块)和 online softmax 在 SRAM(Shared Memory) 中完成计算，只把最终结果写回 HBM。这既节省了内存，又因为减少了 HBM 访问而提高了速度。

    """性能基准测试"""

    torch.cuda.synchronize()
    start = time.perf_counter()

2.2 为什么标准 Attention 是 Memory-bound

Roofline 分析: 标准 Attention 算术强度 = FLOPs / Bytes QK^T: n²d FLOPs / (2nd + n²) Bytes 当 n >> d 时, 算术强度 ≈ n·d / (2d + n) ≈ d H100: 峰值算力 989 TFLOPS, 带宽 3.35 TB/s 拐点 = 989/3.35 ≈ 295 FLOPS/Byte d=128 时, 算术强度 ≈ 128 < 295 → Memory-bound! 受限于HBM带宽 → 优化方向: 减少HBM访问(FlashAttention的核心)

三、FlashAttention 核心原理

3.1 Tiling (分块计算)

FlashAttention 的分块计算: 将 Q 分成 Tr 个行块(每块 Br 行) 将 K/V 分成 Tc 个列块(每块 Bc 列) Q: [Q₁] K: [K₁ K₂ K₃ ...] V: [V₁ V₂ V₃ ...] [Q₂] [Q₃] [...] 对每个 Qᵢ: 初始化: Oᵢ = 0, mᵢ = -∞, lᵢ = 0 for each Kⱼ, Vⱼ: ① 从HBM加载 Qᵢ, Kⱼ, Vⱼ 到SRAM ② 计算局部注意力: Sᵢⱼ = Qᵢ × Kⱼ^T [Br×Bc] ③ 用 online softmax 更新统计量 ④ 累加: Oᵢ += softmax(Sᵢⱼ) × Vⱼ ⑤ 将最终 Oᵢ 写回HBM 关键: 每次只在SRAM中处理一个小块(Br×Bc) SRAM容量: ~200KB (够放 Br×Bc×2B) 不需要存储完整的 n×n 矩阵!

五、反向传播的 Recomputation

FlashAttention 的反向传播: 标准Attention反向: 需要存储 S[n×n] 和 P[n×n] 用于梯度计算 → O(n²) 内存 FlashAttention反向: 不存储 S 和 P 反向传播时重新计算(recomputation) 前向: 计算 O, 保存 m, l (统计量) 反向: 用 m, l 重新计算 S 和 P 然后计算梯度 额外计算: ~1次前向的FLOPs 节省内存: O(n²) → O(n) → 用计算换内存, 总体更快(因为减少了HBM访问)

七、Ring Attention (多GPU长序列)

七、Ring Attention (多GPU长序列)

Ring Attention: 多GPU分布式Attention 问题: 单GPU内存有限, 超长序列(1M+)装不下 方案: 将序列分到多个GPU上 GPU 0: [Q₁, K₁, V₁] GPU 1: [Q₂, K₂, V₂] GPU 2: [Q₃, K₃, V₃] GPU 3: [Q₄, K₄, V₄] 环形通信: Step 1: 每个GPU计算本地 QᵢKᵢᵀ Step 2: K,V 环形传递(GPU0→1→2→3→0) Step 3: 每个GPU计算与下一个K,V块的注意力 Step 4: 重复直到所有K,V块都被处理 计算和通信重叠 → 隐藏通信延迟 效果: 序列长度可以线性扩展到GPU数量

import time
import torch

def profile_inference(model, input_data, warmup=10, iterations=100):
    """推理性能分析：测量延迟、吞吐和显存"""
    # Warmup
    for _ in range(warmup):
        with torch.no_grad():
            model(input_data)
    torch.cuda.synchronize()
    
    # Measure latency
    latencies = []
    for _ in range(iterations):
        start = time.perf_counter()
        with torch.no_grad():
            output = model(input_data)
        torch.cuda.synchronize()
        latencies.append(time.perf_counter() - start)
    
    # Statistics
    avg_ms = sum(latencies) / len(latencies) * 1000
    p50 = sorted(latencies)[len(latencies)//2] * 1000
    p99 = sorted(latencies)[int(len(latencies)*0.99)] * 1000
    mem_mb = torch.cuda.max_memory_allocated() / 1024 / 1024
    
    print(f"Latency: avg={avg_ms:.2f}ms, p50={p50:.2f}ms, p99={p99:.2f}ms")
    print(f"Peak Memory: {mem_mb:.0f}MB")
    print(f"Throughput: {1000/avg_ms:.0f} req/s")
    return {"avg_ms": avg_ms, "p50": p50, "p99": p99, "mem_mb": mem_mb}

    ▶显示答案
    ✅ 参考答案与详解
    1. 核心原理: FlashAttention通过Online Softmax实现精确(数学等价)的注意力计算。标准softmax需要两次遍历(先找max再算exp和)，而Online Softmax通过增量更新实现单次遍历：维护当前最大值m和归一化因子l，每处理一个新块时计算scale=exp(m_old - m_new)来缩放之前的累加结果。数学证明：设x=[x₁...xₖ, xₖ₊₁...xₖ₊ⱼ]，前k个元素的softmax用mₖ计算，加入新块后全局最大值m_new=max(mₖ, max(new_block))。旧分母lₖ需乘以exp(mₖ-m_new)重新缩放，新分母l_new=lₖ×exp(mₖ-m_new)+Σexp(xⱼ-m_new)。这个缩放操作保证了所有exp项都在同一max基准下，结果与标准softmax完全一致
      关键要点: ①增量更新max和归一化因子 ②缩放因子exp(m_old-m_new) ③数学等价于标准softmax ④单次遍历
      实际案例: FA-3处理128K上下文，结果bit-exact等价于标准attention
      关联知识: 与tiling策略、IO感知算法和Forward-Backward数值稳定性相关
    2. 核心原理: FA-3在H100上处理seq=32K, d=128, Br=Bc=128时的SRAM需求计算：SRAM中需同时存放①Q tile: Br×d×2B = 128×128×2 = 32KB ②K tile: Bc×d×2B = 128×128×2 = 32KB ③V tile: Bc×d×2B = 128×128×2 = 32KB ④S矩阵(注意力分数): Br×Bc×2B = 128×128×2 = 32KB ⑤P矩阵(softmax后): Br×Bc×2B = 128×128×2 = 32KB ⑥O累加器: Br×d×4B = 128×128×4 = 64KB(FP32)。总计约224KB。H100每SM有228KB Shared Memory，刚好可以放下。但实践中还需为TMA descriptor、pipeline buffer等预留空间，因此实际Br和Bc可能需略小(如96或100)
      关键要点: ①Q/K/V各32KB ②S/P各32KB ③O累加器64KB(FP32) ④总计~224KB ≈ 228KB上限
      实际案例: FA-3默认Br=Bc=128，SMem利用率~98%
      关联知识: 与SRAM容量约束、tiling参数选择和Occupancy相关
    3. 核心原理: FA-3的Warp Specialization将不同Warp分配专职任务：Warp 0-1(Producer)专責TMA从HBM加载Q/K/V tile到Shared Memory；Warp 2-5(Consumer)专責WGMMA执行QK^T和PV矩阵乘法；Warp 6-7(Store Consumer)专責将结果写回HBM。三个阶段通过Shared Memory和Barrier(名为async barrier)实现流水线重叠：Producer加载tile[i+1]的同时，Consumer计算tile[i]。这比FA-2(所有Warp都做加载+计算+存储)高效的原因：①Producer不需要寄存器保存MMA操作数→更少寄存器→更高Occupancy ②各阶段完美重叠→隐藏HBM延迟 ③TMA/WGMMA异步特性充分发掘。效率提升：FA-3在H100上达到75% TC利用率(FA-2仅35%)
      关键要点: ①Producer/Consumer角色分离 ②TMA加载/WGMMA计算/Store三阶段流水线 ③async barrier同步 ④75% vs 35%利用率提升
      实际案例: FA-3 H100 FP16达740 TFLOPS(标准Attention仅~100 TFLOPS)
      关联知识: 与Hopper TMA、WGMMA和CUTLASS Warp Specialization相关
    4. 核心原理: Ring Attention环形通信重叠机制：将序列分片到N个GPU，每个GPU持有Qᵢ,Kᵢ,Vᵢ。计算分N步：Step t中每个GPU计算本地Qᵢ与Kₜᵀ的注意力(本地计算)，同时将K,V环形传递给下一个GPU(NIC通信)。通信和计算重叠的关键：本地SM计算QK^T时，DMA/NIC独立搬运K,V数据(NVLink到下一GPU)。如果通信延迟>计算时间，会出现计算等待通信的bubble——SM完成当前块计算后，下一块的K,V还未到达，导致GPU空闲。解决方案：①增大K,V块(更多计算/通信比) ②使用更快互连(NVLink vs PCIe) ③双缓冲预取下一个块
      关键要点: ①环形传递K,V ②本地计算与通信重叠 ③通信>计算时产生bubble ④增大块/更快互连缓解
      实际案例: Ring Attention在8×H100上扩展1M上下文，NVLink隐藏通信延迟
      关联知识: 与NVLink通信、FlashAttention变体和超长序列训练相关
    ⚠️ 常见误区：FlashAttention减少了FLOPs：这是最常见的错误理解！FlashAttention的FLOPs和标准Attention完全一样——都是O(n²d)。它减少的是HBM访问量和内存占用。速度提升来自减少了从HBM读写S和P矩阵的开销。标准Attention需要把n×n的S和P矩阵写回HBM再读出来做softmax，FlashAttention在SRAM中完成这一切。
    ⚠️ 常见误区：FlashAttention适用于所有Attention变体：FlashAttention的核心假设是"Attention矩阵太大放不进SRAM"。对于GQA(Grouped Query Attention)，KV头数比Q少，矩阵更小，FA仍然有效。但对于某些稀疏Attention(如Longformer的local attention)，可能有更高效的实现方式。
    🔬 深入探讨：Online Softmax的数学证明

    标准softmax: softmax(x_i) = exp(x_i - max(x)) / Σexp(x_j - max(x))
    问题是max(x)需要看到所有元素。Online Softmax通过增量更新解决：
    假设已处理前k个元素，得到m_k和l_k。处理新块时：
    1. 计算新块最大值: m_new_block = max(new_block)
    2. 更新全局最大值: m_new = max(m_k, m_new_block)
    3. 缩放旧分母: l_k' = l_k × exp(m_k - m_new)
    4. 累加新贡献: l_new = l_k' + Σexp(x_j - m_new)
    5. 缩放旧输出: O_old' = O_old × (l_k' / l_new)
    关键: exp(m_k - m_new)保证数学等价——把之前的exp值重新缩放到新的max基准下。
    🔬 深入探讨：如何选择Br和Bc？

    Br(行tile)和Bc(列tile)的选择直接影响FlashAttention性能：
    约束：Br×d×2B + Bc×d×2B + Br×Bc×2B(×2 for S和P) ≤ Shared Memory
    H100为例(228KB SMem, d=128)：Br=Bc=128需要~128KB → OK；Br=Bc=256需要~512KB → 超了!
    增大Br提高Q复用率；增大Bc提高K/V复用率。但两者受SMem约束。典型配置Br=Bc=128。
    Online Softmax 状态机: 初始化: m = -∞, l = 0, O = 0 Tile 1 (K₁, V₁): S₁ = Q × K₁ᵀ [Br × Bc] m₁ = max(S₁) l₁ = Σexp(S₁-m₁) O₁ = softmax(S₁) × V₁ m = m₁, l = l₁, O = O₁ Tile 2 (K₂, V₂): S₂ = Q × K₂ᵀ m_new = max(m, max(S₂)) ← 更新全局最大值 scale = exp(m - m_new) ← 缩放因子 l = l × scale + Σexp(S₂-m_new) ← 缩放旧分母+累加新 O = O × scale + softmax(S₂)×V₂ ← 缩放旧输出+累加新 m = m_new Tile 3: (同上) 最终: O = O / l ← 只在最后归一化一次! 结果与标准Attention数学等价 ✓
