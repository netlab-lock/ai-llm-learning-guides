2.1 上下文窗口管理

# 短期记忆: 上下文窗口管理
class ContextWindow:
    def __init__(self, max_tokens=8192):
        self.max_tokens = max_tokens
        self.messages = []

    def add(self, role, content):
        self.messages.append({"role": role, "content": content})
        self._trim()

    def _trim(self):
        """当超出窗口时，移除最旧的消息"""
        total = sum(count_tokens(m["content"]) for m in self.messages)
        while total > self.max_tokens and len(self.messages) > 2:
            removed = self.messages.pop(0)  # 移除最旧的
            total -= count_tokens(removed["content"])

    def get_messages(self):
        return self.messages

摘要:"""

    summary = model.generate(prompt)

    ]

3.2 记忆检索策略

策略原理适用场景实现
时间衰减越近的记忆权重越高对话历史score *= exp(-λ·Δt)
相关性语义相似度匹配知识检索cosine_similarity
重要性LLM 评估记忆重要性关键信息LLM 打分
混合加权组合以上三种通用场景α·相关 + β·时间 + γ·重要
# 混合检索策略
def hybrid_retrieve(query, memory, alpha=0.5, beta=0.3, gamma=0.2):
    """混合检索: 相关性 + 时间衰减 + 重要性"""
    candidates = memory.get_all()

    scores = []
    for item in candidates:
        # 相关性分数
        relevance = cosine_similarity(embed(query), embed(item.content))
        # 时间衰减分数
        age_hours = (datetime.now() - item.timestamp).total_seconds() / 3600
        recency = math.exp(-0.01 * age_hours)
        # 重要性分数 (预计算)
        importance = item.importance_score

        score = alpha * relevance + beta * recency + gamma * importance
        scores.append((item, score))

    # 返回 Top-K
    scores.sort(key=lambda x: x[1], reverse=True)
    return [item for item, score in scores[:5]]

6. 记忆压缩技术

通俗类比：Agent 的记忆系统就像人脑的三层记忆：短期记忆像你的"工作台"——只放当前正在处理的东西，满了就清理；长期记忆像你的"笔记本"——记录重要的事情，需要时翻阅；情景记忆像你的"经验库"——记住上次踩过的坑，下次遇到类似情况就知道怎么处理。
常见误区：1) 以为把所有历史对话都塞进上下文窗口就是"记忆"——token 有上限，成本也会飙升；2) 长期记忆只用向量检索，忽略了时间衰减和重要性加权，导致检索到过时或无关的记忆；3) 忘记设计"遗忘"机制，Agent 记住太多噪声信息反而干扰决策。
深入探讨：Letta/MemGPT 的核心创新是让 Agent 自己决定"记什么、忘什么"。传统 RAG 是被动检索，而自编辑记忆是主动管理——Agent 可以调用 core_memory_append 把重要信息写入核心记忆，用 core_memory_replace 更新过时信息。这模拟了人类的"元认知"能力：不仅知道信息，还知道自己知道什么、不知道什么。在实际应用中，这种自编辑能力对长期对话和个性化 Agent 至关重要。

7. 记忆系统的工作机制与数学原理

7.1 向量相似度的数学推导

记忆检索的核心是向量相似度计算。当用户提出查询时，系统需要从数万条记忆中找到最相关的几条。这个过程的数学原理基于余弦相似度（Cosine Similarity）。

给定查询向量 q 和记忆向量 m，余弦相似度定义为：sim(q, m) = (q · m) / (||q|| × ||m||)。其中 q · m 是向量点积，||q|| 和 ||m|| 是向量的 L2 范数。这个公式的几何含义是两个向量夹角的余弦值——值越接近 1，两个向量越相似；值接近 0，两个向量正交（无关）。在高维空间（如 1536 维的 OpenAI Embedding），余弦相似度比欧氏距离更能捕捉语义相似性，因为它对向量的绝对大小不敏感，只关注方向。

深入探讨： 向量数据库的高效检索依赖于近似最近邻（ANN）算法。暴力搜索的时间复杂度是 O(n×d)，其中 n 是记忆数量，d 是向量维度，对于百万级记忆来说太慢了。主流的 ANN 算法包括：(1) HNSW（Hierarchical Navigable Small World）——构建多层图结构，从顶层粗粒度搜索逐层细化，时间复杂度 O(log n)，是 ChromaDB 和 Qdrant 的默认算法；(2) IVF（Inverted File Index）——先将向量空间聚类，查询时只在最近的几个聚类中搜索；(3) PQ（Product Quantization）——将高维向量压缩为低维编码，减少内存占用。在实际应用中，HNSW 在召回率和速度之间取得了最佳平衡，通常能达到 95%+ 的召回率。

7.2 记忆权重的动态调整

记忆系统的核心挑战之一是如何为每条记忆动态分配权重。权重的计算需要综合考虑多个因素：相关性（与当前查询的语义匹配度）、时效性（记忆的创建时间）、重要性（记忆被访问的频率和关键程度）、以及上下文相关性（与当前对话主题的关联度）。这个多因素加权的数学模型可以表示为：score(m, q, t) = α · relevance(m, q) + β · recency(m, t) + γ · importance(m) + δ · context(m, current_topic)。其中 α, β, γ, δ 是可调节的超参数，需要根据具体应用场景进行调优。

权重因子计算方式典型值适用场景
相关性 α余弦相似度0.4-0.6知识检索
时效性 β指数衰减 exp(-λ·Δt)0.2-0.3对话历史
重要性 γ访问频率 + LLM评分0.1-0.2关键信息
上下文 δ主题模型匹配0.1-0.2多主题对话
实践建议： 在实际部署中，建议使用"两阶段检索"策略：第一阶段用向量相似度快速召回 Top-20 候选记忆（粗筛），第二阶段用 LLM 对候选记忆进行精排（精筛），选择最相关的 Top-5 注入上下文。这种两阶段策略比纯向量检索的准确率高 20-30%，虽然增加了少量延迟，但显著提高了记忆质量。对于延迟敏感的场景，可以用交叉编码器（Cross-Encoder）替代 LLM 做精排，速度更快且准确率接近。

8. 记忆系统的常见问题与最佳实践

记忆污染警告： 记忆系统最大的风险是"记忆污染"——当Agent从外部数据源（网页、邮件、用户上传文件）获取信息时，恶意或错误的信息可能被存入长期记忆，并在后续对话中反复被检索使用。例如，如果Agent从一个不可靠的网页中学习到"Python的sorted函数是不稳定的"（实际是稳定的），这条错误信息会持续影响Agent的后续回答。防御策略包括：(1) 为每条记忆标注来源和可信度分数；(2) 对外部来源的记忆设置过期时间；(3) 定期用高可信度知识库校验记忆内容的正确性。
隐私风险： 长期记忆系统存储了大量用户交互数据，这带来了严重的隐私风险。用户的个人信息、对话历史、偏好习惯都被持久化存储，如果被泄露或滥用后果严重。建议实施以下隐私保护措施：(1) 对敏感信息进行脱敏处理后再存储；(2) 实现用户级别的记忆隔离，不同用户的数据严格分开；(3) 提供记忆查看和删除功能，让用户控制自己的数据；(4) 遵循数据最小化原则，只存储必要的信息。
架构选择： 对于不同的应用场景，记忆系统的架构选择至关重要。个人助手场景推荐使用 Letta/MemGPT 的自编辑记忆模式——Agent主动管理记忆，用户体验最好；企业知识库场景推荐使用 RAG + 向量数据库——外部文档作为记忆源，维护成本低；客服机器人场景推荐使用混合模式——短期记忆（对话历史）+ RAG（产品文档）+ 情景记忆（历史工单），覆盖面最广。每种架构都有其适用范围，选择时需要综合考虑数据量、更新频率、延迟要求和隐私约束。
深入探讨： 记忆系统的"遗忘"机制同样值得深入研究。人类大脑的遗忘不是随机的，而是遵循艾宾浩斯遗忘曲线——新记忆的衰减速度随时间递减。受此启发，研究者提出了"可控遗忘"策略：为每条记忆维护一个"记忆强度"参数，初始强度为 1.0，每次未被检索时强度按公式 decay(t) = e^(-t/S) 衰减（t 是时间，S 是稳定性参数）。当记忆被成功检索并使用时，强度会提升（类似人类的"复习"效应）。这种机制确保了常用记忆持久保留，而很少用到的记忆自然消退，模拟了人类大脑的记忆巩固过程。在实现中，S 参数可以由 LLM 根据记忆的重要性自动设定——重要事件的 S 值大（不容易遗忘），琐碎细节的 S 值小（快速遗忘）。
