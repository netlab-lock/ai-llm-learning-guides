1. DDPM：去噪扩散概率模型

DDPM（Ho et al., 2020）是现代扩散模型的基石。其核心思想是：先逐步向数据添加高斯噪声（前向过程），再学习逐步去除噪声（反向过程）。这一看似简单的思想蕴含着深刻的概率论原理——通过定义一个易于采样的前向马尔可夫链，将复杂的数据分布逐步转化为简单的高斯分布，然后学习逆过程来实现从噪声到数据的生成。

前向过程

定义噪声调度 β₁, β₂, ..., β_T，前向过程为马尔可夫链：

q(x_t | x_{t-1}) = N(x_t; √(1-β_t) · x_{t-1}, β_t · I)

利用 α_t = 1 - β_t, ᾱ_t = ∏α_i，可一步到位：
q(x_t | x_0) = N(x_t; √ᾱ_t · x_0, (1-ᾱ_t) · I)

直觉理解：
  ᾱ_t 从1逐渐减小到0
  当t=T时，ᾱ_T ≈ 0，x_T ≈ N(0, I)（纯噪声）
  当t=0时，ᾱ_0 = 1，x_0 = 原始数据

前向过程的数学本质是对原始数据逐步施加方差为β_t的高斯噪声。由于高斯分布的可加性，我们可以直接从x_0计算任意时刻t的x_t，而不需要逐步迭代。这个性质被称为"重参数化技巧"（reparameterization trick），是训练效率的关键——训练时只需要采样一个时间步t，计算x_t，然后让网络预测噪声。

反向过程

学习神经网络 ε_θ 预测噪声，从纯噪声逐步恢复数据：

p_θ(x_{t-1} | x_t) = N(x_{t-1}; μ_θ(x_t, t), σ_t² · I)

其中 μ_θ(x_t, t) = (1/√α_t)(x_t - β_t/√(1-ᾱ_t) · ε_θ(x_t, t))

训练目标：L = E[||ε - ε_θ(x_t, t)||²]

简化训练目标（Ho et al.）：
  L_simple = E_{t,x_0,ε}[||ε - ε_θ(x_t, t)||²]
  其中 t ~ Uniform(1, T)
  x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε
💡 直觉理解

前向过程 = 给照片逐渐撒沙子直到看不清；反向过程 = 学会从沙子中恢复照片。网络学的不是"生成"，而是"去噪"。

2. 潜在扩散模型（LDM / Stable Diffusion）

Rombach et al.（2022）提出在潜在空间而非像素空间进行扩散，大幅降低计算成本：

像素空间 x (512×512×3) → VAE编码器 → 潜在空间 z (64×64×4)
                                 ↓
                        在潜在空间做扩散
                                 ↓
                VAE解码器 → 像素空间 x (512×512×3)

关键优势：计算量降低约48倍（64×64 vs 512×512），同时保留高频细节。

VAE架构细节：
  编码器：输入512×512×3 → 下采样8倍 → 输出64×64×4
  解码器：输入64×64×4 → 上采样8倍 → 输出512×512×3
  KL散度正则化：确保潜在空间接近标准正态分布

LDM的核心洞察是：图像中的高频细节（如纹理、噪声）对于扩散过程来说是冗余信息，真正需要建模的是图像的语义结构和全局布局。通过VAE将图像压缩到低维潜在空间，不仅大幅降低了计算量，还去除了冗余的高频信息，使得扩散模型可以专注于学习语义层面的生成规律。这也是为什么Stable Diffusion能在消费级GPU上运行的根本原因。

3. Flow Matching

Lipman et al.（2023）提出Flow Matching，用更简洁的框架替代DDPM。核心思想是学习从噪声到数据的速度场：

ODE轨迹：dx_t = v_θ(x_t, t) · dt
其中 x_t = (1-t)·x_0 + t·ε  （线性插值路径）
训练目标：L = E[||v_θ(x_t, t) - (ε - x_0)||²]

Flow Matching vs DDPM 数学对比:
  DDPM: 学习预测噪声 ε_θ(x_t, t) ≈ ε
        采样: SDE, 需要随机项
        步数: 20-1000步
        
  FM:   学习预测速度 v_θ(x_t, t) ≈ ε - x_0
        采样: ODE, 确定性轨迹
        步数: 4-28步
        
  关键区别: ODE路径更直，离散化误差更小
🔑 Flow Matching vs DDPM

Flow Matching使用ODE而非SDE，路径更直，采样步数更少（4-8步 vs 20-50步），且训练更稳定。2025年主流模型（SD3、FLUX）均采用此范式。

Flow Matching的数学优雅性在于它将生成过程建模为一个连续的常微分方程（ODE），而不是DDPM中的随机微分方程（SDE）。ODE的确定性意味着给定相同的初始噪声，总是生成相同的图像，这在实际应用中非常重要——用户可以通过改变随机种子来获得不同的结果，但同样的种子总是产生相同的输出。此外，ODE轨迹的直线性使得低阶ODE求解器（如欧拉方法）就能获得很好的近似，从而实现少步数高质量采样。

4. DiT：Diffusion Transformer

Peebles & Xie（2023）用Transformer替换传统UNet骨干网络：

输入：噪声patch序列 + 时间步 + 条件（文本）
  → Patch Embedding（Conv2d 2×2, stride=2）
  → Transformer Blocks × N（自注意力 + FFN）
  → AdaLN-Zero 条件注入（时间步 + 文本嵌入）
  → 输出：噪声预测 / 速度场预测

DiT vs UNet 架构对比:
  UNet: 
    - 卷积为主，局部感受野
    - 跳跃连接，多尺度特征
    - 参数量扩展困难
    - Stable Diffusion 1.x/2.x使用
    
  DiT:
    - 注意力为主，全局感受野
    - Patch处理，统一尺度
    - 参数量扩展容易（增加层数/宽度）
    - SD3/FLUX/DALL-E 3使用

AdaLN-Zero：通过自适应层归一化注入条件信息，比cross-attention更高效。具体来说，AdaLN使用时间步和文本嵌入来调制每个Transformer块中的Layer Normalization的缩放和偏移参数。Zero初始化确保训练初期每个Transformer块等价于恒等映射，使得深层网络的训练更加稳定。

5. 一致性模型（Consistency Models）

Song et al.（2023）提出直接学习ODE轨迹上的一致性函数，实现单步生成：

核心性质：f_θ(x_t, t) = f_θ(x_{t'}, t') = x_0  对任意 t, t'
即：轨迹上任意一点都映射到同一个起点（干净数据）

训练方式：
  - 一致性蒸馏：从预训练扩散模型蒸馏
    损失: L = E[||f_θ(x_{t_{n+1}}, t_{n+1}) - f_{θ^-}(x_{t_n}, t_n)||²]
    其中 θ^- 是EMA更新的教师网络
    
  - 一致性训练：直接从数据训练
    损失: L = E[||f_θ(x_{t_{n+1}}, t_{n+1}) - f_{θ^-}(x_{t_n}, t_n)||²]
    不需要预训练的扩散模型

一致性模型的核心思想是学习一个映射函数，将ODE轨迹上的任意点直接映射到轨迹的起点（即干净数据）。这意味着理论上只需要一步就能从噪声生成图像，实现了"单步生成"的理想。然而，单步生成的质量通常不如多步采样，因此在实际应用中往往使用1-2步采样来平衡速度和质量。LCM（Latent Consistency Models）将一致性蒸馏应用到潜在空间，成为加速Stable Diffusion推理的标准方法。

6. 整流Flow（Rectified Flow）

整流Flow通过重新对齐ODE路径，使采样路径更直，减少离散化误差：

原始Flow路径：弯曲的ODE轨迹 → 需要更多步数
整流后路径：接近直线 → 更少步数即可达到高质量

reflow操作：x'_t = (1-t)·x_0 + t·ε，学习新的v_θ使路径变直
迭代reflow → 路径越来越直 → 1-4步即可生成

整流Flow数学原理:
  原始ODE: dx/dt = v_θ(x, t), 路径弯曲
  reflow: 用当前v_θ生成(x₀, ε)配对
         重新训练v_θ使路径变直
  效果: 采样步数从20步降到1-4步
  
FLUX-schnell就使用了整流Flow:
  4步采样达到28步质量的80-90%

7. Classifier-Free Guidance (CFG)

CFG是提升条件生成质量的关键技术，由Ho & Salimans（2022）提出：

CFG 核心思想:
  同时训练有条件和无条件生成
  推理时: ε̃ = ε_θ(x_t, t, ∅) + s · (ε_θ(x_t, t, c) - ε_θ(x_t, t, ∅))
  
  其中:
    ∅ = 空条件（无条件生成）
    c = 文本条件
    s = guidance scale（通常7-12）
    
  效果:
    s=1: 标准条件生成
    s>1: 增强条件遵循度，但可能降低多样性
    s<1: 增强多样性，但可能降低条件遵循度

CFG的直觉理解:
  无条件生成: 模型自由发挥，生成"通用"图像
  有条件生成: 模型按照文本描述生成
  CFG: 放大"有条件"和"无条件"之间的差异
       → 生成更符合文本描述的图像

CFG的实际效果是增强了生成图像与文本描述的一致性。当guidance scale较高时，模型会更严格地遵循文本描述，但代价是可能降低图像的多样性和自然度。当guidance scale过高时，图像可能出现过饱和、对比度过强等artifacts。因此，选择合适的guidance scale是实际应用中的重要调参环节。不同的模型和任务通常有不同的最优guidance scale范围。

常见误区：1) 以为CFG的guidance scale越高越好——超过15通常会导致图像过饱和和artifacts，最佳范围通常在7-12之间；2) 忽略negative prompt的作用——negative prompt本质上是修改无条件生成的方向，与CFG配合使用效果更好；3) 混淆CFG和ControlNet——CFG控制文本条件的强度，ControlNet控制空间结构条件，两者互补。

8. 噪声调度策略

噪声调度（noise schedule）决定了前向过程中每个时间步的噪声水平，对生成质量有重要影响：

调度策略公式特点适用场景
线性调度β_t 从10⁻⁴线性增加到0.02简单直观DDPM原始论文
余弦调度ᾱ_t = cos²(πt/2T)噪声变化更均匀Improved DDPM
Square调度β_t 按平方增长低噪声区域更精细高质量图像生成
Flow调度线性插值 t∈[0,1]路径最直Flow Matching

噪声调度的选择直接影响模型的生成质量和采样效率。余弦调度相比线性调度的优势在于，它在低噪声区域（对应图像的高频细节）提供了更精细的噪声梯度，使得模型能够更好地学习细节恢复。Flow Matching使用的线性插值调度最为简洁，配合整流Flow可以实现极高的采样效率。

通俗类比：扩散模型就像"去噪画画"——先在画布上洒满随机噪点（前向扩散），然后一步步擦掉噪点露出画作（反向去噪）。就像考古学家清理化石——从一块乱石头里慢慢凿出恐龙骨架。训练过程就是学习"怎么凿"——给模型看很多"石头→化石"的对应关系，它就学会了从任何石头里凿出合理的化石。
通俗类比：Flow Matching和DDPM的区别就像"走直线"和"走弯路"——DDPM的随机采样路径弯弯曲曲，需要很多步才能到达目的地；Flow Matching的ODE路径接近直线，几步就到了。整流Flow更是把弯路修成了高速公路——先用reflow把路径拉直，然后1-4步就能到达。这就是为什么FLUX-schnell只需4步就能生成高质量图像。

扩散过程可视化

深入探讨：扩散模型的核心数学：前向过程q(x_t|x_{t-1})逐步加高斯噪声直到纯噪声，反向过程p_θ(x_{t-1}|x_t)学习去噪。关键简化：训练目标从预测噪声ε简化为预测x_0，损失函数变成简单的MSE。LDM的核心创新：在VAE的潜空间做扩散而不是像素空间，计算量从512×512×3降到64×64×4，效率提升100倍+。Flow Matching是最新改进——用直线ODE路径替代随机SDE，采样步数从50步降到4步，速度提升10倍。
深入探讨：从信息论角度看扩散模型：前向过程逐步损失信息（熵增加），反向过程逐步恢复信息（熵减少）。互信息I(x₀; x_t)随t增加而减小，当t=T时接近0。神经网络的任务是学习条件分布p(x_{t-1}|x_t)来最大化数据似然。变分下界（ELBO）将对数似然分解为T个KL散度项的和，每项对应一个时间步。简化训练目标L_simple是ELBO的一个加权近似，实验表明它比完整的ELBO训练效果更好，可能是因为它隐式地对不同时间步进行了重要性加权。

扩散模型采样步数与生成质量的权衡

扩散模型的核心工程挑战是在采样步数和生成质量之间寻找最优平衡。DDPM 需要 20-1000 步才能生成高质量图像，但每步都需要一次完整的神经网络前向传播，计算成本极高。Flow Matching 通过 ODE 路径直线化将步数降到 4-28 步，整流 Flow 进一步降到 1-4 步，FLUX-schnell 仅需 4 步即可达到 28 步质量的 80-90%。一致性模型更是实现了单步生成的理想。选择哪种方法取决于应用场景：实时生成（如视频渲染）优先选择一致性模型或整流 Flow（1-4步），高质量图像生成优先选择 Flow Matching（4-28步），研究探索优先选择 DDPM（完整理论框架）。

通俗类比：噪声调度就像"调音量旋钮"——线性调度是匀速转动旋钮，余弦调度是先快后慢再快，Flow调度是直接从0拧到1。不同的旋钮转动方式会影响"去噪"的效果——余弦调度在低音量（低噪声）时转动更慢，让模型有更多机会学习细节。
常见误区：1) 以为所有扩散模型都用同样的噪声调度——不同模型（DDPM/SD/FLUX）使用不同的调度策略，混用会导致生成质量下降；2) 忽略时间步嵌入的重要性——时间步t告诉模型当前的噪声水平，没有它模型无法区分不同阶段的去噪任务。
