1. 多模态数据 Pipeline

数据是多模态LLM的灵魂。模型的能力上限由数据质量决定，而非架构设计。构建高质量的多模态训练数据需要经过四个阶段：采集、清洗、增强和混合。每个阶段都有其独特的挑战和技术要求。

💡 实用技巧：关于「本节」，建议关注实际工程中的关键参数调优和边界条件处理，这些往往决定了系统在生产环境中的稳定性和性能表现。
💡 实用技巧：使用GPT-4V生成数据时，prompt的设计至关重要。好的prompt应该明确指定输出格式（JSON）、回答长度（50-200字）、回答风格（客观描述）。建议参考LLaVA官方的prompt模板，它经过了大量实验验证。
⚠️ 注意事项：不要盲目照搬网上的数据配置。不同的模型规模和任务需求下，数据的最优配置可能完全不同。例如，小模型（7B）需要更简洁的回答，大模型（72B）可以处理更长更复杂的回答。务必在自己的场景下做充分的验证实验。
🔍 深入理解：数据构建的核心挑战是"质量-规模"权衡。高质量数据（如GPT-4V生成）成本高但效果好，低质量数据（如网页爬取）成本低但噪声大。最优策略是分阶段使用：Stage 1预训练用大量中等质量数据（图文对），Stage 2微调用少量高质量数据（指令数据）。这个"粗粮+细粮"的策略在LLM训练中普遍适用。

2.2 数据清洗 Pipeline

# 图文对数据清洗 Pipeline
def clean_image_text_pairs(pairs):
    cleaned = []
    seen_hashes = set()

    for img, text in pairs:
        # 1. 图像质量检查
        if img.size[0] < 224 or img.size[1] < 224:
            continue  # 太小的图像
        if img.mode != 'RGB':
            continue  # 非 RGB 图像

        # 2. 文本质量检查
        if len(text) < 10 or len(text) > 1000:
            continue  # 太短或太长
        if text.count(' ') < 3:
            continue  # 可能不是自然语言

        # 3. CLIP 相似度过滤
        clip_score = compute_clip_similarity(img, text)
        if clip_score < 0.25:
            continue  # 图文不匹配

        # 4. 去重
        img_hash = compute_image_hash(img)
        if img_hash in seen_hashes:
            continue
        seen_hashes.add(img_hash)

        # 5. NSFW 过滤
        if is_nsfw(img):
            continue

        cleaned.append((img, text))

    return cleaned

💡 实用技巧：数据清洗是数据构建中最耗时但也最重要的环节。建议先用小规模数据（10K）跑通整个清洗pipeline，验证每个步骤的过滤比例是否合理。如果某个步骤过滤了超过50%的数据，可能是阈值设置太严格。逐步调整阈值，找到质量-规模的最佳平衡点。
⚠️ 注意事项：CLIP相似度阈值的选择需要谨慎。阈值太高（>0.35）会过滤掉很多有效数据，阈值太低（<0.20）会让很多图文不匹配的数据混入。建议先人工检查100个不同CLIP分数的样本，了解分数分布后再决定阈值。
🔍 深入理解：CLIP相似度作为质量指标的原理是：如果图像和文本描述高度匹配，CLIP编码后的相似度就高。这个指标的有效性来源于CLIP预训练时使用的对比学习——它天然学习了图文匹配的语义。但CLIP分数也有局限：它可能给"一只猫"和任何猫的图片都打高分，即使文本描述非常简略。因此CLIP分数只能作为初筛，不能替代人工审核。

💡 实用技巧：生成多轮对话数据时，要注意对话的连贯性。每一轮的问题应该基于上一轮的回答自然延伸，而不是孤立的问题。好的多轮对话应该像人类聊天一样自然流畅。建议在prompt中明确要求GPT-4V生成"连贯的、有逻辑递进的"对话。
⚠️ 注意事项：GPT-4V生成的数据可能包含幻觉（编造不存在的物体或属性）。建议对生成的数据进行抽样人工审核，特别是物体识别和空间关系类的问答。如果幻觉比例超过10%，需要调整prompt或增加后处理过滤。
🔍 深入理解：用GPT-4V生成训练数据的本质是"知识蒸馏"——将GPT-4V的视觉理解能力通过数据传递给更小的模型。这种方法的核心假设是：如果小模型能学会模仿GPT-4V的回答风格和质量，它就能获得类似的理解能力。LLaVA的成功证明了这个假设的可行性。但也要注意，蒸馏数据的质量上限受限于教师模型（GPT-4V）的能力。

# 构建OCR训练数据的关键:
# 1. 多样化文档类型 (发票、合同、简历、论文等)
# 2. 多样化语言 (中文、英文、日文等)
# 3. 多样化布局 (表格、列表、段落等)
# 4. 确保标注准确率 > 99%
💡 实用技巧：OCR数据的质量对多模态LLM的文档理解能力影响巨大。建议使用多种OCR工具（PaddleOCR、Tesseract、GPT-4V）交叉验证标注质量。如果多个工具的结果一致，标注可信度就高。
⚠️ 注意事项：OCR数据中的表格结构标注是最难的部分。简单的文本拼接会丢失表格的行列结构信息。建议使用Markdown表格格式或HTML表格格式来保留结构信息。
🔍 深入理解：OCR数据的特殊性在于它需要"精确理解"而非"大致理解"。自然图像理解允许一定程度的模糊（"一只猫"可以是任何颜色的猫），但OCR要求100%准确（"12.5"不能变成"12.50"或"125"）。这种精度要求使得OCR数据的构建比自然图像数据更难、成本更高。

# 视频数据处理
def process_video_data(video_path):
    """处理视频数据为训练格式"""
    # 1. 采样关键帧
    frames = sample_keyframes(video_path, num_frames=8)

    # 2. 为每帧生成描述
    descriptions = []
    for frame in frames:
        desc = gpt4v.generate(image=frame, prompt="描述这个画面")
        descriptions.append(desc)

    # 3. 生成时序推理问题
    temporal_qa = generate_temporal_qa(frames, descriptions)

    return {
        "video": video_path,
        "frames": frames,
        "descriptions": descriptions,
        "qa_pairs": temporal_qa
    }

# 关键挑战:
# 1. 帧采样策略 (均匀采样 vs 关键帧检测)
# 2. 时序推理 (需要理解事件的先后顺序)
# 3. token数量 (8帧 × 256 tokens = 2048 tokens)
视频数据处理流程: 视频文件 (MP4) │ ├── 帧采样 (均匀/关键帧) │ → 8-32 帧 │ ├── 每帧编码 (ViT) │ → 8 × 256 = 2048 tokens │ ├── 时序标注 │ → 帧描述 + 时序关系 │ └── 生成问答 → 描述类: "视频中发生了什么？" → 时序类: "先发生了什么？" → 推理类: "为什么会出现这种情况？"
💡 实用技巧：视频数据的构建成本很高（需要大量GPU时间来编码帧）。建议先从短视频（<30秒）开始，使用8帧采样。验证模型的视频理解能力后再扩展到长视频。
⚠️ 注意事项：视频数据的token数量是图像的8-32倍，这对推理成本影响巨大。在构建视频数据时要控制帧数，避免token爆炸。建议使用Pixel Shuffle等压缩技术。
🔍 深入理解：视频理解的核心挑战是"时序推理"——不仅要理解每一帧的内容，还要理解帧与帧之间的变化和因果关系。这是当前多模态LLM最薄弱的环节，也是最有价值的研究方向。

8.4 数据构建成本估算

构建大规模多模态训练数据的成本是一个需要认真考虑的问题。以下是各项成本的估算：1）GPT-4V API调用成本——生成一条视觉指令数据大约需要0.01-0.03美元（取决于图片大小和回答长度）。生成100K条数据需要1000-3000美元。2）计算成本——CLIP相似度过滤、NSFW检测等预处理步骤需要GPU计算。处理1M张图片大约需要10-20小时的A100 GPU时间。3）人工标注成本——如果需要人工审核，每小时大约可以审核100-200个样本。审核100K样本需要500-1000小时的人工时间。4）存储成本——100K张图片（平均500KB/张）需要约50GB存储空间。加上文本数据和元数据，总计约60GB。

总的成本估算：构建一个100K高质量视觉指令数据集的总成本约为2000-5000美元（如果完全使用API生成）或500-1500美元（如果使用开源模型生成+人工审核）。对于预算有限的研究团队，建议从10K小规模数据开始，验证数据质量后再逐步扩展。也可以考虑使用开源的视觉语言模型（如LLaVA-1.5）替代GPT-4V来降低成本，但需要注意生成质量可能不如GPT-4V。

8.5 开源数据集推荐

对于资源有限的研究团队，可以直接使用开源的多模态训练数据集，避免从零开始构建数据。以下是推荐的开源数据集及其特点：LLaVA-Pretrain-558K是Stage 1预训练的标准数据集，包含558K高质量图文对，从CC3M中筛选而来。LLaVA-Instruct-158K是Stage 2指令微调的标准数据集，包含158K多轮视觉对话数据，由GPT-4生成。ShareGPT4V包含100K极其详细的图像描述，适合需要精细理解能力的场景。ALLaVA包含更丰富的指令类型和更长的回答。这些数据集都可以在HuggingFace上免费下载。

7. 数据质量评估方法

构建训练数据后，需要系统地评估数据质量。低质量的训练数据会导致模型学到错误的模式，产生幻觉或理解错误。以下是几种常用的数据质量评估方法，每种方法适用于不同维度的质量检查。

7.1 自动化质量评估

自动化质量评估可以在大规模数据上快速筛选出低质量样本。常用的方法包括：1）CLIP相似度评估——计算图文对的CLIP相似度分数，低于阈值的样本可能是图文不匹配的。2）文本困惑度评估——用预训练语言模型计算文本描述的困惑度，困惑度异常高的样本可能是非自然语言或包含错误。3）图像质量评估——使用图像质量评估模型（如MUSIQ、CLIPIQA）检测模糊、低分辨率、过度曝光等问题图像。4）多样性评估——计算样本之间的相似度，去除过于相似的冗余样本。

自动化评估的优势是可以大规模应用，但准确性有限。建议将自动化评估作为初筛手段，对筛选后的样本进行人工抽检。通常，自动化评估可以将数据规模缩小到原来的30-50%，同时保留90%以上的高质量样本。

7.2 人工抽检

人工抽检是数据质量评估的金标准。建议在每个数据源上随机抽取100-200个样本，由人工审核以下维度：图文匹配度、文本质量、信息密度、标注一致性。人工抽检的成本较高，但可以发现自动化方法无法检测的问题。例如，有些图文对的CLIP分数很高，但描述过于简略或遗漏了重要信息。

7.3 数据多样性分析

训练数据的多样性直接影响模型的泛化能力。如果数据中90%都是自然风景图片，模型在文档理解任务上就会表现很差。建议从以下维度分析数据多样性：图像类型分布、文本类型分布、语言分布、难度分布。如果发现某个维度的分布严重不均衡，需要有针对性地补充数据。

7.4 数据版本管理

在多轮数据迭代中，建议使用数据版本管理工具（如DVC、Git LFS）来追踪数据的变化。每次数据更新都应该记录：新增了哪些数据、删除了哪些数据、修改了哪些数据的标注、数据的来源和生成方式。这些信息对于调试模型问题和复现实验结果至关重要。数据版本管理的最佳实践是：每次训练使用一个固定的数据版本号，记录在实验日志中。

8. 数据构建工具与生态

构建高质量的多模态训练数据需要一系列工具和基础设施的支持。以下是常用的数据构建工具和最佳实践，帮助你高效地完成数据构建工作。

8.1 图像采集工具

大规模图像采集通常使用网络爬虫工具。常用的工具包括：1）img2dataset——专门用于下载和处理大规模图文对数据集的工具，支持从URL列表批量下载图片，并自动进行格式转换和质量筛选。2）gallery-dl——支持从多个图片网站（如Flickr、Instagram、Pinterest）批量下载图片和元数据。3）自定义爬虫——对于特定领域的数据（如医学影像、卫星图像），可能需要编写专门的爬虫程序。

在使用爬虫工具时，需要注意版权和隐私问题。建议只使用开放许可的图片（如Creative Commons），并遵守网站的robots.txt规则。对于包含人脸的图片，需要进行隐私脱敏处理（如模糊人脸）。

8.2 标注工具

当需要人工标注数据时，可以使用专业的标注工具。常用的工具包括：1）Label Studio——开源的数据标注平台，支持图像、文本、音频等多种数据类型的标注。2）CVAT——专门用于计算机视觉任务的标注工具，支持目标检测、语义分割、关键点检测等。3）Prodigy——商业标注工具，支持主动学习，可以智能选择最有价值的样本进行标注，提高标注效率。

对于多模态数据的标注，最重要的是标注规范的制定。标注规范应该明确定义：标注的目标（要提取什么信息）、标注的格式（JSON、XML等）、标注的质量标准（准确率要求）、标注的流程（初标、复标、终审）。一个好的标注规范可以大幅提高标注质量和效率。

8.3 数据增强技术

数据增强可以有效扩充训练数据的规模和多样性。常用的多模态数据增强技术包括：1）图像变换——裁剪、翻转、旋转、颜色抖动等，这些变换不会改变图像的语义内容，但可以增加数据的多样性。2）文本改写——用GPT-4或同义词工具对文本描述进行改写，生成语义相同但表述不同的数据。3）多视角生成——对同一张图片生成不同角度、不同粒度的描述（简单描述、详细描述、专业描述）。4）负样本生成——对每张图片生成图文不匹配的负样本，用于训练模型区分匹配和不匹配的图文对。

数据增强的关键原则是保持数据的真实性。增强后的数据应该与真实数据在分布上一致。过度的增强（如过度裁剪、过度颜色变换）可能引入不真实的样本，反而降低模型的泛化能力。建议在增强后人工抽检一批样本，确保增强后的数据质量。
