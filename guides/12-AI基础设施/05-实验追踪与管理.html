为什么需要实验追踪

通俗类比：就像探险家记录航程日志，实验追踪是记录AI模型训练的'探险日记'。没有它，改进就像在迷宫中盲行。
常见误区： 很多人在学习为什么需要实验追踪时容易陷入"只看不练"的误区，以为理解了概念就能在实际项目中应用。实际上，理论知识必须配合动手实验才能真正掌握。建议每学完一个知识点就写一段验证代码，在实践中发现和解决问题。
深入探讨：MLflow标准实验一次可记录数百个指标、几十个参数和上百个模型版本。团队协作时，平均可减少40%的重复实验，节省30%的GPU算力。
训练流程 -> 记录参数 -> 记录指标 -> 记录模型 -> 保存快照
追踪要素作用类比
参数(Parameters)记录超参数配置菜谱中的调料用量
指标(Metrics)记录loss/accuracy等烹饪过程中的温度计
模型(Model)保存模型文件最终成品菜肴
方面要点注意事项
核心概念为什么需要实验追踪的基本原理理解底层机制比记忆参数更重要
实践应用在实际项目中的使用方式先跑通demo再优化
常见问题踩坑记录和解决方案参考社区issue和讨论

LLM训练实验动辄花费数天到数周，消耗大量GPU资源。没有系统化的实验追踪，你将无法回答"上次训练用了什么参数？效果如何？"这类关键问题。实验追踪是科学方法论在ML工程中的落地。

⚠️ 无追踪的代价

缺乏实验追踪会导致：重复实验浪费资源、无法复现最佳结果、团队协作混乱、模型选型缺乏数据支撑。据估计，ML团队平均浪费20-30%的计算资源在重复实验上。

MLflow

通俗类比：像瑞士军刀，MLflow集成了实验追踪、模型打包、部署等多种功能。用好这把'刀'，AI项目管理事半功倍。
常见误区： 很多人在学习为什么需要实验追踪时容易陷入"只看不练"的误区，以为理解了概念就能在实际项目中应用。实际上，理论知识必须配合动手实验才能真正掌握。建议每学完一个知识点就写一段验证代码，在实践中发现和解决问题。
深入探讨：核心组件：Tracking(追踪)、Projects(项目)、Models(模型)、Registry(注册)。单机版可管理1000+实验，分布式版支持PB级数据。典型启动时间<5秒。
本地训练 -> MLflow Tracking -> UI展示 -> 模型注册 -> 部署服务
组件功能典型用法
mlflow.start_run()开始记录实验包裹训练代码
mlflow.log_param()记录参数learning_rate=0.01
mlflow.log_metric()记录指标accuracy=0.92
mlflow.sklearn.log_model()保存模型保存训练好的模型

MLflow是最流行的开源ML生命周期管理平台，包含四大组件：

组件功能说明
MLflow Tracking实验追踪记录参数、指标、产物
MLflow Projects项目打包可复现的实验运行
MLflow Models模型管理统一的模型封装格式
MLflow Registry模型注册版本管理与生命周期
# MLflow基本使用
import mlflow

mlflow.set_tracking_uri("http://mlflow-server:5000")
mlflow.set_experiment("llm-finetune-v2")

with mlflow.start_run(run_name="llama-7b-lora"):
    # 记录超参数
    mlflow.log_params({
        "model": "llama-7b",
        "lr": 2e-4,
        "batch_size": 32,
        "lora_rank": 16,
        "epochs": 3
    })

    # 训练过程中记录指标
    for step, loss in enumerate(train_losses):
        mlflow.log_metric("train_loss", loss, step=step)

    # 记录模型产物
    mlflow.pytorch.log_model(model, "model")

Weights & Biases

通俗类比：像智能仪表盘，W&B将复杂数据转化为直观图表。它不仅是记录工具，更是实验分析的'智能副驾驶'。
常见误区：不要过度依赖自动记录，重要指标需手动标记。就像开车只看自动驾驶，关键时刻可能错过关键信息。
深入探讨：支持20+主流ML框架集成，自动记录GPU利用率、系统指标。单项目可存储TB级实验数据，图表响应时间<200ms。协作团队人均效率提升35%。
代码插入wandb -> 自动收集指标 -> 实时可视化 -> 对比分析 -> 报告生成
功能W&B实现MLflow对比
实时监控Web界面实时更新需手动刷新UI
协作功能内置团队协作需额外配置
自动追踪深度框架集成需手动添加代码

W&B是商业化的MLOps平台，提供更丰富的可视化和协作功能：

# W&B使用示例
import wandb

wandb.init(
    project="llm-training",
    name="mistral-7b-sft",
    config={
        "learning_rate": 2e-5,
        "architecture": "Mistral-7B",
        "dataset": "alpaca-cleaned",
        "epochs": 3
    }
)

# 自动记录训练指标
for epoch in range(num_epochs):
    train_loss = train_one_epoch(model)
    val_loss = evaluate(model, val_data)
    wandb.log({
        "train/loss": train_loss,
        "val/loss": val_loss,
        "train/lr": scheduler.get_last_lr()[0]
    })

# 记录模型
wandb.save("model_checkpoint.pt")
wandb.finish()

💡 实验命名规范

推荐格式：{model}_{task}_{date}_{variant}，例如 llama7b_sft_20240115_lora16。好的命名让实验历史一目了然。

  • 自动记录：使用autolog功能自动捕获框架指标（如MLflow的mlflow.pytorch.autolog()）
  • 分层组织：Experiment → Run → Step 三层结构管理实验
  • 产物管理：训练检查点、评估报告、可视化图表都作为产物保存
  • 标签系统：使用tags标注实验状态（baseline/candidate/production）
  • 对比分析：定期对比关键指标趋势，识别最优实验方向

✅ 参考答案与详解
  1. 核心原理: 实验追踪是科学方法论在ML工程中的落地，核心是记录每次实验的四要素：参数（Parameters，如lr=2e-4、batch_size=32）、指标（Metrics，如train_loss随step的变化）、模型产物（Model artifacts，如checkpoint文件）、运行环境（硬件配置、CUDA版本等）。MLflow是最流行的开源平台，四大组件：Tracking（记录参数/指标/产物）、Projects（可复现的实验运行）、Models（统一的模型封装格式）、Registry（版本管理与生命周期）。W&B提供更丰富的可视化（实时Web界面、交互式Tables）和协作功能（团队报告、Sweeps自动超参数搜索），支持20+主流ML框架集成，单项目可存储TB级实验数据。缺乏实验追踪会导致重复实验浪费20-30%的GPU算力。
    关键要点: ①实验命名规范推荐格式：{model}_{task}_{date}_{variant}（如llama7b_sft_20240115_lora16）；②MLflow的autolog功能（mlflow.pytorch.autolog()）自动捕获框架指标减少手动代码；③W&B Sweeps支持贝叶斯优化自动搜索超参数空间，可优化lr和LoRA rank组合；④分层组织Experiment→Run→Step三层结构管理实验，使用tags标注状态（baseline/candidate/production）；⑤定期对比关键指标趋势，每周清理<0.1提升度的实验，关键实验保留3个checkpoints
    实际案例: LoRA微调实验追踪：使用mlflow.start_run(run_name="llama-7b-lora")包裹训练代码，通过log_params记录model/lr/lora_rank/epochs，训练过程中每100步log_metric("train_loss", loss, step=step)，最终log_model保存模型。同时配置W&B Sweeps搜索lr∈[1e-5,5e-4]和rank∈[8,16,32,64]的最优组合，使用贝叶斯优化策略。
    关联知识: 与模型版本管理（第6章）的Registry阶段流转、CI/CD（第7章）的自动化评估流水线紧密关联。实验追踪的结果是模型注册和上线的决策依据。
