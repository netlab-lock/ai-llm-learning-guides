08 - 拒绝采样与迭代对齐

1. 拒绝采样微调（RFT）

对齐训练的工程实践与最新进展

对齐训练的工程实现涉及多个精密调优的环节。数据管线方面，偏好数据需要经过严格的质量控制：标注者一致性检查(agreement rate需>60%)、异常样本过滤(如明显的标注错误)、以及数据平衡(确保chosen/rejected的质量差异适中——差异太大模型学不到细微差别，差异太小模型分不清好坏)。

训练稳定性是RLHF/PPO的核心挑战。常见问题包括：(1)奖励黑客——模型发现RM的漏洞，生成高分但低质量的输出；(2)KL散度爆炸——模型偏离SFT基线太远，输出变得不可控；(3)训练崩溃——奖励分数突然归零，模型忘记所有已学知识。对策包括：定期更新RM(对抗性训练)、使用clip范围限制策略更新幅度、以及checkpoint回退(检测到异常时恢复到最近的正常checkpoint)。

最新的对齐方法正在向"无需奖励模型"的方向发展。DPO直接从偏好对学习，省去了RM训练的开销。GRPO用组内相对排名替代绝对奖励，进一步简化了流程。KTO只需要好/坏标签(不需要配对)，降低了数据收集成本。RLVR(Reinforcement Learning with Verifiable Rewards)用于数学/代码等可验证任务，奖励信号来自客观的正确性判断而非人类偏好。这些方法使得对齐训练的成本从RLHF的数万美元降到数千美元。

拒绝采样微调（Rejection Sampling Fine-Tuning, RFT）是一种简单但有效的对齐方法：对同一问题生成多个回答，通过奖励模型或规则筛选出最好的回答，然后用这些高质量回答做监督微调。

1.1 基本流程

  1. 生成阶段：对每个 prompt 生成 N 个候选回答（通常 N=8~64）
  2. 评分阶段：用奖励模型（或规则验证器）对每个回答打分
  3. 筛选阶段：选择得分最高的回答（或 top-k 回答）
  4. 训练阶段：用筛选后的高质量数据做 SFT
核心直觉：RFT 本质上是"用搜索代替学习"——不做梯度更新优化策略，而是通过大量采样+筛选来找到好的回答，再让模型模仿这些好回答。这在数学推理等场景特别有效，因为正确答案可以被验证。

1.2 RFT 的理论基础

P_RFT(y|x) ∝ P_base(y|x) · 𝟙[r(x,y) ≥ threshold]

RFT 通过截断基座模型的分布，只保留高奖励区域的质量。当 N→∞ 时，RFT 可以逼近最优策略（在奖励模型准确的前提下）。

⚠️ 注意事项：关于「1. 拒绝采样微调（RFT）」的常见陷阱：不要盲目照搬网上的配置参数。不同的数据分布和模型规模下，拒绝采样微调的最优配置可能完全不同。务必在自己的场景下做充分的验证实验。
🔍 深入理解：「1. 拒绝采样微调（RFT）」的深层理解：RFT 的理论优雅之处在于将对齐问题分解为搜索和模仿两个子问题。与直接优化奖励函数的 RL 方法（如 PPO 不稳定）不同，RFT 将问题拆分为两个成熟的子问题：(1) 高效探索输出空间以找到高奖励样本，(2) 用监督学习让模型复现这些样本。这种分离使得 RFT 比 PPO 稳定得多，同时保持强劲性能。核心洞察来自极值理论：随着采样数 N 增加，Best-of-N 的期望质量趋近理论最优。实践中 N=16-64 已能捕获大部分收益，使 RFT 在计算上可行。
RLHF Pipeline (4个模型): Prompt → [Actor] → Response → [Reward Model] → Score ↓ ↓ [Critic] → Value PPO Update ← KL Penalty ↓ [Reference] → KL DPO Pipeline (2个模型): (Prompt, y_w, y_l) → [Policy] → log π(y_w), log π(y_l) ↓ [Reference] → log π_ref(y_w), log π_ref(y_l) ↓ DPO Loss (无需奖励模型) GRPO Pipeline (2个模型): Prompt → [Actor] × G次采样 → {y_1,...,y_G} ↓ [Rule-based Reward] → {r_1,...,r_G} ↓ 组内排名归一化 → 优势估计A_i ↓ GRPO Update (无需Critic)

Best-of-N是 RFT 的推理时版本：在推理时对同一问题生成 N 个回答，直接选最好的给用户，不做任何训练。

2.1 Best-of-N 的效果

N 值典型效果推理成本
N=1基线性能1x
N=4显著提升4x
N=16接近微调效果16x
N=64接近最优64x
实用价值：Best-of-N 是评估奖励模型质量的重要基线。如果 Best-of-N(N=64) 的效果远优于 PPO 训练的模型，说明奖励模型可能有问题，或者 RL 训练没有充分利用奖励信号。

⚠️ 注意事项：关于「2. Best-of-N 采样」的常见陷阱：常见陷阱：忽略DPO的边界条件和异常情况。这些在小规模测试中可能不会暴露，但在生产环境中往往是最先出问题的地方。
🔍 深入理解：「2. Best-of-N 采样」的深层理解：DPO背后的核心权衡是灵活性与效率之间的平衡。更灵活的方案通常计算开销更大，更高效的方案往往需要更强的假设。在PPO的设计中也能看到类似的权衡。

3. 迭代对齐（Iterative Alignment）

迭代对齐将拒绝采样与模型训练结合，形成"生成→筛选→训练→再生成"的循环。

3.1 迭代对齐的基本框架

for iteration in range(num_iterations):
    # 1. 用当前策略模型生成候选回答
    candidates = generate(policy_model, prompts, n_samples=N)
    
    # 2. 用奖励模型/规则筛选
    best_responses = select_best(candidates, reward_fn)
    
    # 3. 构建训练数据
    train_data = build_dataset(prompts, best_responses)
    
    # 4. SFT 微调当前模型
    policy_model = sft_train(policy_model, train_data)
    
    # 5. （可选）更新奖励模型
    if iteration % reward_update_freq == 0:
        reward_model = update_reward_model(new_preferences)

3.2 迭代对齐的优势

  • 自我改进：每轮迭代后模型能生成更好的候选，进一步提升筛选质量
  • 训练简单：每轮只需要 SFT，比 PPO 简单得多
  • 数据效率：通过在线生成不断获取新的高质量数据
  • 可验证任务友好：数学、代码等可以用执行结果替代奖励模型
💡 实用技巧：关于「3. 迭代对齐（Iterative Alignment）」的实用建议：实战技巧：善用日志和可视化工具来理解系统行为。Iterative相关指标的监控可以帮助你快速定位问题。建立完善的实验记录习惯。
⚠️ 注意事项：关于「3. 迭代对齐（Iterative Alignment）」的常见陷阱：最常见的错误是跳过baseline直接上复杂方案。没有Iterative的baseline对比，你无法判断复杂方案是否真的带来了提升。
🔍 深入理解：「3. 迭代对齐（Iterative Alignment）」的深层理解：从信息论的角度思考——理解Iterative需要信息的压缩与保留、信号与噪声的分离、冗余与效率的权衡，这些都是贯穿整个AI工程的核心主题。

4. Constitutional AI 的迭代自改进

Constitutional AI（CAI）由 Anthropic 提出，是迭代对齐的经典范例。其核心思想是让 AI 根据一组"宪法原则"自我批评和改进。

4.1 CAI 的两阶段流程

阶段一：监督自改进（SL）

  1. 模型生成一个初始回答（可能有害）
  2. 模型根据宪法原则自我批评："这个回答违反了哪条原则？"
  3. 模型修改回答使其符合原则
  4. 用修改后的回答做 SFT

阶段二：强化学习自改进（RL）

  1. 对同一问题生成多个修改后的回答
  2. AI 根据宪法原则对回答进行排序
  3. 用排序数据训练奖励模型
  4. 用 RL 优化策略模型
宪法原则示例："请选择最无害、最诚实、最有帮助的回答"、"请选择不包含歧视性内容的回答"、"请选择不会帮助用户从事危险活动的回答"。
⚠️ 注意事项：关于「4. Constitutional AI 的迭代自改进」的常见陷阱：一次只改一个变量。在调整CAI的参数时，如果同时改多个参数，你无法确定是哪个改动带来了效果变化或引入了问题。
🔍 深入理解：「4. Constitutional AI 的迭代自改进」的深层理解：从历史视角看，当前CAI的主流方法经历了多年的演进。早期被认为过时的技术在大规模场景下可能重新焕发生机。保持对经典方法的了解很重要。

5. DeepSeek-R1 的拒绝采样实践

DeepSeek-R1在其训练流程中大量使用了拒绝采样，特别是将 RL 训练的推理模型的高质量输出蒸馏到更小模型中。

5.1 DeepSeek-R1 的多阶段流程

  1. 冷启动 SFT：用少量高质量推理数据微调基座模型
  2. RL 训练（GRPO）：用可验证奖励训练推理能力
  3. 拒绝采样：从 RL 模型生成大量回答，筛选正确且推理过程清晰的样本
  4. SFT 蒸馏：用筛选后的数据训练小模型（1.5B/7B/14B/32B）
  5. 再次 RL：对小模型再做一轮 RL 训练

5.2 拒绝采样的筛选标准

def deepseek_filter(question, response, ground_truth):
    """DeepSeek-R1 风格的筛选逻辑"""
    # 1. 正确性检查
    is_correct = verify_answer(response, ground_truth)
    
    # 2. 格式检查（是否包含 <think> 标签）
    has_thinking = "<think>" in response and "</think>" in response
    
    # 3. 长度合理性（不太短也不太长）
    thinking_length = extract_thinking(response)
    is_reasonable_length = 200 < len(thinking_length) < 8000
    
    # 4. 语言一致性（不混杂多种语言）
    is_consistent_lang = check_language_consistency(response)
    
    return all([is_correct, has_thinking, is_reasonable_length,
                is_consistent_lang])
💡 实用技巧：关于「5. DeepSeek-R1 的拒绝采样实践」的实用建议：建议优先使用成熟的开源工具和框架，避免重复造轮子。RL生态中有许多经过生产验证的工具可以选择。
⚠️ 注意事项：关于「5. DeepSeek-R1 的拒绝采样实践」的常见陷阱：隐蔽的陷阱是评估数据泄漏。确保RL的评估数据完全没有出现在训练集中。即使部分重叠也会导致评估结果严重失真。
🔍 深入理解：「5. DeepSeek-R1 的拒绝采样实践」的深层理解：工程哲学：在RL的实践中，简单但规模大往往胜过复杂但规模小。Chinchilla定律和Scaling Law都指向同一个方向——数据和计算的规模比精巧的算法设计更重要。

6. 实现模式与工程实践

6.1 大规模拒绝采样实现

import torch
from vllm import LLM, SamplingParams

# 使用 vLLM 进行高效批量生成
llm = LLM(model="your_model_path", tensor_parallel_size=4)
sampling_params = SamplingParams(
    temperature=0.8,
    top_p=0.95,
    n=32,            # 每个 prompt 生成 32 个候选
    max_tokens=2048,
)

# 批量生成
outputs = llm.generate(prompts, sampling_params)

# 筛选最佳回答
selected = []
for output in outputs:
    candidates = output.outputs
    scored = [(c.text, reward_fn(output.prompt, c.text)) 
              for c in candidates]
    best = max(scored, key=lambda x: x[1])
    selected.append({"prompt": output.prompt, "response": best[0]})

6.2 迭代轮次选择

迭代次数效果说明
1 轮显著提升大多数场景已经足够
2-3 轮继续提升但边际递减推荐的迭代次数
4+ 轮趋于饱和收益递减明显
注意事项：迭代对齐可能导致模型逐渐偏离原始分布（distribution drift）。建议每轮迭代后检查模型在通用基准上的表现，避免过拟合到奖励模型的偏好。使用 KL 约束或定期回退到参考数据可以缓解此问题。
💡 实用技巧：关于「6. 实现模式与工程实践」的实用建议：好习惯：定期做checkpoint并保存实验配置。特别是在调整SamplingParams的参数时，记录每组参数的效果对比。这些记录在后续复盘时非常有价值。
🔍 深入理解：「6. 实现模式与工程实践」的深层理解：SamplingParams的发展遵循AI领域的通用演进模式——从手工规则到数据驱动，从监督学习到自监督/强化学习，从单一模态到多模态融合。在KL中也能观察到这个趋势。

  1. 核心原理: 本节介绍了三种核心对齐方法——拒绝采样微调(RFT)、Best-of-N采样、迭代对齐。RFT的本质是"用搜索代替学习"：通过大量采样(N=8~64)生成候选回答，用奖励模型/规则筛选最佳回答，再用高质量数据做SFT。其数学基础是截断分布 P_RFT(y|x) ∝ P_base(y|x)·𝟙[r(x,y)≥threshold]，当N→∞时逼近最优策略。Best-of-N是RFT的推理时版本，不做训练直接选最好的给用户，是评估RM质量的重要基线。迭代对齐将RFT扩展为多轮循环——生成→筛选→训练→再生成，每轮迭代后模型能力提升，进一步生成更高质量候选。这三种方法构成了从简单到复杂的对齐工程体系：Best-of-N(无训练)→RFT(单轮训练)→迭代RFT(多轮训练)。关键要点: ①RFT=采样+筛选+模仿的三步法 ②Best-of-N是评估RM的基线 ③迭代RFT形成自我改进闭环 ④N=16-64已能捕获大部分收益 ⑤可验证任务(数学/代码)中效果最佳。实际案例: DeepSeek-R1在训练流程中大量使用拒绝采样——从RL模型生成大量回答，筛选正确且推理过程清晰的样本用于SFT蒸馏到1.5B-32B小模型。关联知识: 与RLHF/PPO/DPO/GRPO等方法形成互补——RFT更简单稳定，PPO/DPO更灵活但更难调参。
  2. 对比分析: Best-of-N: 零训练成本、N倍推理延迟、上限受采样多样性限制、极稳定。适合推理预算充足的离线条目生成(如数学题求解)。RFT: 单次训练成本、正常推理速度、受初始模型能力上限限制、稳定。适合批量生成高质量训练数据的场景。迭代RFT: 多轮训练成本、正常推理速度、可突破初始模型上限(自我改进)、每轮边际收益递减。适合需要持续提升的场景(如对话系统)。PPO: 在线学习成本高、需要Actor+Critic+RM+Reference四个模型、可学习新策略模式但不稳定。适合对性能要求极高的头部应用。DPO: 只需两个模型(Policy+Reference)、无需RM、简单稳定、但依赖静态数据集。适合偏好数据充足的场景。最佳选择: 快速验证 → Best-of-N; 一次性提升 → RFT; 持续提升 → 迭代RFT; 极致性能 → PPO; 数据充足+稳定优先 → DPO。
  3. 应用设计方案: 场景: 为企业客服系统构建对齐训练Pipeline。步骤一(数据准备): 收集客服对话prompt集1万条，覆盖常见问题类型并标注正确答案(可验证)。步骤二(初始训练): 用基础模型对每个prompt生成N=16个候选回答，用规则验证器(正确答案匹配)+人工抽检筛选最佳回答。步骤三(RFT训练): 用筛选后的高质量数据做SFT，训练1个epoch。步骤四(迭代优化): 用RFT后的模型再次生成候选→筛选→训练，共2-3轮。步骤五(评估): 对比Best-of-N基线、RFT模型、迭代RFT模型在客服评测集上的正确率、满意度、对话轮次效率。步骤六(部署上线): A/B测试验证效果后灰度发布，持续监控在线指标(满意度评分、转人工率)。

🔬 拒绝采样与迭代对齐技术深度剖析：

拒绝采样微调（RFT）的理论基础来自极值理论：当从基座模型P_base(y|x)中采样N个候选时，Best-of-N的期望奖励随N增加而单调递增，且满足E[r(y*)] = E[max_{i=1..N} r(y_i)]。Yuan等人（"Scaling Relationship on Learning Mathematical Reasoning with Large Language Models", 2023）在GSM8K上的实验表明，RFT(N=64)可将Llama-2-7B的准确率从基线26.3%提升至53.8%，而相同计算预算下的PPO仅提升至42.1%，证明RFT在可验证任务上的优越性。DeepSeek-R1的完整训练流程包含5个阶段：(1)冷启动SFT使用约5K高质量推理样本，(2)GRPO训练约8000步（生成温度0.7，G=16），(3)拒绝采样从RL模型生成约800K候选回答并筛选约100K高质量样本（筛选标准包括答案正确性、think标签存在、推理长度在200-8000字符之间、语言一致性），(4)SFT蒸馏训练1.5B/7B/14B/32B小模型，(5)对小模型再做一轮GRPO训练。迭代对齐的边际收益递减明显：第1轮通常带来最大提升（10-20个百分点），第2轮提升3-8个百分点，第3轮及以后趋于饱和。Constitutional AI的迭代自改进流程（Bai et al., 2022）使用约16条宪法原则，每轮SL阶段对约131K有害prompt进行"生成→批评→修正"循环（通常5轮修正），RL阶段使用AI排序的约135K偏好对训练RM，最终将有害输出率从约15%降至约1.5%。工程实现上，vLLM框架支持高效批量生成（4×A100上Llama-2-7B的吞吐约2000 tokens/s），配合tensor_parallel_size=4和n=32的采样配置，可在1小时内完成约10K prompt的拒绝采样流程。
