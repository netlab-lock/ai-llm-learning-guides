01 - 对齐训练全景图

1. 什么是对齐？为什么需要对齐？

大语言模型（LLM）通过海量文本预训练获得了强大的语言理解和生成能力，但预训练目标（下一个词预测）并不能保证模型的行为符合人类期望。对齐（Alignment）就是让模型的行为与人类的价值观、偏好和意图保持一致的过程。

1.1 基座模型 vs 对齐模型

对比维度基座模型（Base Model）对齐模型（Aligned Model）
指令遵循续写文本，不理解指令准确遵循用户指令
安全性可能生成有害、偏见内容拒绝有害请求，输出安全
有用性输出不可预测提供有帮助的回答
对话能力无法进行多轮对话支持流畅的多轮交互
格式控制格式不可控按要求输出结构化内容
关键洞察：基座模型像一个"知识渊博但不听话的学者"，对齐后变成"既有能力又愿意配合的助手"。GPT-4 Base → ChatGPT、Llama-3 Base → Llama-3-Instruct 的巨大差距主要来自对齐训练。

1.2 对齐税（Alignment Tax）

对齐训练在提升有用性和安全性的同时，可能会略微降低模型在某些基准测试上的表现，这被称为对齐税。原因包括：

  • 对齐过程中模型的部分知识可能被"遗忘"或压制
  • 安全约束会限制模型某些边缘能力的表达
  • KL 散度惩罚限制了策略偏离参考模型的程度
实践发现：随着对齐技术的进步，对齐税在持续降低。GRPO、DPO 等新方法在保持甚至提升推理能力的同时完成了对齐。DeepSeek-R1 就是通过 GRPO 训练在推理能力上实现了显著提升。
⚠️ 注意事项：关于「1. 什么是对齐？为什么需要对齐？」的常见陷阱：不要盲目照搬网上的配置参数。不同的数据分布和模型规模下，基座模型的最优配置可能完全不同。务必在自己的场景下做充分的验证实验。
🔍 深入理解：「1. 什么是对齐？为什么需要对齐？」的深层理解：对齐的本质是将人类的价值观——通常归纳为 3H 原则（Helpful、Harmless、Honest）——注入到预训练模型中。Ouyang 等人（InstructGPT, 2022）的实验表明，经过 RLHF 对齐的 1.3B 参数模型在人类评估中胜率高达 85%，击败了未对齐的 175B GPT-3。这说明对齐不是锦上添花，而是让模型能力真正可用的关键步骤。对齐税（Alignment Tax）通常在 1-5% 的基准性能范围内，但随着 DPO、GRPO 等新方法的出现，这一代价已被大幅压缩甚至反转——DeepSeek-R1 通过 GRPO 训练后，数学推理能力反而提升了 15-20%。
RLHF Pipeline (4个模型): Prompt → [Actor] → Response → [Reward Model] → Score ↓ ↓ [Critic] → Value PPO Update ← KL Penalty ↓ [Reference] → KL DPO Pipeline (2个模型): (Prompt, y_w, y_l) → [Policy] → log π(y_w), log π(y_l) ↓ [Reference] → log π_ref(y_w), log π_ref(y_l) ↓ DPO Loss (无需奖励模型) GRPO Pipeline (2个模型): Prompt → [Actor] × G次采样 → {y_1,...,y_G} ↓ [Rule-based Reward] → {r_1,...,r_G} ↓ 组内排名归一化 → 优势估计A_i ↓ GRPO Update (无需Critic)

2. 对齐训练的三个世代

RLHF训练三阶段流程 阶段1: SFT (监督微调) 高质量(指令, 回答)对 → 微调基座模型 → SFT模型 阶段2: RM (奖励模型训练) SFT模型生成多个回答 → 人类排序偏好 → 训练奖励模型 阶段3: PPO/GRPO (强化学习) SFT模型 + 奖励模型 → RL优化 → 对齐后模型 ┌─────────┐ ┌─────────┐ ┌─────────┐ │ SFT │ → │ RM │ → │ PPO │ │ (指令 │ │ (偏好 │ │ (强化 │ │ 微调) │ │ 学习) │ │ 学习) │ └─────────┘ └─────────┘ └─────────┘ 
# RLHF/DPO训练核心代码示例
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

# Step 1: 加载策略模型和参考模型
policy = AutoModelForCausalLM.from_pretrained("model-name")
ref_model = AutoModelForCausalLM.from_pretrained("model-name")
tokenizer = AutoTokenizer.from_pretrained("model-name")

# Step 2: DPO损失函数
def dpo_loss(policy_chosen_logps, policy_rejected_logps,
             ref_chosen_logps, ref_rejected_logps, beta=0.1):
    """DPO核心损失: 基于偏好对的直接优化"""
    chosen_rewards = beta * (policy_chosen_logps - ref_chosen_logps)
    rejected_rewards = beta * (policy_rejected_logps - ref_rejected_logps)
    loss = -torch.nn.functional.logsigmoid(chosen_rewards - rejected_rewards)
    return loss.mean()

# Step 3: 训练循环
for batch in dataloader:
    chosen_logps = get_log_probs(policy, batch["chosen"])
    rejected_logps = get_log_probs(policy, batch["rejected"])
    with torch.no_grad():
        ref_chosen = get_log_probs(ref_model, batch["chosen"])
        ref_rejected = get_log_probs(ref_model, batch["rejected"])
    loss = dpo_loss(chosen_logps, rejected_logps, ref_chosen, ref_rejected)
    loss.backward()
    optimizer.step()

2.1 第一代：监督微调（SFT）

最早也最直接的对齐方式。收集高质量的（指令，回答）对，直接用监督学习微调模型。

L_SFT = -E[log π_θ(y*|x)]，其中 (x, y*) 为高质量指令-回答对

优点：简单直接、训练稳定、计算成本低。缺点：依赖高质量标注数据、无法超越标注者水平、难以优化主观偏好维度。

典型数据来源：人工标注（如 OpenAI 的 InstructGPT 数据）、高质量开源数据集（如 ShareGPT、OpenAssistant）、GPT-4 生成的数据（如 Alpaca、Vicuna 的训练数据）。

2.2 第二代：RLHF（基于人类反馈的强化学习）

引入奖励模型（Reward Model）将人类偏好转化为标量奖励信号，再用 PPO 等强化学习算法优化策略模型。OpenAI 的 InstructGPT 是这一范式的代表。

J(θ) = E[r_φ(x,y)] - β·KL[π_θ(y|x) ‖ π_ref(y|x)]

流程：SFT → 训练奖励模型 → PPO 强化学习。优点：可超越标注者水平、有效利用人类偏好。缺点：训练复杂、需要同时维护4个模型（策略、参考、奖励、价值）、超参敏感。

RLHF 的核心难点在于训练不稳定性：PPO 的裁剪范围、KL 惩罚系数 β、奖励模型的准确性三者之间需要精细平衡。任何一个因素失调都可能导致训练崩溃或效果退化。

2.3 第三代：直接偏好优化（DPO/GRPO 及其变体）

跳过显式奖励模型训练，直接从偏好数据优化策略。DPO 将 RLHF 的奖励最大化问题转化为一个简单的分类损失。

L_DPO = -E[log σ(β·(log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)))]

GRPO 进一步去除价值模型，用组内相对奖励替代基线估计，大幅降低训练复杂度。这一代方法的共同特点是：更简洁的训练流程、更低的计算开销、更好的训练稳定性。

⚠️ 注意事项：关于「2. 对齐训练的三个世代」的常见陷阱：常见陷阱：忽略GPT的边界条件和异常情况。这些在小规模测试中可能不会暴露，但在生产环境中往往是最先出问题的地方。
🔍 深入理解：「2. 对齐训练的三个世代」的深层理解：GPT背后的核心权衡是灵活性与效率之间的平衡。更灵活的方案通常计算开销更大，更高效的方案往往需要更强的假设。在GRPO的设计中也能看到类似的权衡。

3. 对齐方法分类体系

3.1 基于强化学习的方法（RL-based）

  • PPO（Proximal Policy Optimization）：经典 RLHF 的核心算法，通过裁剪目标函数稳定训练。需要奖励模型和价值模型
  • GRPO（Group Relative Policy Optimization）：DeepSeek 提出，去除价值模型，用组内排序计算优势。在推理任务上表现卓越
  • REINFORCE / RLOO：更简洁的策略梯度变体，用留一法（Leave-One-Out）估计基线。计算效率介于 PPO 和 GRPO 之间

3.2 基于偏好的方法（Preference-based）

  • DPO（Direct Preference Optimization）：将 RL 目标转化为二分类损失，无需奖励模型。最广泛使用的偏好优化方法
  • KTO（Kahneman-Tversky Optimization）：基于前景理论，不需要配对数据，只需标记好/坏
  • SimPO（Simple Preference Optimization）：无需参考模型，使用长度归一化奖励，节省约 50% 显存
  • ORPO（Odds Ratio Preference Optimization）：将 SFT 和偏好优化合并为单阶段训练
  • IPO、cDPO、RSO：DPO 的各种改进变体，分别解决过拟合、数据噪声等问题

3.3 基于执行的方法（Execution-based）

  • RLEF（Reinforcement Learning from Execution Feedback）：利用代码执行、数学验证等自动化反馈作为奖励信号
  • RLVR（RL with Verifiable Rewards）：特别适合推理任务，用可验证结果作为奖励。DeepSeek-R1 的核心训练方法
方法演进趋势：从需要多个模型（PPO 需 4 个）→ 需要 2 个模型（DPO）→ 只需 1 个模型（SimPO/ORPO）。训练复杂度持续降低，效果不断提升。
⚠️ 注意事项：关于「3. 对齐方法分类体系」的常见陷阱：最常见的错误是跳过baseline直接上复杂方案。没有RLOO的baseline对比，你无法判断复杂方案是否真的带来了提升。
🔍 深入理解：「3. 对齐方法分类体系」的深层理解：从信息论的角度思考——理解RLOO需要信息的压缩与保留、信号与噪声的分离、冗余与效率的权衡，这些都是贯穿整个AI工程的核心主题。

5. 如何选择对齐方法？

5.1 决策流程

  1. 有配对偏好数据（chosen/rejected）？→ 优先 DPO，资源受限选 SimPO/ORPO
  2. 只有好/坏二元标签？→ 选 KTO
  3. 需要超越标注者水平？→ 选 PPO/GRPO（RL-based）
  4. 推理/代码等可验证任务？→ 选 RLVR
  5. 资源极其有限？→ SimPO 或 ORPO（无需参考模型）

5.2 业界趋势

当前主流趋势：多数团队采用 DPO/KTO 进行通用对齐，GRPO/RLVR 进行推理增强，两阶段结合使用。PPO 逐渐被更简洁的方法替代，但在某些场景（如需要在线探索）仍然不可替代。

🔍 深入理解：「5. 如何选择对齐方法？」的深层理解：工程哲学：在AI的实践中，简单但规模大往往胜过复杂但规模小。Chinchilla定律和Scaling Law都指向同一个方向——数据和计算的规模比精巧的算法设计更重要。

6. 对齐训练的通用原则

6.1 数据质量优先

无论使用哪种对齐方法，数据质量始终是第一优先级。高质量数据应具备以下特征：

  • 多样性：覆盖多种任务类型、难度级别和领域
  • 一致性：标注标准统一，不同标注者之间的一致率应 > 80%
  • 区分度：偏好数据中 chosen 和 rejected 应有明显质量差距
  • 时效性：定期更新数据以适应模型能力的提升

6.2 评估对齐效果

对齐效果的评估需要多维度指标：

评估维度评估方法典型指标
有用性MT-Bench、AlpacaEval胜率、评分
安全性红队测试、有害内容检测拒绝率、安全评分
指令遵循IFEval严格/宽松准确率
推理能力GSM8K、MATH、HumanEval准确率
💡 实用技巧：关于「6. 对齐训练的通用原则」的实用建议：好习惯：定期做checkpoint并保存实验配置。特别是在调整IFEval的参数时，记录每组参数的效果对比。这些记录在后续复盘时非常有价值。
⚠️ 注意事项：关于「6. 对齐训练的通用原则」的常见陷阱：注意：过度优化IFEval可能导致系统其他部分成为新的瓶颈。建议做端到端的性能分析。
🔍 深入理解：「6. 对齐训练的通用原则」的深层理解：IFEval的发展遵循AI领域的通用演进模式——从手工规则到数据驱动，从监督学习到自监督/强化学习，从单一模态到多模态融合。在MATH中也能观察到这个趋势。

RLHF训练深度 · 对齐训练全景图 · 第01章

深入探讨：

对齐训练的技术演进经历了从RLHF到DPO再到GRPO的三代范式转移。第一代RLHF（2022年，InstructGPT开创）需要同时维护4个模型（Actor、Critic、RM、Reference），训练成本高昂——以Llama-2-70B为例，完整的RLHF训练需要约1000张A100 GPU运行数周，成本超过百万美元。第二代DPO（2023年，Stanford提出）通过数学推导证明RLHF的最优解可以用闭式表达式直接从偏好数据学习，去掉了RM和Critic，将训练成本降低约60%。但DPO的局限在于需要精心构建的配对偏好数据，且容易在分布外泛化失败。第三代GRPO（2025年，DeepSeek提出）用组内相对排名替代绝对奖励评分，仅需规则可验证的奖励信号（如数学题的对错判断），在DeepSeek-R1上实现了从基座模型到推理模型的端到端训练，MATH基准准确率从15.6%跃升至71.0%，同时训练成本仅为PPO的1/3。当前前沿趋势是"对齐即训练"（Alignment as Training）——将对齐信号直接整合进预训练阶段，而非作为后处理步骤，如Llama-3的预训练阶段就融入了RLHF偏好信号，显著降低了对齐税。未来的对齐研究方向包括：基于形式化验证的安全保证（provable safety）、多模态对齐（视觉-语言模型的价值观对齐）、以及个性化对齐（让模型适应不同用户的价值偏好）。

✅ 参考答案与详解
  1. 核心原理: 对齐训练的本质是将人类价值观（3H原则：Helpful、Harmless、Honest）注入预训练模型中。基座模型通过下一个词预测获得语言能力，但输出不可控——可能生成有害内容、不遵循指令。对齐训练经历三代演进：第一代RLHF（SFT→RM→PPO）需要4个模型同时运行，PPO通过clipped surrogate objective优化策略，KL散度惩罚防止偏离参考模型太远；第二代DPO证明RM的最优解可用闭式表达，直接从偏好对学习，仅需2个模型；第三代GRPO用组内相对排名替代绝对奖励，DeepSeek-R1在MATH上从15.6%提升至71.0%。
    关键要点: ①对齐税（Alignment Tax）通常在1-5%，但GRPO/DPO已将其压缩甚至反转；②SFT解决格式对齐，RM编码人类偏好，RL在约束下搜索最优策略；③DeepSeek-R1证明纯RL可涌现CoT推理和自我验证能力；④对齐正从后处理阶段前移至预训练阶段（Llama-3）。
    实际案例: InstructGPT用1.3B参数经RLHF对齐后，在人类评估中85%胜率击败175B GPT-3，证明对齐不是锦上添花而是能力释放的关键步骤。DeepSeek-R1仅用GRPO+二值奖励就从基座模型端到端训练出推理模型。
    关联知识: 与强化学习中的策略梯度定理、Bradley-Terry偏好模型、KL约束优化紧密相关，后续章节的DPO、GRPO、RLVR都是对RLHF范式的不同简化路径。
  2. 核心原理: 三代对齐方法的核心区别在于奖励信号的来源和优化策略的复杂度。RLHF使用训练好的奖励模型提供连续分数，PPO在约束下优化；DPO将奖励隐式编码在策略概率比中，用简单的交叉熵式损失替代PPO；GRPO用规则奖励的组内归一化替代学习到的RM，优势估计A_i=(r_i-mean(r))/std(r)无需Critic。
    关键要点: ①RLHF灵活性最高但训练最不稳定（reward hacking），需同时加载4个模型约160GB显存（7B）；②DPO训练最稳定（类似SFT），显存降至约56GB，但需要精心构建的配对偏好数据；③GRPO在可验证任务上效果最佳，显存约85GB（2-3模型），训练吞吐量比PPO提升40%；④KTO进一步简化至单标签数据，性能达DPO的90-95%；⑤RLVR专用于有明确验证标准的任务（数学、代码）。
    实际案例: Llama-3-70B-Instruct使用PPO对齐，Qwen3系列使用DPO，DeepSeek-R1使用GRPO。工程选型上，数据充足配对好选DPO，有规则奖励选GRPO，只有单标签选KTO。
    关联知识: Bradley-Terry模型是RM和DPO共同的理论基础；GRPO的组内归一化本质上是方差缩减技术，与蒙特卡洛方法中的控制变量法思想相通。
  3. 核心原理: 设计实验验证对齐技术需要控制变量、选择合适评估指标和基线对比。以验证DPO vs SFT为例：使用相同基座模型，一组仅SFT训练，另一组SFT+DPO，在MT-Bench、AlpacaEval等自动化评估和人类偏好评估上对比。
    关键要点: ①实验设计需控制：相同基座模型、相同SFT数据、相同训练步数，唯一变量是对齐方法；②评估指标应多维：有用性（MT-Bench评分）、安全性（ToxiGen/RealToxicityPrompts）、指令遵循（IFEval）；③消融实验验证关键参数影响：DPO的β值（0.01~0.5）、GRPO的G值（4~64）、学习率等；④记录训练曲线：loss收敛、KL散度变化、奖励分数趋势。
    实际案例: 可复现DeepSeek-R1的实验——用Qwen2.5-7B-Base作为基座，在GSM8K数据集上分别运行SFT-only、DPO、GRPO三种配置，GRPO使用二值验证奖励，对比MATH-500和AIME准确率。预期GRPO> DPO > SFT-only，但需注意GRPO训练时间约为DPO的3倍。
    关联知识: 与实验设计中的A/B测试方法论、统计显著性检验（如bootstrap置信区间）相关；评估基准（MT-Bench、AlpacaEval、RewardBench）的选择直接影响结论可信度。
