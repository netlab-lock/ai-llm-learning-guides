02 RLHF 经典方法详解

InstructGPT 三阶段训练

OpenAI 的 InstructGPT（2022）开创了 RLHF 范式，至今仍是 ChatGPT、GPT-4 的核心训练方法。

阶段1: SFT (Supervised Fine-Tuning) ┌─────────────────────────────────────┐ │ 数据：人类标注的 (prompt, response) │ │ 目标：教模型"回答问题"的格式 │ │ 方法：标准交叉熵损失 │ │ 效果：模型学会遵循指令 │ └─────────────────────────────────────┘ ↓ 阶段2: RM (Reward Model) 训练 ┌─────────────────────────────────────┐ │ 数据：偏好对 (prompt, chosen, rejected)│ │ 目标：训练一个打分模型 │ │ 方法：Bradley-Terry 排序损失 │ │ 效果：模型能给回答打分 │ └─────────────────────────────────────┘ ↓ 阶段3: PPO 训练 ┌─────────────────────────────────────┐ │ 数据：prompt + 模型生成的回答 │ │ 模型：4 个同时运行 │ │ ┌─────────────────────────────┐ │ │ │ Actor (策略模型) ← 被训练 │ │ │ │ Critic (价值模型) ← 评估状态 │ │ │ │ RM (奖励模型) ← 给分 │ │ │ │ Ref (参考模型) ← KL 锚点 │ │ │ └─────────────────────────────┘ │ │ 目标：最大化奖励 - KL 惩罚 │ └─────────────────────────────────────┘
⚠️ 注意事项：关于「InstructGPT 三阶段训练」的常见陷阱：不要盲目照搬网上的配置参数。不同的数据分布和模型规模下，RLHF的最优配置可能完全不同。务必在自己的场景下做充分的验证实验。
🔍 深入理解：「InstructGPT 三阶段训练」的深层理解：InstructGPT 的三阶段设计（SFT → RM → PPO）并非随意拆分，而是解决了三个本质不同的问题：SFT 解决"格式对齐"（让模型学会对话格式），RM 解决"价值对齐"（将人类偏好编码为标量信号），PPO 解决"策略优化"（在约束下搜索最优策略）。InstructGPT 论文中的关键数据：RM 训练使用了约 33K 个比较对，PPO 训练时 β（KL 惩罚系数）设为 0.02，ε（PPO clip 范围）设为 0.2。有趣的是，RM 的准确率通常在 65-72% 之间就足以支撑有效的 PPO 训练——RM 不需要完美，只需要"比随机好足够多"。
RLHF Pipeline (4个模型): Prompt → [Actor] → Response → [Reward Model] → Score ↓ ↓ [Critic] → Value PPO Update ← KL Penalty ↓ [Reference] → KL DPO Pipeline (2个模型): (Prompt, y_w, y_l) → [Policy] → log π(y_w), log π(y_l) ↓ [Reference] → log π_ref(y_w), log π_ref(y_l) ↓ DPO Loss (无需奖励模型) GRPO Pipeline (2个模型): Prompt → [Actor] × G次采样 → {y_1,...,y_G} ↓ [Rule-based Reward] → {r_1,...,r_G} ↓ 组内排名归一化 → 优势估计A_i ↓ GRPO Update (无需Critic)

Bradley-Terry 模型

RM 的理论基础是 Bradley-Terry 偏好模型：给定两个回答，偏好更好的那个。

P(yw ≻ yl | x) = σ(r(x, yw) - r(x, yl))

损失函数：
LRM = -E(x, y_w, y_l) [log σ(r(x, yw) - r(x, yl))]

r(x, y) = RM(x, y) → 标量分数

RM 架构

class RewardModel(nn.Module):
    def __init__(self, base_model):
        super().__init__()
        self.model = base_model  # LLM backbone
        self.reward_head = nn.Linear(hidden_dim, 1)  # 标量输出

    def forward(self, input_ids, attention_mask):
        outputs = self.model(input_ids, attention_mask)
        last_hidden = outputs.last_hidden_state[:, -1, :]  # 最后一个 token
        reward = self.reward_head(last_hidden).squeeze(-1)
        return reward

# 训练循环
for prompt, chosen, rejected in dataloader:
    r_chosen = RM(concat(prompt, chosen))
    r_rejected = RM(concat(prompt, rejected))
    loss = -torch.log(torch.sigmoid(r_chosen - r_rejected)).mean()
    loss.backward()

PPO 训练详解

策略梯度基础

REINFORCE 梯度：
∇J(θ) = E[∇log πθ(a|s) · R(s,a)]

问题：方差大，训练不稳定
解决：优势函数 + Clipping

优势函数与 GAE

优势函数：A(s,a) = R(s,a) - V(s)
V(s) = Critic 对状态 s 的价值估计
A > 0 → 这个动作比平均好 → 增加概率
A < 0 → 这个动作比平均差 → 减少概率

GAE (Generalized Advantage Estimation)：
At = Σl=0∞ (γλ)l δt+l
δt = rt + γV(st+1) - V(st)
γ = 0.99（折扣因子），λ = 0.95（平滑因子）

PPO Clipping 机制

LPPO = E[min(rt(θ)·At, clip(rt(θ), 1-ε, 1+ε)·At)]

rt(θ) = πθ(a|s) / πold(a|s) ← 新旧策略比率
ε = 0.2 ← clip 范围

直觉：不让策略更新太大幅度，保持训练稳定
PPO Clipping 图示： 当 A > 0（好动作）： r 在 [1-ε, 1+ε] 内 → 用 r·A（正常更新） r > 1+ε → 用 (1+ε)·A（限制更新幅度） 当 A < 0（坏动作）： r 在 [1-ε, 1+ε] 内 → 用 r·A（正常更新） r < 1-ε → 用 (1-ε)·A（限制更新幅度）

KL 散度惩罚

总目标：L = LPPO - β · KL(πθ || πref)

KL 惩罚的作用：
1. 防止策略偏离参考模型太远（避免 reward hacking）
2. 保持模型的语言能力（防止退化）
3. β 通常从 0.01-0.1，可自适应调整

PPO 完整训练循环

for epoch in range(num_epochs):
    # 1. 采样：Actor 生成回答
    responses = actor.generate(prompts)

    # 2. 打分：RM 给每个回答评分
    rewards = reward_model(prompts, responses)

    # 3. 评估：Critic 估计状态价值
    values = critic(prompts, responses)

    # 4. 计算 KL 惩罚
    kl_penalty = beta * KL(actor || ref_model)

    # 5. 计算优势函数（GAE）
    advantages = compute_gae(rewards, values)

    # 6. PPO 更新（多步 mini-batch）
    for ppo_step in range(ppo_epochs):  # 通常 2-4 步
        for batch in create_mini_batches():
            ratio = actor_prob / old_actor_prob
            surr1 = ratio * advantages
            surr2 = torch.clamp(ratio, 1-eps, 1+eps) * advantages
            actor_loss = -torch.min(surr1, surr2).mean()
            critic_loss = F.mse_loss(predicted_values, returns)
            loss = actor_loss + 0.5 * critic_loss - kl_penalty
            loss.backward()
            optimizer.step()

PPO 的工程挑战

显存问题

💡 实用技巧：关于「PPO 的工程挑战」的实用建议：建议：不要一开始就追求完美方案，先跑通一个能工作的版本，再迭代优化。在GB的调优过程中，逐步调整比一次性大改更安全。
🔍 深入理解：「PPO 的工程挑战」的深层理解：从历史视角看，当前GB的主流方法经历了多年的演进。早期被认为过时的技术在大规模场景下可能重新焕发生机。保持对经典方法的了解很重要。

自适应 KL 系数

# 自适应 KL 系数（常用策略）
target_kl = 6.0  # 目标 KL 散度

if kl_divergence > 1.5 * target_kl:
    beta *= 2.0  # KL 太大，增大惩罚
elif kl_divergence < target_kl / 1.5:
    beta *= 0.5  # KL 太小，减小惩罚

# 限制 beta 在合理范围
beta = max(0.001, min(beta, 10.0))
通俗类比：RLHF 就像"训练宠物"——你不会告诉宠物"坐下"的物理原理（神经网络参数），而是当它做对了就给零食（奖励），做错了就摇头（惩罚）。奖励模型就像你内心的"好/坏判断标准"，PPO 就是宠物根据你的反馈调整行为的过程。KL 惩罚就像"不要太离谱"——你可以学新花样，但不能忘记基本规矩。
深入探讨：RLHF 的一个深层问题是"奖励黑客"（reward hacking）——模型可能学会"讨好"奖励模型，而不是真正变得有帮助。例如，模型可能学会生成冗长但空洞的回答，因为奖励模型倾向于给长回答高分。这引出了一个根本性问题：我们能否设计出"不可被黑客"的奖励模型？目前的解决方案包括：① 多奖励模型集成 ② 对抗训练 ③ 人工定期校准。但这个问题本质上是"Goodhart 定律"的体现——当一个指标成为目标时，它就不再是好的指标。
⚠️ 注意事项：关于「自适应 KL 系数」的常见陷阱：隐蔽的陷阱是评估数据泄漏。确保Goodhart的评估数据完全没有出现在训练集中。即使部分重叠也会导致评估结果严重失真。

RLHF的经典实现需要深入理解PPO算法在语言模型训练中的适配细节。PPO的核心是clipped surrogate objective：L_CLIP = E[min(r_t(θ)A_t, clip(r_t(θ), 1-ε, 1+ε)A_t)]，其中r_t(θ)是新旧策略的概率比，ε通常设为0.2。在LLM场景下，A_t（优势函数）由Critic Model估计，Critic的训练目标是最小化TD-error的均方损失。InstructGPT论文报告的关键超参包括：β（KL惩罚系数）= 0.02，学习率 = 1e-6（Adam优化器），batch_size = 64，每步PPO更新仅做1次梯度更新（避免过度优化）。训练稳定性方面，PPO容易出现"奖励黑客"（reward hacking）——模型发现RM的漏洞生成高分但无意义的输出。Gao等人(2023)发现reward score与KL散度呈近似线性关系：reward约等于根号KL，即RM的过优化遵循幂律缩放。对策包括：(1)增大RM规模（更大的RM更难被hack），(2)定期用新策略生成的数据更新RM，(3)使用多个RM的集成（ensemble）作为更鲁棒的奖励信号。TRL（Transformer Reinforcement Learning）库提供了开箱即用的PPO训练实现，支持Llama、Qwen等主流模型架构，配合DeepSpeed ZeRO-3可在8×A100上完成7B模型的完整RLHF训练，约需4-8小时。

✅ 参考答案与详解
  1. 核心原理: RLHF（Reinforcement Learning from Human Feedback）由InstructGPT开创，包含三个阶段：SFT阶段用高质量(指令,回答)对微调基座模型，教会模型对话格式；RM阶段基于Bradley-Terry偏好模型训练奖励模型，损失函数L_RM=-E[log σ(r_chosen-r_rejected)]，通常需要50K-500K偏好对；PPO阶段同时运行4个模型（Actor策略模型、Critic价值模型、RM奖励模型、Ref参考模型），通过clipped surrogate objective L_PPO=E[min(r_t·A_t, clip(r_t,1-ε,1+ε)·A_t)]优化策略，ε=0.2防止策略更新幅度过大。优势函数A(s,a)=R(s,a)-V(s)由Critic估计，使用GAE平滑：A_t=Σ(γλ)^l·δ_{t+l}。
    关键要点: ①RM准确率仅需65-72%即可支撑有效PPO训练；②PPO的reward hacking问题：reward≈√KL，RM过优化遵循幂律缩放；③对策包括增大RM规模、定期更新RM、RM集成；④InstructGPT超参：β=0.02, lr=1e-6, batch=64, 每步仅1次梯度更新。
    实际案例: OpenAI使用约33K人类排序数据训练6B RM，对175B策略模型评分。Gao等人(2023)发现reward score与KL散度呈近似线性关系，RM规模达策略模型1/10至1/3即可胜任。Llama-2使用超过100万偏好对训练RM。
    关联知识: PPO源自Schulman等人(2017)的策略梯度方法，Bradley-Terry模型是经典的配对比较统计模型。与DPO的核心区别在于DPO证明了RM可用策略模型的闭式解表示，从而跳过RM训练。
  2. 核心原理: RLHF的核心组件对比——RM（奖励模型）架构通常在基座模型最后token隐藏状态后接线性层投影到标量。PPO的训练需要精心管理4个模型的交互：Actor生成回答→RM打分→Critic估计状态价值→PPO计算优势和更新策略→KL惩罚约束偏离。DPO则通过数学推导将这4步简化为2步：直接从偏好对中隐式学习奖励函数。
    关键要点: ①RM架构：LLM backbone + Linear(hidden_dim→1)标量头，最后一个token的隐藏状态作为整体表示；②PPO工程难点：4模型显存管理（7B约160GB）、Critic与Actor的训练节奏协调、KL系数β的动态调整；③DPO优势：显存降约65%（7B约56GB）、训练稳定性高（无PPO方差问题）、实现简单；④DPO局限：需要配对偏好数据、分布外泛化可能失败。
    实际案例: ChatGPT/GPT-4使用RLHF(PPO)训练，开源社区的HuggingFace Zephyr使用DPO训练。PPO适合有充足计算资源的大厂，DPO适合中小团队快速迭代。
    关联知识: PPO的GAE（Generalized Advantage Estimation）与TD-learning的关系；RM的Bradley-Terry损失与逻辑回归的等价性；DPO的闭式解推导基于KL约束优化的拉格朗日对偶。
  3. 核心原理: 验证RLHF关键技术的实验设计应覆盖三阶段各环节。以验证RM质量对PPO效果的影响为例：固定SFT模型和PPO配置，变量为RM规模（1B/3B/6B）和RM训练数据量（10K/50K/200K偏好对），评估指标包括reward score分布、KL散度曲线、最终模型的MT-Bench评分。
    关键要点: ①RM质量实验：不同数据量训练RM→评估pairwise accuracy→用各RM跑PPO→比较最终模型质量，验证RM accuracy与下游效果的关系；②PPO稳定性实验：监控reward score vs KL散度曲线，观察是否出现reward hacking（reward飙升但质量下降）；③超参敏感性实验：β(0.01/0.02/0.05)、ε(0.1/0.2/0.3)、lr(5e-7/1e-6/2e-6)的网格搜索；④数据效率实验：对比RLAIF（AI标注）vs人工标注RM的训练效果差距。
    实际案例: 可在Qwen2.5-7B上复现：先SFT（5K指令数据）→训练3个不同规模RM（1B/3B/7B）→各自PPO训练500步→在AlpacaEval上对比。预期7B RM > 3B RM > 1B RM，但差距可能仅2-5个百分点。
    关联知识: 与超参优化（Optuna/Ray Tune）、分布式训练（DeepSpeed ZeRO-3）、实验追踪（WandB）等工程实践密切相关。
