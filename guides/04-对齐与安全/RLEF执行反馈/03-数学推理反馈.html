1. RLVR (Reinforcement Learning with Verifiable Rewards)

1. RLVR (Reinforcement Learning with Verifiable Rewards)的核心组件与工作原理

DeepSeek-R1 的核心方法：用数学答案正确性作为可验证的奖励信号。

1.1 RLVR 的核心思想

RLVR 训练流程: 问题: "计算 17 × 23" │ ▼ 模型生成 K 个推理轨迹: 轨迹 1: "17 × 23 = 17 × 20 + 17 × 3 = 340 + 51 = 391" 轨迹 2: "17 × 23 = 17 × 25 - 17 × 2 = 425 - 34 = 391" 轨迹 3: "17 × 23 = 390" (错误) 轨迹 4: "17 × 23 = 17 × 23 = 391" (正确但步骤少) │ ▼ 验证答案正确性: 轨迹 1: 391 = 391 → 正确 → 奖励 = 1.0 轨迹 2: 391 = 391 → 正确 → 奖励 = 1.0 轨迹 3: 390 ≠ 391 → 错误 → 奖励 = 0.0 轨迹 4: 391 = 391 → 正确 → 奖励 = 1.0 │ ▼ GRPO 更新: 强化正确轨迹，弱化错误轨迹

1.2 奖励函数设计

# RLVR 奖励函数
def rlvr_reward(question, generated_answer, correct_answer):
    """可验证的数学答案奖励"""
    # 1. 提取最终答案
    extracted = extract_final_answer(generated_answer)

    # 2. 验证正确性
    if is_equivalent(extracted, correct_answer):
        return 1.0  # 正确
    else:
        return 0.0  # 错误

def extract_final_answer(text):
    """从推理文本中提取最终答案"""
    # 方法 1: 提取 "答案是 X" 或 "= X"
    patterns = [
        r'答案是\s*[：:]?\s*(.+)',
        r'=\s*(.+?)(?:\s*$|\s*\n)',
        r'\\boxed\{(.+?)\}',
    ]
    for pattern in patterns:
        match = re.search(pattern, text)
        if match:
            return match.group(1).strip()
    return text.split('\n')[-1].strip()  # 最后一行

def is_equivalent(a, b):
    """判断两个答案是否等价"""
    # 数值比较
    try:
        return abs(float(a) - float(b)) < 1e-6
    except:
        pass
    # 符号比较
    return a.strip() == b.strip()
通俗类比： RLHF训练就像训练导盲犬——第一步(SFT)教它基本技能，第二步(奖励模型)让它学习人类对"好行为"的偏好，第三步(PPO)通过反复实践强化正确行为。奖励模型就是那个"打分的训练师"。
常见误区： RLHF最容易出现"奖励黑客"(Reward Hacking)——模型学会了讨好评分器而非真正变好。比如模型发现加"让我为您详细解答"这种套话能提高奖励分，就疯狂使用。对策是定期更新奖励模型和KL约束。
深入探讨： DPO的核心推导从RLHF的最优策略出发。RLHF目标：max_π E[r(x,y)] - β·KL(π||π_ref)。其闭式解为π*(y|x) = π_ref(y|x)·exp(r(x,y)/β)/Z(x)。反解奖励：r(x,y) = β·log(π*(y|x)/π_ref(y|x)) + β·log(Z(x))。代入Bradley-Terry偏好模型P(y_w≻y_l|x) = σ(r(y_w)-r(y_l))，Z(x)被消去，得到DPO损失：L = -E[log σ(β·(log π(y_w)/π_ref(y_w) - log π(y_l)/π_ref(y_l)))]。

2. PRM (Process Reward Model)

2. PRM (Process Reward Model)的核心组件与工作原理

2.1 ORM vs PRM

ORM (Outcome Reward Model): 整个推理过程 → 最终答案 → 奖励 只看结果，不看过程 问题: 如果最终答案碰巧对了但推理过程有误 ORM 会给高奖励 → 强化错误推理 PRM (Process Reward Model): Step 1 → 奖励 ✓ (这步正确) Step 2 → 奖励 ✓ (这步正确) Step 3 → 奖励 ✗ (这步错误!) Step 4 → 奖励 ? (后续步骤可能弥补) 优势: 能精确定位推理过程中的错误步骤 挑战: 需要步骤级标注数据

2.2 PRM 训练

# PRM 训练数据格式
{
    "question": "计算 17 × 23",
    "steps": [
        {"text": "17 × 23 = 17 × 20 + 17 × 3", "correct": true},

# PRM 模型: 对每个步骤预测正确性
class ProcessRewardModel(nn.Module):
    def __init__(self, base_model):
        super().__init__()
        self.model = base_model
        self.reward_head = nn.Linear(hidden_dim, 1)

    def forward(self, question, steps):
        rewards = []
        for step in steps:
            hidden = self.model.encode(question + step)
            reward = torch.sigmoid(self.reward_head(hidden))
            rewards.append(reward)
        return rewards  # 每步一个奖励

5. DeepSeek-R1 的数学训练

5. DeepSeek-R1 的数学训练的训练流程与优化策略

5. DeepSeek-R1 的数学训练的训练流程通常分为三个阶段：预训练(Pre-training)在大规模无标注语料上学习语言表示，微调(Fine-tuning)在任务特定数据上适配下游任务，对齐(Alignment)通过人类偏好反馈优化输出质量。每个阶段的数据配比、学习率调度和正则化策略都需要精心设计。

DeepSeek-R1 数学训练流程: 基础模型: DeepSeek-V3 │ ▼ GRPO 训练: 奖励: 数学答案正确性 (二值) 数据: GSM8K + MATH + 合成数据 采样: K=64 个响应/问题 │ ▼ 结果: - AIME 2024: 79.8% (匹配 o1) - 自发学会 Chain-of-Thought - 自发学会自我验证 - 自发学会 "等一下，让我重新想想"
💡 通俗类比：ORM vs PRM 就像老师批改数学考试：ORM 只看最终答案对不对（结果奖励），PRM 则会检查每一步推导过程（过程奖励）。一个学生可能答案对了但过程有误（碰巧猜对），ORM 会给满分，PRM 会指出中间步骤的错误。这就是为什么 PRM 更适合训练"会思考"的模型。
⚠️ 常见误区：① 认为 RLVR 的二值奖励太简单无法训练——DeepSeek-R1 证明，即使只有对/错两种奖励，配合 GRPO 和大量采样（K=64），模型也能涌现出复杂的推理能力；② 忽略答案提取的难度——数学答案格式多样（分数、根号、表达式），提取错误会导致正确答案被误判为错误，严重影响训练效果。
🔬 深入探讨：Math-Shepherd 的蒙特卡洛标注方法揭示了一个有趣的统计现象：如果一个中间步骤是"正确的"，那么从该步骤出发继续推理，得到正确最终答案的概率应该显著高于随机水平。这本质上是在用"条件概率"来评估步骤质量。但这种方法有偏差：如果模型本身能力较弱，即使中间步骤正确，后续推理也可能失败，导致正确步骤被误标为错误。这就是为什么需要多次采样（通常 10-100 次）来降低方差。

数学答案验证的形式化方法与多格式解析

RLVR（Reinforcement Learning with Verifiable Rewards）训练的核心环节是数学答案的自动验证，而这一看似简单的"对错判断"在实际工程中充满挑战。数学答案的表达形式极为多样：同一结果"391"可以写成"391"、"17×23"、"√152881"、"3.91×10²"、"三百九十一"等多种形式，简单的字符串匹配完全无法胜任。生产级的答案验证器需要实现"多格式归一化+等价性判定"的两步流程。第一步格式归一化包括：(1) 数值标准化——将科学计数法、分数、根号等统一转换为标准数值格式，使用 Python 的 `sympy` 库进行符号化简；(2) LaTeX 解析——从 `\boxed{391}`、`$391$`、`\frac{391}{1}` 等 LaTeX 格式中提取数值；(3) 中文数字转换——将"三百九十一"转换为 391；(4) 单位处理——识别并标准化"391个"、"391.0"、"约391"等带有修饰词的答案。第二步等价性判定包括：数值等价（`abs(a-b) < 1e-6`）、符号等价（使用 sympy 的 `simplify(a-b) == 0`）、以及集合等价（对于有多组解的问题，判断两个集合是否相同）。DeepSeek-R1 的训练实践表明，答案验证器的准确率直接影响训练效果——将验证器准确率从 92% 提升到 98% 后，模型在 AIME 基准上的最终准确率提升了约 5 个百分点。这是因为验证器的误判会引入噪声奖励信号，误导模型的优化方向。特别需要注意的是"格式惩罚"问题——如果模型给出了正确的推理过程和正确的数值答案，但答案格式与标准答案不一致（如标准答案写"391"而模型写"17×23=391"），验证器不应将其判为错误，否则会惩罚模型的"详细解答"行为。最佳实践是使用"答案提取+归一化+比较"的三级流水线，确保格式差异不影响正确性判定。

蒙特卡洛采样步骤标注的偏差校正与方差缩减

Math-Shepherd 提出的蒙特卡洛采样步骤标注方法虽然优雅，但存在两个统计偏差问题需要工程上的校正。第一个偏差是"模型能力偏差"（Model Capability Bias）：如果用于采样的模型本身能力较弱，即使中间步骤完全正确，后续推理也可能失败，导致正确步骤被误标为错误。例如，一个 7B 模型在采样时可能因为语言理解能力不足而误解题意，导致后续推理全部偏离——但这不是中间步骤的错，而是模型本身的局限。校正方法是使用"能力基线"：对每个步骤，不仅计算"从该步骤出发得到正确答案的条件概率 P(correct | step_i)"，还计算"从该步骤出发的无条件正确率 P(correct)"，两者的比值 P(correct | step_i) / P(correct) 才是该步骤的"真实贡献度"。第二个偏差是"采样方差"（Sampling Variance）：当采样次数 K 不够大时，蒙特卡洛估计的方差会很高。例如，如果某步骤的真实正确概率是 0.7，但只采样 5 次，有 10.8% 的概率全部失败（0.3^5），导致该步骤被错误标记。方差缩减的工程策略包括：(1) 分层采样（Stratified Sampling）——将温度参数分层（如 T=0.3, 0.7, 1.0），在每个温度下独立采样，确保探索的多样性；(2) 控制变量法（Control Variates）——利用已知正确步骤的采样结果作为控制变量，降低估计方差；(3) 重要性采样（Importance Sampling）——对模型不确定度高的步骤分配更多采样次数。实验表明，结合这三种方差缩减技术后，步骤标注的准确率从 78% 提升至 89%，接近人工标注的 93% 水平，而采样成本仅增加约 40%（从 10 次增加到 14 次）。

# 数学答案验证器完整实现
import re
import sympy
from typing import Optional, Tuple

class MathAnswerVerifier:
    """生产级数学答案验证器"""

    def __init__(self):
        # LaTeX 答案提取模式
        self.boxed_pattern = re.compile(r'\\boxed\{([^}]+)\}')
        self.dollar_pattern = re.compile(r'\$\s*(.+?)\s*\$')
        self.equals_pattern = re.compile(r'=\s*(.+?)(?:\s*$|\s*\\|\s*\n)')

    def extract_answer(self, text: str) -> Optional[str]:
        """从推理文本中提取最终答案"""
        # 优先级: boxed > 最后一个等号 > 最后一行
        for pattern in [self.boxed_pattern, self.dollar_pattern, self.equals_pattern]:
            matches = pattern.findall(text)
            if matches:
                return matches[-1].strip()

        # 回退: 取最后一行非空文本
        lines = [l.strip() for l in text.strip().split('\n') if l.strip()]
        return lines[-1] if lines else None

    def normalize(self, answer: str) -> Optional[float]:
        """将答案归一化为数值"""
        answer = answer.strip().rstrip('.')

        # 移除单位和修饰词
        answer = re.sub(r'[个只条元美元￥]', '', answer)
        answer = re.sub(r'^(约|大概|approximately|about)\s*', '', answer, flags=re.I)

        # 尝试直接数值解析
        try:
            return float(answer)
        except ValueError:
            pass

        # 尝试 sympy 符号计算
        try:
            expr = sympy.sympify(answer)
            if expr.is_number:
                return float(expr)
        except (sympy.SympifyError, TypeError):
            pass

        return None

    def is_equivalent(
        self, generated: str, reference: str, tolerance: float = 1e-6
    ) -> Tuple[bool, str]:
        """判断生成答案与参考答案是否等价"""
        gen_norm = self.normalize(generated)
        ref_norm = self.normalize(reference)

        # 数值等价
        if gen_norm is not None and ref_norm is not None:
            if abs(gen_norm - ref_norm) < tolerance:
                return True, "numeric_match"

        # 符号等价 (使用 sympy)
        try:
            gen_expr = sympy.sympify(generated)
            ref_expr = sympy.sympify(reference)
            if sympy.simplify(gen_expr - ref_expr) == 0:
                return True, "symbolic_match"
        except:
            pass

        # 集合等价 (处理多解问题)
        try:
            gen_set = set(sympy.sympify(generated.replace('{', '').replace('}', '')))
            ref_set = set(sympy.sympify(reference.replace('{', '').replace('}', '')))
            if gen_set == ref_set:
                return True, "set_match"
        except:
            pass

        return False, "no_match"

    def compute_reward(
        self, question: str, generated_text: str, reference_answer: str
    ) -> float:
        """计算 RLVR 奖励"""
        extracted = self.extract_answer(generated_text)
        if extracted is None:
            return 0.0  # 无法提取答案

        equivalent, method = self.is_equivalent(extracted, reference_answer)
        if equivalent:
            return 1.0  # 正确
        else:
            return 0.0  # 错误
