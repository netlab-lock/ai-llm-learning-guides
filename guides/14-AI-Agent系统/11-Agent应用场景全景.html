🌍 模块11：Agent应用场景全景

编程、科研、客服、游戏、企业、机器人——Agent正在改变每个领域

1. 背景与动机

Agent不是学术概念——它正在各个行业落地。根据MIT 2025 AI Agent Index，截至2025年已有30+个主要Agent系统在6个类别中投入生产使用。本模块将全面介绍Agent的主要应用场景。

2. 编程Agent

编程是Agent最成熟的应用领域，因为代码有明确的正确性标准（测试通过/失败），非常适合Agent的迭代改进模式。

2.3 编程Agent的内部架构

# 编程Agent的核心循环 (伪代码)
class CodingAgent:
    def __init__(self, repo_path):
        self.repo = Repo(repo_path)
        self.tools = {{
            "read_file": ReadFileTool(),
            "edit_file": EditFileTool(),
            "run_tests": RunTestsTool(),
            "search_code": SearchCodeTool(),
            "execute": ExecuteCodeTool(),
        }}

    def solve_issue(self, issue_description):
        # Step 1: 理解Issue
        context = self.tools["search_code"].run(issue_description)
        plan = self.llm.plan(f"Issue: {{issue_description}}\nContext: {{context}}")

        # Step 2: 迭代修复 (核心!)
        for attempt in range(5):
            # 2a. 生成修复代码
            patch = self.llm.generate_fix(plan, context)

            # 2b. 应用修改
            self.tools["edit_file"].run(patch)

            # 2c. 运行测试验证
            test_result = self.tools["run_tests"].run()

            if test_result.passed:
                return {{"status": "success", "attempts": attempt + 1}}

            # 2d. 分析失败原因，调整策略
            context += f"\n尝试{{attempt+1}}失败: {{test_result.error}}"
            plan = self.llm.replan(plan, test_result.error)

        return {{"status": "failed", "attempts": 5}}
🔬 编程Agent的技术突破：2025年编程Agent的核心突破是"测试驱动的自我修正"（Test-Driven Self-Correction）。Agent不再一次性生成所有代码，而是：(1) 生成代码 (2) 运行测试 (3) 分析失败原因 (4) 修复代码 (5) 重复直到通过。SWE-bench的Top Agent都采用了这种模式。另一个突破是"代码库理解"——Agent能索引整个代码库的AST（抽象语法树），理解函数调用关系、类型系统和依赖图，而不仅仅是逐文件处理。

🔬 企业Agent的ROI分析：根据McKinsey 2025年报告，企业Agent的投资回报率差异巨大。(1) 客服Agent: ROI最高，平均节省40-60%的客服成本，部署周期2-4周。(2) 数据分析Agent: ROI中等，节省分析师30%的重复工作，但需要数据治理基础。(3) 流程自动化Agent: ROI取决于流程标准化程度，标准化流程ROI可达5-10x，非标准化流程ROI可能为负。关键教训：先标准化流程，再部署Agent。

8. Agent应用场景选型指南

┌─────────────────────────────────────────────────────────────────┐ │ Agent应用选型决策树 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 你的场景有什么特点? │ │ │ │ │ ├─→ 任务有明确的成功标准? (如测试通过/答案正确) │ │ │ └─→ 是 → 编程Agent、问答Agent │ │ │ │ │ ├─→ 需要调用多个外部API/工具? │ │ │ └─→ 是 → 工具增强Agent │ │ │ │ │ ├─→ 需要处理大量文档/数据? │ │ │ └─→ 是 → RAG Agent、数据分析Agent │ │ │ │ │ ├─→ 需要多人协作完成? │ │ │ └─→ 是 → 多Agent系统 │ │ │ │ │ ├─→ 需要与物理世界交互? │ │ │ └─→ 是 → 机器人Agent、IoT Agent │ │ │ │ │ └─→ 需要持续学习和改进? │ │ └─→ 是 → 自进化Agent │ │ │ └─────────────────────────────────────────────────────────────────┘
💡 落地建议：Agent项目成功的关键不是技术选型，而是场景选择。选择一个：(1)有明确价值 (2)容错率高 (3)数据充足 (4)用户愿意接受AI辅助的场景。从MVP开始，逐步扩展Agent的自主程度。
⚠️ 常见失败原因：Agent项目失败的三大原因：(1)场景选择错误——选了容错率低的场景（如医疗诊断） (2)期望过高——期望Agent100%自主完成 (3)缺乏评估——没有定义清晰的成功指标。

# 选择一个简单场景，用最少代码构建Agent
# 场景: 自动回答技术问题（带搜索能力）

from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain.tools import Tool
from langchain_core.prompts import ChatPromptTemplate

# 1. 定义工具
def search_web(query: str) -> str:
    # 接入搜索API
    return f"Search results for: {{query}}"

tools = [Tool(name="search", func=search_web, description="搜索互联网")]

# 2. 定义Prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个技术问答助手。遇到不确定的问题，先搜索再回答。"),
    ("human", "{{input}}"),
    ("placeholder", "{{agent_scratchpad}}"),
])

# 3. 创建Agent
llm = ChatOpenAI(model="gpt-4o", temperature=0)
agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 4. 测试
result = executor.invoke({{"input": "Python 3.14有什么新特性?"}})
print(result["output"])

解题思路: 构建第一个Agent的核心是：选场景(简单明确) → 定义工具(搜索+计算) → 创建Prompt(系统指令) → 组装Agent(create_tool_calling_agent) → 运行(AgentExecutor)。从最简单的ReAct模式开始，逐步添加记忆和评估。

# 完整可运行的第一个Agent (带搜索能力的技术问答助手)
# pip install langchain langchain-openai langchain-community

from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain.tools import Tool
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain.memory import ConversationBufferMemory

# ── 1. 定义工具 ──
def search_knowledge(query: str) -> str:
    """模拟知识库搜索 (生产中接入RAG或搜索API)"""
    kb = {
        "python": "Python是一种解释型、面向对象的高级编程语言，由Guido van Rossum于1991年创建。",
        "agent": "AI Agent是以LLM为核心推理引擎，具备感知、推理、行动、记忆、规划能力的自主系统。",
        "rag": "RAG(Retrieval-Augmented Generation)通过检索外部知识库增强LLM的回答质量，减少幻觉。",
    }
    for key, val in kb.items():
        if key in query.lower():
            return val
    return f"未找到关于'{query}'的信息，建议用更具体的关键词搜索。"

def calculator(expression: str) -> str:
    """安全的数学计算"""
    try:
        return str(eval(expression, {"__builtins__": {}}))
    except: return "计算错误"

tools = [
    Tool(name="knowledge_search", func=search_knowledge, description="搜索技术知识库，输入搜索关键词"),
    Tool(name="calculator", func=calculator, description="数学计算，输入数学表达式"),
]

# ── 2. 定义Prompt ──
prompt = ChatPromptTemplate.from_messages([
    ("system", """你是一个技术问答助手。
规则：
1. 遇到不确定的技术问题，先搜索知识库再回答
2. 涉及数学计算时使用calculator工具
3. 回答要简洁准确，不确定时说明
4. 引用知识库内容时注明来源"""),
    ("human", "{input}"),
    MessagesPlaceholder(variable_name="chat_history", optional=True),
    ("placeholder", "{agent_scratchpad}"),
])

# ── 3. 创建Agent (带记忆) ──
llm = ChatOpenAI(model="gpt-4o", temperature=0)
memory = ConversationBufferMemory(memory_key="chat_history", return_messages=True)
agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True, memory=memory)

# ── 4. 测试 ──
tests = [
    "什么是AI Agent?",
    "RAG技术解决什么问题?",
    "计算 128 * 256 + 1024",
]
for q in tests:
    print(f"\n{'='*50}\nQ: {q}")
    result = executor.invoke({"input": q})
    print(f"A: {result['output']}")

关键知识点: (1) create_tool_calling_agent是2026年推荐的Agent创建方式(替代旧的create_react_agent) (2) MessagesPlaceholder用于注入对话历史实现记忆 (3) 工具描述(description)的质量直接影响LLM的工具选择准确率 (4) 从简单场景开始(3-5个工具)，验证可行性后再扩展 (5) 生产化路径：添加记忆→添加评估→添加安全防护→优化成本。
