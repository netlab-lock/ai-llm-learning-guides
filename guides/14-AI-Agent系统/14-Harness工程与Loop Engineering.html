⚙️ 模块14：Harness工程与Loop Engineering 2026新概念

从Prompt Engineering到Harness Engineering——Agent治理层的工程化

🔬 概念演变：2026年Agent工程的核心变化是从Prompt Engineering（提示工程）演进为Harness Engineering（治理工程）。Prompt只是Agent的一小部分；Harness是包裹LLM的完整治理层——包括上下文管理、工具编排、预算控制、终止条件、轨迹评估。这是2026年Agent岗位的核心考察点。

1. 从Prompt到Harness的演进

2. Harness的七大组件

2.1 上下文管理

Agent的上下文窗口是稀缺资源。一个复杂任务可能产生数万tokens的对话历史、工具结果和中间状态。

策略方法适用场景
滑动窗口只保留最近N轮对话简单对话Agent
摘要压缩用LLM将旧对话压缩为摘要长对话Agent
选择性保留只保留关键信息（工具结果、决策点）复杂任务Agent
分层存储热数据在上下文，冷数据在向量DB长期运行Agent

2.2 工具编排

Harness控制哪些工具可用、何时调用、如何处理结果：

# Harness层的工具编排
class ToolHarness:
    def __init__(self):
        self.tools = {{"search": SearchTool(), "code": CodeTool()}}
        self.call_count = 0
        self.max_calls = 20
        self.budget = TokenBudget(max_tokens=50000)

    def execute(self, tool_name, args):
        # 1. 权限检查
        if tool_name not in self.tools:
            return {{"error": "Tool not available"}}
        
        # 2. 预算检查
        if self.call_count >= self.max_calls:
            return {{"error": "Max tool calls reached"}}
        
        # 3. 参数验证
        if not self.validate_args(tool_name, args):
            return {{"error": "Invalid arguments"}}
        
        # 4. 执行
        result = self.tools[tool_name].run(args)
        self.call_count += 1
        
        # 5. 结果截断 (防止超长结果填满上下文)
        return self.truncate_result(result, max_tokens=2000)

2.4 终止条件

Agent必须有明确的终止条件，否则可能无限循环：

终止条件说明
目标达成Agent判断任务已完成（LLM自行判断）
最大步数超过N步自动终止
重复检测连续3步行动相同 → 可能陷入循环
质量阈值输出质量低于阈值 → 终止并降级
人类干预请求人类确认是否继续

2.7 安全防护

Harness层的安全防护比Prompt层更全面：

  • 输入防护：检测prompt injection、敏感信息
  • 行动防护：检查工具调用的权限和参数
  • 输出防护：检查输出是否包含敏感信息
  • 行为防护：检测异常行为模式（如突然偏离目标）

3. Loop Engineering：Agent循环工程化

Agent的核心是一个循环（Loop）。Loop Engineering关注如何让这个循环更可靠、更高效。

3.3 完整Harness实现示例

class AgentHarness:
    """Agent治理层核心实现"""
    def __init__(self, config):
        self.max_steps = config.get("max_steps", 20)
        self.max_tokens = config.get("max_tokens", 50000)
        self.timeout = config.get("timeout", 300)
        self.allowed_tools = config.get("allowed_tools", [])
        self.step_count = 0
        self.total_tokens = 0
        self.history = []

    def should_terminate(self, state):
        """终止条件检查"""
        # 1. 最大步数
        if self.step_count >= self.max_steps:
            return True, "max_steps_reached"
        # 2. Token预算
        if self.total_tokens >= self.max_tokens:
            return True, "token_budget_exceeded"
        # 3. 重复检测 (连续3步相同行动)
        if len(self.history) >= 3:
            last_3 = [h["action"] for h in self.history[-3:]]
            if len(set(str(a) for a in last_3)) == 1:
                return True, "repeated_action_loop"
        # 4. 目标达成 (LLM自行判断)
        if state.get("goal_achieved"):
            return True, "goal_achieved"
        return False, None

    def validate_tool_call(self, tool_name, args):
        """工具调用验证"""
        if tool_name not in self.allowed_tools:
            return False, f"Tool '{tool_name}' not in allowed list"
        # 参数大小限制
        if len(str(args)) > 10000:
            return False, "Arguments too large"
        return True, None

    def compress_context(self, messages, max_tokens=4000):
        """上下文压缩: 保留最近N轮 + 摘要旧历史"""
        if self.count_tokens(messages) <= max_tokens:
            return messages
        # 分离: 旧历史 + 最近对话
        recent = messages[-6:]  # 保留最近3轮
        old = messages[:-6]
        # 用LLM压缩旧历史
        summary = self.llm_summarize(old)
        return [{{"role": "system", "content": f"历史摘要: {{summary}}"}}] + recent

    def run(self, agent, task):
        """Harness主循环"""
        state = {{"task": task, "messages": []}}
        while True:
            # 检查终止条件
            should_stop, reason = self.should_terminate(state)
            if should_stop:
                return {{"result": state.get("output"), "reason": reason,
                         "steps": self.step_count, "tokens": self.total_tokens}}
            # 执行一步
            action = agent.step(state)
            # 验证工具调用
            if action.type == "tool_call":
                valid, msg = self.validate_tool_call(action.tool, action.args)
                if not valid:
                    state["messages"].append({{"role": "system", "content": f"工具调用被拒绝: {{msg}}"}})
                    continue
            # 执行并记录
            result = action.execute()
            self.step_count += 1
            self.total_tokens += result.tokens
            self.history.append({{"action": action, "result": result}})
            state["messages"].extend([action.to_message(), result.to_message()])

✅ 参考答案与详解
  1. 核心原理: Prompt Engineering关注的是如何写好系统提示词和用户提示词来引导LLM的输出，它只管理Agent的"输入端"。而Harness Engineering是对整个Agent系统的治理层设计，包括上下文管理、工具编排、预算控制、终止条件、轨迹评估、错误恢复和安全防护七大组件。2026年Agent工程转向Harness的根本原因是：随着Agent从简单对话走向复杂任务执行，单纯优化Prompt已经无法解决Agent在生产环境中的可靠性问题——上下文溢出、工具调用失控、无限循环、安全风险等系统级问题需要一个完整的治理框架来应对。
    关键要点: (1) Prompt只管输入，Harness管整个生命周期 (2) Harness是系统级治理，Prompt是技巧级优化 (3) 生产环境可靠性需要系统化治理 (4) Harness将Agent从"demo可用"提升到"生产可靠"
    实际案例: 一个客服Agent如果有优秀的Prompt但缺乏Harness，可能在用户反复追问时上下文溢出导致崩溃；而有了Harness的摘要压缩+预算控制+终止条件三重保障，Agent能在达到限制时优雅降级并转接人工。
  2. 核心原理: "LLM自行判断目标达成"存在风险是因为LLM的判断本身可能出错——Agent可能"以为自己完成了"但实际上遗漏了关键步骤，或者在早期步骤中产生了幻觉导致错误地认为任务已完成。更可靠的混合终止策略应包含多层检查：(1)硬性约束层：最大步数、最大token、最大时间 (2)软性判断层：LLM自评+独立评估模型双重验证 (3)语义匹配层：将最终输出与期望目标做语义相似度计算 (4)规则验证层：对可量化的任务（如代码执行、数据提取）用程序化验证结果正确性。
    关键要点: (1) LLM自评可能有误判 (2) 需要硬性约束作为兜底 (3) 多层验证提高可靠性 (4) 规则验证适用于结构化任务
    实际案例: 代码生成Agent声称"已完成"，但实际代码缺少异常处理。通过添加测试用例执行作为规则验证层，只有所有测试通过的代码才被接受为"任务完成"。
  3. 核心原理: 多Agent错误隔离机制的核心是"故障域隔离"——将错误限制在发生错误的Agent及其直接影响范围内。具体设计包括：(1)错误传播阻断：当Agent A调用Agent B的服务时，如果B返回错误，A的Harness立即捕获该错误并决定是重试、降级还是换策略，而不是将错误向上传播 (2)级联保护：设置全局错误计数器，当系统中任意Agent的错误率超过阈值时触发全局降级 (3)超时机制：Agent间调用设置超时，超时即视为失败 (4)降级路径：每个Agent在协作链路中都有预定义的降级方案 (5)隔离墙：Agent间通过消息队列传递结果，错误结果被拦截在队列消费者层面。
    关键要点: (1) 错误捕获在第一跳 (2) 设置全局错误率阈值 (3) 超时=失败原则 (4) 每个节点有降级预案 (5) 消息队列作为缓冲隔离
    实际案例: 在一个"研究→分析→写作"的三Agent流水线中，如果"分析"Agent的工具调用失败，"研究"Agent的Harness捕获到该错误后，尝试切换到备用分析策略，而不是让错误传递到"写作"Agent。
  4. 核心原理: 可量化的轨迹评分算法框架设计如下：设总分为S=w1×A+w2×E+w3×Sa+w4×P，其中A为目标对齐度(0~1)，E为效率得分(0~1)，Sa为安全得分(0~1)，P为进度得分(0~1)，w1+w2+w3+w4=1。目标对齐度A通过将当前行动描述与原始任务目标做语义相似度计算；效率得分E=1-(已用步数/最大步数)×(已用token/最大token)；安全得分Sa通过规则引擎检查当前行动是否触发安全规则；进度得分P通过对比已完成子任务数/总子任务数。当S<阈值时触发干预（如强制换方向或请求人类确认）。
    关键要点: (1) 四维度加权评分 (2) 每步实时计算 (3) 阈值触发干预机制 (4) 权重可根据任务类型动态调整
    实际案例: 数据分析Agent第8步时，A=0.8(仍在分析数据)，E=0.6(已用8/20步)，Sa=1.0(无安全风险)，P=0.5(5/10子任务完成)，S=0.4×0.8+0.2×0.6+0.2×1.0+0.2×0.5=0.74，低于0.8阈值，系统提示Agent加速或重新规划路径。

        self.action_hashes = []  # 用于重复检测
        self.error_counts = {}  # 各工具错误计数
        self.intervention_log = []
    
    def run(self, agent, task):
        """主循环 - 治理下的Agent执行"""
        self.start_time = time.time()
        state = {"task": task, "messages": [], "output": None, "goal_achieved": False}
        
        while True:
            # 1. 检查终止条件
            should_stop, reason = self.should_terminate(state)
            if should_stop:
                return {
                    "result": state.get("output"),
                    "reason": reason,
                    "steps": self.step_count,
                    "tokens": self.total_tokens,
                    "duration": time.time() - self.start_time,
                    "interventions": self.intervention_log
                }
            
            # 2. 执行Agent一步
            try:
                action = agent.step(state)
            except Exception as e:
                recovery = self.handle_error(e, state)
                if recovery["action"] == "continue":
                    continue
                elif recovery["action"] == "retry":
                    action = agent.step(state)
                else:  # terminate
                    return {"result": None, "reason": f"unrecoverable_error: {e}",
                            "steps": self.step_count, "tokens": self.total_tokens}
            
            # 3. 验证工具调用
            if action.get("type") == "tool_call":
                valid, msg = self.validate_tool_call(action.get("tool"), action.get("args"))
                if not valid:
                    state["messages"].append({
                        "role": "system",
                        "content": f"工具调用被拒绝: {msg}"
                    })
                    self.intervention_log.append({
                        "step": self.step_count, "type": "tool_rejected", "detail": msg
                    })
                    continue
            
            # 4. 执行行动
            result = self.execute_action(action)
            if result.get("error"):
                recovery = self.handle_error(result["error"], state)
                if recovery["action"] != "continue":
                    continue
            
            # 5. 更新状态
            self.step_count += 1
            self.total_tokens += result.get("tokens", 0)
            self.history.append({"action": action, "result": result})
            action_hash = hashlib.md5(str(action).encode()).hexdigest()
            self.action_hashes.append(action_hash)
            state["messages"].extend([
                {"role": "assistant", "content": str(action)},
                {"role": "system", "content": str(result)}
            ])
            
            # 6. 上下文管理
            if self.count_tokens(state["messages"]) > self.context_limit:
                state["messages"] = self.compress_context(state["messages"])
            
            # 7. 轨迹评估
            score = self.evaluate_trajectory(action, state)
            if score < 0.5:
                self.intervention_log.append({
                    "step": self.step_count, "type": "low_score",
                    "score": score, "action": "注入目标提醒"
                })
                state["messages"].append({
                    "role": "system",
                    "content": f"[Harness提醒] 原始任务: {state['task']}。请评估当前行动是否有效推进目标。"
                })
            
            # 8. 更新状态
            if result.get("output"):
                state["output"] = result["output"]
            if result.get("goal_achieved"):
                state["goal_achieved"] = True
    
    def should_terminate(self, state):
        """多维度终止检查"""
        # 1. 最大步数
        if self.step_count >= self.max_steps:
            return True, "max_steps_reached"
        # 2. Token预算
        if self.total_tokens >= self.max_tokens:
            return True, "token_budget_exceeded"
        # 3. 超时
        if time.time() - self.start_time > self.timeout:
            return True, "timeout"
        # 4. 重复检测 (连续4步相同)
        if len(self.action_hashes) >= 4:
            last_4 = self.action_hashes[-4:]
            if len(set(last_4)) == 1:
                return True, "repeated_action_loop"
        # 5. 错误率过高
        if self.step_count > 5:
            total_errors = sum(self.error_counts.values())
            if total_errors / self.step_count > 0.5:
                return True, "error_rate_too_high"
        # 6. 目标达成
        if state.get("goal_achieved"):
            return True, "goal_achieved"
        return False, None
    
    def compress_context(self, messages):
        """上下文压缩 - 保留最近对话 + 压缩旧历史"""
        if self.count_tokens(messages) <= self.context_limit:
            return messages
        
        # 保留最近6条消息(3轮对话)
        recent = messages[-6:]
        old = messages[:-6]
        
        if not old:
            return recent
        
        # 压缩旧历史为摘要（实际项目用LLM生成，这里简化为截断）
        old_text = "\n".join(str(m.get("content", "")) for m in old)
        summary = old_text[:int(len(old_text) * self.compression_ratio)] + "...[已压缩]"
        
        return [
            {"role": "system", "content": f"历史对话摘要: {summary}"}
        ] + recent
    
    def evaluate_trajectory(self, action, state):
        """轨迹评估 - 四维度加权评分"""
        w = self.eval_weights
        
        # 目标对齐度 (简化实现: 检查action中是否包含任务关键词)
        task_keywords = set(state["task"].lower().split()[:5])
        action_text = str(action).lower()
        alignment = len(task_keywords & set(action_text.split())) / max(len(task_keywords), 1)
        alignment = min(alignment * 2, 1.0)  # 缩放
        
        # 效率得分
        step_eff = 1.0 - (self.step_count / self.max_steps)
        token_eff = 1.0 - (self.total_tokens / self.max_tokens)
        efficiency = (step_eff + token_eff) / 2
        
        # 安全得分 (默认1.0, 检测到危险行动时降低)
        safety = 1.0
        dangerous_keywords = ["delete", "drop", "remove", "destroy", "rm -rf"]
        if any(kw in action_text for kw in dangerous_keywords):
            safety = 0.3
        
        # 进度得分 (根据步骤和历史的启发式估计)
        progress = min(self.step_count / (self.max_steps * 0.7), 1.0)
        if state.get("output"):
            progress = max(progress, 0.8)
        
        score = (w["alignment"] * alignment + 
                 w["efficiency"] * efficiency + 
                 w["safety"] * safety + 
                 w["progress"] * progress)
        return round(score, 3)
    
    def handle_error(self, error, context):
        """错误恢复 - 分级处理策略"""
        error_str = str(error)
        
        # 记录错误
        tool_name = context.get("last_tool", "unknown")
        self.error_counts[tool_name] = self.error_counts.get(tool_name, 0) + 1
        
        # 分级恢复
        if "rate_limit" in error_str:
            # 限流: 等待后重试
            time.sleep(2)
            return {"action": "retry", "delay": 2}
        elif "timeout" in error_str:
            # 超时: 降级到缓存
            context["messages"].append({
                "role": "system",
                "content": "[Harness] 请求超时，尝试使用缓存结果或简化参数重试"
            })
            return {"action": "continue"}
        elif "not_found" in error_str or "invalid" in error_str:
            # 无效请求: 不重试，让Agent换策略
            context["messages"].append({
                "role": "system", 
                "content": f"[Harness] 请求无效: {error_str}。请尝试其他方法。"
            })
            return {"action": "continue"}
        else:
            # 未知错误: 如果错误率可控则继续，否则终止
            if sum(self.error_counts.values()) > self.step_count * 0.5:
                return {"action": "terminate"}
            return {"action": "continue"}
    
    def validate_tool_call(self, tool_name, args):
        """工具调用验证"""
        if tool_name not in self.allowed_tools:
            return False, f"Tool '{tool_name}' not in allowed list"
        if len(str(args)) > 10000:
            return False, "Arguments too large (>10KB)"
        return True, None
    
    def count_tokens(self, messages):
        """简易token估算 (实际项目用tiktoken)"""
        total_chars = sum(len(str(m.get("content", ""))) for m in messages)
        return total_chars // 4  # 粗略估算
    
    def execute_action(self, action):
        """执行行动 (模拟)"""
        return {"tokens": 100, "output": None, "error": None, "goal_achieved": False}
