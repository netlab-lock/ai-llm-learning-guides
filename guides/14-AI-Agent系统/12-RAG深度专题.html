2.2 相似度计算

cosine_sim(A, B) = (A · B) / (|A| × |B|) 范围: [-1, 1]
L2_distance(A, B) = √(Σ(aᵢ - bᵢ)²) 越小越相似
inner_product(A, B) = Σ(aᵢ × bᵢ) 越大越相似(归一化后等价cosine)
💡 面试回答：当面试官问"为什么用cosine而不是L2"时，回答：cosine相似度只关注方向不关注 magnitude（向量长度），对文本长度不敏感。两个意思相同但长度不同的文本，cosine相似度高，L2距离可能大。实际应用中，归一化后的向量用内积等价于cosine，且内积计算更快。

2.3 ANN近似最近邻搜索

精确的最近邻搜索O(n·d)太慢，生产中使用ANN算法：

算法原理速度召回率代表系统
HNSW分层可导航小世界图极快95-99%FAISS, Qdrant, Milvus
IVF倒排索引+聚类快90-95%FAISS
PQ乘积量化压缩极快85-90%FAISS
SCANNGoogle开源，各向异性量化极快95%+Google ScaNN
🔬 HNSW原理：HNSW（Hierarchical Navigable Small World）构建一个多层图结构。顶层是稀疏的"高速公路"，快速定位大致区域；底层是密集的精确搜索层。搜索时从顶层开始，逐层下降，每层找到最近的节点作为下一层的起点。这类似于地图导航：先在高速公路上开到目标城市附近，再走城市道路找到具体地址。时间复杂度O(log n)，空间复杂度O(n·M)（M是每节点连接数）。

3. 混合检索 面试高频

纯向量检索的弱点：关键词匹配差。"Python 3.12新特性"可能检索到"Python教程"而不是"3.12特性"。混合检索结合Dense和Sparse的优势。

3.1 Dense vs Sparse

维度Dense (向量)Sparse (BM25)
原理语义相似度词频-逆文档频率
优势理解同义词、语义精确关键词匹配
劣势可能漏掉精确匹配不理解同义词
示例"开心"≈"高兴""Python 3.12"精确匹配

3.2 融合策略：RRF

RRF_score(d) = Σ 1/(k + rank_i(d)) 其中 k=60 (常数)

RRF（Reciprocal Rank Fusion）将Dense和Sparse的排名融合。k=60是经验值，避免高排名文档 dominate。

# 混合检索实现 (LangChain)
from langchain.retrievers import EnsembleRetriever
from langchain_community.retrievers import BM25Retriever
from langchain_community.vectorstores import Chroma

# Dense检索
vectorstore = Chroma(embedding_function=embeddings)
dense_retriever = vectorstore.as_retriever(search_kwargs={{"k": 20}})

# Sparse检索
bm25_retriever = BM25Retriever.from_texts(texts, k=20)

# RRF融合
ensemble = EnsembleRetriever(
    retrievers=[dense_retriever, bm25_retriever],
    weights=[0.6, 0.4]  # Dense权重更高
)

4. Rerank重排序 面试高频

Rerank是RAG质量提升的关键环节。Embedding检索是Bi-encoder（query和doc分别编码），Rerank是Cross-encoder（query和doc一起编码），精度更高但更慢。

┌─────────────────────────────────────────────────────────────────┐ │ Bi-encoder vs Cross-encoder │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ Bi-encoder (Embedding检索): │ │ query → [Encoder] → vec_q │ │ doc → [Encoder] → vec_d │ │ score = cosine(vec_q, vec_d) │ │ 特点: 快 (可预计算doc向量), 精度中等 │ │ │ │ Cross-encoder (Rerank): │ │ [query, doc] → [Encoder] → score │ │ 特点: 慢 (每次都要重新计算), 精度高 │ │ │ │ 生产策略: │ │ 1. Bi-encoder快速召回 Top-50 │ │ 2. Cross-encoder精排 Top-5 │ │ 3. 平衡速度和精度 │ │ │ └─────────────────────────────────────────────────────────────────┘
Rerank模型提供商特点
Cohere RerankCohereAPI服务，多语言，效果最好
bge-reranker-v2-m3BAAI开源，中文优秀，可本地部署
jina-reranker-v2Jina开源，支持长文本

5. GraphRAG 面试进阶

GraphRAG（Microsoft, 2024）将知识图谱引入RAG，解决传统RAG无法处理多跳推理和全局总结的问题。

┌─────────────────────────────────────────────────────────────────┐ │ 传统RAG vs GraphRAG │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 传统RAG: │ │ 查询: "公司CEO的母校在哪个城市?" │ │ 检索: 找到"CEO毕业于清华" → 但没有"清华在北京" │ │ 结果: 无法回答 ✗ │ │ │ │ GraphRAG: │ │ 知识图谱: │ │ [CEO] --毕业于--> [清华大学] --位于--> [北京] │ │ 查询: 沿图谱路径 [CEO→清华→北京] │ │ 结果: "清华大学，位于北京" ✓ │ │ │ │ GraphRAG的核心创新: │ │ 1. 社区检测: 将图谱分为社区，每个社区生成摘要 │ │ 2. 层级检索: 先定位社区，再在社区内检索 │ │ 3. 全局查询: "这个数据集的主要主题是什么?" │ │ → 检索所有社区摘要，综合回答 │ │ │ └─────────────────────────────────────────────────────────────────┘

6. Agentic RAG 面试前沿

Agentic RAG让Agent自主决定何时检索、检索什么、如何使用检索结果，而非固定流程。

模式说明优势
主动检索Agent判断是否需要检索，不总是检索减少不必要的检索开销
多源检索Agent同时检索文档、数据库、API、网页信息更全面
迭代检索第一次结果不满意，Agent调整query重新检索提高检索质量
验证检索Agent对检索结果做事实验证减少幻觉

7. RAG幻觉防控

幻觉类型原因解决方案

8.2 RAG常见踩坑与解决方案

坑现象解决方案
文档解析丢信息PDF表格/图片内容丢失用marker-pdf或Unstructured做智能解析
Chunk边界切断一个论点被切成两半增加overlap(200 tokens)，按语义边界切分
Embedding不准语义相似但检索不到换Embedding模型，或用HyDE改写query
检索到不相关结果生成被干扰加Rerank精排+相关性阈值过滤
生成不引用来源回答无法溯源Prompt中要求"必须引用来源编号"
知识库更新不及时回答过时信息增量更新索引，加时间戳过滤

8.3 RAG降级策略

生产环境必须有降级方案：

  • 检索失败：降级到纯LLM回答 + 声明"以下为通用建议"
  • Rerank超时：跳过Rerank，直接用Embedding检索结果
  • 向量DB不可用：降级到BM25关键词检索
  • LLM不可用：返回检索结果的原文摘要

9. RAG评估指标

指标衡量什么计算方法
Context Relevance检索结果与问题的相关性LLM评估或人工标注
Faithfulness回答是否忠实于检索结果逐句检查是否有证据支持
Answer Relevance回答是否切题LLM评估回答与问题的匹配度
Recall@KTop-K中包含正确答案的比例有标注数据时计算
MRR正确答案的排名1/rank的平均值

# 参考回答要点:
# 1. 离线层: 文档处理→分块→Embedding→向量DB
# 2. 在线层: Query改写→混合检索→Rerank→生成
# 3. 评估层: 自动评估(Faithfulness/Relevance)+人工抽检
# 4. 运维层: 索引更新策略、缓存、降级、监控
# 5. 关键选型: Embedding模型、向量DB、Rerank模型、LLM
▶显示答案
💡 参考答案与详解

解题思路: 生产级RAG系统需要考虑全链路7个环节的技术选型和优化。回答时应分层说明：离线层(文档→Chunk→Embed→索引)、在线层(Query改写→检索→Rerank→生成)、评估层(Faithfulness/Relevance)、运维层(缓存/降级/监控)。

# 生产级RAG系统架构设计 (面试回答模板)
#
# ┌──────────────────── 离线层 ────────────────────┐
# │  文档处理(PDF/Word/HTML → 纯文本)               │
# │     ↓                                           │
# │  分块(RecursiveCharacterTextSplitter, 800/200)  │
# │     ↓                                           │
# │  Embedding(bge-large-zh-v1.5 中文最优)          │
# │     ↓                                           │
# │  向量DB(Milvus分布式/Qdrant/Chroma原型)         │
# └─────────────────────────────────────────────────┘
#
# ┌──────────────────── 在线层 ────────────────────┐
# │  Query改写(LLM改写模糊query + HyDE)            │
# │     ↓                                           │
# │  混合检索(Dense向量 + BM25 + RRF融合)           │
# │     ↓                                           │
# │  Rerank(bge-reranker-v2-m3, Top50→Top5)        │
# │     ↓                                           │
# │  生成(Prompt: "只基于context回答，引用来源")      │
# └─────────────────────────────────────────────────┘

from langchain.retrievers import EnsembleRetriever, ContextualCompressionRetriever
from langchain_community.retrievers import BM25Retriever
from langchain_community.vectorstores import Chroma
from langchain.retrievers.document_compressors import CrossEncoderReranker
from langchain_community.cross_encoders import HuggingFaceCrossEncoder

# 1. 混合检索 (Dense + BM25 + RRF)
dense = vectorstore.as_retriever(search_kwargs={"k": 20})
bm25 = BM25Retriever.from_texts(texts, k=20)
hybrid = EnsembleRetriever(retrievers=[dense, bm25], weights=[0.6, 0.4])

# 2. Rerank精排 (Top-50 → Top-5)
reranker = CrossEncoderReranker(
    model=HuggingFaceCrossEncoder(model_name="BAAI/bge-reranker-v2-m3"),
    top_n=5
)
reranked = ContextualCompressionRetriever(base_compressor=reranker, base_retriever=hybrid)

# 3. 生成 (带Grounding约束)
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("""
基于以下参考资料回答问题。如果资料不足，请说"我没有找到相关信息"。
回答时必须引用来源编号。

参考资料:
{context}

问题: {question}
""")
llm = ChatOpenAI(model="gpt-4o", temperature=0)

关键知识点: (1) 混合检索(Dense+BM25+RRF)比纯向量检索召回率高15-25% (2) Rerank用Cross-encoder精排，精度远高于Bi-encoder但更慢，所以先粗排再精排 (3) Prompt约束"只基于context回答"是减少幻觉的最简单有效方法 (4) 必须有降级策略：检索失败→纯LLM、Rerank超时→跳过、向量DB不可用→BM25 (5) 评估用Faithfulness+Context Relevance+Answer Relevance三指标。

10. RAG面试高频追问汇总

追问回答要点
为什么用cosine不用L2?cosine只关注方向不关注magnitude，对文本长度不敏感
BM25的原理?TF-IDF的改进版，考虑词频饱和和文档长度归一化
RRF的k值怎么选?k=60是经验值，避免高排名dominate，实际可网格搜索
Rerank为什么比Embedding准?Cross-encoder联合编码query-doc，Bi-encoder分别编码
GraphRAG的社区检测?用Leiden算法将知识图谱分为社区，每个社区生成摘要
RAG vs 长上下文?长上下文解决"放得下"，RAG解决"找得到"，两者互补
怎么评估RAG效果?Context Relevance(检索质量) + Faithfulness(忠实度) + Answer Relevance(切题度)
RAG的延迟怎么优化?Embedding缓存 + 检索结果缓存 + 异步Rerank + 流式生成
