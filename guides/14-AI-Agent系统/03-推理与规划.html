🧠 模块03：推理与规划

Chain-of-Thought、Tree-of-Thought、Graph-of-Thought——Agent的"思考"能力

1. 背景与动机

推理与规划是Agent最核心的能力。没有推理，Agent就是一个"随机行动器"；没有规划，Agent就无法处理复杂任务。2022年以来，一系列推理技术的突破直接推动了Agent的实用化。

2. Chain-of-Thought (CoT) 链式思维

CoT（Wei et al., 2022）是推理技术的基石。核心思想：让LLM在给出答案前，先展示推理过程。

2.1 为什么CoT有效？

┌─────────────────────────────────────────────────────────────────┐ │ CoT 的工作原理 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 无CoT (直接回答): │ │ Q: "一个农夫有17只羊，除了9只都死了，还剩几只?" │ │ A: "8只" ✗ │ │ (模型被"除了...都死了"误导，做了减法) │ │ │ │ 有CoT (逐步推理): │ │ Q: "一个农夫有17只羊，除了9只都死了，还剩几只?" │ │ A: "让我逐步分析: │ │ 1. 题目说'除了9只都死了' │ │ 2. '除了'意思是'not'，即不是全部死了 │ │ 3. 所以有9只没有死 │ │ 4. 答案是9只" ✓ │ │ │ │ 关键: 显式的推理步骤暴露了中间逻辑，减少了歧义 │ │ │ └─────────────────────────────────────────────────────────────────┘

2.2 CoT的变体

变体方法论文效果
Few-shot CoT在prompt中提供推理示例Wei et al., 2022GSM8K: 56.6%→74.4%
Zero-shot CoT添加"Let's think step by step"Kojima et al., 2022MultiArith: 17.7%→78.7%
Auto-CoT自动生成推理示例Zhang et al., 2022与手动Few-shot相当

💡 实践建议："Let's think step by step"是性价比最高的prompt技巧。只需在问题末尾加上这句话，就能显著提升推理准确率。但注意：对于简单任务（如翻译、分类），CoT反而可能降低性能——因为增加了不必要的复杂性。

3. Self-Consistency 自一致性

Self-Consistency（Wang et al., 2022）是对CoT的重要改进。核心思想：多次采样不同的推理路径，取多数投票的结果。

4. Tree-of-Thought (ToT) 树状思维

ToT（Yao et al., 2023）将推理过程建模为一棵搜索树，每个节点是一个"思考步骤"，通过BFS或DFS探索最优路径。

┌─────────────────────────────────────────────────────────────────┐ │ Tree-of-Thought 示意图 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 任务: "24点游戏: 用 4, 5, 6, 10 算出 24" │ │ │ │ [4,5,6,10] │ │ / | \ │ │ / | \ │ │ [4+5=9] [4×5=20] [10-4=6] │ │ [6,10] [6,10] [5,6] │ │ / \ | | │ │ / \ | | │ │ [9+6=15] [9×6=54] [20+6=26] [6×5=30] │ │ [10] [10] [10] [6] │ │ | | | | │ │ 15+10=25 54+10=64 26+10=36 30+6=36 │ │ ✗ 失败 ✗ 失败 ✗ 失败 ✗ 失败 │ │ │ │ ← 回溯到 [4×5=20], [6,10] │ │ [20-6=14] │ │ [10] │ │ 14+10=24 ✓ 成功! │ │ │ │ 关键: LLM同时充当"生成器"(产生思考)和"评估器"(评判可行性) │ │ │ └─────────────────────────────────────────────────────────────────┘

ToT的搜索策略

策略特点适用场景
BFS (广度优先)逐层探索，保留最优b个节点解空间较浅，需要全局视野
DFS (深度优先)深入探索单条路径，失败时回溯解空间较深，有明确的评估信号

5. Least-to-Most Prompting 由简到难

Least-to-Most（Zhou et al., 2022）是另一种重要的推理策略。核心思想：将复杂问题分解为一系列由简到难的子问题，逐步解决。

┌─────────────────────────────────────────────────────────────────┐ │ Least-to-Most Prompting │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 问题: "计算 (3x² + 2x)(x - 1) 在 x=2 时的值" │ │ │ │ CoT方式 (直接推理): │ │ "展开: 3x³ - 3x² + 2x² - 2x = 3x³ - x² - 2x │ │ 代入x=2: 3(8) - 4 - 4 = 24 - 8 = 16" │ │ 问题: 一步到位，容易出错 │ │ │ │ Least-to-Most方式 (逐步递进): │ │ 子问题1 (最简): "x=2时, x²=? x³=?" │ │ → x²=4, x³=8 │ │ 子问题2 (中等): "3x³=? -x²=? -2x=?" │ │ → 24, -4, -4 │ │ 子问题3 (最终): "24 + (-4) + (-4) = ?" │ │ → 16 │ │ 优势: 每步都简单，累积错误少 │ │ │ │ 与CoT的区别: │ │ • CoT: 一条推理链，中间可能有复杂步骤 │ │ • Least-to-Most: 先解决简单子问题，再组合 │ │ • 适用: 多步复合问题、需要前置知识的推理 │ │ │ └─────────────────────────────────────────────────────────────────┘

6. GoT：Graph-of-Thought 图状思维

GoT（Besta et al., 2024）进一步将推理从树扩展为有向无环图（DAG），允许不同推理分支之间的合并和交互。

┌─────────────────────────────────────────────────────────────────┐ │ Graph-of-Thought vs ToT vs CoT │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ CoT (链): A → B → C → D → Answer │ │ │ │ ToT (树): A │ │ / \ │ │ B C │ │ / \ | │ │ D E F │ │ \ / │ │ G ← 分支独立，无法合并 │ │ │ │ GoT (图): A │ │ / \ │ │ B C │ │ / \ / \ │ │ D E-F G ← E和F可以合并! │ │ \ / \ / │ │ H I ← 多个分支的结果可以聚合 │ │ │ │ GoT的核心创新: 允许思维节点之间的合并、聚合、精炼 │ │ │ └─────────────────────────────────────────────────────────────────┘

GoT的操作类型

操作说明示例
生成 (Generate)从一个节点生成多个后续节点一个想法→多个方案
聚合 (Aggregate)将多个节点合并为一个多个方案→综合方案
精炼 (Refine)改进一个节点的内容初稿→修改稿
分割 (Split)将一个节点分解为多个复杂问题→子问题

6. Agent中的规划能力

6.1 任务分解 (Task Decomposition)

复杂任务需要分解为可管理的子任务。常见的分解策略：

策略方法适用场景
顺序分解将任务分为有序步骤 [S1→S2→S3]线性流程（如：先搜索→再分析→最后写报告）
层次分解将大任务递归分解为子树复杂项目（如：软件开发→前端+后端+测试）
并行分解识别可并行执行的子任务独立子任务（如：同时搜索多个主题）
条件分解根据条件分支执行不同路径决策任务（如：如果数据量大用Spark，否则用Pandas）

6.2 自适应规划

现代Agent的规划不是静态的，而是根据执行结果动态调整：

┌─────────────────────────────────────────────────────────────────┐ │ 自适应规划流程 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 初始计划: [S1, S2, S3, S4, S5] │ │ │ │ 执行 S1 → 成功 ✓ │ │ 执行 S2 → 失败 ✗ │ │ │ │ │ ▼ │ │ 重规划: 分析失败原因 │ │ 新计划: [S1✓, S2', S3', S4, S5] (调整了S2和S3) │ │ │ │ │ ▼ │ │ 执行 S2' → 成功 ✓ │ │ 执行 S3' → 成功 ✓ │ │ 执行 S4 → 成功 ✓ │ │ 执行 S5 → 成功 ✓ │ │ │ │ │ ▼ │ │ 任务完成! │ │ │ └─────────────────────────────────────────────────────────────────┘
🔬 CoT为什么有效？——信息论视角：从信息论角度，CoT的有效性可以用"信息瓶颈"理论解释。直接回答需要LLM在单次前向传播中完成所有推理，信息瓶颈很窄。CoT通过生成中间步骤，将推理过程"展开"为多个步骤，每一步的信息瓶颈更宽。实验表明：对于需要3步以上推理的任务，CoT的准确率提升最为显著（从~20%提升到~70%）。对于1-2步的简单任务，CoT反而可能降低性能（因为增加了不必要的噪声）。

7. 综合对比

技术推理结构探索能力成本适用场景
CoT线性链低（单路径）1x简单推理、数学
Self-Consistency多条链+投票中（多路径采样）Nx有明确答案的任务
ToT树高（分支+回溯）高创意任务、博弈
GoT图（DAG）最高（合并+聚合）最高复杂综合推理
ReAct链+行动低中工具调用任务
LATS树+行动高高需要探索的决策
⚠️ 实际应用：在生产环境中，大多数Agent使用简单的CoT或ReAct。ToT/GoT主要用于研究和需要深度推理的特定场景（如数学证明、博弈策略）。不要为了"看起来高级"而使用复杂推理——简单有效才是王道。
🧪 动手练习： 
# 体验Zero-shot CoT的威力
prompt_no_cot = "15 * 17 + 23 = ?"
prompt_with_cot = "15 * 17 + 23 = ? Let's think step by step."

# 对比两个prompt的回答准确率
# 在GPT-4上，no_cot偶尔出错，with_cot几乎100%正确
▶显示答案
💡 参考答案与详解

解题思路: Zero-shot CoT只需在prompt末尾添加"Let's think step by step"即可显著提升推理准确率。对于需要多步推理的数学题，CoT通过显式暴露中间步骤，将单步推理分解为多步，每步的错误率更低。

# 完整对比实验代码
from openai import OpenAI
client = OpenAI()

def ask(prompt, model="gpt-4o"):
    response = client.chat.completions.create(
        model=model,
        messages=[{"role": "user", "content": prompt}],
        temperature=0
    )
    return response.choices[0].message.content

# 测试 (需要多步计算的题目)
questions = ["15 * 17 + 23 = ?", "打8折后售价120元，原价?", "速度15km/h走45分钟，距离?"]

for q in questions:
    ans_no_cot = ask(q)
    ans_cot = ask(q + " Let's think step by step.")
    print(f"问题: {q}")
    print(f"  无CoT: {ans_no_cot[:80]}")
    print(f"  有CoT: {ans_cot[:80]}")
    print("-" * 50)

# 关键发现:
# 简单计算(15*17+23): 两者都对，但有CoT可看到推理过程
# 推理题(打折/行程): CoT显著减少错误，且过程可调试
# CoT最大价值: 可解释性 + 可调试性

关键知识点: (1) CoT的核心机制是将隐式推理变为显式推理，降低每步的认知负荷 (2) "Let's think step by step" 是性价比最高的prompt技巧，MultiArith数据集上从17.7%提升到78.7% (3) CoT对简单任务可能略增延迟但不降准确率，对复杂任务显著提升 (4) 在Agent场景中，CoT常作为ReAct的Thought步骤使用，是Agent推理能力的基石。

8. 推理优化：DSPy与自动Prompt工程

手动设计推理prompt是低效的。DSPy（Stanford NLP, 2023）将推理过程模块化，通过自动优化提升Agent的推理能力。

小结

概念一句话
CoT让LLM展示推理过程，从"直接回答"变为"逐步推理"
Self-Consistency多次采样不同推理路径，取多数投票结果
ToT将推理建模为搜索树，通过BFS/DFS探索最优路径
GoT将推理扩展为图结构，支持分支合并和聚合
