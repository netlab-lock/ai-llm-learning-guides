1. Embedding模型原理

1.1 什么是Embedding？

Embedding将离散的文本映射为连续的高维向量，使得语义相似的文本在向量空间中距离近。

Embedding: text → Rd (d通常768-3072)
┌─────────────────────────────────────────────────────────────────┐ │ Embedding 向量空间示意 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ "Python教程" ● │ │ \ │ │ "编程入门" ● \ ● "机器学习入门" │ │ \ / │ │ ●────●────● │ │ "AI教程" "深度学习" │ │ │ │ "今天天气" ●────● "明天预报" │ │ │ │ 语义相似 → 向量距离近 │ │ 语义不同 → 向量距离远 │ │ │ └─────────────────────────────────────────────────────────────────┘

1.2 Embedding模型训练：对比学习

现代Embedding模型通过对比学习（Contrastive Learning）训练：

InfoNCE Loss: L = -log [exp(sim(q,k⁺)/τ) / Σ exp(sim(q,kᵢ)/τ)]

直觉：让query和正样本(k⁺)的相似度最大化，和负样本(k⁻)的相似度最小化。

┌─────────────────────────────────────────────────────────────────┐ │ 对比学习训练流程 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ 训练数据: (query, positive, negative) 三元组 │ │ 例: ("如何学Python", "Python入门指南", "今天天气") │ │ │ │ Step 1: 编码 │ │ q = Encoder("如何学Python") → [0.12, 0.85, ...] │ │ k⁺ = Encoder("Python入门指南") → [0.15, 0.82, ...] │ │ k⁻ = Encoder("今天天气") → [0.91, 0.23, ...] │ │ │ │ Step 2: 计算相似度 │ │ sim(q, k⁺) = 0.95 (应该高) │ │ sim(q, k⁻) = 0.12 (应该低) │ │ │ │ Step 3: 更新参数，让sim(q,k⁺)↑, sim(q,k⁻)↓ │ │ │ │ 关键: 负样本的质量决定模型质量 │ │ • 简单负样本(随机): 模型学不到有用信息 │ │ • 困难负样本(语义相近但不同): 模型学到细粒度区分 │ │ │ └─────────────────────────────────────────────────────────────────┘

1.3 Matryoshka表征学习

Matryoshka（俄罗斯套娃）Embedding允许在不同维度上使用同一个向量：

完整向量: [d₁, d₂, d₃, ..., d₁₀₂₄] 精度最高
截断向量: [d₁, d₂, d₃, ..., d₂₅₆] 精度略低，速度快4倍

text-embedding-3-large支持Matryoshka：可以用256维做粗筛，再用3072维精排。

2. ANN近似最近邻算法 面试高频

2.1 为什么需要ANN？

精确最近邻: O(n·d) 对100万向量×3072维 = 3万亿次运算
ANN近似最近邻: O(log n·d) 100万向量只需~20次比较

2.2 HNSW详解 面试必问

┌─────────────────────────────────────────────────────────────────┐ │ HNSW 分层可导航小世界图 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ Layer 2 (最稀疏, 高速公路): [A]────────────[G] │ │ | | │ │ Layer 1 (中等密度): [A]──[C]──[E] [G]──[I] │ │ [B]──[D]──[F] [H]──[J] │ │ │ │ Layer 0 (最密集, 所有点): 所有数据点都存在, 精确搜索 │ │ │ │ 搜索过程: │ │ 1. 从Layer 2的入口点开始 │ │ 2. 在当前层贪心搜索最近邻居 │ │ 3. 找到当前层最近点后, 下降到下一层 │ │ 4. 重复直到Layer 0, 得到最终结果 │ │ │ │ 类比: 高速公路→城市道路→小区道路 │ │ 时间: O(log n) 空间: O(n·M) 召回率: 95-99% │ │ │ └─────────────────────────────────────────────────────────────────┘

✅ 参考答案与详解
  1. 核心原理: HNSW(Hierarchical Navigable Small World)构建过程：(1)每个新数据点插入时，从最高层开始，在当前层贪心搜索找到最近邻居 (2)逐层下降直到Layer 0 (3)在每一层中，将新节点连接至多M个最近邻居，形成局部小世界网络 (4)层数由公式l = floor(-ln(random()) × m_L)决定，m_L是归一化因子，高层概率低(类似跳表)。搜索过程：(1)从最高层的入口点开始 (2)在当前层贪心搜索——移动到距离目标更近的邻居，直到找不到更近的点 (3)下降到下一层，以下一层找到的最近点作为新入口 (4)重复直到Layer 0，在Layer 0上用ef_search控制搜索范围。参数影响：M(每层连接数)越大——构建时间越长(每层需要找更多邻居)、内存越大(每个点存更多边)、搜索精度越高(更多路径选择)、搜索速度略慢(检查更多邻居)。推荐M=16-64。ef_construction越大——构建时间越长(插入时搜索范围更大)、索引质量越高(连接更优)、不影响搜索时间。推荐ef_construction=100-200。ef_search越大——搜索越准但越慢，是精度-速度的运行时权衡。
    关键要点: (1) 构建类似跳表的多层结构 (2) 搜索从粗到细逐步精确 (3) M影响质量和内存 (4) ef_construction影响构建质量 (5) ef_search是运行时trade-off
    实际案例: 100万128维向量，M=16/ef_construction=200构建时间约30分钟，内存500MB，95%召回率@10。M=64时构建时间2小时，内存1.5GB，98%召回率@10——质量提升3%但内存增加3倍。
  2. 核心原理: 场景A(个人开发者，10万文档)：选Chroma。理由：(1)零配置，pip install即可用 (2)内存模式适合原型开发，持久化模式满足10万文档需求 (3)内置HNSW索引 (4)Python原生，与LangChain/LLamaIndex无缝集成 (5)10万文档×768维约300MB内存，单机完全足够。场景B(企业客服，1000万文档)：选Milvus。理由：(1)1000万文档×768维约30GB，单机内存紧张，需要Milvus的分布式架构 (2)Milvus支持DiskANN处理超大规模数据 (3)支持混合检索(向量+标量过滤)，适合客服场景的"按部门/时间过滤" (4)生产级高可用 (5)支持IVF/HNSW/PQ多种索引，可按成本-精度权衡选择。场景C(已有PostgreSQL团队，100万文档)：选pgvector。理由：(1)不需要引入新的数据库组件，降低运维复杂度 (2)100万×768维约3GB，pgvector完全胜任 (3)可以用SQL同时做向量检索和结构化过滤 (4)团队无需学习新技术栈 (5)备份恢复复用PostgreSQL生态。
    关键要点: (1) Chroma零配置适合小规模 (2) Milvus分布式适合大规模 (3) pgvector利用已有PG (4) 规模是核心选型依据 (5) 团队技术栈也要考虑
    实际案例: 某创业公司从Chroma(原型)→Qdrant(100万文档生产)→Milvus(1000万+文档)的演进路径，每一步都是数据量增长10倍驱动的。
  3. 核心原理: 困难负样本(Hard Negative)是指在向量空间中与query语义相近但实际上不是正确答案的样本。例如query="Python异步编程"，正样本="Python asyncio教程"，简单负样本="红烧肉做法"，困难负样本="Python多线程编程"(都是Python并发编程，但技术方案不同)。困难负样本比简单负样本更有效的原因：(1)简单负样本(随机选取)与query差异太大，模型很容易区分，无法学到细粒度语义差异 (2)困难负样本迫使模型学习到"异步"vs"多线程"这样的细微区别，提升区分能力 (3)没有困难负样本的模型在实际检索中，对于排名靠前的困难负样本会错误地给出高分。三种挖掘方法：(1)BM25检索——用BM25找出与query关键词重叠高但语义不同的文档 (2)模型自身的negatives——用当前模型检索top-K，排除正样本后排名最高的就是hard negatives (3)Cross-encoder打分——用更准确的cross-encoder对候选重排，选择被cross-encoder判为不相关但embedding模型判为相关的样本。
    关键要点: (1) 困难负样本与query近似但非正解 (2) 迫使模型学细粒度区分 (3) BM25/模型自身/Cross-encoder三种挖掘 (4) 负样本质量决定模型质量上限
    实际案例: BGE训练中使用BM25+模型自身检索+Cross-encoder三种方法混合挖掘困难负样本，相比只用随机负样本，MTEB得分从58.2提升到64.7。
  4. 核心原理: Matryoshka的核心特性是训练时让向量的不同维度子空间保留不同粒度的信息——前256维已经包含了大部分语义信息，后续维度逐步补充细节。在RAG管线中的应用价值：(1)两级检索优化——第一级用256维做粗筛(速度快4倍)，从100万文档中选出top-100；第二级用完整3072维做精排，从100个中选top-10。总耗时≈粗筛(1ms)+精排(0.5ms)=1.5ms，vs 直接用3072维检索约5ms，速度提升3倍且精度几乎无损 (2)存储成本优化——对老文档/低频文档只存256维向量，对新文档/高频文档存完整向量，节省50-75%存储 (3)缓存友好——粗筛向量可以完全放入CPU L3缓存，减少内存读取 (4)渐进式检索——先快速返回初步结果(256维检索)，后台用完整维度做精排并更新结果。
    关键要点: (1) 粗筛+精排两级检索 (2) 存储分级降低存储成本 (3) 粗筛适合缓存 (4) 渐进式用户体验
    实际案例: OpenAI的text-embedding-3-large支持Matryoshka。在一个100万文档的RAG系统中，用256维粗筛(耗时1.2ms) + 3072维精排(耗时0.3ms) vs 全程3072维检索(耗时4.8ms)，速度提升3.2倍，检索精度(NDCG@10)从0.847降到0.841，仅损失0.7%。

参考答案与详解

解题思路: 实现向量检索系统需要理解余弦相似度计算、RRF融合排序、Matryoshka两级检索流程。混合检索将向量检索(BM25风格关键词)的结果用RRF公式融合：RRF_score = Σ 1/(k+rank_i)，k为平滑常数(通常60)。Matryoshka两级检索先用低维向量粗筛，再用完整维度精排。

import math
import numpy as np
from collections import Counter
from typing import List, Dict, Callable, Tuple, Optional

class VectorSearchEngine:
    """向量检索引擎 - 支持混合检索和Matryoshka两级检索"""
    
    def __init__(self, dim=768, use_matryoshka=True, short_dim=256):
        self.dim = dim
        self.short_dim = short_dim if use_matryoshka else dim
        self.documents = []       # 原始文档列表
        self.vectors = []         # 完整维度向量(N×dim)
        self.short_vectors = []   # 低维度向量(N×short_dim, Matryoshka)
        self.id_to_idx = {}       # doc_id → 索引位置
        # 倒排索引(用于关键词检索)
        self.inverted_index = {}  # term → set(idx)
        self.doc_term_freq = []   # 每篇文档的词频统计
        self.avg_doc_len = 0      # 平均文档长度(用于BM25)
    
    def add_documents(self, docs: List[Dict], embed_fn: Callable):
        """
        添加文档并建立索引
        docs: [{"id": "doc1", "text": "...", "metadata": {...}}]
        embed_fn: 文本→向量的embedding函数，需返回(完整向量, 短向量)元组
        """
        start_idx = len(self.documents)
        
        for doc in docs:
            doc_id = doc["id"]
            text = doc["text"]
            
            # 生成embedding
            if self.short_dim < self.dim:
                full_vec, short_vec = embed_fn(text)
            else:
                full_vec = embed_fn(text)
                short_vec = full_vec
            
            idx = len(self.documents)
            self.documents.append(doc)
            self.vectors.append(np.array(full_vec))
            self.short_vectors.append(np.array(short_vec))
            self.id_to_idx[doc_id] = idx
            
            # 建立倒排索引
            terms = self._tokenize(text)
            term_freq = Counter(terms)
            self.doc_term_freq.append(term_freq)
            for term in set(terms):
                if term not in self.inverted_index:
                    self.inverted_index[term] = set()
                self.inverted_index[term].add(idx)
        
        # 计算平均文档长度
        total_len = sum(len(d) for d in self.doc_term_freq)
        self.avg_doc_len = total_len / max(len(self.doc_term_freq), 1)
        
        # 转为numpy数组加速计算
        if self.vectors:
            self.vectors_np = np.array(self.vectors)
            self.short_vectors_np = np.array(self.short_vectors)
        
        return f"已添加{len(docs)}篇文档，总计{len(self.documents)}篇"
    
    def search(self, query_vec, top_k=10, keyword_filter=None, mode="hybrid",
               query_text=None):
        """
        检索
        mode: "vector" | "keyword" | "hybrid"
        返回: [(doc, score), ...]
        """
        if mode == "vector":
            return self._vector_search(query_vec, top_k)
        elif mode == "keyword":
            if query_text is None:
                raise ValueError("关键词检索需要提供query_text")
            return self._keyword_search(query_text, top_k)
        elif mode == "hybrid":
            return self._hybrid_search(query_vec, query_text, top_k)
        else:
            raise ValueError(f"未知检索模式: {mode}")
    
    def matryoshka_two_stage(self, query_vec_full, query_vec_short=None, 
                              coarse_k=100, fine_k=10):
        """
        Matryoshka两级检索:
        1. 用短向量从全量中粗筛coarse_k个
        2. 用完整维度对粗筛结果精排，返回fine_k个
        """
        if query_vec_short is None and self.short_dim < self.dim:
            query_vec_short = query_vec_full[:self.short_dim]
        
        # 第一阶段: 粗筛 (在低维空间中搜索)
        coarse_results = self._vector_search_single(
            query_vec_short if query_vec_short is not None else query_vec_full,
            coarse_k, vectors_attr="short_vectors_np" if self.short_dim < self.dim else "vectors_np"
        )
        
        # 第二阶段: 精排 (在完整维度空间中重排)
        coarse_indices = [idx for idx, _ in coarse_results]
        if not coarse_indices:
            return []
        
        # 取粗筛结果的完整维度向量
        coarse_full_vectors = self.vectors_np[coarse_indices]
        query_full = np.array(query_vec_full[:self.dim])
        
        # 计算完整维度相似度
        similarities = self._cosine_similarity_batch(query_full, coarse_full_vectors)
        
        # 排序取top fine_k
        top_indices = np.argsort(similarities)[::-1][:fine_k]
        
        results = []
        for i in top_indices:
            orig_idx = coarse_indices[i]
            results.append((self.documents[orig_idx], float(similarities[i])))
        
        return results
    
    def _vector_search(self, query_vec, top_k):
        """向量检索"""
        query = np.array(query_vec[:self.dim])
        similarities = self._cosine_similarity_batch(query, self.vectors_np)
        top_indices = np.argsort(similarities)[::-1][:top_k]
        return [(self.documents[i], float(similarities[i])) for i in top_indices]
    
    def _vector_search_single(self, query_vec, top_k, vectors_attr="vectors_np"):
        """对指定向量矩阵搜索"""
        vectors = getattr(self, vectors_attr, self.vectors_np)
        if len(vectors) == 0:
            return []
        query = np.array(query_vec)
        similarities = self._cosine_similarity_batch(query, vectors)
        top_indices = np.argsort(similarities)[::-1][:top_k]
        return [(i, float(similarities[i])) for i in top_indices]
    
    def _keyword_search(self, query_text, top_k):
        """BM25关键词检索"""
        query_terms = self._tokenize(query_text)
        if not query_terms:
            return []
        
        # 计算每篇文档的BM25分数
        scores = {}
        k1, b = 1.5, 0.75  # BM25参数
        N = len(self.documents)
        
        for term in query_terms:
            if term not in self.inverted_index:
                continue
            df = len(self.inverted_index[term])  # 文档频率
            idf = math.log((N - df + 0.5) / (df + 0.5) + 1)
            
            for idx in self.inverted_index[term]:
                tf = self.doc_term_freq[idx].get(term, 0)
                doc_len = sum(self.doc_term_freq[idx].values())
                # BM25公式
                score = idf * (tf * (k1 + 1)) / (tf + k1 * (1 - b + b * doc_len / self.avg_doc_len))
                scores[idx] = scores.get(idx, 0) + score
        
        # 排序
        sorted_results = sorted(scores.items(), key=lambda x: -x[1])[:top_k]
        return [(self.documents[idx], score) for idx, score in sorted_results]
    
    def _hybrid_search(self, query_vec, query_text, top_k, k=60):
        """
        混合检索 - RRF融合排序
        RRF_score(d) = Σ 1/(k + rank_i(d))
        """
        # 向量检索排名
        vector_results = self._vector_search(query_vec, top_k * 3)
        vector_ranks = {self.documents.index(doc): rank for rank, (doc, _) in enumerate(vector_results)}
        
        # 关键词检索排名
        keyword_results = self._keyword_search(query_text, top_k * 3)
        keyword_ranks = {self.documents.index(doc): rank for rank, (doc, _) in enumerate(keyword_results)}
        
        # RRF融合
        all_doc_indices = set(vector_ranks.keys()) | set(keyword_ranks.keys())
        rrf_scores = {}
        for idx in all_doc_indices:
            score = 0
            if idx in vector_ranks:
                score += 1.0 / (k + vector_ranks[idx])
            if idx in keyword_ranks:
                score += 1.0 / (k + keyword_ranks[idx])
            rrf_scores[idx] = score
        
        # 排序
        sorted_results = sorted(rrf_scores.items(), key=lambda x: -x[1])[:top_k]
        return [(self.documents[idx], score) for idx, score in sorted_results]
    
    def _cosine_similarity_batch(self, query, vectors):
        """批量计算余弦相似度"""
        query_norm = query / (np.linalg.norm(query) + 1e-10)
        vectors_norm = vectors / (np.linalg.norm(vectors, axis=1, keepdims=True) + 1e-10)
        return np.dot(vectors_norm, query_norm)
    
    def _tokenize(self, text):
        """简易中文/英文分词 - 按字符级分词"""
        # 简化实现: 对英文按空格分词, 对中文按字符
        text = text.lower()
        # 简单处理: 空格分词 + 中文单字
        words = []
        for token in text.split():
            # 分离中文字符
            current_word = ""
            for char in token:
                if '\u4e00' <= char <= '\u9fff':
                    if current_word:
                        words.append(current_word)
                        current_word = ""
                    words.append(char)
                else:
                    current_word += char
            if current_word:
                words.append(current_word)
        return words
    
    def stats(self):
        """索引统计信息"""
        return {
            "total_documents": len(self.documents),
            "vector_dim": self.dim,
            "short_dim": self.short_dim,
            "vocab_size": len(self.inverted_index),
            "matryoshka": self.short_dim < self.dim
        }
