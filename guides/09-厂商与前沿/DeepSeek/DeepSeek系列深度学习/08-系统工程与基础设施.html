理解1. 概览需要从DeepSeek的技术演进脉络入手。DeepSeek从最初的模型架构探索，到大规模预训练的工程实践，再到多模态能力的拓展，每一步都建立在前一步的技术积累之上。这种渐进式的技术发展路线，使得DeepSeek能够在每个阶段都保持技术竞争力，同时降低技术风险。

1.1 MoE 架构设计的核心权衡

DeepSeek 的系统工程哲学可以概括为"用架构创新换取计算效率"。MoE 架构中每个 token 仅激活约 8-10% 的参数，这意味着同等性能下推理 FLOPs 和显存需求大幅降低。但这也带来了新的工程挑战：专家路由的负载均衡、All-to-All 通信开销、以及推理时的动态批处理复杂度。

通俗类比：DeepSeek就像AI界的“性价比之王”。它用更少的训练成本（1/10的算力）做出了和顶级模型相当的效果，秘诀在于创新的MoE架构（只用部分参数处理任务）和系统级优化。

4. 确定性训练

DeepSeek在4. 确定性训练方面的技术积累体现在数据质量控制和训练策略优化两个维度。在数据方面，DeepSeek建立了多层数据过滤管线：基于规则的过滤、基于分类器的质量评分和基于多样性的采样。在训练策略方面，DeepSeek采用了学习率warmup、梯度裁剪和z-loss正则化等技术保障训练稳定性。这些措施确保了模型在大规模训练中的一致性和可复现性。

4.0 位级可复现性的工程意义

确定性训练是大规模模型开发中被低估的关键能力。在数千 GPU 的分布式训练中，非确定性操作会导致同一配置产生微妙不同的权重，使得 debug 和消融实验变得极其困难。DeepSeek 通过消除 atomicAdd、禁用 Split-K、固定 cuBLAS 算法选择等手段，实现了位级可复现的训练过程。

训练过程中的关键技术包括：混合精度训练（BF16/FP16）以减少显存占用，梯度累积以支持更大的有效批大小，学习率warmup和cosine退火以确保训练稳定性，以及梯度裁剪以防止训练过程中的数值不稳定。这些技术的组合使得大规模模型训练变得更加高效和可靠。

DeepSeek的训练流程采用多阶段策略，从大规模无监督预训练到高质量有监督微调，每个阶段都有明确的优化目标。预训练阶段使用海量文本数据学习语言的基本规律和世界知识，微调阶段使用精选的指令-回答对训练模型遵循指令和生成高质量输出。

通俗类比：确定性训练就像每次做菜都严格按食谱操作，确保味道一模一样。在AI训练中，它指相同输入、超参和随机种子，能100%复现相同的模型权重和梯度，这对调试和科学验证至关重要。

5.1 异构 KV Cache 推理

  • 稀疏注意力核：与 KV Cache 布局协同设计的高性能 kernel

5.2 磁盘 KV Cache

利用 3FS (Fire-Flyer File System) 分布式文件系统存储共享前缀的压缩 KV：

  • 首次计算的 KV 条目持久化到 3FS
  • 后续请求直接读取，避免重复 prefill
  • 支持按场景选择存储策略（纯内存、内存+磁盘、磁盘优先）

6. 3FS (Fire-Flyer File System)

DeepSeek在6. 3FS (Fire-Flyer File System方面的技术实现体现了其对工程质量和创新的追求。从技术原理到工程落地，DeepSeek在每个环节都进行了深入的优化。这些优化不仅提升了模型的性能指标，更重要的是提升了模型在实际应用中的可靠性和用户体验。DeepSeek的技术团队在实践中积累的"隐性知识"是其核心竞争力之一。

6.1 AI 原生存储架构

3FS 的设计哲学是"为 AI 工作负载量身定制"。与通用分布式文件系统不同，它针对大文件顺序读写、高聚合带宽和低元数据延迟进行了深度优化。通过 RDMA 网络和纠删码容错，3FS 在数千节点规模下实现了百 GB/s 级吞吐，同时将存储开销控制在 1.5 倍冗余度以内。

评估DeepSeek模型的能力需要多维度的基准测试。知识能力通过MMLU(57科目)和C-Eval(中文)评估；推理能力通过GSM8K(数学)和ARC(科学)评估；代码能力通过HumanEval(Python)和MBPP(多语言)评估；对话质量通过Chatbot Arena(人工排名)评估。单一基准的高分不代表模型优秀——需要综合评估在实际场景下的表现。

DeepSeek在技术实现方面的技术演进遵循"验证-优化-规模化"的三阶段路径。第一阶段通过小规模实验验证核心技术假设的可行性；第二阶段在中等规模上优化性能指标和工程实现；第三阶段在大规模生产环境中部署并持续监控。每个阶段都有明确的质量门控标准，只有达到标准才能进入下一阶段。

通俗类比：3FS就像为AI集群打造的“超级仓库管理员”。它把成百上千个计算节点的本地SSD硬盘组成一个共享的高速文件池，让任何节点都能像访问本地硬盘一样快速读写海量训练数据。

7. 训练框架优化

7. 训练框架优化的工程实践涉及数据管线、分布式训练和checkpoint管理三大子系统。DeepSeek的数据管线支持PB级数据的实时处理和质量监控；分布式训练系统采用数据并行+张量并行+流水线并行的三维并行策略；checkpoint管理支持增量保存和快速恢复。这些基础设施的完善程度直接决定了模型训练的效率和稳定性。

7.0 3D 并行与显存优化策略

万亿参数模型的训练框架需要在计算效率、显存占用和通信开销之间找到最优平衡。DeepSeek 采用 3D 并行（数据并行 DP + 张量并行 TP + 流水线并行 PP）与 ZeRO 优化器分片的混合策略，并通过梯度检查点和混合精度训练进一步压缩显存，使得在 2048 张 H800 GPU 上完成万亿 token 训练成为可能。

训练过程中的关键技术包括：混合精度训练（BF16/FP16）以减少显存占用，梯度累积以支持更大的有效批大小，学习率warmup和cosine退火以确保训练稳定性，以及梯度裁剪以防止训练过程中的数值不稳定。这些技术的组合使得大规模模型训练变得更加高效和可靠。

DeepSeek的训练流程采用多阶段策略，从大规模无监督预训练到高质量有监督微调，每个阶段都有明确的优化目标。预训练阶段使用海量文本数据学习语言的基本规律和世界知识，微调阶段使用精选的指令-回答对训练模型遵循指令和生成高质量输出。

7.2 mHC 高效实现

mHC 的流形约束投影操作通过重计算 + 融合 kernel 实现：

  • 正交化操作不保存中间结果（节省显存）
  • 反向传播时重新计算（用计算换显存）
  • 前向和反向的投影操作融合为单一 kernel
