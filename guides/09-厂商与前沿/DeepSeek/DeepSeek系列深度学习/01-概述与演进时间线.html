3.1 DeepSeek-V1 (2024.01)

DeepSeek 的第一个开源模型，标志着其正式进入 LLM 领域。

  • 架构：标准 Transformer，使用 GQA (Grouped Query Attention)
  • 规模：7B 和 67B 两个版本
  • 训练数据：2T tokens，以中英文为主
  • 特色：在代码和数学任务上表现出色，中文能力强
  • 定位：验证技术路线，积累经验

DeepSeek-V1 架构 (标准 Transformer Dense Model)

3.2 DeepSeek-V2 (2024.05) — 架构革命

V2 是 DeepSeek 架构创新的起点，引入了两个改变游戏规则的技术：

  • Multi-head Latent Attention (MLA)：将 KV Cache 压缩到低维潜空间，显存占用降低至 GQA 的约 1/5
  • DeepSeekMoE：细粒度 MoE 架构，使用大量小专家 + 共享专家
  • 参数：236B 总参数，但每个 token 仅激活 21B（约 8.9%）
  • 训练数据：8.1T tokens
  • 性能：与 Llama-3-70B 相当，但推理成本仅为其一小部分

DeepSeek-V2 架构革新

3.3 DeepSeek-V3 (2024.12) — 里程碑

V3 是 DeepSeek 最具影响力的版本，在 V2 的基础上进一步优化：

  • 规模：671B 总参数 / 37B 激活，370 亿 MoE 专家
  • 无辅助损失负载均衡：不使用传统的 auxiliary loss，而是通过动态偏置项平衡专家负载，避免了性能损失
  • Multi-Token Prediction (MTP)：同时预测多个未来 token，增强训练信号
  • FP8 混合精度训练：首个大规模使用 FP8 训练的模型，训练效率提升约 40%
  • DeepSeek Sparse Attention (DSA)：V3.2 引入的稀疏注意力，V4 进一步发展为 CSA
  • 训练成本：仅 557.6 万美元（2048 × H800，约 2 个月）
  • 性能：在多项基准上媲美 GPT-4o 和 Claude 3.5 Sonnet
🎯 V3 核心突破

V3 证明了一个重要事实：顶级 LLM 的训练成本可以比业界预期低一个数量级。这直接改变了行业对"大模型 = 大投入"的认知，推动了全球范围内对高效训练方法的研究。

3.4 DeepSeek-R1 (2025.01) — 推理革命

R1 系列是 DeepSeek 在推理模型方向的重要探索：

  • R1-Zero：完全不使用 SFT（监督微调），仅通过 GRPO 强化学习就涌现出了思维链推理能力。这证明了推理能力可以从纯 RL 中自发产生。
  • R1：在 R1-Zero 基础上增加了冷启动 SFT 数据，进一步提升推理质量和可读性。
  • GRPO (Group Relative Policy Optimization)：不需要训练单独的价值模型，通过组内相对排序来估计优势函数，大幅降低 RL 训练成本。
  • 性能：在 AIME 2024（数学竞赛）和 Codeforces（编程竞赛）上达到了 OpenAI o1 级别。

3.5 DeepSeek-V4 (2026) — 百万 token 时代

V4 系列是截至目前 DeepSeek 最先进的版本，核心目标是突破超长上下文的效率瓶颈：

  • V4-Pro：1.6T 总参数 / 49B 激活，61 层 Transformer
  • V4-Flash：284B 总参数 / 13B 激活，更高效的轻量版
  • 上下文长度：原生支持 100 万 token
  • 训练数据：32T-33T tokens
