1. 核心论文列表

GLM的技术论文是理解其技术路线的最佳入口。通过精读这些论文，可以深入了解GLM在模型架构、训练方法和推理优化方面的创新。建议的学习顺序是：先读概述性论文了解全局，再读具体技术论文深入细节，最后读实验论文了解实际效果。每篇论文建议花2-3小时精读，并做笔记总结关键创新点。

1.1 智谱AI评估论文导读指南

智谱AI的论文体系呈现出清晰的演进脉络：从GLM-130B的自回归填空预训练范式，到ChatGLM系列的对话能力构建，再到GLM-4的多模态扩展，最终到GLM-5的Agent能力突破。每篇论文都包含详细的评估实验和消融研究，建议读者重点关注各论文中的评测方法论部分——这些评估方法论本身就是智谱AI对LLM能力评估的重要贡献，对理解国产大模型的技术水平具有参考价值。

通俗类比：GLM在技术实现方面的技术积累是多年工程实践的结果。从最初的架构探索到大规模部署的工程优化，每一步都建立在对前一步经验的总结之上。这种渐进式的技术演进路线确保了每次改进都是可验证和可复现的。
常见误区：GLM在1. 核心论文列表方面的常见误区是以为MoE计算量等于总参数量。实际上MoE每次只激活部分专家，计算量接近激活参数量。GLM的MoE设计正是利用这一特性实现高参数效率。

以下是理解 GLM 系列必须精读的论文，按推荐顺序排列：

📄 GLM-130B: An Open Bilingual Pre-Trained Model (2022)

作者: Zeng et al. | 会议: ICLR 2023 | 奠基之作 必读

核心贡献：首个开源的千亿参数双语模型。提出自回归空白填充（Autoregressive Blank Infilling）预训练目标，设计二维位置编码（2D RoPE），实现 INT4 量化下的低门槛部署。论文详细介绍了训练稳定性技术（嵌入层梯度缩放、FP16 混合精度等）。

精读重点：第3节（GLM预训练框架）、第4节（训练稳定性）、第6节（量化策略）

📄 ChatGLM: An Open Bilingual Dialogue Language Model (2023)

作者: GLM Team | 阶段: ChatGLM → ChatGLM2 → ChatGLM3 对话系统 必读

核心贡献：将 GLM-130B 蒸馏到 6B 规模，通过监督微调（SFT）和 RLHF 实现高质量对话。ChatGLM2 引入 GQA 和 FlashAttention，将上下文从 2K 扩展到 32K。ChatGLM3 加入工具调用和代码能力。

精读重点：蒸馏策略、SFT 数据构建、GQA 架构改进

📄 GLM-4 Technical Report (2024)

作者: GLM Team | 类型: 技术报告 多模态 重要

核心贡献：GLM-4 系列是智谱从学术走向商业化的转折点。支持 128K 上下文、多模态输入（GLM-4V）、Function Calling、All Tools 模式。报告详细披露了数据清洗、对齐训练、安全评估的工程细节。

精读重点：多模态融合架构、长上下文训练、All Tools 实现

📄 GLM-5 Technical Report (2026)

作者: GLM Team | 类型: 技术报告 最新 必读

核心贡献：GLM-5 定位为 agentic LLM，核心创新包括：(1) 大规模合成数据用于 Agent 训练；(2) 交错式思维链推理；(3) 多轮自主纠错机制；(4) GLM-5.1 变体专注 agentic coding。开源权重，支持商用。

精读重点：Agent 训练数据合成、推理架构、评测体系设计

📄 CogVLM: Visual Expert for Pretrained Language Models (2023)

作者: Wang et al. | 类型: 视觉语言模型 多模态 重要

核心贡献：提出视觉专家模块（Visual Expert），在每个 Transformer 层中并行插入视觉特征的 QKV 和 FFN，实现深度融合而非浅层拼接。CogVLM-Chat 在多个视觉理解基准上超越同期模型。

精读重点：Visual Expert 架构设计、与 Flamingo/LLaVA 的对比

📄 AutoGLM: Autonomous Agents for Human-Level Computer Control (2024)

作者: GLM Team | 类型: Agent 系统 Agent 必读

核心贡献：首个支持真实手机和 PC GUI 操作的自主 Agent 系统。核心思想是将 GUI 理解与语言模型决策结合，通过"观察-思考-行动"循环实现 50+ 步自主操作。GLM-PC 扩展到桌面环境控制。

精读重点：GUI grounding 方法、长序列决策规划、安全约束设计

2. 推荐阅读顺序

2. 推荐阅读顺序是GLM技术体系中的重要组成部分。从系统工程角度看，这一技术的实现需要平衡计算效率、内存占用和输出质量三个维度。GLM通过精心的架构设计和工程优化，在这三个维度上都达到了令人满意的平衡点。这种平衡能力是GLM技术成熟度的重要体现。

2.1 分阶段阅读策略

推荐阅读路线按照「入门→进阶→深入」三阶段设计。入门阶段（1-2周）聚焦GLM-130B原始论文和ChatGLM系列，掌握核心预训练思想和工程演进脉络；进阶阶段（3-4周）精读GLM-4和GLM-5技术报告，关注多模态融合、Agent训练和评测体系设计；深入阶段（5-6周）阅读AutoGLM和CogVLM论文，结合竞品论文（Llama、Qwen、DeepSeek）形成全面的技术视野。

，GLM的技术选择反映了对市场需求和技术趋势的深刻理解。在模型规模、训练数据、对齐方法和部署效率等方面的决策，都体现了在性能、成本和安全性之间的精细权衡。

通俗类比：如果把推荐阅读顺序比作矿石冶炼，原矿(原始数据)→选矿(过滤)→冶炼(训练)→精炼(微调)。
常见误区：GLM在1. 核心论文列表方面的一个误区是以为Agent就是让LLM循环调工具。真正的Agent需要可靠的规划、错误恢复和状态管理。GLM的Agent设计采用有限状态机避免无限循环。

3. 外部学习资源

GLM的技术论文是理解其技术路线的最佳入口。通过精读这些论文，可以深入了解GLM在模型架构、训练方法和推理优化方面的创新。建议的学习顺序是：先读概述性论文了解全局，再读具体技术论文深入细节，最后读实验论文了解实际效果。每篇论文建议花2-3小时精读，并做笔记总结关键创新点。

外部学习资源的工作流程与数据流

3. 外部学习资源的训练流程通常分为三个阶段：预训练(Pre-training)在大规模无标注语料上学习语言表示，微调(Fine-tuning)在任务特定数据上适配下游任务，对齐(Alignment)通过人类偏好反馈优化输出质量。每个阶段的数据配比、学习率调度和正则化策略都需要精心设计。

通俗类比：外部学习资源就像婴儿学语言——先大量听(互联网文本)，学会语言规律(语法、常识)，再学具体任务(微调)。
常见误区：GLM在1. 核心论文列表方面的一个误区是只看MMLU分数选模型。MMLU有数据泄漏风险，且只测知识不测推理。GLM建议在自己数据上做A/B测试。
深入探讨：GLM在1. 核心论文列表方面的技术经济性分析表明，总拥有成本等于固定成本(GPU硬件)加边际成本(推理token数乘以单token成本)。通过模型量化和批处理优化，GLM将单token成本降低3-5倍，在保持利润率的同时提供有竞争力的API价格。
