2.1 Transformer 基础

DeepSeek 全系列基于 Transformer 架构。你需要理解：

  • Self-Attention: Q、K、V 矩阵的计算，注意力分数 Softmax(QK^T/√d)V
  • Multi-Head Attention (MHA): 多头并行计算注意力，拼接输出
  • KV Cache: 自回归推理时缓存历史 K、V 避免重复计算
  • FFN (Feed-Forward Network): 两层线性变换 + 激活函数
  • 位置编码: RoPE (Rotary Position Embedding) 的旋转矩阵

2.2 MoE 基础

Mixture-of-Experts 是 DeepSeek 的核心架构选择：

  • 稀疏激活: 每个 token 只激活部分专家，降低计算量
  • 路由机制: Gate 网络决定 token 分配给哪些专家
  • 负载均衡: 防止专家坍缩（所有 token 涌向少数专家）
  • Top-K 路由: 每个 token 选择 K 个最相关的专家

2.3 强化学习基础

理解 GRPO 需要以下 RL 知识：

  • 策略梯度: ∇J(θ) = E[∇log π(a|s) · A(s,a)]
  • PPO: 近端策略优化，裁剪目标函数防止策略剧变
  • Actor-Critic: 策略网络 (Actor) + 价值网络 (Critic)
  • 奖励模型: 从人类偏好中学习奖励信号

3.1 问题背景

标准 Multi-Head Attention 的 KV Cache 随序列长度线性增长：

KV Cache = 2 × n_layers × n_heads × d_head × seq_len × batch_size

对于 128K 上下文的模型，KV Cache 可达数百 GB，成为推理瓶颈。

3.2 MLA 的数学原理

MLA 的核心是将高维 KV 投影到低维潜在向量：

压缩阶段（训练时）：

c_KV = W_DKV · h_t    // 将隐状态 h_t 压缩为 d_c 维潜在向量

K = W_UK · c_KV     // 从潜在向量恢复 K
V = W_UV · c_KV     // 从潜在向量恢复 V

其中：
d_c << n_heads × d_head   // 潜在维度远小于原始 KV 维度
W_DKV ∈ R^{d_c × d_model}   // 下投影矩阵 (压缩)
W_UK ∈ R^{n_heads×d_head × d_c}   // 上投影矩阵 (恢复 K)
W_UV ∈ R^{n_heads×d_head × d_c}   // 上投影矩阵 (恢复 V)

3.3 MLA vs MHA vs GQA 对比

方法KV Cache 大小性能原理
MHA2 × n_heads × d_head基准完整多头 KV
GQA2 × n_groups × d_head略降多头共享 KV 组
MQA2 × d_head下降明显所有头共享单个 KV
MLAd_c (潜在维度)≈ MHA低秩压缩 KV 到潜在空间
关键优势：MLA 在 KV Cache 压缩到比 MQA 更小的同时，性能保持与 MHA 相当。这是因为低秩投影保留了最重要的信息，而上投影矩阵在推理时可以与 Q 投影合并，不增加额外计算。

3.4 推理时的计算合并

MLA 的精妙之处在于推理时的优化：

# 训练时：先压缩再恢复
c_KV = W_DKV @ h_t          # 压缩: d_model -> d_c (缓存这个)
K = W_UK @ c_KV              # 恢复: d_c -> n_heads * d_head
V = W_UV @ c_KV              # 恢复: d_c -> n_heads * d_head

# 推理时：将 W_UK 吸收到 W_Q 中
# Q @ K^T = Q @ (W_UK @ c_KV)^T = (W_UK^T @ Q)^T @ c_KV
# 只需缓存 c_KV (维度 d_c)，不需要完整 K, V
# 显存节省: (n_heads × d_head) / d_c 倍

4.2 架构细节

DeepSeekMoE 前向传播：

y = Σ(i∈TopK) g_i · E_i(x) + Σ(j∈Shared) E_j(x)

g_i = Softmax(TopK(Gate(x)))_i   // 路由权重
Gate(x) = W_g · x              // 门控网络
TopK: 选择 K 个得分最高的路由专家
Shared: 1-2 个始终激活的共享专家

4.3 负载均衡策略

DeepSeek-V3 引入了辅助损失无关的负载均衡：

  • 传统方法: 添加辅助损失惩罚不均衡路由，但会干扰主损失
  • V3 创新: 使用 bias 项调整路由得分，不引入额外损失梯度
  • 每个专家维护一个可调 bias，通过监控负载动态更新

5. ⚡ 混合注意力 CSA + HCA

DeepSeek在5. ⚡ 混合注意力 CSA + HCA上的创新主要体现在计算效率和建模能力的平衡上。传统Softmax注意力的O(N²)复杂度限制了长序列处理能力，而DeepSeek通过引入稀疏注意力模式、线性近似或Flash Attention等技术，在保持注意力精度的同时显著降低了计算成本。这些优化使得模型能够处理更长的上下文窗口，同时保持较低的推理延迟。

注意力机制的设计直接影响模型的长距离依赖建模能力。DeepSeek的方案通过优化注意力头的分组策略和KV Cache管理，在显存占用和计算效率之间取得了良好平衡。实验表明，这种设计在保持短距离精度的同时，显著提升了长距离信息检索的准确率。

通俗类比：CSA和HCA就像餐厅的两种点菜方式。CSA(压缩稀疏注意力)像"套餐"——先把相似的菜品分组压缩成几道大菜，快速浏览全局；HCA(分层交叉注意力)像"自助餐"——从压缩的套餐中挑选细节补充。两种方式结合，既能快速概览又能关注细节。

5.1 Compressed Sparse Attention (CSA)

CSA 将 KV Cache 分为两部分：

  • 跨头共享压缩 KV: 多个注意力头共享一份压缩后的 KV，降低冗余
  • 头特定压缩 KV: 每个头保留自己压缩后的 KV，保持头间差异性
CSA-KV = [K_shared_compressed; K_head_specific_compressed]
V4-Pro 在 1M 上下文时只需 10% KV Cache (vs V3.2)

5.2 Heavily Compressed Attention (HCA)

HCA 对 KV 进行更激进的压缩，但保持稠密注意力（非稀疏）：

  • 压缩比更高，KV Cache 更小
  • 保持对所有 token 的注意力计算（稠密），避免信息丢失
  • 适合需要全局理解的任务

8. 🎮 GRPO 强化学习算法

DeepSeek在技术实现方面的技术实现需要在多个工程约束下寻找最优解。计算资源有限时，需要在模型大小和推理速度之间做权衡；数据质量参差不齐时，需要在数据量和数据质量之间做取舍；部署环境多样时，需要在通用性和专用性之间做选择。DeepSeek通过系统性的性能分析和A/B测试来量化这些权衡的影响，确保每个技术决策都是数据驱动的。

通俗类比：GRPO就像训练导盲犬的新方法。传统RLHF需要另请一个"训犬教练"(奖励模型)来打分，成本高且可能被狗"钻空子"。GRPO直接让一群狗互相比较——同一批训练中表现好的奖励多、表现差的奖励少，用群体相对排名代替绝对打分，省掉了独立奖励模型。
常见误区：DeepSeek在1. 📋 学习路线图方面的一个误区是以为训练数据越多越好。数据质量远比数量重要——低质量数据不仅浪费算力还可能污染模型能力。DeepSeek通过多层过滤管线确保数据质量。
深入探讨：GRPO损失函数：L_GRPO = -E[min(r(θ)·A_i, clip(r(θ), 1-ε, 1+ε)·A_i)]，其中 r(θ)=π_θ(a|s)/π_old(a|s) 是概率比，A_i是组内优势估计。关键区别：A_i = (R_i - mean(R_group))/std(R_group)，使用同一prompt下G个回复的相对奖励标准化，而非独立奖励模型。DeepSeek-R1实验中G=64，ε=0.2，KL惩罚系数β=0.01。相比PPO减少约40%显存占用。

核心创新：Group Relative Policy Optimization — 无需 Critic 模型的策略优化，大幅降低 RL 训练成本。

8.1 传统 PPO 的问题

  • 需要训练一个与策略模型同等规模的 Critic (价值) 网络
  • Critic 的训练不稳定，可能引入偏差
  • 显存和计算开销翻倍

8.2 GRPO 算法

GRPO 目标函数：

J_GRPO(θ) = E_q[ Σ_i ( min(ρ_i · A_i, clip(ρ_i, 1-ε, 1+ε) · A_i) - β · D_KL(π_θ || π_ref) ) ]

ρ_i = π_θ(o_i|q) / π_old(o_i|q)    // 重要性采样比率
A_i = (r_i - mean(r)) / std(r)    // 组内相对优势
β · D_KL(...): KL 散度正则项

关键区别：A_i 通过组内相对排名计算，不需要 Critic 网络

8.3 GRPO 的工作流程

1. 对每个 prompt q，从旧策略 π_old 采样 G 个输出 {o_1, ..., o_G}
2. 对每个输出计算奖励 r_i = Reward(q, o_i)
3. 组内标准化优势: A_i = (r_i - mean(r_1..r_G)) / std(r_1..r_G)
4. 用 PPO-clip 目标更新策略 π_θ
5. 重复直到收敛

优势:
- 无需 Critic 模型 (省 50% 显存)
- 组内标准化消除了奖励尺度的影响
- 训练更稳定

8.4 DeepSeek-R1 的 RL 实验

R1-Zero 的重要发现：

  • 纯 RL 训练: 不使用 SFT，直接从 base model 开始 RL
  • 涌现推理: 模型自发学会了 Chain-of-Thought、自我验证、反思
  • Aha moment: 训练中出现"让我重新想想"的推理模式
  • 规则奖励: 使用规则而非奖励模型（数学用答案验证，代码用测试用例）

通俗类比：LLM推理就像“外卖配送网络”。训练是中央厨房研发菜谱（成本极高），部署是把厨房开到各个小区（边缘节点）。推理优化就是设计更小的标准化餐盒（量化）、优化配送路线（KV Cache）、使用更快的电动车（高效推理框架），让送餐（响应）又快又省。
常见误区：关于DeepSeek的1. 📋 学习路线图，很多人以为注意力权重代表token重要性。实际上注意力只是Q-K点积的相关性度量。DeepSeek的多头注意力设计通过多个视角综合判断token的真正重要性。
深入探讨：对于DeepSeek-V2(236B参数)的部署：1) **量化**：使用GPTQ或AWQ进行INT4量化，模型体积可从~472GB(FP16)压缩至~60GB，可在2-4张80G A100上运行。2) **KV Cache优化**：采用PagedAttention（如vLLM框架）技术，将KV Cache分块管理，减少内存碎片，提升吞吐量约2-3倍。3) **批处理**：动态批处理和连续批处理是关键，能将吞吐量从单请求~10 tokens/s提升至批量下~2000 tokens/s。
