🌊 Baichuan-Omni-1.5 — 全模态模型

Baichuan-Omni-1.5 (~11B) 是百川智能推出的全模态（Omni-modal）模型，是国产模型中少数同时支持文本、图像、视频、音频四种输入并生成文本+音频输出的模型：

  • 架构：7B LLM 核心 + 视觉编码器 + 音频编码器，Dense 架构
  • 输入：文本 + 图像 + 视频 + 音频（四模态输入）
  • 输出：文本 + 音频（含语音克隆、可控实时语音）
  • 评测：OpenCompass 平均 73.3，GMAI-MMBench 83.8%（最佳，超越 Qwen2-VL-72B）
  • 医学影像：医疗图像理解能力业界最佳
  • 图像分辨率：支持最高 1.8M 像素输入
  • 语音能力：语音克隆 + 可控实时语音生成
  • 协议：Apache 2.0（商用 >100万 DAU 有限制）

💻 使用示例

# 使用 Hugging Face Transformers 加载 Baichuan2-7B
from transformers import AutoTokenizer, AutoModelForCausalLM

tokenizer = AutoTokenizer.from_pretrained(
    "baichuan-inc/Baichuan2-7B-Chat",
    trust_remote_code=True
)
model = AutoModelForCausalLM.from_pretrained(
    "baichuan-inc/Baichuan2-7B-Chat",
    trust_remote_code=True,
    device_map="auto"
)

messages = [
    {"role": "user", "content": "请解释什么是大语言模型"}
]
response = model.chat(tokenizer, messages)
print(response)

# ── 使用百川API (Baichuan-3/4) ──
import requests

resp = requests.post(
    "https://api.baichuan-ai.com/v1/chat/completions",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    json={
        "model": "Baichuan4",
        "messages": [{"role": "user", "content": "你好"}]
    }
)
print(resp.json()["choices"][0]["message"]["content"])

💻 使用示例

# 使用 Hugging Face 加载 Yi-1.5-34B-Chat
from transformers import AutoTokenizer, AutoModelForCausalLM

model_path = "01-ai/Yi-1.5-34B-Chat"
tokenizer = AutoTokenizer.from_pretrained(model_path, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_path,
    trust_remote_code=True,
    device_map="auto",
    torch_dtype="auto"
)

messages = [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "用Python写一个快速排序"}
]

input_ids = tokenizer.apply_chat_template(
    messages, tokenize=True, add_generation_prompt=True, return_tensors="pt"
).to(model.device)

output = model.generate(input_ids, max_new_tokens=1024)
print(tokenizer.decode(output[0][input_ids.shape[1]:], skip_special_tokens=True))

📊 百川 vs Yi 综合对比

通俗类比：选模型就像选交通工具：百川像SUV——稳重全能；Yi像跑车——速度快、长续航。没有绝对好坏，只有适不适合你的场景。
常见误区：单一指标论英雄是大忌！模型选择要看综合表现：推理速度、显存占用、中文能力、开源协议都要考虑，就像买房不只看面积。
