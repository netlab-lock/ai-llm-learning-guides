4.1 DPO (Direct Preference Optimization)

Baichuan2 从 RLHF 迁移到 DPO，这是一个更简洁高效的对齐方案：

4.2 DPO 实现

# DPO 损失函数
def dpo_loss(policy_chosen_logps, policy_rejected_logps,
            ref_chosen_logps, ref_rejected_logps, beta=0.1):
    """
    DPO: 直接从偏好对中学习策略
    policy_chosen_logps: 策略模型对 chosen 回复的 log 概率
    ref_chosen_logps:    参考模型对 chosen 回复的 log 概率
    """
    # 计算隐式奖励差
    chosen_rewards = beta * (policy_chosen_logps - ref_chosen_logps)
    rejected_rewards = beta * (policy_rejected_logps - ref_rejected_logps)

    # DPO 损失
    loss = -F.logsigmoid(chosen_rewards - rejected_rewards).mean()

    # 统计指标
    chosen_reward_mean = chosen_rewards.mean().item()
    rejected_reward_mean = rejected_rewards.mean().item()
    reward_margin = chosen_reward_mean - rejected_reward_mean
    accuracy = (chosen_rewards > rejected_rewards).float().mean()

    return loss, {
        "chosen_reward": chosen_reward_mean,
        "rejected_reward": rejected_reward_mean,
        "margin": reward_margin,
        "accuracy": accuracy,
    }
✅ DPO 的优势
  • 无需训练 RM：省去奖励模型训练阶段
  • 训练稳定：避免 PPO 中的不稳定性问题
  • 超参简单：主要调节 β (通常 0.1-0.5)
  • 计算高效：只需 2 次前向传播 (policy + reference)

搜索增强的对齐数据构造与RLAIF

百川在对齐阶段创新性地将搜索引擎作为外部知识验证工具，用于构造高质量的偏好数据对。在RLAIF（Reinforcement Learning from AI Feedback）流程中，模型生成候选回答后，系统会自动调用搜索引擎检索相关网页，将检索结果作为"证据"输入给另一个评分模型，由评分模型判断回答是否与搜索结果一致、是否存在事实错误。这种"搜索验证"机制显著提升了偏好数据的质量，使得基于这些数据训练的奖励模型在事实性判断上的准确率比纯人工标注方案高出约8%。

在DPO训练中，百川还采用了"搜索增强对比采样"策略：对于同一个问题，分别在有搜索上下文和无搜索上下文条件下生成两个回答，将搜索增强后的回答作为优选（chosen），无搜索的回答作为劣选（rejected）。这种自动化的偏好对构造方式大幅降低了人工标注成本，同时确保了优选回答在事实准确性上的优势。实验结果显示，使用搜索增强偏好数据训练的Baichuan-4在TruthfulQA基准上达到了72.3%的准确率，比不使用搜索增强的基线高出5.8个百分点。

  1. SFT：~100K 高质量指令数据，只对 assistant 部分计算 loss
  2. RLHF：RM + PPO + KL 约束，复杂但效果好
  3. DPO：Baichuan2 采用 DPO，无需单独 RM，更简洁高效
  4. 安全对齐：红队测试 + 安全数据，确保模型输出安全
  5. 数据质量 > 数量：精选 100K 数据效果优于海量低质数据

DPO损失函数的数学推导与收敛性证明

从数学角度深入理解DPO的推导过程对于掌握对齐技术的本质至关重要。DPO的核心推导从RLHF的最优策略出发：给定奖励函数r(x,y)和KL散度约束系数β，RLHF的优化目标为 max_π E_{y∼π}[r(x,y)] - β·KL(π||π_ref)。该优化问题的闭式解可以通过变分法严格证明得到：π*(y|x) = π_ref(y|x)·exp(r(x,y)/β) / Z(x)，其中Z(x) = Σ_y π_ref(y|x)·exp(r(x,y)/β)是配分函数。对等式两边取对数并反解奖励函数，得到 r(x,y) = β·log(π*(y|x)/π_ref(y|x)) + β·log(Z(x))。关键的数学技巧在于：将此表达式代入Bradley-Terry偏好模型P(y_w≻y_l|x) = σ(r(x,y_w) - r(x,y_l))时，配分函数Z(x)在两个奖励的差值中被精确消去，从而得到DPO的损失函数：L_DPO = -E[log σ(β·(log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)))]。这一推导的深刻含义是：DPO无需显式训练奖励模型，而是将偏好优化直接映射到策略空间，同时保持与RLHF在数学上的最优解等价性。

GRPO算法的工作机制与形式化定义

GRPO（Group Relative Policy Optimization）是DeepSeek-R1采用的对齐算法，其形式化定义可以追溯到策略梯度定理和优势函数估计。给定一个问题x，GRPO首先从当前策略π_θ中采样一组候选回答{y_1, y_2, ..., y_G}，然后通过奖励函数r(x,y_i)计算每个回答的得分。与标准PPO不同的是，GRPO不使用全局基线来估计优势函数，而是采用组内相对排名作为优势估计：A_i = (r(x,y_i) - mean(r)) / std(r)。这种组内归一化策略的优势在于消除了奖励函数的绝对尺度影响，使训练更加稳定。从数学证明的角度，可以证明GRPO的梯度估计是策略梯度的无偏估计（在组大小G趋于无穷时），其方差随着G的增大以O(1/G)的速度递减。在实践中，G=8到G=64的采样规模即可获得足够低方差的梯度估计。GRPO的另一个关键创新是将奖励信号分解为多个维度——正确性、格式规范性和安全性——并通过加权求和得到最终奖励，这种多维度奖励设计使得对齐训练可以在多个目标之间进行精细权衡。
