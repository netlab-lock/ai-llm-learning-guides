GQA是Qwen2及之后模型的标准注意力机制，将Query头分组共享Key/Value头，在保持性能的同时大幅降低KV cache内存。

GQA核心参数 (以Qwen2-72B为例)

  • Query Heads: 64
  • KV Heads: 8 (GQA-8)
  • 分组比: 8:1 (每8个Q头共享1组KV)
  • Head Dim: 128
# GQA 计算流程伪代码 Q = Linear(x) # [batch, seq, n_heads, head_dim] → 64 heads K = Linear(x) # [batch, seq, n_kv_heads, head_dim] → 8 heads V = Linear(x) # [batch, seq, n_kv_heads, head_dim] → 8 heads # 将KV heads扩展到与Q heads一致 (repeat) K = K.repeat_interleave(n_heads // n_kv_heads, dim=2) V = V.repeat_interleave(n_heads // n_kv_heads, dim=2) # 标准scaled dot-product attention output = softmax(Q @ K^T / sqrt(head_dim)) @ V

YaRN 在 Qwen 中的实现细节

YaRN (Yet another RoPE extensioN) 是Qwen从Qwen2开始采用的核心上下文扩展技术。其核心思想是将RoPE的频率维度分为三个区间分别处理：高频维度直接外推以保持局部位置精度，低频维度线性插值以压缩到训练范围内，中频维度通过ramp函数平滑过渡。Qwen2.5将基础上下文从4K扩展至128K，Qwen3.5进一步通过YaRN+稀疏注意力扩展至1M token。实现中的关键参数包括attention_factor（调整注意力缩放因子）和beta_fast/beta_slow（控制频率分组边界），训练时使用4K-8K短上下文，推理时动态扩展。

注意力机制的设计直接影响模型的长距离依赖建模能力。Qwen的方案通过优化注意力头的分组策略和KV Cache管理，在显存占用和计算效率之间取得了良好平衡。实验表明，这种设计在保持短距离精度的同时，显著提升了长距离信息检索的准确率。

MLA vs GQA 关键区别

MLA (Multi-head Latent Attention): 将KV压缩到低秩潜空间，推理时从潜变量恢复KV，实现极低KV cache。DeepSeek-V2/V3采用。

GQA: 通过共享KV头减少cache大小，实现简单，Qwen/LLaMA主流选择。

Linear Attention: 用核函数近似softmax，将复杂度从O(n²)降至O(n)，MiniMax在超长序列场景使用。

Qwen系列在注意力机制上的持续演进体现了阿里云对大模型核心技术的深刻理解和前瞻性布局。从Qwen1的纯MHA到Qwen2+的GQA，再到Qwen3的MoE+GQA组合，每一次升级都精准回应了当时的技术瓶颈和应用需求。GQA在Qwen2中的成功应用验证了一个关键洞察：在推理场景中，KV Cache的内存占用往往比计算量更先成为瓶颈，因此通过减少KV头数来降低内存消耗的策略比优化计算本身更具实用价值。Qwen2-72B采用的64个Query头和8个KV头（8:1的分组比）在性能和效率之间取得了近乎最优的平衡——实验表明，与标准MHA相比，GQA仅带来不到1%的性能损失，却将KV Cache内存降低了8倍，这一数据有力地证明了注意力机制设计中"做减法"的可行性。

展望未来，Qwen注意力机制的创新方向正在向三个前沿领域延伸。第一是超长上下文支持，Qwen3.5已将上下文长度推至1M+ token，这要求RoPE位置编码的YaRN扩展方案能够稳定处理远超训练长度的序列，同时保持注意力分布的合理性；第二是MoE架构中的注意力共享优化，Qwen3-235B-A22B的128专家Top-8路由配合共享专家机制，在激活仅22B参数的条件下实现了235B参数量级的模型能力，这种设计的核心挑战在于如何让路由网络学会将不同类型的知识分配给不同的专家，同时让共享专家捕获跨领域的通用知识；第三是与线性注意力等替代方案的融合探索，虽然Qwen目前仍坚持Softmax注意力为主，但MiniMax等公司在混合注意力架构上的成功实践表明，将线性注意力与标准注意力在不同层中交替使用，有可能在保持模型质量的同时将推理复杂度降低一个数量级。阿里云在全模态（Qwen-VL、Qwen-Audio）方面的布局也为注意力机制的跨模态统一设计提供了新的研究空间。

注意力优化的工程实践

注意力优化通过GQA减少KV头数、Flash Attention减少内存访问、KV Cache量化减少显存占用三项技术组合，使长序列推理成为可能。

从数学原理来看，大模型推理的性能瓶颈可以用阿姆达尔定律推导：加速比 S = 1/((1-P) + P/N)，其中P为可并行化比例，N为并行度。当P=0.95且N=8时，理论加速比为S=5.9x。这一数学模型解释了为什么即使增加更多GPU，推理速度也不会线性提升——瓶颈在于不可并行的串行部分（如路由决策和结果聚合）。通过优化串行部分的执行效率，可以将P从0.95提升到0.98，从而使加速比从5.9x提升到12.3x。
