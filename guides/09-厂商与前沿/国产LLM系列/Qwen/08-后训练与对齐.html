8.1 SFT + DPO + GRPO 训练管线

Qwen的训练流程采用多阶段策略，从大规模无监督预训练到高质量有监督微调，每个阶段都有明确的优化目标。预训练阶段使用海量文本数据学习语言的基本规律和世界知识，微调阶段使用精选的指令-回答对训练模型遵循指令和生成高质量输出。

通俗类比：从工程实践看，1 SFT + DPO + GRPO 训练管线就像矿石冶炼——原矿(原始数据)→选矿(过滤)→冶炼(训练)→精炼(微调)。这也是阿里云Qwen3/3.5区别于竞品的关键设计之一。
常见误区：Qwen在8.1 SFT + DP方面的误区是以为看到很多技术就都要学。实际上大部分技术互补非互斥。Qwen建议先搞懂核心概念，其他用到再学。
深入探讨：从可靠性工程角度看，Qwen在8.1 SFT + DP方面的生产部署需要考虑故障恢复、性能降级和容量规划。Qwen的系统设计遵循"故障是常态"原则——每个组件都有备份方案，关键路径有重试机制，非关键功能有降级策略，整体可用性从99.9%提升到99.99%。

Qwen3的后训练采用三阶段管线：

阶段一: SFT (Supervised Fine-Tuning)

  • 目标 教会模型遵循指令的基本能力
  • 数据 百万级高质量指令-回复对
  • 格式 多轮对话、代码生成、数学推理等
  • 数据来源: 人工标注 + 强模型蒸馏

阶段二: DPO (Direct Preference Optimization)

  • 目标 基于偏好数据优化回复质量
  • 数据 chosen/rejected对比对
  • 优势: 无需训练奖励模型，直接优化策略

阶段三: GRPO (Group Relative Policy Optimization)

  • 目标 强化推理和复杂任务能力
  • 特点 组内相对排名替代绝对奖励值
  • 优势: 更稳定的训练，避免奖励黑客
# GRPO 核心算法伪代码 def grpo_step(policy, ref_policy, prompts): # 对每个prompt采样一组回复 responses = [policy.sample(prompt) for _ in range(G)] # 计算组内相对奖励 (无需绝对奖励值) rewards = [reward_fn(r) for r in responses] advantages = (rewards - mean(rewards)) / std(rewards) # 策略优化 (带KL约束) for resp, adv in zip(responses, advantages): ratio = policy.log_prob(resp) / ref_policy.log_prob(resp) loss = -min(ratio * adv, clip(ratio, 1-ε, 1+ε) * adv) loss += β * kl_div(policy, ref_policy) return loss

8.3 Thinking/Non-Thinking 统一模式

Qwen在8.3 Thinking/Non-Thinking 统一模式方面的技术实现体现了其对工程质量和创新的追求。从技术原理到工程落地，Qwen在每个环节都进行了深入的优化。这些优化不仅提升了模型的性能指标，更重要的是提升了模型在实际应用中的可靠性和用户体验。Qwen的技术团队在实践中积累的"隐性知识"是其核心竞争力之一。

双模推理的训练与切换机制

Qwen3的Thinking/Non-Thinking统一模式通过在SFT阶段混合两种格式的数据实现。Thinking数据包含...标签包裹的推理过程，Non-Thinking数据为直接回答格式。训练时模型学习根据输入格式自适应切换。推理时用户通过/think或/no_think指令触发对应模式，也可通过API参数enable_thinking=True全局启用。Thinking Budget参数限制推理token上限，预算耗尽时自动回退到Non-Thinking模式给出当前最优答案。与OpenAI将o1和GPT-4o分为两个模型不同，Qwen3的单一模型方案降低了部署复杂度和运维成本。

GRPO 在 Qwen3 后训练中的具体应用

GRPO (Group Relative Policy Optimization) 是Qwen3后训练阶段的核心强化学习算法。与PPO需要训练独立的critic模型不同，GRPO对每个prompt采样一组回复（通常G=8-16），通过组内相对排名计算优势值（advantage = (reward - mean) / std），无需绝对奖励值。Qwen3的三阶段后训练中，阶段二使用GRPO强化Thinking模式的推理能力——模型在数学、代码等任务上采样多条推理路径，保留高分路径、抑制低分路径；阶段三使用DPO优化Non-Thinking模式的对话质量。GRPO相比PPO的训练稳定性更高，避免了奖励黑客问题，且无需额外的奖励模型训练开销。

通俗类比：理解3 Thinking/Non-Thinking 统一模式的关键在于规模。类比高考备考：大量刷题总结规律查漏补缺，阿里云的Qwen架构正是基于这一思想。
常见误区：Qwen在8.1 SFT + DPO +方面的常见误区是"换框架就能解决所有问题"。实际上框架只是变速箱，瓶颈可能在模型本身、硬件或数据。Qwen的工程团队建议先用profiler定位真正的瓶颈，再针对性优化。
深入探讨：Qwen在8.1 SFT + DP方面的训练稳定性保障涉及梯度裁剪、学习率warmup和z-loss正则化。从数学角度看，训练稳定性本质是优化曲面上的病态条件问题。Qwen通过多种技术组合确保训练过程的平稳收敛。

Qwen3 GRPO: 结合强到弱蒸馏，在SFT阶段注入推理轨迹后用GRPO精调。强调思维/非思维统一。

DeepSeek GRPO: 首次提出GRPO概念，用组内相对排名替代PPO中的critic模型，大幅简化RL流程。

MiniMax CISPO: Clipped Importance Sampling Policy Optimization，通过对重要性权重裁剪实现更稳定的策略更新。

GLM Sequential RL: 将复杂推理任务分解为多个步骤，每步分别施加RL信号，适合长链推理场景。

  1. 核心原理: SFT+DPO+GRPO是Qwen3的三阶段后训练管线，将预训练基座模型转化为可用的对话和推理模型。阶段一SFT（Supervised Fine-Tuning）：使用百万级高质量指令-回复对训练模型遵循指令的基本能力，数据格式包括多轮对话、代码生成、数学推理等，来源为人工标注+强模型蒸馏。阶段二DPO（Direct Preference Optimization）：基于chosen/rejected偏好对比数据优化回复质量，无需训练独立的奖励模型——直接从偏好数据优化策略，损失函数为L=-logσ(β·(logπ(y_w)/π_ref(y_w)-logπ(y_l)/π_ref(y_l)))。阶段三GRPO（Group Relative Policy Optimization）：对每个prompt采样一组回复（G=8-16），通过组内相对排名计算优势值（advantage=(reward-mean)/std），无需绝对奖励值，避免了PPO中critic模型的训练开销和奖励黑客问题。在Qwen3中，三阶段管线进一步细化为：模式融合SFT（统一thinking+non-thinking格式）→ Thinking RL/GRPO（强化推理能力）→ Non-Thinking RL/DPO（优化对话质量）。这三阶段管线是Qwen从基座模型到可用助手的"最后一公里"，直接决定了模型的指令遵循能力、回复质量和推理深度。
    关键要点: ①SFT教会模型遵循指令，DPO基于偏好优化质量，GRPO通过组内相对排名强化推理；②DPO无需奖励模型，比PPO简化了训练流程；③GRPO用组内相对排名替代绝对奖励值，训练更稳定；④Qwen3将三阶段细化为模式融合SFT→Thinking GRPO→Non-Thinking DPO；⑤GRPO避免了奖励黑客（reward hacking）问题。
    实际案例: Qwen3-235B通过三阶段后训练，在Chatbot Arena排名中位居前列。其中GRPO阶段的贡献最为显著——在AIME 2024数学竞赛上，经过GRPO训练的Thinking模式准确率从SFT阶段的45%提升至72%，证明了组内相对排名优化对推理能力的强化效果。

  2. 核心原理: SFT+DPO+GRPO管线和强到弱蒸馏是Qwen3后训练体系中两个互补的组件，分别针对不同规模的模型。SFT+DPO+GRPO管线应用于大模型（32B及以上）的训练——这些模型有足够能力从RL信号中学习，GRPO的组内采样和相对排名优化能有效提升推理能力。强到弱蒸馏应用于小模型（0.6B-8B）——这些模型的探索能力有限，直接RL效率低下，因此从大模型（32B/235B）的推理轨迹中蒸馏知识更为高效。具体实现中，蒸馏在SFT阶段注入教师模型的长思维链（CoT）推理过程，混合off-policy（从教师完整输出学习）和on-policy（学生自主生成+教师评分）两种信号。实验表明，Qwen3-4B通过蒸馏达到Qwen2.5-72B的性能水平，而直接对4B模型进行GRPO训练则远达不到这一效果。两者的核心差异在于：三阶段管线是"自我进化"（通过采样和排名优化自身策略），蒸馏是"知识转移"（将大模型的能力压缩到小模型中）。在Qwen3的实际训练中，大模型先完成三阶段管线，然后其推理轨迹作为蒸馏数据传递给小模型。
    关键要点: ①三阶段管线用于大模型（≥32B），蒸馏用于小模型（≤8B）；②蒸馏混合off-policy和on-policy两种信号效果最佳；③Qwen3-4B蒸馏后达到Qwen2.5-72B性能，证明蒸馏效率远高于直接RL；④三阶段管线是"自我进化"，蒸馏是"知识转移"；⑤两者在训练流程中串联使用——先训练大模型再蒸馏小模型。

    关键要点: ①SFT是基础，DPO/GRPO是锦上添花——SFT数据质量比对齐方法选择更重要；②偏好数据可用强模型自动标注+人工审核的混合方式构造；③DPO的β参数需要在目标任务上通过消融实验确定；④GRPO适合需要强化推理能力的场景但需要可靠的奖励信号；⑤评估需要在真实场景上做A/B测试而非仅依赖基准分数。

    关键要点: ①SFT是格式对齐（教会模型遵循指令），DPO是质量优化（教会模型区分好坏），GRPO是能力强化（教会模型深度推理）；②DPO的数学基础是Bradley-Terry偏好模型的简化；③GRPO用组内相对排名替代绝对奖励值，消除了critic模型的训练开销；④三阶段的数据格式和损失函数各不相同；⑤Qwen3将此管线进一步细化为模式融合SFT→Thinking GRPO→Non-Thinking DPO。

    关键要点: ①Qwen选择DPO+GRPO而非PPO是因为两者都消除了额外模型的训练开销；②KTO适合数据受限场景但效果略弱；③CISPO和Sequential RL是更前沿的探索方向；④纯SFT+DPO适合不需要深度推理的场景；⑤方法选择取决于任务需求和可用数据类型。

    关联知识: PPO的clip机制与GRPO的clip机制数学形式相同，但GRPO用组内均值替代了critic模型的value估计；DPO的理论推导需要理解RLHF的KL约束优化框架；CISPO的重要性采样裁剪与off-policy RL的方差控制技术相关。
