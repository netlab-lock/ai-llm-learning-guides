8.1 SFT + DPO + GRPO 训练管线

Qwen的训练流程采用多阶段策略，从大规模无监督预训练到高质量有监督微调，每个阶段都有明确的优化目标。预训练阶段使用海量文本数据学习语言的基本规律和世界知识，微调阶段使用精选的指令-回答对训练模型遵循指令和生成高质量输出。

通俗类比：从工程实践看，1 SFT + DPO + GRPO 训练管线就像矿石冶炼——原矿(原始数据)→选矿(过滤)→冶炼(训练)→精炼(微调)。这也是阿里云Qwen3/3.5区别于竞品的关键设计之一。
常见误区：Qwen在8.1 SFT + DP方面的误区是以为看到很多技术就都要学。实际上大部分技术互补非互斥。Qwen建议先搞懂核心概念，其他用到再学。
深入探讨：从可靠性工程角度看，Qwen在8.1 SFT + DP方面的生产部署需要考虑故障恢复、性能降级和容量规划。Qwen的系统设计遵循"故障是常态"原则——每个组件都有备份方案，关键路径有重试机制，非关键功能有降级策略，整体可用性从99.9%提升到99.99%。

Qwen3的后训练采用三阶段管线：

阶段一: SFT (Supervised Fine-Tuning)

  • 目标 教会模型遵循指令的基本能力
  • 数据 百万级高质量指令-回复对
  • 格式 多轮对话、代码生成、数学推理等
  • 数据来源: 人工标注 + 强模型蒸馏

阶段二: DPO (Direct Preference Optimization)

  • 目标 基于偏好数据优化回复质量
  • 数据 chosen/rejected对比对
  • 优势: 无需训练奖励模型，直接优化策略

阶段三: GRPO (Group Relative Policy Optimization)

  • 目标 强化推理和复杂任务能力
  • 特点 组内相对排名替代绝对奖励值
  • 优势: 更稳定的训练，避免奖励黑客
# GRPO 核心算法伪代码 def grpo_step(policy, ref_policy, prompts): # 对每个prompt采样一组回复 responses = [policy.sample(prompt) for _ in range(G)] # 计算组内相对奖励 (无需绝对奖励值) rewards = [reward_fn(r) for r in responses] advantages = (rewards - mean(rewards)) / std(rewards) # 策略优化 (带KL约束) for resp, adv in zip(responses, advantages): ratio = policy.log_prob(resp) / ref_policy.log_prob(resp) loss = -min(ratio * adv, clip(ratio, 1-ε, 1+ε) * adv) loss += β * kl_div(policy, ref_policy) return loss

8.3 Thinking/Non-Thinking 统一模式

Qwen在8.3 Thinking/Non-Thinking 统一模式方面的技术实现体现了其对工程质量和创新的追求。从技术原理到工程落地，Qwen在每个环节都进行了深入的优化。这些优化不仅提升了模型的性能指标，更重要的是提升了模型在实际应用中的可靠性和用户体验。Qwen的技术团队在实践中积累的"隐性知识"是其核心竞争力之一。

双模推理的训练与切换机制

Qwen3的Thinking/Non-Thinking统一模式通过在SFT阶段混合两种格式的数据实现。Thinking数据包含...标签包裹的推理过程，Non-Thinking数据为直接回答格式。训练时模型学习根据输入格式自适应切换。推理时用户通过/think或/no_think指令触发对应模式，也可通过API参数enable_thinking=True全局启用。Thinking Budget参数限制推理token上限，预算耗尽时自动回退到Non-Thinking模式给出当前最优答案。与OpenAI将o1和GPT-4o分为两个模型不同，Qwen3的单一模型方案降低了部署复杂度和运维成本。

GRPO 在 Qwen3 后训练中的具体应用

GRPO (Group Relative Policy Optimization) 是Qwen3后训练阶段的核心强化学习算法。与PPO需要训练独立的critic模型不同，GRPO对每个prompt采样一组回复（通常G=8-16），通过组内相对排名计算优势值（advantage = (reward - mean) / std），无需绝对奖励值。Qwen3的三阶段后训练中，阶段二使用GRPO强化Thinking模式的推理能力——模型在数学、代码等任务上采样多条推理路径，保留高分路径、抑制低分路径；阶段三使用DPO优化Non-Thinking模式的对话质量。GRPO相比PPO的训练稳定性更高，避免了奖励黑客问题，且无需额外的奖励模型训练开销。

通俗类比：理解3 Thinking/Non-Thinking 统一模式的关键在于规模。类比高考备考：大量刷题总结规律查漏补缺，阿里云的Qwen架构正是基于这一思想。
常见误区：Qwen在8.1 SFT + DPO +方面的常见误区是"换框架就能解决所有问题"。实际上框架只是变速箱，瓶颈可能在模型本身、硬件或数据。Qwen的工程团队建议先用profiler定位真正的瓶颈，再针对性优化。
深入探讨：Qwen在8.1 SFT + DP方面的训练稳定性保障涉及梯度裁剪、学习率warmup和z-loss正则化。从数学角度看，训练稳定性本质是优化曲面上的病态条件问题。Qwen通过多种技术组合确保训练过程的平稳收敛。

Qwen的数据质量控制与清洗流程

数据质量是模型能力的上限。Qwen建立了多层数据过滤管线：第一层是基于规则的过滤（去除HTML标签、重复段落、过短文本），第二层是基于分类器的质量评分（训练一个二分类器区分高质量和低质量文本），第三层是基于多样性的采样（确保不同领域、语言、风格的文本比例均衡）。这个流程的核心原理是：模型学到的是训练数据分布的统计规律，数据中的噪声和偏差会直接传递到模型行为中。因此，数据清洗的ROI远高于模型架构优化。

Qwen的分布式训练架构与通信优化

大模型训练需要跨数百甚至数千张GPU进行分布式计算。Qwen采用了数据并行(DP)+张量并行(TP)+流水线并行(PP)的三维并行策略。DP将数据分片到不同GPU上独立计算梯度，然后通过AllReduce操作同步；TP将单个矩阵运算分片到多张GPU上，每层需要2次AllReduce通信；PP将模型的不同层分配到不同GPU上，通过micro-batch减少气泡时间。通信效率的关键是计算/通信比——当单层计算时间是通信时间的5倍以上时，并行效率可达80%以上。
