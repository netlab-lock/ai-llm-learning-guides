3.2.2 默认行为

Qwen3 的默认行为是 Thinking 模式。如果用户未显式指定 /no_think，模型会根据问题复杂度自行决定是否进行深度思考。这种自适应能力通过 RL 训练获得。

Thinking Budget 是 Qwen3 引入的推理成本控制机制，允许用户限制模型思考过程的 token 数量。

核心思想：通过设置最大思考 token 数，在推理质量和推理成本之间实现精确控制。这对于 API 服务的计费和延迟管理至关重要。 
# Thinking Budget 使用示例
# 设置最大思考 token 数为 1024
response = model.generate(
    prompt="解释量子纠缠的原理",
    thinking_budget=1024,  # 最多使用1024个token进行思考
)
# 思考过程在达到budget后自动结束，输出最终答案

3.4 训练 Pipeline

Qwen在3.4 训练 Pipeline方面的技术积累体现在数据质量控制和训练策略优化两个维度。在数据方面，Qwen建立了多层数据过滤管线：基于规则的过滤、基于分类器的质量评分和基于多样性的采样。在训练策略方面，Qwen采用了学习率warmup、梯度裁剪和z-loss正则化等技术保障训练稳定性。这些措施确保了模型在大规模训练中的一致性和可复现性。

4 训练 Pipeline的技术原理与实现细节

3.4 训练 Pipeline的训练流程通常分为三个阶段：预训练(Pre-training)在大规模无标注语料上学习语言表示，微调(Fine-tuning)在任务特定数据上适配下游任务，对齐(Alignment)通过人类偏好反馈优化输出质量。每个阶段的数据配比、学习率调度和正则化策略都需要精心设计。

通俗类比：Qwen在技术实现方面的技术路线体现了对效率和质量的双重追求。通过创新的架构设计和精细的工程优化实现了接近最优的模型性能。

阶段1：Cold Start（冷启动）

  • 使用长链式推理（Long-CoT）数据进行 SFT
  • 数据经过精标注，包含详细的推理步骤
  • 建立模型的基础思考能力

阶段2：RL 扩展

  • 使用 GRPO（Group Relative Policy Optimization）算法
  • 大规模强化学习训练，探索更优推理路径
  • 显著提升数学、代码、逻辑推理能力

阶段3：Thinking 模式融合

  • 将 Thinking 和 Non-thinking 数据混合训练
  • 确保两种模式在同一模型中共存
  • 使用 /think 和 /no_think 标记区分

阶段4：Thinking Budget 约束

  • 在不同 Budget 限制下进行训练
  • 使模型学会在有限 token 内完成推理
  • 实现推理质量和成本的可控平衡

3.5 与其他推理方案对比

3.5 与其他推理方案对比的工程实践需要平衡延迟、吞吐量和成本三个维度。Qwen通过多种技术手段实现这一平衡：模型量化将显存占用降低50-75%，连续批处理将GPU利用率提升到90%以上，KV Cache分页管理消除了显存碎片。在实际部署中，Qwen的推理服务可以达到P99延迟<500ms、吞吐量>1000 tokens/s的性能水平。

3.5 与其他推理方案对比的优化技术与工程实践

3.5 与其他推理方案对比的优化可以从三个层面入手：算法层面改进核心计算逻辑(如注意力机制的复杂度从O(n²)优化到O(n))，系统层面优化内存管理和调度策略(如PagedAttention的块管理)，硬件层面充分利用GPU特性(如FlashAttention的tiling策略利用SRAM)。
