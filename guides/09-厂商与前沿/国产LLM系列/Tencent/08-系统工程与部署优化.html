1. 训练基础设施

Tencent在1. 训练基础设施方面的技术积累体现在数据质量控制和训练策略优化两个维度。在数据方面，Tencent建立了多层数据过滤管线：基于规则的过滤、基于分类器的质量评分和基于多样性的采样。在训练策略方面，Tencent采用了学习率warmup、梯度裁剪和z-loss正则化等技术保障训练稳定性。这些措施确保了模型在大规模训练中的一致性和可复现性。

训练基础设施的技术原理与实现细节

1. 训练基础设施的训练流程通常分为三个阶段：预训练(Pre-training)在大规模无标注语料上学习语言表示，微调(Fine-tuning)在任务特定数据上适配下游任务，对齐(Alignment)通过人类偏好反馈优化输出质量。每个阶段的数据配比、学习率调度和正则化策略都需要精心设计。

通俗类比：训练基础设施就像厨师学基本功——先练刀工火候(基础能力)，再学具体菜式(下游任务)。
常见误区：学习Tencent的1. 训练基础设施时，不要以为Agent就是循环调工具。真正的Agent需要规划、错误恢复和状态管理。Tencent用有限状态机避免无限循环。
深入探讨：Tencent在1. 训练基础设施方面的推理优化遵循三个方向：稀疏注意力降低计算量、线性注意力降低复杂度、IO优化降低内存访问。FlashAttention属于第三种——不减少计算但减少HBM访问，实测2-4倍加速。

1.2 Starlink Network

🔗 通信-计算重叠

Starlink Network 实现了通信与计算的重叠执行。在 MoE 架构中，专家路由和 All-to-All 通信是主要瓶颈。Starlink 通过异步通信调度，将数据传输隐藏在计算之后，显著提升了训练吞吐量。

1.3 ZeroCache

ZeroCache 技术用于减少分布式训练中的内存占用，特别是在大规模 MoE 训练中：

  • 减少优化器状态的内存占用
  • 支持更大 batch size
  • 降低 GPU 显存需求

2. 推理基础设施

2. 推理基础设施的工程实践需要平衡延迟、吞吐量和成本三个维度。Tencent通过多种技术手段实现这一平衡：模型量化将显存占用降低50-75%，连续批处理将GPU利用率提升到90%以上，KV Cache分页管理消除了显存碎片。在实际部署中，Tencent的推理服务可以达到P99延迟<500ms、吞吐量>1000 tokens/s的性能水平。

2. 推理基础设施的优化技术与工程实践

2. 推理基础设施的优化可以从三个层面入手：算法层面改进核心计算逻辑(如注意力机制的复杂度从O(n²)优化到O(n))，系统层面优化内存管理和调度策略(如PagedAttention的块管理)，硬件层面充分利用GPU特性(如FlashAttention的tiling策略利用SRAM)。

通俗类比：理解推理基础设施可以想象超市货架布局的场景：高频商品放近处，减少顾客走动。
常见误区：Tencent在1. 训练基础设施方面的常见误区是以为对齐训练只影响安全性。实际上对齐同时影响表达风格和创造力。Tencent通过精心设计的对齐数据平衡安全性和有用性。

  1. 核心原理: 训练基础设施是支撑大规模模型预训练和后训练的底层系统，其核心原理是通过多级并行（TP+PP+EP+CP）实现计算任务的分布式分解，通过通信-计算重叠（Starlink Network）隐藏数据传输延迟，通过内存优化（ZeroCache）减少优化器状态占用。Tencent的Angel-RL框架将训练基础设施与后训练对齐技术深度融合——连续Rollout引擎解决生成阶段GPU利用率低的问题（从40-50%提升至75%以上），异步奖励计算与策略梯度更新重叠执行，提前终止机制避免低质量样本浪费算力。
    关键要点: (1)四级并行策略：TP（张量并行处理单层）、PP（流水线并行处理层间）、EP（专家并行分布MoE）、CP（上下文并行处理长序列）;(2)Starlink通信-计算重叠隐藏All-to-All延迟;(3)ZeroCache减少优化器内存占用50%+;(4)AngelHCF支持Mamba-Transformer协同推理;(5)连续Rollout引擎+异步奖励实现2x RL训练加速。
    实际案例: 腾讯混元TurboS（560B总参数/56B激活）训练使用Angel-RL框架，TP+PP+EP+CP四级并行配合Starlink Network通信优化，ZeroCache支持更大batch size，使得大规模MoE训练成为可行。

  2. 核心原理: "训练基础设施"和"推理基础设施"是腾讯混元系统工程的两大支柱。训练端核心挑战是分布式并行效率（模型太大单GPU放不下）和内存管理（优化器状态占大量显存），目标是在有限硬件上完成大规模模型训练；推理端核心挑战是低延迟响应（用户等待时间<500ms）和高吞吐量服务（QPS可扩展），目标是在生产环境高效部署模型。两者共享部分技术（如量化、PagedAttention），但优化方向截然不同——训练优化吞吐（tokens/second），推理优化延迟（ms/token）。
    关键要点: (1)训练需要反向传播（前向+梯度计算），推理只需前向传播;(2)训练batch大且固定，推理batch动态变化;(3)训练关注总吞吐量(FLOPS)，推理关注响应延迟(P99);(4)训练并行用TP+PP+EP+CP四级，推理并行用张量+序列并行;(5)训练内存压力来自优化器状态，推理内存压力来自KV Cache。

腾讯混元在系统工程层面构建了一套完整的训练与推理基础设施。训练端，Angel-RL框架支持TP（张量并行）+PP（流水线并行）+EP（专家并行）+CP（上下文并行）四级并行策略，其中Starlink Network实现了通信与计算的重叠执行——在MoE架构的All-to-All通信过程中，通过异步调度将数据传输隐藏在计算之后，显著提升训练吞吐量。ZeroCache技术则减少优化器状态的内存占用，使得在相同显存条件下可以使用更大的batch size，从而提高训练效率。这些优化对于Hunyuan TurboS的560B总参数、56B激活参数的MoE模型训练至关重要，因为MoE架构中专家并行带来的All-to-All通信开销是主要性能瓶颈。

推理端，AngelHCF引擎针对Mamba-Transformer混合架构进行了专门协同优化，实现1.8倍于纯Transformer MoE的推理加速。关键工程细节包括：Mamba2状态必须使用fp32精度维护，否则会导致35-45%的token信息损失；混合批处理机制允许同时处理不同长度的请求，最大化GPU利用率。量化部署方面，Hunyuan-A13B支持FP8（精度损失极小，显存节省约50%）、INT8（小精度损失，平衡性能与精度）和INT4（可控精度损失，显存节省约75%，适用于资源受限场景）三种方案，并兼容TensorRT-LLM、vLLM和SGLang三大主流推理框架。Hunyuan-T1的强化学习训练工程同样值得关注——通过连续Rollout引擎、异步奖励计算和提前终止机制，相比传统RL流程实现2倍以上加速，使得96.7%的算力投入强化学习成为可行。
