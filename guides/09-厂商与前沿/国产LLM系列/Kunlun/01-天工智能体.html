天工智能体的产品定位与技术愿景

天工智能体平台代表了昆仑万维从基础大模型向AI应用生态的战略跃迁。不同于单一聊天机器人产品，天工智能体将五大专业Agent整合为统一工作空间，覆盖文档、演示、数据、音频和网络检索五大知识工作场景。其核心设计理念是"Agent即服务"——每个Agent不仅是独立的AI工具，更是可与其他Agent协同编排的工作流节点。昆仑万维通过Skywork OR1-32B和R1V4等自研模型为平台提供底层推理能力，同时将模型以Apache 2.0和MIT协议完全开源，构建开放的Agent生态。

通俗类比：理解概述的最佳方式是类比矿石冶炼：原矿(原始数据)→选矿(过滤)→冶炼(训练)→精炼(微调)。这正是昆仑万维AI Agent的技术基础。
常见误区：学习Kunlun的五大超级智能体详解时，不要以为Agent就是循环调工具。真正的Agent需要规划、错误恢复和状态管理。Kunlun用有限状态机避免无限循环。

天工智能体（Skywork Super Agents）是昆仑万维推出的全能AI工作空间平台。不同于单一功能的AI工具，天工智能体将5个专业领域的AI Agent集成在一个统一平台中，用户可以在同一界面内完成文档写作、幻灯片制作、数据分析、播客创作和网络信息检索等任务。

核心理念：一个平台、五大智能体、覆盖知识工作全流程。每个Agent既可独立使用，也能协同工作 — 例如Web Agent搜索资料后，Docs Agent自动整合成报告，PPT Agent再将其转化为演示文稿。

📄 Docs Agent — 文档智能体

长文写作、报告生成、摘要提炼、多语言翻译。支持Markdown和多种文档格式导出。可基于用户提供的资料自动生成结构化文档。

📊 PPT Agent — 演示智能体

底层模型：Skywork OR1-32B

底层模型：Skywork OR1-32B的设计哲学体现了Kunlun对模型效率和能力平衡的深刻理解。通过引入混合专家(MoE)架构、分组查询注意力(GQA)等技术，Kunlun在保持模型能力的同时显著降低了推理成本。这种"以效率换规模"的技术路线，使得Kunlun能够在有限的计算资源下部署更大、更强的模型。

OR1-32B的强化学习训练与推理能力

Skywork OR1-32B采用了基于GRPO（Group Relative Policy Optimization）的强化学习训练策略，无需显式训练价值模型即可优化策略。在数学推理方面，OR1-32B在MATH-500上达到与DeepSeek-R1-32B相当的水平；在代码生成方面，于LiveCodeBench上的Pass@1指标同样对标R1系列。32B的参数规模使其可在单张A100-80G或2张RTX 4090上运行，大幅降低了企业部署门槛。模型支持最大32K上下文窗口，配合YaRN外推可扩展至128K。

通俗类比：底层模型：Skywork OR1-32B的核心挑战在于稳定性。类比酿酒：粮食(数据)+酒曲(算法)+时间(算力)=好酒(模型)，昆仑万维的方案优雅地解决了这个问题。
常见误区：不仅需要关注技术实现，还需要考虑数据质量、团队能力和业务需求。Kunlun的技术选型遵循"技术就绪度"(TRL)评估框架——只有TRL≥6的技术才会进入生产系统。过度追求前沿技术而忽略工程可行性是常见的失败原因。

Skywork R1V4 SOTA Multimodal Agent

Agent系统的关键挑战在于错误恢复和长程规划。当工具调用失败时，Agent需要分析失败原因并尝试替代方案。当任务复杂时，Agent需要将任务分解为可管理的子任务，并按最优顺序执行。Kunlun通过强化学习训练Agent的规划和反思能力。

Kunlun的Agent能力使模型能够与外部世界交互——调用API、执行代码、操作界面。Agent的核心组件包括：规划器（将复杂任务分解为子步骤）、工具选择器（根据任务选择合适的工具）、执行器（调用工具并处理结果）、反思器（评估执行结果并调整策略）。

通俗类比：Kunlun在天工智能体 / Skywork Supe方面的工程实践就像医生看病——先做全面检查(profiling)找到病因(瓶颈)，再对症下药(针对性优化)，最后复查(回归测试)确认疗效。<
常见误区：关于Kunlun的五大超级智能体详解，不要以为LoRA能完全替代全量微调。LoRA在95%任务接近全量微调，但改变行为模式时全量更好。Kunlun根据任务复杂度选择。

昆仑万维最新开源的 MoE 多模态智能体模型，~30B 总参数 / 3B 激活参数，首创"Thinking With Images"范式，实现视觉推理与图像操作的深度融合。

核心创新：Thinking With Images

  • 主动图像操控 — 模型能主动对图像进行裁剪、缩放等操作，而非被动接收固定视觉输入
  • 多阶段裁剪 + 区域注意力 — Multi-stage cropping 与 region attention 结合，精细感知图像细节
  • 交错推理 — 视觉推理、图像操作、Web 搜索三者无缝交替，形成完整的推理链路
  • 涌现式长程推理 — 能自主执行 10+ 工具调用的长程推理任务，无需人工干预

训练方法

纯 SFT 训练：不依赖强化学习（RL），仅使用不到 30,000 条高质量轨迹数据进行 SFT 训练，即达到 SOTA 性能。这证明了数据质量远比数据规模重要。

性能指标

  • MMSearch: 66.1 — SOTA，多模态搜索任务新纪录
  • FVQA: 67.2 — SOTA，视觉问答新纪录
  • 全面超越 Gemini 2.5 Flash — 在全部 11 项评测指标上均胜出
  • MoE 高效架构 — ~30B 总参数 / 3B 激活参数，兼顾能力与效率
  • MIT License — 完全开源，可自由商用

MuisCoT — 全球首个音乐推理模型

MuisCoT（Music Chain-of-Thought）将大语言模型中的"思维链"推理范式引入音乐生成领域。不同于传统的端到端音乐生成模型，MuisCoT能够像人类作曲家一样，先"思考"音乐的结构、调性、节奏和情感走向，再逐步生成音乐，实现了更高质量、更可控的AI作曲。

Mureka V6 — AI音乐服务平台

Mureka是昆仑万维面向创作者和企业的AI音乐商业平台，最新V6版本提供：

  • 文本生成音乐 — 用自然语言描述想要的音乐风格和情感
  • 风格迁移 — 将已有音乐转换为不同风格
  • 伴奏生成 — 为人声或旋律自动生成伴奏
  • 多风格支持 — 覆盖流行、古典、电子、嘻哈等多种音乐类型
  • 商用授权 — 提供清晰的版权和商用授权方案

  1. 核心原理: 天工智能体平台的核心原理是"多Agent协同"——将五大专业智能体（Docs/PPT/Excel/Podcast/Web Agent）统一集成在一个工作空间中，每个Agent既可独立处理特定类型的知识工作，也能通过标准化的中间表示格式进行数据传递、串联成自动化流水线。平台以Skywork OR1-32B和R1V4为底层推理引擎，Apache 2.0/MIT协议完全开源。其设计理念区别于单一聊天机器人——Agent不仅是工具调用循环，而是具备规划、执行、反思和错误恢复能力的完整系统，采用有限状态机避免无限循环。

    实际案例: 市场研究报告场景：用户需要制作一份行业竞争分析报告。传统流程需用搜索引擎查资料→用文档工具写报告→用幻灯片工具做演示→用音频工具录制解读播客，各环节孤立。天工智能体通过"端到端自动化流水线"：Web Agent实时联网搜索行业数据 → Docs Agent自动生成结构化市场分析报告 → PPT Agent制作包含图表的汇报演示 → Podcast Agent将核心内容转化为对话式播客，将数小时的多工具协作流程压缩到分钟级别。

    关键要点: ①推理延迟累积——多个Agent串联时推理延迟叠加 ②显存竞争——30B模型权重在所有并发Agent间共享 ③工具调用不可靠——外部API/网络搜索的成功率影响整体SLA ④Agent间数据传递格式不一致导致信息损失 ⑤错误在Agent间级联传播
