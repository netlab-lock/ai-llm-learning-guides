5.2 RLHF对齐

InternLM在5.2 RLHF对齐方面的技术路线涵盖了从SFT到RLHF/DPO的完整对齐流程。SFT阶段通过高质量的指令-回答对训练模型的指令跟随能力；RLHF阶段通过奖励模型引导模型的生成分布向人类偏好方向偏移。InternLM的创新在于将安全对齐和能力增强统一到同一个训练框架中，避免了"对齐税"——即对齐训练导致模型能力下降的问题。

RLHF（基于人类反馈的强化学习）是InternLM采用的主要对齐方法之一，包含三个阶段：

奖励模型训练

使用人类标注的偏好数据对训练奖励模型（RM）。标注员对同一提示的多个回答进行排序，RM学习预测人类偏好的分数。

# RM训练核心
# 偏好数据格式: (prompt, chosen_response, rejected_response)
# 训练目标: Bradley-Terry模型
loss = -log(sigmoid(rm(chosen) - rm(rejected)))

# InternLM RM架构: 基于InternLM-base + 线性head
# 输入: 完整prompt+response
# 输出: 标量reward score

PPO优化

使用PPO算法优化语言模型策略，最大化奖励模型的得分，同时通过KL散度惩罚防止策略偏离SFT模型过远。

# PPO目标函数
objective = E[r(x,y)] - β * KL(π_θ || π_ref)

# InternLM PPO超参:
clip_range: 0.2
kl_coeff: 0.05
batch_size: 512
mini_batch: 64
ppo_epochs: 4
value_loss_coeff: 0.5

5.3 DPO直接偏好优化

5.3 DPO直接偏好优化的核心挑战是在安全性和有用性之间找到平衡。InternLM通过多层安全机制实现这一目标：输入过滤、输出审查和模型内化安全。在训练数据方面，InternLM精心设计了包含"安全但有用"正例的对齐数据集，确保模型不会因为过度对齐而变得过于保守。这种精细化的对齐策略是InternLM模型在安全性和实用性上都表现优秀的关键。

DPO的理论基础与实现机制

DPO通过证明RLHF最优策略的解析解，实现了无需奖励模型的直接偏好优化。核心公式π*(y|x)∝π_ref(y|x)×exp(r(x,y)/β)表明最优策略可从偏好数据和参考模型直接计算，大幅简化了对齐训练流程。关键超参β控制模型偏离参考模型的程度，需要在安全性和有用性之间寻找平衡。

通俗类比：DPO就像"跳过考试阅卷老师，直接看考生互相PK"。RLHF需要先训练一个"阅卷老师"（奖励模型），再用它来指导学生（语言模型）学习。DPO更直接——拿两份答卷直接比较，"这份比那份好"，然后让模型自己学会偏好。省去了训练奖励模型的中间环节，就像跳过裁判，直接让选手看比赛录像自学。
常见误区：InternLM在5.1 监督微调（SFT方面的误区是以为MoE计算量等于总参数量。实际上每次只激活部分专家。InternLM的MoE设计利用稀疏激活实现高参数效率低推理成本。
深入探讨：DPO的理论核心是证明了RLHF的最优策略可以解析求解：π*(y|x) ∝ π_ref(y|x) × exp(r(x,y)/β)。这意味着给定偏好数据对和参考模型，可以直接算出最优策略，无需显式训练奖励模型和PPO迭代。但这个推导依赖Bradley-Terry偏好模型假设——即人类偏好可以用标量奖励函数建模。实践中，人类偏好往往是非传递的（A>B>C>A），这违反了BT假设。KTO和IPO等改进方法正在尝试突破这一限制。

DPO是RLHF的简化替代方案，无需训练独立的奖励模型，直接从偏好数据中优化策略。InternLM在部分场景中采用DPO替代PPO。

def dpo_loss(policy_chosen_logps, policy_rejected_logps,
             ref_chosen_logps, ref_rejected_logps, beta=0.1):
    logits = beta * (
        (policy_chosen_logps - ref_chosen_logps) -
        (policy_rejected_logps - ref_rejected_logps)
    )
    return -F.logsigmoid(logits).mean()

5.6 对齐数据构造

InternLM在5.6 对齐数据构造方面的技术积累体现在数据质量控制和训练策略优化两个维度。在数据方面，InternLM建立了多层数据过滤管线：基于规则的过滤、基于分类器的质量评分和基于多样性的采样。在训练策略方面，InternLM采用了学习率warmup、梯度裁剪和z-loss正则化等技术保障训练稳定性。这些措施确保了模型在大规模训练中的一致性和可复现性。

5.6 对齐数据构造的工程实践与性能优化

在实际工程应用中，5.6 对齐数据构造的实现需要在准确率、推理速度和资源消耗之间找到最优平衡。关键优化策略包括：(1)算子融合(Operator Fusion)将多个小kernel合并为一个大kernel，减少GPU启动开销和中间结果的显存读写；(2)混合精度计算在保持数值稳定性的前提下使用FP16/BF16，将显存占用降低约50%；(3)动态批处理(Dynamic Batching)根据请求的序列长度自适应调整batch大小，最大化GPU利用率。

开源工具链支撑的后训练流水线

InternLM的后训练流水线深度整合了上海AI实验室开源的一系列工具链，形成了从数据准备到模型部署的完整闭环。在SFT阶段，团队使用开源的XTuner框架进行参数高效微调，该框架原生支持InternLM系列模型，提供了LoRA、QLoRA和全量微调三种模式的统一接口。XTuner的核心优势在于其"自动并行策略"——根据可用GPU数量和模型规模自动选择最优的张量并行和数据并行配置，用户无需手动调整分布式训练参数。在数据处理方面，InternLM开源了数据清洗工具包（InternLM-Utils），包含文本去重（基于MinHash）、质量评分（基于困惑度过滤）和安全检测（基于关键词和分类器）等模块，这些工具以流水线方式组合，可处理TB级别的原始语料。在偏好对齐阶段，团队开源了RLHF训练框架，支持PPO、DPO和KTO三种对齐方法，其中PPO实现采用了创新的"异步奖励模型推理"策略——奖励模型的评分计算与策略模型的训练迭代并行执行，将训练吞吐量提升了约40%。

InternLM开源工具链的另一大特色是其完善的评估和调试工具集。InternLM-Eval是一个标准化的模型评估框架，内置了50余个评测基准（包括MMLU、HumanEval、GSM8K、C-Eval等），支持一键式批量评测和结果对比分析。该框架还提供了"能力雷达图"可视化功能，从语言理解、数学推理、代码生成、知识问答等六个维度对模型能力进行综合画像。在调试方面，InternLM-Insight工具可以实时监控训练过程中的梯度分布、注意力权重变化和损失函数曲线，帮助开发者快速定位训练异常（如梯度爆炸、loss spike等）。这些开源工具的协同使用，使得从零开始训练一个InternLM系列模型的门槛大幅降低——一个具备基本深度学习知识的开发者，借助完整的工具链文档和示例代码，可以在约两周内完成从数据准备到模型发布的全流程。

在人类偏好对齐方面，InternLM同时支持RLHF（PPO）、DPO、KTO和GRPO四种方法，根据不同场景灵活选择。RLHF/PPO方案效果上限最高但训练复杂度也最大——需要先训练奖励模型（基于Bradley-Terry偏好模型），再用PPO算法优化语言模型策略，通过KL散度惩罚防止策略偏离SFT模型过远。DPO方案作为RLHF的简化替代，直接从偏好数据中解析求得最优策略，无需显式训练奖励模型，其理论核心是证明了最优策略π*(y|x)与参考模型和奖励函数之间的解析关系。InternLM在对齐实践中的关键创新是\"条件对齐\"机制——模型可根据系统提示中的安全等级标签（宽松/标准/严格）动态调整输出策略，\"宽松\"模式适合创意写作、\"标准\"模式适合通用对话、\"严格\"模式适合医疗法律等敏感场景，这种动态安全策略避免了\"一刀切\"的过度拒绝问题。对齐数据构造采用迭代式数据增强方法：用当前最好的模型生成候选回答，用奖励模型打分筛选，再用筛选后的数据训练更好的模型，形成正向飞轮。安全对齐方面，InternLM采用宪法AI（Constitutional AI）方法，用预定义的安全规则让模型自我批评和修正输出，同时保持安全样本与能力样本1:5到1:10的比例，确保模型不会因过度对齐而丧失正常能力。

从工程实践的角度审视InternLM的后训练与对齐体系，其最具特色的创新是"条件对齐"机制与"对齐税最小化"策略的有机结合。条件对齐机制通过在系统提示中嵌入安全等级标签（宽松/标准/严格），使同一个基础模型能够根据不同应用场景动态调整安全阈值——在创意写作场景中采用"宽松"模式以最大化表达自由度，在通用对话中采用"标准"模式以平衡安全性和有用性，在医疗法律等敏感场景中采用"严格"模式以最大限度降低风险。这种"一个模型、多套策略"的设计避免了为每个场景训练独立模型的高昂成本，同时解决了传统"一刀切"对齐方案中普遍存在的过度拒绝问题。对齐税最小化策略则通过精心设计的数据配比（安全样本与能力样本保持1:5到1:10的比例）和渐进式对齐流程（先SFT建立基础能力，再RLHF精细调优），确保对齐过程不会显著降低模型在预训练阶段获得的知识和推理能力。在偏好对齐方法的选择上，InternLM同时支持RLHF/PPO、DPO、KTO和GRPO四种方案，其中PPO实现采用了创新的"异步奖励模型推理"策略——将奖励模型的评分计算与策略模型的训练迭代并行执行，将训练吞吐量提升了约40%。DPO作为更轻量的替代方案，在快速迭代场景中被广泛使用，其核心超参β（温度系数）的默认值为0.1，过大导致模型过度保守、过小导致模型偏离太远。安全对齐方面，InternLM借鉴Anthropic的Constitutional AI思想，用预定义的安全规则让模型自我批评和修正输出，同时配合红队测试和对抗训练持续提升模型的鲁棒性，形成了"规则自检+红队攻击+持续监控"的三层安全防护体系。

综合本章所介绍的InternLM后训练与对齐技术体系，可以总结出上海AI实验室在大模型对齐领域的三个核心设计理念及其对国产大模型行业的启示。第一，"对齐税最小化"是后训练的核心约束条件——InternLM的实践表明，对齐训练存在一个"甜蜜点"，超过这个点模型的安全性提升了但有用性会急剧下降（过度拒绝正常请求），因此需要通过精心设计的数据配比（安全样本与能力样本1:5到1:10）和渐进式对齐流程来控制对齐税在5%以内。第二，"条件对齐"代表了对齐技术从"硬编码"向"软控制"的范式转变——传统对齐将安全行为固化到模型参数中，而InternLM让模型根据系统提示中的安全等级标签动态调整输出策略，这种设计使得同一个模型能够服务于从创意写作到医疗咨询的多样化场景，避免了为每个场景训练独立模型的高昂成本。第三，"迭代式数据增强"形成了对齐质量的正向飞轮——用当前最好的模型生成候选回答，用奖励模型打分筛选，再用筛选后的数据训练更好的模型，这种闭环方法论使得对齐效果能够持续改进而非一次性优化。在偏好对齐方法的选择上，InternLM同时支持RLHF/PPO（效果上限最高但训练复杂）、DPO（简单稳定但离线方法）、KTO（无需配对数据）和GRPO（自生成奖励）四种方案，开发者可根据具体场景灵活选择。对于国产大模型行业而言，InternLM的对齐实践表明：好的对齐不是简单的"加安全过滤器"，而是需要在数据构造、训练策略、评估方法和部署机制四个层面进行系统性的设计和优化。
