1.1 Softmax 注意力 vs 线性注意力

1.2 线性注意力的数学原理

Softmax 注意力:
Attention(Q,K,V) = softmax(QK^T/√d) · V
复杂度: O(n² · d)

线性注意力:
Attention(Q,K,V) = φ(Q) · (φ(K)^T · V)
先计算 K^T · V (状态矩阵)，再乘 Q
复杂度: O(n · d²) — 当 d < n 时更高效

Lightning Attention 的 I/O 优化:
- 分块计算，优化 GPU 内存访问模式
- 利用 tile-level 并行性
- Delta decay 机制保持序列建模能力

2. 🏗️ MoE 架构

MoE架构的核心挑战在于负载均衡和专家利用率。MiniMax通过精心设计的路由策略和辅助损失函数，确保各专家的激活频率接近均匀分布，避免了"专家坍缩"问题。同时，高效的All-to-All通信机制使得专家并行能够在多GPU环境下实现接近线性的扩展效率。

MiniMax的MoE（Mixture of Experts）架构通过条件计算实现了"大参数量、低计算量"的帕累托最优。路由网络根据输入特征动态选择最相关的专家子网络，每个token只激活部分专家进行计算。这种设计使得模型能够拥有更大的知识容量，同时保持与密集模型相当的推理成本。

CISPO: Clipped Importance Sampling Policy Optimization — 专为 Lightning Attention 设计的 RL 算法，利用线性注意力的效率优势加速 RL 训练。

3.1 为什么需要 CISPO

  • 传统 PPO/GRPO 在长序列 RL 训练中计算成本高
  • Lightning Attention 的 O(n) 特性天然适合长序列
  • CISPO 充分利用线性注意力的效率进行 RL 训练

3.2 CISPO 的核心思想

CISPO 结合了:
1. 重要性采样 (Importance Sampling)
2. 裁剪机制 (Clipping)
3. 线性注意力的高效序列处理

结果: RL 训练在 512 H800 GPU 上仅需 3 周
成本: 仅 $534,700

  1. 核心原理: Lightning Attention原理的核心是用线性注意力替代softmax注意力，将O(n²)复杂度降为O(n)，实现超长上下文的高效推理。数学原理基于核函数近似——将softmax(QK^T/√d)·V分解为φ(Q)·(φ(K)^T·V)，先计算K^T·V得到固定大小的状态矩阵S（d×d维），再与Q相乘得到输出。这种"先累积状态、后查询"的计算模式天然支持增量推理——每一步Decode只需更新状态矩阵S（S_t = S_{t-1} + k_t ⊗ v_t）而无需重新计算全部历史。Lightning Attention进一步通过IO感知的分块计算优化GPU内存访问：将序列分为256大小的block，block内使用标准因果注意力（利用GPU SRAM高带宽），block间通过递推形式传播状态矩阵，将HBM访问次数减少约70%，在H100上达到理论峰值82%。delta decay机制（S_t = δ·S_{t-1} + k_t ⊗ v_t）解决了线性注意力在长序列中的记忆衰退问题。
    关键要点: (1)核函数近似——φ(Q)·(φ(K)^T·V)替代softmax(QK^T)·V (2)增量推理——每步仅更新状态矩阵 (3)分块计算——block内标准注意力+block间递推 (4)delta decay——衰减因子解决记忆衰退 (5)混合架构——7:1的Lightning:Softmax比例平衡效率与质量
    实际案例: 在MiniMax-M1中，456B总参数采用混合架构（每8层为一组，7层Lightning Attention + 1层Softmax Attention），实现了1M原生上下文和4M外推推理，在RULER 128K测试达到95.2%（超过GPT-4o的91.8%）。
