08 混合精度量化策略

通俗类比：混合精度量化就像"聪明的行李打包"——你不会把所有衣服都用同样大小的压缩袋。大衣(关键层)用大袋(FP16/INT8)保持品质，T恤(非关键层)用小袋(INT4)节省空间。混合精度就是给模型的每一层"定制压缩方案"，在体积和品质之间找到最优平衡。

一、为什么需要混合精度？

一、为什么需要混合精度？

对精度的敏感度差异。有些层（如注意力的QK投影）对误差极敏感，量化后质量暴跌；有些层（如FFN的中间层）对误差很鲁棒，量化几乎无影响。混合精度量化的核心思想是：敏感层高精度，鲁棒层低精度。

不同层对量化的敏感度差异（以 Llama 3 8B 为例） 层名 INT4 量化后 ppl 变化 敏感度 ───────────────────────────────────────────────────── Embedding +1.2 ★★★★★ (极敏感) Attention Q/K 投影 +0.8 ★★★★☆ Attention V 投影 +0.3 ★★★☆☆ Attention Output +0.2 ★★☆☆☆ FFN Gate (up_proj) +0.15 ★★☆☆☆ FFN Down +0.1 ★☆☆☆☆ (最鲁棒) LayerNorm +0.05 ★☆☆☆☆ LM Head +0.6 ★★★★☆ 结论：Embedding 和 Q/K 投影是最不能量化的层！
常见误区： 一、为什么需要混合精度？ 整体架构 ┌─────────────────────────────────────────────┐ │ 输入层 (Input) │ ├─────────────────────────────────────────────┤ │ 处理层 (Processing) │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ │ │ 模块 A │→│ 模块 B │→│ 模块 C │ │ │ └──────────┘ └──────────┘ └──────────┘ │ ├─────────────────────────────────────────────┤ │ 输出层 (Output) │ └─────────────────────────────────────────────┘
strong>很多人以为"模型越大，量化越安全"。实际上，大模型（如70B）的某些关键层（特别是Embedding和LM Head）对量化更敏感，因为它们负责"词汇空间"的映射，精度损失会直接体现为输出质量下降。小模型反而因为参数冗余少，量化时每层都需要更谨慎。

二、SmoothQuant：激活量化的救星

二、SmoothQuant：激活量化的救星

激活值量化困难的原因：存在outlier 通道（某些通道的激活值比其他通道大 100x+）。SmoothQuant 把激活的 outlier "转移"到权重上。

SmoothQuant 平滑公式：
Y = X · W = (X · diag(s)-1) · (diag(s) · W) = X̂ · Ŵ

sj = max(|Xj|)α / max(|Wj|)1-α

α = 0.5（默认）：激活和权重各承担一半难度
α → 1：更多难度转给权重
α → 0：更多难度留给激活
SmoothQuant 效果： 量化前： 激活: [1, 2, 150, 3, 1, 200, 2, 1] ← 有 outlier (150, 200) 权重: [0.5, 0.3, 0.4, 0.6, 0.5, 0.3, 0.4, 0.6] ← 均匀 平滑后（α=0.5）： 激活: [1, 2, 12, 3, 1, 14, 2, 1] ← outlier 被压平 权重: [0.5, 0.3, 5, 0.6, 0.5, 4.3, 0.4, 0.6] ← 权重承担缩放 → 两者都可以安全量化为 INT8
深入探讨：SmoothQuant的核心洞察是：激活值的outlier不是随机的——它们出现在固定的通道上（称为"outlier channels"）。这意味着我们可以提前知道哪些通道有outlier，并预先做缩放处理。这个发现来自LLM.int8()的研究者Tim Dettmers，他发现LLM中大约0.1%的通道是"outlier通道"，它们在不同输入中始终保持异常大的值。SmoothQuant利用这个规律，用数学方式把outlier"转移"到权重上，使得激活和权重都可以安全量化。

三、LLM.int8()

bitsandbytes 的方案：99.9% 的值用 INT8，0.1% 的 outlier 用 FP16。

LLM.int8() 混合精度： 输入矩阵 X │ ├── 检测 outlier 通道（绝对值 > 阈值，通常是 6.0） │ ├── 99.9% 通道 → INT8 矩阵乘法（快，利用 INT8 Tensor Core） │ └── 0.1% outlier 通道 → FP16 矩阵乘法（准，避免溢出） │ └── 合并两个结果 → 最终输出 原理：outlier 通道数量极少（<0.1%）， 但它们包含的信息很重要（负责"方向"）， 所以单独用高精度处理

通俗类比：LLM.int8()的策略就像"重要文件单独保管"——99.9%的普通文件放在普通文件柜(INT8)，0.1%的重要文件放在保险柜(FP16)。虽然保险柜更贵更占空间，但因为重要文件很少，总体成本增加很小，却能确保重要信息不丢失。

四、KV-Cache 量化

四、KV-Cache 量化

推理时 KV-Cache 是显存大户。长上下文场景下，KV-Cache 可能比模型本身还大。

KV-Cache 显存 = 2 × num_layers × num_heads × head_dim × seq_len × batch × dtype_bytes

Llama 3 70B, seq_len=128K, batch=1:
= 2 × 80 × 64 × 128 × 131072 × 1 × 2 (BF16)
≈ 34 GB（比模型参数还大！）

FP8 KV-Cache：17 GB（减半）
INT4 KV-Cache：8.5 GB（四分之一）

五、双重量化 (Double Quantization)

五、双重量化 (Double Quantization)

QLoRA 的技巧：量化参数本身也有开销（FP32 的 scale）。双重量化把这些 scale 也量化了。

双重量化原理： 正常量化：每 64 个权重共享一个 FP32 scale [64 weights] → FP32 scale → INT4 值 scale 开销：4 bytes / 64 weights = 0.0625 bytes/weight 双重量化：scale 本身也量化为 FP8 [64 weights] → FP32 scale → FP8 scale (per-block) + FP32 rescale 开销降到：~0.01 bytes/weight 总效果（7B模型）： 普通 INT4：0.0625 × 7B = 0.44 GB 开销 双重量化：0.01 × 7B = 0.07 GB 开销 节省：0.37 GB（约5%模型大小）
深入探讨：双重量化的关键洞察是：量化scale本身的变化范围很小——大部分scale值集中在某个小范围内。这意味着用FP8甚至INT8来存储scale几乎不会损失精度。这个技巧在QLoRA中首次提出，后来被广泛应用于各种量化方案。看似微小的0.37GB节省，在8卡并行时意味着2.96GB显存释放，可能刚好够用。

七、层间精度分配策略

混合精度量化的数学本质：Pareto最优精度分配推导

混合精度量化的底层原理是一个约束优化问题。设模型有L层，第i层精度为b_i bits，量化后第i层的精度损失为ε_i(b_i)。为什么需要混合精度？推导：如果所有层用相同比特数b，总精度损失 = Σε_i(b)。但不同层的ε_i(b)曲线形状不同——敏感层ε下降慢，鲁棒层ε下降快。Pareto最优解满足min Σε_i(b_i)，约束 Σb_i ≤ B_total。数学本质上这等价于拉格朗日乘子法：∂ε_i/∂b_i = λ（常数），即所有层的精度边际改善相等时达到最优分配。

策略1：均匀量化（最简单但最差） 所有层 → INT4 策略2：首尾高精度（简单有效） 前2层 + 后2层 → INT8 其他层 → INT4 策略3：敏感度感知（推荐） 根据每层的量化敏感度评分，动态分配精度 Embedding → FP16 Attention Q/K → INT8 Attention V/O → INT4 FFN layers → INT4 LM Head → INT8 策略4：Pareto最优（前沿研究） 使用搜索算法找到给定预算下的最优精度分配 目标：min(ppl) s.t. total_bits ≤ budget 
# 敏感度感知的混合精度量化示例
import torch

def compute_layer_sensitivity(model, calibration_data):
    """计算每层对量化的敏感度"""
    sensitivities = {}
    for name, module in model.named_modules():
        if isinstance(module, torch.nn.Linear):
            # 保存原始权重
            original_weight = module.weight.data.clone()
            
            # 模拟 INT4 量化

            module.weight.data = quantized

            sensitivities[name] = loss

def assign_precision(sensitivities, bit_budget):
    """根据敏感度分配精度"""
    sorted_layers = sorted(sensitivities.items(), key=lambda x: -x[1])
    assignments = {}
    remaining_bits = bit_budget
    
    for name, sensitivity in sorted_layers:
        if sensitivity > HIGH_THRESHOLD and remaining_bits >= 8:
            assignments[name] = 8  # 高敏感层用 INT8
            remaining_bits -= 8
        elif sensitivity > MED_THRESHOLD and remaining_bits >= 6:
            assignments[name] = 6  # 中敏感层用 INT6
            remaining_bits -= 6
        else:
            assignments[name] = 4  # 低敏感层用 INT4
    
    return assignments

八、W4A8：权重量化与激活量化的融合

七、层间精度分配策略

最热门的混合精度方案，兼顾了4-bit的存储效率和8-bit的计算精度。

W4A8 核心思路：

权重存储：INT4 (每参数 0.5 字节，与纯INT4相同)
激活计算：INT8 (计算精度比INT4高一个数量级)
解量化：在计算时将INT4权重临时解量化为INT8

优势：存储 = INT4 速度，精度 ≈ INT8 级别
深入探讨：W4A8的精妙之处在于"分离存储精度和计算精度"。权重用INT4存储（节省显存和带宽），但在做矩阵乘法时先解量化为INT8再计算（保持精度）。这之所以可行，是因为GPU的INT8 Tensor Core吞吐量很高，解量化操作几乎"免费"。Marlin和Machete kernel的核心优化就是高效实现这个"存储-计算分离"。

      实际案例: 对 Llama 3 8B 使用混合精度：Embedding→FP16, Q/K→INT8, V/O→INT4, FFN→INT4, LM Head→INT8。对比统一 INT4，MMLU 提升 1.5%，显存仅增加 0.3GB。
      关联知识: 敏感度分析是第07章 Benchmark 方法论的延伸。
    2. 核心原理: SmoothQuant 解决激活值量化的核心难题——outlier 通道。公式为 Y=X·W=(X·diag(s)⁻¹)·(diag(s)·W)，其中 s_j=max(|X_j|)^α / max(|W_j|)^(1-α)。通过此变换，激活的 outlier 被"转移"到权重上：α=0.5 时各承担一半难度。核心洞察来自 Tim Dettmers 的发现：LLM 中约 0.1% 的通道是"outlier 通道"，在不同输入中始终保持异常大的值。
      关键要点: ①激活值存在 outlier 通道（大 100x+）；②SmoothQuant 将 outlier 转移到权重；③α 参数控制难度分配（默认 0.5）；④outlier 通道固定且可预测；⑤使激活和权重都可安全量化为 INT8。
      实际案例: 对 Llama 3 8B 应用 SmoothQuant（α=0.5），激活值从 [1,2,150,3,1,200,2,1] 平滑为 [1,2,12,3,1,14,2,1]，INT8 量化后 PPL 仅增加 0.05。
      关联知识: SmoothQuant 与 LLM.int8() 是互补方案。
    3. 核心原理: LLM.int8() 是 bitsandbytes 提出的混合精度方案，核心策略是"99.9% 的值用 INT8，0.1% 的 outlier 用 FP16"。流程：检测 outlier 通道（绝对值 > 6.0）→ 正常通道用 INT8 矩阵乘法 → outlier 通道用 FP16 矩阵乘法 → 合并结果。效果：精度接近 FP16，速度 1.5x，显存约 9GB，易用性极高，适合 7B+ 模型在消费卡上推理。
      关键要点: ①99.9% 值用 INT8，0.1% outlier 用 FP16；②outlier 阈值通常为 6.0；③精度接近 FP16；④速度 1.5x，显存减半；⑤易用性极高。
      实际案例: 在 RTX 3090（24GB）上部署 Llama 3 8B，FP16 需要 16GB（放不下其他内容），使用 LLM.int8() 后显存降至 9GB，正常运行且 MMLU 差异 <0.3%。
      关联知识: LLM.int8() 的 outlier 检测机制与 SmoothQuant 的 outlier 通道发现是同一现象的两种利用方式。
    4. 核心原理: KV-Cache 量化解决长上下文推理的显存瓶颈。显存公式：2×num_layers×num_heads×head_dim×seq_len×batch×dtype_bytes。以 Llama 3 70B、seq_len=128K 为例，BF16 KV-Cache 约 34GB（比模型参数还大）。FP8 KV-Cache 可将显存减半（17GB），INT4 KV-Cache 降为四分之一（8.5GB）。关键洞察：KV-Cache 量化和权重量化是独立的，可以叠加使用。FP8 KV-Cache 几乎无损（PPL 增加 <0.1）。
      关键要点: ①KV-Cache 在长上下文下是显存大户；②seq_len=32K 时追平模型大小；③FP8 KV-Cache 显存减半且几乎无损；④KV-Cache 量化与权重量化可叠加；⑤INT4 KV-Cache 显存降为四分之一。
      实际案例: 在 vLLM 中部署 Llama 3 70B 支持 128K 上下文，设置 kv_cache_dtype="fp8"，KV-Cache 从 64GB 降至 32GB。
      关联知识: KV-Cache 量化是第05章 FP8 推理实现的具体应用。
    5. 核心原理: 双重量化是 QLoRA 提出的技巧，核心是"量化参数本身也量化"。正常量化中每 64 个权重共享一个 FP32 scale，开销 0.0625 bytes/weight。双重量化将 scale 量化为 FP8，开销降至约 0.01 bytes/weight。以 7B 模型为例：普通 INT4 的 scale 开销 0.44GB，双重量化降至 0.07GB，节省 0.37GB。关键洞察：scale 值变化范围很小，用 FP8 存储几乎不损失精度。
      关键要点: ①量化参数本身有存储开销；②双重量化将 scale 量化为 FP8；③7B 模型节省约 0.37GB；④scale 值范围小，量化几乎无损；⑤8 卡并行时节省 2.96GB。
      实际案例: 使用 QLoRA 微调 Llama 3 8B，双重量化将模型显存从 5.5GB 降至 5.1GB，在 RTX 3090 上可支持更大 batch_size（从 4 提升至 6）。
      关联知识: 双重量化是 QLoRA 的核心技术，与第02章 GPTQ 的 group_size 参数有概念关联。
