大语言模型（LLM）的参数规模持续增长，部署成本成为核心瓶颈。以 LLaMA-70B 为例，使用 FP16 存储需要约 140GB 显存，这远超单张消费级 GPU 的容量。通过 4-bit 量化，同样模型仅需 35GB，使得在单卡 A100 80GB 上运行成为可能。

核心动机：量化是降低模型内存占用、提升推理速度的最实用手段，无需重新训练即可获得 2-4 倍的压缩比。

量化带来的三大收益

  • 内存节省：从 FP16 的 2 bytes/param 降低到 INT4 的 0.5 bytes/param，压缩 4 倍
  • 速度提升：整数运算吞吐量更高，内存带宽需求更低
  • 成本降低：同等硬件可部署更大模型或服务更多并发请求

BF16 vs FP16

BF16（Brain Float 16）由 Google 提出，保留了 FP32 的指数位（8位），牺牲尾数精度。这使得 BF16 的数值范围与 FP32 相同，但精度较低，更适合训练场景，因为它不容易出现溢出/下溢问题。

FP16 保留了更多尾数位（10位），精度更高但范围较小，更适合推理场景。目前大多数推理框架默认使用 FP16。

整数格式

INT8 将数值映射到 [-128, 127] 或 [0, 255] 的整数范围，INT4 则映射到 [-8, 7] 或 [0, 15]。整数运算在硬件上更高效，吞吐量通常为浮点的 2-4 倍。

为什么不用 INT8 推理？实际上 INT8 推理已经非常成熟，许多推理框架默认使用 INT8 进行矩阵乘法。但对于 LLM 来说，4-bit 量化能提供更大的压缩比，且质量损失在可接受范围内。

均匀量化（Uniform Quantization）

最常用的量化方式，将连续浮点值均匀映射到离散整数：

q = round(x / s) + z
x_hat = (q - z) × s

其中：s = (x_max - x_min) / (2^b - 1) 为缩放因子
z = round(-x_min / s) 为零点偏移
b 为量化位数

对称量化 vs 非对称量化

对称量化假设数值以零为中心对称分布，不需要零点偏移（z=0），计算更简单。权重量化通常采用对称方式。

非对称量化可以更好地处理非零中心的分布，适合激活值的量化，因为激活值往往呈现非对称分布。

非均匀量化（Non-Uniform Quantization）

非均匀量化允许量化点之间的间距不等，可以更有效地覆盖数据分布。典型方法包括：

  • 对数量化：对数间隔的量化点，适合长尾分布
  • 向量量化：将多个值组合为一个码本索引
  • NormalFloat（NF4）：QLoRA 提出的针对正态分布优化的量化格式

按时间点分类

  • 训练后量化（PTQ）：在训练完成后对权重进行量化，不需要重新训练。代表：GPTQ、AWQ、GGUF
  • 量化感知训练（QAT）：在训练过程中模拟量化效果，让模型学习适应低精度表示。代表：LLM-QAT、BitNet

Round-to-Nearest（RTN）

最简单的量化方法，直接将每个权重四舍五入到最近的量化点：

W_q = round(W / s) × s
误差 = |W - W_q| ≤ s/2

RTN 的优点是实现简单、速度快，但在低比特（4-bit 及以下）时质量下降明显，因为它没有考虑权重之间的相关性。

校准数据的作用

校准数据的作用

据（128-512 条样本）来确定最优的量化参数。校准数据用于：

  • 确定激活值的分布范围，计算合适的缩放因子
  • 在 GPTQ 中用于计算 Hessian 矩阵，指导误差补偿方向
  • 在 AWQ 中用于识别对模型输出影响最大的权重通道

实践建议：校准数据应尽量覆盖目标应用场景的分布。对于通用模型，使用 C4 或 WikiText 数据集的随机子集即可。校准数据量通常 128 条足够，增加到 512 条带来的改善有限。

常见误区：很多人以为量化只是"把FP16的权重转成INT4"这么简单。实际上，校准数据的选择对量化质量影响巨大——用英文维基百科校准的模型在中文任务上可能损失10%以上的精度。务必用目标任务分布的数据做校准。

量化本质上是有损压缩，不同比特宽度对模型质量的影响差异显著。以下以 LLaMA-2-7B 在 WikiText-2 上的困惑度（PPL）为例：

  • 8-bit：几乎无损，PPL 增加通常小于 0.1
  • 4-bit：轻微下降，PPL 增加 0.2-0.5，对大多数任务影响不大
  • 3-bit：明显下降，PPL 增加 1-3，开始影响复杂推理和长文本生成
  • 2-bit：严重下降，PPL 增加 5+，仅适用于特定场景

注意：量化影响因模型架构而异。通常，MoE 模型（如 Mixtral）对量化更敏感，因为每个专家的参数量较小，量化误差的影响更为集中。同时，小模型（如 1B-3B）比大模型（如 70B+）对量化更敏感。

量化并非万能方案，存在以下局限：

  • 有损压缩：低于 4-bit 时质量退化明显
  • 任务敏感：数学推理、编程等复杂任务对量化更敏感
  • 硬件依赖：实际加速效果取决于硬件对低精度运算的支持
  • 不改变架构：量化不能解决模型本身的设计问题

常见误区：对称量化（z=0）虽然计算更简单，但并不总是优于非对称量化。当权重或激活值的分布明显偏离零中心时（如ReLU后的激活值全为正数），非对称量化能更有效地利用量化范围，精度损失可减少30-50%。不要图省事一律用对称量化。
为什么需要量化？的技术原理与数学推导： 从底层硬件角度分析，为什么需要量化？的性能瓶颈通常在于内存带宽而非计算能力。 推导：以A100为例，FP16峰值算力312 TFLOPS，HBM带宽2TB/s， 算术强度转折点 = 312T/2T = 156 FLOPs/Byte。为什么这很重要？ 因为当实际算术强度低于此值时，任务是内存受限的——增加更多计算单元不会提升性能。 这一数学本质决定了为什么需要量化？的所有优化策略都应围绕减少内存访问展开。 从工作机制看，算子融合（减少中间结果写回）和批处理（共享参数读取） 是提升算术强度的两大核心手段。
为什么需要量化？的系统架构分析： 从系统设计的原理出发，为什么需要量化？采用分层架构来管理复杂性。 为什么分层是主流选择？推导如下： 设系统有N个组件，全连接交互复杂度为O(N²)，分层后降低到O(N·L)，L为层数。 数学本质上，这是用少量层间通信开销换取大幅降低的组件间耦合。 在实际工作机制中，每一层只与相邻层通信，通过标准接口解耦—— 这类似于TCP/IP协议栈的设计原理，每层独立优化，层内改动不影响整体。 
# 为什么需要量化？ - 性能基准测试
import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    """测量推理延迟和吞吐"""
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
    elapsed = (time.perf_counter() - start) / iters
    print(f"Latency: {elapsed*1000:.2f}ms")
    print(f"Throughput: {1/elapsed:.0f} req/s")
    return elapsed

  1. 为什么需要量化？
    核心原理: 大语言模型参数规模持续增长（如LLaMA-70B需要140GB FP16显存），远超单卡GPU容量。量化通过将权重从FP16(2字节/参数)压缩到INT4(0.5字节/参数)，实现4倍压缩比。从硬件角度看，推理瓶颈通常在于内存带宽而非计算能力——以A100为例，FP16峰值算力312 TFLOPS，HBM带宽2TB/s，算术强度转折点=312T/2T=156 FLOPs/Byte，大多数LLM工作负载低于此值，属于内存受限任务。量化减少参数字节数，直接降低内存访问需求，从而提升推理速度。
    关键要点: ①量化提供2-4倍压缩比，无需重新训练 ②整数运算吞吐量是浮点的2-4倍 ③从FP16到INT4可让70B模型从多卡部署变为单卡A100部署 ④推理成本是持续性运营成本，比训练一次性成本高得多
    实际案例: LLaMA-70B在FP16下需140GB显存，使用4-bit量化后仅需35GB，单张A100 80GB即可运行，使企业从需要多卡A100集群降低到单卡部署，硬件成本降低75%以上。
    关联知识: 量化收益与屋顶模型(Roofline Model)密切相关——只有理解内存受限任务的本质，才能真正理解为何减少参数位宽能带来速度提升。同时量化是PTQ方法的入口，后续GPTQ/AWQ/GGUF都是此动机的具体技术实现。
  2. 浮点数格式详解
    核心原理: 不同浮点数格式通过分配符号位、指数位、尾数位来权衡精度与范围。FP32(1-8-23)提供±3.4×10³⁸范围和高精度；FP16(1-5-10)牺牲范围和精度换取半存储；BF16(1-8-7)保留FP32的指数位(8位)但大幅牺牲尾数(7位)，数值范围与FP32相同适合训练；FP8(E4M3/E5M2)是8位格式，分别针对精度和范围优化。整数格式INT8映射到[-128,127]，INT4映射到[-8,7]，硬件上整数GEMM吞吐量是浮点的2-4倍。
    关键要点: ①BF16范围=FP32范围但精度低，适合训练的梯度动态范围 ②FP16精度更高但范围小，适合推理 ③FP8是新一代硬件原生格式(H100原生支持) ④整数格式的推理效率远高于浮点格式
    实际案例: 训练LLaMA-2使用BF16混合精度（利用其大范围避免溢出），推理时使用FP16（利用其更高精度），部署时量化到INT4（利用整数运算的高效性）。三种格式在同一个模型生命周期的不同阶段各司其职。
    关联知识: 格式选择本质是精度-效率权衡。理解各格式的位布局是理解量化原理的基础——量化就是将FP16/BF16的连续值映射到INT4/INT8的离散整数空间，映射过程中的精度损失取决于格式间的数值范围匹配程度。
  3. 量化数学原理
    核心原理: 均匀量化将连续浮点值映射到离散整数：q=round(x/s)+z，其中s=(x_max-x_min)/(2^b-1)是缩放因子，z=round(-x_min/s)是零點偏移。对称量化(z=0)假设数值零中心分布，计算简单适合权重；非对称量化(z≠0)处理非零中心分布，适合激活值。非均匀量化如对数量化(对数间隔)、向量量化(码本)和NormalFloat(NF4)能更好地覆盖数据分布。Round-to-Nearest(RTN)直接四舍五入，误差≤s/2，但低比特时质量下降明显。
    关键要点: ①缩放因子s决定量化精度，s越小精度越高 ②对称量化计算更快但精度可能不如非对称 ③非均匀量化可适配长尾分布 ④量化误差上界为s/2，group_size越小s越小误差越低
    实际案例: 对于ReLU后的激活值(全为正数，分布严重非零中心)，非对称量化能利用全部量化范围，精度比对称量化高30-50%。QLoRA使用NF4(4-bit NormalFloat)专门针对预训练权重的正态分布优化。
    关联知识: 量化数学是GPTQ和AWQ的理论基础。GPTQ的Hessian误差补偿本质是最优量化方向搜索；AWQ的通道缩放本质是通过缩放s来减小重要通道的量化误差。理解均匀量化的s和z参数是理解所有PTQ方法的前提。
  4. 量化方法分类
    核心原理: 量化方法按时间点分为PTQ(训练后量化)和QAT(量化感知训练)。PTQ在训练完成后量化权重，不需重新训练，代表方法有GPTQ(Hessian误差补偿)、AWQ(激活感知通道缩放)、GGUF(混合精度分组量化)。QAT在训练中模拟量化效果让模型适应低精度表示，代表有LLM-QAT、BitNet。按精度方式可分为RTN(直接舍入，最快但精度低)、GPTQ(需校准数据+Hessian补偿)、AWQ(需校准数据+激活统计)。不同方法在速度-质量-资源三角中各有侧重。
    关键要点: ①PTQ训练成本低但精度上限受限于原始模型 ②QAT精度上限更高但需要重新训练 ③RTN适合快速实验和极致速度场景 ④GPTQ需要GPU做校准阶段计算 ⑤AWQ比GPTQ量化更快但同样优秀
    实际案例: 研究者需要在单卡A100上部署LLaMA-13B做文本分类。如果追求最快速度且精度要求不高，选择RTN 4-bit；如果追求最佳精度且时间充裕，选择GPTQ 4-bit(group_size=64)；如果需要快速部署且精度良好，选择AWQ 4-bit(GEMM kernel)。三种选择对应三种不同的实际约束。
    关联知识: 量化方法分类是选型指南(第10章)的理论基础。每种方法背后有不同的数学假设和优化目标，理解这些才能在实际项目中做出合理选择。RTN是基线方法，理解RTN的局限性(低比特质量崩溃)就能理解为什么需要GPTQ/AWQ的误差补偿。
  5. 校准数据的作用
    核心原理: 校准数据(128-512条样本)用于确定最优量化参数。在GPTQ中，校准数据用于计算Hessian矩阵H=2X^TX，该矩阵指导误差补偿方向；在AWQ中，校准数据用于识别salient channels(激活值top 1%的通道)，这些通道在量化时需要额外保护；在校准数据中用于计算激活值分布范围确定缩放因子s和零点z。校准数据的质量(分布匹配度)比数量更重要——128条场景匹配数据优于1000条通用数据。
    关键要点: ①128条通常为最低有效量，512条基本饱和 ②数据分布必须匹配目标场景(对话用ShareGPT，代码用Code-Alpaca) ③序列长度通常2048 tokens覆盖典型推理 ④错误分布校准可导致10%以上精度损失
    实际案例: 部署一个中文客服LLM。如果用英文维基百科校准(分布严重不匹配中文)，量化后模型在中文理解任务上PPL可能增加15%以上；改用中文客服对话数据(128条)校准后，PPL增加控制在3%以内。
    关联知识: 校准数据的设计直接影响量化质量。这是连接理论(量化数学)与实践(部署质量)的桥梁。校准数据的分布决定了Hessian矩阵的表征范围、salient channel的识别准确性，最终决定了量化模型在实际任务上的保真度。这也是PTQ方法的核心假设：少量代表性数据能代表模型在推理时的真实激活分布。
向前串联： 为什么需要量化？的知识将在后续的推理框架选型、性能基准测试、生产环境调优中反复应用。 建议在本章学习后，立即用真实模型（如LLaMA-7B）进行动手实验， 将概念理解转化为可测量的性能数据（延迟、吞吐、显存占用）， 这些数据将成为后续优化决策的量化依据。
学习路径建议： 学习为什么需要量化？建议按"概念→原理→实验→调优"四步走。先理解核心概念和术语， 再深入数学原理和工作机制，然后用最小化实验验证理解，最后在真实场景中调优参数。 每步都应记录关键指标（延迟/吞吐/精度），建立数据驱动的优化习惯。
