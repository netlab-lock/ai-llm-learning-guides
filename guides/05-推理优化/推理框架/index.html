💡 为什么要学 LLM 推理框架？

通俗类比：推理框架就像餐厅的「点餐系统+出餐流水线」——训练好的厨师要高效接待成千上万客人，必须优化点单、配菜、传菜等每个环节，否则再厉害的厨师也会手忙脚乱。
常见误区：很多人以为有了GPU就能跑推理，忽略系统优化，导致单卡只能服务1-2个用户，成本比API服务贵10倍以上。
深入探讨：以Llama-2-70B为例，纯模型推理需要140GB显存（FP16），而vLLM通过PagedAttention技术，单卡40GB即可服务，QPS提升5-10倍。推理优化直接决定百万级部署成本。
训练成本：$2M (一次)
推理成本：$500/天 (持续)

优化前：1 GPU → 2 QPS
优化后：1 GPU → 20 QPS

核心观点：训练一个 LLM 可能需要数百万美元，但推理（inference）才是 LLM 真正产生价值的环节。每一次 ChatGPT 的回答、每一次 Copilot 的代码补全，背后都是一次推理过程。推理框架的效率直接决定了服务成本、响应速度和用户体验。

💰 成本敏感

推理占 LLM 总运营成本的 80-90%。优化推理 = 省钱。一个高效的推理框架可以将吞吐量提升 5-20 倍。

⚡ 延迟决定体验

用户期望毫秒级响应。理解推理优化（KV Cache、投机解码等）才能理解为什么有的服务快、有的慢。

🔧 工程必备

无论做 AI 产品、模型部署还是研究落地，都绕不开推理框架。vLLM、TensorRT-LLM 已成为行业标准。

常见误区：只关注模型计算层，忽略数据预处理、序列调度、负载均衡等环节，就像只优化炒菜速度却让客人等半小时配菜。

通俗类比：框架选择像选交通工具——vLLM是「地铁」（高并发稳定）、TGI是「公交车」（HuggingFace生态）、TensorRT-LLM是「高铁」（极致速度但路线固定）。

📊 核心性能指标

通俗类比：性能指标像餐厅的「体检报告」——吞吐量是「每小时接待客人数」，延迟是「从点菜到上菜时间」，显存是「厨房面积」，成本是「每道菜原料费」。
常见误区：只看吞吐量忽略延迟，就像餐厅每小时接待100人但每桌等2小时，用户体验极差。
深入探讨：关键指标：1) 吞吐量：tok/s或req/s；2) 延迟：TTFT（首token延迟）、TPS（每token延迟）；3) 显存占用：峰值显存；4) 成本：$/M tokens。例如：vLLM在A100上Llama-2-70B可达2000 tok/s（输入+输出）。
TTFT：首token延迟
TPS：生成速度
吞吐量：总token/时间
显存：峰值显存占用

# 安装 vLLM
pip install vllm

# 启动推理服务（自动下载模型）
python -m vllm.entrypoints.openai.api_server \
    --model meta-llama/Llama-3.1-8B-Instruct \
    --port 8000

# 用 curl 调用
curl http://localhost:8000/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{"model":"meta-llama/Llama-3.1-8B-Instruct","messages":[{"role":"user","content":"Hello!"}]}'
