🧱 Transformer 架构基础

本节目标：快速回顾 Transformer 架构的核心组件，为理解推理优化打下基础。假设你已了解基本概念，这里聚焦于推理相关的细节。

1. Transformer 的核心组件

Transformer 的核心组件

ng>通俗类比：Transformer像一座多层办公楼：自注意力是每层的‘圆桌会议室’，让信息自由交流；前馈网络是独立的‘分析师工位’，对信息做深度加工。每一层都这样‘开会-分析’。
常见误区：初学者常混淆‘层’与‘组件’。一层Transformer包含自注意力和前馈网络两个子层，前者负责上下文关联，后者负责非线性变换，二者缺一不可。
深入探讨：以GPT-3为例，它拥有96层Transformer。每一层中的自注意力子层包含96个注意力头。模型总参数量达1750亿，其中大部分存储于各层FFN的权重矩阵中。

2. Self-Attention 机制详解

通俗类比：自注意力就像一个高效的会议室：每个词（成员）都会生成‘询问’(Query)、‘提供资料’(Key)和‘实际观点’(Value)。通过Q与所有K的匹配度，来决定最终吸收多少来自各个词的V。
常见误区：容易忽略对注意力分数的缩放（除以根号d_k）。这防止了因维度d_k过大导致的点积结果过大，使softmax梯度饱和，从而导致训练困难。
深入探讨：计算公式为：Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V。对于序列长度为L，head维度为d_k的输入，计算复杂度为O(L^2 * d_k)。这就是长序列计算成本高的根源。

Attention 是 Transformer 的核心，也是推理优化的重点：

2.1 标准 Self-Attention 计算

Attention(Q, K, V) = softmax(QK^T / √d_k) × V

计算步骤分解：

  1. Q、K、V 投影：将输入分别乘以三个权重矩阵 W_q, W_k, W_v
  2. 注意力分数：Q × K^T，得到 [seq_len, seq_len] 的矩阵
  3. 缩放：除以 √d_k，防止梯度消失
  4. Softmax：归一化为概率分布
  5. 加权求和：用注意力权重对 V 加权

3. FFN（前馈网络）

优化决策。

FFN（前馈网络）

通俗类比：FFN像是一个生产线上的‘质检与精加工’环节：自注意力层混合了上下文信息后，FFN对每个位置（词）独立地进行两次线性变换和一次非线性激活，提取更高级的特征模式。
常见误区：误以为FFN对序列全局建模。FFN是逐位置（position-wise）独立的，即序列中每个token的FFN计算互不影响，这与自注意力层的全局交互形成鲜明对比。
深入探讨：标准FFN结构为：FFN(x) = max(0, xW1 + b1)W2 + b2。其中W1将模型维度d_model（如768）升维到d_ff（如3072，通常是4倍），W2再降回d_model。这带来了主要的参数量。
[输入x] -> [线性层W1, b1] -> [ReLU/GELU] -> [线性层W2, b2] -> [输出]
(d_model)      (d_ff)                     (d_ff)       (d_model)
激活函数公式特点
ReLUmax(0, x)简单，可能产生‘死神经元’
GELUx * Φ(x)平滑，性能更佳，主流选择
SILU/Swishx * sigmoid(x)非单调，类似GELU

每个 Transformer Block 的 FFN 通常包含模型 2/3 的参数：

FFN 结构 (以 Llama 为例): 输入 x: [batch, seq_len, d_model] │ ├──→ W_gate (d_model → d_ffn) ──→ SiLU 激活 │ │ └──→ W_up (d_model → d_ffn) ─────────┤ │ 逐元素相乘 │ ▼ W_down (d_ffn → d_model) │ ▼ 输出: [batch, seq_len, d_model] 其中 d_ffn = 2/3 × 4 × d_model (SwiGLU 变体) Llama-7B: d_model=4096, d_ffn=11008

3.1 推理中的 FFN 特点

  • 参数量大：FFN 占模型参数的 ~2/3，是量化的主要目标
  • 计算密集：Prefill 阶段 FFN 是主要计算量
  • MoE 变体：Mixtral 等模型用多个专家 FFN + 路由，只激活部分专家

4. 模型规模与计算量

模型规模与计算量

通俗类比：模型规模好比汽车的排量和重量：参数量（Params）像发动机排量，决定了模型的‘理论能力上限’；计算量（FLOPs）像油耗和行驶阻力，决定了实际处理数据时需要消耗多少计算资源（时间和电费）。
常见误区：不要只关注参数量，而忽略推理时的计算量（FLOPs）。一个参数少但序列长的模型，其计算成本可能远超一个参数多但序列短的模型。二者需权衡。
深入探讨：GPT-3 (175B) 的一次前向传播，对于长度为1024的序列，约需3.14 * 10^14 FLOPs。训练它消耗了约3.14 * 10^23 FLOPs，相当于在上千个GPU上运行数月。
计算量 FLOPs ≈ 2 * 参数量 * 序列长度

5. 不同架构变体

通俗类比：Transformer架构变体如同不同菜系：Encoder-Only（如BERT）是‘凉菜师傅’，擅长理解；Decoder-Only（如GPT）是‘主菜大师’，擅长生成；Encoder-Decoder（如T5）是‘全能套餐’，擅长序列到序列任务。
常见误区：没有‘万能’架构。选择取决于任务：理解用BERT类，生成用GPT类，翻译/摘要用T5类。盲目追求最新架构可能不如选择合适架构有效。
深入探讨：Decoder-Only（GPT）采用因果掩码（Causal Mask），确保每个位置只能关注之前的位置。Encoder-Only（BERT）采用双向掩码。这从根本上决定了它们的训练和推理方式。
Encoder-Only: [输入] => [双向Transformer堆叠] => [输出]
Decoder-Only: [输入] => [带掩码的单向Transformer堆叠] => [输出]
Enc-Dec:      [输入] => [Encoder] => [Decoder] => [输出]
架构类型代表模型掩码方式擅长任务
Encoder-OnlyBERT, RoBERTa双向（无掩码）分类、NER、理解
Decoder-OnlyGPT, LLaMA, PaLM因果（向左看）文本生成、续写
Encoder-DecoderT5, BARTEnc双向，Dec因果翻译、摘要、问答

5.1 仅解码器（Decoder-Only）——当前主流

  • GPT 系列、Llama、Qwen、Mistral 等
  • 特点：单向注意力（causal mask），自回归生成
  • 推理优化最成熟，本指南主要覆盖此类

5.2 编码器-解码器（Encoder-Decoder）

  • T5、BART 等
  • 特点：双向编码 + 单向解码
  • 推理时需要分别处理编码和解码阶段

5.3 混合专家（MoE）

  • Mixtral、DeepSeek-V2/V3、Qwen MoE
  • 特点：多个专家 FFN，每次只激活 top-k 个
  • 推理挑战：专家路由、负载均衡、All-to-All 通信

6. 代码示例：一次前向传播

代码示例：一次前向传播

前向传播就像按食谱做菜：输入张量是‘食材’，经过一层层‘处理’（线性变换、激活函数、注意力），最终输出结果张量。每一层的处理都遵循固定的‘配方’（模型结构）。

常见误区：初学者最常犯的错误是张量形状不匹配。务必时刻关注输入输出的维度（如 batch_size, seq_len, d_model），这是调试深度学习代码的关键。
深入探讨：在PyTorch中，一个标准的TransformerLayer调用大致为：`output = self.attn(x, x, x)[0]; output = self.ffn(output)`。注意注意力层的输入Q、K、V通常都是同一个x（自注意力）。
import torch

# 假设 x 形状为 (batch=4, seq_len=10, d_model=512)
x = torch.randn(4, 10, 512)
# 通过一层Transformer
layer = TransformerLayer(d_model=512, nhead=8)
out = layer(x) # 输出形状仍为 (4, 10, 512)

FFNLinear -> Act -> Linear(B, L, d) -> (B, L, d_ff) -> (B, L, d)

import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

# 加载模型
model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3.1-8B",
    torch_dtype=torch.float16, device_map="auto")
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-3.1-8B")

# 编码输入
inputs = tokenizer("The capital of France is", return_tensors="pt").to("cuda")

# 一次前向传播 = Prefill 阶段
with torch.no_grad():
    outputs = model(**inputs)
    # outputs.logits shape: [1, seq_len, vocab_size]
    next_token = outputs.logits[:, -1, :].argmax(dim=-1)
    # → token_id for " Paris"

# Decode 阶段（逐 token）
for i in range(20):
    with torch.no_grad():
        outputs = model(next_token.unsqueeze(0))
        next_token = outputs.logits[:, -1, :].argmax(dim=-1)
    print(tokenizer.decode(next_token), end="")
⚠️ 注意：上面的代码是为了演示原理，实际生产中请使用 vLLM、TGI 等框架，它们已经实现了 KV Cache 管理、连续批处理等优化。

✅ 参考答案与详解
  1. 核心原理: Transformer的核心组件包括自注意力（Self-Attention）、前馈网络（FFN）、残差连接（Residual Connection）和层归一化（Layer Norm）。自注意力层像"圆桌会议室"——序列中每个token都会生成Query、Key、Value三个向量，通过Q与所有K的点积计算注意力权重，再对V加权求和，实现上下文信息的全局交互。FFN则像"独立工位"——对每个token位置独立进行两次线性变换和一次非线性激活（如GELU），提取高级特征模式。残差连接通过跳跃连接缓解梯度消失，Layer Norm稳定训练。一个Transformer Block = LayerNorm + Self-Attention + Residual + LayerNorm + FFN + Residual，此结构堆叠N层（GPT-3为96层）构成完整模型。
    关键要点: ①自注意力负责全局上下文交互，FFN负责逐位置非线性变换，二者缺一不可；②残差连接和Layer Norm是训练稳定性的关键保障；③GPT-3有96层Transformer，96个注意力头，1750亿参数；④推理时仅需前向传播，无需反向传播，计算量约为2×参数量×序列长度 FLOPs；⑤FFN占模型约2/3参数量，是量化优化的主要目标。
    实际案例: 以Llama-3-8B为例，32层Transformer Block，每层包含GQA自注意力（32个头，KV头8个）+ SwiGLU FFN（d_model=4096, d_ffn=14336）。处理输入"The capital of France is"时：先经Tokenizer转为token序列，经Embedding和RoPE位置编码后，依次通过32层Block——每层先做自注意力（所有token交互建模上下文依赖），再做FFN（逐位置提取高级特征），最后经LM Head输出logits，argmax得到下一个token "Paris"。
    关联知识: 核心组件的选择直接影响推理性能——自注意力的O(n²)复杂度驱动了FlashAttention和稀疏注意力优化（模块2）；FFN的参数量大驱动了INT4量化和MoE架构（模块3-5）；GQA/MQA通过减少KV头数降低Decode阶段内存带宽需求。理解组件是理解所有推理优化的前提。
  2. 核心原理: Self-Attention是Transformer的核心机制，其计算公式为Attention(Q,K,V) = softmax(QK^T/√d_k)×V。具体步骤：1）将输入向量分别乘以W_Q、W_K、W_V投影矩阵得到Q、K、V；2）Q×K^T计算注意力分数矩阵（[seq_len, seq_len]）；3）除以√d_k缩放，防止因维度d_k过大导致点积值过大，使softmax梯度饱和；4）softmax归一化为概率分布；5）用注意力权重对V加权求和得到输出。多头注意力（MHA）将d_model拆分为h个头并行计算，每个头独立学习不同的注意力模式，最后拼接输出。计算复杂度为O(L²×d_k)，这是长序列计算成本高的根源。
    关键要点: ①QKV分别由输入乘以W_Q/W_K/W_V得到，Q和K的点积衡量token间的相关性；②√d_k缩放是防止梯度消失的关键细节，初学者常忽略；③计算复杂度O(L²×d_k)，序列长度翻倍计算量增4倍；④多头注意力让模型同时关注不同类型的依赖关系（语法、语义等）；⑤推理时KV Cache避免重复计算历史token的K和V。
    实际案例: 推理优化中，Attention是最关键的优化对象。KV Cache机制：Decode阶段生成第n个token时，只需计算新token的Q_n、K_n、V_n，然后用Q_n与缓存的K_1..K_n做注意力计算，将K_n、V_n追加到缓存。GQA（Grouped Query Attention）将多个Query头共享一组KV头，如Llama-2-70B用64个Q头但只有8个KV头，KV Cache缩小8倍。FlashAttention通过分块计算避免显式构造O(n²)注意力矩阵，将内存从O(n²)降至O(n)。
    关联知识: Self-Attention是推理中内存和计算的双重瓶颈——Prefill阶段计算密集（O(n²)），Decode阶段内存密集（反复读KV Cache）。FlashAttention优化算子层（模块2），GQA/MQA优化架构层减少KV头数，PagedAttention优化系统层管理KV内存，滑动窗口注意力限制上下文长度。理解Attention机制是理解所有这些优化的前提。
  3. 核心原理: 前馈网络（FFN）是Transformer中每个Block的第二个子层，对每个token位置独立进行非线性特征变换。标准FFN结构为：FFN(x) = max(0, xW₁+b₁)W₂+b₂，其中W₁将d_model升维到d_ff（通常4倍，如768→3072），W₂再降回d_model。现代LLM普遍采用SwiGLU变体（如Llama系列）：FFN(x) = (x·W_gate经SiLU激活) ⊙ (x·W_up) · W_down，其中d_ffn=2/3×4×d_model。FFN的关键特点是逐位置独立——序列中每个token的FFN计算互不影响，这与自注意力的全局交互形成鲜明对比。FFN占模型参数量的约2/3，是模型存储和量化优化的主要目标。
    关键要点: ①FFN逐位置独立处理，不建模序列全局关系（那是Attention的工作）；②标准FFN升维-激活-降维，SwiGLU变体用门控机制效果更好；③FFN占模型~2/3参数量，Llama-7B中d_ffn=11008；④推理中FFN在Prefill阶段是主要计算量，在Decode阶段是主要内存访问量；⑤MoE架构的核心就是用多个专家FFN+路由替代单一FFN，每次只激活top-k个专家。
    实际案例: 以Llama-7B为例，每层FFN包含W_gate（4096×11008）、W_up（4096×11008）、W_down（11008×4096）三个矩阵，共约3×4096×11008≈1.35亿参数，32层FFN总计约43亿参数（占总6.7B参数的~64%）。推理时，INT4量化将每个参数从2字节压缩到0.5字节，FFN部分从8.6GB降至2.15GB，这是单卡运行7B模型的关键。MoE模型如Mixtral-8x7B有8个专家FFN，路由网络每次只激活2个，总参数45B但活跃参数仅12B，实现了大模型能力+小模型推理成本。
    关联知识: FFN是量化和MoE优化的核心目标——INT4量化直接压缩FFN权重矩阵（模块3）；MoE将FFN替换为多个专家+路由器，实现参数量与计算量的解耦（模块5）；FFN的计算模式（GEMM）是算子融合和CUDA优化的重点。理解FFN的参数分布有助于制定合理的量化策略——通常FFN层可以比Attention层使用更激进的量化。
  4. 核心原理: 模型规模由参数量（Params）决定能力上限和存储需求，计算量（FLOPs）决定推理时的资源消耗。核心公式：FP16模型内存≈参数量×2字节（7B≈14GB，70B≈140GB）；前向传播FLOPs≈2×参数量×序列长度。模型参数主要分布在：Embedding层（vocab_size×d_model）、自注意力层（3×d_model²×n_heads/head_dim相关）、FFN层（~2/3参数量）、LM Head（与Embedding共享权重或独立）。随着参数量增大，不仅存储需求线性增长，推理时的计算量和内存带宽需求也线性增长。序列长度增长的影响更严重——自注意力计算量是O(n²)。
    关键要点: ①FP16内存≈参数量×2字节，这是选择GPU的第一参考；②FLOPs≈2×参数量×序列长度，GPT-3(175B)处理1024 token约需3.14×10¹⁴ FLOPs；③FFN占~2/3参数，Attention参数相对较少但计算量因O(n²)可能更大；④更长序列意味着更大的KV Cache——2×层数×头数×头维度×序列长度×精度；⑤DeepSeek-V3的MoE架构总参671B但活跃参数仅37B，展示了稀疏激活的优势。
    实际案例: 部署Llama-3-70B：FP16需140GB显存（至少2张A100-80G）；INT8量化需70GB（1张H100-80G勉强）；INT4量化需35GB（单张A100-80G可运行）。处理2048 token的推理：FLOPs≈2×70B×2048≈2.86×10¹⁴，在H100（990 TFLOPS FP16）上理论耗时0.29ms，但实际受内存带宽限制（3.35TB/s需读取70GB权重），实际约20-50ms/层。这说明大模型推理瓶颈往往在内存带宽而非算力。
    关联知识: 模型规模分析是硬件选型和优化策略的基础——参数量决定最低显存需求→驱动量化技术选择；FLOPs决定计算成本→驱动批处理和投机解码优化；KV Cache大小决定内存增长→驱动PagedAttention和KV压缩。后续模块的所有优化本质上都是在减少有效参数量、FLOPs或内存占用。
  5. 核心原理: Transformer有三大架构变体：1）Encoder-Only（如BERT）——采用双向注意力（无掩码），每个token可以看到序列中所有其他token，擅长理解任务（分类、NER、问答）。2）Decoder-Only（如GPT/LLaMA/Qwen）——采用因果掩码（Causal Mask），每个token只能看到当前位置及之前的token，通过自回归方式逐token生成，是当前LLM的主流架构。3）Encoder-Decoder（如T5/BART）——编码器双向处理输入，解码器单向生成输出，擅长序列到序列任务（翻译、摘要）。此外，MoE（混合专家）是一种正交的架构创新——用多个专家FFN+路由网络替代单一FFN，每次推理只激活top-k个专家（如Mixtral激活2/8），实现"大参数量、小计算量"的效果。
    关键要点: ①掩码方式是架构的根本区别——双向（BERT）、因果单向（GPT）、编码双向+解码单向（T5）；②Decoder-Only是当前LLM主流，推理优化技术栈最成熟；③MoE是正交创新，DeepSeek-V3总参671B活跃仅37B，推理成本远低于同能力密集模型；④GQA通过共享KV头减少内存，Llama-3全部采用GQA；⑤MLA（Multi-Latent Attention）是DeepSeek-V2/V3的创新，进一步压缩KV Cache。
    实际案例: 不同架构的实际选择：文本生成（ChatGPT类产品）→Decoder-Only（GPT-4、Llama-3、Qwen2.5）；文本分类/NER→Encoder-Only（BERT-base/large）；机器翻译→Encoder-Decoder（T5、NLLB）或Decoder-Only（大模型通用能力）。MoE的实际部署挑战：Mixtral-8x7B在多GPU上需要专家并行（Expert Parallelism），All-to-All通信开销可能成为瓶颈。路由不均衡会导致部分专家过载、部分闲置，需要辅助损失函数平衡负载。
    关联知识: 架构变体决定了推理优化的方向——Decoder-Only的KV Cache管理是核心（模块2）；MoE的专家路由和负载均衡是独特挑战（模块5）；Encoder-Decoder需要分别优化编码和解码阶段。理解架构差异有助于在实际项目中做出正确的技术选型：不是所有场景都需要最大的Decoder-Only模型，有时一个经过蒸馏的小模型或特定架构（如BERT做分类）反而更高效。
向前串联： Transformer 的核心组件的知识将在后续的推理框架选型、性能基准测试、生产环境调优中反复应用。 建议在本章学习后，立即用真实模型（如LLaMA-7B）进行动手实验， 将概念理解转化为可测量的性能数据（延迟、吞吐、显存占用）， 这些数据将成为后续优化决策的量化依据。
