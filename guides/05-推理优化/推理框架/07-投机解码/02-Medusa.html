🐍 Medusa

本节目标：理解 Medusa 如何通过额外的预测头实现无需小模型的投机解码。

1. 核心思想

在大模型上添加多个额外的预测头，每个头预测未来第 K 个 token：

1.1 Medusa Head 的架构设计

每个 Medusa Head 是一个轻量级的 MLP（通常 1-2 层），输入为 Transformer 最后一层的隐藏状态 h，输出为第 k 个未来位置的 logits 分布。头 0 就是原始 LM Head（预测 t+1），头 1 预测 t+2，以此类推。所有头共享同一个隐藏状态，因此只需一次前向传播。

1.2 Top-K 候选生成与候选树构建

每个 Medusa Head 不只预测 1 个 token，而是输出 Top-K 个候选（通常 K=5-10）。将所有头的 Top-K 候选组合成一棵候选树，树的宽度为 K，深度为头的数量。然后用原始模型一次前向传播验证所有候选路径，找到最长的匹配前缀。

1.3 典型注意力掩码与位置编码处理

Medusa 的验证阶段需要特殊的 attention mask：每个候选 token 只能看到自己在树中的祖先节点。位置编码也需要特殊处理——Medusa Head 预测的是"未来第 k 步"的 token，其位置 ID 应该是当前位置 + k，而不是候选树中的位置。

Medusa 的验证阶段需要特殊的 attention mask：每个候选 token 只能看到自己在树中的祖先节点。位置编码也需要特殊处理——Medusa Head 预测的是"未来第 k 步"的 token，其位置 ID 应该是当前位置 + k，而不是候选树中的位置。

🔬 深入：Medusa Head 与 Speculative Decoding 的理论联系
Medusa 可以看作 speculative decoding 的一个特例——"draft model"被替换为"多个轻量级预测头"。从信息论角度看，Medusa Head 的预测能力受限于它只能访问当前隐藏状态 h，而无法看到未来的 token 分布。这解释了为什么头越远（k 越大），预测准确率越低。Medusa-2 通过联合训练让模型"学会"产生更可预测的隐藏状态，相当于在信息论意义上最大化了 h 对未来 token 的互信息。
通俗类比：标准模型就像一个射手一次只射一支箭，而 Medusa 就像一个章鱼（Medusa 的本意是美杜莎/水母，多触手）——同时伸出多条触手去抓不同的目标。触手 0（原始 LM Head）最准，触手 1 稍差，触手 3 可能抓错，但总有一条能抓对。关键是：这些触手共享同一个"大脑"（隐藏状态），所以几乎不增加额外计算。
常见误区：很多人以为 Medusa Head 和原始 LM Head 是并行的独立模型。实际上，Medusa Head 是非常轻量的线性层（通常只有 1-2 层 MLP），它们共享同一个 Transformer 主干的隐藏状态。额外参数量通常只有原始模型的 1-2%，推理开销增加约 5-10%，但可以一次预测多个 token，净收益显著。
深入探讨：Medusa 的设计哲学是"最小化额外开销"。与草稿模型法（Draft Model）相比，Medusa 有三个关键优势：(1) 无需维护和加载额外模型，显存占用更少；(2) 无需额外的前向传播，所有预测头共享一次前向传播；(3) 训练简单——冻结原始模型，只训练 Medusa Head，通常只需几千条数据和几小时训练。但 Medusa 也有局限：预测头之间的独立性意味着无法捕捉 token 之间的依赖关系（如"t+2 依赖于 t+1"），这在一定程度上限制了接受率。

2. 优势

2.1 显存效率对比

草稿模型法需要加载完整的 draft model（即使是小模型，7B 的 1.3B draft 也需要 ~2.6GB 显存）。Medusa 只增加 4 个 MLP 头，额外参数约 100-200MB，显存开销几乎可以忽略。这使得 Medusa 特别适合显存紧张的单 GPU 部署场景。

2.2 计算开销分析

Medusa 的额外计算只来自预测头的前向传播（几个矩阵乘法），约占主模型前向传播的 5-10%。而草稿模型需要一次完整的前向传播（虽然模型更小）。在 GPU 计算资源紧张时，Medusa 的轻量级优势更加明显。

2.3 部署简易性

Medusa 的部署极其简单——只需在原始模型上附加几个线性层，无需修改推理框架的核心逻辑。vLLM 和 SGLang 都原生支持 Medusa，配置几行代码即可开启。

⚠ Medusa 的局限性：Medusa Head 之间是独立的——头 2 预测 t+3 时不知道头 1 选了什么 token 作为 t+2。这意味着 Medusa 无法捕捉 token 之间的依赖关系（如语法结构、代码缩进）。对于确定性高的任务（如代码生成），这个限制影响较小；但对于需要长程依赖的任务（如诗歌生成），接受率会显著下降。EAGLE 通过引入特征头来建模这种依赖，是 Medusa 的一个重要改进方向。
  • 不需要额外的小模型
  • 所有预测共享同一个前向传播
  • Medusa Head 很小，额外开销极小
通俗类比：Medusa 的优势就像"批量采购"——你去超市买 4 件东西，不需要跑 4 趟（4 次前向传播），而是一趟买齐（1 次前向传播 + 4 个预测头）。虽然第 3、4 件可能买错（预测不准），但大部分时候能省很多时间。
常见误区：Medusa 的加速效果高度依赖接受率。如果 Medusa Head 的预测准确率很低（比如 <30%），那么大部分预测的 token 都会被拒绝，反而浪费了验证的计算。因此，Medusa Head 的训练质量至关重要。在实际部署中，建议先在目标任务上评估 Medusa Head 的 Top-K 接受率，再决定是否开启投机解码。对于随机性很高的任务（如创意写作），Medusa 的加速效果可能不明显。
深入探讨：Medusa 的训练策略分两种：(1) Medusa-1：冻结原始模型，只训练 Medusa Head。这是最简单的方式，适合已有部署模型的场景。(2) Medusa-2：联合训练原始模型和 Medusa Head，让模型"学会"被 Medusa 预测。Medusa-2 的接受率更高，但需要更多的训练数据和计算资源。在实践中，Medusa-1 通常能达到 2-3x 加速，Medusa-2 可以达到 3-4x 加速。

3. 训练

3.1 Medusa-1 vs Medusa-2 训练策略

Medusa-1 冻结原始模型，只训练 Medusa Head，适合已有部署模型的场景。Medusa-2 联合训练原始模型和 Medusa Head，让模型"学会"被预测，接受率更高但需要更多计算资源。实践中，Medusa-1 可达 2-2.5x 加速，Medusa-2 可达 3-4x 加速。

3.2 训练数据与损失函数设计

Medusa Head 的训练使用标准的交叉熵损失，但每个头的 target 是不同的——头 k 的 target 是位置 t+k+1 的真实 token。训练数据应覆盖目标任务的分布，否则在特定领域上接受率会大幅下降。建议至少 10K-50K 条多样化样本。

3.3 接受率评估与调优

训练完成后，需要在验证集上评估每个头的 Top-K 接受率。如果某个头的接受率过低（如头 3 的 Top-1 接受率 < 20%），可以考虑减少头的数量或增加训练数据。接受率的提升直接转化为加速比的提升。

# Medusa 训练代码示例
import torch
import torch.nn as nn
from transformers import AutoModelForCausalLM

class MedusaHead(nn.Module):
    """单个 Medusa Head: 轻量级 MLP 预测未来第 k 个 token"""
    def __init__(self, hidden_size, vocab_size, k):
        super().__init__()
        self.k = k
        self.mlp = nn.Sequential(
            nn.Linear(hidden_size, hidden_size),
            nn.SiLU(),
            nn.Linear(hidden_size, vocab_size),
        )

    def forward(self, hidden_states):
        return self.mlp(hidden_states)  # [batch, seq_len, vocab_size]

class MedusaModel(nn.Module):
    def __init__(self, base_model_path, num_heads=4):
        super().__init__()
        self.base_model = AutoModelForCausalLM.from_pretrained(base_model_path)
        hidden_size = self.base_model.config.hidden_size
        vocab_size = self.base_model.config.vocab_size

        # 冻结原始模型
        for param in self.base_model.parameters():
            param.requires_grad = False

        # 创建 Medusa Heads
        self.medusa_heads = nn.ModuleList([
            MedusaHead(hidden_size, vocab_size, k=i+1)
            for i in range(num_heads)
        ])

    def forward(self, input_ids, labels=None):
        outputs = self.base_model(input_ids, output_hidden_states=True)
        hidden = outputs.hidden_states[-1]  # 最后一层隐藏状态

        loss = 0
        logits_list = []
        for k, head in enumerate(self.medusa_heads):
            head_logits = head(hidden)
            logits_list.append(head_logits)
            if labels is not None:
                # 头 k 预测位置 t+k+1 的 token
                shifted_logits = head_logits[:, :-(k+1), :]
                shifted_labels = labels[:, (k+1):]
                loss += nn.functional.cross_entropy(
                    shifted_logits.reshape(-1, shifted_logits.size(-1)),
                    shifted_labels.reshape(-1)
                )

        return loss / len(self.medusa_heads), logits_list

# 训练循环
model = MedusaModel("meta-llama/Llama-3-8B", num_heads=4)
optimizer = torch.optim.AdamW(model.medusa_heads.parameters(), lr=1e-4)

# Medusa 推理代码示例
import torch

def medusa_generate(model, input_ids, max_new_tokens=50, top_k=5):
    """Medusa 投机解码推理"""
    generated = input_ids.clone()

    for _ in range(max_new_tokens):
        # 1. 一次前向传播获得所有头的预测
        with torch.no_grad():
            outputs = model.base_model(generated, output_hidden_states=True)
            hidden = outputs.hidden_states[-1]

        # 2. 收集每个头的 Top-K 候选
        candidates = {}
        base_logits = model.base_model.lm_head(hidden[:, -1:])
        candidates[0] = torch.topk(base_logits[:, -1], top_k, dim=-1)

        for k, head in enumerate(model.medusa_heads):
            head_logits = head(hidden[:, -1:])
            candidates[k+1] = torch.topk(head_logits[:, -1], top_k, dim=-1)

        # 3. 构建候选树并验证（简化版：线性验证）
        accepted = [candidates[0].indices[0, 0].item()]
        for k in range(1, len(candidates)):
            best_candidate = candidates[k].indices[0, 0].item()
            # 实际实现中需要用主模型验证
            accepted.append(best_candidate)

        # 4. 接受最长匹配前缀
        new_tokens = torch.tensor([accepted], device=generated.device)
        generated = torch.cat([generated, new_tokens], dim=1)

    return generated

Medusa 的数学建模与接受率分析

Medusa Head 的独立性假设与概率模型

Medusa 的核心原理建立在一个关键假设之上：每个预测头可以独立预测未来第 k 个 token。从数学本质角度来看，这意味着：

P(t_{i+k} | h_i) ≠ P(t_{i+k} | h_i, t_{i+1}, ..., t_{i+k-1})

左边是 Medusa 的建模方式——每个头只基于当前隐藏状态 h_i 独立预测。右边是真实条件概率——第 k 个 token 依赖于前面所有 token。为什么存在差距？因为 Transformer 的隐藏状态 h_i 编码的是截至位置 i 的所有信息，但无法完美编码未来 token 的条件依赖。从信息论推导可以证明，当 k 增大时，h_i 对 t_{i+k} 的互信息 I(h_i; t_{i+k}) 递减，这就是为什么头越远准确率越低的底层原理。

Medusa Head 准确率衰减的信息论推导
设 H(t_{i+k} | h_i) 为给定隐藏状态后预测第 k 个未来 token 的条件熵。根据链式法则：

H(t_{i+k} | h_i) = H(t_{i+k} | t_{i+1},...,t_{i+k-1}, h_i) + I(t_{i+1},...,t_{i+k-1}; t_{i+k} | h_i)

由于 Medusa Head 无法访问中间 token t_{i+1}~t_{i+k-1}，它丢失了互信息项 I(t_{i+1},...,t_{i+k-1}; t_{i+k} | h_i)。推导可得，这个丢失的互信息随 k 线性增长（近似），因此 Medusa-1 的头 3 接受率通常只有头 0 的 40-60%。为什么 Medusa-2 能改善？因为联合训练让模型的隐藏状态 h_i "主动"编码更多关于未来 token 的信息，本质上是最大化了 I(h_i; t_{i+k})。

候选树的组合优化与验证效率

Medusa 使用 Top-K 候选构建候选树，树的宽度为 K，深度为预测头数量 N。从公式角度看，候选路径总数为 K^N——4 个头各取 Top-5 时，候选路径数为 5^4 = 625 条。

为什么不需要验证所有路径？底层的优化原理在于：只需找到最长匹配前缀。这意味着可以采用贪心策略——从头 0 开始，逐层验证，一旦某层所有候选都被拒绝就停止。数学本质上，这是一个树搜索问题，其时间复杂度从 O(K^N) 降低到 O(K·N)。

候选树的验证工作机制还需要特殊的 attention mask。为什么？因为候选树中不同路径的 token 之间不应互相看到——它们代表的是"可能的未来"，而非"确定的过去"。构造 mask 矩阵 M 使得 M[i][j] = 1 当且仅当 j 是 i 在树中的祖先节点，这保证了每个候选路径内部的因果性。

# Medusa Head 接受率评估脚本
import torch
from transformers import AutoTokenizer

def evaluate_acceptance_rate(model, tokenizer, eval_texts, k=4, top_k=5):
    """评估每个 Medusa Head 的接受率"""
    results = {i: {"correct": 0, "total": 0} for i in range(k+1)}

    for text in eval_texts:
        inputs = tokenizer(text, return_tensors="pt").to(model.device)
        input_ids = inputs["input_ids"]

        with torch.no_grad():
            outputs = model.base_model(input_ids, output_hidden_states=True)
            hidden = outputs.hidden_states[-1]

        # 原始 LM Head 的预测 (头 0)
        base_logits = model.base_model.lm_head(hidden[0])
        base_probs = torch.softmax(base_logits, dim=-1)
        base_topk = torch.topk(base_probs, top_k, dim=-1)

        # 评估每个头的接受率
        for pos in range(input_ids.shape[1] - k - 1):
            # 头 0: 预测下一个 token
            real_next = input_ids[0, pos + 1].item()
            if real_next in base_topk.indices[pos]:
                results[0]["correct"] += 1
            results[0]["total"] += 1

            # 头 k: 预测未来第 k+1 个 token
            for head_idx, head in enumerate(model.medusa_heads):
                head_logits = head(hidden[0:1, pos:pos+1])
                head_probs = torch.softmax(head_logits[0, 0], dim=-1)
                head_topk = torch.topk(head_probs, top_k, dim=-1)
                real_future = input_ids[0, pos + head_idx + 2].item()
                if real_future in head_topk.indices:
                    results[head_idx+1]["correct"] += 1
                results[head_idx+1]["total"] += 1

    # 输出结果
    print("Medusa Head 接受率评估:")
    print("-" * 50)
    for head_id, stats in results.items():
        rate = stats["correct"] / max(stats["total"], 1)
        print(f"头 {head_id}: Top-{top_k} 接受率 = {rate:.1%} "
              f"({stats['correct']}/{stats['total']})")

    return results
