🎲 采样策略

本节目标：理解模型输出 logits 后，如何选择下一个 token。采样策略直接影响生成文本的质量、多样性和确定性。

Temperature效果适用场景
0.0确定性（等同贪心）代码、数学、事实问答
0.1 - 0.3低随机性，高质量翻译、摘要
0.7 - 1.0平衡质量和多样性对话、创意写作
1.5 - 2.0高随机性，可能不连贯头脑风暴、创意激发

只从概率最高的 k 个 token 中采样：

# Top-k 采样 (k=50)
topk_values, topk_indices = logits.topk(50)
probs = softmax(topk_values)
next_token = topk_indices[multinomial(probs)]
  • k=1：等同贪心解码
  • k=50-100：常用范围
  • k=vocab_size：等同标准采样

5. Top-p (Nucleus) 采样

Top-p (Nucleus) 采样

维度训练(Training)推理(Inference)
目标学习参数生成输出
计算模式前向+反向传播仅前向传播
批处理固定大batch动态batch(请求驱动)
延迟要求天/周级别毫秒/秒级
成本特征一次性(GPU集群)持续性(按QPS扩展)

选择累积概率达到 p 的最小 token 集合：

# Top-p 采样 (p=0.9)
sorted_probs, sorted_indices = logits.softmax(dim=-1).sort(descending=True)
cumsum = sorted_probs.cumsum(dim=-1)
# 找到累积概率超过 p 的位置
cutoff_idx = (cumsum > 0.9).nonzero()[0]
# 只从 cutoff 之前的 token 中采样
sorted_probs[cutoff_idx+1:] = 0
sorted_probs /= sorted_probs.sum()  # 重新归一化
next_token = sorted_indices[multinomial(sorted_probs)]
💡 Top-p 的优势：自适应——当模型很确定时（如 "中国的首都是" → "北"），只保留少数 token；当模型不确定时（如 "今天我想吃"），保留更多 token。比固定 k 更灵活。

从 Logits 到 Token的技术原理与数学推导： 从底层硬件角度分析，从 Logits 到 Token的性能瓶颈通常在于内存带宽而非计算能力。 推导：以A100为例，FP16峰值算力312 TFLOPS，HBM带宽2TB/s， 算术强度转折点 = 312T/2T = 156 FLOPs/Byte。为什么这很重要？ 因为当实际算术强度低于此值时，任务是内存受限的——增加更多计算单元不会提升性能。 这一数学本质决定了从 Logits 到 Token的所有优化策略都应围绕减少内存访问展开。 从工作机制看，算子融合（减少中间结果写回）和批处理（共享参数读取） 是提升算术强度的两大核心手段。

✅ 参考答案与详解
  1. 核心原理: 从 Logits 到 Token 的过程是 LLM 推理的核心采样管线。模型的 Transformer 层处理输入后，在最后一个位置输出一个 hidden_state 向量，该向量经过 LM Head（一个线性映射层，将隐藏维度映射到词表大小 vocab_size）得到 logits 向量——即每个 token 的"原始得分"。logits 本身不是概率，需要先通过温度缩放（logits / temperature）调整分布的锐度，再经过 softmax 函数归一化为概率分布。最后，采样策略（argmax 或 multinomial）从概率分布中选择一个 token 作为输出。这一过程在每一步解码中重复执行，形成自回归生成循环。
    关键要点: ① logits 是未归一化的原始得分，不是概率；② 温度缩放在 softmax 之前应用，控制分布锐度；③ LM Head 实现从隐藏状态到词表空间的映射；④ 采样策略决定了从概率分布到最终 token 的选择方式；⑤ 整个过程在每个解码步骤中循环执行。
    实际案例: 当用户输入"今天天气"时，模型处理后输出 logits 如 [2.1, -0.5, 3.8, 1.2, -1.0, 4.2, 0.3]，对应词表中"的""了""是""在""和""很""也"等 token。经过 softmax 后，"很"（logit=4.2）获得最高概率。若使用 greedy 策略则直接选"很"，若使用随机采样则按概率分布抽取。
    关联知识: Logits→Token 的过程与 Tokenizer（模块01）紧密相关，因为词表大小直接决定 logits 向量维度；与自回归解码（模块02）配合形成完整的生成循环；温度参数在此处的应用直接影响后续所有采样策略的效果。
  2. 核心原理: Greedy Decoding（贪心解码）是最简单的采样策略：在每个解码步骤中，直接选择概率最高的 token（即 argmax 操作）。数学表达为 next_token = argmax(P(token|context))，它完全消除了随机性，对于相同的输入 prompt 总是产生完全相同的输出。贪心解码的计算开销极小，因为只需一次 argmax 运算，不需要维护多个候选序列。但其根本缺陷在于局部最优——每步只选当前最优的 token，无法保证全局序列的最优性，且容易陷入重复循环（如反复生成"的的的"）。
    关键要点: ① 每步选择概率最高的 token（argmax），完全确定性；② 相同输入总是产生相同输出，可复现性强；③ 无法保证全局最优，只追求局部最优；④ 计算开销最小，速度最快；⑤ 容易产生重复、单调的文本。
    实际案例: 在代码生成场景中，给定函数签名"def calculate_area(radius):"，greedy decoding 会稳定地生成"return 3.14159 * radius ** 2"这类确定性代码，因为每步都有明确的最高概率 token。这比随机采样更可靠，避免了引入语法错误或无关变量名。
    关联知识: Greedy decoding 是 beam_width=1 的特殊情况，可视为 beam search 的退化形式；与 temperature=0 等价（温度趋近0时 softmax 退化为 argmax）；在需要确定性的场景（如代码生成、数学推理）中是首选策略。
  3. 核心原理: Temperature 是控制概率分布"尖锐程度"的超参数，通过在 softmax 之前对 logits 进行缩放实现：probs = softmax(logits / temperature)。当 temperature < 1 时，logits 被放大，概率分布变得更尖锐（高概率 token 概率更高，低概率 token 被抑制），接近贪心行为；当 temperature > 1 时，logits 被缩小，分布变得更平坦（各 token 概率趋于均匀），增加随机性和多样性。temperature = 1.0 保持原始分布不变。本质上，temperature 是信息论中 softmax 的"温度"参数，控制着从确定性（T→0）到完全随机（T→∞）的连续谱。
    关键要点: ① 温度在 logits 上做除法，再送入 softmax；② T<1 使分布更尖锐，趋向确定性；③ T>1 使分布更平坦，增加随机性；④ T=0 等价于 greedy decoding；⑤ 通常与 top-p/top-k 组合使用。
    实际案例: 对于同一个 prompt "写一首关于月亮的诗"，temperature=0.2 时模型可能每次都输出相似的"明月照窗前"，适合需要稳定性的翻译任务；而 temperature=1.5 时，每次生成可能产出截然不同的诗句如"银盘挂夜空""嫦娥舞广寒"等，适合创意写作场景中激发灵感。
    关联知识: Temperature 与 softmax 函数的数学性质密切相关；与 top-p/top-k 采样策略是正交关系，通常组合使用（如 vLLM 中 temperature=0.7 + top_p=0.9）；在存在惩罚（presence_penalty）和频率惩罚（frequency_penalty）的框架中，温度调整在这些惩罚之后应用。
  4. 核心原理: Top-k 采样将候选 token 集合限制为概率最高的 k 个，然后在这 k 个 token 中按概率重新归一化后随机采样。具体实现为：先对 logits 做 softmax 得到概率分布，取 topk(k) 得到前 k 个最大概率及其索引，然后对这 k 个概率做 multinomial 采样。k=1 时退化为 greedy decoding，k=vocab_size 时退化为标准随机采样。Top-k 的核心思想是过滤掉极低概率的"噪声" token（如拼写错误、无关词汇），同时保留足够的多样性。常用范围是 k=50-100。
    关键要点: ① 只从概率最高的 k 个 token 中采样，其余 token 概率设为0；② k=1 等价于贪心解码，k=vocab_size 等价于无限制采样；③ 常用范围 k=50-100，在多样性和质量之间取得平衡；④ 实现简单，计算开销低（topk 操作高效）；⑤ 缺点是 k 值固定，不适应模型置信度的变化。
    实际案例: 在对话机器人中设置 top_k=50，当用户问"今天几号？"时，虽然模型很确定答案，但 top-k=50 仍可能从50个候选中随机选到非最优 token，导致输出不够精确。相比之下，当模型对"今天我想吃____"不确定时，k=50 又可能过滤掉一些合理的长尾食物名称（如"螺蛳粉"），因为它们不在 top-50 内。
    关联知识: Top-k 采样的固定 k 值缺陷正是 Top-p（Nucleus）采样要解决的问题；与 temperature 组合使用时，temperature 在 top-k 之前应用；在实际框架如 vLLM、TGI 中，top-k 和 top-p 通常同时配置，取两者的交集作为最终候选集。
  5. 核心原理: Top-p（Nucleus）采样采用自适应策略：将所有 token 按概率从高到低排序，然后累加概率直到总和达到阈值 p，只从这个"核心"（nucleus）token 集合中采样。具体实现为：softmax 得到概率分布 → 按降序排序 → 计算累积概率 → 找到第一个 cumsum > p 的位置作为截断点 → 将截断点之后的 token 概率置零 → 重新归一化 → 采样。其核心优势是自适应性：当模型很确定时（如"中国的首都是→北"），nucleus 只包含极少数 token；当模型不确定时（如"今天我想吃→"），nucleus 自动扩大包含更多候选。
    关键要点: ① 按累积概率阈值动态确定候选集大小，自适应模型置信度；② p=0.9 或 p=0.95 是常用值；③ 比固定 k 的 top-k 更灵活，避免了 k 值选择的困境；④ 实现需要排序、累积求和、截断、重新归一化四步；⑤ 是目前最推荐的通用采样策略。
    实际案例: 设置 top_p=0.9 时，对于"巴黎是____的首都"，概率分布极度集中在"法国"上（概率约0.98），累积概率超过0.9只需1个 token，nucleus 大小=1，几乎等同于 greedy；而对于"这首歌让我想起了____"，概率分散在多个候选上（"你""童年""故乡"等），需要约10-20个 token 才达到0.9的累积概率，nucleus 自动扩大，保留了丰富的多样性。
    关联知识: Top-p 与 Top-k 是互补关系，实际部署中常同时使用（如 vLLM 中 top_k=50, top_p=0.9）；Nucleus 采样的"核心"概念与信息论中的熵密切相关——高熵分布需要更大的 nucleus；在 beam search 无法使用的场景（如实时对话），top-p 是质量与多样性平衡的最佳选择。
