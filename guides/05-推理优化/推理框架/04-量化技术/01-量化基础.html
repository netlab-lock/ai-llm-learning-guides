# 手动 INT8 对称量化
import torch

weight = torch.randn(4096, 4096)  # FP32 权重

# 计算 scale
scale = weight.abs().max() / 127.0

# 量化
weight_int8 = torch.round(weight / scale).clamp(-128, 127).to(torch.int8)

# 反量化
weight_dequant = weight_int8.float() * scale

# 误差
error = (weight - weight_dequant).abs().mean()
print(f"量化误差: {error:.6f}")  # ~0.001

什么是量化？的技术原理与数学推导： 从底层硬件角度分析，什么是量化？的性能瓶颈通常在于内存带宽而非计算能力。 推导：以A100为例，FP16峰值算力312 TFLOPS，HBM带宽2TB/s， 算术强度转折点 = 312T/2T = 156 FLOPs/Byte。为什么这很重要？ 因为当实际算术强度低于此值时，任务是内存受限的——增加更多计算单元不会提升性能。 这一数学本质决定了什么是量化？的所有优化策略都应围绕减少内存访问展开。 从工作机制看，算子融合（减少中间结果写回）和批处理（共享参数读取） 是提升算术强度的两大核心手段。
什么是量化？的系统架构分析： 从系统设计的原理出发，什么是量化？采用分层架构来管理复杂性。 为什么分层是主流选择？推导如下： 设系统有N个组件，全连接交互复杂度为O(N²)，分层后降低到O(N·L)，L为层数。 数学本质上，这是用少量层间通信开销换取大幅降低的组件间耦合。 在实际工作机制中，每一层只与相邻层通信，通过标准接口解耦—— 这类似于TCP/IP协议栈的设计原理，每层独立优化，层内改动不影响整体。

什么是量化？的性能优化策略

优化什么是量化？性能需要系统性的方法论。首先用profiling工具定位瓶颈—— 从底层看，GPU推理的瓶颈通常表现为三种模式： 1) 计算受限（GPU利用率>80%，SM活跃）； 2) 内存受限（带宽利用率高，但计算单元空闲）； 3) 通信受限（多卡场景，NCCL等待时间长）。 为什么要区分这三种模式？因为优化策略完全不同： 计算受限→减少FLOPs（量化/剪枝）；内存受限→减少数据搬运（算子融合/缓存）； 通信受限→优化通信拓扑（梯度压缩/异步通信）。 这一原理是推理性能调优的基础框架。

# 什么是量化？ - 性能基准测试
import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    """测量推理延迟和吞吐"""
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
    elapsed = (time.perf_counter() - start) / iters
    print(f"Latency: {elapsed*1000:.2f}ms")
    print(f"Throughput: {1/elapsed:.0f} req/s")
    return elapsed

✅ 参考答案与详解
  1. 核心原理: 量化的核心原理是将连续的高精度浮点数值映射到离散的低精度整数集合，从而减少存储空间和计算资源需求。具体来说，一个 FP32 权重占用 32 位（4 字节），而 INT8 仅需 8 位（1 字节），INT4 仅需 4 位（0.5 字节）。量化的核心数学关系是：quantized_value = round(real_value / scale) + zero_point，反量化则是 real_value = scale × (quantized_value - zero_point)。其中 scale（缩放因子）决定了浮点数到整数的映射比例，zero_point（零点偏移）确保浮点零能被精确表示。量化分为权重量化（Weight Quantization）和激活量化（Activation Quantization），在 LLM 推理场景中，权重量化是最常用的，因为权重在推理过程中不变，可以在部署前离线完成。量化本质上是一种有损压缩技术，其精度损失来源有两方面：截断误差（超出量化范围的值被钳位）和舍入误差（连续值映射到离散值的精度损失）。量化粒度（per-tensor/per-channel/per-group）决定了 scale 和 zero_point 的共享范围，粒度越细精度越高但存储开销越大。
    关键要点: ①量化公式 q = round(r/s) + zp 是所有量化方法的数学基础 ②scale 决定映射精度，zero_point 决定偏移 ③权重量化可在部署前离线完成，激活量化需要运行时动态处理 ④精度损失来自截断和舍入两种误差 ⑤量化粒度（per-tensor/channel/group）在精度和存储开销之间权衡
    实际案例: 以 LLaMA-2-7B 为例，FP16 权重约 14GB，单张 RTX 4090（24GB 显存）虽然能加载但留给 KV Cache 的空间很小。进行 INT8 量化后权重降至约 7GB，显存占用减半，有充足空间做长序列推理。INT4 量化后进一步降至约 3.5GB，在消费级 GPU 上可以轻松部署，且 perplexity 仅上升约 2-3%。
    关联知识: 量化是整个推理优化体系的基础技术，与后续 GPTQ（OBS 方法优化量化误差）、AWQ（保护重要权重通道）、GGUF（CPU/GPU 混合推理格式）、FP8（硬件原生支持）等具体方法直接关联。量化也与模型剪枝、知识蒸馏共同构成模型压缩的三大技术路线。
  2. 核心原理: 对称量化和非对称量化的本质区别在于浮点数零点在整数空间中的映射位置。对称量化强制将浮点零映射到整数零，公式为 q = round(r / scale)，scale = max(|x|) / (2^(bits-1) - 1)。其映射关系是一条过原点的直线，量化范围关于零点对称分布，如 INT8 为 [-127, 127]。优点是计算简单（无需处理 zero_point 偏移），适合权重分布接近零均值对称的情况（如经过良好初始化的权重矩阵）。非对称量化允许浮点零映射到任意整数值，公式为 q = round(r / scale) + zp，其中 scale = (max - min) / (2^bits - 1)，zp = round(-min / scale)。其映射关系是一条不过原点的直线，量化范围覆盖 [min, max] → [0, 2^bits - 1]，能充分利用全部量化级别。当数据分布呈现明显的非零均值时（如 ReLU 后的激活值全部 ≥ 0），非对称量化能避免大量量化级别的浪费。在实际选择上，对称量化适合计算受限场景（省去 zero_point 的加减运算），非对称量化适合精度敏感场景（充分利用量化范围减少误差）。GPTQ 选择非对称量化的原因是 INT4 仅有 16 个离散值，每一个量化级别的有效利用都至关重要。
    关键要点: ①对称量化：零→零，只存 scale，映射线过原点 ②非对称量化：零→zp，存 scale + zp，映射线不过原点 ③非对称量化在非零均值分布时量化范围利用率更高 ④对称量化计算更简洁（省去 zp 运算） ⑤INT4 场景下非对称量化精度优势更明显（16个值的利用率差异更大）
    实际案例: 在 Transformer 的注意力层中，Softmax 输出的注意力权重值域为 [0, 1] 且高度偏向小值区域（长尾分布）。如果用对称量化，scale 按 max=1 计算，[-1, 0) 的负半轴完全浪费，仅利用了 16 个 INT4 值中约 8 个；非对称量化用 [0, 1] → [0, 15] 映射，全部 16 个值都用于实际数据范围，精度损失显著降低。
    关联知识: 与本文的量化公式和 scale 计算直接关联，与 02-GPTQ 的误差补偿算法深度耦合（GPTQ 的 Hessian 引导量化依赖精确的量化/反量化循环），与 AWQ 的权重重要性评估方法互补（AWQ 从另一个角度解决精度问题）。在 SmoothQuant 中，还通过迁移量化难度从激活到权重来平衡两者的量化难度。
  3. 核心原理: 量化粒度是指共享同一个 scale 和 zero_point 的数值范围大小，它在量化精度和元数据存储开销之间形成核心权衡。粒度从粗到细分为四个层次：Per-Tensor（整个张量共用一个 scale/zp）、Per-Channel（每个输出通道共用一个 scale/zp）、Per-Group（每 group_size 个连续元素共用一个 scale/zp，典型值为 128）、Per-Token（每个 token 的激活值共用一个 scale/zp）。粒度越细，每个 scale/zp 覆盖的数值范围越小，该范围内的数值分布越均匀，因此量化误差越小。以 4096×4096 权重矩阵为例：Per-Tensor 仅需 1 个 scale（4 字节），但全局 max(|x|) 作为 scale 会严重浪费量化范围（只要一个极端异常值就会影响全部元素）；Per-Channel 需要 4096 个 scale（16KB），每行独立计算 max，精度显著提升；Per-Group (128) 需要 4096×32 = 131072 个 scale（约 512KB），每 128 个元素独立量化，进一步减少异常值的影响。在实际部署中，Per-Group 是 LLM 量化的主流选择（GPTQ/AWQ 默认 group_size=128），在精度和存储开销之间取得最佳平衡。GGUF 格式还支持混合粒度——对精度敏感的注意力层使用 Per-Channel，对精度容忍度高的 FFN 层使用 Per-Group。
    关键要点: ①粒度越细→精度越高→存储开销越大 ②Per-Tensor 精度最差但开销最小，Per-Token/Per-Element 精度最高但开销最大 ③group_size=128 是 LLM 量化的事实标准 ④混合粒度策略可在不同层使用不同粒度以优化整体效果 ⑤粒度选择直接影响 outlier（异常值）对量化精度的冲击范围
    实际案例: 以 LLaMA 的 FFN 层 gate_proj 权重（shape 11008×4096）为例，该层存在典型的 outlier 现象——某些通道的权重值可能是其他通道的 10-100 倍。Per-Tensor 量化时，这些 outlier 主导了 scale 的计算，导致正常范围的权重被压缩到极少的量化级别中，perplexity 恶化明显；切换到 Per-Group (128) 后，每个 group 独立计算 scale，outlier 的影响被限制在所在的 group 内，perplexity 可改善 0.5-1.0 个点。
    关联知识: 量化粒度是 02-GPTQ 和 03-AWQ 的核心配置参数（group_size），与 SmoothQuant 的 per-channel 激活量化策略密切相关。粒度选择还影响推理引擎的计算效率——Per-Group 量化需要额外的 dequantization 计算，这在 memory-bound 的 Decode 阶段是可接受的，但在 compute-bound 的 Prefill 阶段可能成为额外开销。
  4. 核心原理: PTQ（Post-Training Quantization，训练后量化）和 QAT（Quantization-Aware Training，量化感知训练）是实现量化的两种根本不同的方法论。PTQ 在模型训练完成后直接对权重进行量化，无需重新训练，只需少量校准数据（通常 128-512 条样本）来统计激活值分布以确定最优的 scale 和 zero_point。其核心思想是：已训练好的模型权重已经收敛到一个好的解，量化只需尽量减少对这个解的扰动。典型方法包括 GPTQ（基于 OBS/Hessian 的逐层最优量化）和 AWQ（基于激活值感知的权重保护量化）。QAT 则在训练过程中模拟量化效果，在前向传播中插入伪量化节点（fake quantization），让模型在训练时就学习适应量化带来的精度损失。其核心思想是：与其在量化后修补误差，不如让模型从一开始就适应低精度表示。QAT 的典型流程是在每个量化层的前向传播中插入 round + clamp 操作，但反向传播使用 Straight-Through Estimator (STE) 来绕过 round 函数的不可导性。对于 LLM 推理场景，PTQ 是绝对主流，因为 LLM 训练成本极高（LLaMA-3-70B 训练需要数千万美元的 GPU 资源），重新训练不现实；而 QAT 只在训练新模型或对精度要求极高的特定场景中使用。
    关键要点: ①PTQ 不需要训练，只需校准数据，适合已训练好的 LLM ②QAT 需要完整训练流程，精度最好但成本极高 ③PTQ 典型方法：GPTQ、AWQ、SmoothQuant ④QAT 使用 STE（Straight-Through Estimator）解决 round 函数不可导问题 ⑤LLM 推理几乎只用 PTQ，QAT 用于移动端小模型或从零训练场景
    实际案例: 对 LLaMA-3-8B 进行 INT4 量化对比：PTQ 方法（GPTQ）仅需 128 条 WikiText-2 样本做校准，耗时约 30 分钟（单卡 A100），perplexity 约从 5.8 升至 6.1；QAT 方法需要完整训练（约 1000 个 GPU 小时），perplexity 可控制在 5.9 以内，但成本是 PTQ 的 2000 倍以上。在移动端部署中（如手机端运行 BERT-tiny），QAT（如 PyTorch 的 torch.quantization.quantize_dynamic）是常用方法，因为小模型训练成本低且精度收益明显。
    关联知识: PTQ/QAT 的选择是量化的第一个决策点，直接影响后续技术路线（PTQ→GPTQ/AWQ/GGUF，QAT→训练框架集成）。与本文的量化粒度、对称/非对称选择共同构成量化的三大基础决策。在 06-量化实战中，会展示如何用 AutoGPTQ (PTQ) 和 TorchAO (QAT) 两种方式完成实际量化。
  5. 核心原理: 文中的量化示例展示了 INT8 对称量化的完整实现流程，这是理解所有量化方法的最基础代码。核心步骤分为四步：(1) 计算 scale = weight.abs().max() / 127.0，找到权重张量的绝对值最大值并除以 INT8 的最大正值 127，得到映射比例；(2) 量化 weight_int8 = torch.round(weight / scale).clamp(-128, 127).to(torch.int8)，先将每个权重值除以 scale 映射到整数范围，再 round 取整，clamp 钳位到 [-128, 127] 防止溢出，最后转为 int8 类型；(3) 反量化 weight_dequant = weight_int8.float() * scale，将整数权重乘以 scale 还原为浮点数；(4) 计算误差 error = (weight - weight_dequant).abs().mean()，衡量量化前后的平均绝对误差。这个示例的关键洞察是：量化误差约 0.001，相对 FP32 的权重值范围很小，说明 INT8 对称量化对大多数权重是"足够好"的。但需要注意的是，这个简单的 per-tensor 量化是最粗糙的粒度，实际应用中 GPTQ/AWQ 使用 per-group 量化可将误差再降低 3-5 倍。扩展到 INT4 时，量化范围从 [-128, 127] 缩小到 [-8, 7]（仅 16 个值），误差会显著增大，这就是为什么 GPTQ 需要 Hessian 引导的误差补偿算法。
    关键要点: ①scale 计公式是所有量化方法的共同基础 ②round + clamp 是量化的核心算子（round 映射 + clamp 防溢出） ③反量化是量化的逆过程，用于在推理时恢复浮点数 ④per-tensor INT8 量化误差约 0.001，已经是生产可用的精度 ⑤从 INT8 扩展到 INT4 时误差急剧增大，需要更复杂的算法（GPTQ/AWQ）来控制
    实际案例: 可以在 PyTorch 中扩展此示例进行对比实验：分别用 per-tensor 和 per-group (128) 两种粒度对同一个 4096×4096 矩阵做 INT8 量化，per-tensor 的平均误差约 0.001，per-group 约 0.0003（3 倍精度改善）。再尝试 INT4 量化：per-tensor INT4 误差约 0.02（是 INT8 的 20 倍），per-group INT4 误差约 0.008。这个实验直观展示了量化粒度和位数对精度的影响，也是理解 GPTQ/AWQ 必要性的起点。
    关联知识: 此示例的代码模式是所有量化框架的底层实现基础——AutoGPTQ、AutoAWQ、BitsAndBytes 的核心都基于此 scale/round/clamp 循环。与本文的对称/非对称量化公式直接对应，与量化粒度讨论中 per-tensor vs per-group 的精度差异形成实证。在 02-GPTQ 中，这个基础量化流程会被扩展为带 Hessian 补偿的逐列量化算法。
