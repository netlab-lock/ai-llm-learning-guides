剪枝通过移除模型中不重要的权重或结构来减小模型规模。

1.1 非结构化剪枝

将单个权重置零，形成稀疏矩阵。压缩率高但需要专用稀疏计算支持。

import torch.nn.utils.prune as prune
# 对线性层进行 L1 非结构化剪枝（移除 30% 权重）
prune.l1_unstructured(module, name='weight', amount=0.3)
prune.remove(module, 'weight')  # 永久化剪枝

1.2 结构化剪枝

移除整个注意力头、FFN 神经元或 Transformer 层。兼容标准硬件，但压缩率较低。

方法粒度压缩率硬件兼容质量影响
非结构化单个权重50-90%需稀疏支持中等
结构化-通道注意力头/神经元10-30%✅ 原生较小
结构化-层整个层10-50%✅ 原生较大
⚠️ 剪枝注意

LLM 的剪枝需要在剪枝后进行恢复微调 (fine-tuning)，否则质量会大幅下降。推荐使用 Wanda 或 SparseGPT 等 LLM 专用方法。

⚠️ 常见误区：非结构化剪枝可以直接加速推理

非结构化剪枝（将权重置零）虽然能达到 50-90% 的稀疏率，但在标准硬件上几乎不会带来实际加速。原因是：通用 GPU/CPU 的矩阵乘法内核不支持稀疏计算，零值权重仍然占用计算周期和内存带宽。要获得实际加速，需要：1) NVIDIA 的 2:4 结构化稀疏（Ampere+ GPU）；2) 专用稀疏计算库（如 DeepSparse）；3) 将非结构化稀疏转换为块稀疏格式。对于端侧部署，结构化剪枝（移除整个注意力头/FFN 神经元）才是更实用的选择。

量化是端侧部署最重要的压缩手段，将高精度浮点数转为低精度表示。

2.1 GPTQ

基于二阶信息的训练后量化方法。逐层量化，使用少量校准数据（128-256 样本）实现 INT4 精度。

# GPTQ 量化示例
from transformers import AutoModelForCausalLM, GPTQConfig
quantization_config = GPTQConfig(bits=4, dataset="c4", group_size=128)
model = AutoModelForCausalLM.from_pretrained("model", quantization_config=quantization_config)

2.2 AWQ (Activation-aware Weight Quantization)

基于激活值感知的权重量化。识别对输出影响大的权重通道并保持高精度，其余通道量化为 INT4。通常质量优于 GPTQ。

2.3 GGUF 量化

llama.cpp 专用量化方案，支持从 Q2 到 Q8 的多种精度。使用 importance matrix (imatrix) 可进一步提升量化质量。

💡 量化选型

端侧部署首选 AWQ 或 GGUF Q4_K_M——AWQ 适合 GPU 推理，GGUF 适合 CPU/NPU 混合推理。

使用大模型（教师）的输出指导小模型（学生）训练，让小模型获得接近大模型的能力。

  • 输出蒸馏：学生学习教师的 token 概率分布（KL 散度损失）
  • 特征蒸馏：学生学习教师的中间层表示
  • 数据增强：教师生成训练数据，用于学生模型的监督微调

典型案例：Phi-3 Mini (3.8B) 通过蒸馏和高质量数据达到了接近 7B 模型的效果。

⚠️ 常见误区：知识蒸馏可以无损压缩任何模型

知识蒸馏的效果高度依赖教师-学生架构的匹配度。当教师和学生的架构差异过大（如从 Transformer 蒸馏到 RNN），或学生模型容量不足以表达教师的知识时，蒸馏效果会大幅下降。常见陷阱：1) 直接用教师的 logits 做蒸馏但忽略了 temperature 参数调节——temperature 太低会让分布过于尖锐，学生只学到 argmax；2) 没有使用数据增强——教师生成的合成数据对蒸馏效果有决定性影响；3) 忽视了中间层特征蒸馏——仅用输出层蒸馏会丢失大量结构化知识。

实际部署中通常组合多种压缩技术：

知识蒸馏 (7B→3B) → 结构化剪枝 (层/头) → AWQ INT4 量化 → GGUF 转换

这种级联压缩可将 7B 模型从 14GB 压缩到约 2GB，同时保持 85-90% 的原始性能。

一、模型剪枝的技术原理与数学推导： 从底层硬件角度分析，一、模型剪枝的性能瓶颈通常在于内存带宽而非计算能力。 推导：以A100为例，FP16峰值算力312 TFLOPS，HBM带宽2TB/s， 算术强度转折点 = 312T/2T = 156 FLOPs/Byte。为什么这很重要？ 因为当实际算术强度低于此值时，任务是内存受限的——增加更多计算单元不会提升性能。 这一数学本质决定了一、模型剪枝的所有优化策略都应围绕减少内存访问展开。 从工作机制看，算子融合（减少中间结果写回）和批处理（共享参数读取） 是提升算术强度的两大核心手段。

✅ 参考答案与详解
  1. 核心原理: 模型剪枝通过识别并移除神经网络中不重要的权重或结构来减小模型规模，分为非结构化和结构化两种路径。非结构化剪枝(如L1 unstructured)将单个权重置零，可移除50-90%权重形成稀疏矩阵，但标准GPU/CPU的矩阵乘法内核不支持稀疏计算，零值仍占用计算周期和内存带宽，因此在通用硬件上几乎不会带来实际加速——需要NVIDIA 2:4结构化稀疏(Ampere+)或DeepSparse等专用库。结构化剪枝移除整个注意力头、FFN神经元或Transformer层，兼容标准硬件但压缩率较低(10-50%)。对于LLM必须在剪枝后进行恢复微调(fine-tuning)，推荐使用Wanda或SparseGPT等LLM专用方法。
    关键要点: ①非结构化剪枝压缩率高(50-90%)但标准硬件无法加速；②结构化剪枝压缩率低(10-50%)但兼容原生硬件；③LLM剪枝后必须恢复微调否则质量大幅下降；④Wanda/SparseGPT 是LLM专用剪枝方法；⑤端侧部署优先选择结构化剪枝
    实际案例: 对 Llama-2-13B 进行结构化注意力头剪枝，移除25%的注意力头后模型从26GB降至约20GB，在通用GPU上推理速度提升约20%，且perplexity仅增加5%
    关联知识: 结构化剪枝后的模型尺寸减小使其更适合后续INT4量化，形成"剪枝+量化"级联压缩链
  2. 核心原理: 量化技术通过将高精度浮点数(FP16/BF16)转换为低精度表示(INT8/INT4)来减小模型大小和计算开销，是端侧部署最重要的压缩手段。GPTQ基于二阶信息逐层量化，使用128-256样本校准数据实现INT4精度，通过最优脑外科手术(Optimal Brain Surgery)框架逐层优化量化误差。AWQ(Activation-aware Weight Quantization)基于激活值感知，识别对输出影响大的权重通道保持高精度，其余量化为INT4，通常质量优于GPTQ。GGUF量化是llama.cpp专用方案，支持Q2到Q8多种精度，imatrix量化通过校准数据进一步提升质量。量化选型建议：端侧部署首选AWQ(GPU推理)或GGUF Q4_K_M(CPU/NPU混合推理)。
    关键要点: ①GPTQ：基于二阶信息的训练后量化，适合INT4；②AWQ：激活值感知，质量通常优于GPTQ；③GGUF：llama.cpp专用，Q4_K_M为最佳平衡；④量化后必须验证输出质量(至少3-5种任务)；⑤AWQ 对 Qwen 系列效果好，对不同架构兼容性有差异
    实际案例: Qwen2.5-7B 从FP16(14GB)量化为AWQ INT4(3.8GB)后，在旗舰手机上推理延迟从120ms/token降至55ms/token，MMLU得分从68.0降至66.2(损失2.6%)
    关联知识: 量化带来的精度损失因任务而异——代码生成和数学推理对量化更敏感，建议用更高端位宽
  3. 核心原理: 知识蒸馏的核心思想是让一个小容量"学生模型"学习大容量"教师模型"的行为和表示，从而获得接近大模型的能力。包含三种主要方法：(a)输出蒸馏——学生学习教师token概率分布，使用KL散度损失函数，temperature参数控制分布平滑度，过低的temperature会让分布过于尖锐使学生只学到argmax；(b)特征蒸馏——学生学习教师中间层的表示，保留结构化知识；(c)数据增强——教师模型生成合成训练数据用于学生监督微调，这对蒸馏效果有决定性影响。典型案例是Phi-3 Mini(3.8B)通过蒸馏和高质量数据达到接近7B模型的效果。效果高度依赖教师-学生架构匹配度。
    关键要点: ①输出蒸馏：KL散度损失+温度参数调节是关键；②特征蒸馏：中间层表示匹配保留结构化知识；③数据增强：教师生成合成数据影响蒸馏效果；④架构匹配度影响蒸馏效果——差异过大效果骤降；⑤temperature参数过低导致分布尖锐，学生只学到argmax
    实际案例: Microsoft的 Phi-3 Mini(3.8B)通过从Phi-3 Medium蒸馏和使用高质量合成数据，在多个benchmark上接近7B模型的性能，证明了蒸馏+优质数据的威力
    关联知识: 蒸馏后的小模型进一步进行INT4量化形成级联压缩，可将7B模型从14GB压缩到约2GB同时保持85-90%性能
  4. 核心原理: 权重聚类通过k-means等算法将连续的权重值映射到有限的离散聚类中心(centroids)，用聚类中心索引替代原始权重值来实现压缩。具体流程为：(1)收集所有权重的值域分布；(2)用k-means算法迭代计算k个聚类中心；(3)每个权重替换为最近聚类中心的索引；(4)配合霍夫曼编码(Huffman Coding)可进一步压缩存储——出现频率高的聚类中心用短码字。当k=16(4-bit)时，每个权重只需4bit存储索引，加上聚类码本(codebook)的存储开销，总体可大幅降低模型尺寸。与量化不同，权重聚类的离散值是数据驱动的自适应值，而非固定的均匀间隔。
    关键要点: ①k-means聚类：数据驱动的自适应离散值；②配合霍夫曼编码进一步压缩；③聚类中心(codebook)有额外存储开销；④与量化相比：量化用均匀间隔，聚类用自适应值；⑤适合与推理引擎配合在运行时通过查表还原权重
    实际案例: 对7B模型使用16个聚类中心(4-bit索引)进行权重聚类，配合霍夫曼编码，模型从14GB压缩到约3.5GB，配合运行时查表还原推理
    关联知识: 权重聚类是非线性量化的一种形式，其自适应值域使其在某些层上比均匀量化更接近原始分布
  5. 核心原理: 组合策略将多种压缩技术级联应用，实现远超单一技术的压缩效果。典型级联路径为：知识蒸馏(7B→3B) → 结构化剪枝(层/头) → AWQ INT4量化 → GGUF转换。这种组合可将7B模型从14GB压缩到约2GB(压缩率85%)同时保持85-90%的原始性能。各阶段的作用：蒸馏负责减小模型基数，结构化剪枝在兼容硬件的前提下进一步减薄，INT4量化将位宽从16bit降至4bit(4倍压缩)，GGUF转换适配端侧推理引擎。关键是各阶段之间的质量衰减累积——每步损失叠加，最终需要控制在可接受范围内。
    关键要点: ①典型级联：蒸馏→剪枝→量化→格式转换；②累积质量衰减需控制在可接受范围；③每步压缩率相乘而非相加(蒸馏50%×剪枝70%×量化25%);④端侧部署首选AWQ或GGUF Q4_K_M；⑤组合方案可保持85-90%原始性能
    实际案例: Llama-2-7B(14GB FP16) → 蒸馏为4B结构 → 结构化剪枝移除20%层 → AWQ INT4量化 → 最终约2.1GB，MMLU从68.0降至62.5(损失8%)
    关联知识: 组合策略的效果取决于各阶段的协调——过度量化会导致"垃圾进垃圾出"，需在各阶段间插入质量评估
