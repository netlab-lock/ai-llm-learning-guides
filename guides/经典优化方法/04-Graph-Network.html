四、图论与网络优化

图是表达关系最自然的结构——节点是实体，边是关系。本篇覆盖最短路径、最小生成树、最大流/最小割、二分匹配、网络单纯形等经典算法，附完整推导和应用场景。

1. 最短路径

常见误区：易犯错误：使用Dijkstra算法时忽略负权边，导致结果不正确。应改用Bellman-Ford算法处理负权。
深入探讨：Dijkstra算法假设非负权边，时间复杂度O((V+E)log V)。Bellman-Ford可处理负权，时间复杂度O(VE)。Floyd-Warshall用于所有节点对，O(V^3)。

|        |
2        3
v        v
C --4--> D
最短路径A到D：A->B->D (4)
算法时间复杂度特点
DijkstraO((V+E)log V)非负权，贪心
Bellman-FordO(VE)可处理负权
Floyd-WarshallO(V^3)所有节点对
算法条件复杂度核心思想
Dijkstra非负权O((V+E)logV)贪心扩展最近节点
Bellman-Ford可有负权O(VE)松弛所有边V-1轮
Floyd-Warshall所有点对O(V³)枚举中间节点

1.1 Bellman-Ford：处理负权边

Dijkstra不能处理负权边。Bellman-Ford可以，还能检测负权环。

图: A→B(1), B→C(-2), C→A(-1), B→D(4)

Bellman-Ford: 对所有边松弛 |V|-1 轮
初始化: dist[A]=0, dist[B,C,D]=∞

第1轮 (松弛所有边):
  A→B: dist[B] = min(∞, 0+1) = 1
  B→C: dist[C] = min(∞, 1+(-2)) = -1
  C→A: dist[A] = min(0, -1+(-1)) = -2 ← 更新了!
  B→D: dist[D] = min(∞, 1+4) = 5

第2轮 (继续松弛):
  A→B: dist[B] = min(1, -2+1) = -1
  B→C: dist[C] = min(-1, -1+(-2)) = -3
  C→A: dist[A] = min(-2, -3+(-1)) = -4 ← 还在变!

第|V|-1轮后，再做一轮:
  如果还能更新 → 存在负权环!

时间: O(VE) — 比Dijkstra慢，但能力更强

1.2 Floyd-Warshall：所有点对最短路

一次计算任意两点之间的最短距离。O(V³)。

核心思想: 枚举中间节点k，看"经过k"是否更短

dp[k][i][j] = min(dp[k-1][i][j], dp[k-1][i][k] + dp[k-1][k][j])
                不经过k的最短路      经过k的最短路

伪代码 (可以压缩到二维):
for k in range(n):
    for i in range(n):
        for j in range(n):
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

还可以检测负权环: 如果 dist[i][i] < 0，说明i在负权环上

应用: 社交网络中任意两人最短"距离"、路由表预计算

1.3 A*算法：启发式引导搜索

Dijkstra的升级版。用启发函数h(n)估计当前节点到目标的距离，优先探索最有希望的节点。

A*: f(n) = g(n) + h(n)
  g(n) = 从起点到n的实际距离(已知)
  h(n) = 从n到终点的估计距离(启发函数)

关键要求: h(n)必须是"可接受的"(admissible) — 不高估真实距离
  • 曼哈顿距离: h = |x₁-x₂| + |y₁-y₂|  (网格地图)
  • 欧几里得距离: h = √((x₁-x₂)² + (y₁-y₂)²)  (连续空间)
  • h=0: 退化为Dijkstra

为什么A*更快:
  Dijkstra: 均匀地向所有方向扩展 (无信息)
  A*:       朝着目标方向集中搜索 (有h引导)
  在大规模地图上，A*可能比Dijkstra快10-100倍

Dijkstra算法详解

图: A→B(1), A→C(4), B→C(2), B→D(6), C→D(3)

初始化: dist[A]=0, 其他=∞
优先队列: [(0,A)]

Step 1: 弹出(0,A)
  更新B: 0+1=1 < ∞ → dist[B]=1
  更新C: 0+4=4 < ∞ → dist[C]=4
  队列: [(1,B),(4,C)]

Step 2: 弹出(1,B)
  更新C: 1+2=3 < 4 → dist[C]=3
  更新D: 1+6=7 < ∞ → dist[D]=7
  队列: [(3,C),(7,D)]

Step 3: 弹出(3,C)
  更新D: 3+3=6 < 7 → dist[D]=6
  队列: [(6,D)]

结果: dist[A]=0, dist[B]=1, dist[C]=3, dist[D]=6

1.4 拓扑排序 (Topological Sort)

对有向无环图(DAG)的顶点进行线性排序，使得每条边(u,v)中u排在v前面。

算法1: DFS后序的逆序
  对每个未访问节点做DFS, 记录退出顺序, 反转即为拓扑序

算法2: Kahn's Algorithm (BFS)
  1. 计算所有节点入度
  2. 将入度=0的节点入队
  3. 弹出节点u, 对其邻居v: 入度[v]--, 若入度[v]=0则入队
  4. 若输出节点数 < 总节点数 → 图有环(不是DAG!)

应用:
  • 任务调度: 有依赖关系的任务执行顺序
  • 编译顺序: 模块依赖的编译顺序
  • 课程规划: 先修课约束下的选课顺序
  • Makefile/CMake: 构建系统的依赖解析

1.5 强连通分量 (SCC)

有向图中，最大的子图使得任意两点互相可达。

Tarjan算法: O(V+E)
  用DFS + 栈 + low-link值, 一次DFS找出所有SCC

Kosaraju算法: O(V+E)
  1. 对原图做DFS, 记录退出顺序
  2. 对反向图按退出顺序逆序做DFS
  3. 每次DFS访问到的节点构成一个SCC

应用:
  • 2-SAT求解: 构建蕴含图, 若x和¬x在同一SCC → 不可满足
  • 社交网络: 发现紧密社群
  • 编译器: 函数调用图的循环依赖分析
  • Web分析: 网页链接的社区结构

2. 最小生成树 (MST)

常见误区： 很多人在学习经典优化方法时容易陷入"只看不练"的误区，以为理解了概念就能在实际项目中应用。实际上，理论知识必须配合动手实验才能真正掌握。建议每学完一个知识点就写一段验证代码，在实践中发现和解决问题。
深入探讨：Kruskal算法使用并查集，按边权排序添加边，时间复杂度O(E log E)。Prim算法从单个节点扩展，使用优先队列，O((V+E)log V)。MST的边数为V-1。
A ---1--- B
|         |
2         3
|         |
C ---4--- D
MST边：A-B, A-C, B-D
算法策略时间复杂度
Kruskal边排序，添加无环边O(E log E)
Prim从节点扩展树O((V+E)log V)
Borůvka并行分治O(E log V)

连通图中总边权最小的树。P问题，O(E log V)。

Kruskal算法

按边权从小到大排序，依次加入不形成环的边。

核心：并查集(Union-Find)判环。

Prim算法

从一个节点出发，每次加入连接树内和树外的最小边。

类似Dijkstra的贪心扩展。

2.1 Kruskal算法详解

图: 边集 {(A,B,1),(A,C,4),(B,C,2),(B,D,6),(C,D,3),(C,E,5)}

步骤1: 按边权排序
  (A,B,1) < (B,C,2) < (C,D,3) < (A,C,4) < (C,E,5) < (B,D,6)

步骤2: 依次加入不形成环的边 (用并查集)
  加(A,B,1): A-B  ✓ 不成环
  加(B,C,2): A-B-C ✓ 不成环
  加(C,D,3): A-B-C-D ✓ 不成环
  加(A,C,4): A-C → A和C已连通 → 跳过! (会形成环)
  加(C,E,5): A-B-C-D, E ✓ 不成环

结果: MST = {(A,B,1),(B,C,2),(C,D,3),(C,E,5)}, 总权=11

并查集(Union-Find): O(α(n)) ≈ O(1) 每次操作
总复杂度: O(E log E) (排序) + O(E·α(V)) (并查集) = O(E log E)

2.2 Prim算法详解

同样的图，从A出发:

初始: MST={A}, 候选边: {A-B(1), A-C(4)}
Step 1: 选最小边 A-B(1), MST={A,B}
        候选边: {A-C(4), B-C(2), B-D(6)}
Step 2: 选最小边 B-C(2), MST={A,B,C}
        候选边: {A-C(4)←跳过(C已在), B-D(6), C-D(3), C-E(5)}
Step 3: 选最小边 C-D(3), MST={A,B,C,D}
        候选边: {B-D(6)←跳过, C-E(5)}
Step 4: 选最小边 C-E(5), MST={A,B,C,D,E}

结果: MST总权 = 1+2+3+5 = 11 ✓ (与Kruskal相同)

复杂度: O(E log V) (用优先队列)
Kruskal vs Prim: 边少用Kruskal, 边密集用Prim

3. 最大流与最小割

常见误区： 很多人在学习经典优化方法时容易陷入"只看不练"的误区，以为理解了概念就能在实际项目中应用。实际上，理论知识必须配合动手实验才能真正掌握。建议每学完一个知识点就写一段验证代码，在实践中发现和解决问题。
深入探讨：最大流最小割定理：最大流值等于最小割容量。Ford-Fulkerson算法使用增广路径，时间复杂度O(E * max_flow)。Edmonds-Karp用BFS找最短增广路径，O(VE^2)。
S --10--> A --5--> T
|         |
10        10
v         v
B ------15--> T
最大流：15
算法方法时间复杂度
Ford-Fulkerson增广路径O(E * max_flow)
Edmonds-KarpBFS增广O(VE^2)
Dinic阻塞流O(V^2 E)

Ford-Fulkerson方法

反复在残余图中找增广路径，沿路径增加流量。
找不到增广路径时停止 → 当前流即为最大流。

增广路径用BFS(Edmonds-Karp) → O(VE²)
增广路径用DFS → 可能不终止(无理数容量)

最大流的对偶：最小割

将节点分为S侧和T侧，割的容量 = 从S侧到T侧的边容量之和。最大流=最小割定理是网络优化的核心。

E2 --T2
E3 --T3

匈牙利算法

在二部图中找最大基数匹配。O(V³)。

二部图: 左{A,B,C} 右{1,2,3}
边: A-1, A-2, B-2, B-3, C-1

匈牙利算法:
1. 初始匹配: A-1
2. 尝试B: B-2 (无冲突) → 匹配{A-1, B-2}
3. 尝试C: C-1 (与A冲突!) → 寻找增广路
   C-1-A-2-B-3 (增广路) → 翻转得{C-1, A-2, B-3}
4. 最大匹配: 3 ✓

5. 最小费用流 (Min-Cost Flow)

常见误区： 很多人在学习经典优化方法时容易陷入"只看不练"的误区，以为理解了概念就能在实际项目中应用。实际上，理论知识必须配合动手实验才能真正掌握。建议每学完一个知识点就写一段验证代码，在实践中发现和解决问题。
深入探讨：最小费用流问题：在满足供需和容量约束下，最小化总费用。常用算法：网络单纯形法或使用增广路径的费用流算法。时间复杂度通常为O(V^2 E)或更高。
S --(cost 2, cap 10)--> A --(cost 1, cap 5)--> T
最小费用流：发送5单位，费用2*5+1*5=15
方法描述效率
网络单纯形法基于线性规划迭代通常高效
增广路径法逐步增广最小费用路径易实现
费用缩放近似优化O(V^2 E log V)

在满足供需约束的前提下，以最小总费用运送指定流量。

min Σ cost(i,j) · f(i,j)    (最小化运输费用) s.t. Σ f(i,j) - Σ f(j,i) = b(i)    (流量守恒: 每个节点有净供给/需求b(i)) 0 ≤ f(i,j) ≤ cap(i,j)    (容量约束) 
例子: 工厂→仓库→零售店的运输网络

工厂A(供给100) → 仓库X(容量50, 费用2) → 零售店P(需求60)
                → 仓库Y(容量80, 费用3) → 零售店P
工厂B(供给60)  → 仓库X(容量30, 费用1) → 零售店Q(需求100)
                → 仓库Y(容量40, 费用4) → 零售店Q

最小费用流 = 在满足P需60、Q需100的前提下，最小化总运输费用

特殊形式:
• 运输问题: 二部图(工厂→客户)，无中间节点 → 网络单纯形法
• 指派问题: 运输问题的特例(供需都=1) → 匈牙利算法
• 最大流: 所有边费用=0, 求最大流量

6. 网络单纯形法

常见误区： 很多人在学习经典优化方法时容易陷入"只看不练"的误区，以为理解了概念就能在实际项目中应用。实际上，理论知识必须配合动手实验才能真正掌握。建议每学完一个知识点就写一段验证代码，在实践中发现和解决问题。
深入探讨：网络单纯形法是线性规划单纯形法在流网络上的特化，使用生成树结构。每次迭代通过改变树边优化，平均时间复杂度O(V^2 E)，实践中高效。
Iteration 1: Current flow
Iteration 2: Adjust edge
Iteration 3: Optimal flow
方面网络单纯形法传统单纯形法
应用流网络问题一般线性规划
数据结构生成树矩阵
效率更高效于网络问题通用但较慢

LP单纯形法在网络结构上的特化。运输问题、最小费用流的最高效算法。

标准单纯形法: 每步维护一个基矩阵B, 求解Bx=b → O(m³)
网络单纯形法: 基矩阵对应一棵生成树 → 利用树结构 O(m)

核心操作:
1. 维护一棵"生成树"作为当前基可行解
2. 选择非树边入基(负reduced cost)
3. 找到环路, 确定出基边
4. 沿环路调整流量

为什么快:
• 网络矩阵是全幺模的(TU) → LP松弛自动得整数解
• 树结构的操作比矩阵求逆快得多
• 适合大规模稀疏网络(百万节点)

应用：运输问题、生产调度、库存管理、现金流优化。OR-Tools和Gurobi内置网络单纯形求解器。

    ① 取S(0)：更新A=4, B=2
    ② 取B(2)：更新A=min(4,3)=3, C=7
    ③ 取A(3)：更新C=min(7,6)=6, D=9
    ④ 取C(6)：更新D=min(9,8)=8
    最短路径：S→B→A→C→D，距离=8。注意S→A直接走(4)不如S→B→A(3)短。
    关联知识: 与Bellman-Ford（负权边）、Floyd-Warshall（全源最短路）、A*搜索（启发式）
  2. 核心原理: Kruskal算法求解最小生成树（MST），核心思想是贪心+并查集：① 将所有边按权重排序；② 从小到大依次考虑每条边；③ 如果边的两个端点不在同一连通分量（用并查集判断），则加入MST；④ 重复直到MST有V-1条边。时间O(ElogE)。
    关键要点: ① 贪心：先选最短边 ② 并查集判环 ③ 时间O(ElogE) ④ 适用于稀疏图
    实际案例: 6条边无向图：(A,B,4),(A,C,2),(B,C,1),(B,D,5),(C,D,8),(D,E,2)。
    排序：(B,C,1),(A,C,2),(D,E,2),(A,B,4),(B,D,5),(C,D,8)
    ① 选(B,C,1) ✓ ② 选(A,C,2) ✓ ③ 选(D,E,2) ✓
    ④ 选(A,B,4)：A-B已连通，跳过 ✗
    ⑤ 选(B,D,5) ✓（连接两个分量）
    MST = {(B,C,1),(A,C,2),(D,E,2),(B,D,5)}，总权重=10。
    关联知识: 与Prim算法（稠密图更优）、网络设计应用
  3. 核心原理: 最大流-最小割定理：从源点s到汇点t的最大流量等于最小割的容量。求解方法：① 在残量图中找s→t的增广路径；② 沿路径增加流量（路径上最小剩余容量）；③ 更新残量图；④ 重复直到无增广路。时间O(VE²)（Edmonds-Karp）。
    关键要点: ① 最大流=最小割（对偶关系） ② 增广路方法求解 ③ 残量图是核心数据结构 ④ 广泛应用于网络流、匹配
    实际案例: NetworkX求解示例代码：
    import networkx as nx
    G = nx.DiGraph()
    edges = [("s","a",10),("s","b",8),("a","b",5),("a","c",7),
    ("b","c",10),("b","d",6),("c","t",10),("d","t",8)]
    G.add_weighted_edges_from([(u,v,{"capacity":w}) for u,v,w in edges])
    flow_value, _ = nx.maximum_flow(G, "s", "t")
    cut_value, _ = nx.minimum_cut(G, "s", "t")
    print(f"最大流={flow_value}, 最小割={cut_value}") # 均为15
    最大流=最小割=15，验证了Ford-Fulkerson定理。
    关联知识: 与二部图匹配、多商品流、网络可靠性分析
