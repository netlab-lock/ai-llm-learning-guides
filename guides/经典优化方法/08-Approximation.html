八、近似算法

NP-hard问题没有已知的多项式时间精确算法。近似算法在多项式时间内给出有可证明保证的近似解：算法解 ≤ ρ × 最优解（ρ为近似比）。本篇按问题分类，详解每个经典近似算法的思路、证明、实现和应用。

1. 近似算法基础理论

通俗类比：考试目标不是满分100分，而是‘至少得90分’。近似比α就是那个‘90分’的下限保证，保证算法在最坏情况下的表现底线。
常见误区：混淆‘对所有实例成立’和‘对某个实例成立’。近似比是对问题所有可能输入的最坏情况比值保证，而非平均或特定情况。
深入探讨：近似比 α(n) 是问题规模 n 的函数。若 α 是常数，则称为常数因子近似算法。PTAS (多项式时间近似方案) 可以达到任意近似比 (1+ε)，但时间可能是 n^{f(1/ε)}。
近似比 α\n最大值  |  (最大化问题)\n    /\n   / \n  /   \n /     \nOPT----->APX
概念定义例子

PTAS对任意ε>0，有(1+ε)近似算法欧几里得TSP
FPTAS时间关于1/ε和n都是多项式的PTAS背包问题

1.1 近似比的定义

ρ-近似算法: 对于最小化问题, 算法解 A(I) ≤ ρ · OPT(I), ρ ≥ 1
对于最大化问题, 算法解 A(I) ≥ (1/ρ) · OPT(I), 或 A(I) ≥ ρ · OPT(I), ρ ≤ 1

ρ越接近1越好。ρ=1意味着精确算法。

1.2 近似类 (Approximation Classes)

类含义典型问题
PTAS对任意ε>0存在(1+ε)-近似算法，但时间可能随ε指数增长欧氏TSP、背包
FPTAS完全多项式时间近似方案——时间关于1/ε也是多项式背包、子集和
APX存在常数近似比的多项式算法顶点覆盖(2)、集合覆盖(ln n)
不可近似除非P=NP，不存在常数近似比一般图着色、TSP(无三角不等式)

1.3 不可近似性结果

基于PCP定理(1992, Arora-Safra, Arora-Lund-Motwani)的不可近似性:

• 顶点覆盖: 不可近似到 2-ε (除非P=NP)  ← 2-近似算法恰好最优!
• 集合覆盖: 不可近似到 (1-ε)ln n (除非NP⊆DTIME(n^log log n))
• TSP(无三角不等式): 不可近似到任何常数
• MAX-3SAT: 不可近似到 7/8+ε
• 图着色: 不可近似到 n^{1-ε}

这些结果告诉我们: 近似算法的近似比已经接近理论极限

2. 顶点覆盖 (Vertex Cover)

通俗类比：想象公司需要雇佣保安覆盖所有走廊。最优解是精打细算用最少人，2-近似解是‘每个走廊两端各派一人’，虽然可能有人重叠，但保证不超过最优人数的2倍。
常见误区：贪心算法（每次选度数最高的边）可能找不到常数因子近似解。其性能可能差到O(log n)倍，这不是一个好的近似算法保证。
深入探讨：经典2-近似算法：1) 计算一个极大匹配M（没有边可以再加入）。2) 将M中所有边的两端点都选入覆盖集。证明：任何顶点覆盖必须至少包含|M|个顶点来覆盖M的每条边，而我们选了2|M|个。
   1 --- 2     极大匹配M：{(1,2), (3,4)}\n   |     |     选入覆盖集：{1,2,3,4}\n   3 --- 4     顶点数 |C| = 2|M|

极大匹配法2O(V+E)
线性规划舍入2多项式时间
贪心(度数)O(log n)O(V^2)

2.1 问题

给定图G=(V,E)，找最小的顶点集合S使得每条边至少有一个端点在S中。

2.2 经典2-近似算法

算法: 近似顶点覆盖 (Approx-Vertex-Cover)
1. C = ∅
2. 当E非空时:
   a. 任取一条边 (u,v) ∈ E
   b. C = C ∪ {u, v}     ← 把两个端点都加入覆盖
   c. 删除与u或v关联的所有边
3. 返回 C

示例:
  图: A---B---C---D
        \  |  /
         \ | /
          \|/
           E

  取边(A,B): C={A,B}, 删除关联边
  取边(C,D): C={A,B,C,D}, 删除关联边
  取边(B,E): C={A,B,C,D,E}, 删除关联边
  所有边被覆盖, 返回C={A,B,C,D,E}
  
  最优解: {B,C} (大小2), 算法返回大小5
  近似比: 5/2 = 2.5 ≤ 2 (在最坏情况下)

2.3 近似比证明

证明: |C| ≤ 2·OPT

设算法选了k条边 e₁,e₂,...,eₖ（它们两两不相邻）
• 每条边贡献2个顶点给C → |C| = 2k
• 这k条边互不相邻 → 任何顶点覆盖必须包含每条边的至少一个端点
  → OPT ≥ k
• 因此 |C| = 2k ≤ 2·OPT  □

注意: 这是紧的! 考虑完全二部图 K_{n,n}:
  算法可能选n条匹配边 → C=2n
  OPT=n (只需选一侧)
  近似比恰好=2

2.4 整数规划视角

IP: min Σ xᵥ  s.t.  xᵤ + xᵥ ≥ 1  ∀(u,v)∈E,  xᵥ ∈ {0,1}

LP松弛: 放宽 xᵥ ∈ {0,1} → xᵥ ∈ [0,1]
LP最优解 x*: 每条边约束 xᵤ*+xᵥ* ≥ 1
→ 取 {v : xᵥ* ≥ 1/2} 构成顶点覆盖
→ 大小 ≤ 2·Σxᵥ* = 2·LP* ≤ 2·OPT

这也是2-近似! LP对偶提供了一个下界。
💡 应用场景： • 网络安全：最少监控点覆盖所有链路 • 设施监控：最少摄像头覆盖所有路段 • 漏洞修复：修复最少组件消除所有攻击路径 • 生物信息：最少标记覆盖所有基因交互

3. 集合覆盖 (Set Cover)

通俗类比：给全家手机充电，你有多个插排，每个插排有几个孔。目标是选最少的插排让所有手机都充上电。贪心法就是每次选‘能充最多没充上手机’的那个插排。
常见误区：贪心算法 (每次选覆盖最多未覆盖元素的集合) 是一个O(log n)近似算法，但不是常数因子近似。对于某些实例，它选的集合数量可以是最佳解的log n倍。
深入探讨：贪心算法近似比为 H(d)，其中 d 是最大集合大小，H(d) 是第 d 个调和数，H(d) ≈ ln(d) + 0.577。当 d 很大时，H(d) ≈ ln(d)。线性规划舍入可得 O(ln n) 近似。
全集 U: {a,b,c,d,e,f}\n集合 S1: {a,b,c}  S2: {b,d,e}  S3: {c,f}\n贪心选 S1 → 未覆盖{d,e,f}\n再选 S2 → 未覆盖{f}，选 S3。共3个。
算法近似比关键思想
贪心法H(n) ≈ ln n每轮选覆盖比最高的集合
线性规划舍入O(ln n)分数解舍入到整数解
随机法O(ln n)以某种概率随机选取集合

3.1 问题

给定全集U={1,...,n}和子集族S={S₁,...,Sₘ}，每个子集有代价c(Sⱼ)。选择代价最小的子集族使其并集=U。

3.2 贪心算法 (O(ln n)-近似)

算法: Greedy-Set-Cover
1. C = ∅ (已选子集), U' = U (未覆盖元素)
2. 当U'非空时:
   a. 选择性价比最高的子集: Sⱼ = argmax |Sⱼ∩U'| / c(Sⱼ)
   b. C = C ∪ {Sⱼ}
   c. U' = U' \ Sⱼ
3. 返回 C

示例:
  U = {1,2,3,4,5,6,7,8,9}
  S₁={1,2,3,6} c=2  → 性价比 4/2=2
  S₂={4,5,6,7} c=3  → 性价比 4/3=1.33
  S₃={1,4,8,9} c=4  → 性价比 4/4=1
  S₄={2,5,7,8} c=2  → 性价比 4/2=2

  Step1: 选S₁或S₄(性价比2), 假设选S₁: 覆盖{1,2,3,6}
  Step2: 未覆盖{4,5,7,8,9}
         S₂覆盖{4,5,7}性价比3/3=1, S₃覆盖{4,8,9}性价比3/4=0.75
         选S₂: 覆盖{4,5,6,7}
  Step3: 未覆盖{8,9}, 选S₃: 覆盖{1,4,8,9}
  总代价: 2+3+4=9

3.3 近似比证明

定理: Greedy-Set-Cover 的代价 ≤ H(max|Sⱼ|) · OPT
其中 H(n) = 1 + 1/2 + 1/3 + ... + 1/n ≈ ln n

证明思路:
• 对每个元素e定义"价格"price(e) = e被覆盖时所选子集的代价/该子集新覆盖的元素数
• 每个子集的代价 = 它覆盖的元素的价格之和
• 对于最优解中的子集Sⱼ*: 它的每个元素的价格 ≤ c(Sⱼ*)/|Sⱼ*| (贪心选择性)
• 因此总代价 ≤ Σe∈U price(e) ≤ ΣSⱼ*∈OPT |Sⱼ*|·c(Sⱼ*)/|Sⱼ*| = OPT... (需精细分析)

结果: H(n) ≈ ln n, 对于n=1000, ln n ≈ 7
→ 贪心解最多是最优解的7倍
💡 应用场景： • 无线基站选址：最少基站覆盖所有用户 • 传感器部署：最少传感器覆盖所有区域 • 广告投放：最少渠道覆盖所有目标人群 • 生物探针：最少探针覆盖所有基因 • 物流中心：最少仓库覆盖所有客户

4. 度量TSP (Metric TSP)

通俗类比：快递员送完所有包裹再回仓库，要求总路程最短。‘三角不等式’就像实际道路：从A到B，直接去总比绕道C更快或相等。Christofides算法能保证路线不超过最优解的1.5倍。
常见误区：必须满足三角不等式(d(i,k) ≤ d(i,j) + d(j,k))才能保证常数近似比。一般的TSP问题（无三角不等式）无法获得常数因子近似，除非P=NP。
深入探讨：Christofides算法：1) 构建最小生成树T。2) 找出T中度数为奇数的顶点集合O。3) 在O上计算最小权完美匹配M。4) 将T和M的边合并，得到欧拉图，找欧拉回路。5) 将回路‘短路’成哈密顿回路。近似比1.5。
1) MST:     2) 奇数度点 O = {1,3}\n 1---2       3) 匹配 M: (1,3)\n |            4) T + M 形成欧拉图\n 3            5) 欧拉回路 → 哈密顿回路
算法近似比前提条件

Christofides1.5三角不等式

4.1 问题

TSP + 三角不等式: d(i,j) ≤ d(i,k) + d(k,j)。这是TSP中最有实际意义的子类。

4.2 MST启发式 (2-近似)

算法: MST-TSP
1. 计算最小生成树T
2. 对T做DFS(深度优先遍历), 记录访问顺序
3. 按访问顺序跳过已访问的节点, 形成回路

为什么是2-近似?
• MST的总权 ≤ OPT (去掉TSP回路的任意一条边得到一棵生成树)
• DFS遍历每条边走两次 → 回路长度 ≤ 2·MST ≤ 2·OPT

示例:
  MST:    A---B     DFS遍历: A→B→C→B→A→D→A
              |     跳过重复: A→B→C→D→A
          C---D     回路长度 ≤ 2·MST总权

4.3 Christofides算法 (3/2-近似)

算法: Christofides (1976)
1. 计算最小生成树T
2. 找T中所有奇数度顶点的集合O (偶数个)
3. 在O上计算最小权完美匹配M
4. 构造欧拉图: T ∪ M (合并树和匹配)
5. 在欧拉图中找欧拉回路
6. 跳过已访问节点, 形成哈密顿回路

近似比证明:
• MST ≤ OPT
• 奇数度顶点的最小匹配M ≤ OPT/2 (利用三角不等式和TSP回路的性质)
• 总权 ≤ MST + M ≤ OPT + OPT/2 = 3/2·OPT  □

这是40多年来度量TSP最好的近似算法!
2020年Karlin-Klein-Oveis Gharan取得微小改进: 3/2 - 10^{-36}
💡 应用场景： 所有满足三角不等式的TSP场景（实际中几乎所有距离度量都满足）： 物流配送、电路板钻孔、仓库拣货、DNA片段组装

5. 背包问题 (Knapsack)

通俗类比：超市大促销，你有一个购物车容量限制，想装总价值最高的商品。‘价值密度’(价值/重量)高的商品就像明星商品，是优先考虑的，但必须留空间给其他‘高性价比’的组合。
常见误区：纯贪心法（只按价值密度排序）可能非常差，因为可能忽略一个重量大、价值极高但密度不高的‘大件’。完全贪心甚至没有常数近似比保证。
深入探讨：FPTAS (完全多项式时间近似方案)：1) 设定一个缩放因子 K = ε * (max p_i) / n。2) 将每个价值 p_i 替换为 p_i' = floor(p_i / K)。3) 对缩放后的问题运行动态规划。时间复杂度 O(n^3/ε)，得到 (1-ε) 近似解。
容量 W=10\n物品 (重量,价值): A(6,60), B(4,40), C(1,10)\n价值密度: A=10, B=10, C=10\n最优解: B+C (价值50)\n贪心(只按价值): A (价值60) 但超重!
算法近似比时间复杂度
贪心(价值密度)无常数保证O(n log n)
贪心+最优物品0.5 (最小值)O(n)

5.1 FPTAS (完全多项式时间近似方案)

问题: n个物品, 重量w[i], 价值v[i], 容量W. 最大化总价值.
FPTAS思路: 对价值做"舍入", 使DP表变小

算法:
1. 设 v_max = max(v[i])
2. 设 K = ε·v_max/n  (舍入精度)
3. 对每个物品, 令 v'[i] = ⌊v[i]/K⌋  (舍入后的价值)
4. 用DP求解舍入后的背包问题: O(n²·v_max/K) = O(n³/ε)

时间: O(n³/ε) — 关于1/ε是多项式!
近似比: (1-ε) — 可以任意接近最优

关键洞察: 舍入造成的总误差 ≤ n·K = ε·v_max ≤ ε·OPT

5.2 贪心近似 (PTAS)

贪心算法: 按性价比 v[i]/w[i] 降序排列, 依次装入
近似比: 1/2 (即 ≥ OPT/2)

证明: 设贪心解G, 最优解OPT
  • 如果最优解中最贵的物品 ≤ G → G=OPT
  • 否则: 最优解去掉最贵物品后 ≤ G (因为贪心按性价比排)
  → OPT ≤ 最贵物品 + G ≤ 2G → G ≥ OPT/2

改进: 选择贪心解和单个最贵物品中更好的
  → 近似比提升到 ≥ OPT/2 (更严格的分析可得更好界)
💡 应用场景： 投资组合选择、资源分配、货物装载、预算分配、内存管理

6. MAX-SAT 与布尔可满足性

通俗类比：设计一个电路，有多个开关，每个‘门’(子句)需要特定的开关组合才能让灯亮。MAX-SAT就是调整开关状态，尽可能让更多的灯亮。
常见误区：不要只关注‘找到一个满足所有子句的赋值’（SAT问题）。MAX-SAT是关于‘最大化满足子句的比例’，它本身是NP难的，但存在非常好的近似算法。
深入探讨：对于 MAX-k-SAT（每个子句最多k个文字），随机赋值算法能满足至少 (1 - 1/2^k) 比例的子句的期望值。例如，MAX-3-SAT 期望满足 7/8 的子句。可以通过条件期望去随机化得到确定性 (1 - 1/2^k) 近似算法。
子句1: (x1 ∨ ¬x2)   子句2: (¬x1 ∨ x3)\n随机赋值: x1=T, x2=F, x3=T\n满足: 子句1 (T∨T=T), 子句2 (F∨T=T)\n满足率: 2/2 = 100% > 期望的75%
问题变种最佳已知近似比算法核心

6.1 问题

给定CNF公式，找变量赋值使得满足的子句数最多。

6.2 随机化算法 (1/2-近似)

算法: Random-Assignment
  对每个变量, 独立地以1/2概率设为True

分析: 每个子句(k个文字)被满足的概率 = 1 - (1/2)^k ≥ 1/2
  → 期望满足的子句数 ≥ m/2 ≥ OPT/2
  → 至少1/2近似

改进 (条件期望法): 逐个确定变量, 每次选使期望更大的值
  → 确定性1/2-近似

6.3 Goemans-Williamson算法 (7/8-近似, MAX-3SAT)

算法: SDP-based (1994)
1. 将MAX-3SAT编码为半正定规划(SDP)
2. 求解SDP得到单位向量解
3. 用随机超平面将向量映射为布尔值

近似比: 7/8 = 0.875 (对于MAX-3SAT)
  证明依赖于SDP松弛的几何性质和随机超平面的分析

这是第一个用SDP得到的近似算法, 开创了"SDP rounding"范式
不可近似性: 除非P=NP, 不可近似到 7/8+ε (Håstad, 2001)
→ Goemans-Williamson恰好达到理论极限!
💡 应用场景： • MAX-SAT: 硬件验证中"尽可能多满足"规格 • MAX-kSAT: 配置问题中最大化满意度 • 电路优化: 最大化满足的时序约束

7. 图着色 (Graph Coloring)

通俗类比：给地图上的国家着色，相邻国家不能同色。就像安排会议，有冲突的会议不能在同一时间。目标是用最少的时间段（颜色）开完所有会。
常见误区：贪心着色算法（按某种顺序，给每个节点分配最小可用颜色）的效果严重依赖节点顺序，并且没有常数近似比保证。对于某些图，它可能用掉远超过色数的颜色。

7.2 贪心着色

算法: 按某种顺序遍历顶点, 每个顶点用最小可用颜色
近似比: 一般图无常数保证, 但对于:
  • 二部图: 精确(2色)

  • 区间图: 精确
  • 弦图: 精确

  每次选"饱和度最高"(邻居颜色数最多)的顶点着色
  对很多实际图类可以找到最优着色

7.3 SDP松弛 (色数的近似)

Lovász theta函数 ϑ(G):
  • χ(G) ≥ ϑ(G) ≥ α(G)  (色数的下界 = 独立数的上界)
  • 可用SDP在多项式时间计算
  • 对于完美图: ϑ(G) = χ(G) (精确!)

应用: 频率分配、寄存器分配、考试排期

8. 机器调度近似算法

通俗类比：多个工人同时做一批任务，目标是尽快完成所有任务（最小化最大完工时间）。就像安排宿舍值日，把打扫任务分配给室友，让最后一个干完活的人早点休息。
常见误区：贪心法（按任务到达顺序，将每个任务分配给当前负载最小的机器）称为 List Scheduling，其近似比为 (2 - 1/m)，其中 m 是机器数。但它不是最优的，特别是当任务长度差异很大时。
深入探讨：LPT (最长处理时间优先) 算法：1) 将任务按处理时间降序排列。2) 按此顺序执行 List Scheduling。近似比为 (4/3 - 1/(3m))。当 m=2 时，近似比为 5/4 = 1.25。

8.1 P||Cmax (并行机最小化makespan)

LPT算法 (Longest Processing Time first):
1. 将作业按处理时间降序排列
2. 依次将每个作业分配给当前负载最小的机器

近似比: 4/3 - 1/(3m) (m为机器数)

示例: 3台机器, 作业=[8,7,6,5,4,3,2]
  排序: [8,7,6,5,4,3,2]
  8→机器1(8), 7→机器2(7), 6→机器3(6)
  5→机器3(6+5=11), 4→机器2(7+4=11), 3→机器1(8+3=11), 2→机器1(11+2=13)
  makespan=13, OPT=12, 近似比=13/12≈1.08 ✓

8.2 PTAS for P||Cmax

Hochbaum-Shmoys PTAS (1987):
1. 猜测最优makespan T (二分搜索)
2. 将作业分为"大"(pⱼ > εT)和"小"(pⱼ ≤ εT)
3. 大作业: 用First-Fit (每个机器最多1/ε个大作业)
4. 小作业: 贪心分配
5. 如果成功 → T是可行的makespan

时间: O(n log n) 对固定ε
近似比: 1+ε
💡 应用场景： 并行计算任务分配、云服务器负载均衡、工厂多机排产

10. 近似算法设计范式

近似算法设计范式

通俗类比：设计近似算法就像解数学题，有几种常用‘解题套路’：贪心策略、线性规划舍入、原始对偶法、局部搜索、度量方法等。
常见误区：不要认为一种范式能解决所有问题。贪心法对某些问题很好（如集合覆盖），但对另一些问题很差（如背包）。需要根据问题的组合结构选择合适的范式。
深入探讨：常见范式：1) 贪心法：每步做局部最优决策。2) 线性规划舍入：将组合问题松弛为LP，求解分数解，再舍入为整数解。3) 原始对偶法：同时构建原始解和对偶解，保证互补松弛条件。4) 局部搜索：从一个解开始，不断用小改进来优化。
近似算法设计范式\n├── 贪心法 (集合覆盖, 顶点覆盖)\n├── LP舍入 (顶点覆盖, 机时调度)\n├── 原始对偶 (网络设计, Steiner树)\n└── 局部搜索 (度量TSP, k-中值)

  2. 实现集合覆盖的贪心算法，验证近似比是否≤ln(n)
  3. 实现Christofides算法的前3步（MST+奇度匹配），分析近似比

▶显示答案
✅ 参考答案与详解
  1. 核心原理: 顶点覆盖2-近似算法的核心是"极大匹配法"：(1)计算图G的一个极大匹配M（无法再加入更多不相邻的边）；(2)取M中所有边的两个端点构成覆盖集C。证明|C|≤2·OPT：M中边两两不相邻→任何顶点覆盖必须包含每条边至少一个端点→OPT≥|M|，而|C|=2|M|，所以|C|≤2·OPT。测试方法：生成随机图(如Erdős-Rényi G(n,p))，运行2-近似算法得到C，同时用ILP求精确最优OPT，计算|C|/OPT。
    关键要点: 1. 近似比2是紧的——完全二部图K_{n,n}上恰好达到2倍；2. 算法时间复杂度O(V+E)，非常高效；3. LP松弛+阈值舍入也给出2-近似：LP最优≤OPT，取{xᵥ≥1/2}得到|C|≤2·LP≤2·OPT；4. PCP定理证明不可近似到2-ε(除非P=NP)；5. 随机图上实际近似比通常在1.2~1.5之间。
    实际案例: 网络安全中最小监控点问题：100个网络节点，500条链路。2-近似算法选出约35个监控节点，精确最优为28个，比值1.25。在完全二部图上比值可达2.0。
    关联知识: 顶点覆盖是NP-complete(模块七·§2.4)；与独立集互补（顶点覆盖的补集是独立集）；集合覆盖的贪心近似比为O(ln n)(§3)。
  2. 核心原理: 集合覆盖贪心算法的近似比验证需要系统实验。算法：每次选"新覆盖元素数/代价"最大的集合，直到全覆盖。近似比H(n)=1+1/2+...+1/n≈ln(n)。验证方法：(1)构造不同规模实例(n=50,100,500)；(2)运行贪心算法得到解大小S_greedy；(3)用Gurobi求精确最优S_opt；(4)计算比值S_greedy/S_opt，绘制随n变化的趋势图。理论上比值应≤ln(n)，实际通常远小于此上界。
    关键要点: 1. H(n)是第n个调和数，增长极慢——n=1000时H≈7.5；2. 贪心算法的近似比是紧的(Chvátal, 1979)——存在构造性实例使比值达到H(n)；3. "性价比"标准(覆盖数/代价)优于"覆盖数最多"标准；4. 当代价相同时等价于"每次选最大的未覆盖集合"；5. 实验应多次随机运行取平均值。
    实际案例: 广告投放优化：100万用户需被覆盖，500个广告渠道各有不同覆盖面和成本。贪心算法约需ln(1000000)≈14倍最优解，实际测试中约需2~3倍最优解。
    关联知识: 集合覆盖的不可近似性基于PCP定理(模块七·Cook-Levin)——不可近似到(1-ε)ln n；顶点覆盖是特例有2-近似(§2)；FPTAS(模块八·§5)适用于背包但不适用于覆盖问题。
  3. 核心原理: Christofides算法求解度量TSP(满足三角不等式)的1.5-近似算法。前3步：(1)计算最小生成树T(MST)——时间O(n²log n)；(2)找出T中度数为奇数的顶点集合O(必有偶数个)——时间O(n)；(3)在O上计算最小权完美匹配M——时间O(n³)。后续步骤：(4)合并T∪M形成欧拉图；(5)找欧拉回路；(6)短路重复顶点成哈密顿回路。近似比证明：MST≤OPT(去掉TSP回路任意一边得生成树)，匹配M≤OPT/2(利用三角不等式和奇度顶点性质)，总权≤MST+M≤OPT+OPT/2=1.5·OPT。
    关键要点: 1. 前3步已构成算法的核心——MST+奇度匹配是最关键的组合；2. 奇度顶点个数必为偶数(图论基本定理)；3. 最小权完美匹配可用Blossom算法(Edmonds, 1965)O(n³)求解；4. 40年来(1976-2020)无人改进1.5近似比，直到2020年Karlin等取得3/2-10^{-36}的微小改进；5. 实际中LKH启发式效果远好于Christofides但无理论保证。
    实际案例: 物流配送20个地点：MST总距离150km，最优TSP回路约180km，Christofides给出≤270km的回路(实际约195km)。LKH启发式给出182km(更接近最优)。
    关联知识: TSP是NP-hard(模块七·§3.1)；MST可由Kruskal/Prim算法求解(模块四·图论)；匹配问题(模块四·匈牙利算法)是关键子问题。
  4. 核心原理: MAX-3SAT可达到7/8近似的根本原因是每个子句的结构允许"概率方法"的巧妙应用。对3-SAT的每个子句(含3个文字)，随机赋值使其不满足的概率为(1/2)³=1/8，即满足概率为7/8。通过条件期望去随机化可得到确定性7/8近似算法。更深层的SDP方法(Goemans-Williamson)也达到7/8且证明了匹配的不可近似性下界(除非P=NP不可改进到7/8+ε)。而一般图着色不能达到常数近似是因为：图着色的难度来自"全局约束"——每个顶点的颜色必须与所有邻居不同，局部信息无法控制全局色数。具体地，判断图是否3-可着色已是NP-complete，且不存在n^{1-ε}近似算法(除非P=NP)。
    关键要点: 1. MAX-3SAT的7/8来源于"每个子句独立的3个文字"的组合结构；2. 条件期望去随机化：逐个确定变量，每次选使期望满足子句数更大的赋值；3. 图着色的不可近似性远强于覆盖问题——甚至不能区分色数=3和色数=n^{1-ε}；4. 着色的困难在于"无冲突"约束是全局耦合的，不像覆盖问题有局部松弛；5. 特殊图类(二部图、弦图、完美图)可精确求解。
    实际案例: 硬件验证中MAX-3SAT：有1000个时序约束(子句)，随机赋值期望满足875个，条件期望法确定性地满足≥875个。图着色：100节点随机图，色数可能为5~20，贪心着色可能用30+种颜色且无理论保证。
    关联知识: MAX-SAT与SAT(模块七·§2.1)紧密关联；Goemans-Williamson的SDP舍入技术也用于MAX-CUT(模块八·§6)；图着色在特殊图类上的可解性(模块八·§7)。
