一、线性规划与整数规划

LP/IP是整个优化领域的基石。线性规划是所有优化方法中最成熟、应用最广泛的；整数规划则是从"容易"到"困难"的分水岭。本篇从标准形式讲起，覆盖单纯形法、内点法、分支定界、割平面、装箱问题、列生成，附完整手算过程。

1. 线性规划 (LP) 标准形式

通俗类比：把线性规划的「标准形式」想象成在一个统一的「竞技场规则」下进行比赛。原始的LP问题可能有 `≤`, `≥`, `=` 各种约束，变量可能有下界或无界，就像不同的比赛有不同的场地和规则。通过引入「松弛变量」和「剩余变量」（好比为不等式加缓冲），以及「拆分自由变量」（如x = x⁺ - x⁻），我们将所有约束都变成等式 `Ax = b`，所有变量都变成非负 `x ≥ 0`。这就像所有比赛都统一到「标准田径跑道」上，使用统一的计时和测量规则，方便求解器这个「裁判」用单纯形法高效工作。
常见误区：初学者容易犯的误区是认为必须将目标函数严格转为「最小化」形式才能使用标准形式。实际上，标准形式的核心是等式约束和非负变量。最大化 `c^T x` 等价于最小化 `-c^T x`，这只需将目标系数取反即可，不影响可行域。真正容易出错的是处理自由变量（取值可正可负）时，错误的拆分或遗漏，导致问题被错误地限制或求解器报错。
深入探讨：标准形式 `min c^T x, s.t. Ax=b, x>=0` 的深层价值在于它为「对偶理论」提供了完美框架。对偶问题 `max b^T y, s.t. A^T y <= c` 的最优解 `y*` 的每个分量，都对应原始问题约束的「影子价格」或「边际价值」。例如，在资源分配问题中，约束 `b_i` 表示第 `i` 种资源的总量，其对偶变量 `y_i*` 就表示该资源每增加一单位，目标函数最优值改善的幅度。这是线性规划在经济学中如此重要的理论基础。参见Dantzig的原始著作《Linear Programming and Extensions》(1963)。

1.1 LP的几何直觉

💡 关键定理：如果LP有最优解，则一定存在一个顶点最优解。这就是单纯形法的理论基础——只在顶点上搜索。

1.2 单纯形法 (Simplex Method) · 1947

George Dantzig 发明。沿可行多面体的棱从一个顶点走到相邻的更优顶点。

手算示例

max  z = 3x₁ + 2x₂
s.t. x₁ + x₂ ≤ 4      (约束1)
     x₁ + 3x₂ ≤ 6     (约束2)
     x₁, x₂ ≥ 0

步骤1: 引入松弛变量 s₁, s₂ 化为标准形式
     x₁ + x₂ + s₁ = 4
     x₁ + 3x₂ + s₂ = 6

步骤2: 初始基可行解 (x₁=0, x₂=0, s₁=4, s₂=6), z=0

步骤3: 选择入基变量（目标函数系数最大的非基变量）→ x₁ (系数3)
       选择出基变量（最小比值检验）:
         行1: 4/1 = 4
         行2: 6/1 = 6   → s₁ 出基 (比值最小)

步骤4: 枢轴运算 → 新基可行解 (x₁=4, x₂=0, s₁=0, s₂=2), z=12

步骤5: 继续迭代 → 最终 x₁=3, x₂=1, z=11 ✓
特性说明
迭代方式从一个顶点移动到相邻的更优顶点
最坏复杂度指数级（Klee-Minty反例）
实际表现极好！通常 2m~3m 次迭代（m=约束数）
适用规模百万变量、十万约束
优点成熟、可靠、天然提供灵敏度分析

1.3 对偶理论

每个LP（原问题）都有一个对偶问题。对偶的对偶回到原问题。

原问题 (P):                  对偶问题 (D):
min c^T x                    max b^T y
s.t. Ax ≥ b                  s.t. A^T y ≤ c
     x ≥ 0                        y ≥ 0

核心定理:
• 弱对偶: 任何对偶可行解 ≥ 任何原可行解的目标值
• 强对偶: 若原问题有最优解，则对偶也有，且 c*Tx* = b*Ty*
• 互补松弛: x*i(aiTx* - bi) = 0

1.4 内点法 (Interior Point) · 1984

Karmarkar提出的多项式时间算法，沿可行域内部的"中心路径"趋近最优解。

2. 整数规划 (IP / MIP)

通俗类比：整数规划就像用乐高积木搭建一个指定尺寸和承重的结构。线性规划（LP）好比是用橡皮泥或沙子，可以任意分割、塑形，很容易填满任何形状的空间（连续可行域），得到「最优解」。但整数规划（IP）要求你只能用一整块、一整块的乐高积木（离散变量）来搭建。你不能切开一块积木，因此即使你知道橡皮泥最优解在哪里，你也必须在它附近的积木堆里去寻找最接近的可行整数解，这个过程往往非常困难（NP-hard），且最优的「整数解」可能和「橡皮泥解」差别很大。
常见误区：最常见的误区是认为「先解对应的线性规划松弛（LP Relaxation），再四舍五入取整」就能得到一个不错的整数解。这通常会导致两个严重问题：1）四舍五入后的解可能违反某些约束（不可行）。2）即使可行，其目标函数值可能非常差。例如，在0-1规划中，LP松弛解可能是0.5，但最优整数解可能是0或1，目标函数值可能天差地别。四舍五入法是一种糟糕的启发式，应使用专门的整数规划求解器。
深入探讨：整数规划的计算复杂性是其核心特征。证明IP是NP-hard的经典方法是将已知的NP完全问题（如3-SAT，旅行商问题TSP）多项式时间归约到IP。在工程上，分支定界（Branch and Bound）算法是求解IP/MIP的基石。其效率高度依赖于两个关键技术：1）「线性规划松弛」提供下界/上界，以剪枝。2）「割平面」（Cutting Planes）不断收紧松弛的可行域，使其更接近整数包（Convex Hull）。Gomory在1958年提出了第一种系统生成割平面的方法。现代求解器的性能提升，很大程度上归功于割平面和分支策略的创新。

1.5 椭球法 (Ellipsoid Method) · 1979

Khachiyan提出的第一个多项式时间LP算法。理论上重要，但实践中比单纯形法慢得多。

核心思想: 用椭球不断缩小包含最优解的区域
  1. 从一个包含可行域的大椭球开始
  2. 检查椭球中心是否可行
  3. 若不可行, 用切割平面切掉一半, 构造更小的椭球
  4. 重复直到椭球足够小

理论意义: 证明了LP ∈ P (多项式时间可解)
实践意义: 几乎不用(太慢), 但启发了内点法和组合优化中的分离算法

3. 分解方法：拉格朗日松弛与Dantzig-Wolfe

通俗类比：处理一个庞大、复杂的大公司（大规模优化问题）时，「分解方法」就像实施「分权管理」。「拉格朗日松弛」好比将总部（目标函数）的某些严格规定（难以处理的耦合约束）暂时放宽，但要求各分公司（子问题）为违反这些规定「缴纳罚款」（拉格朗日乘子），然后不断调整罚款标准，寻找最优的管理方案。「Dantzig-Wolfe分解」则更像「联邦制」，将大问题拆分为独立的「邦」（子问题），各自产生自己的「方案」（极点/射线），然后由一个「联邦协调者」（主问题）来权衡和选择这些方案的最佳组合。
常见误区：一个关键的误区是认为拉格朗日松弛提供了问题的可行解。实际上，拉格朗日松弛得到的是原问题最优值的一个「下界」（对极小化问题），其对应的「解」通常不满足被松弛掉的约束（即对偶变量所乘的那些约束）。因此，需要额外的「次梯度优化」或「增广拉格朗日」等方法来更新乘子，并最终通过「可行性恢复」技术（如启发式）将松弛解转化为可行解。忽略这个后处理步骤，会得到无法实施的「解」。
深入探讨：拉格朗日松弛的数学核心是定义对偶函数 `g(λ) = min_{x in X} { L(x, λ) }`，其中 `L(x, λ) = f(x) + λ^T (Ax - b)`。`g(λ)` 是凹函数，其最大值 `max_λ g(λ)` 提供原问题 `min f(x) s.t. Ax=b, x in X` 的最强下界。次梯度法是更新 `λ` 的常用方法：`λ_{k+1} = λ_k + s_k * (Ax_k - b)`，其中 `s_k` 是步长。Dantzig-Wolfe分解则依赖于多面体的Minkowski表示定理，将可行域表示为极点和射线的凸组合。论文推荐：Geoffrion, A. M. (1974). Lagrangean relaxation for integer programming. *Mathematical Programming Study*, 2, 82-114.

3.1 拉格朗日松弛 (Lagrangian Relaxation)

核心思想：把"麻烦的约束"（耦合约束）通过拉格朗日乘子松弛到目标函数中，使剩余问题变得容易求解。

原问题 (困难):
  min c^T x
  s.t. Ax ≤ b    ← 耦合约束(使问题难解)
       Dx ≤ d    ← 简单约束(去掉耦合约束后问题容易)
       x ∈ X     ← 整数约束

拉格朗日松弛 (对λ≥0):
  L(λ) = min c^T x + λ^T(Ax - b)
         s.t. Dx ≤ d, x ∈ X
  = min (c + A^Tλ)^T x - λ^T b
    s.t. Dx ≤ d, x ∈ X
  → 对每个固定的λ，这是一个"简单"的IP/LP！

对偶问题:
  max_{λ≥0} L(λ)   ← 用次梯度方法求解

关键性质:
• L(λ) 是原问题最优值的下界(最小化问题)
• 拉格朗日对偶间隙 = L(λ*) - z*  (衡量松弛质量)
• 间隙越小，松弛越紧，分支定界效率越高

典型应用：选址问题、网络设计、调度问题——耦合约束通常是"能力约束"或"需求满足约束"。

3.2 次梯度优化拉格朗日乘子

拉格朗日对偶函数L(λ)是凹的但不可导（因为内层IP有多个最优解时L(λ)不光滑）。用次梯度方法求解：

λ^{k+1} = max(0, λ^k + t_k · (Ax^k - b))

其中 x^k 是 L(λ^k) 的最优解，t_k 是步长。
步长选择: t_k = α_k · (L(λ^k) - UB) / ||Ax^k - b||²
(UB是原问题的已知上界)

3.3 Dantzig-Wolfe分解

当LP/MIP的可行域可以表示为多个"小"多面体的凸组合时，用列生成思想分解。

原问题:
  min c^T x
  s.t. Ax = b       ← 主约束(耦合)
       x ∈ P₁ ∩ P₂  ← 子问题约束(可分解为独立块)

Dantzig-Wolfe形式:
  将x表示为 P₁ 和 P₂ 的极点的凸组合:
  x = Σ λᵢ vᵢ   (vᵢ是P₁的极点)
  x = Σ μⱼ wⱼ   (wⱼ是P₂的极点)

主问题(Master): 选择最优的λ和μ
子问题(Subproblem): 给定对偶价格，找有负reduced cost的新极点

→ 列生成的推广！适合块角结构(block-angular)的大规模LP/MIP
💡 拉格朗日松弛 vs Dantzig-Wolfe：两者是对偶的！拉格朗日松弛是"放松约束到目标"，Dantzig-Wolfe是"展开可行域为极点"。在实践中，选择哪个取决于问题结构：块角结构用D-W，耦合约束少用拉格朗日。

4. 装箱与切割问题 (Bin Packing)

通俗类比：装箱问题就是经典的「行李打包」或「集装箱装货」难题。你有一些不同大小的物品（数据包、订单任务、钢板需求），和许多容量完全相同的箱子（服务器内存块、运输车辆、标准板材）。目标是用最少的箱子装下所有物品。贪心算法（如「首次适应」算法）就像你随手拿起一件行李，看到第一个还没满的箱子就塞进去。这通常很快，但结果往往不是最优——可能因为一个不凑巧的放置顺序，导致最后多用了好几个箱子。寻找全局最优解则需要更聪明的策略。
常见误区：一个常见误区是认为装箱问题就是简单的「除法」（总物品大小除以箱子容量），或者简单地按照物品大小降序排列（如首次适应递减算法FFD）就一定能得到最优解。事实上，1维装箱问题是NP-hard的。FFD算法虽然实践表现好，但最坏情况性能比 `FFD(I) <= (11/9)OPT(I) + 6/9`（其中OPT(I)是最优解箱数），并非最优。过度依赖简单启发式，会错失在成本敏感场景下节约10%-20%资源的机会。
深入探讨：装箱问题是组合优化的核心问题之一，与调度、切割库存、内存分配密切相关。除了FFD，更优的「精确算法」包括：1）分支定界法，需要设计强下界（如基于连续松弛、对偶可行函数）。2）列生成法，将每个可行的箱子装法视为一「列」，主问题选择哪些列使用，子问题是「背包问题」来生成新列。一个有趣的工程细节是：在云计算的虚拟机调度中，装箱问题被称为「向量装箱」，因为每个虚拟机对CPU、内存、网络的需求是多维的。理论参考：Johnson, D. S., et al. (1974). Worst-case performance bounds for simple one-dimensional packing algorithms. *SIAM Journal on Computing*, 3(4), 256-278.

5. 建模技巧：问题转化与线性化

通俗类比：优化建模的「转化与线性化」技巧，就像一位出色的「翻译官」。你遇到的现实问题（如「生产与否的决策」、「固定成本」、「最大值约束」）是口语，充满模糊和非线性的表达。而标准的线性/整数规划求解器只听得懂「书面语」（线性、整数约束）。转化技巧就是将口语翻译成书面语的过程。例如，`y = max(x1, x2)` 不是线性，但你可以翻译成：`y >= x1` 且 `y >= x2`，并让目标函数尽可能地把 `y` 压下去（因为是min问题），这样 `y` 自然就等于 `x1` 和 `x2` 中的最大者了。这个翻译过程就是建模的核心艺术。
常见误区：一个危险的误区是滥用「大M法」（Big-M）来线性化逻辑约束。大M法通过引入一个足够大的常数M和二进制变量，将 `IF-THEN` 逻辑转化为线性不等式。例如，`if (y=1) then x <= 10` 可以建模为 `x <= 10 + M*(1-y)`。误区在于：1）M选择过大会导致数值不稳定，求解困难（LP松弛太弱）。2）选择过小可能无意中切掉了可行解。应该尽可能寻找问题结构中的自然上界来替代M，或者使用「指示约束」（Indicator Constraints）等现代求解器功能。
深入探讨：线性化非线性项是建模的基础。对于双线性项 `z = x*y`（其中x,y为连续变量，或一个连续一个二进制），有标准技巧：当x和y是二进制变量时，`z`就是逻辑与，可以线性化。当它们连续但有界时，可以使用McCormick包络进行松弛或精确线性化。对于分段线性函数（如阶梯电价、分段线性成本），可以使用「SOS2变量」（特殊有序集）或增量法建模。一个高级技巧是「隐式建模」：对于复杂的约束，有时不需要显式写出来，而是通过巧妙设计子问题（如列生成中的子问题）来隐式地满足它，这能大幅降低模型规模。参见：Williams, H. P. (2013). *Model Building in Mathematical Programming*. John Wiley & Sons.

实际问题的目标函数和约束往往不是"天然线性"的。掌握转化技巧，把非线性/逻辑约束变成LP/MIP能处理的形式，是建模的核心能力。

5.1 绝对值

目标中出现 |x|:  min Σ|xᵢ - aᵢ|
转化: 引入辅助变量 tᵢ ≥ 0
  min Σ tᵢ
  s.t. tᵢ ≥ xᵢ - aᵢ
       tᵢ ≥ -(xᵢ - aᵢ)    ← 两个约束确保 tᵢ = |xᵢ - aᵢ|

5.2 min / max 函数

目标中出现 max(f₁, f₂):
  引入辅助变量 z,  min z
  s.t. z ≥ f₁,  z ≥ f₂      ← z 自然取 max(f₁,f₂)

约束中出现 max(f₁, f₂) ≤ b:
  等价于 f₁ ≤ b 且 f₂ ≤ b   ← 直接拆成两个约束

目标中出现 min(f₁, f₂):
  引入二元变量 y ∈ {0,1},  min z
  s.t. z ≥ f₁ - M(1-y)
       z ≥ f₂ - My           ← y=0时约束1生效, y=1时约束2生效

5.3 大M法 (Big-M Method)

用一个足够大的常数M来"关闭"约束。将逻辑条件转化为线性约束的核心技巧。

"如果y=0，则约束必须满足; 如果y=1，则约束可以违反"
  f(x) ≤ b + M·y
  y ∈ {0,1}

"选A或选B (至少选一个)"
  x_A + x_B ≥ 1,  x_A, x_B ∈ {0,1}

"如果选了仓库j (yⱼ=1)，才能服务客户i"
  xᵢⱼ ≤ yⱼ     ← xᵢⱼ=客户i由仓库j服务的比例

⚠️ M要"足够大但不要太大":
  M太小 → 可行域被错误地裁掉
  M太大 → LP松弛很弱, 分支定界效率极差
  经验法则: M取约束右端可能最大值的10倍以内

5.4 分段线性函数

目标函数是分段线性的(如折扣定价、阶梯电价):
  f(x) = { 2x     if 0 ≤ x ≤ 100
         { 200+1.5x if 100 < x ≤ 300
         { 450+x    if x > 300

转化: 用SOS2变量(Special Ordered Set type 2)
  x = Σ λᵢ · xᵢ,  Σλᵢ = 1,  λᵢ ≥ 0
  最多两个相邻的λᵢ非零
  f(x) = Σ λᵢ · f(xᵢ)

或者用二元变量:
  x = x₁ + x₂ + x₃,  0≤x₁≤100, 0≤x₂≤200, x₃≥0
  y₁,y₂,y₃ ∈ {0,1} 表示落在哪个段
  x₁ ≤ 100y₁, x₂ ≤ 200y₂, y₁+y₂+y₃=1
  f(x) = 2x₁ + 1.5x₂ + x₃ + 200y₂ + 450y₃

5.5 逻辑约束

"如果A则B":       x_A ≤ x_B
"如果A则非B":      x_A + x_B ≤ 1
"A当且仅当B":      x_A = x_B
"恰好选k个":       Σxᵢ = k
"至少连续选3个":   xᵢ + xᵢ₊₁ + xᵢ₊₂ ≥ 3yᵢ  (yᵢ表示起始位置)
"互斥约束":        xᵢ + xⱼ ≤ 1

"如果f(x)>0, 则y=1" (指示变量):
  y ≥ (f(x) - ε) / M    (ε是小正数, M是大M)
  即: f(x) > 0 时 y 必须为 1

  1. 核心原理: 线性规划的标准形式为 max cᵀx, s.t. Ax≤b, x≥0。求解方法：① 画出可行域（约束条件围成的多边形）；② 目标函数等值线沿梯度方向移动；③ 最优解在可行域的顶点处取得（基本定理）。手算步骤：先求约束交点，再代入目标函数比较。
    关键要点: ① 可行域是凸多边形 ② 最优解在顶点 ③ 等值线法直观判断方向
    实际案例: max z=5x₁+4x₂, s.t. 6x₁+4x₂≤24, x₁+2x₂≤6, x₁,x₂≥0。约束交点：(0,0),(4,0),(3,1.5),(0,3)。代入：z(4,0)=20, z(3,1.5)=21, z(0,3)=12。最优解为x₁=3, x₁=1.5, z*=21。等值线5x₁+4x₂=21恰好切于(3,1.5)点。
    关联知识: 与单纯形法（沿顶点移动）、内点法（穿越可行域内部）的求解路径对比
  2. 核心原理: 将连续变量改为整数变量后，问题从LP变为ILP（整数线性规划），复杂度从P变为NP-hard。关键变化：① 可行域从连续多边形变为离散格点；② 最优解可能不在LP最优解的顶点处；③ 需要用分支定界、割平面等方法求解。整数约束的"四舍五入"不一定可行——可能违反约束。
    关键要点: ① ILP是NP-hard ② 可行域变为离散格点 ③ 四舍五入不一定可行 ④ 分支定界是标准求解法
    实际案例: 上题LP最优解x₁=3, x₂=1.5。加整数约束后，x₂必须为整数。检查可行整数点：(3,1)→z=19, (4,0)→z=20, (2,2)→z=18。整数最优解为x₁=4, x₂=0, z*=20，比LP最优解z*=21差。这就是整数规划的"间隙"（Integrality Gap）。
    关联知识: 与松弛问题（LP relaxation）、分支定界法、割平面法的关系
  3. 核心原理: PuLP是Python的LP/ILP建模工具，灵敏度分析考察参数变化对最优解的影响。影子价格（Shadow Price）是约束右端项bᵢ增加1单位时目标函数的增量，反映资源的边际价值。在PuLP中，通过constraint.pi属性获取影子价格。
    关键要点: ① 影子价格=资源边际价值 ② 非紧约束的影子价格为0 ③ PuLP用constraint.pi获取
    实际案例: 5变量3约束LP示例代码：
    from pulp import *
    prob = LpProblem("LP", LpMaximize)
    x = [LpVariable(f"x{i}", 0) for i in range(5)]
    prob += 3*x[0]+2*x[1]+4*x[2]+x[3]+5*x[4] # 目标
    prob += x[0]+x[1]+x[2] <= 10 # 约束1
    prob += 2*x[2]+x[3]+x[4] <= 15 # 约束2
    prob += x[0]+3*x[4] <= 12 # 约束3
    prob.solve()
    求解后用prob.constraints[0].pi获取约束1的影子价格。若影子价格=2，表示约束1的右端项从10增加到11时，最优目标值增加2。
    关联知识: 与对偶理论（影子价格=对偶变量）、经济解释（资源定价）
