六、随机规划与鲁棒优化

当问题参数不确定时（需求波动、天气变化、市场价格），标准优化方法可能给出在实际中不可行或次优的解。本篇覆盖处理不确定性的两大范式（随机规划 vs 鲁棒优化）、在线算法、以及强化学习视角。

1. 不确定性的来源

不确定性的来源

通俗类比：就像天气预报总有误差，现实世界的数据、参数和外部环境都存在‘不确定性’。它是系统固有的属性，而非模型的缺陷。
常见误区：最大的误区是忽略或假设不确定性不存在。这会导致模型在实际应用中变得脆弱，性能急剧下降，就像在晴天建造的纸房子，雨天必然倒塌。
深入探讨：不确定性主要分为三类：1) **参数不确定性**：模型系数（如成本、需求）不精确。2) **结构不确定性**：模型方程形式本身可能有误。3) **数据不确定性**：观测数据存在噪声或缺失。量化这些不确定性是后续所有随机方法的基础。

2. 随机规划 (Stochastic Programming)

通俗类比：做旅行计划时，你不仅考虑最可能的天气（期望），还会为晴、雨、多云制定不同的备用方案。随机规划就是系统地管理这些‘备用方案’。
常见误区：生成具有代表性的‘场景’（场景树）非常关键。场景太少或偏差大，会导致解质量差；场景太多，计算又不堪重负。需在精确性和计算效率间权衡。
深入探讨：其标准形式为两阶段问题：第一阶段做‘这里-现在’决策，第二阶段在观察到随机变量实现后做‘补偿’决策。目标是最小化第一阶段成本与未来所有可能场景下期望补偿成本之和。数学形式：min c^Tx + E_Q[Q(x,ξ)]。

3. 鲁棒优化 (Robust Optimization)

通俗类比：设计建筑时，不仅要考虑平均风力，更要确保能抵御‘百年一遇’的极端天气。鲁棒优化就是为系统寻找在‘最坏情况’下依然表现良好的方案。
常见误区：保守是其特性也是代价。追求‘免疫’所有极端情况的解，可能在大多数普通情况下表现平庸。需要在鲁棒性和平均性能间找到平衡点。
深入探讨：核心是定义**不确定集合** Ω，优化目标变为 min_x max_{ξ∈Ω} f(x, ξ)。集合形状至关重要：盒式集合导致线性规划增长；椭球集合（SOCP）能权衡保守性与概率保证；数据驱动集合则从历史数据中学习。

确定性优化不确定性为零最优解无（脆弱）

3.2 对等转化

鲁棒优化的核心技巧：将min-max问题转化为确定性等价问题（通常是LP或SOCP）。

原问题: min c^T x  s.t.  a^T x ≤ b, ∀a∈U
盒式不确定集: aᵢ ∈ [āᵢ-δᵢ, āᵢ+δᵢ]
等价转化: ā^T x + Σδᵢ|xᵢ| ≤ b  (绝对值可线性化)

2.3 多阶段随机规划

两阶段的推广：决策在多个时间点逐步做出，不确定参数在阶段间逐步揭示。

三阶段示例 — 电网调度:
  阶段1(提前1天): 确定发电计划 x₁
  阶段2(当天早上): 观测天气(风力/光照), 调整调度 x₂
  阶段3(实时): 观测实际需求, 启动备用机组 x₃

  min c₁ᵀx₁ + E[c₂ᵀx₂(ξ₂) + E[c₃ᵀx₃(ξ₂,ξ₃)]]
      ↑         ↑                 ↑
    确定性    条件期望          条件期望

求解: 场景树方法 — 将连续分布离散化为有限场景
  阶段1: 1个节点(根)
  阶段2: K₂个节点(每种天气场景)
  阶段3: K₂×K₃个节点(天气×需求组合)
  → 大规模LP/MIP, 但有特殊结构可用Benders/L形方法分解

2.4 滚动时域优化 (Model Predictive Control, MPC)

不是一次性规划整个时间跨度，而是只规划未来N步，执行第1步，然后重新规划。

MPC框架 (每步重复):
  1. 观测当前状态 x(t)
  2. 求解优化问题:
     min Σᵢ₌₀ᴺ L(x(t+i), u(t+i)) + V(x(t+N))
     s.t. x(t+i+1) = f(x(t+i), u(t+i))  ← 系统动力学
          约束条件
  3. 执行 u(t) (只用第一步控制)
  4. t ← t+1, 回到步骤1

为什么滚动而不是一次性规划?
  • 模型不完美 → 逐步修正
  • 不确定性 → 用最新信息更新
  • 计算可行 → 只优化短窗口

应用: 化工过程控制、自动驾驶、机器人、电力调度

2.5 双层优化 (Bilevel Optimization)

上层决策者和下层决策者嵌套优化：上层的选择影响下层的问题，下层的反应影响上层的目标。

双层优化形式:
  上层: min F(x, y*(x))
  s.t.  G(x) ≤ 0
        y*(x) = argmin_y f(x,y)  ← 下层优化问题
              s.t. g(x,y) ≤ 0

含义: 上层选x, 下层对x做出最优反应y*(x), 上层考虑y*(x)来优化

典型应用:
  • 定价: 公司定价x, 消费者最优购买y*(x), 公司利润最大化
  • 网络设计: 政府建路x, 用户选最优路径y*(x), 最小化总拥堵
  • 对抗ML: 攻击者选扰动x, 防御者最优响应y*(x)

求解难度: 即使上下层都是LP, 双层问题也是NP-hard!
方法: KKT转化(将下层替换为KKT条件)、罚函数、启发式
💡 分阶段优化的谱系： 两阶段随机规划 → 多阶段随机规划 → 滚动时域MPC → 强化学习(MDP) ← 这是一条从"确定性模型"到"完全在线学习"的连续谱系。模型越精确用左边，不确定性越大用右边。

4. 机会约束规划 (Chance-Constrained Programming)

机会约束规划 (Chance-Constrained Programming)

通俗类比：制定航班计划时，目标不是‘绝对准点’（不可能），而是‘准点概率≥95%’。机会约束规划直接对解的‘可靠性’设定了概率门槛。
常见误区：最大的陷阱是：即使模型满足概率约束，实际运行中也可能因小概率事件连续发生而导致严重失败。它是‘大概率可行’，而非‘绝对安全’。
深入探讨：其形式为 min c^Tx s.t. Pr{g(x, ξ) ≤ 0} ≥ 1-ε。当ξ服从正态分布时，可转化为二阶锥规划(SOCP)。例如，若g(x, ξ)=a^Tx + ξ，约束变为 a^Tx + Φ^{-1}(1-ε) * σ ≤ 0，其中Φ是标准正态CDF。

要求约束以至少一定概率满足，而非对所有场景都满足。

P( a(ξ)Tx ≤ b ) ≥ 1 − ε

含义：允许约束被违反的概率不超过ε（如5%）。比鲁棒优化更灵活，比随机规划更安全。

例子 — 供应链:
  需求D ~ N(μ, σ²)
  机会约束: P(D ≤ 供应量) ≥ 0.95  (95%的服务水平)
  等价: 供应量 ≥ μ + 1.645σ  (正态分布下)

一般情况: 需要对不确定分布积分，通常用场景近似或CVaR转化

5. 马尔可夫决策过程 (MDP)<决策过程 (MDP)的工作原理，有助于在实际工程中做出更合理的技术选择和优化决策。

马尔可夫决策过程 (MDP)

通俗类比：下棋时，每走一步（决策）都会改变棋盘状态（环境），并影响后续所有可能性（未来收益）。MDP就是为这类序贯决策问题建立的数学框架。
常见误区：状态定义是MDP的灵魂。定义得太粗糙，模型失去预测能力；定义得太细致，状态空间爆炸（维数灾难），难以求解。需要领域知识和抽象的巧妙平衡。
深入探讨：一个五元组 (S, A, P, R, γ)：S状态集，A动作集，P转移概率，R即时奖励，γ折扣因子。核心是求解贝尔曼方程：V(s) = max_a [ R(s,a) + γ * Σ P(s'|s,a) * V(s') ]。常用值迭代、策略迭代求解。
  [状态 S_t]\n       | (采取动作 A_t)\n       v\n  [获得奖励 R_t]\n       | (环境转移到新状态)\n       v\n  [状态 S_{t+1}] --> 循环，目标是最大化累积折扣奖励
特性有限期MDP无限期MDP (折扣)
时间范围固定T步无限长
目标函数期望总回报 Σ_{t=0}^T γ^t R_t期望折扣回报 E[Σ_{t=0}^∞ γ^t R_t]
策略性质通常是非平稳的（与时间t有关）是平稳的（仅与状态s有关）
策略通信量适用互联典型场景
张量并行(TP)高NVLink节点内多卡
流水线并行(PP)低PCIe/IB跨节点
数据并行(DP)中任意多数据源
专家并行(EP)高NVLinkMoE模型

多阶段序贯决策的数学框架。每步观察状态，选择动作，获得奖励，转移到下一状态。

MDP五元组: (S, A, P, R, γ)
  S: 状态集合
  A: 动作集合
  P(s'|s,a): 状态转移概率
  R(s,a): 即时奖励
  γ: 折扣因子(0<γ<1)

Bellman方程 (最优值函数):
  V*(s) = max_a { R(s,a) + γ·Σ P(s'|s,a)·V*(s') }

求解方法:
• 值迭代: V_{k+1}(s) = max_a { R(s,a) + γ·Σ P(s'|s,a)·V_k(s') }
• 策略迭代: 交替评估当前策略+改进策略
• 线性规划: min ΣV(s) s.t. V(s) ≥ R(s,a) + γ·ΣP(s'|s,a)·V(s')
💡 MDP与强化学习的关系：RL = MDP + 未知模型(P和R未知)。当转移概率P已知时，用值迭代/策略迭代精确求解。当P未知时，用RL（Q-learning、PPO等）通过采样学习。

6. 在线算法

在线算法

通俗类比：玩在线游戏时，你只能根据已出现的地图信息做决策，无法预知未来。在线算法就是处理这种‘边做边看’、信息逐步到来场景的策略。
常见误区：不要将‘在线’与‘随机’混淆。在线算法关注的是输入序列的‘到达顺序’和‘未知性’，而随机算法关注的是算法内部的‘随机选择’。
深入探讨：其性能用**竞争比**衡量：在线算法成本/离线最优成本 ≤ c。经典结果：对在线滑雪租赁问题，‘租用直到租用总花费等于购买价，然后购买’的策略竞争比为2。**后悔值**是另一个核心概念，衡量与事后最优决策的差距。

3.3 分布鲁棒优化 (Distributionally Robust Optimization)

介于随机规划(已知分布)和鲁棒优化(不知分布)之间：知道分布属于某个模糊集(ambiguity set)。

min_{x} max_{P∈P} E_P[f(x,ξ)]
  P = 模糊集: 所有与观测数据一致的概率分布
  优化最坏情况分布下的期望

模糊集的选择:
• 矩模糊集: 均值和方差在某个范围内 → SDP可解
• Wasserstein球: 与经验分布的Wasserstein距离≤ε → LP可解
• φ-散度球: KL散度/χ²散度在某个范围内

优势: 比鲁棒优化不那么保守, 比随机规划更鲁棒
应用: 金融风险管理、供应链、医疗决策

7. 多目标优化

多目标优化

通俗类比：买手机时，你希望‘性能强’且‘价格低’且‘续航久’。多目标优化就是在这些相互冲突的目标间，寻找一组最佳的权衡方案（帕累托前沿）。

常见误区：避免直接为多个目标简单赋权求和。权重的选择非常主观且困难，可能导致丢失重要的非劣解。探索完整的权衡前沿更具参考价值。
深入探讨：解 x* 是**帕累托最优**的，当且仅当不存在另一个解 x 支配它（即在所有目标上不差，且至少在一个目标上严格更好）。求解帕累托前沿的方法：1）加权法（需多次运行）。2）ε-约束法（优化一个目标，其他转为约束）。3）多目标进化算法（如NSGA-II）。
目标空间 (假设最小化两个目标 f1, f2)\n      f2 ^\n         |        (不可接受区域)\n         |        * (被支配解)\n         |       *   *\n         |------*-------*----- 帕累托前沿\n         |    *   ★★★   *     (非支配解集)\n         |   *         *  --> f1\n         |  *\n         +------------------->\n         (理想点：每个目标都达到最小)
方法核心思想优点缺点
加权和法将多目标转为单目标：min Σ w_i f_i简单，可用现有单目标算法权重敏感，无法求解非凸前沿
ε-约束法优化一个主目标，其他转为约束：g_j(x) ≤ ε_j可获得边界解，控制计算量需要多次运行，选择ε值困难
帕累托搜索法直接搜索非支配解集（如进化算法）一次运行得整个近似前沿计算量大，解的分布可能不均

✅ 参考答案与详解
  1. 核心原理: 报童问题是两阶段随机规划的经典案例。第一阶段：订购x份报纸(决策变量)。第二阶段：需求D揭示后，如果D>x则损失销售机会(欠货成本)，如果D关键要点: 1. 报童模型的关键是"临界比率"(p-c)/(p-s)；2. 临界比率越高(利润高、残值低)，应订购更多；3. 需求不确定性(σ)越大，订购量与均值的偏差越大；4. 该模型可推广到多产品(关联需求)、多阶段(动态订购)场景；5. 当需求分布未知时，可用鲁棒优化替代。
    实际案例: 鲜花店母亲节备货：成本8元/束，售价20元/束，未售出花束残值3元/束，需求N(200,50²)。临界比率=(20-8)/(20-3)=0.706，Φ⁻¹(0.706)≈0.54，最优订购量=200+0.54×50=227束。
    关联知识: 报童问题是MDP(§5)的单步特例；与机会约束规划(§4)可结合——改为P(缺货率≤5%)≥0.95的服务水平约束；与鲁棒优化(§3)结合可处理分布未知情况。
  2. 核心原理: 鲁棒对等问题是将含不确定参数的优化问题转化为确定性等价问题。设原LP为min c^Tx s.t. aᵢ^Tx ≤ b, ∀i，其中约束系数aᵢ在盒式不确定集U={aᵢ : |aᵢⱼ-āᵢⱼ| ≤ δᵢⱼ}中变化。鲁棒对等问题要求约束对所有可能的aᵢ∈U都满足。数学上：āᵢ^Tx + Σⱼδᵢⱼ|xⱼ| ≤ b。绝对值|xⱼ|可通过引入辅助变量线性化：xⱼ=xⱼ⁺-xⱼ⁻, xⱼ⁺≥0, xⱼ⁻≥0, 则|xⱼ|=xⱼ⁺+xⱼ⁻。
    关键要点: 1. 盒式不确定集的鲁棒对等是LP（保持了计算可处理性）；2. 椭球不确定集的鲁棒对等是SOCP（二阶锥规划）；3. Bertsimas-Sim预算不确定集可控制保守度——设参数Γ控制最多Γ个参数同时取极端值；4. 鲁棒对等问题的可行域是原问题可行域的子集（更保守）；5. Γ=0退化为名义问题，Γ=n退化为完全盒式鲁棒。
    实际案例: 供应链LP：运输成本系数c不确定(受油价波动)，设c∈[c̄-δ, c̄+δ]。鲁棒对等问题：min c̄^Tx + δ^T|x| s.t. Ax=b, x≥0。求解后得到在最坏成本下仍最优的运输方案。
    关联知识: 鲁棒优化与随机规划(§2)互补——有分布信息用随机规划，无分布信息用鲁棒优化；与分布鲁棒优化(§3.3)结合可得到折中方案；椭球不确定集的对等问题涉及SOCP(模块二·凸优化)。
