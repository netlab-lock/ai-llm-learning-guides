06 PyTorch FSDP2 完全指南

FSDP1 vs FSDP2 的核心区别

FSDP2（Fully Sharded Data Parallel 2）是 PyTorch 原生的分布式训练方案，相比 FSDP1 有重大架构改进。

特性FSDP1FSDP2
参数表示FlatParameter（所有参数拼成一个大张量）每参数独立 shard（DTensor）
混合精度WrappedAMPMixedPrecisionPolicy（更灵活）
与 TP 结合困难（FlatParameter 不兼容）原生支持（DTensor 统一）
torch.compile不支持支持
分片粒度整个模块单个参数
💡 实用技巧：FSDP2 最关键的改进是用 DTensor 替代 FlatParameter：每个参数独立分片（而非拼成一个大张量），这使得 FSDP2 天然兼容 TP 和 torch.compile。如果你在新项目中选择框架，FSDP2 是 PyTorch 原生方案的首选。迁移成本：将 FSDP 的 FullyShardedDataParallel(model) 替换为 fully_shard(layer) 即可。
⚠️ 注意事项：FSDP1 的 FlatParameter 方案在实践中有很多坑：参数名改变导致 checkpoint 加载困难、与 TP 不兼容、无法用 torch.compile。迁移到 FSDP2 可以一次性解决这些问题。但注意 FSDP2 需要较新版本的 PyTorch（2.2+）。
🔍 深入理解：「FSDP1 vs FSDP2 的核心区别」的深层理解：从系统设计的角度看，TP的设计体现了关注点分离的原则——每个组件只负责一个明确的功能，通过清晰的接口协作。这种设计哲学是构建可维护、可扩展系统的基础。

FSDP2 核心 API

import torch
from torch.distributed._composable.fsdp import fully_shard
from torch.distributed._tensor import DTensor

# 模型定义
model = TransformerModel(config)

# 对每一层应用 FSDP
for layer in model.layers:
    fully_shard(layer)  # 每层独立分片

# 最外层也应用 FSDP
fully_shard(model)

# 配置混合精度
from torch.distributed._composable.fsdp import MixedPrecisionPolicy

mp_policy = MixedPrecisionPolicy(
    param_dtype=torch.bfloat16,      # 参数精度
    reduce_dtype=torch.float32,       # 梯度归约精度
    output_dtype=torch.bfloat16,      # 输出精度
)

for layer in model.layers:
    fully_shard(layer, mp_policy=mp_policy)
fully_shard(model, mp_policy=mp_policy)
💡 实用技巧：FSDP2 的核心 API 只有两个：fully_shard(layer) 和 MixedPrecisionPolicy。最佳实践：先对每层调用 fully_shard(layer)，再对整个模型调用 fully_shard(model)。混合精度推荐 BF16 参数 + FP32 梯度归约（reduce_dtype=torch.float32），兼顾精度和显存。
⚠️ 注意事项：FSDP2 API 的一个常见陷阱是分片顺序：必须先分片子模块，再分片父模块。如果反过来，子模块的参数会被父模块的分片覆盖，导致意外行为。另外 MixedPrecisionPolicy 的 output_dtype 参数容易被忽略，它控制前向输出的精度。
🔍 深入理解：FSDP2 的 API 设计哲学是「composable」（可组合的）——fully_shard 不是像 FSDP1 那样包装整个模型，而是对每一层独立应用。这种设计使得 FSDP2 可以与 TP、SP 等其他并行策略自由组合，每层可以有不同的分片策略。

FSDP2 的执行流程

FSDP2 前向传播： ┌─────────────────────────────────────────┐ │ Layer 0 前向 │ │ 1. AllGather 收集完整参数 │ │ 2. 计算前向 │ │ 3. 释放非本地 shard（保留完整参数用于反向）│ ├─────────────────────────────────────────┤ │ Layer 1 前向 │ │ ...同上... │ └─────────────────────────────────────────┘ FSDP2 反向传播： ┌─────────────────────────────────────────┤ │ Layer 1 反向 │ │ 1. AllGather 收集完整参数 │ │ 2. 计算梯度 │ │ 3. ReduceScatter 分片梯度 │ │ 4. 释放完整参数 │ ├─────────────────────────────────────────┤ │ Layer 0 反向 │ │ ...同上... │ └─────────────────────────────────────────┘
💡 实用技巧：FSDP2 的执行流程本质就是 ZeRO-3 的 PyTorch 原生实现。前向：AllGather → 计算 → 释放非本分片；反向：AllGather → 计算梯度 → ReduceScatter → 释放。理解这个流程对调试 OOM 和通信瓶颈至关重要。用 torch.profiler 可以可视化每层的 AllGather/ReduceScatter 耗时。
⚠️ 注意事项：FSDP2 执行流程中容易被忽略的细节是「释放时机」。前向时非本分片参数在计算完后立即释放，但完整参数会保留到反向结束后才释放（因为反向需要完整参数计算梯度）。这意味着峰值显存 = 分片参数 + 单层完整参数 + 激活值。
🔍 深入理解：FSDP2 的执行流程与 DeepSpeed ZeRO-3 几乎相同，但实现方式不同：DeepSpeed 用 C++ 扩展和自定义 CUDA kernel，FSDP2 用纯 Python + DTensor。FSDP2 的优势是更易调试和扩展，劣势是某些极端场景下性能可能略低。

FSDP2 + torch.compile

# FSDP2 与 torch.compile 组合使用
model = TransformerModel(config)

for layer in model.layers:
    fully_shard(layer)
fully_shard(model)

# 编译模型（FSDP2 原生支持）
compiled_model = torch.compile(model)

# 训练循环
for batch in dataloader:
    loss = compiled_model(batch)
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()
💡 实用技巧：FSDP2 是第一个原生支持 torch.compile 的全分片方案。使用方式：compiled_model = torch.compile(model)，无需额外配置。实测收益：compile 可将训练吞吐提升 10-25%（取决于模型结构）。注意：自定义 CUDA 算子可能与 compile 不兼容，需要先检查。
⚠️ 注意事项：FSDP2 + torch.compile 的一个已知限制是动态 shape 支持。如果输入序列长度变化较大，compile 的 graph break 会增多，性能收益下降。建议在训练时使用固定的 padding 策略，确保输入 shape 尽量一致。
🔍 深入理解：FSDP2 与 torch.compile 的兼容性得益于 DTensor 的设计：DTensor 的分片和收集操作是 torch.compile 可以追踪的 Python 操作，不像 FSDP1 的 FlatParameter 需要特殊的 C++ 处理。这是 DTensor 统一抽象带来的额外好处。

FSDP2 + TP（DTensor 统一）

# FSDP2 与张量并行结合
from torch.distributed.tensor.parallel import parallelize_module, ColwiseParallel, RowwiseParallel

# 先应用 TP
tp_plan = {
    "attention.q_proj": ColwiseParallel(),
    "attention.k_proj": ColwiseParallel(),
    "attention.v_proj": ColwiseParallel(),
    "attention.out_proj": RowwiseParallel(),
    "mlp.fc1": ColwiseParallel(),
    "mlp.fc2": RowwiseParallel(),
}
model = parallelize_module(model, device_mesh=tp_mesh, parallelize_plan=tp_plan)

# 再应用 FSDP
for layer in model.layers:
    fully_shard(layer)
fully_shard(model)
💡 实用技巧：FSDP2 + TP 的混合并行是 DTensor 统一抽象的核心优势。实现步骤：先用 parallelize_module() 应用 TP（ColwiseParallel/RowwiseParallel），再用 fully_shard() 应用 FSDP。TP 组内用 NVLink 做 AllReduce，TP 组间用 IB 做 FSDP 的 AllGather/ReduceScatter。
⚠️ 注意事项：FSDP2 + TP 配置的一个关键决策是 TP 度和 FSDP 度的分配。8 GPU 单机：TP=8（全 NVLink）或 TP=2+FSDP=4 是常见选择。TP 度越大，TP 通信越快（NVLink 带宽高），但 FSDP 的分片度越小（显存节省越少）。需要根据模型大小和通信带宽做权衡。
🔍 深入理解：FSDP2 + TP 的组合展示了 DTensor 统一抽象的价值：在 DTensor 框架下，TP 和 FSDP 只是同一张量的不同维度的分片方式。TP 沿列/行维度分片，FSDP 沿参数维度分片，两者可以自由组合而无需特殊处理。这是 FSDP2 相比 FSDP1 最大的架构优势。

实战：用 FSDP2 训练 7B 模型

import torch
import torch.distributed as dist
from torch.distributed._composable.fsdp import fully_shard, MixedPrecisionPolicy

def main():
    dist.init_process_group("nccl")
    local_rank = int(os.environ["LOCAL_RANK"])
    torch.cuda.set_device(local_rank)

    # 模型
    model = build_llama_7b(config).cuda()

    # 混合精度策略
    mp = MixedPrecisionPolicy(
        param_dtype=torch.bfloat16,
        reduce_dtype=torch.bfloat32,
    )

    # 分片
    for layer in model.layers:
        fully_shard(layer, mp_policy=mp)
    fully_shard(model, mp_policy=mp)

    # 优化器
    optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4)

    # 训练
    for step, batch in enumerate(dataloader):
        batch = {k: v.cuda() for k, v in batch.items()}
        loss = model(**batch).loss
        loss.backward()
        optimizer.step()
        optimizer.zero_grad()

        if step % 100 == 0:
            print(f"Step {step}, Loss: {loss.item():.4f}")

if __name__ == "__main__":
    main()

FSDP2 优势：PyTorch 原生，无需额外安装；API 简洁；与 torch.compile 和 TP 无缝集成。适合 7B-70B 模型训练。

⚠️ 注意事项：7B 模型训练中的常见问题：① OOM 时检查是否开启了梯度检查点（activation_checkpointing）；② Loss 爆炸时检查混合精度配置，reduce_dtype 建议用 float32；③ 训练速度慢时检查是否开启了 forward_prefetch 和 backward_prefetch。
🔍 深入理解：FSDP2 训练 7B 模型的完整流程展示了 PyTorch 原生分布式训练的最佳实践：DTensor 分片 + 混合精度 + 通信预取 + 梯度累积。相比 DeepSpeed 的 JSON 配置方式，FSDP2 的代码方式更透明，适合需要深度定制的场景。

FSDP2 的核心架构与实现细节

PyTorch FSDP2（Fully Sharded Data Parallel 2）是 PyTorch 原生的全分片数据并行框架，相比 FSDP1 进行了大量架构改进。FSDP2 采用了更细粒度的分片策略和更灵活的通信调度，使得大模型训练更加高效。

FSDP2 与 FSDP1 的架构对比

FSDP2 不是 FSDP1 的简单升级，而是基于 DTensor 的全新设计。其核心变化包括参数管理、通信调度和混合并行支持等方面。

FSDP2 的参数分片与收集流程

FSDP2 将模型参数按 DTensor 的方式分片存储在不同 GPU 上。在前向传播时，通过 AllGather 收集完整参数；计算完成后立即释放非本 GPU 负责的参数分片。反向传播时再次 AllGather 收集参数，计算梯度后通过 ReduceScatter 分发梯度。

FSDP2 参数分片与 AllGather 流程（2 GPU）： 前向: GPU0: [shard_0] →AllGather→ [full_param] →compute→ release shard_1 GPU1: [shard_1] →AllGather→ [full_param] →compute→ release shard_0 反向: GPU0: [shard_0] →AllGather→ [full_param] →backward→ ReduceScatter→ [grad_0] GPU1: [shard_1] →AllGather→ [full_param] →backward→ ReduceScatter→ [grad_1] 更新: GPU0: [shard_0] -= lr * [grad_0] GPU1: [shard_1] -= lr * [grad_1]
FSDP2 通信计算重叠示意： 时间 → Layer N: [AllGather][Compute ][RS ] Layer N+1: [AllGather][Compute][RS] ↑ 预取 Layer N 反向: [AG][Backward ][RS ] Layer N-1 反向: [AG][Backward][RS] ↑ 预取 通信节省: 40-60% (取决于计算/通信比) 配置: forward_prefetch=True, backward_prefetch=True
