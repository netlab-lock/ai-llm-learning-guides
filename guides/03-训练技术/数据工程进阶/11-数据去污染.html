1. 什么是数据污染

数据污染（Data Contamination）是指测试集或评估数据被包含在训练数据中，导致模型性能评估结果虚高。这是当前LLM评估领域最严重且最难完全解决的问题之一。当训练语料中混入了MMLU、GSM8K、HumanEval等主流benchmark的题目时，模型并非真正"学会"了解题能力，而是"记住了"答案。这种记忆效应会导致研究者和产业界对模型能力的严重误判。

数据污染的来源非常多样。最直接的来源是Common Crawl——作为最大规模的网页爬取数据集，它不可避免地包含了各种benchmark题目。此外，GitHub上的开源代码仓库中也经常包含benchmark的解答代码，论坛（如Reddit、Stack Overflow）上的讨论也会泄露评估数据。即使经过精心的数据清洗，这些问题仍然难以完全避免。

⚠️ 污染的严重性

Shi et al.（2023）的研究发现，GPT-3.5在MMLU测试集中约有10%的样本存在训练数据污染嫌疑。即使是看似微小的污染率（1-2%），也可能使评估结果提升5-15个百分点，严重误导模型能力的判断。更令人担忧的是，这种污染在闭源模型中几乎无法被外部审计。

1.1 污染的分类体系

根据污染程度和检测难度，可以将数据污染分为以下几个层次：

1.2 污染的量化指标

为了科学地度量污染程度，研究者提出了多种量化指标：

  • 污染率（Contamination Rate）：测试集中被污染样本的比例，是最直观的指标
  • 性能偏移（Performance Shift）：去除疑似污染样本前后，模型在benchmark上得分的变化量
  • 记忆分数（Memorization Score）：模型能够完整复述训练数据中特定片段的能力指标
  • 困惑度差异（Perplexity Gap）：模型对已知vs未知数据的困惑度差异，差异越大说明记忆效应越强
💡 实用技巧：在实际项目中，建议对所有用于评估的benchmark都建立污染检测流水线。具体做法是将benchmark题目作为"探针"，在训练数据中进行n-gram检索，记录每个benchmark的污染率。污染率超过5%的benchmark应当从评估结果中标注，或者替换为更新的、未被污染的评估集。
⚠️ 注意事项：污染率低不等于没有污染。部分污染（如只有题目没有答案）虽然难以检测，但模型仍可能从相关解题模式中学到"捷径"。因此，除了量化污染率，还需要分析模型在疑似污染样本上的行为模式（如是否异常快速地给出正确答案）。
🔍 深入理解：数据污染问题本质上揭示了当前LLM评估范式的根本缺陷——我们用固定benchmark来衡量不断进化的模型，而训练数据和评估数据来自同一个互联网语料池。这种"裁判员和运动员来自同一支队伍"的结构性矛盾，使得污染问题几乎不可能被完全解决。真正的解决方向可能需要转向动态评估（如实时更新的benchmark）或能力测试（如探针测试）。

2. 污染检测方法总览

目前业界已经发展出多种污染检测方法，每种方法在检测精度、计算成本和适用场景上各有不同。选择合适的检测方法需要综合考虑数据规模、污染类型和可用计算资源。

💡 实用技巧：在实际操作中，推荐采用"漏斗式"检测策略：先用精确匹配快速过滤掉明显的精确污染（计算成本极低），再用n-gram重叠检测近似污染，最后对高价值benchmark使用嵌入相似度或成员推理做深度检测。这种分层策略可以在保证检测率的同时控制计算成本。

3. n-gram重叠方法

n-gram重叠是最常用的污染检测方法，也是OpenAI在GPT-3和GPT-4中采用的核心去污染策略。其基本思想是计算测试集样本与训练集文档之间的n-gram重叠率。如果一条测试数据的大部分n-gram都能在训练数据中找到匹配，则认为该样本可能已被污染。

3.1 n-gram的基本原理

n-gram是指文本中连续n个词（或字符）构成的序列。给定一个长度为L的文本，可以提取出L-n+1个n-gram。例如，对于文本"大语言模型是强大的"（假设按字切分），其3-gram包括："大语言"、"语言模"、"言模型"、"模型是"、"型是强"、"是强大"、"强大的"。

n-gram重叠检测算法实现：

from collections import Counter
import re

def extract_ngrams(text, n=13):
    """提取文本的所有n-gram"""
    tokens = list(text)  # 字符级n-gram
    ngrams = []
    for i in range(len(tokens) - n + 1):
        ngrams.append(''.join(tokens[i:i+n]))
    return set(ngrams)

def compute_overlap(test_ngrams, train_ngrams):
    """计算两个n-gram集合的重叠率"""
    if not test_ngrams:
        return 0.0
    overlap = test_ngrams & train_ngrams
    return len(overlap) / len(test_ngrams)

def detect_contamination(test_doc, train_docs, n=13, threshold=0.8):
    """检测test_doc是否被训练数据污染"""
    test_ngrams = extract_ngrams(test_doc, n)
    contaminated = False
    max_overlap = 0.0
    
    for train_doc in train_docs:
        train_ngrams = extract_ngrams(train_doc, n)
        overlap = compute_overlap(test_ngrams, train_ngrams)
        max_overlap = max(max_overlap, overlap)
        if overlap >= threshold:
            contaminated = True
            break
    
    return contaminated, max_overlap

3.2 n值的选择与权衡

n值的选择是n-gram方法中最关键的超参数。不同的n值会导致截然不同的检测效果：

n值n-gram空间大小误报率漏报率计算成本典型应用
1-3极小极高低极低不推荐
5-8小高中低快速初筛
10-13中等低低中标准检测（GPT-3/4选择13）
20-30大极低中高高精确检测长文本
💡 GPT-4的去污染实践

OpenAI在GPT-4技术报告中描述了其去污染流程：使用13-gram匹配检测训练集与所有已知benchmark的重叠，移除匹配率超过50%的训练文档。此外，还对新型benchmark进行持续监控，在发布前移除可能污染的训练数据。值得注意的是，GPT-4的去污染不仅限于英文benchmark，还包括多语言评估集的翻译版本。

3.3 高效实现：MinHash近似

在大规模数据集上进行精确n-gram匹配的计算成本过高。MinHash是一种高效的近似方法，通过将n-gram集合压缩为固定长度的签名向量，实现快速的相似度估计。具体实现可以参考本系列的05-去重与MinHash章节。

基于MinHash的快速去污染：

from datasketch import MinHash, MinHashLSH

def build_minhash_index(train_docs, n=13, num_perm=128):
    """为训练集构建MinHash索引"""
    lsh = MinHashLSH(threshold=0.5, num_perm=num_perm)
    for idx, doc in enumerate(train_docs):
        m = MinHash(num_perm=num_perm)
        for ngram in extract_ngrams(doc, n):
            m.update(ngram.encode('utf-8'))
        lsh.insert(str(idx), m)
    return lsh

def query_contamination(lsh, test_doc, n=13, num_perm=128):
    """查询测试文档是否与训练数据存在重叠"""
    m = MinHash(num_perm=num_perm)
    for ngram in extract_ngrams(test_doc, n):
        m.update(ngram.encode('utf-8'))
    results = lsh.query(m)  # 返回相似度超阈值的文档ID
    return len(results) > 0, len(results)
⚠️ 注意事项：n-gram方法对文本的预处理非常敏感。标点符号、空格、Unicode编码的差异都可能导致漏检。建议在检测前统一进行文本规范化：去除多余空白、统一Unicode编码（NFC/NFKC）、统一数字和标点格式。此外，对于代码类benchmark，还需要考虑缩进和注释的标准化处理。
🔍 深入理解：n-gram重叠方法的核心假设是"如果测试数据在训练集中出现过，那么它们共享足够多的连续片段"。但这个假设在以下情况下可能失效：(1)训练数据经过深度改写但保留了语义；(2)测试数据是通过模板生成的，训练集中存在大量同模板不同内容的数据；(3)跨语言污染，翻译版本的n-gram完全不同。这些边界情况需要结合嵌入相似度等语义级方法来补充检测。

4. 成员推理攻击

成员推理（Membership Inference）是一种从模型行为推断某条数据是否在训练集中的技术。其原理是：模型对训练数据的"记忆"会导致其在见过的数据上表现出不同的置信度分布。与n-gram方法不同，成员推理不依赖于训练数据的直接访问，而是通过分析模型的输出来间接判断。

4.1 基于Loss的推理方法

最经典的成员推理方法是基于模型loss的分析。直觉上，模型对训练数据的loss应该低于非训练数据，因为训练过程就是最小化训练数据上的loss。

成员推理攻击实现：

import torch
import numpy as np

def membership_inference_loss(model, tokenizer, text, threshold=2.0):
    """基于loss的成员推理"""
    inputs = tokenizer(text, return_tensors="pt", truncation=True)
    with torch.no_grad():
        outputs = model(**inputs, labels=inputs["input_ids"])
    loss = outputs.loss.item()
    
    # 与基准分布比较
    # 如果loss显著低于基准，则可能在训练集中
    return {
        "loss": loss,
        "is_member_likely": loss < threshold,
        "confidence": max(0, (threshold - loss) / threshold)
    }

def min_k_percent_prob(model, tokenizer, text, k=20):
    """Min-K% Prob方法：取概率最低的k%token的平均log概率"""
    inputs = tokenizer(text, return_tensors="pt", truncation=True)
    with torch.no_grad():
        outputs = model(**inputs)
        logits = outputs.logits
    
    # 计算每个token的log概率
    log_probs = torch.log_softmax(logits, dim=-1)
    token_log_probs = log_probs[0, :-1].gather(
        1, inputs["input_ids"][0, 1:].unsqueeze(1)
    ).squeeze()
    
    # 取最低k%的token
    k_count = max(1, int(len(token_log_probs) * k / 100))
    sorted_probs, _ = torch.sort(token_log_probs)
    min_k_avg = sorted_probs[:k_count].mean().item()
    
    return min_k_avg  # 值越小，越可能是成员

4.2 Reference Model方法

Reference Model方法通过对比目标模型和一个"干净"参考模型的行为差异来判断成员关系。该方法的核心洞察是：如果目标模型在某条数据上的loss显著低于参考模型，说明这条数据很可能在目标模型的训练集中。

Reference Model方法流程： 目标模型M_target 参考模型M_ref (可能被污染) (确认干净) │ │ ▼ ▼ ┌─────────┐ ┌─────────┐ │计算loss │ │计算loss │ │L_target │ │L_ref │ └────┬────┘ └────┬────┘ │ │ └───────────┬────────────┘ ▼ ┌──────────────┐ │ 计算差异 │ │ Δ = L_ref - │ │ L_target│ └───────┬──────┘ │ ┌──────┴──────┐ ▼ ▼ Δ > threshold Δ ≤ threshold │ │ ▼ ▼ 可能被污染 可能未污染
💡 实用技巧：选择参考模型时，应确保其训练数据与目标模型尽可能不同但模型架构和规模相似。例如，如果目标模型基于Common Crawl训练，可以选择一个主要基于学术论文训练的模型作为参考。此外，建议使用多个参考模型取平均，以减少单个参考模型的偏差。
⚠️ 注意事项：成员推理方法存在固有的局限性。首先，它只能给出概率性的判断，无法100%确定一条数据是否在训练集中。其次，对于高度"可预测"的文本（如标准公式、常见事实），即使是未见过的数据也可能有很低的loss，导致误判。最后，推理成本较高，需要对每条待检测数据运行一次完整的前向传播。

5. Canary字符串注入

Canary字符串是一种主动污染检测方法：在训练数据中故意插入特殊标记，之后通过检测模型是否"记住"这些标记来评估污染程度。与被动检测方法不同，Canary方法提供了对污染程度的精确控制和度量。

5.1 Canary设计原则

有效的Canary字符串需要满足以下设计原则：

设计原则说明示例
唯一性Canary不应自然出现在任何文本中使用特殊编码：如Base64编码的随机序列
可检测性模型必须能够被明确测试是否记住设计为问答对或数字序列
可控性可以调节出现频率以测试记忆阈值同一Canary在训练集中出现1/10/100次
不可猜测性不能通过上下文推断出Canary内容使用密码学安全的随机数生成器
📖 Google的Canary实验

Google在PaLM的训练中注入了特殊的数字序列（如特定格式的电话号码），然后在推理时测试模型是否能完整回忆这些序列。结果显示，高频出现的canary序列被记忆的概率显著更高。具体而言，出现100次的Canary序列被完整回忆的概率约为60%，而仅出现1次的序列回忆概率不到5%。这一实验不仅验证了污染检测的可行性，也揭示了模型记忆的频率依赖特性。

Canary注入与检测实现：

import random
import string
import hashlib

def generate_canary(seed, length=32):
    """生成确定性的Canary字符串"""
    random.seed(seed)
    chars = string.ascii_uppercase + string.digits
    canary = ''.join(random.choices(chars, k=length))
    return f"CANARY[{canary}]"

def inject_canaries(dataset, canary, frequency=10):
    """将Canary注入到训练数据中"""
    # 在数据集的随机位置插入Canary
    positions = random.sample(range(len(dataset)), 
                              min(frequency, len(dataset)))
    for pos in positions:
        # 将Canary嵌入到自然文本中
        dataset[pos] = dataset[pos] + f"\n记住这个编号：{canary}"
    return dataset

def detect_canary(model, tokenizer, canary, prompt="记住这个编号是："):
    """检测模型是否记住了Canary"""
    inputs = tokenizer(prompt, return_tensors="pt")
    outputs = model.generate(**inputs, max_new_tokens=len(canary)+10)
    generated = tokenizer.decode(outputs[0], skip_special_tokens=True)
    return canary in generated
🔍 深入理解：Canary方法的理论基础来自差分隐私（Differential Privacy）领域。从信息论的角度看，每条训练数据对模型参数的影响可以通过梯度来量化。Canary的注入频率与记忆概率之间的关系，实际上反映了模型容量与训练数据量之间的关系——当模型容量远大于训练数据量时，模型倾向于"记住"而非"泛化"。这也是为什么大模型（如175B参数的GPT-3）比小模型更容易受到数据污染的影响。

6. 嵌入相似度检测

嵌入相似度方法使用预训练的文本编码器（如Sentence-BERT、E5、BGE等）将文本映射到向量空间，然后通过向量相似度（如余弦相似度）来检测语义级别的污染。该方法的优势在于能够检测到经过改写、翻译或重组的污染数据。

6.1 实现流程

嵌入相似度去污染：

from sentence_transformers import SentenceTransformer
import numpy as np
import faiss

def build_embedding_index(train_docs, model_name="BAAI/bge-large-zh-v1.5"):
    """构建训练数据的向量索引"""
    model = SentenceTransformer(model_name)
    embeddings = model.encode(train_docs, show_progress_bar=True, 
                              normalize_embeddings=True)
    
    # 使用FAISS构建高效索引
    dim = embeddings.shape[1]
    index = faiss.IndexFlatIP(dim)  # 内积 = 余弦相似度(归一化后)
    index.add(embeddings.astype('float32'))
    return model, index

def detect_semantic_contamination(model, index, test_doc, threshold=0.92):
    """检测语义级别的污染"""
    test_emb = model.encode([test_doc], normalize_embeddings=True)
    scores, indices = index.search(test_emb.astype('float32'), k=5)
    
    max_score = scores[0][0]
    return {
        "is_contaminated": max_score >= threshold,
        "max_similarity": float(max_score),
        "top_matches": indices[0].tolist()
    }

6.2 阈值设定与校准

嵌入相似度的阈值设定是该方法的核心挑战。阈值过低会导致误报（将语义相似但非污染的数据标记为污染），阈值过高则会导致漏报。

相似度范围判断处理策略
≥ 0.98高度疑似精确污染直接移除
0.92 - 0.98疑似近似污染人工审核或自动改写
0.85 - 0.92可能的间接污染标注警告，评估时考虑
< 0.85大概率未污染保留
💡 实用技巧：阈值的校准应该使用已知的污染/非污染样本对来进行。具体做法是：手动构造一组"已知污染"（将测试集原文微调后放入训练集）和"已知非污染"样本，计算它们的相似度分布，然后选择使F1分数最大化的阈值。建议使用验证集上的ROC曲线来确定最优阈值。

# Dolma去污染流程示例
# Step 1: 准备benchmark数据
dolma decon prepare_benchmarks \
    --benchmarks mmlu,gsm8k,humaneval \
    --output benchmarks_ngrams.jsonl

# Step 2: 对训练数据进行去污染
dolma decon run \
    --documents "s3://data/train/*.jsonl.gz" \
    --benchmarks benchmarks_ngrams.jsonl \

    --output "s3://data/clean/*.jsonl.gz"

dolma decon report \
    --input "s3://data/clean/*.jsonl.gz" \
    --output contamination_report.html

8.2 去污染的频率与时机

去污染应该在数据管线的多个节点执行：

  • 预处理阶段：在数据清洗后、训练前执行完整的去污染扫描
  • 增量阶段：对新增的合成数据或补充数据执行增量去污染
  • 评估前：每次评估前重新确认训练集与评估集的重叠情况
  • 发布前：模型发布前进行最终的污染审计

🎯 练习题

1. 实现一个完整的13-gram重叠检测器：输入两个文本文件，输出重叠的n-gram数量、重叠率和疑似污染片段。要求支持中英文混合文本，并处理Unicode编码问题。

2. 分析实验：为什么选择13-gram而不是更短的n-gram（如5-gram）？构造一个包含100对已知污染/非污染样本的测试集，计算不同n值（5/8/10/13/20）下的精确率、召回率和F1分数，绘制n值与检测性能的关系曲线。

3. 设计一个完整的去污染pipeline：从CommonCrawl数据中移除所有与MMLU、GSM8K、HumanEval、ARC等主流benchmark的重叠内容。要求：(a)支持多语言检测；(b)生成详细的污染报告；(c)计算成本不超过100 GPU小时。

4. 对比实验：使用同一组测试数据，分别用n-gram方法和嵌入相似度方法检测污染，比较两种方法的检测结果差异。分析在什么场景下嵌入方法能检测到n-gram方法遗漏的污染。

9. 学习路径与进阶建议

数据去污染是数据工程中技术门槛较高的领域，建议按以下路径学习：

9.1 推荐学习顺序

  1. 基础阶段：理解n-gram原理和基本实现 → 实践精确匹配检测
  2. 进阶阶段：学习MinHash/LSH（参考05-去重与MinHash）→ 实现大规模近似检测
  3. 高级阶段：掌握嵌入相似度方法 → 研究成员推理攻击原理
  4. 工程阶段：设计工业级Pipeline → 集成到数据工程工具链（参考12 数据工程工具链）

💡 实用技巧：建议在学习过程中动手实现每种检测方法的最小可行版本，而不是仅仅理解理论。例如，先写一个10行的精确匹配脚本，再逐步扩展到支持n-gram、MinHash和嵌入相似度。实践中的细节（如文本预处理、内存优化、并行计算）往往比理论更重要。
🔍 深入理解：从更宏观的视角看，数据污染问题反映了AI领域的一个根本矛盾：我们用互联网上产生的数据来训练模型，又用同样来源的数据来评估模型。只要评估基准是静态的、公开的，污染就几乎不可避免。未来的解决方向可能包括：(1)动态benchmark，定期更新题目；(2)私有评估集，不公开评估数据；(3)能力测试，测试泛化能力而非记忆能力；(4)对抗性评估，使用对抗样本测试鲁棒性。这些方向都值得深入探索。
