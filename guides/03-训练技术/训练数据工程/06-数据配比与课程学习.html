⚖️ 06 · 数据配比与课程学习 — 数据的配方

Domain Weighting · DoReMi · Data Mixing Laws · 多阶段训练 · 各大模型配比策略

一、背景与动机

LLM 的训练数据来自多个"领域"(domain)：网页、代码、书籍、学术论文、百科等。这些数据源应该如何混合？网页占 60% 还是 80%？代码应该占 5% 还是 15%？

数据配比(Data Mixing)直接影响模型能力分布。网页比例高→通用能力强但代码差；代码比例高→代码能力强但通用知识弱。找到最优配比是数据工程中最关键的决策之一。

💡 2024-2025 关键发现：代码数据对推理能力有显著帮助(不仅提升代码能力)。各大模型的代码占比从 5% 提升到 15-17%。同时"过训练"(超出 Chinchilla 最优比)成为标准做法——用更多数据训练更小模型以降低推理成本。
⚠️ 注意事项：数据配比没有"万能公式"。Llama 3的配比（网页55%+代码17%）不一定适合你的场景。配比取决于：①目标能力（代码助手vs知识问答需要不同配比）；②模型规模（小模型需要更多代码数据提升推理）；③数据质量（高质量数据可以少占比例，低质量数据需要更多过滤）。务必在自己的场景下做消融实验。
🔍 深入理解：数据配比的核心矛盾：网页数据量大但质量低，高质量数据（代码/学术/书籍）量小但价值高。自然比例会导致模型"偏食"——过度学习网页的写作风格，而缺乏结构化推理能力。配比优化的本质是在"数据量"和"数据质量"之间找到帕累托最优前沿。代码数据的"意外收获"（不仅提升代码能力还提升推理能力）是2024年最重要的发现之一。

2.3 DoReMi: 自动学习配比

DoReMi 算法 (Xie et al. 2023): Step 1: 训练参考模型 用自然比例训练小型参考模型(280M) 记录每个领域的损失 L_ref(domain_i) Step 2: 训练代理模型+学习权重 代理模型(280M)在线学习领域权重 αᵢ 目标: 每个领域的损失都不能比参考模型差太多 min_α max_i [L_domain_i(α) - L_ref(domain_i)] ───────────────────────────────────────────── minimax: 最小化最差领域的超出量 Step 3: 用学到的权重训练大模型 DoReMi 学到的权重通常: ↑ 上采样: 代码、数学、学术(高质量) ↓ 下采样: 普通网页(低质量) 效果: 7B 模型用 DoReMi 配比比自然配比提升 2.6% MMLU
# DoReMi 配比优化的核心实现原理 (简化版)
import torch

def doremi_step(proxy_model, ref_losses, domain_weights, domain_data):
    """DoReMi 的单步权重更新机制"""
    # 1. 计算加权损失
    total_loss = 0
    domain_losses = {}
    for domain, data in domain_data.items():
        loss = proxy_model(data)  # 前向传播计算各域损失
        domain_losses[domain] = loss
        total_loss += domain_weights[domain] * loss

    # 2. Minimax 更新: 关注"最差领域"的超出量
    excess_losses = {
        d: domain_losses[d] - ref_losses[d]
        for d in domain_data
    }
    worst_domain = max(excess_losses, key=excess_losses.get)

    # 3. 用指数梯度上升更新权重
    # 最差领域的权重被上调(获得更多训练数据)
    lr = 0.01
    for domain in domain_weights:
        if domain == worst_domain:
            domain_weights[domain] *= torch.exp(lr * excess_losses[domain])
    # 归一化确保权重之和为1
    total = sum(domain_weights.values())
    domain_weights = {d: w/total for d, w in domain_weights.items()}

    return domain_weights, total_loss

2.4 Data Mixing Laws

Ye et al. (2024) 发现数据配比和损失之间存在幂律关系：

L(α) ≈ Σᵢ αᵢ^βᵢ · Lᵢ
其中 αᵢ 是领域 i 的比例，βᵢ 是领域特定的指数

这意味着可以通过小规模实验拟合 βᵢ，然后预测大规模的最优配比——避免了昂贵的大规模网格搜索。

2.5 代码数据的特殊作用

💡 实用技巧：数据配比的四步实验法：①用自然比例训练1-3B小模型作为baseline；②固定其他域，逐一调整每个域的比例（如代码5%→10%→15%→20%），做消融实验；③用DoReMi或Data Mixing Laws做联合优化；④在目标模型规模上验证最优配比。每个配比至少训练3次取平均以减少随机波动。
⚠️ 注意事项：不同规模模型的最优配比可能不同！小模型（1-3B）可能需要更多代码数据（15-20%）来提升推理能力，而大模型（70B+）可能只需10-12%就能获得相同效果。在小模型上通过DoReMi学到的配比不一定直接适用于大模型，建议在目标规模上做验证实验。

💡 实用技巧：课程学习的三阶段实践：Phase 1（预训练）用大规模混合数据+余弦衰减学习率，消耗80-90%计算量；Phase 2（退火）切换到高质量数据+低学习率（原始的10%），消耗5-10%计算量，可提升1-2%性能；Phase 3（长上下文）用长文档数据扩展窗口（8K→128K），消耗5%计算量。Llama 3和DeepSeek-V3都采用此策略。阶段切换时机通过小规模实验确定。
⚠️ 注意事项：退火阶段的数据质量至关重要：如果退火数据中混入低质量内容，不仅不能提升模型上限，反而可能导致"灾难性遗忘"——模型丢失预训练阶段学到的知识。建议退火数据只使用经过严格筛选的高质量子集（如教科书、学术论文、高质量代码）。长上下文扩展阶段同理，长文档中的噪声会严重影响长文本处理能力。
🔍 深入理解：课程学习的理论基础是"易→难"的学习曲线与人类认知类似。但LLM的课程学习与传统ML不同：传统ML的课程学习关注样本的"难度"（损失值），LLM更关注数据的"质量"和"领域"。退火阶段的学习率降低是关键——低学习率让模型在高质量数据上做精细调整，而不会破坏预训练阶段学到的广泛知识。这种"粗调→精调"的范式类似于图像处理中的多尺度分析。

四、配比优化的实验方法

方法成本精度适用
网格搜索高(多次训练)高小规模
小模型代理中中DoReMi
Data Mixing Laws低中快速预测
在线学习中高DoReMi
💡 实用技巧：DoReMi实践要点：用280M小模型做代理即可获得可靠的配比指导（成本降低10倍+），核心是最小化最差域的超出量（minimax目标）。Data Mixing Laws实践要点：先在3-5种不同配比下训练小模型，拟合幂律参数βᵢ，然后外推预测大规模最优配比。网格搜索虽然成本高但最可靠，建议在资源允许时优先使用网格搜索确定范围，再用DoReMi做精细调优。
⚠️ 注意事项：配比优化中的"指标幻觉"：过度优化单一benchmark（如MMLU）可能导致"偏科"。例如大幅增加学术论文比例可能提升MMLU，但降低对话和创意写作能力。建议使用多维度评估：语言理解、推理、代码、知识、对话能力同时评估。此外，配比优化结果存在随机性，每个配比至少训练3次取平均。
🔍 深入理解：Data Mixing Laws的公式L(α)≈Σᵢαᵢ^βᵢ·Lᵢ揭示了配比与损失的幂律关系，但其核心假设是各域独立。实际上域间存在交互效应——代码数据和数学数据之间有正向交互（代码提升推理→数学也受益），这使得独立假设偏乐观。DoReMi的minimax优化更鲁棒，因为它不假设域独立，而是直接优化最差域的表现。两种方法互补：Laws提供理论指导，DoReMi提供实操优化。

五、使用场景

  • 预训练数据规划：确定各领域数据的最优配比
  • 持续预训练：注入新领域数据时的配比调整
  • 退火训练：训练末期切换到高质量数据
  • 长上下文扩展：扩展上下文窗口的数据策略
💡 实用技巧：不同场景的配比策略：预训练数据规划→参考Llama 3配比（网页55%+代码17%+书籍8%+学术5%+其他15%）作为起点；持续预训练→新领域数据占比10-30%（过高会灾难性遗忘）；退火训练→训练最后5-10%步数切换到100%高质量数据+学习率降至10%；长上下文扩展→长文档占比30-50%，逐步增加序列长度（8K→32K→128K）。
⚠️ 注意事项：持续预训练的"灾难性遗忘"风险：注入新领域数据时，如果新数据比例过高（>30%），模型可能丢失原有的通用能力。安全的做法是：新领域数据占10-20%，同时保留80-90%的原始混合数据。此外，新数据的学习率应低于原始训练学习率，避免大幅偏离已学到的参数。
🔍 深入理解："过训练"（超出Chinchilla最优比）已成为2024-2025年的行业标准。Chinchilla建议数据量:参数量=20:1，但Llama 3 8B用了15T tokens（比例约1875:1），远超最优比。原因：推理成本远高于训练成本——用更多数据训练更小模型，可以在相同推理预算下获得更好的效果。这改变了配比优化的目标函数：不再是"给定计算预算最大化性能"，而是"给定推理预算最大化性能"。

import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
    elapsed = (time.perf_counter() - start) / iters
    print(f"Latency: {elapsed*1000:.2f}ms")
    return elapsed

💡 实用技巧：实际效果对比要点：DoReMi配比比自然配比提升2.6% MMLU（7B模型），Llama 3的人工优化配比提升约5% MMLU（8B模型），退火训练额外提升1-2%。配比优化的投入产出比很高——相同的计算预算下，好的配比可以带来相当于"免费"增加30%数据量的效果。建议先用DoReMi快速获得初始配比，再在该基础上做人工微调。
⚠️ 注意事项：配比优化实验的常见错误：①在太小的模型上实验（<1B的配比结论不可靠）；②训练步数不足（配比差异需要足够多的训练步数才能显现）；③评估不全面（只看loss不看下游任务）；④没有控制变量（同时改配比和学习率）。建议使用1-3B模型、至少训练100B tokens、在5个以上benchmark上评估。
🔍 深入理解：配比优化的核心哲学是"数据即模型能力的配方"。每个域的数据对应一种能力维度：网页→通用知识和语言能力，代码→逻辑推理和编程，学术→专业知识和严谨表达，书籍→长文本理解和叙事能力。最优配比不是固定的——它取决于你想要什么样的模型。面向代码助手的模型需要更高代码比例，面向知识问答的模型需要更高学术和百科比例。没有"万能配比"，只有"最适合目标的配比"。

数据配比的优化是一个高度实验驱动的过程。最直接的方法是网格搜索，但成本很高。DoReMi提出了一种更高效的自动优化方法，核心思想是用小代理模型在线学习最优的域权重，优化目标是最小化最差域的超出量。DoReMi学到的权重通常会大幅上采样高质量数据，大幅下采样低质量数据。Data Mixing Laws发现了数据配比与损失之间的幂律关系，通过小规模实验拟合参数，可以预测大规模训练的最优配比。

多阶段训练策略是2024-2025年的主流做法。Llama 3采用了三阶段训练：预训练阶段使用15T tokens的混合数据，以余弦衰减的学习率训练；退火阶段切换到高质量数据，学习率降低到原始的10%；长上下文扩展阶段使用长文档数据，将上下文窗口从8K扩展到128K。每个阶段的数据配比和学习率都需要精心设计，阶段之间的切换时机也需要通过实验确定。代码数据对推理能力的提升是一个非常有趣的研究发现，增加代码数据比例不仅提升了编程能力，还显著增强了逻辑推理和数学能力。

DoReMi的核心公式是min_alpha max_i L_domain_i(alpha)减去L_ref(domain_i)，这个minimax目标确保没有任何一个域的性能比参考模型差太多。DoReMi学到的权重通常会大幅上采样高质量数据如代码和学术，大幅下采样普通网页数据。效果方面，7B模型用DoReMi配比比自然配比提升2.6%的MMLU分数。代码数据对推理能力的提升是一个非常有趣的研究发现，Llama 3将代码占比从8%提升到17%，结果发现代码能力大幅提升而通用能力不降反升。

Data Mixing Laws的公式表明每个域的损失与其配比之间存在幂律关系，通过小规模实验拟合参数可以预测大规模训练的最优配比。多阶段训练中退火阶段的效果尤为显著，使用高质量数据和低学习率可以在预训练基础上进一步提升1到2个百分点的性能。长上下文扩展阶段需要注意数据的质量，长文档中如果包含大量噪声反而会降低模型的长文本处理能力。建议使用经过精心筛选的长文档数据，如书籍、学术论文和高质量的网页长文。在实际的配比优化实验中，建议每个配比至少训练3次取平均以减少随机波动的影响，同时使用多维度评估指标确保配比优化不会导致偏科。不同规模的模型可能需要不同的最优配比，小模型可能需要更多的代码数据来提升推理能力，而大模型可能只需要较少的代码数据就能获得足够的推理能力。

七、学习路径与配比优化实践

数据配比是数据工程中最需要实验验证的环节。掌握配比方法后，下一步是学习去污染技术，保护评测的公正性。

数据配比优化实验流程: ┌──────────────────────────────────────────────────────┐ │ 数据配比优化实验方法论 │ ├──────────────────────────────────────────────────────┤ │ │ │ 阶段1: 基线实验 │ │ ┌────────────────────────────────────────────┐ │ │ │ 用自然比例训练小模型(1-3B)作为基线 │ │ │ │ 评估各能力维度: 语言/推理/代码/知识 │ │ │ └────────────────────────────────────────────┘ │ │ │ │ │ 阶段2: 消融实验 │ │ ┌────────────────────────────────────────────┐ │ │ │ 固定其他域, 逐一调整每个域的比例 │ │ │ │ 代码: 5%→10%→15%→20% │ │ │ │ 学术: 2%→5%→8%→12% │ │ │ │ 记录每个配比下的评估结果 │ │ │ └────────────────────────────────────────────┘ │ │ │ │ │ 阶段3: 联合优化 │ │ ┌────────────────────────────────────────────┐ │ │ │ 方法A: 网格搜索 (成本高但精确) │ │ │ │ 方法B: DoReMi 自动优化 (成本中) │ │ │ │ 方法C: Data Mixing Laws 预测 (成本低) │ │ │ └────────────────────────────────────────────┘ │ │ │ │ │ 阶段4: 验证 │ │ ┌────────────────────────────────────────────┐ │ │ │ 用最优配比训练大模型(7B+) │ │ │ │ 全面评估各能力维度 │ │ │ │ 与基线对比, 确认提升 │ │ │ └────────────────────────────────────────────┘ │ └──────────────────────────────────────────────────────┘

数据配比的优化是一个高度实验驱动的过程。虽然DoReMi和Data Mixing Laws等方法可以提供初始指导，但最终的最优配比仍然需要在目标模型规模上进行验证。一个重要但常被忽视的因素是：不同规模的模型可能需要不同的最优配比。小模型（1-3B）可能需要更多的代码数据来提升推理能力，而大模型（70B+）可能只需要较少的代码数据就能获得足够的推理能力。因此，在小模型上通过DoReMi学到的配比不一定适用于大模型。

多阶段训练策略是2024-2025年的主流做法。典型的三阶段训练包括：Phase 1（预训练阶段）使用大规模混合数据，以较高的学习率训练，目标是学习通用知识和语言能力，通常消耗总计算量的80-90%；Phase 2（退火阶段）切换到高质量数据，以较低的学习率（原始的10%）训练，目标是提升模型的上限性能，通常消耗总计算量的5-10%；Phase 3（长上下文扩展阶段）使用长文档数据，扩展模型的上下文窗口，通常消耗总计算量的5%。Llama 3和DeepSeek-V3都采用了类似的多阶段策略。

代码数据对推理能力的提升是一个非常有趣的研究发现。多项实验表明，增加代码数据的训练比例不仅提升了模型的代码生成能力，还显著提升了数学推理、逻辑推理等看似无关的能力。一种解释是代码具有严格的逻辑结构（条件判断、循环、函数调用等），学习代码本质上是在学习结构化的逻辑推理。另一种解释是代码中的注释和文档字符串天然形成了"指令-响应"对，有助于提升模型的指令遵循能力。无论哪种解释，15-17%的代码配比已经成为2024年后的行业标准。

💡 实践建议：下一步继续学习去污染与评测泄漏模块。同时建议在小规模实验中尝试不同的数据配比：准备网页、代码、学术三种数据各100K条，分别以不同的比例混合（如80:10:10、60:20:20、40:30:30），用小模型训练后比较评估结果，直观感受配比对模型能力的影响。
⚠️ 配比优化陷阱：不要只关注单一指标。一个常见的错误是过度优化某个特定benchmark（如MMLU）的分数，而忽视了其他能力维度。例如，大幅增加学术论文比例可能提升MMLU分数，但可能降低模型的对话能力和创意写作能力。建议使用多维度评估指标，确保配比优化不会导致"偏科"。此外，配比优化的结果往往存在随机性，建议每个配比至少训练3次取平均，以减少随机波动的影响。
🔬 深入理解：Data Mixing Laws揭示了数据配比与模型损失之间的幂律关系，这为配比优化提供了理论基础。其核心公式L(α)≈Σᵢαᵢ^βᵢ·Lᵢ表明，每个领域的损失与其配比之间存在幂律关系，指数βᵢ是领域特定的。通过小规模实验拟合βᵢ，可以在不进行大规模训练的情况下预测最优配比。但需要注意的是，这个公式假设各领域之间是独立的，忽略了领域间的交互效应。在实践中，代码数据和数学数据之间可能存在正向交互（代码提升推理能力），这使得独立假设不一定成立。

配比优化的工程实践还需要考虑数据的质量分层，同一领域内的数据质量差异可能很大例如网页数据中既有高质量的百科文章也有低质量的广告页面，一种有效的策略是将每个领域的数据按质量分为高中低三层然后为不同质量层分配不同的采样权重，高质量数据可以被上采样重复训练多次低质量数据可以被下采样或完全丢弃，这种质量分层的配比策略比简单的领域级配比更加精细可以进一步提升模型性能，Llama 3就采用了类似的质量分层策略在退火阶段只使用最高质量的数据子集。

配比优化的实验设计需要特别注意控制变量，每次只改变一个域的比例同时保持其他域的比例不变，这样才能准确评估该域对模型能力的影响，如果同时改变多个域的比例将无法确定是哪个域的变化带来了效果变化，此外实验的随机性也需要控制建议固定所有随机种子包括数据打乱顺序模型初始化权重等。

配比优化的实验记录应该包含完整的训练配置包括学习率批量大小训练步数数据配比等信息，这些记录对于后续的消融实验和结果复现至关重要，建议使用实验追踪工具如Weights and Biases或MLflow来自动化记录和管理实验结果。

配比优化的结果需要在多个评测维度上验证确保模型不会出现偏科现象，单一指标的提升可能掩盖其他能力的下降。

配比优化是一个需要持续迭代的过程，随着模型规模和训练目标的变化最优配比也会相应调整，建议建立标准化的配比评估流程以便快速验证新的配比方案。

配比优化的网格搜索虽然成本高但结果最可靠，建议在资源允许的情况下优先使用网格搜索方法来确定最优配比，然后用DoReMi等自动方法做进一步微调，这种组合策略可以在成本和效果之间取得最佳平衡。

配比优化的实验成本可以通过使用更小的代理模型来降低，通常1B到3B参数的模型就可以提供足够可靠的配比指导而不需要在7B或更大模型上进行完整的网格搜索，这种代理策略可以将实验成本降低十倍以上同时保持配比选择的准确性。

配比优化的最终验证需要在目标模型规模上进行，小模型上学到的最优配比不一定完全适用于大模型，但在大多数情况下可以提供很好的初始配比作为大模型训练的起点，建议在小模型配比的基础上进行小幅调整来适配大模型的特殊需求。

课程学习的核心理念是从简单到复杂逐步安排训练数据，这一方法在实践中已被证明可以提升训练效率和最终模型性能。

持续学习和迭代优化是数据配比工程的核心方法论。

def evaluate_method_applicability(scenario_constraints):
    """
    评估技术方法在不同场景下的适用性
    
    Parameters:
    scenario_constraints: dict, 各场景的约束条件
    
    Returns:
    适用性评分和建议
    """
    dimensions = ["latency", "power", "area", "cost", "complexity"]
    
    for scenario, constraints in scenario_constraints.items():
        score = 0
        suggestions = []
        
        # 延迟约束
        if constraints.get("latency_budget_ms", 100) < 10:
            score += 20
        elif constraints.get("latency_budget_ms", 100) < 100:
            score += 15
        else:
            score += 10
        
        # 功耗约束
        if constraints.get("power_budget_w", 100) < 5:
            score += 20
            suggestions.append("⚠️ 极低功耗场景需特别注意泄漏电流")
        elif constraints.get("power_budget_w", 100) < 75:
            score += 15
        else:
            score += 10
        
        # 面积约束
        if constraints.get("area_budget_mm2", 500) < 50:
            score += 20
        elif constraints.get("area_budget_mm2", 500) < 200:
            score += 15
        else:
            score += 10
        
        print(f"场景: {scenario}")
        print(f"  适用性评分: {score}/60")
        if suggestions:
            print(f"  建议: {suggestions[0]}")
        print()
