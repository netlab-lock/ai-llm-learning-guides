⚖️ 02 · 权重平均方法

权重平均是最简单、最直接的模型融合方法。它直接在参数空间中对多个模型的权重进行加权求和，得到融合后的模型。本章详细介绍三种主要的权重平均方法：Simple Averaging、SLERP 和 Weighted Averaging，并分析它们的数学原理、适用场景和实际效果。

1. Simple Averaging

1.1 基本公式

θ_merged = (1/N) · Σ θ_i = (θ_1 + θ_2 + ... + θ_N) / N

Simple Averaging（简单平均）是最基础的融合方法，将 N 个模型的权重逐元素取算术平均。其核心假设是：多个微调模型都在 Loss Landscape 的同一个平坦区域内，因此它们的平均值仍然位于低损失区域。

1.2 数学分析

从数学角度看，简单平均是一种特殊的加权平均，其中所有权重 w_i = 1/N。设 θ_pre 为预训练模型，τ_i = θ_i - θ_pre 为第 i 个模型的 Task Vector，则简单平均可以改写为：

θ_merged = θ_pre + (1/N) · Σ τ_i

这意味着简单平均等价于在预训练模型的基础上，叠加所有 Task Vector 的平均值。如果各个 Task Vector 编码的是不同的能力（如数学、代码、中文），那么平均后的结果会保留这些能力的"公共部分"。

1.3 实现思路

Simple Averaging 的实现非常简单，核心逻辑只有几步：加载所有源模型的权重字典，对每个参数键进行逐元素平均，保存合并后的权重。在 PyTorch 中，这可以通过 state_dict 的字典操作轻松实现。MergeKit 工具则提供了更方便的 YAML 配置驱动的融合流程。

1.4 优缺点分析

维度优点缺点
实现难度极低，几行代码即可—
计算成本几乎为零—
融合质量模型相似时效果好模型差异大时干扰严重
可扩展性支持任意数量模型模型越多干扰越大
理论基础有平坦区域假说支撑假设不一定总成立
实用建议：Simple Averaging 适合作为融合实验的 baseline。建议在尝试任何复杂方法之前，先用简单平均跑一遍，看看效果如何。如果简单平均已经能达到满意的效果，就不需要使用更复杂的方法了。
注意事项：当融合的模型数量超过 3-4 个时，简单平均的效果通常会开始下降。这是因为更多的模型意味着更大的参数差异，干扰效应会逐渐累积。建议先融合 2 个最相关的模型，观察效果后再考虑加入更多模型。
深入理解：从贝叶斯角度看，简单平均可以理解为对模型后验分布的一种近似。如果假设每个微调模型的后验分布是一个高斯分布，那么简单平均对应的是一种"均匀先验"下的模型集成。这种近似在模型后验分布重叠较大时是合理的，但在后验分布差异较大时可能不准确。

2. SLERP (Spherical Linear Interpolation)

2.1 基本原理

θ_merged = sin((1-t)·α)/sin(α) · θ_1 + sin(t·α)/sin(α) · θ_2 其中 α = arccos(θ_1 · θ_2 / (|θ_1| · |θ_2|))

SLERP（球面线性插值）最初用于计算机图形学中的旋转插值，后来被引入模型融合领域。与简单线性插值不同，SLERP 在高维球面上插值，保持权重向量的范数和方向关系。

2.2 几何直觉

SLERP vs 线性插值的几何对比: ╭─────────╮ ╱ ╲ ○ SLERP 结果 (在球面上) ╱ ○ SLERP ╲ ╱ ╱ ╲ ╲ ● 线性插值结果 (在球面内) │ ╱ ● ╲ │ │ ╱ Linear ╲ │ │╱ Interp ╲│ ○───────────────────○ θ_1 θ_2 在高维参数空间中: - 线性插值: 结果的范数变小，方向被"拉向"中间 - SLERP: 保持范数不变，只改变方向

在高维参数空间中（如 7B 模型有数十亿参数），这种差异可能对融合效果产生显著影响。SLERP 保持参数向量的范数，避免了线性插值可能导致的"权重衰减"问题。

2.3 参数 t 的选择

t 值含义适用场景
t = 0.3偏向模型 A (70%)模型 A 质量明显更高
t = 0.5等比例混合两个模型质量相当
t = 0.7偏向模型 B (70%)模型 B 质量明显更高
t = 0.0 或 1.0退化为单模型调试用

2.4 SLERP 的局限

标准 SLERP 只支持两个模型之间的插值。如果需要融合多个模型，可以通过嵌套使用来实现：先融合模型 A 和 B 得到 AB，再用 AB 与模型 C 融合。但嵌套顺序可能影响最终结果，这是一个已知的局限。

实用建议：SLERP 特别适合融合两个高质量模型的场景。在使用时，建议在 t ∈ [0.3, 0.7] 范围内以 0.1 为步长搜索最优 t 值。通常 t=0.5 是一个不错的默认选择。
常见误区：① 认为 SLERP 只能融合两个模型——确实，标准 SLERP 只支持两个模型，但可以通过嵌套使用来扩展；② 忽略权重选择的影响——等权平均不一定最优，性能加权通常效果更好。
深入理解：SLERP 在模型融合中的有效性暗示了一个深层事实：预训练模型的参数空间可能具有某种"球面结构"。这意味着不同微调模型的参数向量可能分布在一个高维球面上，而不是在一个平坦的欧几里得空间中。如果这个假设成立，那么在球面上插值（SLERP）自然比在平坦空间中插值更合理。

3. Weighted Averaging

3.1 基本公式

θ_merged = Σ w_i · θ_i, 其中 Σ w_i = 1

Weighted Averaging（加权平均）是 Simple Averaging 的推广，允许为每个模型分配不同的权重。权重的选择是加权平均的核心问题。

3.2 权重选择策略

策略权重公式优点缺点
等权w_i = 1/N简单，无需调参忽略模型质量差异
性能加权w_i ∝ perf_i考虑模型质量需要评估基准
温度缩放w_i ∝ exp(perf_i/T)可调节集中度需要选择温度 T
数据量加权w_i ∝ data_i考虑训练数据量数据量不等于质量
网格搜索在验证集上搜索最优理论上最优计算成本高

3.3 温度缩放详解

温度缩放是一种常用的权重调整策略。通过温度参数 T 控制权重的"集中度"：T→0 时退化为选择最佳模型，T→∞ 时退化为等权平均。

w_i = exp(perf_i / T) / Σ exp(perf_j / T) T → 0: w_best → 1, 其他 → 0 (选择最佳模型) T → ∞: w_i → 1/N (等权平均) T = 1: 标准 softmax 权重
实用建议：在实际应用中，性能加权通常比等权平均效果好 1-3 个百分点。如果你有可靠的评估基准，建议优先使用性能加权策略。如果没有评估基准，等权平均是一个合理的默认选择。
常见误区：最常见的错误是跳过 baseline 直接上复杂方案。没有 Simple Averaging 的 baseline 对比，你无法判断加权平均是否真的带来了提升。建议始终保留等权平均作为对照组。
深入理解：从信息论角度看，加权平均可以理解为一种"信息加权"操作。权重 w_i 反映了我们对第 i 个模型"信息质量"的信任度。这种视角与贝叶斯推理中的后验加权有深刻联系：在贝叶斯框架下，最优的模型组合权重应该正比于每个模型的后验概率。

5. 实践指南

在实际项目中选择融合方法时，建议遵循以下决策流程：

  1. 如果只需要融合 2 个模型，优先尝试 SLERP（t=0.5）
  2. 如果需要融合 3+ 个模型，使用加权平均
  3. 如果有评估基准，用性能加权；否则用等权平均
  4. 始终以 Simple Averaging 作为 baseline 对比
实用建议：在评估融合效果时，除了看整体指标，还要关注每个子任务的性能变化。有时候整体指标略有提升，但某些子任务可能严重退化。这种"跷跷板效应"是模型融合中常见的问题。
注意事项：融合 7B 以上的模型时，需要至少 16GB 的显存。如果没有 GPU，可以使用 CPU 进行融合，但速度会慢很多。MergeKit 工具支持在 Google Colab 上免费运行。
向前串联： Simple Averaging的知识将在后续的推理框架选型、性能基准测试、生产部署中反复应用。 建议结合实际框架动手验证本章所学内容。

5.2 常见失败模式与诊断

融合后可能出现多种失败模式。第一种是"能力退化"——融合后某个任务的性能明显下降。这通常是因为不同模型的参数变化方向冲突导致的。诊断方法是分别计算各源模型在目标任务上的性能，确认哪个源模型的贡献导致了退化。第二种是"输出退化"——模型产生重复文本、乱码或无意义输出。这通常是因为融合后的权重分布异常，导致解码过程不稳定。诊断方法是检查融合后权重的统计分布（均值、方差、最大值等）是否正常。

5.3 权重平均的数学性质

从数学角度深入分析权重平均的性质，可以帮助我们更好地理解其行为。设 θ₁ 和 θ₂ 是两个微调模型的权重，θ_pre 是预训练模型的权重。定义 τ₁ = θ₁ - θ_pre 和 τ₂ = θ₂ - θ_pre 为两个 Task Vector。则加权平均可以写为：

θ_merged = (1-t)·θ₁ + t·θ₂ = θ_pre + (1-t)·τ₁ + t·τ₂ 这意味着加权平均等价于在预训练模型上叠加 两个 Task Vector 的凸组合

这个分解揭示了一个重要事实：权重平均本质上是在 Task Vector 空间中的凸组合。由于凸组合保持在凸包内，如果两个 Task Vector 所在的区域是"低损失"的，那么它们的凸组合也应该在"低损失"区域内。这就是权重平均能够工作的数学基础。

进一步分析，考虑损失函数在融合点的二阶泰勒展开：

L(θ_merged) ≈ L(θ_pre) + ∇L·δ + (1/2)·δᵀ·H·δ 其中 δ = (1-t)·τ₁ + t·τ₂ 是融合带来的总偏移 H 是 Hessian 矩阵 如果 H 的最大特征值很小（平坦区域）， 则 L(θ_merged) ≈ L(θ_pre) + ∇L·δ 损失值主要由一阶项决定

这个分析表明，权重平均在"平坦区域"内效果良好，因为二阶项（曲率效应）的影响很小。但如果融合点接近损失景观的"陡峭"区域，二阶项可能变得显著，导致融合后损失值急剧增加。

SLERP 数学原理示意图: 在 N 维参数空间中，两个向量 θ₁ 和 θ₂ 定义了一个二维平面。 在这个平面上: θ₂ ○ ╱| ╱ | ╱ | sin(t·α) ╱ | SLERP ●───┤ ╱ ╲ | ╱ t·α| ╱ | ○───────┴ θ₁ cos(α) (原点) SLERP 在这个平面上沿着单位圆弧插值: θ(t) = sin((1-t)·α)/sin(α) · θ₁ + sin(t·α)/sin(α) · θ₂ 关键性质: |θ(t)| = 1 (始终在单位球面上) θ(0) = θ₁, θ(1) = θ₂ (端点正确) 当 t=0.5 时，θ 是 θ₁ 和 θ₂ 的"球面中点"
实用建议：在选择融合参数时，可以利用数学分析来缩小搜索范围。例如，如果已知两个模型的 Task Vector 的夹角很小（余弦相似度 > 0.9），那么 t=0.5 的简单平均就是一个很好的选择。如果夹角较大，则需要搜索更广泛的 t 值范围。
常见误区：忽略数值精度的影响。在实际实现中，权重通常以 FP16 或 BF16 存储，而融合计算可能需要 FP32 精度以避免数值问题。特别是在 SLERP 中，如果两个向量非常接近（α 很小），sin(α) 可能接近零，导致数值不稳定。此时需要特殊处理（如退化为线性插值）。
练习：推导 SLERP 在 α→0 时的极限形式。提示：利用 sin(x)/x → 1 (当 x→0) 的性质。证明当两个模型非常接近时，SLERP 退化为线性插值。

6.1 融合模型的量化与部署

融合后的模型可以像普通模型一样进行量化和部署。但在量化过程中，需要注意融合可能改变权重的分布特征。例如，简单平均会使权重分布变得更"平滑"（方差减小），而 SLERP 会保持原始的方差。这些变化可能影响量化的效果。

在实际部署中，推荐的量化流程是：首先对融合后的模型进行 FP16 推理测试，确认基本功能正常。然后使用 GPTQ 或 AWQ 等量化方法进行 4-bit 量化，在量化校准数据集上评估量化后的性能。如果量化后性能下降明显，可以尝试 8-bit 量化或混合精度量化。

对于推理框架的选择，vLLM 是当前最流行的开源推理框架，支持 PagedAttention 等优化技术，可以显著提高推理吞吐量。TensorRT-LLM 则提供了更底层的优化，适合对延迟要求极高的场景。llama.cpp 适合在 CPU 或消费级 GPU 上运行量化后的模型。

6.2 多模型融合的批量实验框架

在实际项目中，通常需要进行大量的融合实验来找到最优策略。建议建立一个自动化的批量实验框架，包括以下组件：(1) 实验配置生成器——根据预定义的参数范围自动生成融合配置；(2) 融合执行器——自动调用 MergeKit 执行融合；(3) 评估器——自动在标准基准上评估融合效果；(4) 结果收集器——自动收集和整理实验结果。

这种自动化框架可以大大加速融合实验的迭代速度。例如，如果需要在 5 个不同的 t 值和 3 种不同的模型组合上进行 SLERP 融合实验，总共需要 15 次实验。手动执行这些实验可能需要数天时间，而自动化框架可以在数小时内完成。

6.3 权重平均的完整实现代码

以下是使用 PyTorch 实现三种权重平均方法的完整代码：

import torch
import math

def simple_average(model_states):
    """简单平均融合"""
    merged = {}
    for key in model_states[0]:
        merged[key] = sum(s[key].float() for s in model_states) / len(model_states)
    return merged

def slerp(state1, state2, t=0.5):
    """SLERP 球面线性插值"""
    merged = {}
    for key in state1:
        v1 = state1[key].float().flatten()
        v2 = state2[key].float().flatten()
        # 计算夹角
        cos_angle = torch.dot(v1, v2) / (v1.norm() * v2.norm() + 1e-8)
        cos_angle = cos_angle.clamp(-1, 1)
        angle = torch.acos(cos_angle)
        if angle.abs() < 1e-6:
            # 夹角太小，退化为线性插值
            merged[key] = (1-t) * v1 + t * v2
        else:
            merged[key] = (torch.sin((1-t)*angle)/torch.sin(angle)) * v1 + \
                          (torch.sin(t*angle)/torch.sin(angle)) * v2
        merged[key] = merged[key].reshape(state1[key].shape)
    return merged

def weighted_average(model_states, weights):
    """加权平均融合"""
    assert abs(sum(weights) - 1.0) < 1e-6
    merged = {}
    for key in model_states[0]:
        merged[key] = sum(w * s[key].float() for w, s in zip(weights, model_states))
    return merged

权重平均方法的一个重要性质是线性性——简单平均和加权平均都是线性操作，满足叠加原理。这意味着多个简单平均的结果可以被合并为一个等效的加权平均。例如，先对模型 A 和 B 做等权平均得到 AB，再对 AB 和模型 C 做等权平均，等价于对 A、B、C 做权重为 (1/4, 1/4, 1/2) 的加权平均。这种线性性使得权重平均方法非常灵活，可以方便地进行增量融合。

在实际的工程实践中，权重平均的实现通常分为两个阶段：第一阶段是"加载与对齐"——将所有源模型的权重加载到内存中，并确保它们的参数键（parameter key）完全对齐。第二阶段是"计算与保存"——逐元素执行加权平均操作，然后将结果保存为标准的模型格式（如 Hugging Face 的 safetensors 格式）。对于大规模模型（如 70B 参数），整个过程可能需要数百 GB 的内存和数十分钟的计算时间。MergeKit 工具通过智能的内存管理（如逐层加载、CPU offload 等优化）大大降低了硬件要求。

值得注意的是，权重平均的效果与模型的训练方式密切相关。如果两个模型使用了不同的学习率调度器（如一个使用 cosine schedule，另一个使用 linear schedule），它们的权重可能位于不同的"训练轨迹"上，导致融合效果不佳。因此，在进行权重平均之前，最好确认源模型使用了相似的训练配置。此外，训练的随机性（如不同的随机种子）也会影响融合效果，但在大多数情况下这种影响较小。
