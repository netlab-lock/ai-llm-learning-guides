➕ 03 · Task Arithmetic

Task Arithmetic (Ilharco et al. 2023) 引入了一个优雅的概念：Task Vector——微调带来的"知识增量"可以用向量表示，支持加法和减法运算。这一发现深刻揭示了大模型能力的可分解性和可组合性。

1. Task Vector 的概念

1.1 定义

τ = θ_finetuned - θ_pretrained 含义: 微调过程中，模型权重的变化量 这个变化量编码了微调带来的"知识增量"

Task Vector 的定义极其简单：它就是微调后模型权重减去预训练模型权重。这个简单的减法操作却蕴含着深刻的意义——它将微调过程中的"知识学习"从绝对权重空间中解耦出来，变成了一个独立的、可操作的"增量向量"。

1.2 直觉理解

Task Vector 的直觉: ┌──────────────────────────────────────────────────────────────┐ │ 预训练模型 θ_pre: "通用语言能力" │ │ │ │ │ │ 微调 (数学数据) │ │ ▼ │ │ 微调模型 θ_finetuned: "通用 + 数学能力" │ │ │ │ Task Vector τ = θ_finetuned - θ_pre │ │ 含义: "数学能力的增量" │ │ │ │ 关键洞察: 这个增量可以被提取、存储、组合、缩放! │ └──────────────────────────────────────────────────────────────┘

1.3 为什么 Task Vector 有效

Task Vector 之所以能够独立编码特定能力，根本原因在于微调的局部性。研究表明，微调主要改变模型中与目标任务相关的参数子集，而大部分参数变化很小。这意味着 Task Vector 是"稀疏的"——大部分元素接近零，只有少量元素有显著变化。

另一个重要原因是预训练的锚定效应。由于所有微调都从同一个预训练模型出发，Task Vector 之间具有可比性——它们都在同一个"参考系"下定义。这使得不同任务的 Task Vector 可以直接进行加减运算。

1.4 Task Vector 的数学性质

性质描述实际意义
可加性τ_A + τ_B 可以组合两种能力多任务融合
可减性θ - τ 可以去除特定能力能力遗忘/安全编辑
可缩放λ·τ 可以调节能力强度精细控制
稀疏性大部分元素接近零可以压缩存储
可迁移同一任务的不同模型的 τ 相似跨模型知识迁移
实用建议：在计算 Task Vector 时，建议使用 FP32 精度。由于 Task Vector 是两个大数相减的结果，低精度（如 FP16）可能导致数值不稳定。计算完成后可以转换回 FP16 存储以节省空间。
注意事项：Task Vector 的计算需要确保预训练模型和微调模型使用完全相同的架构和 tokenizer。即使 tokenizer 有微小差异，也可能导致权重不对齐，使 Task Vector 失去意义。
融合的理论与局限的核心概念与技术分析： 融合的理论局限源于高维参数空间中的几何结构。设 N 个 Task Vector {δ₁, δ₂, ..., δₙ}，朴素融合 θ_base + λΣδᵢ 的风险在于：当 δᵢ 和 δⱼ 在同一参数上符号相反时，它们会相互抵消。从 PAC-Bayes 理论角度，融合后模型的泛化界取决于各任务参数变化的联合分布。实验表明，当融合任务数超过模型参数的有效维度时，性能会急剧下降。这一现象的数学本质是：在高维空间中，随机向量几乎正交，但任务向量之间的相关结构导致它们并非独立，使得简单的线性叠加不再是最优策略。

7.2 开放问题深度分析

问题一：融合的有效性预测。 目前，判断两个模型是否适合融合的唯一可靠方法是"试一试"。这显然效率低下。理想情况下，我们希望能够仅通过检查两个模型的权重（而不需要运行任何评估），就预测它们融合后的效果。一种可能的思路是分析 Task Vector 的"频谱特性"——如果两个 Task Vector 的频谱重叠很小，说明它们编码的信息互补性强，融合效果可能很好。另一种思路是利用 Hessian 矩阵的信息来估计融合后的损失值变化。这些方向目前都在积极探索中。

问题二：最优融合权重的理论确定。 目前确定最优融合权重的方法主要是经验性的——在验证集上搜索。这不仅效率低下，还可能导致过拟合到验证集。理论上的最优权重应该是什么？一种来自贝叶斯统计的观点是：最优权重应该正比于每个模型的后验概率。但在实践中，估计后验概率本身就是一个困难的问题。

问题三：融合的极限。 可以融合多少个模型？经验上，2-4 个模型的融合效果最好，超过这个数量通常会出现性能下降。但这可能不是理论上的极限——也许通过更好的融合方法（如 DARE+TIES），可以融合更多模型而不损失性能。确定这个理论极限是一个重要的开放问题。

7.3 融合失败的案例分析

理解融合失败的案例与理解融合成功的案例同样重要。以下是几个典型的失败模式：

案例一：跨预训练模型融合。 尝试融合从不同预训练模型出发的微调模型（如 Llama-2 和 Mistral-7B 的微调版本）。结果：融合后的模型产生无意义输出。原因：两个预训练模型的内部表示完全不同，参数没有对应关系。

案例二：过度微调模型的融合。 将经过数十个 epoch 微调的模型进行融合。结果：融合后性能严重退化。原因：过度微调使模型偏离了预训练时的平坦区域，两个模型可能位于不同的"尖锐最小值"中。

案例三：安全模型与通用模型融合。 将经过 RLHF 对齐的安全模型与未经对齐的通用能力模型融合。结果：安全对齐被部分破坏。原因：安全对齐的参数变化与通用能力的参数变化方向可能冲突。

实用建议：从失败案例中学习是提高融合技能的有效途径。建议建立一个"融合实验日志"，记录每次实验的配置、结果和分析。长期积累后，你会形成对"什么能融、什么不能融"的直觉判断。
安全警告：融合操作可能无意中破坏模型的安全对齐。在发布任何融合模型之前，必须进行全面的安全评估。建议建立一套标准化的安全检查流程，包括红队测试、自动化安全评估和人工审核。
终极思考：模型融合的理论研究还处于早期阶段。当前的理论解释（如平坦区域假说、线性模式连接）只是"必要条件"的近似，离"充分条件"还有很大距离。未来的研究可能会发现全新的理论框架，从根本上改变我们对模型融合的理解。也许有一天，我们能够像"配方"一样精确地描述如何融合任意两个模型——选择什么方法、用什么参数、预期什么效果。但在此之前，"实验为王"仍然是最可靠的指导原则。

8.1 融合与模型编辑的关系

模型融合和模型编辑（Model Editing）是两个密切相关的领域。模型编辑的目标是精确修改模型的特定知识（如将"埃菲尔铁塔在伦敦"改为"埃菲尔铁塔在巴黎"），而模型融合的目标是组合多个模型的能力。两者在操作层面有相似之处——都是直接修改模型的参数。

一个有趣的统一视角是：Task Arithmetic 中的"任务减法"可以看作一种特殊的模型编辑——通过减去编码了"错误知识"的 Task Vector 来修正模型的错误。反过来，模型编辑中的"定位-修改"范式也可以用于改善融合效果——先定位融合后出现错误的参数，再进行针对性修正。

这种融合与编辑的统一视角引出了一个实用的建议：在融合后发现模型有特定错误时，不要重新融合，而是使用模型编辑技术进行局部修正。这比重新融合更高效，也更精确。

8.2 融合的可逆性研究

一个前沿的研究问题是：能否从融合后的模型中"恢复"出源模型？如果可以，那么融合就是一种"无损"的知识压缩方法。如果不行，那么融合过程中必然有信息丢失。

从信息论角度看，融合是一种"有损"操作——将多个模型的信息压缩到一个模型中，必然会丢失部分信息。但在某些特殊情况下（如两个模型的任务完全正交），融合可能是近似"无损"的。确定什么条件下融合是"无损"的，是一个重要的理论问题。

初步研究表明，对于简单平均融合，如果已知融合权重和其中一个源模型，可以通过简单的代数运算恢复出另一个源模型。但对于更复杂的融合方法（如 DARE+TIES），恢复源模型的难度大大增加，可能需要额外的信息或约束。

8.3 跨模态融合的可能性

当前的模型融合技术主要应用于语言模型。一个自然的问题是：这些技术能否扩展到多模态模型（如视觉-语言模型）？初步研究表明，答案是肯定的——Task Arithmetic 可以用于融合不同多模态能力。例如，可以将一个擅长图像描述的模型与一个擅长视觉问答的模型进行融合。

多模态融合面临一些独特的挑战。首先，不同模态的编码器（如视觉编码器和文本编码器）通常有不同的架构，不能直接进行参数空间融合。其次，不同模态之间的对齐关系更复杂，简单的参数平均可能破坏模态间的对齐。这些挑战需要新的融合方法来解决。

8.4 融合效果评估代码

以下是评估融合效果的核心代码：

import torch
import numpy as np

def compute_cosine_similarity(state1, state2):
    """计算两个模型权重的余弦相似度"""
    vec1 = torch.cat([v.float().flatten() for v in state1.values()])
    vec2 = torch.cat([v.float().flatten() for v in state2.values()])
    return torch.cosine_similarity(vec1.unsqueeze(0), vec2.unsqueeze(0)).item()

def compute_task_vector_norm(task_vector):
    """计算 Task Vector 的 L2 范数"""
    total_norm = 0
    for key in task_vector:
        total_norm += task_vector[key].float().norm() ** 2
    return total_norm.sqrt().item()

def compute_task_vector_sparsity(task_vector):
    """计算 Task Vector 的稀疏度（接近零的参数比例）"""
    total = 0
    near_zero = 0
    threshold = 1e-6
    for key in task_vector:
        delta = task_vector[key].float()
        total += delta.numel()
        near_zero += (delta.abs() < threshold).sum().item()
    return near_zero / total

def evaluate_merge_compatibility(state_a, state_b, pretrained_state):
    """评估两个模型的融合兼容性"""
    tau_a = {k: state_a[k].float() - pretrained_state[k].float() for k in state_a}
    tau_b = {k: state_b[k].float() - pretrained_state[k].float() for k in state_b}
    
    cos_sim = compute_cosine_similarity(tau_a, tau_b)
    norm_a = compute_task_vector_norm(tau_a)
    norm_b = compute_task_vector_norm(tau_b)
    sparsity_a = compute_task_vector_sparsity(tau_a)
    sparsity_b = compute_task_vector_sparsity(tau_b)
    
    print(f"Task Vector 余弦相似度: {cos_sim:.4f}")
    print(f"Task Vector A 范数: {norm_a:.4f}")
    print(f"Task Vector B 范数: {norm_b:.4f}")
    print(f"Task Vector A 稀疏度: {sparsity_a:.4f}")
    print(f"Task Vector B 稀疏度: {sparsity_b:.4f}")
    
    if cos_sim > 0.5:
        print("兼容性: 高 - 推荐融合")
    elif cos_sim > 0.3:
        print("兼容性: 中 - 可以尝试，注意调参")
    else:
        print("兼容性: 低 - 融合风险较高")

模型融合的理论研究也与"神经网络的可塑性"(Plasticity) 研究有深刻联系。可塑性研究关注的是神经网络在学习新知识时保持旧知识的能力。模型融合可以看作是一种"事后可塑性"——在训练完成后的参数空间中实现知识的整合。理解可塑性的理论可能为融合提供新的理论工具。

另一个重要的理论联系是与"损失景观的拓扑结构"。近年来的研究发现，深度学习模型的损失景观具有复杂的拓扑结构，包括大量的鞍点、局部最小值和连接它们的"隧道"。模型融合的成功可能与这种拓扑结构有关——如果两个模型位于同一个"隧道"中，它们之间的线性路径就是低损失的。这种拓扑视角为理解融合提供了新的几何工具。

从实践角度看，模型融合的理论局限并不妨碍其在实际中的广泛应用。就像深度学习的理论基础至今仍不完善，但这不妨碍它在各个领域取得巨大成功。关键是在实践中积累经验，建立对"什么能融、什么不能融"的直觉判断。随着融合技术的不断发展和理论研究的深入，我们有理由相信，模型融合将成为构建下一代AI系统的核心技术之一。

def verify_linear_connectivity(model_a, model_b, eval_fn, steps=20):
    """验证两个模型之间的线性模式连接"""
    results = []
    for t in torch.linspace(0, 1, steps):
        # 线性插值
        merged = {}
        for key in model_a:
            merged[key] = (1-t) * model_a[key].float() + t * model_b[key].float()
        # 评估
        loss = eval_fn(merged)
        results.append((t.item(), loss))
        print(f"t={t:.2f}: loss={loss:.4f}")
    return results

以下是计算参数空间距离的辅助代码：

def compute_param_distance(state_a, state_b):
    """计算两个模型之间的参数空间距离"""
    l2_dist = 0
    cos_sim_num = 0
    cos_sim_den_a = 0
    cos_sim_den_b = 0
    for key in state_a:
        a = state_a[key].float().flatten()
        b = state_b[key].float().flatten()
        l2_dist += (a - b).norm() ** 2
        cos_sim_num += (a * b).sum()
        cos_sim_den_a += a.norm() ** 2
        cos_sim_den_b += b.norm() ** 2
    l2_dist = l2_dist.sqrt().item()
    cos_sim = (cos_sim_num / (cos_sim_den_a.sqrt() * cos_sim_den_b.sqrt())).item()
    print(f"L2 距离: {l2_dist:.4f}")
    print(f"余弦相似度: {cos_sim:.4f}")
