➕ 03 · Task Arithmetic

Task Arithmetic (Ilharco et al. 2023) 引入了一个优雅的概念：Task Vector——微调带来的"知识增量"可以用向量表示，支持加法和减法运算。这一发现深刻揭示了大模型能力的可分解性和可组合性。

1. Task Vector 的概念

1.1 定义

τ = θ_finetuned - θ_pretrained 含义: 微调过程中，模型权重的变化量 这个变化量编码了微调带来的"知识增量"

Task Vector 的定义极其简单：它就是微调后模型权重减去预训练模型权重。这个简单的减法操作却蕴含着深刻的意义——它将微调过程中的"知识学习"从绝对权重空间中解耦出来，变成了一个独立的、可操作的"增量向量"。

1.2 直觉理解

Task Vector 的直觉: ┌──────────────────────────────────────────────────────────────┐ │ 预训练模型 θ_pre: "通用语言能力" │ │ │ │ │ │ 微调 (数学数据) │ │ ▼ │ │ 微调模型 θ_finetuned: "通用 + 数学能力" │ │ │ │ Task Vector τ = θ_finetuned - θ_pre │ │ 含义: "数学能力的增量" │ │ │ │ 关键洞察: 这个增量可以被提取、存储、组合、缩放! │ └──────────────────────────────────────────────────────────────┘

1.3 为什么 Task Vector 有效

Task Vector 之所以能够独立编码特定能力，根本原因在于微调的局部性。研究表明，微调主要改变模型中与目标任务相关的参数子集，而大部分参数变化很小。这意味着 Task Vector 是"稀疏的"——大部分元素接近零，只有少量元素有显著变化。

另一个重要原因是预训练的锚定效应。由于所有微调都从同一个预训练模型出发，Task Vector 之间具有可比性——它们都在同一个"参考系"下定义。这使得不同任务的 Task Vector 可以直接进行加减运算。

1.4 Task Vector 的数学性质

性质描述实际意义
可加性τ_A + τ_B 可以组合两种能力多任务融合
可减性θ - τ 可以去除特定能力能力遗忘/安全编辑
可缩放λ·τ 可以调节能力强度精细控制
稀疏性大部分元素接近零可以压缩存储
可迁移同一任务的不同模型的 τ 相似跨模型知识迁移
实用建议：在计算 Task Vector 时，建议使用 FP32 精度。由于 Task Vector 是两个大数相减的结果，低精度（如 FP16）可能导致数值不稳定。计算完成后可以转换回 FP16 存储以节省空间。
注意事项：Task Vector 的计算需要确保预训练模型和微调模型使用完全相同的架构和 tokenizer。即使 tokenizer 有微小差异，也可能导致权重不对齐，使 Task Vector 失去意义。
DARE 与 TIES-Merging的核心概念与技术分析： DARE 方法的核心洞察是 Task Vector 中大部分参数变化幅度很小，对任务性能贡献有限但会干扰其他任务的参数。DARE 先以概率 p 随机丢弃这些参数，再用 1/(1-p) 缩放保留的参数以保持期望值不变。数学上，若 δ 为原始 Task Vector，DARE 后的 δ' = (m ⊙ δ) / (1-p)，其中 m 为 Bernoulli 随机掩码。TIES-Merging 则先将低于阈值的参数截断为零，然后通过多数投票确定每个参数的符号方向，只保留与多数方向一致的参数。这两种方法的计算开销主要在参数遍历和统计阶段，属于内存带宽受限的操作，实际融合过程本身是逐元素计算，计算量极小。

6. 学习建议与下一步

Task Arithmetic 是理解高级融合方法的基础。掌握了 Task Vector 的概念后，可以继续学习 DARE 和 TIES-Merging，这两种方法专门解决 Task Arithmetic 中的"干扰问题"。

5.1 DARE 的理论保证

DARE 方法有一个重要的理论保证：在一定条件下，DARE 融合的效果可以近似等于原始（未丢弃）的融合效果。这个理论保证基于以下数学分析：

设 δ 为原始 Task Vector，δ' 为 DARE 处理后的 Task Vector E[δ'_i] = E[m_i · δ_i / (1-p)] = δ_i · E[m_i] / (1-p) = δ_i · (1-p) / (1-p) = δ_i 结论: DARE 处理后的参数期望值等于原始值 这意味着平均来看，DARE 不会引入系统性偏差

但需要注意的是，虽然期望值保持不变，方差却增加了。具体来说：

Var[δ'_i] = δ_i² · p/(1-p) 当 p=0.9 时，方差放大了 9 倍 这意味着单次 DARE 的结果可能波动较大 需要多次运行取平均，或使用足够大的模型来"平均掉"波动

在实践中，由于大模型的参数量巨大（数十亿），单个参数的方差波动会被整体"平均掉"，因此 DARE 在大模型上的效果通常比小模型更稳定。

5.2 TIES 的阈值选择策略

TIES 的 Trim 步骤需要选择一个阈值 τ，决定保留多少参数。这个选择对融合效果有显著影响。以下是几种常用的阈值选择策略：

策略方法保留比例优缺点
绝对阈值去除 |δ| < τ 的参数取决于分布简单但不自适应
百分位阈值保留 top-k% 的参数固定 k%自适应，推荐
标准差阈值保留 |δ| > n·σ 的参数取决于分布统计意义明确
能量阈值保留累积能量的 top-k%动态理论最优但计算复杂

论文推荐使用百分位阈值，保留 top 20% 的参数（即去除绝对值最小的 80% 参数）。这个选择基于一个经验观察：微调过程中，大部分参数变化是"噪声"（绝对值很小且方向随机），只有少部分参数变化携带了真正的"知识"（绝对值较大且方向一致）。

5.3 DARE 与 Dropout 的区别

DARE 看起来很像 Dropout，但两者有本质区别。Dropout 是在训练过程中随机"关闭"一些神经元，目的是防止过拟合。DARE 则是在融合过程中随机"关闭"一些参数变化，目的是减少干扰。关键区别在于：

  • 时机不同：Dropout 在训练时使用，DARE 在融合时使用
  • 目的不同：Dropout 防止过拟合，DARE 减少融合干扰
  • 缩放不同：Dropout 通常在训练和推理时行为不同（训练时丢弃，推理时不丢弃），DARE 只在融合时丢弃并重缩放
  • 理论基础不同：Dropout 基于正则化理论，DARE 基于压缩感知理论

实用建议：在资源有限的情况下，建议优先使用 DARE 方法。它的计算成本与 Simple Merge 相当（只需要额外的随机采样和重缩放），但效果通常更好。TIES 的效果可能略优于 DARE，但计算成本更高（需要排序操作）。
注意事项：在使用 DARE 时，要注意随机种子的管理。如果需要可复现的结果，必须固定随机种子。如果需要探索不同的融合结果，可以使用不同的种子多次运行，选择效果最好的结果。

6.1 DARE 和 TIES 的最新进展

DARE 和 TIES 自提出以来，已经有了许多改进和变体。其中一个重要的改进方向是自适应丢弃率——根据每个参数的重要性自动调整丢弃概率，而不是使用统一的丢弃率。重要性高的参数（绝对值大的）保留概率更高，重要性低的参数保留概率更低。

另一个改进方向是层级 TIES——对模型的不同层使用不同的阈值。如前所述，底层和高层的 Task Vector 特征不同，使用统一的阈值可能不是最优的。层级 TIES 可以为每层独立选择最优阈值，从而提升整体融合效果。

还有一个有趣的方向是DARE 与其他方法的组合。除了 DARE+TIES 的组合外，还可以将 DARE 与 Permutation 对齐结合使用——先通过 Permutation 对齐解决参数的对称性问题，再用 DARE 减少残余干扰。这种三阶段的融合策略在某些场景下可以取得最优效果。

6.2 融合方法的选择决策树

融合方法选择决策树: 开始 │ ├─ 只融合 2 个模型? │ ├─ 是 → 模型非常相似? │ │ ├─ 是 → Simple Averaging │ │ └─ 否 → SLERP (t=0.5) │ └─ 否 → 融合 3+ 个模型 │ ├─ 需要精细控制? │ ├─ 是 → Task Arithmetic + TIES │ └─ 否 → 继续判断 │ ├─ 干扰严重? │ ├─ 是 → DARE + TIES 组合 │ └─ 否 → Simple Averaging 或 SLERP │ └─ 需要自动化? ├─ 是 → Evolutionary Merge └─ 否 → 手动调参

6.3 DARE 和 TIES 实现代码

以下是 DARE 和 TIES 的核心实现代码：

import torch

def dare_merge(task_vectors, drop_rate=0.9):
    """DARE: 随机丢弃 + 重缩放"""
    merged_tv = {}
    for key in task_vectors[0]:
        result = torch.zeros_like(task_vectors[0][key].float())
        for tv in task_vectors:
            delta = tv[key].float()
            # 随机丢弃
            mask = torch.bernoulli(torch.full_like(delta, 1 - drop_rate))
            # 重缩放
            delta_scaled = delta * mask / (1 - drop_rate)
            result += delta_scaled
        merged_tv[key] = result
    return merged_tv

def ties_trim(task_vector, threshold=0.1):
    """TIES Step 1: Trim - 去除绝对值小于阈值的参数"""
    trimmed = {}
    for key in task_vector:
        delta = task_vector[key].float()
        mask = (delta.abs() >= threshold).float()
        trimmed[key] = delta * mask
    return trimmed

def ties_elect_sign(task_vectors):
    """TIES Step 2: Elect Sign - 投票选方向"""
    sign_sum = {}
    for key in task_vectors[0]:
        sign_sum[key] = sum(tv[key].sign() for tv in task_vectors)
    consensus = {key: sign_sum[key].sign() for key in sign_sum}
    return consensus

def ties_disjoint_merge(task_vectors, consensus_sign):
    """TIES Step 3: Disjoint Merge"""
    merged = {}
    for key in task_vectors[0]:
        aligned = []
        for tv in task_vectors:
            mask = (tv[key].sign() == consensus_sign[key]).float()
            aligned.append(tv[key] * mask)
        # 只对方向一致的参数取平均
        count = sum((tv[key].sign() == consensus_sign[key]).float() for tv in task_vectors)
        count = count.clamp(min=1)
        merged[key] = sum(aligned) / count
    return merged

DARE 和 TIES 的成功也启发了其他领域的方法创新。例如，在联邦学习中，类似的"去噪后聚合"思路被用于处理不同客户端模型之间的差异。在多任务学习中，"选择性共享"的概念与 TIES 的 Trim 和 Elect Sign 步骤有异曲同工之妙。在模型压缩中，"重要参数选择"的方法与 TIES 的 Trim 步骤使用了相似的技术。

从工程实现角度看，DARE 和 TIES 的计算瓶颈不同。DARE 的主要计算开销在于随机数生成和掩码操作，这些操作可以高效地在 GPU 上并行执行。TIES 的主要计算开销在于排序操作（用于确定 Trim 的阈值）和符号统计（用于 Elect Sign），这些操作需要全局信息，不太容易并行化。因此，在大规模模型上，DARE 的实际运行速度通常比 TIES 快 2-3 倍。

在使用 DARE 和 TIES 时，还需要注意一些实现细节。首先，DARE 的随机种子管理很重要——如果使用不同的随机种子，融合结果可能差异较大。建议在正式评估时固定随机种子，在探索阶段使用不同种子。其次，TIES 的阈值 τ 需要根据 Task Vector 的分布动态调整——如果 Task Vector 的方差很大，阈值也应该相应增大。第三，DARE+TIES 的组合顺序也很重要——建议先执行 TIES 的 Trim 和 Elect Sign，再执行 DARE 的随机丢弃。
