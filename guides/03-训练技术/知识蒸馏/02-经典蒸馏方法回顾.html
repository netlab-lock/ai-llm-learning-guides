📚 02 · 经典蒸馏方法回顾

在深入 LLM 蒸馏之前，先回顾经典的知识蒸馏方法。这些方法为 LLM 蒸馏奠定了理论基础。

1. Hinton 蒸馏 (2015)

1.1 核心思想

Hinton et al. (2015) 的核心洞察：teacher 的软标签 (soft labels) 包含比硬标签 (hard labels) 更多的信息。

1.2 Hinton 蒸馏公式

Hinton 蒸馏公式

L = α · L_CE(y, p_s) + (1-α) · T² · KL(p_t^T, p_s^T)

其中:
L_CE = 交叉熵损失 (student 预测 vs 真实标签)
KL = KL 散度 (teacher 软标签 vs student 软标签)
T = 温度参数 (平滑概率分布)
α = 权重系数

1.3 温度 T 的作用

温度 T 控制 softmax 分布的平滑程度：

1.4 暗知识 (Dark Knowledge)

teacher 的软标签包含了"暗知识"——类别之间的相似性关系。例如：

  • 猫和狗的相似度高于猫和汽车
  • 这种信息在硬标签中丢失了
  • 蒸馏让 student 学到这些隐含的关系
实现要点：Hinton蒸馏的关键参数选择：温度T=2~5（推荐从T=3开始），权重α=0.5~0.9（软标签权重越高，暗知识迁移越多）。建议先在小数据集上对比不同T值下的KL散度曲线，选择KL下降最平稳的T值。注意loss中T²的缩放因子——它补偿了温度缩放带来的梯度缩小，遗漏此因子是初学者最常见的错误。
常见实现错误：① 忘记T²缩放因子——不乘T²会导致高温时梯度过小，Student几乎学不到知识；② KL散度的reduction模式错误——应使用'batchmean'而非'mean'，否则batch size变化会影响loss量级；③ Teacher未设为eval模式——Teacher的Dropout和BatchNorm在推理时应处于eval状态，否则输出不稳定。
数学本质：Hinton蒸馏的数学优雅之处在于将"暗知识"概念形式化。软标签中的非目标类别概率并非噪声，而是Teacher学到的类别间语义关系的编码。从贝叶斯视角看，软标签相当于Teacher对后验分布的完整估计，而硬标签只是最大后验估计（MAP）。蒸馏本质上是让Student学习Teacher的完整后验分布——这比点估计包含远更多的信息。

2. Feature-based 蒸馏

2.1 原理

除了匹配输出分布，还可以匹配中间层的特征表示：

L_feature = Σ_l ||f_l^teacher - g(f_l^student)||²

其中:
f_l^teacher = teacher 第 l 层的特征
g = 映射函数 (维度对齐)

2.2 Feature-based 蒸馏的图解

层对齐策略：Feature-based蒸馏的关键是层对齐。Teacher 12层→Student 4层时，常见对齐方式：均匀映射（T层{1,4,7,10}→S层{1,2,3,4}）或最后几层对齐。映射函数g通常使用单层线性层（维度不同时必须用）。建议先用MSE损失，效果不好再尝试余弦相似度损失。对齐层数不宜超过Student层数的一半，否则计算开销过大。
常见陷阱：① 维度不匹配——Teacher和Student的隐藏层维度通常不同（如1024 vs 768），必须用映射函数g做维度对齐，直接匹配未经对齐的特征会导致训练不稳定；② 对齐层过多——选择过多的对齐层会大幅增加计算开销而收益有限，建议选取3-5个关键层；③ 忽略特征归一化——不同层的特征尺度可能差异很大，建议对每层特征做L2归一化后再匹配。
理论动机：Feature-based蒸馏基于"表示学习"假说：模型的中间层特征编码了输入的不同层次语义信息。浅层关注表面模式（词法、句法），深层关注抽象概念（语义、逻辑）。通过匹配中间表示，Student不仅学到"答案是什么"，还学到"如何提取和组织信息"。这种方法在极端压缩比（>5x）下特别有效，因为单纯匹配输出层无法传递足够的中间知识。

3. DistilBERT

3.1 方法

DistilBERT (Sanh et al. 2019) 是 NLP 领域的经典蒸馏案例：

  • Teacher: BERT-base (110M 参数)
  • Student: DistilBERT (66M 参数, 6 层)
  • 蒸馏方法: 软标签 + 特征匹配 + 余弦相似度损失
  • 结果：保留 97% 性能，推理速度提升 60%

3.2 DistilBERT 的损失函数

L = α · L_CE + β · L_cos + γ · L_MLM

L_CE: 软标签交叉熵损失
L_cos: 隐藏层余弦相似度损失
L_MLM: 掩码语言模型损失
复现DistilBERT的要点：① 三种损失的权重比例对效果影响大，建议先只用KL+CE（α=0.5），效果稳定后再加入cosine损失；② Student初始化应从Teacher偶数层复制权重（12层→6层取第0,2,4,6,8,10层），比随机初始化效果好很多；③ 冻结Teacher参数（requires_grad=False），否则Teacher会被Student"带偏"；④ 学习率建议1e-5~5e-5，比预训练低一个量级。
易错点：① 误以为层数减半=速度翻倍——Transformer的计算复杂度与层数线性相关，但注意力计算是O(n²)的，实际加速约1.6x而非2x；② 忽略Tokenizer一致性——Teacher和Student必须使用相同的Tokenizer，否则软标签无法对齐；③ 混淆软标签CE和标准CE——蒸馏时的交叉熵是对Teacher软标签计算的，不是对ground truth计算的。
架构洞察：DistilBERT的成功揭示了一个重要洞察：在Transformer架构中，冗余主要集中在层数而非宽度。6层Student保留97%性能说明12层BERT的相邻层之间存在大量冗余信息。这与信息瓶颈理论一致——深层网络在训练中逐渐压缩信息，中间层可能已充分提取有用特征。DistilBERT通过"跳层"利用了这种冗余，这也启发了后续的层剪枝和层共享研究。

4. TinyBERT

4.1 方法

TinyBERT (Jiao et al. 2020) 更激进地蒸馏 BERT：

  • Teacher: BERT-base (12 层)
  • Student: TinyBERT-4 (4 层) 或 TinyBERT-6 (6 层)
  • 蒸馏内容: 注意力矩阵 + 隐藏层 + 预测层
  • 结果：TinyBERT-4 保留 96% 性能，体积缩小 7.5x

4.2 TinyBERT 的两阶段蒸馏

两阶段蒸馏要点：TinyBERT两阶段蒸馏核心：阶段1（通用蒸馏）在预训练数据上进行，学习通用语言表示，建议训练20-40个epoch；阶段2（任务蒸馏）在下游任务数据上进行，学习任务特定能力，建议训练3-10个epoch。注意力矩阵蒸馏使用MSE损失，需要对注意力头做归一化处理。两阶段不可颠倒——必须先通用再任务。
关键陷阱：① 两阶段顺序不可颠倒——必须先通用再任务，否则Student会过拟合到特定任务；② 注意力头对齐——如果Teacher和Student的头数不同（如12 vs 4），需要做头映射或选择性匹配中间层；③ 阶段2数据量不宜过大——否则会覆盖阶段1学到的通用知识，建议任务数据量不超过通用数据量的1/10。
教学哲学：TinyBERT的两阶段蒸馏体现了"先学语法再学应用"的教学哲学。阶段1让Student获得通用的语言理解能力（相当于"识字"），阶段2在此基础上学习特定任务的处理方式（相当于"应用"）。这种分层学习比端到端蒸馏更高效，因为它避免了通用能力和任务能力之间的干扰。从优化角度看，两阶段降低了参数空间的复杂度，使训练更容易收敛到好的解。

5. 经典方法在 LLM 上的局限

局限说明
无法访问 logits闭源模型 (GPT-4, Claude) 只提供 API，不暴露 logits
序列级蒸馏LLM 是自回归生成，不是分类，需要新的蒸馏方法
数据规模LLM 需要海量数据，经典方法的数据效率不够
能力多样性LLM 有多种能力 (推理、代码、数学)，需要针对性蒸馏
经典→LLM改造思路：将经典蒸馏方法用于LLM的核心改造：(1) logit蒸馏→序列级蒸馏（逐token计算KL散度，注意mask padding位置）；(2) 固定数据集→动态数据生成（利用Teacher生成无限训练数据，重点转向数据质量筛选）；(3) 准确率评估→多维评估（perplexity+下游benchmark+人工评测）。推荐使用Hugging Face Transformers库作为实现起点。
三大陷阱：① LLM词表有32K-128K个token，存储完整logit分布的成本比传统分类任务高1000倍+，必须使用Top-K或Top-P截断；② 自回归生成的序列长度不固定，无法使用固定维度的特征匹配，需要动态padding和masking；③ 经典方法的数据效率不够——LLM需要的训练数据量远超传统NLP任务，建议至少准备10K条高质量数据。
范式转移：经典方法到LLM蒸馏的演进反映了AI领域的通用范式转移：从"小数据+复杂算法"到"大数据+简单算法"。经典蒸馏方法在小模型上表现优异，但当模型规模增长到数十亿参数时，数据的规模和质量变得比算法精巧程度更重要。Chinchilla定律和Scaling Law都指向这一方向——Phi系列模型用精选数据训练小模型达到大模型性能，就是这一范式的最佳例证。

6. 经典方法到 LLM 蒸馏的演进

通俗类比：软标签就像"模糊的照片"——硬标签是黑白分明的轮廓，软标签是带有灰度层次的照片。模糊的照片虽然不如清晰照片精确，但它包含了更多的"上下文信息"（比如猫和虎斑猫的相似度）。DistilBERT 的成功证明：一个6层的"模糊版"BERT，通过学习12层BERT的软标签，可以保留97%的性能。
常见误区：① 认为经典蒸馏方法可以直接用于 LLM——Hinton 蒸馏需要完整的 logit 分布，但 LLM 的词表有 32K-128K 个 token，存储和传输 logit 的成本极高；② 忽略 Feature-based 蒸馏的局限性——不同架构的模型（如 GPT vs BERT）内部表示空间完全不同，无法直接匹配中间层特征。

7. 经典蒸馏方法的数学基础

7.1 KL 散度的推导

KL 散度（Kullback-Leibler Divergence）是知识蒸馏中最核心的数学工具。它衡量两个概率分布之间的"距离"，虽然严格来说 KL 散度不是对称的距离度量，但在蒸馏场景中，我们通常使用 KL(p_teacher || p_student) 的方向，这等价于最小化 Teacher 分布下的 Student 交叉熵。

KL(p || q) = Σ_i p(i) · log(p(i) / q(i)) = Σ_i p(i) · log(p(i)) - Σ_i p(i) · log(q(i)) = -H(p) + H(p, q) = H(p, q) - H(p) 其中: H(p) = -Σ p(i)·log(p(i)) (p 的熵) H(p,q) = -Σ p(i)·log(q(i)) (p 和 q 的交叉熵) 
# KL 散度计算示例
import torch
import torch.nn.functional as F

def compute_kl_divergence(teacher_logits, student_logits, T=2.0):
    """计算温度缩放后的 KL 散度"""
    # 温度缩放
    teacher_soft = F.softmax(teacher_logits / T, dim=-1)
    student_log_soft = F.log_softmax(student_logits / T, dim=-1)
    # KL 散度 (乘以 T² 补偿梯度缩放)
    kl_loss = F.kl_div(student_log_soft, teacher_soft,
                       reduction='batchmean') * (T ** 2)
    return kl_loss

# 使用示例
teacher_logits = torch.tensor([5.0, 3.0, 1.0, 0.0])
student_logits = torch.tensor([4.0, 3.5, 1.5, 0.5])
kl = compute_kl_divergence(teacher_logits, student_logits, T=2.0)
print(f"KL散度: {kl.item():.4f}")

7.2 交叉熵与 KL 散度的关系

KL 散度KL(p_t || p_s)分布差异度量匹配 Teacher 的输出分布
交叉熵H(p_t, p_s)编码 p_t 所需的额外比特KL + 常数 (Teacher 熵)
MSE||p_t - p_s||²欧氏距离直接匹配概率值

8. 经典方法的代码复现

8.1 Hinton 蒸馏完整实现

# Hinton 蒸馏完整实现
import torch
import torch.nn as nn
import torch.nn.functional as F

class HintonDistillation:
    def __init__(self, teacher, student, T=2.0, alpha=0.5):
        self.teacher = teacher
        self.student = student
        self.T = T
        self.alpha = alpha

    def compute_loss(self, input_ids, labels):
        # Teacher 前向 (不计算梯度)
        with torch.no_grad():
            teacher_logits = self.teacher(input_ids).logits
        # Student 前向
        student_logits = self.student(input_ids).logits
        # 软标签 KL 散度损失
        soft_loss = F.kl_div(
            F.log_softmax(student_logits / self.T, dim=-1),
            F.softmax(teacher_logits / self.T, dim=-1),
            reduction='batchmean'
        ) * (self.T ** 2)
        # 硬标签交叉熵损失
        hard_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            labels.view(-1)
        )
        # 总损失
        total = self.alpha * soft_loss + (1 - self.alpha) * hard_loss
        return total, soft_loss, hard_loss

9.2 DistilBERT 与 TinyBERT 的详细对比

DistilBERT 和 TinyBERT 是 NLP 领域两个最具影响力的蒸馏方法，它们代表了两种不同的蒸馏策略。DistilBERT 主要使用软标签蒸馏，只匹配 Teacher 和 Student 的输出层概率分布，实现简单但效果已经很好（保留 97% 性能）。TinyBERT 则采用了更激进的多层次蒸馏策略，不仅匹配输出层，还匹配注意力矩阵和隐藏层表示，实现了更深层的知识迁移。

从工程实践的角度来看，DistilBERT 的方法更容易实现和推广——只需要在训练时用 Teacher 的 logits 替代硬标签即可。而 TinyBERT 的方法需要修改模型架构以暴露中间层表示，并且需要设计合理的层对齐策略（Teacher 的 12 层如何映射到 Student 的 4 层）。然而，TinyBERT 的方法在极端压缩场景下（如 12 层压缩到 4 层）效果明显更好，因为它保留了更多的中间层知识。

两种方法的另一个关键区别在于训练阶段。DistilBERT 采用单阶段蒸馏，在预训练阶段直接进行蒸馏。TinyBERT 采用两阶段蒸馏：第一阶段在通用语料上进行通用蒸馏，学习基础的语言表示能力；第二阶段在任务特定数据上进行任务蒸馏，学习下游任务的特定能力。这种两阶段策略让 TinyBERT 在下游任务上的表现通常优于 DistilBERT。

# DistilBERT 风格蒸馏实现
class DistilBERTStyleDistillation:
    def __init__(self, teacher, student, T=2.0, alpha=0.5):
        self.teacher = teacher.eval()
        self.student = student
        self.T = T
        self.alpha = alpha
        # 冻结 Teacher
        for param in self.teacher.parameters():
            param.requires_grad = False

    def forward(self, input_ids, attention_mask, labels):
        # Teacher 前向传播
        with torch.no_grad():
            teacher_outputs = self.teacher(
                input_ids=input_ids,
                attention_mask=attention_mask
            )
            teacher_logits = teacher_outputs.logits

        # Student 前向传播
        student_outputs = self.student(
            input_ids=input_ids,
            attention_mask=attention_mask
        )
        student_logits = student_outputs.logits

        # 软标签损失 (KL散度)
        kd_loss = F.kl_div(
            F.log_softmax(student_logits / self.T, dim=-1),
            F.softmax(teacher_logits / self.T, dim=-1),
            reduction='batchmean'
        ) * (self.T ** 2)

        # 硬标签损失 (交叉熵)
        ce_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            labels.view(-1)
        )

        # 总损失
        total_loss = self.alpha * kd_loss + (1 - self.alpha) * ce_loss
        return total_loss
