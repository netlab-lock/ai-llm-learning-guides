1. 蒸馏实验设计

1.1 Teacher/Student 模型选择

维度考虑因素
Teacher能力越强越好；考虑 API 成本和速率限制
Student参数量通常是 teacher 的 1/5 ~ 1/10
架构同架构蒸馏效果最好

1.2 训练超参数

参数建议范围说明
学习率1e-5 ~ 5e-5比预训练低
温度 T1.0 ~ 5.0越高越平滑
数据量10K ~ 1M 条取决于质量
训练步数1K ~ 10K通常不需要太多
实验设计要点：蒸馏实验的核心决策：(1) Teacher/Student参数比建议1:5~1:10（如70B→7B或7B→1B），差距过大会导致容量鸿沟；(2) 同架构蒸馏效果最好（如Llama→Llama），跨架构蒸馏需要额外的特征对齐；(3) 学习率建议1e-5~5e-5（比预训练低一个量级），温度T从2.0起步，α从0.5起步；(4) 先用1K条数据做快速验证，确认方向正确后再扩展到10K-1M条。
设计陷阱：① 跳过baseline——蒸馏前必须评估Student的原始性能（无蒸馏SFT），否则无法量化蒸馏收益；② 盲目照搬参数——别人的最优配置（T、α、学习率）在你的数据和模型上可能完全不同，务必做消融实验；③ 评估指标单一——只看loss下降不够，必须在多个下游benchmark上评估，loss下降≠能力提升。
蒸馏实验设计的核心概念与技术分析： 蒸馏实验设计的技术要点包括：(1) Teacher-Student 配对选择——Teacher 模型的容量应显著大于 Student，但差距过大会导致 Student 难以学习。经验法则是参数量比在 5-20 倍之间效果最佳；(2) 数据构造——黑盒蒸馏需要从 Teacher 生成软标签或推理链，生成成本是主要开销。建议先用小样本测试生成质量，再批量生成；(3) 训练超参数——蒸馏温度 T 通常在 2-5 之间，损失权重 α 在 0.5-0.9 之间（偏向蒸馏损失）。学习率建议比直接微调小 2-5 倍以避免灾难性遗忘；(4) 评估方法——除标准 benchmark 外，还应在分布外数据上测试泛化能力，使用 perplexity、BLEU/ROUGE、人工评估等多维度指标。

案例启示：Gemma 2和Phi-3代表了两种不同的蒸馏哲学：Gemma 2采用"同架构压缩"策略（27B→9B→2.6B），通过渐进式蒸馏逐步缩小容量差距；Phi-3采用"数据驱动"策略（用精选合成数据训练小模型），通过极致的数据质量弥补模型规模的不足。两种策略各有优劣：同架构压缩保留率更高但需要Teacher权重，数据驱动更灵活但对数据质量要求极高。选择哪种策略取决于你的资源约束。

5. 蒸馏训练的完整代码流程

5.1 使用 Transformers 实现黑盒蒸馏

以下是一个完整的黑盒蒸馏训练流程示例，展示了如何使用 Hugging Face Transformers 库进行蒸馏训练。该流程包括数据准备、模型加载、训练循环和评估四个核心步骤。

# 黑盒蒸馏完整示例
import torch
from transformers import AutoTokenizer, AutoModelForCausalLM
from torch.optim import AdamW

# 1. 加载 Teacher 和 Student
teacher = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3-70B")
student = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3-8B")
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-3-8B")

# 2. 准备蒸馏数据
def prepare_distill_data(teacher, prompts, tokenizer):
    distill_dataset = []
    for prompt in prompts:
        inputs = tokenizer(prompt, return_tensors="pt")
        with torch.no_grad():
            output = teacher.generate(**inputs, max_new_tokens=512)
        response = tokenizer.decode(output[0], skip_special_tokens=True)
        distill_dataset.append({"input": prompt, "output": response})
    return distill_dataset

# 3. 训练循环
optimizer = AdamW(student.parameters(), lr=2e-5)
for epoch in range(3):

        outputs = student(**batch, labels=batch["input_ids"])

        loss.backward()
        optimizer.step()
        optimizer.zero_grad()

5.2 使用 Logit 蒸馏的代码实现

白盒蒸馏需要同时访问 Teacher 和 Student 的 logits，通过 KL 散度损失函数将 Teacher 的知识迁移到 Student。温度参数 T 在此过程中起到平滑概率分布的关键作用。

# Logit 蒸馏核心代码
import torch.nn.functional as F

def distillation_loss(student_logits, teacher_logits, labels, T=2.0, alpha=0.5):
    """计算蒸馏损失: KL散度 + 交叉熵"""
    # 软标签损失 (KL散度)
    soft_student = F.log_softmax(student_logits / T, dim=-1)
    soft_teacher = F.softmax(teacher_logits / T, dim=-1)
    kd_loss = F.kl_div(soft_student, soft_teacher, reduction='batchmean') * (T ** 2)
    # 硬标签损失 (交叉熵)
    ce_loss = F.cross_entropy(student_logits, labels)
    # 加权组合
    total_loss = alpha * kd_loss + (1 - alpha) * ce_loss
    return total_loss, kd_loss, ce_loss

5.3 GKD (On-policy) 蒸馏实现

GKD 蒸馏让 Student 自己生成文本，然后用 Teacher 的 logits 来指导学习。这种方法避免了分布偏移问题，通常比传统的 Off-policy 蒸馏效果更好。

# GKD On-policy 蒸馏
def gkd_training_step(student, teacher, input_ids, tokenizer, T=2.0):
    """GKD: Student 生成, Teacher 打分"""
    # 1. Student 自己生成文本 (on-policy)
    student_output = student.generate(input_ids, max_new_tokens=256,
                                       do_sample=True, temperature=0.7)
    # 2. Teacher 对 Student 的输出打分
    with torch.no_grad():
        teacher_logits = teacher(student_output).logits
    # 3. Student 在自己的输出上学习
    student_logits = student(student_output).logits
    # 4. 计算 KL 散度
    loss = F.kl_div(
        F.log_softmax(student_logits / T, dim=-1),
        F.softmax(teacher_logits / T, dim=-1),
        reduction='batchmean'
    ) * (T ** 2)
    return loss

6. 蒸馏监控与调试

6.1 训练过程监控指标

在蒸馏训练过程中，需要实时监控多个关键指标来确保训练正常进行。这些指标不仅包括传统的损失函数值，还包括 KL 散度、Teacher-Student 相似度等蒸馏特有的指标。

监控指标正常范围异常信号应对措施
总损失持续下降震荡或上升降低学习率
KL 散度逐渐减小不再下降增加温度 T
CE 损失稳定下降突然上升检查数据质量
温度梯度平稳梯度爆炸梯度裁剪

实用建议：蒸馏训练的最佳实践是从较小的学习率（如1e-5）开始，观察前500步的损失下降曲线。如果KL散度下降缓慢，可以尝试增大温度参数T到3-5；如果CE损失上升，说明Teacher的数据可能有噪声，需要检查数据质量。建议使用wandb或tensorboard进行实时监控。
常见陷阱：① 忽视评估时机——不要等到训练结束才评估，建议每500-1000步做一次中间评估，及时发现问题；② 数据泄漏——确保评估集完全没有出现在训练数据中，否则评估结果虚高，上线后效果会大打折扣；③ 过拟合——如果Student在训练集上loss持续下降但验证集开始上升，说明过拟合，需要提前停止或增加数据量。
深入理解：蒸馏训练中的一个关键洞察是"容量差距"（Capacity Gap）问题。当Teacher和Student的参数量差距过大时（如70B→3B），Student可能无法完全吸收Teacher的知识。解决方法包括：(1) 渐进式蒸馏（70B→13B→3B），(2) 使用中间大小的Teacher，(3) 调整温度参数T来控制知识的"粒度"。实验表明，T的选择与Teacher-Student的容量比密切相关——差距越大，T应该越高（更平滑的分布）。

超参数调优技巧：建议先固定温度T=2.0和α=0.5，只调学习率找到最佳范围。然后固定学习率，调T和α。使用网格搜索或贝叶斯优化来找到最优组合。记录每组实验的超参数和结果，方便后续复盘和选择最佳配置。
排错注意：如果蒸馏后Student在某些特定任务上表现异常差，很可能是数据分布不均衡导致的。检查蒸馏数据中各任务的占比，确保多样性。此外，注意检查Tokenizer的一致性——如果Teacher和Student使用不同的Tokenizer，需要额外的对齐步骤。
