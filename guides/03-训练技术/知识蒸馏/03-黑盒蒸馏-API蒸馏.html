📦 03 · 黑盒蒸馏 (API蒸馏)

黑盒蒸馏是 LLM 时代最常用的蒸馏方法。它只需要访问 teacher 的文本输出，不需要访问模型权重或 logits。

1. 黑盒蒸馏的场景

黑盒蒸馏入门建议：最佳入门路径：选择一个开源Teacher（如Llama 3 70B或Qwen 2 72B）+ 轻量Student（如Llama 3 8B），使用500-1000条高质量prompt作为种子数据，让Teacher生成详细回答后训练Student。先跑通端到端流程，验证Student确实在目标能力上有提升，再逐步扩展数据量和优化质量。不建议一开始就用商业API——成本高且有法律风险。
关键风险：① 法律风险——使用商业API（GPT-4）的输出训练竞品模型可能违反服务条款（OpenAI TOS明确禁止），建议使用开源Teacher模型；② 质量退化——Teacher在弱项领域（如数学推理）的输出质量直接决定Student的能力上限，蒸馏过程本身存在信息损失；③ 成本失控——大规模API调用的费用可能远超预期，务必提前估算并设置预算上限。
黑盒蒸馏（API 蒸馏）的核心概念与技术分析： 黑盒蒸馏的技术实现路径包括：(1) 数据生成——通过多样化的 prompt 模板从 Teacher API 获取高质量响应。关键在于 prompt 的多样性设计：使用不同难度、不同领域、不同风格的 prompt 以覆盖 Student 需要学习的能力分布。建议使用 Self-Instruct 或 Evol-Instruct 框架自动化 prompt 生成；(2) 数据过滤——Teacher 输出并非都适合用于训练。需要过滤低质量响应（过短、重复、包含幻觉）和敏感内容。可以使用规则过滤（长度、重复率）和模型过滤（用独立模型评估质量）结合；(3) 训练策略——使用标准 SFT 流程，但需要注意：Teacher 输出的格式和风格要与 Student 的目标一致；如果 Teacher 使用了思维链（CoT），Student 也需要学习推理过程而非仅学习答案。

# 数据质量过滤示例
import tiktoken

def filter_quality(data, min_tokens=50, max_tokens=2000, max_repeat=0.3):
    enc = tiktoken.get_encoding("cl100k_base")
    filtered = []
    for item in data:
        tokens = enc.encode(item["output"])
        # 长度过滤
        if len(tokens) < min_tokens or len(tokens) > max_tokens:
            continue
        # 重复度过滤
        text = item["output"]
        words = text.split()
        unique_ratio = len(set(words)) / max(len(words), 1)
        if unique_ratio < (1 - max_repeat):
            continue
        filtered.append(item)
    return filtered

7.3 数据增强与多样性保证

蒸馏数据的多样性直接影响 Student 模型的泛化能力。如果蒸馏数据过于集中在某几个领域或风格，Student 会在这些领域表现很好，但在其他领域严重退化。这种现象被称为"蒸馏版灾难性遗忘"。因此，在生成蒸馏数据时，必须有意识地保证数据的多样性。

一个有效的多样性保证策略是使用"主题矩阵"。首先定义一个二维矩阵，一维是任务类型（推理、创作、分析、翻译、对话等），另一维是领域（科学、历史、技术、文学、商业等）。然后在每个交叉点生成一定数量的数据，确保所有重要组合都有覆盖。这种方法虽然看起来简单，但在实践中非常有效——Phi 系列模型就使用了类似的策略来保证数据的全面覆盖。

除了主题多样性，还需要注意难度多样性。太简单的数据无法让 Student 学到有价值的知识，太难的数据可能超出 Student 的能力范围，导致学习效率低下。建议将数据分为简单、中等、困难三个级别，分别占 30%、50%、20%。对于困难数据，可以要求 Teacher 提供更详细的推理过程（通过特殊的 Prompt 设计），让 Student 能够理解解题的每一步。

# 多样性检查工具
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.cluster import KMeans

def check_diversity(texts, n_clusters=10):
    """检查蒸馏数据的多样性"""
    # TF-IDF 向量化
    vectorizer = TfidfVectorizer(max_features=5000)
    tfidf_matrix = vectorizer.fit_transform(texts)
    # K-Means 聚类
    kmeans = KMeans(n_clusters=n_clusters, random_state=42)
    clusters = kmeans.fit_predict(tfidf_matrix)
    # 计算每个簇的大小
    cluster_sizes = [sum(clusters == i) for i in range(n_clusters)]
    # 多样性得分 (越均匀越好)
    expected = len(texts) / n_clusters
    diversity_score = 1 - sum(abs(s - expected) for s in cluster_sizes) / (2 * len(texts))
    return diversity_score, cluster_sizes

# 使用示例
texts = [item["output"] for item in distill_data]
score, sizes = check_diversity(texts)
print(f"多样性得分: {score:.2f}")  # 越接近 1 越好
print(f"各簇大小: {sizes}")

7.4 蒸馏数据的评估与筛选

生成蒸馏数据后，需要进行系统化的评估和筛选，确保只有高质量的数据被用于训练 Student。评估维度包括内容质量（准确性、完整性、逻辑性）、格式质量（结构清晰、无语法错误）和多样性（覆盖不同任务和难度级别）。一个常用的策略是使用多个评估指标的加权组合来计算每条数据的"蒸馏价值"，然后按照价值排序，选择价值最高的数据子集用于训练。

数据筛选的一个关键原则是"宁缺毋滥"。研究表明，10 万条精心筛选的高质量数据通常比 100 万条未筛选的数据效果更好。这是因为低质量数据不仅无法提供有价值的学习信号，还可能引入噪声，干扰 Student 的学习过程。Phi 系列模型的成功就是这一原则的最佳证明——他们使用 GPT-4 生成并精心筛选的"教科书级"数据，让 3.8B 参数的模型达到了 10 倍大小模型的性能水平。

在实际操作中，可以使用以下策略来提升数据筛选效率。首先使用自动过滤器（如长度检查、重复度检查、安全检查）快速排除明显不合格的数据。然后使用 Teacher 自身作为评估器——让 Teacher 对自己的输出进行自我评估和改进，只保留改进后的版本。最后，可以使用人工抽样检查来验证自动筛选的质量，确保筛选标准的合理性。

质量评分多指标加权淘汰 20-30%筛选高质量数据
多样性筛选聚类 + 采样保留多样性保证覆盖面

8.2 黑盒蒸馏的规模化实践

在工业级的黑盒蒸馏项目中，规模化是一个核心挑战。从实验室的几百条数据到生产环境的百万条数据，需要解决数据生成、质量控制、训练效率和成本管理等一系列工程问题。以下是一个经过验证的规模化蒸馏实践方案，已在多个商业项目中成功应用。

第一阶段是数据生成的规模化。使用异步并发 API 调用可以将数据生成速度提升 10-50 倍。建议使用 Python 的 asyncio 库配合 aiohttp 进行并发调用，同时实现完善的重试机制和速率限制处理。对于大规模数据生成，可以使用分布式任务队列（如 Celery）将任务分配到多个 worker 并行执行。在成本控制方面，建议对不同难度的任务使用不同级别的 Teacher——简单任务使用 GPT-3.5-turbo，复杂任务才使用 GPT-4。

第二阶段是质量控制的规模化。人工审核在小规模下可行，但在大规模下必须依赖自动化。建议构建一个多层过滤管线：第一层使用规则过滤（长度、格式、安全性），第二层使用自动评分模型（如训练一个质量评估分类器），第三层使用 Teacher 自评（让 Teacher 对自己的输出进行改进和评分）。只有通过所有三层过滤的数据才会被用于训练。这种管线可以将数据处理速度提升到每天处理 10 万条以上。

第三阶段是训练的规模化。大规模蒸馏训练需要使用分布式训练框架（如 DeepSpeed、FSDP）来处理大模型和大数据。建议使用梯度累积来模拟大 batch size，使用混合精度训练来降低显存需求。对于超大规模的训练，可以使用检查点恢复机制来应对训练中断。在训练过程中，建议定期保存中间模型并进行快速评估，以便及时发现和纠正问题。

# 异步 API 调用示例
import asyncio
import aiohttp

async def call_api_async(session, prompt, semaphore):
    async with semaphore:
        async with session.post(
            "https://api.openai.com/v1/chat/completions",
            json={"model": "gpt-4", "messages": [{"role": "user", "content": prompt}]},
            headers={"Authorization": f"Bearer {API_KEY}"}
        ) as resp:
            data = await resp.json()
            return data["choices"][0]["message"]["content"]

async def batch_generate(prompts, max_concurrent=50):
    semaphore = asyncio.Semaphore(max_concurrent)
    async with aiohttp.ClientSession() as session:
        tasks = [call_api_async(session, p, semaphore) for p in prompts]
        results = await asyncio.gather(*tasks)
    return results
