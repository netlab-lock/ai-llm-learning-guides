✨ 05 · 涌现能力与相变

中级 理论争议 前沿研究

1. 什么是涌现能力（Emergent Ability）？

Scaling Law 描述的是平滑、可预测的性能提升。但研究者发现，某些能力不是平滑出现的——它们在模型达到某个规模阈值之前几乎不存在，一旦跨过阈值就突然「涌现」出来。这一现象是 Scaling Law 体系中最令人兴奋（也最具争议）的发现之一。

涌现能力 = 在小模型中不存在 + 在大模型中突然出现

涌现能力的概念最早由 Google Research 的 Jason Wei 团队在 2022 年的论文中系统性地提出。他们发现，在 BIG-Bench（Beyond the Imitation Game Benchmark，包含 200+ 个多样化的语言任务）上，某些任务的性能曲线呈现出典型的「阶梯状」模式——小模型的性能接近随机猜测，而当模型规模跨过某个阈值后，性能突然跃升到远超随机的水平。

普通能力 vs 涌现能力： 普通能力（平滑提升）： 涌现能力（突然出现）： 性能 性能 │ ····· │ ····· │ ··· │ · │ ·· │ · │ · │ · │· │ ·········· └──────────────→ └──────────────→ 模型规模 模型规模 ↑ 涌现阈值 关键区别： - 普通能力：随规模增大平滑提升，可以用 Scaling Law 精确预测 - 涌现能力：在阈值前几乎不存在，跨过阈值后突然出现 - 涌现能力的存在使得 Scaling Law 的外推变得不可靠

💡 通俗类比：涌现能力就像烧开水——水温从 20°C 升到 99°C，水的外观几乎没有变化（只是温度升高）。但当温度跨过 100°C 这个阈值时，水突然沸腾——出现了一个全新的现象（气泡、蒸汽）。模型参数量就像水温，某些能力就像沸腾——在达到阈值之前看不出迹象，一旦达到就突然展现。详见 模块 01 对 Scaling Law 基本概念的介绍。

4. 涌现的物理解释

相变类比

涌现能力类似于物理学中的相变（Phase Transition）。在物理学中，相变是指物质在某个临界点附近突然改变状态的现象——水在 0°C 结冰、在 100°C 沸腾。这些变化不是渐进的，而是突然的、质的变化。

水的相变 vs 模型的涌现： 水： 模型： 温度 -1°C → 温度 0°C → 温度 +1°C 参数 10B → 参数 50B → 参数 100B 冰 冰→水 水 不能推理 临界点 能推理！ │ │ │ │ │ │ 固态 相变 液态 无能力 突变 有能力 物理相变的关键特征： 模型涌现的关键特征： - 存在明确的临界温度 - 存在明确的参数阈值 - 宏观性质的突变 - 任务性能的突变 - 微观结构的重组 - 参数表示的重组 - 可用序参量描述 - 可用任务准确率描述

为什么会出现涌现？

目前有几种主要假说，每种都有其合理性和局限性：

假说一：组合能力假说

涌现能力是多种基础能力的组合。只有当每种基础能力都达到一定水平时，组合才能成功。例如，多步数学推理需要：（1）理解题意的能力，（2）分解问题的能力，（3）执行单步计算的能力，（4）整合结果的能力。如果每一步的成功率是 80%，四步都成功的概率就是 0.8⁴ ≈ 41%；如果每步的成功率提升到 95%，四步都成功的概率跃升到 0.95⁴ ≈ 81%。这种非线性的组合效应可以产生看似突然的性能跃升。

假说二：相空间假说

模型在高维参数空间中的「有效维度」随规模增长而增加，达到某个阈值后解锁新的能力区域。这类似于物理学中的相空间理论——系统在相空间中的可达区域随能量增大而扩展。当模型参数量增大时，其在参数空间中的「探索能力」增强，能够到达之前无法到达的「能力区域」。

假说三：任务分解假说

复杂任务需要多个子步骤，只有当每步的成功率都超过某个阈值时，整体成功率才会从 ~0 跳到显著水平。这可以用数学公式来描述：

🔬 深入探讨：涌现的数学模型

假设一个任务需要 k 个子步骤，每步的成功率为 p(N)——取决于模型规模 N。在最简单的情况下，整体成功率为：

P_total(N) = ∏ᵢ₌₁ᵏ pᵢ(N)

如果每步的 p(N) 都遵循幂律关系（如 模块 02 所述），例如 p(N) = 1 - (N₀/N)^α，那么：

  • 当 N 较小时，p(N) 接近 0，P_total ≈ 0（每步都几乎不可能成功）
  • 当 N 增大时，p(N) 缓慢增加，但 P_total 仍然接近 0（因为乘积效应）
  • 当 N 超过某个阈值时，每步的 p(N) 都足够高，P_total 突然从 ~0 跳到显著水平

这种「乘积效应」是涌现能力突然出现的数学机制。具体来说，如果 k=5 个子步骤，每步的成功率从 0.5 提升到 0.9，整体成功率从 0.5⁵ = 3.1% 跳到 0.9⁵ = 59.0%——这是一个 19 倍的跃升，尽管每步只提升了 80%。

论文 "Emergent Abilities of Large Language Models" (Wei et al., 2022) 的分析表明，BIG-Bench 上的涌现任务通常包含 3-8 个子步骤，这与上述数学模型的预测一致。当模型的单步能力从「勉强可用」提升到「基本可靠」时，多步组合的成功率就会发生质变。

⚠️ 常见误区：很多人认为涌现意味着「模型在大到一定程度后突然学到了新东西」。但更准确的理解是：模型在小规模时已经「知道」了各个子步骤，只是每个子步骤的成功率太低，无法组合成完整的能力。当规模增大后，每个子步骤的可靠性都提高了，组合起来就产生了「突然出现」的效果。涌现不是「无中生有」，而是「从不可用到可用」的质变。

5. 争议：涌现能力是海市蜃楼吗？

2023 年，Schaeffer 等人发表了 "Are Emergent Abilities a Mirage?"，提出了尖锐的质疑：

⚠️ 核心论点：涌现能力可能是度量标准的假象（metric artifact），而不是模型的内在特性。当你改变评估方式时，「突然涌现」的阶梯状曲线会变成平滑的渐变曲线。

论证过程

精确匹配 (Exact Match)： 连续度量 (Brier Score)： 性能 性能 │ ██ │ ····· │ ██ │ ·· │ ██ │ ·· │ ██ ██ │ · │ ██ ██ ██ │ · │ ██ ██ ██ │· └──────────────→ └──────────────→ 模型规模 模型规模 看起来「突然涌现」 看起来「平滑提升」 （阶梯状跳跃） （连续曲线） 度量标准的选择决定了你看到的「现实」 这就像用二进制 vs 十进制看温度变化—— 前者看起来是突变，后者是渐变

解释：

  • 当使用精确匹配（Exact Match，答案必须完全正确才得分）时，小模型的得分是 0，大模型突然跳到高分——看起来像涌现
  • 当使用连续度量（如 Brier Score，部分正确也得分）时，性能是平滑提升的——没有涌现
  • 这就像用「及格/不及格」看成绩 vs 用「百分制」看成绩——前者看起来是突变，后者是渐变

学术界的回应

立场观点代表论据

🔬 深入探讨：度量标准的哲学问题

这场争论的核心实际上是一个科学哲学问题：「我们观察到的现象是客观存在的，还是我们观察方式的产物？」在量子力学中，观察者效应告诉我们测量本身会影响被测量的对象。在 AI 评估中，度量标准的选择同样会影响我们看到的「现实」。

论文 "Are Emergent Abilities a Mirage?" 的核心贡献不是「证伪」涌现能力，而是提醒我们：在报告任何实验结果之前，必须仔细选择和论证度量标准。如果一个结论只在特定的度量标准下成立，那它可能不是关于模型本身的结论，而是关于度量标准的结论。

从实践角度看，这场争论的意义在于：如果涌现是真实的，那么 AI 能力的预测就变得极其困难——我们无法从小模型的表现推断大模型的能力。如果涌现只是度量假象，那么 AI 的发展更加可预测。对于投资者和政策制定者来说，这两种观点会导致截然不同的决策。

6. 与 Scaling Law 的关系

涌现能力对 Scaling Law 的挑战在于：

  • 预测性：如果能力是涌现的，就无法用 Scaling Law 预测何时出现。Scaling Law 只能预测平均损失，而涌现能力是关于特定任务的质变。
  • 可外推性：Scaling Law 的核心价值在于外推（从小规模预测大规模），但涌现破坏了外推的可靠性——因为小规模上不存在的能力无法被拟合。
  • 资源分配：如果不知道能力何时涌现，就无法精确计算需要多少算力。这对价值数十亿美元的训练决策有重大影响。

💡 通俗类比：这就像在黑暗中走路——Scaling Law 告诉你「往前走 100 米，海拔会下降 10 米」（趋势可预测），但涌现能力意味着「走着走着可能突然掉进一个洞里」（存在不可预测的突变）。你需要同时关注趋势和突变，才能安全前行。

涌现与 Scaling Law 的统一视角

一个更深层的问题是：涌现能力和 Scaling Law 是否矛盾？答案是：不矛盾。Scaling Law 描述的是平均损失（一个聚合指标）的趋势，而涌现能力描述的是特定任务的行为。一个模型的平均损失可能按照幂律平滑下降，但某些特定任务的性能却可能呈现阶梯状跳跃。这就像一个班级的平均成绩在平稳提升，但某个学生的某个科目的成绩可能突然从不及格跳到优秀。

理解这一点对实践很重要：

  • Scaling Law 仍然可以可靠地预测总体损失趋势
  • 但对于特定任务的能力预测，需要额外考虑涌现效应
  • 最安全的策略是：用 Scaling Law 做总体规划，用任务特定的评估做精细调整

7. 2024-2025 的新理解

随着推理模型（o1、R1）的出现，涌现的讨论有了新维度：

  • 推理时 Scaling：不仅模型大小能带来涌现，推理时的计算量也能。o1 在「思考更久」后突然能解决之前解不了的数学题。这开辟了一条全新的涌现路径——不是增大模型参数，而是增加推理时间。详见 模块 08。
  • 训练方法的影响：GRPO/RLHF 等对齐方法可能激活了预训练中已经习得但未被「解锁」的能力。DeepSeek-R1 的实验表明，纯强化学习训练（不使用人类标注数据）就能让模型自发地学会思维链推理——这暗示涌现能力可能不是「新能力的产生」，而是「已有能力的解锁」。
  • 涌现 vs 能力解锁：越来越多研究者认为，涌现不是「新能力的产生」，而是「已有能力的解锁」。预训练过程中，模型已经学到了各种能力的「碎片」，但只有通过适当的训练方法（如 RLHF、GRPO）或足够的规模，这些碎片才能被组装成完整的能力。

推理时涌现的新发现

2024-2025 年，推理时缩放（Test-time Compute Scaling）带来了涌现研究的新突破。与传统的「增大模型参数」不同，推理时缩放通过增加推理阶段的计算量来提升性能。关键发现包括：

  • 思维链长度与能力的关系：o1 模型的思维链越长，解题能力越强。但这种增强不是线性的——当思维链长度超过某个阈值时，模型突然能解决之前完全无法解决的问题。
  • 搜索深度与涌现：使用 MCTS（蒙特卡洛树搜索）等搜索算法时，搜索深度的增加也会导致涌现。模型在浅层搜索时无法找到正确答案，但在深层搜索时突然能发现解题路径。
  • 自涌现：R1 模型展示了「自涌现」现象——通过纯 RL 训练，模型自发地学会了何时需要「思考更久」，何时可以快速回答。这种元认知能力本身就是一种涌现。

8. 涌现能力的实际影响

涌现能力的存在对 AI 研究和产业有深远影响：

影响领域具体影响应对策略

⚠️ 常见误区：很多人在评估小模型时发现它「不能做 X」，就认为大模型也做不了。但涌现能力告诉我们，这种推理是不可靠的。小模型不具备的能力，大模型可能突然具备。反过来也不安全——小模型具备的能力，大模型可能因为不同的训练策略而丧失。在 AI 领域，任何从小模型到大模型的外推都需要谨慎对待。

9. 如何设计涌现实验

如果你是一名研究者，想验证某个能力是否是涌现的，需要遵循以下步骤：

  1. 选择任务：选择一个明确定义的任务，有可量化的评估标准
  2. 选择度量：同时使用精确匹配和连续度量，避免度量假象
  3. 选择模型：在多个规模上测试（如 1B, 3B, 7B, 13B, 30B, 70B, 100B+）
  4. 控制变量：确保不同规模的模型使用相同的训练数据和方法
  5. 统计分析：使用统计检验（如 Changepoint Detection）识别突变点
  6. 消融实验：改变度量标准，验证结论的鲁棒性
# 涌现检测的伪代码示例
import numpy as np

def detect_emergence(model_sizes, accuracies, threshold=0.1):
    """
    检测涌现能力：在连续度量下寻找非线性跳跃
    model_sizes: 模型参数量列表 (如 [1e9, 3e9, 7e9, 13e9, 70e9])
    accuracies: 对应的准确率列表 (如 [0.12, 0.15, 0.18, 0.55, 0.82])
    threshold: 判断涌现的阈值 (二阶差分超过此值)
    """
    # 转换为对数坐标
    log_sizes = np.log10(model_sizes)
    
    # 计算一阶差分（斜率）
    first_diff = np.diff(accuracies) / np.diff(log_sizes)
    
    # 计算二阶差分（曲率）
    second_diff = np.diff(first_diff)
    
    # 二阶差分大的地方可能是涌现点
    emergence_points = np.where(np.abs(second_diff) > threshold)[0]
    
    if len(emergence_points) > 0:
        for idx in emergence_points:
            size = model_sizes[idx + 1]  # 涌现点的模型规模
            print(f"检测到涌现点: ~{size:.0e} 参数")
            print(f"  准确率从 {accuracies[idx]:.1%} 跃升到 {accuracies[idx+1]:.1%}")
    else:
        print("未检测到涌现点——性能是平滑提升的")
    
    return emergence_points

10. 涌现能力的分类学

根据不同的标准，涌现能力可以有多种分类方式：

按涌现机制分类

机制描述例子阈值特征
组合涌现多种基础能力的组合多步数学推理阈值取决于最弱的子能力
表示涌现高维表示空间的结构变化概念理解阈值取决于表示空间的有效维度
工具涌现学会使用外部工具计算器调用阈值取决于指令遵循能力
元认知涌现学会「知道自己不知道」拒绝回答不确定的问题阈值最高，通常 100B+

按可预测性分类

并非所有涌现能力都同样难以预测：

  • 可预测涌现：基于基础能力的组合，可以大致推断涌现时机（如多步推理 = 多个单步推理的组合）
  • 半可预测涌现：知道可能涌现，但不知道确切阈值（如代码生成）
  • 不可预测涌现：完全无法从小模型表现推断（如某些创造性能力）

💡 通俗类比：涌现能力的可预测性就像天气预报——短期（1-3 天）很准，中期（3-7 天）大致对，长期（7+ 天）基本靠猜。同样，「组合涌现」相对容易预测（就像看云就能判断会不会下雨），「元认知涌现」最难预测（就像预测一个月后的具体天气）。

11. 小结与展望

要点内容
定义小模型中不存在、大模型中突然出现的能力
阈值通常在 10B-100B 参数之间
类比类似物理学中的相变——临界点附近的质变
争议可能是度量假象（连续度量下平滑提升）
影响破坏了 Scaling Law 的可预测性
新发展推理时 Scaling 也展现涌现特性
实践建议同时使用精确匹配和连续度量评估模型

# 什么是涌现能力（Emergent Ability）？ 性能基准测试
import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
    elapsed = (time.perf_counter() - start) / iters
    print(f"Latency: {elapsed*1000:.2f}ms")
    return elapsed
