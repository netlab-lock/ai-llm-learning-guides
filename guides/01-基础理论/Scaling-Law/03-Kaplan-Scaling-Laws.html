1. 论文背景

项目内容
论文标题Scaling Laws for Neural Language Models
作者Jared Kaplan, Sam McCandlish, Tom Henighan, Tom B. Brown, Benjamin Chess, Rewon Child, Scott Gray, Alec Radford, Jeffrey Wu, Dario Amodei
机构OpenAI
发表时间2020 年 1 月
核心发现语言模型的损失与 N、D、C 各自呈独立幂律关系

📜 历史背景：这篇论文发表时，GPT-2（1.5B）刚发布不久，GPT-3 还在训练中。OpenAI 的核心问题是：「如果我们继续加大模型，性能会怎样变化？」Kaplan 等人通过训练数百个不同规模的模型（从 768 参数到 15 亿参数），首次给出了精确的数学答案。这篇论文直接影响了 GPT-3（175B）的设计决策。

值得注意的是，Kaplan 团队在实验设计上投入了大量精力来控制变量。他们固定了学习率调度策略、batch size 的选择、以及训练数据的预处理流程，确保不同规模的模型之间具有可比性。这种严谨的实验方法论本身就是这篇论文的重要贡献之一——后续的 Chinchilla、DeepSeek 等研究都沿用了类似的控制变量范式。

2. 核心发现：三条独立幂律

Kaplan 发现，当其他因素固定时，损失与每个变量各自呈幂律关系：

① 参数量 Scaling：L(N)

L(N) = (Nc / N)^0.076

含义：参数量每翻倍，损失降低约 5%。虽然 5% 听起来很小，但在幂律的长期曲线上，这意味着每增加一个数量级（10 倍）的参数，损失就能下降约 16%。对于语言模型来说，损失的微小差异可能对应着模型能力的巨大飞跃——从"偶尔正确"到"大部分正确"的跨越往往只需要损失降低几个百分点。

参数量 N相对损失直观感受
10M (1000万)1.00 (基准)基础水平
100M0.84改善 16%
1B (10亿)0.70改善 30%
10B0.59改善 41%
100B0.50改善 50%
1T (万亿)0.42改善 58%

💡 直觉理解：幂律的「边际递减」特性意味着，虽然每次翻倍的回报相同（约 5%），但你需要投入的绝对资源在不断增长。从 10M 到 100M 是 90M 新参数，从 100B 到 200B 是 100B 新参数——后者是前者的 1000 多倍资源，但获得的损失改善同样是 5%。这就是为什么「边际回报递减」是 Scaling Law 的核心特征。

② 数据量 Scaling：L(D)

L(D) = (Dc / D)^0.095

含义：数据量每翻倍，损失降低约 6.4%。注意 α_D > α_N，说明数据的边际回报略高于参数。这个发现意味着在资源有限的情况下，增加训练数据可能比增加模型参数更划算。然而，Kaplan 的后续分析中，由于他的实验设置中数据量普遍不足（模型没有被充分训练），导致他在「参数优先 vs 数据优先」的权衡中偏向了参数。这一偏差在两年后被 Chinchilla 论文纠正。

③ 计算量 Scaling：L(C)

L(C) = (Cc / C)^0.050

含义：计算量每翻倍，损失降低约 3.4%。这是最直接的「投入产出」关系。计算量 C 包括了训练过程中所有的浮点运算——前向传播、反向传播、梯度更新等。由于 C ≈ 6ND（参见模块 02），计算量同时受到参数量和数据量的约束。Kaplan 用这个公式来回答最实际的问题：「给定预算，能达到什么水平？」

⚠️ 三个幂指数的差异来源：α_N ≈ 0.076、α_D ≈ 0.095、α_C ≈ 0.050，这三个值看起来差异不大，但它们反映了完全不同的缩放维度。α_C < α_N < α_D 这个排序说明：在 Kaplan 的实验设置下，单独增加数据的回报 > 单独增加参数的回报 > 同时增加两者的回报。但需要注意，这些指数是在「固定其他变量」的条件下测量的，实际中 N 和 D 往往同时变化，联合效应更为复杂。

3. Kaplan 的关键结论

结论一：性能主要取决于规模，而非架构细节

Kaplan 测试了不同深度、宽度、注意力头数的模型，发现：只要参数量相同，不同架构的损失差异很小。这意味着选择 Transformer 的具体配置（多少层、多少头）远不如增加总参数量重要。

具体来说，Kaplan 测试了以下架构变体：不同的 FFN 隐藏层维度（从 256 到 4096）、不同的注意力头数（从 1 到 16）、不同的层数（从 2 到 64），以及是否使用位置编码、不同的 embedding 维度等。在控制总参数量不变的情况下，这些架构选择带来的损失差异通常不超过 0.5%。这个结论的重要性在于：它将研究焦点从"如何设计更好的架构"转移到了"如何高效地扩大规模"。

结论二：大模型更高效（参数优先策略）

Kaplan 的一个重要推论是：在固定计算预算下，应该训练尽可能大的模型，然后用较少的数据训练到收敛前停止（early stopping）。这意味着「过参数化」（over-parameterization）不仅不是问题，反而是最优策略。一个 10 倍大的模型，即使没有被充分训练，其性能也优于被充分训练的小模型。

🔬 参数优先策略的数学直觉：为什么大模型+少数据优于小模型+多数据？Kaplan 的分析基于一个关键观察：参数量的幂指数 α_N ≈ 0.076 和数据量的幂指数 α_D ≈ 0.095 虽然接近，但在联合损失函数 L(N,D) = (Nc/N)^α_N + (Dc/D)^α_D + E 中，两者之间存在一个微妙的平衡关系。当总计算预算 C = 6ND 固定时，Kaplan 通过拉格朗日乘数法求解最优分配，发现最优解倾向于 N 大 D 小。具体推导如下：

设 C = 6ND，要最小化 L(N,D)。令 ∂L/∂N = λ·∂C/∂N 和 ∂L/∂D = λ·∂C/∂D，可以得到 N* ∝ C^(α_D/(α_N+α_D))，D* ∝ C^(α_N/(α_N+α_D))。由于 α_N/(α_N+α_D) ≈ 0.445 < 0.5，这意味着 D 的增长率慢于 C 的平方根——数据量「不足」于匹配计算增长。

结论三：存在不可约损失

即使模型无限大、数据无限多，损失也不会降到零。存在一个不可约损失（irreducible loss）E，代表数据本身的内在熵。对于英语文本，Kaplan 估计 E ≈ 1.69 nats/token。这个值来自语言本身的不确定性——即使是一个完美的人类专家，也无法 100% 准确预测下一个 token（因为语言具有内在的随机性和多义性）。

4. 联合损失函数

当 N 和 D 同时受限时，Kaplan 给出的联合形式为：

L(N, D) = [(Nc/N)^α_N] + [(Dc/D)^α_D] + E

这个公式是两个独立幂律的加法组合。它意味着：

  • 增加 N 和增加 D 都能降低损失，但效果是独立的
  • 如果 N 很大但 D 很小，瓶颈在数据——继续增加 N 的收益递减
  • 如果 D 很大但 N 很小，瓶颈在参数——继续增加 D 的收益递减
  • 最优策略需要平衡两者，而 Kaplan 认为应该偏重 N

⚠️ 加法组合的局限性：Kaplan 的联合损失公式使用加法组合，这意味着 N 和 D 的效应是「可加的」（additive）。然而，后续研究（尤其是 Chinchilla）表明，乘法形式 L(N,D) = A/N^α + B/D^β + E 可能更为精确，其中 A、B 是独立的常数。两者的区别在于：加法形式下，N 和 D 互不影响；而更精确的模型可能需要考虑 N 和 D 之间的交互效应。这一细微差异导致了 Kaplan 和 Chinchilla 在最优分配比例上的巨大分歧。

🔬 Scaling Law 如何改变了 AI 投资逻辑：在 Kaplan 之前，AI 公司很难向投资者解释「为什么需要更多算力」。Kaplan 的 Scaling Law 提供了一个清晰的数学框架：投入 X 算力 → 预期 Y 性能提升。这使得 AI 训练从「凭感觉」变成了「可规划的工程」。投资者可以用 Scaling Law 来评估一个 AI 项目的技术可行性：「你说要训练一个比 GPT-3 更好的模型？请用 Scaling Law 证明你的算力够不够。」这种量化思维直接推动了 AI 行业的资本化和产业化进程。

6. Kaplan 的局限性

两年后被修正：Kaplan 的「参数优先」策略在 2022 年被 DeepMind 的 Chinchilla 论文证明是次优的。Kaplan 的实验中，模型没有被充分训练（数据不够），导致他高估了参数的重要性，低估了数据的重要性。详见模块 04。

具体来说，Kaplan 的实验中每个模型只训练了固定数量的 step（而不是训练到收敛），这意味着较大的模型没有看到足够多的数据就被停止了。在这种「未充分训练」的设置下，大模型确实表现更好——因为它们即使只训练了一小部分，也能利用其容量优势。但如果让所有模型都训练到收敛，情况就不同了。

幂律拟合方法

Kaplan 使用了严格的统计方法来验证幂律关系：首先在双对数坐标中绘制所有数据点，然后用最小二乘法拟合直线。关键的验证步骤是检查残差——如果残差在双对数坐标中随机分布且无明显模式，则确认幂律成立。Kaplan 报告称，幂律拟合的 R² 值通常超过 0.99，表明拟合质量极高。此外，Kaplan 还进行了敏感性分析：移除部分数据点后重新拟合，检查 α 值是否稳定。结果表明，只要数据点覆盖足够的范围（至少 2 个数量级），α 值的估计是稳健的。

值得注意的是，Kaplan 的幂律拟合不是在原始空间中进行的，而是在对数空间中。具体步骤为：(1) 对参数量和损失分别取 log；(2) 在 (log N, log L) 空间中做线性回归；(3) 回归直线的斜率即为 -α，截距给出 log(Nc)。这种方法等价于在原始空间中拟合幂律，但在对数空间中更直观、更稳定（因为不同数量级的数据点在对数坐标中均匀分布）。

💡 如何复现 Kaplan 的实验：你不需要训练数百个模型就能体验 Scaling Law。一个简单的实验方案是：(1) 准备一个小型语料库（如 WikiText-103）；(2) 训练 5-10 个不同规模的 Transformer（从 1M 到 100M 参数）；(3) 在双对数坐标中绘制 (参数量, 损失) 散点图；(4) 用线性回归拟合幂指数 α。即使在小规模实验中，你也能清楚地看到幂律直线！

8. Kaplan 对后续研究的启发

Kaplan 的工作不仅是一篇实证研究，更开创了一种全新的研究范式。在 Kaplan 之前，AI 研究主要关注「如何设计更好的架构」和「如何训练更有效的模型」；Kaplan 之后，研究焦点转向了「如何预测不同规模下的模型性能」和「如何在给定预算下找到最优配置」。这种思维方式的转变，使得 AI 研究从经验驱动走向了理论驱动。

Kaplan 的方法论——训练大量不同规模的模型、在双对数坐标中拟合幂律、用幂律预测更大规模的行为——成为了后续所有 Scaling Law 研究的标准模板。Chinchilla（2022）、DeepSeek（2024）、以及各种 MoE Scaling Law 研究都沿用了这一范式。可以说，Kaplan 不仅发现了一组幂律关系，更建立了一套研究方法论。

Kaplan 论文的核心遗产： ┌─────────────────────────────────────────────┐ │ 1. 实证发现：Loss ∝ N^(-α) 的幂律关系 │ │ 2. 策略推论：参数优先 > 数据优先 │ │ 3. 方法论：大规模实证 + 幂律拟合 + 预测 │ │ 4. 催化效应：催生 GPT-3 和算力军备竞赛 │ │ │ │ 遗产 #1 被 Chinchilla 修正 │ │ 遗产 #2 被 Chinchilla 推翻 │ │ 遗产 #3 至今仍是标准范式 ✓ │ │ 遗产 #4 改变了整个 AI 行业 ✓ │ └─────────────────────────────────────────────┘

💡 从 Kaplan 到 Chinchilla：科学的自我修正：Kaplan 的「参数优先」策略在两年后被 Chinchilla 推翻，但这并不意味着 Kaplan 的工作是失败的。恰恰相反，正是因为 Kaplan 给出了明确的数学预测，Chinchilla 才能通过更严格的实验来验证或推翻这些预测。这就是科学进步的方式：一个理论被提出、被检验、被修正或被取代。Kaplan 的贡献在于提出了正确的研究问题和方法论，即使具体的结论后来被修正，他的方法论仍然是无价的。

10. 代码复现

下面的代码不仅复现了 Kaplan 的幂律曲线，还展示了如何从实验数据中提取幂指数 α。这段代码可以作为你自己进行 Scaling Law 实验的模板——只需替换数据点即可。

import numpy as np
import matplotlib.pyplot as plt
from scipy.optimize import curve_fit

# 定义幂律函数
def power_law(x, a, alpha):
    return a * np.power(x, -alpha)

# Kaplan 的参数（用于对比验证）
Nc = 8.8e13
alpha_N = 0.076

# 生成理论曲线数据
N_theory = np.logspace(6, 13, 100)
L_theory = (Nc / N_theory) ** alpha_N

# 模拟实验数据点（带噪声）
np.random.seed(42)
N_exp = np.logspace(7, 12, 8)
L_exp = (Nc / N_exp) ** alpha_N * (1 + 0.02 * np.random.randn(len(N_exp)))

# 用 curve_fit 拟合幂律
popt, pcov = curve_fit(power_law, N_exp, L_exp, p0=[1e3, 0.08])
a_fit, alpha_fit = popt
print(f"拟合结果: Loss = {a_fit:.2e} * N^(-{alpha_fit:.4f})")
print(f"真实值:   Loss = {Nc**alpha_N:.2e} * N^(-{alpha_N})")
print(f"拟合误差: Δα = {abs(alpha_fit - alpha_N):.4f}")

# 绘图对比
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 5))
ax1.plot(N_theory, L_theory, 'b-', linewidth=2, label='理论曲线')
ax1.scatter(N_exp, L_exp, c='red', s=50, zorder=5, label='实验数据')
ax1.set_xscale('log')
ax1.set_xlabel('Parameters (N)')
ax1.set_ylabel('Loss')
ax1.set_title('Kaplan L(N) - 普通坐标')
ax1.legend()

ax2.loglog(N_theory, L_theory, 'b-', linewidth=2, label='理论曲线')
ax2.loglog(N_exp, L_exp, 'ro', markersize=8, zorder=5, label='实验数据')
ax2.loglog(N_exp, power_law(N_exp, *popt), 'g--', linewidth=1.5, label='拟合曲线')
ax2.set_xlabel('log(Parameters)')
ax2.set_ylabel('log(Loss)')
ax2.set_title('Kaplan L(N) - 双对数坐标')
ax2.legend()
ax2.grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig('kaplan_scaling.png', dpi=150)
plt.show()

import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
