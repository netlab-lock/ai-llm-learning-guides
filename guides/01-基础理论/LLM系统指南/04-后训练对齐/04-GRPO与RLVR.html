🚀 4. GRPO 与 RLVR — 2025 新标准

4.1 GRPO (Group Relative Policy Optimization)

GRPO 是 DeepSeek 团队提出的一种高效策略优化算法，2025 年已成为后训练的新标准。它的核心创新是用"组内相对排名"代替传统 PPO 中的 Value Model（Critic），从而将训练所需的模型数量从 4 个减少到 2 个，同时保持甚至超越 PPO 的效果。GRPO 的设计哲学是：与其让一个额外的神经网络学习"绝对价值"，不如让同一问题的多个回答相互比较，用组内统计量来估计相对优势。

GRPO 的核心创新: PPO: 需要 Value Model (critic) 来估计优势函数 GRPO: 用"组内相对排名"代替 Value Model GRPO 流程: 1. 对同一个问题 x，采样一组回答 {y_1, y_2, ..., y_G} (G=8~64) 2. 对每个回答计算奖励 r_i (可以是 RM 或规则) 3. 用组内统计计算优势: A_i = (r_i - mean(r_1..r_G)) / std(r_1..r_G) 4. 用类 PPO 目标更新: J = E[min(ratio·A, clip(ratio)·A)] - β·KL(π||π_ref) 核心: 组内相对排名 → 不需要 Value Model!
💡 通俗类比：GRPO 像"班级排名"
PPO 需要一个"评分老师"（Value Model）给每个回答打绝对分。GRPO 的创新是：让同一道题的 G 个回答互相比较，用组内平均分和标准差做归一化。就像高考 — 不需要知道"绝对分数"，只需要知道"你在全省的排名"。这种相对评价的优势在于：(1) 不需要额外的 Value Model；(2) 天然适应不同难度的题目；(3) 对奖励的尺度不敏感。
⚠️ 常见误区：GRPO 的采样数量很重要
GRPO 对每个问题采样 G 个回答（通常 G=8~64）。G 太小 → 组内统计不稳定（排名不可靠）；G 太大 → 计算成本高。实践中 G=16~32 是常用范围。采样时 temperature 也需要适当调高（如 0.7~1.0），以保证组内多样性。如果 temperature 太低，所有回答几乎相同，组内排名就失去了意义。

4.1.1 GRPO 的数学细节

# GRPO 的目标函数
J_GRPO(θ) = E_{x~D} [ 1/G · Σ_i (
    min(r_i(θ)·Â_i, clip(r_i(θ), 1-ε, 1+ε)·Â_i)

# 其中:
r_i(θ) = π_θ(y_i|x) / π_old(y_i|x)     # 策略比率
Â_i = (r_i - mean(r)) / std(r)           # 组内归一化优势
G = 采样数量 (通常 16~32)                  # 组大小
ε = 0.2                                   # 裁剪范围
🔬 深入原理：为什么组内归一化有效？——从方差缩减角度
GRPO 的组内归一化本质上是一种方差缩减（Variance Reduction）技术。在标准的策略梯度方法中，优势函数 A(s,a) = R(s,a) - V(s) 的估计方差很大，因为 V(s) 是通过一个独立的神经网络估计的，本身就有误差。 GRPO 通过采样同一问题的 G 个回答，利用这些回答之间的奖励差异来估计优势。数学上，当 G 足够大时： E[Â_i | r_i > mean(r)] > 0 （高于平均的回答获得正优势） E[Â_i | r_i < mean(r)] < 0 （低于平均的回答获得负优势） 这种估计是无偏的（当 G→∞），并且方差随 G 增大而减小。更重要的是，它不需要额外的 Value Model 参数，减少了模型的总参数量和训练复杂度。 一个关键的实现细节是：组内归一化是在每个 mini-batch 内进行的，而不是在整个数据集上。这意味着每个 mini-batch 内需要有足够多的样本（每个问题 G 个），以确保归一化的统计量稳定。

4.2 RLVR (Reinforcement Learning with Verifiable Rewards)

RLVR 是一种革命性的方法，它用可自动验证的任务作为奖励信号，完全消除了对人类偏好数据和奖励模型的依赖。RLVR 的核心思想是：对于数学题、代码题等有明确正确答案的任务，我们可以用程序自动判断回答是否正确，从而获得零噪声的奖励信号。

# RLVR 的奖励来源
数学题: answer == correct_answer ? 1.0 : 0.0
代码题: run_tests(code).pass_rate           # 0.0 ~ 1.0
逻辑题: verify_reasoning_chain(response)     # 基于规则

# 完美奖励信号:
# - 不需要人类标注
# - 不需要训练 RM
# - 奖励完全准确 (不是近似)
RLVR 奖励信号 vs RLHF 奖励信号: RLHF (RM 代理奖励): 问题: "2+3=?" 回答: "5" → RM评分: 0.87 (近似，可能不准) 回答: "4" → RM评分: 0.42 (近似，可能不准) RLVR (可验证奖励): 问题: "2+3=?" 回答: "5" → 验证器: 1.0 (精确！) 回答: "4" → 验证器: 0.0 (精确！) RLVR 的优势: 奖励信号零噪声，训练更稳定
🔬 深入原理：RLVR 的适用范围与局限性
RLVR 只适用于有明确正确答案的任务：数学题（答案可验证）、代码题（测试用例可运行）、逻辑推理（规则可检查）。对于开放式任务（写作、创意、对话），仍需要 RM 或人类偏好。 然而，RLVR 的意义远超其适用范围。DeepSeek-R1-Zero 的实验表明：仅用数学和代码的可验证奖励进行 RL 训练，模型就自发涌现出了通用的推理能力（Chain-of-Thought、自我验证、反思纠错）。这意味着推理能力可能是某种"元能力"——在可验证任务上训练的推理能力可以迁移到不可验证的任务上。 2025 年的最佳实践是：用 RLVR 训练推理"硬实力"（数学、代码、逻辑），用 DPO 做偏好"软对齐"（对话风格、安全性、有帮助性）。两者互补，缺一不可。

4.3 DeepSeek-R1: GRPO + RLVR 的典范

DeepSeek-R1-Zero 训练: 基座: DeepSeek-V3 数据: 数学题 + 代码题 (可验证) 奖励: 答案正确性 (自动验证) 算法: GRPO 惊人结果: 模型自发学会了: ✓ 长链推理 (Chain-of-Thought) ✓ 自我验证 ("让我检查一下...") ✓ 反思纠错 ("等等，这不对，让我重新...") ✓ 尝试多种方法 ✓ "Aha moment" — 突然领悟推理策略 没有任何人工 CoT 数据！纯 RL 涌现！
💡 通俗类比：DeepSeek-R1-Zero 像"自学成才"
传统方法需要人类先写好"推理示范"（CoT 数据），模型照着学。R1-Zero 完全不需要 — 只给题目和答案，模型自己摸索出推理方法。就像给小孩一堆数学题和答案，不教解题方法，小孩自己总结出了"先算括号再算乘除"的规则。这就是 RL 的涌现能力——当奖励信号足够清晰（可验证），模型能够自发地发展出复杂的推理策略。
⚠️ 常见误区：R1-Zero ≠ R1
R1-Zero 是纯 RL 训练（无 SFT 数据），虽然推理能力强但输出格式不规范（如混用中英文、格式混乱、可读性差）。正式版 R1 先用少量 SFT 数据做格式对齐，再用 GRPO+RLVR 训练推理能力。所以实际部署时，SFT + RLVR 的组合比纯 RLVR 更实用。这个经验教训是：RL 擅长提升"能力"，但不擅长控制"格式"；两者需要分阶段处理。

4.3.1 R1 的完整训练流程

# DeepSeek-R1 的四阶段训练流程

# 阶段1: 冷启动 SFT
- 收集少量高质量推理数据（含 CoT 格式）
- 对基座模型进行 SFT，建立基本的推理输出格式

# 阶段2: 推理 RL (GRPO + RLVR)

# 阶段3: 拒绝采样 + SFT

- 用筛选后的数据做 SFT，对齐格式

# 阶段4: 全场景 RL
- 同时使用可验证奖励和 RM 奖励
- 训练推理能力 + 通用对话能力

4.5 "RLHF 之死" 与范式转变

⚠️ 范式转变 (2025-2026 已验证)
"The Death of RLHF" — RLHF 正在被 RLVR + GRPO 取代:
- RLHF 需要人类偏好数据 → 贵且有噪声
- RLHF 需要 RM + Value Model → 成本高
- RLHF 训练不稳定 → 超参敏感
- GRPO + RLVR 更简单、更便宜、更可扩展
- 2025 年的头条转变就是这个
- 但 RLHF 并未完全消亡——在安全对齐等不可验证场景中仍有价值
💡 通俗类比：后训练方法的"三国演义"
RLHF 像"请专家评审" — 贵但效果好。DPO 像"自己对比学习" — 便宜但需要好数据。GRPO+RLVR 像"考试刷题" — 有标准答案的题自动批改，便宜、准确、可扩展。2025 年的趋势是：用 GRPO+RLVR 训练推理"硬实力"，用 DPO 做偏好"软对齐"。三者不是替代关系，而是在不同场景下各有所长。

4.7.1 GRPO 实践中的关键超参数

GRPO 的训练效果对超参数敏感，以下是各超参数的调优指南：

超参数典型值影响调优建议

4.7.2 RLVR 验证器设计最佳实践

验证器的质量直接决定了 RLVR 训练的效果。以下是设计高质量验证器的关键原则：

  • 准确性优先：验证器必须给出正确的奖励信号。一个错误的验证器比没有验证器更糟糕——模型会学到错误的模式。
  • 处理边界情况：答案可能有多种等价表达方式。例如 "1/2" 和 "0.5" 和 "50%" 都是正确的。验证器需要处理这些等价形式。
  • 部分分数：对于多步推理，可以考虑给部分正确的回答部分分数（如 0.5），而不是简单的 0/1。这提供了更细粒度的学习信号。
  • 格式容错：模型可能在答案前后添加额外文本（如 "答案是 42"），验证器需要能够提取核心答案。
  • 避免假阳性：确保正确答案的推导过程也是合理的，而不仅仅是巧合得到正确数字。
RLVR 验证器设计模式: 模式1: 精确匹配 (Exact Match) 输入: "答案是 42" 提取: "42" 验证: "42" == "42" → 1.0 适用: 数学计算题 模式2: 测试用例执行 (Code Execution) 输入: Python 代码 执行: 运行测试用例 验证: 10/12 测试通过 → 0.83 适用: 代码生成 模式3: 形式化证明 (Formal Verification) 输入: Lean4 证明 验证: 编译器检查 → 0.0 或 1.0 适用: 数学证明 模式4: 规则匹配 (Rule-based) 输入: 推理链 验证: 检查逻辑步骤是否合理 适用: 逻辑推理、常识推理

⚠️ 实践陷阱：GRPO 训练中的常见错误
GRPO 训练中常见的错误包括： 1. 采样温度设置不当：温度太低导致组内回答几乎相同，排名失去意义；温度太高导致回答质量下降 2. 组大小太小：G=4 时组内统计量方差大，优势估计不稳定 3. KL 惩罚被忽视：没有 KL 惩罚时模型会快速退化，生成无意义文本 4. 验证器有漏洞：模型找到验证器的"后门"，生成表面正确但实际错误的答案 5. 学习率过大：GRPO 需要较小的学习率（1e-6），过大会导致训练不稳定

4.7.4 GRPO 与 RLVR 的未来展望

GRPO 和 RLVR 技术仍在快速演进中。以下是几个值得关注的发展方向：

首先，过程奖励模型（PRM）的兴起将为 RLVR 带来质的飞跃。传统的 RLVR 只对最终答案进行验证，而 PRM 可以对推理过程的每一步进行评分。这意味着即使是部分正确的推理也能获得正向反馈，极大地提高了学习效率。DeepSeek 和 OpenAI 都在积极探索 PRM 的自动化训练方法，以降低标注成本。

其次，多模态 RLVR 是一个充满潜力的方向。将 RLVR 从纯文本扩展到图像、视频和音频领域，可以让模型在视觉推理、空间理解等任务上也获得类似 R1 的涌现能力。挑战在于如何为这些多模态任务设计有效的验证器——图像描述的正确性比数学答案更难自动验证。

第三，GRPO 与 Agent 训练的结合是一个热门方向。Agent 需要在多步交互中做出决策，每一步都涉及工具调用和环境反馈。GRPO 的组内相对排名机制天然适合这种场景——对同一任务采样多条执行路径，用环境反馈作为奖励信号进行优化。这可能催生出新一代的 Agent 训练范式。

最后，GRPO 在非推理任务中的应用也在探索中。虽然 GRPO 最初是为推理任务设计的，但其组内相对排名的思想可以推广到任何有明确评估标准的任务。例如，在创意写作中可以用 LLM-as-Judge 作为奖励信号，对同一主题的多个故事进行组内排名。这将使 GRPO 的适用范围从推理扩展到更广泛的领域。

深入探讨： 从GRPO的核心原理出发，Group Relative Policy Optimization通过组内相对排名来估计优势函数，避免了单独训练价值网络的开销。为什么GRPO适合大语言模型的强化学习？推导：传统PPO需要价值网络V(s)来计算优势A=r+γV(s')-V(s)，但LLM的输出空间巨大，价值网络难以准确估计。GRPO对同一prompt生成G个响应，用组内奖励的均值和方差做归一化。数学本质上，这是用蒙特卡洛采样替代了价值网络的偏差估计，在数学推理等可验证任务上（RLVR）尤为有效，DeepSeek-R1就是基于此方法训练的。

✅ 参考答案与详解
  1. 核心原理: GRPO（组相对策略优化）的核心创新是用"组内相对排名"代替传统PPO中的Value Model来估计优势函数。具体流程为：(1)对同一问题x采样一组回答{y_1, y_2, ..., y_G}（G通常为16-32）；(2)对每个回答计算奖励r_i（可以是RM评分或规则验证器）；(3)用组内统计量计算归一化优势Â_i = (r_i - mean(r_1..r_G)) / std(r_1..r_G)；(4)用类PPO目标更新策略J = E[min(ratio·Â, clip(ratio,1-ε,1+ε)·Â)] - β·KL(π||π_ref)。这种方法的本质是一种方差缩减技术——通过采样同一问题的多个回答，利用它们之间的奖励差异来估计优势，当G足够大时估计是无偏的且方差随G增大而减小。RLVR（可验证奖励强化学习）进一步用程序自动验证（如数学答案精确匹配、代码测试用例执行）代替RM，获得零噪声的奖励信号。两者结合（GRPO+RLVR）是DeepSeek-R1成功的核心技术。
    关键要点: ①GRPO将训练所需模型从4个（PPO需要Policy+Ref+RM+Value）减少到2个（Policy+Ref），大幅降低显存需求和训练复杂度 ②组大小G是关键超参数——G太小(如4)导致组内统计不稳定，G太大(如64)计算成本高，16-32是常用范围 ③采样温度需适当调高(0.7-1.0)保证组内多样性，温度太低则所有回答几乎相同，排名失去意义 ④RLVR只适用于有明确正确答案的任务（数学、代码、逻辑），但DeepSeek-R1证明在这些任务上训练的推理能力可以迁移到不可验证任务 ⑤DeepSeek-R1-Zero的惊人发现：仅用数学和代码的可验证奖励，模型自发涌现出了Chain-of-Thought、自我验证、反思纠错等推理能力，没有任何人工CoT数据
    实际案例: DeepSeek-R1的四阶段训练流程是GRPO+RLVR的最佳实践：阶段1冷启动SFT（少量高质量推理数据建立格式）→阶段2 GRPO+RLVR（数学题验证答案正确性+代码题运行测试用例，用GRPO优化推理能力）→阶段3拒绝采样+SFT（筛选高质量推理样本做格式对齐）→阶段4全场景RL（同时使用可验证奖励和RM奖励）。使用Unsloth的GRPO教程在Colab上训练Qwen2-1.5B的典型配置：G=8, lr=5e-7, β=0.1, 用GSM8K数据集+答案正确性验证器，训练200步后可观察到模型推理长度从平均50 token增长到300 token，且出现了自发的Chain-of-Thought。
    关联知识: GRPO继承了PPO的裁剪机制（ε=0.2）和KL惩罚结构，但用蒙特卡洛采样的组内统计量代替了Value Model的偏差估计。这在数学上等价于用"班级排名"代替"绝对分数"——不需要知道"绝对分数"只需要知道"你在全省的排名"。GRPO与DPO的关系：DPO通过闭式解消除了RM，GRPO通过组内排名消除了Value Model，两者从不同角度简化了RLHF。2025年最佳实践是组合使用：SFT→GRPO+RLVR(推理能力)→DPO(偏好对齐)→安全RLHF(安全过滤)，每个阶段解决不同问题。
  2. 核心原理: GRPO+RLVR在实际工程中的主要性能瓶颈包括：(1) 采样计算开销——每个问题需要采样G个回答（G=16-32），这意味着每步的推理计算量是单次采样的16-32倍，对于长推理序列（1000+ token）尤为昂贵；(2) 组大小与统计稳定性的权衡——G太小导致优势估计方差大、训练不稳定，G太大导致计算成本过高；(3) 采样温度敏感——温度太低组内回答趋同、排名无意义，温度太高回答质量下降、奖励信号噪声增大；(4) 验证器设计挑战——数学答案需要处理多种等价表达（"1/2"="0.5"="50%"），代码题需要设计高覆盖率的测试用例，验证器的漏洞会导致模型学到"欺骗"验证器的策略；(5) KL惩罚被忽视——没有KL惩罚时模型会快速退化生成无意义文本。
    关键要点: ①GRPO的典型超参数调优：G从16开始，采样温度0.7-1.0，KL系数β从0.04开始（观察KL散度调整），裁剪范围ε=0.2通常不需调整，学习率1e-6比SFT小 ②验证器设计最佳实践：准确性优先（错误验证器比没有更糟）、处理等价表达、考虑部分分数（0-1连续值而非0/1二值）、格式容错（提取核心答案）、避免假阳性 ③DeepSeek-R1-Zero vs R1的教训：纯RLVR训练的模型推理能力强但输出格式混乱（混用中英文、可读性差），需要先SFT做格式对齐再RLVR ④监控训练过程中的KL散度、奖励分布、推理长度变化——推理长度增长通常是涌现能力的信号 ⑤使用vLLM或TGI加速采样阶段的推理，显著降低每步计算时间
    实际案例: 在Colab上用Unsloth训练GRPO模型时，一个常见问题是采样阶段耗时过长——G=16、序列长度1024时，每步采样需要约30秒。解决方案：(1)使用vLLM的批量推理加速采样，可将采样时间降低到约8秒；(2)从G=8开始训练，确认有效后再增大到16-32。另一个典型案例是验证器漏洞——某团队的数学验证器只做精确字符串匹配，模型学会了在答案中堆砌数字（如"答案是42，因为21×2=42，42是正确答案"）来提高匹配概率。修复方案是先用正则表达式提取最后一个数字再比较。
    关联知识: GRPO的采样计算开销问题与RLHF中PPO的在线采样类似，但GRPO的采样可以高度并行化（同一问题的G个回答独立生成）。验证器设计的知识与软件工程中的测试用例设计密切相关——本质上都是"如何判断输出是否正确"。RLVR的适用范围限制（只适用于可验证任务）推动了过程奖励模型（PRM）的发展——PRM对推理每步打分而非仅看最终答案，可以为部分正确的推理提供反馈。GRPO与Agent训练的结合是未来方向——对同一任务采样多条执行路径，用环境反馈作为奖励信号。
  3. 核心原理: GRPO+RLVR的主要替代方案包括RLHF(PPO)、DPO、KTO。核心权衡如下：(1) GRPO vs PPO——PPO需要4个模型（Policy+Ref+RM+Value），GRPO只需2个（Policy+Ref），用组内排名代替Value Model、用验证器代替RM。PPO的奖励信号来自RM（有噪声），GRPO+RLVR的奖励来自规则验证（零噪声）。训练稳定性GRPO更高（组内归一化天然稳定）。(2) GRPO vs DPO——DPO适合通用偏好对齐（对话风格、安全性），GRPO适合推理能力训练（数学、代码）。DPO是离线方法（不需要在线采样），GRPO需要在线采样但奖励信号更准确。(3) GRPO vs KTO——KTO数据要求最低（只需好/坏标签），但对推理能力的提升有限。(4) 2025年范式转变：GRPO+RLVR正在取代RLHF成为后训练主流，但RLHF在安全对齐等不可验证场景中仍有价值。
    关键要点: ①方法选择决策树：格式对齐→SFT，推理能力→GRPO+RLVR（有标准答案时首选），偏好对齐→DPO（有成对偏好）或KTO（只有好/坏标签） ②GRPO+RLVR的革命性在于：不需要人工CoT数据（模型自己学会推理）、不需要RM（规则验证代替）、不需要Value Model（组内排名代替）、奖励信号零噪声（可精确验证） ③RLHF并未完全消亡——在写作、对话、安全等不可验证任务中，RM和人类偏好仍然不可替代 ④最佳实践是分阶段组合：SFT(格式)→GRPO+RLVR(推理)→拒绝采样+SFT(格式对齐)→DPO(偏好)→安全RLHF(安全) ⑤成本效益：GRPO+RLVR的总成本约$3K-10K（7B模型），远低于RLHF的$15K-60K，且推理能力提升更显著
    实际案例: 一个完整的后训练流水线实例：以DeepSeek-V3为基座，(1)SFT阶段用500条高质量推理数据（含CoT格式）做冷启动，1×A100训练2小时；(2)GRPO+RLVR阶段用GSM8K+MATH数据集训练推理能力，G=16, lr=1e-6, 4×A100训练24小时；(3)拒绝采样阶段用训练好的模型生成64个候选/题，筛选答案正确+格式规范的样本做SFT；(4)DPO阶段用人工标注的对话偏好数据做偏好对齐。最终模型在MATH上达到78%准确率（接近GPT-4o），在MT-Bench上达到8.2分。对比纯RLHF方案，总成本降低了约70%，推理能力提升更显著。
    关联知识: GRPO+RLVR代表了后训练方法从"人类偏好驱动"到"可验证奖励驱动"的范式转变。这个转变的理论基础是：推理能力可能是预训练阶段已经学到的"元能力"，RLVR的零噪声奖励信号提供了正确的"激活方向"。这与SFT的原理相通——SFT通过loss mask只在response部分计算loss来"激活"问答能力，RLVR通过可验证奖励来"激活"推理能力。GRPO的组内归一化思想可以推广到任何有明确评估标准的任务——如创意写作中用LLM-as-Judge对同一主题的多个故事进行组内排名。过程奖励模型（PRM）和多模态RLVR是GRPO+RLVR的未来扩展方向。
