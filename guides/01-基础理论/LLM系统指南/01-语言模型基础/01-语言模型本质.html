🔤 1. 语言模型的本质

1.1 什么是语言模型

语言模型 (Language Model) 的核心任务是：给定前面的文本，预测下一个词出现的概率。

这是一个概率分布建模问题：

P(wt | w1, w2, ..., wt-1)

例如给定 "今天天气"，模型需要计算出下一个词是 "很"、"不"、"真" 等的概率。

💡 通俗类比：图书管理员猜下一页
想象你是一个图书馆的管理员，读者正在读一本书，翻到某一页问你："下一页可能写什么？"你需要根据前面所有内容来猜测。语言模型就像这个管理员 — 它读过海量书籍（训练数据），对"语言的规律"有深刻理解，所以能给出合理的概率预测。
⚠️ 常见误区：概率最高 = 最正确？
语言模型输出的是概率分布，不是"正确答案"。"今天天气"后面跟"很好"概率高，但"很差"也是合理的。模型选择的是"最可能"而非"最正确"。这在代码生成中尤其危险 — 模型可能生成高概率但逻辑错误的代码。
语言模型 = 概率分布生成器: ┌──────────────────────┐ "今天天气" ───→ │ 语言模型 (LM) │ ───→ P("很") = 0.35 │ │ P("不") = 0.25 "今天天气很" ───→ │ 看到前文，猜下一个 │ ───→ P("好") = 0.40 │ │ P("热") = 0.20 "机器学习" ───→ │ 同一个模型 │ ───→ P("是") = 0.30 └──────────────────────┘ P("很") = 0.22

自回归 (Autoregressive) 生成

自回归生成过程: 输入: "今天天气" ↓ 模型计算概率分布 P("很") = 0.35, P("不") = 0.25, P("真") = 0.15, ... ↓ 采样选择 "很" 输入变成: "今天天气很" ↓ 再次计算 P("好") = 0.40, P("热") = 0.20, P("冷") = 0.15, ... ↓ 采样选择 "好" 输入变成: "今天天气很好" ↓ 重复...直到生成结束符 <EOS>

这个过程的数学本质是链式法则：

P("今天天气很好") = P("今") × P("天"|"今") × P("天"|"今天") × P("气"|"今天天") × P("很"|"今天天气") × P("好"|"今天天气很")
🔬 深入探讨：自回归 vs 掩码预测的本质区别
自回归 (GPT) 每一步都在做条件概率 P(w_t|w_<t)，天然适配生成任务。而 BERT 的掩码预测 P(w_mask|context) 是"填空题" — 它知道上下文但不知道哪个位置需要填，所以不能直接用于逐步生成。这也解释了为什么 BERT 做生成需要复杂的迭代策略，而 GPT 只需简单地一个接一个采样。

1.2 语言模型的演进

💡 通俗类比：语言模型演进就像交通工具
N-gram 像骑自行车 — 只能看眼前几步；RNN/LSTM 像开汽车 — 视野更远但有限；Transformer 像坐飞机 — 鸟瞰全局，每个位置都能直接看到其他所有位置。
时代方法代表上下文长度核心限制
统计语言模型N-gram2-gram, 3-gram2~5 个词稀疏问题，无法泛化
神经语言模型RNN / LSTMWord2Vec, ELMo~100 词长距离依赖丢失，训练慢
预训练语言模型TransformerGPT, BERT, LLaMA4K~10M tokens计算成本高

N-gram 模型的局限

N-gram 只看前 N-1 个词来预测下一个词。比如 3-gram (trigram)：

P("好"|"今天天气很") ≈ Count("今天天气很好") / Count("今天天气很")

问题：很多合理的组合在训练语料中从未出现过，导致概率为 0。这就是稀疏问题。

RNN / LSTM 的改进

循环神经网络 (RNN) 用隐藏状态压缩历史信息：

h_t = f(h_{t-1}, x_t)   # 隐藏状态递归更新
P(w_{t+1}) = softmax(W · h_t)

问题：长距离依赖仍然丢失 — 梯度在反向传播时会指数衰减（梯度消失）。LSTM 用门控机制缓解了这个问题，但效果有限。

Transformer 的突破

2017 年 "Attention Is All You Need" 论文提出了 Transformer：

  • 自注意力：每个位置可以直接关注其他任何位置，解决了长距离依赖
  • 并行训练：不像 RNN 需要逐步计算，所有位置可以同时处理
  • 可扩展性：可以轻松扩展到数十亿参数
三种架构的计算方式对比: RNN (串行): x1 → [h1] → x2 → [h2] → x3 → [h3] → x4 → [h4] └──────────────────────────────────────→ 长距离信息衰减 Transformer (并行): x1 ←→ x2 ←→ x3 ←→ x4 所有位置互相可见，同时计算注意力 CNN (局部并行): [x1 x2] [x2 x3] [x3 x4] 局部窗口并行，需多层才能看到远距离
🔬 深入探讨：为什么 Transformer 能并行？
RNN 的 h_t 依赖 h_{t-1}，必须串行计算。而 Transformer 的自注意力矩阵可以一次性算出所有位置之间的关联分数。代价是内存 — 注意力矩阵的大小是 O(n²)，这就是为什么上下文长度一直是瓶颈。Flash Attention 等技术正是为了优化这个 O(n²) 的内存瓶颈。
⚠️ 常见误区：模型"理解"了语言？
LLM 并不真正"理解"语言 — 它只是在统计上非常擅长预测下一个 token。但有趣的是，要在这个简单任务上做到极致，模型不得不学到很多"看起来像理解"的能力。这引发了一个哲学问题：统计模式匹配到什么程度才算"理解"？

⚠️ 常见误区：涌现能力是"突然出现"的？
2023 年有研究指出，涌现能力可能是评估指标的假象 — 用连续指标（如 BPE perplexity）衡量时，能力提升是平滑的。只有用"全对或全错"的离散指标时，才看起来像突然涌现。这提醒我们：模型能力的评估方式会影响我们对它的认知。
评估方式看起来的表现实际情况
离散指标 (如准确率)突然涌现可能是阈值效应
连续指标 (如 perplexity)平滑提升能力渐进增长
多指标综合更准确避免单一指标偏差
💡 关键洞察
Next Token Prediction 看似简单，但要预测准确，模型必须"理解"语法、语义、世界知识、甚至推理。这是一种自监督学习— 不需要人工标注数据。

1.4 GPT vs BERT: 两种预训练范式

两种预训练范式的注意力模式: GPT (Decoder-Only) — 因果掩码 (只能看左边): ┌───────────────┐ │ ✓ · · · │ token1 只看自己 │ ✓ ✓ · · │ │ ✓ ✓ ✓ · │ │ ✓ ✓ ✓ ✓ │ token4 看所有左边 └───────────────┘ BERT (Encoder-Only) — 双向注意力 (看左右): ┌───────────────┐ │ ✓ ✓ ✓ ✓ │ │ ✓ ✓ ✓ ✓ │ │ ✓ ✓ ✓ ✓ │ │ ✓ ✓ ✓ ✓ │ 每个 token 看到全部 └───────────────┘
特性GPT (Decoder-Only)BERT (Encoder-Only)

💡 为什么 GPT 路线胜出？
CLM (Causal Language Modeling) 只需要从左到右预测下一个词，但这个简单的目标迫使模型学习几乎所有语言知识。BERT 的 MLM 只能做"填空题"，无法直接用于对话和生成。
🔬 深入探讨：Encoder-Decoder 架构去哪了？
T5、BART 等 Encoder-Decoder 模型曾经也很流行（用于翻译、摘要），但如今几乎被 Decoder-Only 取代。原因是：(1) Decoder-Only 通过 in-context learning 可以做几乎所有任务；(2) 工程上更简单，只需维护一套模型；(3) 规模化效果更好。这体现了"简单目标 + 大规模 = 强能力"的规律。

1.5 语言模型的训练目标：最大似然估计

语言模型的训练本质上是一个最大似然估计（Maximum Likelihood Estimation, MLE）问题。给定一个训练语料库 D = {s₁, s₂, ..., sₙ}，我们的目标是找到一组模型参数 θ，使得训练数据在模型下的概率最大化：

θ* = argmaxθ ∏i=1N P(sᵢ | θ) = argmaxθ ∑i=1N log P(sᵢ | θ)

取对数的原因有两个：一是将连乘转化为求和，避免数值下溢；二是对数函数是单调递增的，不会改变最优解的位置。在实践中，我们通常优化的是平均负对数似然（Negative Log-Loss, NLL），也就是交叉熵损失函数。

交叉熵损失函数详解

交叉熵（Cross Entropy）是衡量两个概率分布差异的指标。在语言模型中，真实分布是 one-hot（正确答案概率为1），模型分布是 softmax 输出。交叉熵的计算公式为：

H(p, q) = -∑i p(i) · log q(i)

当真实分布 p 是 one-hot 时，交叉熵简化为 -log q(correct_token)，即模型给正确 token 的概率越大，损失越小。例如如果模型给正确 token 的概率是 0.8，则损失为 -log(0.8) ≈ 0.22；如果概率只有 0.01，则损失为 -log(0.01) ≈ 4.6。

🔬 深入探讨：Perplexity（困惑度）— 语言模型的核心评估指标

Perplexity（PPL）是交叉熵的指数形式：PPL = exp(H) = exp(-1/N ∑ log P(w_t|w_<t))。直觉上，PPL 表示模型在每个位置平均需要从多少个等概率的选项中"猜"出正确答案。PPL=1 意味着模型每次都100%猜对；PPL=100 意味着模型平均需要从100个选项中猜。现代大语言模型在标准测试集上的 PPL 通常在 5~20 之间。

PPL 的数学性质非常优雅：它等于模型分配给测试集的平均概率的倒数。PPL 越低，模型对语言的建模越好。但要注意，PPL 只衡量"预测下一个词"的能力，不直接衡量"生成质量"或"事实准确性"。一个 PPL 很低的模型可能仍然会"一本正经地胡说八道"。

⚠️ 训练不稳定的常见原因

大语言模型训练中经常遇到训练不稳定的问题，表现为 loss 突然飙升（loss spike）或 loss 不再下降。常见原因包括：(1) 学习率过大导致参数更新过猛；(2) 数据质量问题，如含有大量重复文本或异常字符；(3) 梯度爆炸，特别是在深层网络中。解决方案包括降低学习率、加强梯度裁剪、检查数据质量。LLaMA-3 的技术报告中提到，他们在训练过程中多次遇到 loss spike，需要回退到之前的检查点重新训练。

1.6 Scaling Law：规模法则

2020 年 OpenAI 发表的 Scaling Law 论文揭示了一个惊人的规律：语言模型的性能（以 PPL 衡量）与三个因素呈幂律关系：

L(N) ∝ N-αN，L(D) ∝ D-αD，L(C) ∝ C-αC

其中 N 是模型参数量，D 是训练数据量（token 数），C 是计算量（FLOPs），α 是幂律指数。这意味着只要增加参数、数据或计算，模型性能就会持续提升，而且这种提升是可预测的。

Scaling Law 的三条曲线: log(Loss) ↑ |╲ | ╲ ← 更多参数 (N) | ╲ | ╲____ | ╲____ | ╲___________ | | ╲ | ╲ ← 更多数据 (D) | ╲___ | ╲___________ | | ╲ | ╲ ← 更多计算 (C) | ╲__ | ╲___________ └──────────────────────────→ log(N), log(D), log(C) 关键发现： - 三条曲线都是直线（在对数坐标下）→ 幂律关系 - 没有出现"饱和"迹象 → 增加规模总能提升性能 - 但收益递减 → 10倍参数只带来固定比例的 Loss 降低
💡 Scaling Law 的实际意义

Scaling Law 让 AI 实验室可以"算命" — 不需要真正训练一个万亿参数模型，只需在小规模上做实验，拟合幂律曲线，就能预测大模型的性能。这就是为什么 OpenAI、Google、Meta 敢于投入数十亿美元训练大模型 — 因为他们事先就知道结果。Chinchilla 论文进一步发现，在固定计算预算下，数据量和参数量应该等比例增长，这改变了整个行业的训练策略。

🔬 深入探讨：为什么现代模型"过度训练"？

Chinchilla 论文建议参数量和数据量 1:20 的比例，但现代模型（如 LLaMA-3）的数据量远超这个比例。原因是 Chinchilla 的最优比例只考虑了训练成本，没有考虑推理成本。一个 7B 模型训练 15T tokens 虽然不是训练最优，但推理时只需要 7B 参数的计算量，远小于 70B 模型。在大规模部署场景下，推理成本远超训练成本，所以"过度训练"小模型反而是总成本最优的策略。这就是所谓的"推理最优训练"（Inference-Optimal Training）。

⚠️ 语言模型的根本局限

语言模型本质上是一个概率分布估计器，它学到的是训练数据中的统计模式。这意味着：(1) 它无法区分"相关"和"因果" — 只是因为 A 和 B 经常一起出现，不代表 A 导致了 B；(2) 它的知识有截止日期 — 训练数据之外的信息它不知道；(3) 它可能"过度自信" — 对于不确定的问题仍然给出流畅而自信的回答。理解这些局限性是正确使用 LLM 的前提。

🎯 练习：用 HuggingFace 训练一个微型语言模型

用 PyTorch 实现一个简单的 Bigram 语言模型，理解训练过程：

import torch
import torch.nn as nn

class BigramLM(nn.Module):
    """最简单的语言模型：只看前一个 token 预测下一个"""
    def __init__(self, vocab_size, d_model=64):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, d_model)
        self.head = nn.Linear(d_model, vocab_size)
    
    def forward(self, x):
        # x: [batch, seq_len]
        emb = self.embedding(x)        # [batch, seq_len, d_model]
        logits = self.head(emb)        # [batch, seq_len, vocab_size]
        return logits

# 训练循环
model = BigramLM(vocab_size=1000)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

    optimizer.step()                   # 更新参数
    optimizer.zero_grad()
    print(f"Epoch {epoch}: loss = {loss.item():.4f}")

观察：随着训练进行，loss 应该逐渐下降。这个模型虽然简单，但完整体现了语言模型的训练流程。

深入探讨： 从语言模型的核心原理出发，自回归语言模型本质上是一个条件概率分解器。根据概率链式法则P(x₁,x₂,...,xₙ)=∏P(xᵢ|x₁,...,xᵢ₋₁)，模型通过逐token预测来建模整个序列的联合分布。为什么这种自回归分解是主流选择？推导：任意联合分布都可以按链式法则分解，而自回归方式天然支持变长序列生成，每个位置只需关注左侧上下文。数学本质上，这是将高维联合分布的估计转化为一系列条件概率的乘积，使模型能在O(V)时间内完成每步预测（V为词表大小），避免了指数级的联合分布建模复杂度。
