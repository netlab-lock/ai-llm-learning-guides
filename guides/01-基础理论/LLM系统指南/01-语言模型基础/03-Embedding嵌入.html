📐 3. Embedding (嵌入)

3.1 从 Token ID 到向量

Token 是一个整数 ID（如 15339），但神经网络需要的是向量（一列浮点数）。Embedding 就是把 ID 转为向量的过程。

💡 通俗类比：Embedding 就像"学号查档案"
每个学生有一个学号（token ID），学校的花名册（Embedding 矩阵）里每个学号对应一条档案（向量）。查询时不需要计算 — 直接用学号找对应行就行。这就是为什么 Embedding 层是"查表"操作，计算量几乎为零。
Embedding 查表过程: token_id = 42 ──→ E[42] = [0.12, -0.34, 0.56, ...] (d维向量) token_id = 15339 ──→ E[15339] = [0.02, -0.16, 0.09, ...] token_id = 7 ──→ E[7] = [0.45, 0.11, -0.22, ...] 本质: 矩阵乘法的特殊情况 one-hot(42) × E = E[42] 但实际实现中直接用索引取行，比矩阵乘法快得多
⚠️ 常见误区：Embedding 是训练好的"固定"向量？
Embedding 矩阵是模型参数的一部分，会随训练不断更新。同一个 token 在不同模型中的向量完全不同。甚至同一个模型在训练的不同阶段，同一个 token 的向量也在变化。所以不能把某个模型的 Embedding 直接用在另一个模型中。

3.2 语义空间

训练过程中，语义相近的词会在向量空间中聚集：

# 经典的向量运算
vector("king") - vector("man") + vector("woman") ≈ vector("queen")
vector("Paris") - vector("France") + vector("Japan") ≈ vector("Tokyo")

这意味着 Embedding 空间中编码了丰富的语义关系。

语义空间中的向量关系 (简化为2维示意): ↑ 维度2 | | queen ● ● king | ↗ (性别方向) | ● woman ● man | | Tokyo ● ● Paris | ↗ (首都方向) | ● Japan ● France | └──────────────────────→ 维度1 king - man + woman ≈ queen 含义: "王"减去"男性"加上"女性"≈ "女王"
💡 通俗类比：向量空间就像"语义地图"
想象一张地图，语义相近的词会聚集在同一区域。"猫"和"狗"在宠物区，"汽车"和"飞机"在交通工具区。向量之间的方向代表"关系" — 从"猫"到"小猫"的方向，和从"狗"到"小狗"的方向是平行的，因为都表示"成年→幼年"的关系。
🔬 深入探讨：LLM 中的 Embedding 为什么不如 Word2Vec "干净"？
Word2Vec 的全部训练目标就是学好词向量，所以语义关系非常清晰。而 LLM 的 Embedding 只是整个模型的一小部分 — 它需要编码的信息远不止单词语义，还要编码位置、上下文、甚至后续层需要的"快捷信息"。所以 LLM 的 Embedding 在经典类比任务上可能不如 Word2Vec，但在上下文理解上更强。

维度 d 的意义 — 每个 token 用多少个数字来描述: GPT-2: d=768 → 每个 token 用 768 个浮点数表示 像用 768 个属性描述一个人 (身高、体重、年龄、发色...768个维度) LLaMA-3: d=8192 → 每个 token 用 8192 个浮点数表示 描述更精细，但计算量也更大 存储开销: GPT-2 Embedding: 50,257 × 768 × 4 bytes = 148 MB LLaMA-3 Embedding: 128,256 × 8192 × 4 bytes = 4.0 GB
⚠️ 常见误区：维度越大越好？
更大的维度确实能编码更多信息，但也带来更高的计算和内存开销。而且当维度大到一定程度后，性能提升会递减。LLaMA-2 7B 用 4096 维，而 GPT-2 124M 用 768 维 — 但提升不仅仅来自维度增大，更来自训练数据量和模型架构的改进。

3.4 权重共享 (Weight Tying)

很多模型让 Embedding 矩阵和输出层的投影矩阵共享权重：

# 不共享: 两套参数
Embedding: E [V × d]     ← 输入端
Output:    W [d × V]     ← 输出端
总参数: 2 × V × d

# 共享 (Weight Tying): 同一套参数
Output = Embedding^T
总参数: V × d   ← 节省一半！

GPT-2、BERT、LLaMA-1 都使用了 Weight Tying。但 LLaMA-2/3 和 DeepSeek-V3 没有使用 — 因为大模型中 Embedding 的参数占比已经很小了。

💡 为什么大模型不用 Weight Tying？
在 7B 模型中，Embedding 只占 ~2% 参数。共享带来的收益很小，但会限制输入和输出的表达能力。对于大模型，分开训练反而更好。
Weight Tying 的直觉: 不共享 (两个独立矩阵): 输入端: token → E[token] → 向量 → [Transformer层] → 向量 → W → 下一个token概率 (E和W是两个不同的矩阵，各自学习) 共享 (同一个矩阵): 输入端: token → E[token] → 向量 → [Transformer层] → 向量 → E^T → 下一个token概率 (E和E^T是同一个矩阵的正反方向) 直觉: "一个词作为输入时的表示" 和 "预测一个词时的权重" 应该有某种对称性。 如果"猫"的向量擅长预测"毛茸茸"，那么"毛茸茸"的预测权重也应该指向"猫"。
🔬 深入探讨：Positional Embedding — 位置信息从哪来？
Embedding 只编码了"这个词是什么"，但没有编码"这个词在哪个位置"。所以还需要加上 Positional Embedding（位置编码）。Transformer 原论文用正弦函数生成固定的位置编码，现代 LLM 多用 RoPE（旋转位置编码）— 它把位置信息注入到注意力计算中，而不是加到 Embedding 上。这让模型能更好地处理变长序列和外推到更长的上下文。
位置编码方式方法代表模型特点
绝对位置编码正弦函数 / 可学习原始 Transformer, GPT-2简单但外推差
相对位置编码ALiBiBLOOM直接建模相对距离
旋转位置编码RoPELLaMA, Qwen, DeepSeek主流方案，外推性好

3.5 位置编码的数学原理

Transformer 的自注意力机制是"位置无关"的 — 它不知道 token 的顺序。如果不加位置信息，"猫追狗"和"狗追猫"对模型来说是一样的。因此需要额外注入位置信息。

正弦位置编码（Sinusoidal）

原始 Transformer 论文使用正弦和余弦函数生成位置编码：

PE(pos, 2i) = sin(pos / 100002i/d)，PE(pos, 2i+1) = cos(pos / 100002i/d)

其中 pos 是位置，i 是维度索引，d 是模型维度。这种编码方式有两个关键优点：(1) 每个位置的编码都是唯一的；(2) 任意两个位置之间的相对距离可以通过线性变换表示。

正弦位置编码的可视化 (d=8, 只展示前4个维度): 位置 维度0(sin) 维度1(cos) 维度2(sin) 维度3(cos) 0: 0.000 1.000 0.000 1.000 1: 0.841 0.540 0.010 1.000 2: 0.909 -0.416 0.020 1.000 3: 0.141 -0.990 0.030 1.000 4: -0.757 -0.654 0.040 1.000 5: -0.959 0.284 0.050 1.000 ... 特点: - 低维度 (dim 0,1) 变化快 → 编码"近距离"信息 - 高维度 (dim 2,3) 变化慢 → 编码"远距离"信息 - 类似于二进制编码: 低位变化快，高位变化慢

RoPE 旋转位置编码

RoPE（Rotary Position Embedding）是现代 LLM 的主流位置编码方案。它的核心思想是：将位置信息编码为向量空间中的旋转操作。对于位置 m 上的向量 x，RoPE 将其旋转 mθ 角度：

f(x, m) = R(mθ) · x，其中 R 是旋转矩阵

RoPE 的优雅之处在于：两个位置 m 和 n 之间的注意力分数只依赖于相对距离 m-n，而不是绝对位置。这使得模型天然具有相对位置感知能力。

🔬 深入探讨：RoPE 的数学推导

RoPE 的推导从一个目标出发：找到一种位置编码函数 f(x, pos)，使得两个位置的内积只依赖于相对距离：

⟨f(q, m), f(k, n)⟩ = g(q, k, m-n)

在二维情况下，这个条件唯一确定了旋转操作。将二维推广到高维时，RoPE 将向量分成 d/2 对，每对独立旋转。每对使用不同的旋转频率 θᵢ = 10000-2i/d，这与正弦编码的频率设计是一致的。

RoPE 的一个重要扩展是 NTK-aware Scaling — 当需要将上下文长度从 4K 扩展到 128K 时，不是简单地插值位置编码，而是调整旋转频率的基数（从 10000 改为更大的值）。这种技术被 LLaMA-3 和 Qwen2.5 广泛使用。

位置编码数学形式相对位置感知外推能力计算开销
正弦编码sin/cos 函数间接（需学习）差极低
可学习编码参数矩阵无无极低
ALiBi注意力偏置直接好低
RoPE旋转变换直接好（配合NTK）低

3.6 Embedding 的训练与微调

Embedding 矩阵的训练过程值得深入理解。在预训练阶段，Embedding 矩阵与其他参数一起通过反向传播更新。但在某些场景下，我们可能需要对 Embedding 进行特殊处理。

冻结 vs 微调 Embedding

策略适用场景优点缺点
完全冻结领域迁移小训练快，不会过拟合无法适应新领域
完全微调领域迁移大适应性强需要更多数据，可能过拟合
部分微调添加新 token平衡效率和效果需要选择哪些层微调
LoRA 微调资源受限参数效率高表达能力有限
💡 添加新 token 的实践技巧

当需要让模型理解新的特殊词汇（如专业术语、品牌名、代码关键字）时，可以向 tokenizer 添加新 token 并初始化对应的 Embedding 向量。常用的初始化策略是：(1) 用新 token 的子词 Embedding 的平均值初始化；(2) 随机初始化后用少量数据微调。注意：添加大量新 token 后需要重新训练输出层的投影矩阵（如果没有使用 Weight Tying 的话）。

Embedding 的量化压缩

Embedding 层在大模型中可能占用数 GB 显存。量化压缩是一种有效的减小方法：

Embedding 量化方案: 原始 FP32: 每个参数 4 字节 [0.021, -0.156, 0.089, ..., 0.034] × 4 bytes = 4 bytes/param FP16/BF16: 每个参数 2 字节 [0.021, -0.156, 0.089, ..., 0.034] × 2 bytes = 2 bytes/param 精度损失: 极小 INT8 量化: 每个参数 1 字节 scale = max(abs(row)) / 127 [2, -20, 11, ..., 4] × 1 byte = 1 byte/param 精度损失: 小 INT4 量化: 每个参数 0.5 字节 scale = max(abs(row)) / 7 [0, -3, 1, ..., 1] × 0.5 byte = 0.5 byte/param 精度损失: 中等 LLaMA-3 70B Embedding 压缩效果: FP32: 1.05B × 4 bytes = 4.0 GB FP16: 1.05B × 2 bytes = 2.0 GB INT8: 1.05B × 1 byte = 1.0 GB INT4: 1.05B × 0.5 byte = 0.5 GB
⚠️ Embedding 量化的注意事项

Embedding 量化与权重量化不同。由于 Embedding 是通过索引直接查表的，量化误差会直接影响每个 token 的输入表示，误差会在后续所有层中累积。因此，Embedding 通常采用比其他层更保守的量化策略。例如，GPTQ 和 AWQ 等量化方法通常对 Embedding 保持 FP16，只对 Transformer 层做 INT4 量化。

3.7 Embedding 的可视化与分析

理解 Embedding 的最佳方式之一是将其可视化。虽然高维向量无法直接可视化，但可以通过降维技术（如 t-SNE、UMAP）将其投影到 2D 空间。

t-SNE 降维后的 Embedding 空间示意: ↑ 维度2 | 动物区 | 猫 ● 狗 ● | 鸟 ● | 鱼 ● | |───────────────────────── | 国家区 | 中国 ● 美国 ● | 日本 ● 法国 ● | |───────────────────────── | 动作区 | 跑 ● 走 ● | 飞 ● 游 ● | └──────────────────────→ 维度1 观察: - 语义相近的词在空间中聚集 - 不同类别的词形成明显的簇 - 簇之间的距离反映了语义差异
🔬 深入探讨：Embedding 空间中的偏见问题

Embedding 空间不仅编码了语义信息，还编码了训练数据中的社会偏见。经典研究发现，Word2Vec 的 Embedding 空间中存在性别偏见 — "医生"的向量更接近"男人"，"护士"的向量更接近"女人"。这种偏见会传递到下游任务中，导致模型在简历筛选、贷款审批等场景中产生歧视性结果。现代 LLM 的 Embedding 同样存在这个问题，但通过更大规模和更多样化的训练数据，偏见程度有所减轻。在实际应用中，需要对 Embedding 进行偏见检测和缓解。

🎯 练习：用 t-SNE 可视化 Embedding 空间

加载预训练模型的 Embedding，用 t-SNE 降维并可视化：

import torch
from transformers import AutoModel, AutoTokenizer
from sklearn.manifold import TSNE
import matplotlib.pyplot as plt

# 加载模型
tokenizer = AutoTokenizer.from_pretrained("bert-base-chinese")
model = AutoModel.from_pretrained("bert-base-chinese")

# 获取 Embedding 矩阵
embedding_matrix = model.embeddings.word_embeddings.weight.detach().numpy()

# 选择一些有趣的词
words = ["猫", "狗", "鸟", "鱼", "汽车", "飞机", "火车", 
         "国王", "女王", "男人", "女人", "北京", "东京", "巴黎"]
ids = [tokenizer.convert_tokens_to_ids(w) for w in words]
vectors = embedding_matrix[ids]

# t-SNE 降维
tsne = TSNE(n_components=2, random_state=42, perplexity=5)
coords = tsne.fit_transform(vectors)

# 可视化
plt.figure(figsize=(10, 8))
for i, word in enumerate(words):
    plt.scatter(coords[i, 0], coords[i, 1])
    plt.annotate(word, (coords[i, 0], coords[i, 1]), fontsize=12)
plt.title("Embedding 空间 t-SNE 可视化")
plt.show()

  2. 如果要为一个中文医疗领域模型添加 500 个专业术语 token，如何初始化这些 token 的 Embedding 向量？
  3. 对比 Word2Vec 和 LLM Embedding 在语义类比任务上的表现差异，分析原因。
✅ 参考答案与详解
  1. 核心原理: Embedding 层的计算量几乎为零，因为它本质上是一张查找表（Lookup Table）而非矩阵乘法。Embedding 矩阵 E 的形状为 [vocab_size × d_model]，其中每一行对应一个 token 的向量表示。给定 token_id（如 15339），Embedding 操作直接用索引取矩阵的第 15339 行，即 E[15339]，得到一个 d 维向量。这个操作的时间复杂度是 O(d)，与词汇表大小 V 无关——无论词表有 5 万还是 15 万个 token，取一行的时间都一样。从数学角度看，Embedding 等价于 one-hot 向量与 Embedding 矩阵的乘法：one_hot(15339) × E = E[15339]。one-hot 向量只有一个位置为 1，其余为 0，所以矩阵乘法退化为简单的行选取。但实际实现中，框架（如 PyTorch 的 nn.Embedding）直接用索引操作替代矩阵乘法，避免了构造 one-hot 向量和执行完整的矩阵乘法，从而将计算开销降到最低。相比之下，Transformer 层中的线性变换 W × x 需要 O(d²) 的计算量，注意力机制需要 O(n² × d)，而 Embedding 只需 O(d)。这也是为什么 Embedding 层在大模型中虽然参数量可能很大（LLaMA-3 70B 的 Embedding 有 1.05B 参数），但训练和推理时的计算开销可以忽略不计。
    关键要点: ① Embedding 本质是索引取行操作，非矩阵乘法 ② 数学上等价于 one-hot × E，但实现为直接索引 ③ 时间复杂度 O(d)，与词表大小 V 无关 ④ 相比 Transformer 层的 O(d²) 和注意力的 O(n²d)，计算量可忽略 ⑤ 参数量大但计算量小，是"大存储低计算"的典型
    实际案例: 在批量推理中，输入 shape 为 [batch_size, seq_len]，Embedding 操作将其转为 [batch_size, seq_len, d_model]。假设 batch=32, seq_len=512, d=4096，Embedding 只需 32×512=16384 次索引操作，而后续 Transformer 层每层需要约 32×512×4096×4096 ≈ 10¹² 次浮点运算。
    关联知识: Embedding 的查表特性使其成为连接 Tokenization（离散 ID）和 Transformer（连续向量）的桥梁。权重共享（Weight Tying）让 Embedding 矩阵与输出层投影矩阵共享，在小模型中可节省约一半参数。位置编码（RoPE/正弦编码）需要与 Embedding 向量相加或在注意力中注入，补充 token 的位置信息。
  2. 核心原理: 为中文医疗领域模型添加 500 个专业术语 token 的 Embedding 初始化是一个典型的领域适配问题。初始化策略有三种：(1) 子词组合初始化——将新术语用现有 tokenizer 拆分为子词，取这些子词 Embedding 的平均值作为新 token 的初始向量。例如"阿莫西林"拆为["阿","莫","西","林"]，取这四个子词向量的均值。这种方法保留了子词级别的语义信息，收敛最快，通常比随机初始化快 2-3 倍。(2) 随机初始化后微调——用与模型 Embedding 相同分布（均值 0，标准差与模型一致）的随机向量初始化，然后用领域语料微调。适用于术语无法被现有子词有效拆分的情况。(3) 预训练模型特征初始化——用领域知识图谱或医疗文本中术语的上下文 Embedding（通过模型前向传播获取上下文相关的 hidden state）作为初始化。添加 token 的具体步骤：先用 tokenizer.add_tokens(["阿莫西林", ...]) 添加到词表，然后通过 model.resize_token_embeddings(len(tokenizer)) 扩展 Embedding 矩阵，新行自动初始化为正态分布。然后手动用子词均值覆盖这些行。注意：如果模型使用了 Weight Tying，添加 token 后还需更新输出层的 lm_head；如果没有 Weight Tying，输出层投影矩阵也需要同步扩展。添加 500 个 token 在 d=4096 的模型中增加约 500×4096×2 bytes ≈ 4MB 参数，影响很小。
    关键要点: ① 推荐子词均值初始化，收敛比随机初始化快 2-3 倍 ② 使用 tokenizer.add_tokens() + model.resize_token_embeddings() 添加 ③ 需检查 Weight Tying，同步更新 lm_head ④ 初始化后用领域语料微调新 token 的 Embedding ⑤ 500 个 token 增加参数量约 4MB，影响可忽略
    实际案例: 在中文医疗问答系统中，添加"阿莫西林"、"布洛芬"、"头孢克肟"等 500 个药品名 token。若不添加，这些术语会被拆成多个无意义的子词碎片（如"阿莫西林"→["阿","莫","西","林"]4 个 token），不仅浪费上下文窗口，还可能因子词组合的歧义导致理解错误。添加后每个术语映射为单个 token，模型可以通过微调学习其专业语义。
    关联知识: 添加 token 涉及 Embedding 层参数量变化（V×d 增加），与词汇表大小权衡相关。微调新 token 的 Embedding 时，冻结策略（完全冻结/部分微调/LoRA）的选择影响训练效率。Embedding 量化（FP16/INT8）在添加新 token 后需要重新校准量化参数。
  3. 核心原理: Word2Vec 和 LLM Embedding 在语义类比任务上的表现差异源于训练目标的根本不同。Word2Vec 的全部训练目标就是学习高质量的词向量——无论是 Skip-gram（用中心词预测上下文）还是 CBOW（用上下文预测中心词），模型的每一层参数都在优化同一个目标：让语义相近的词在向量空间中距离更近。因此 Word2Vec 在经典类比任务（如 king - man + woman ≈ queen）上表现非常出色，向量空间中的语义关系非常清晰。而 LLM 的 Embedding 只是整个模型的一个输入层——它需要编码的信息远不止单词语义，还要编码位置信息、上下文依赖、甚至后续 Transformer 层需要的"快捷信息"。LLM 的训练目标是自回归预测下一个 token（CLM），Embedding 矩阵通过反向传播与其他所有参数一起更新，其优化方向由整个模型的损失函数决定，而非专门为词向量质量优化。此外，Word2Vec 的每个词只有一个固定向量（静态 Embedding），而 LLM 的 Embedding 也是静态的，但经过 Transformer 层后会生成上下文相关的动态表示（contextualized embedding），这才是 LLM 真正强大的地方。例如"苹果"在 Word2Vec 中只有一个向量，而在 LLM 中，经过 Transformer 层处理后，"吃苹果"和"苹果手机"中的"苹果"会有完全不同的表示。
    关键要点: ① Word2Vec 专门为词向量优化，类比任务表现好 ② LLM Embedding 是模型的一小部分，需编码多种信息 ③ LLM 的真正优势在于上下文相关的动态表示（Transformer 层输出） ④ 静态 Embedding 类比：Word2Vec 胜；上下文理解：LLM 胜 ⑤ 不同模型的 Embedding 空间不可直接比较
    实际案例: 在"king - man + woman ≈ queen"类比任务中，Word2Vec 的准确率可达 70-80%，而 LLM 的原始 Embedding（不经过 Transformer 层）准确率可能只有 40-50%。但如果使用 LLM 最后一层的 hidden state，上下文相关的表示在下游任务（如文本分类、问答）上的表现远超 Word2Vec。
    关联知识: Embedding 的质量直接影响 Transformer 层的输入质量。位置编码（RoPE）补充了 Embedding 缺失的位置信息。权重共享（Weight Tying）建立了输入 Embedding 与输出预测之间的对称关系。Embedding 的量化压缩（FP16→INT8→INT4）在部署时需要权衡精度和内存占用。
向前串联： Embedding 输出的向量将作为 Transformer 的输入，经过 Self-Attention 和 FFN 层逐步加工。理解 Embedding 的维度和语义空间特性，有助于理解后续章节中注意力机制如何利用这些向量进行信息交互。
深入探讨： Embedding 空间的维度选择遵循信息论原则：每个维度编码约 log₂(V) 比特的信息。当 d 过小时，信息瓶颈限制模型表达能力；当 d 过大时，参数冗余导致过拟合风险增加。经验法则：d 与训练数据量的对数成正比，与词汇表大小的对数成正比。
常见误区： 不要直接把不同模型的 Embedding 向量做比较或迁移。即使两个模型使用相同的 tokenizer，由于训练数据和架构不同，它们的 Embedding 空间完全不可比较。同理，t-SNE 降维后的可视化结果不能跨模型对比。
常见误区： Embedding 量化时不要对所有层使用相同的量化策略。Embedding 是每个 token 的"第一印象"，量化误差会逐层放大。建议对 Embedding 保持 FP16，只对 Transformer 层做 INT4 量化（如 GPTQ/AWQ 的标准做法）。
相关章节： Embedding 是 Transformer 的输入层，与 Self-Attention（如何利用 Embedding 向量计算相关性）、位置编码（如何补充位置信息）形成完整的输入表示。建议结合学习形成完整知识体系。
