🏔️ 13 · ResNet — 残差连接，突破深度极限

2015 · Kaiming He 等 (Microsoft) · 152层CNN · 残差连接影响了所有后续深度网络

一、背景与动机

2015年，VGG 证明了19层CNN比8层更好，但更深的网络(30+层)反而性能下降——不是过拟合(训练误差也高)，而是退化问题(Degradation Problem)：更深的网络更难优化。

Kaiming He 的洞察：如果加一层什么都不做(恒等映射)不会变差，那为什么加一层反而变差了？因为让网络学习恒等映射 H(x)=x 本身很难。解决方案：让网络学习残差 F(x)=H(x)-x，即学习"和恒等映射的差"——这比直接学习 H(x)=x 容易得多。

💡 核心思想：不用学习"输出=输入"，只需学习"输出=0"——这太容易了！残差连接让网络在最坏情况下也不会比浅层网络差。

二、名字由来

💡 通俗类比：ResNet(Residual Network)的"Residual"是"残差"的意思。核心思想：不直接学习目标映射 H(x)，而是学习"残差" F(x) = H(x) - x，然后输出 F(x) + x。这就像"不重写文章，只修改需要改的地方"。
⚠️ 常见误区：残差连接不是"简单地把输入加到输出上"。它是 H(x) = F(x) + x——F(x) 是"需要学习的变化"，x 是"恒等映射"。如果网络认为"不需要改变"，F(x) 可以轻松学到0——这比学到恒等映射容易得多。
🔬 深入探讨：ResNet(2015)解决了"退化问题"——更深的网络反而更难训练。这不是过拟合(训练误差也更高)，而是优化困难。残差连接让损失曲面变得更"平滑"，优化器更容易找到好解。

ResNet：Res(Residual, 残差) + Net(网络)。

Residual(残差/残差)：统计学中指观测值与预测值之差。这里指"输出与输入之差"——F(x) = H(x) - x。恒等映射 x 是"直连通道"(shortcut/skip connection)。

三、核心原理

💡 通俗类比：残差学习就像"修路"——如果路已经不错了，你只需要做小修补(残差≈0)；如果路很差，你只需要重点修坏的部分。这比重建整条路(学习完整映射)容易得多。
⚠️ 常见误区：很多人以为残差连接"只是让梯度更容易流动"。这确实是一个好处，但更深层的原因是：残差连接让网络可以选择性地"跳过"不需要的层——每层只需要学习"增量改进"。
🔬 深入探讨：ResNet 的论文中有一个关键实验：把一个训练好的 ResNet 中间层删掉，性能几乎不变。这说明很多层学到了"接近恒等"的映射——残差连接让网络可以"选择不用"某些层。这种"自适应深度"是残差连接的深层优势。

3.2 为什么残差有效？

梯度传播对比: 普通网络: ∂L/∂x = ∂L/∂H · ∂H/∂x ← 梯度经过多个层的乘法，容易消失 残差网络: ∂L/∂x = ∂L/∂H · (∂F/∂x + 1) ↑ +1 保证梯度至少为 1，不会消失！ 即使 F(x) 的梯度为 0，还有 shortcut 的梯度 1 → 信息和梯度可以"无损"地跳过任何层

四、技术细节

💡 通俗类比：Bottleneck 结构(1×1→3×3→1×1)就像"漏斗"——先用1×1降维(减少计算)，再用3×3做空间卷积，最后用1×1升维(恢复维度)。这比直接做3×3高效得多。
⚠️ 常见误区：ResNet 的 Batch Normalization(BN)层放在卷积之后、激活之前(不是之后)。这个顺序很重要——先归一化再激活，才能保证每层输入的分布稳定。放错顺序会严重影响性能。
🔬 深入探讨：为什么 ResNet 能训练到152层而 VGG 只能到19层？数学上，残差连接让反向传播的梯度可以"跳过"中间层直接回传——∂L/∂x = ∂L/∂H · (∂F/∂x + 1)。这个"+1"确保梯度永远不会完全消失。

4.1 Projection Shortcut

当输入和输出维度不匹配时(如通道数变化)，用 1×1 卷积做投影：

y = F(x, W) + Wₛ·x (Wₛ 是 1×1 卷积投影)

4.2 Pre-Activation ResNet

2016年 Kaiming He 提出改进：把 BN 和 ReLU 放到卷积前面(BN-ReLU-Conv)，效果更好。

五、使用场景

💡 通俗类比：ResNet 就像"高速公路的匝道"——主路(残差路径)畅通无阻，辅路(卷积路径)负责"调整方向"。如果不需要调整，车直接走主路；需要调整时，辅路发挥作用。
⚠️ 常见误区：不是所有任务都需要很深的网络。ResNet-18(11M参数)在很多任务上就够用了。用 ResNet-152(60M参数)处理简单分类任务是浪费——选择合适的深度很重要。
🔬 深入探讨：残差连接的思想已经"无处不在"——Transformer 的每个子层都有残差连接(Add & Norm)，U-Net 的跳跃连接本质上也是残差连接。这个"简单而深刻"的设计已经成为深度学习的"标准配置"。
  • 图像分类：ImageNet SOTA 保持多年
  • 目标检测：Faster R-CNN、YOLO 的标准骨干网络
  • 语义分割：FCN、DeepLab 的特征提取器
  • 人脸识别：ArcFace 用 ResNet 做 backbone
  • 风格迁移、超分辨率：残差块用于各种图像处理网络
  • Transformer 中：每个 Attention/FFN 层都用残差连接

六、相似技术对比

💡 通俗类比：ResNet vs DenseNet vs PlainNet 就像三种"团队协作"方式——PlainNet 是"流水线"(信息只往前传)，ResNet 是"跳级"(可以直接跳到后面)，DenseNet 是"全连接"(每个人都能看到前面所有人的输出)。
⚠️ 常见误区：DenseNet 的"密集连接"比 ResNet 的"跳跃连接"更激进——每层都能访问所有前面层的输出。但 DenseNet 的内存消耗远大于 ResNet——"更多信息共享"的代价是"更多内存"。
🔬 深入探讨：残差连接的成功启发了一个更一般的"高速公路网络"(Highway Network)概念——用门控机制控制"多少信息走捷径，多少信息走变换"。这与 LSTM 的门控思想异曲同工——"控制信息流动"是深度学习的核心设计模式。

七、关联技术

💡 通俗类比：ResNet 与 BatchNorm 的关系就像"盐和胡椒"——几乎总是一起使用。BN 稳定训练，残差连接缓解退化，两者相辅相成，共同让深层网络成为可能。
⚠️ 常见误区：ResNet 的预训练权重在迁移学习中非常有用——但要注意，ImageNet 预训练假设输入是224×224的RGB图像。如果你的数据不同(如医学灰度图、高光谱图)，需要调整第一层。
🔬 深入探讨：ResNet 论文(2015)是计算机视觉领域引用量最高的论文之一(超过17万次)。它的影响力不仅在于"解决了退化问题"，更在于改变了深度学习的"可训练深度上限"——从20层提升到1000+层。

  • ViT(模型19)：Transformer 的 FFN 层用残差连接

九、残差连接的数学分析

💡 数学直觉：残差连接的本质是将学习目标从"映射 H(x)"变为"学习残差 F(x) = H(x) - x"。当最优映射接近恒等映射时(深层网络中很常见)，学习"接近零的残差"比学习"接近恒等的映射"容易得多。

9.1 梯度传播分析

标准网络的梯度传播：∂Loss/∂xₗ = ∂Loss/∂x_L · Πᵢ₌ₗᴸ ∂Fᵢ/∂xᵢ。连乘项导致梯度消失或爆炸。有残差连接时：∂Loss/∂xₗ = ∂Loss/∂x_L · (1 + ∂ΣFᵢ/∂xₗ)。其中的"1"保证了梯度至少为 1，不会消失！这就是残差连接能训练极深网络的根本原因。

🔬 数学推导：设残差块为 x_{l+1} = x_l + F(x_l)。展开到 L 层：x_L = x_l + Σ_{i=l}^{L-1} F(x_i)。对 x_l 求梯度：∂x_L/∂x_l = 1 + Σ_{i=l}^{L-1} ∂F(x_i)/∂x_l。即使 ∂F/∂x 很小，"1"仍然保证梯度不会完全消失。同时，如果 F 恒为零(最坏情况)，网络退化为恒等映射，不会比浅层网络更差。

9.2 残差块的变体

残差块有多种设计变体：Basic Block(两个 3×3 卷积)用于 ResNet-18/34，Bottleneck Block(1×1 + 3×3 + 1×1 卷积)用于 ResNet-50/101/152。Bottleneck 通过 1×1 卷积先降维再升维，减少了计算量。

Pre-Activation ResNet(He, 2016)将 BN 和 ReLU 移到卷积之前：x_{l+1} = x_l + F(BN(ReLU(x_l)))。这种顺序对训练更深的网络更友好，因为恒等映射路径完全不经过非线性变换。

残差块变体对比: Basic Block (ResNet-18/34): x ──┬──→ [3×3 Conv → BN → ReLU → 3×3 Conv → BN] ──→ + → ReLU → output │ F(x) ↑ └──────────────────────────────────────────────────┘ 恒等映射 Bottleneck Block (ResNet-50/101/152): x ──┬──→ [1×1 Conv → BN → ReLU] 降维(d→d/4) │ [3×3 Conv → BN → ReLU] 空间卷积 │ [1×1 Conv → BN] 升维(d/4→d) │ F(x) ↑ └──────────────────────────────────────────────────┘ Pre-Activation Block: x ──┬──→ [BN → ReLU → 1×1 Conv → BN → ReLU → 3×3 Conv → 1×1 Conv] ──→ + → output │ F(x) ↑ └─────────────────────────────────────────────────────────────────────┘

9.3 残差连接与梯度缩放

在实践中，残差连接有时会配合一个缩放因子 α：x_{l+1} = α · x_l + F(x_l)。在初始化阶段使用 α < 1(如 0.1)可以防止初始阶段特征值过大，稳定训练。随着训练进行，α 逐渐增大到 1。

十、残差连接在 Transformer 中的应用

💡 关键联系：Transformer 的每个子层(自注意力、前馈网络)都使用了残差连接：output = LayerNorm(x + Sublayer(x))。没有残差连接，Transformer 根本无法训练。

10.1 Pre-Norm vs Post-Norm

原始 Transformer 使用 Post-Norm：x + Sublayer(x) → LayerNorm。GPT 系列使用 Pre-Norm：x + Sublayer(LayerNorm(x))。Pre-Norm 的梯度传播更稳定，训练更容易收敛，但最终性能可能略低于精心调参的 Post-Norm。

为什么 Pre-Norm 更稳定？因为 Pre-Norm 中残差路径完全不经过 LayerNorm，梯度可以直接从深层传到浅层。Post-Norm 中梯度需要经过 LayerNorm，可能导致梯度幅值变化。DeepNet (2022) 通过特殊的初始化方法使 Post-Norm 可以训练 1000 层 Transformer。

⚠️ 设计选择：大多数现代大模型(LLaMA、GPT、PaLM)使用 Pre-Norm，因为训练稳定性在大规模训练中比微小的性能差异更重要。一个训练不稳定的模型根本无法收敛，而 Pre-Norm 保证了训练的稳定性。

10.2 残差连接与网络深度

残差连接使得训练任意深度的网络成为可能，但这不意味着更深总是更好。ResNet 论文的实验表明：1202 层 ResNet 的测试误差高于 110 层，原因是过拟合。网络深度的选择需要在表达能力和泛化能力之间权衡。

🔬 深入探讨：残差连接可以看作是一种"集成学习"——L 层残差网络等价于 2^L 个不同深度的路径的隐式集成。每条路径通过不同的残差块组合，最终输出是所有路径的平均。这解释了为什么残差网络具有更好的泛化能力——它天然地进行了路径级别的 Dropout 正则化。

12.0 残差连接的代码实现

# PyTorch BasicBlock
import torch, torch.nn as nn

class BasicBlock(nn.Module):
    expansion = 1
    def __init__(self, in_ch, out_ch, stride=1):
        super().__init__()
        self.conv1 = nn.Conv2d(in_ch, out_ch, 3, stride, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_ch)
        self.conv2 = nn.Conv2d(out_ch, out_ch, 3, 1, 1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_ch)
        self.shortcut = nn.Sequential()
        if stride != 1 or in_ch != out_ch:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_ch, out_ch, 1, stride, bias=False),
                nn.BatchNorm2d(out_ch))
    def forward(self, x):
        out = torch.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += self.shortcut(x)
        return torch.relu(out)
# Transformer Pre-Norm 残差连接
class TransformerBlock(nn.Module):
    def __init__(self, d_model, nhead, d_ff):
        super().__init__()
        self.attn = nn.MultiheadAttention(d_model, nhead, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model))
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
    def forward(self, x):
        x = x + self.attn(self.norm1(x), self.norm1(x), self.norm1(x))[0]
        x = x + self.ffn(self.norm2(x))
        return x
# 残差连接梯度验证
x = torch.randn(1, 64, requires_grad=True)
F_x = torch.nn.Linear(64, 64)(x)
y = x + F_x; y.sum().backward()
print(f"残差梯度范数: {x.grad.norm():.4f}")
x.grad = None
y2 = F_x; y2.sum().backward()

  1. ResNet中残差块的公式是y = F(x) + x。请从反向传播角度证明：即使∂F/∂x趋近0，梯度∂L/∂x也不为0。这对训练超深网络有什么意义？
  2. 为什么更深的普通网络(如56层plain)比浅层网络(如22层plain)表现更差？这不是过拟合而是优化困难。请解释"退化问题"(degradation problem)并分析残差连接如何解决。
  3. ResNet的Bottleneck结构是1×1降维→3×3卷积→1×1升维。请计算Bottleneck结构与直接使用3×3卷积的参数量之比，说明Bottleneck如何同时降低参数和增加深度。
  4. 残差连接不仅存在于ResNet，还出现在Transformer(每个子层都有残差)、GPT、甚至Mamba中。为什么残差连接如此普及？请从信号传播和优化的角度分析。
  5. Pre-Activation ResNet将BatchNorm+ReLU放在卷积层之前。从梯度传播角度分析，为什么Pre-Activation比原始ResNet(激活在残差分支内)梯度流动更顺畅？
✅ 参考答案与详解
  1. 核心原理: 反向传播中∂L/∂x = ∂L/∂y · ∂y/∂x = ∂L/∂y · (∂F/∂x + 1)。当∂F/∂x≈0时，∂L/∂x ≈ ∂L/∂y · 1 = ∂L/∂y——梯度无损流过残差连接！即使残差分支F没有学到有用信息，梯度也能通过恒等映射无损回传，保证深层权重的有效更新。这就是为什么ResNet可以训练152层甚至1000+层的关键：残差连接确保至少有一条"梯度高速公路"存在。
    关键要点: (1)∂L/∂x = ∂L/∂y · (∂F/∂x + 1)；(2)即使F无学习梯度也不消失；(3)恒等映射提供梯度高速公路；(4)支持超深网络训练；(5)是通用设计范式。
    实际案例: ResNet-152比普通152层plain网络错误率低约3%，证明了退化不是过拟合而是优化困难。
  2. 核心原理: 退化问题是：更深的plain网络训练误差和测试误差都高于浅层网络。这不是过拟合(训练误差也高)，而是深层网络更难优化——梯度消失使底层无法学习到好的特征。理论上，如果深层网络的浅层能学到恒等映射，深层网络不应比其浅层版本差。但在plain网络中，让多层非线性层学习恒等映射很困难(需要精确的权重配置)。残差连接直接提供恒等映射路径：即使所有残差分支F都学习0，网络退化为浅层的恒等映射，性能至少不降。
    关键要点: (1)退化是指训练误差也升高；(2)不是过拟合是优化困难；(3)学习恒等映射困难；(4)残差连接自然提供恒等映射；(5)至少不差于浅层版本。
    实际案例: He等人(2015)发现56层plain比22层plain训练误差高1.7%。添加残差连接后56层ResNet训练误差大幅降低。
  3. 核心原理: 对于输入/输出均为256通道的3×3卷积：参数量=256×256×9=589,824。Bottleneck(256→64→64→256)：1×1降维(256×64=16,384)+3×3卷积(64×64×9=36,864)+1×1升维(64×256=16,384)=69,632。Bottleneck仅为直接3×3的11.8%！ResNet-50有16个Bottleneck块，使其可在保持高表达力的同时达到50层深度。
    关键要点: (1)Bottleneck参数量为直接3×3的11.8%；(2)可堆叠更深；(3)先降维再计算；(4)ResNet-50有16个Bottleneck块；(5)参数效率和深度兼得。
    实际案例: ResNet-50使用Bottleneck仅2500万参数，性能远超VGG-16的1.38亿参数。
  4. 核心原理: 残差连接普及的原因：(1)梯度传播：提供无损梯度路径，解决深层优化问题；(2)信号传播：输入信号可跳过非线性变换直接传递到深层，使各层只需学习残差修正；(3)优化视角：残差参数化使优化目标变为学习F(x)=H(x)-x，若最优映射接近恒等则F≈0更易学习；(4)集成视角：ResNet可视为多个深度子网络的集成(veit等人证明)。这些优势使残差连接成为深度学习的标配。
    关键要点: (1)无损梯度传播；(2)信号直接通过；(3)残差学习更易优化；(4)隐式网络集成；(5)几乎所有深度模型的标配。
    实际案例: Transformer每个子层(attention+FFN)都有残差连接。Mamba的MLP有残差。甚至浅层的Transformer(如GPT-3)也依赖残差连接稳定训练。
  5. 核心原理: 原始ResNet中残差路径：x → BN→ReLU→Conv → BN→ReLU→Conv → +x。梯度需经过两个BN和ReLU的非线性变换才能到达浅层。Pre-Activation将BN和ReLU移到卷积层之前：x → BN→ReLU→Conv → BN→ReLU→Conv → +x。这样梯度通过恒等映射回传时不需要经过任何非线性变换，完全无损。He等人实验证明Pre-Activation ResNet-1001比原始版本错误率低约0.5%，使其成为现代ResNet的标准。
    关键要点: (1)原始版本梯度需通过BN和ReLU；(2)Pre-Activation使梯度完全无损通过；(3)更少的信息瓶颈；(4)深层网络训练更稳定；(5)是现代标准做法。
    实际案例: PyTorch的torchvision.models.resnet50默认使用Pre-Activation结构。几乎所有新实现都采用这种设计。

import torch, torch.nn as nn

class BasicBlock(nn.Module):
    def __init__super().__init__()
        self.conv1 = nn.Conv2d(in_ch, out_ch, 3, stride, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(out_ch)
        self.conv2 = nn.Conv2d(out_ch, out_ch, 3, 1, 1, bias=False)
        self.bn2 = nn.BatchNorm2d(out_ch)
        self.shortcut = nn.Sequential()
        if stride != 1 or in_ch != out_ch:
            self.shortcut = nn.Sequential(nn.Conv2d(in_ch, out_ch, 1, stride, bias=False), nn.BatchNorm2d(out_ch))
    def forward(self, x):
        out = torch.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        return torch.relu(out + self.shortcut(x))

class Bottleneck(nn.Module):
    def __init__super().__init__()
        self.net = nn.Sequential(
            nn.Conv2d(in_ch, mid_ch, 1, bias=False), nn.BatchNorm2d(mid_ch), nn.ReLU(),
            nn.Conv2d(mid_ch, mid_ch, 3, stride, 1, bias=False), nn.BatchNorm2d(mid_ch), nn.ReLU(),
            nn.Conv2d(mid_ch, out_ch, 1, bias=False), nn.BatchNorm2d(out_ch))
        self.shortcut = nn.Sequential() if stride == 1 and in_ch == out_ch else \
            nn.Sequential(nn.Conv2d(in_ch, out_ch, 1, stride, bias=False), nn.BatchNorm2d(out_ch))
    def forward(self, x):
        return torch.relu(self.net(x) + self.shortcut(x))
