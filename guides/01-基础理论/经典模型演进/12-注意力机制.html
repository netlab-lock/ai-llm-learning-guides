👁️ 12 · 注意力机制 (Attention Mechanism)

2014/2015 · Bahdanau / Luong · "让解码器回头看" · Transformer 的核心组件

一、背景与动机

Seq2Seq 的信息瓶颈问题：长句子的所有信息被压缩到一个向量中，信息严重丢失。2014年，Bahdanau 等人提出：不要只看最后一个隐藏态，让解码器在生成每个词时都"回头看"编码器的所有隐藏态，自动决定该关注输入的哪些位置。

这就是注意力机制(Attention Mechanism)——模仿人类翻译时的注意力：翻译到某个词时，重点看原文中对应的词。

💡 一句话总结：Attention 让模型在生成每个输出时，能动态地"聚焦"于输入的不同部分。它解决了信息瓶颈，更重要的是——它后来取代了 RNN 本身(Transformer)。

二、名字由来

💡 通俗类比："注意力"来自认知心理学——人类在看一张图时不会"均匀"看每个像素，而是"注意"最相关的部分(如人脸)。注意力机制让神经网络也学会"选择性关注"。
⚠️ 常见误区："注意力"不是"看到了什么"，而是"看了多少"——它是一个权重分配机制。注意力权重高不等于"看到了"，只是"分配了更多权重"。模型可能分配了高权重但仍然理解错误。
🔬 深入探讨：Bahdanau Attention(2014)和 Luong Attention(2015)是两种经典的注意力变体——区别在于对齐函数(如何计算相关性分数)的不同。这个看似微小的设计选择会显著影响翻译质量。

Attention(注意力)：来自认知心理学——人类视觉系统会聚焦于图像的特定区域(注意力焦点)，而不是同时处理整个视野。

Bahdanau 的论文标题是《Neural Machine Translation by Jointly Learning to Align and Translate》，所以也叫对齐(Alignment)机制。

三、核心原理

💡 通俗类比：Q(Query)、K(Key)、V(Value)就像在图书馆找书——Q是你想查的问题("深度学习")，K是每本书的标签(书脊上的标题)，V是书的内容。注意力就是"根据标签匹配度(Q·K)来决定看哪本书的内容(V)"。
⚠️ 常见误区：注意力分数的计算有多种方式——点积(dot-product)、加性(additive)、缩放点积(scaled dot-product)。Transformer 用的是"缩放点积"——除以√d_k 是为了防止点积值太大导致 softmax 输出接近 one-hot。
🔬 深入探讨：自注意力(Self-Attention)是 Transformer 的核心创新——Q、K、V都来自同一个序列。这让序列中的每个位置都能"看到"所有其他位置——任意距离的依赖关系都能被直接捕获。这是 RNN 做不到的。

3.1 Bahdanau Attention (加性注意力)

注意力计算过程: 编码器隐藏态: h₁, h₂, h₃, h₄ (所有时间步) 解码器当前态: sₜ Step 1: 计算注意力分数(对齐分数) eₜᵢ = vᵀ · tanh(W₁·sₜ₋₁ + W₂·hᵢ) 对每个 i Step 2: Softmax 归一化 → 注意力权重 αₜᵢ = softmax(eₜᵢ) (权重之和=1) Step 3: 加权求和 → 上下文向量 cₜ = Σᵢ αₜᵢ · hᵢ (编码器隐藏态的加权平均) Step 4: 解码时把 cₜ 和 sₜ 拼接 sₜ = LSTM([yₜ₋₁; cₜ], sₜ₋₁) 注意力权重 α 的可视化示例: 输入: "I love deep learning" 翻译: "我 喜欢 深度 学习" α: [.1 .05 .05 .05] ← 生成"我"时主要看"I" [.05 .7 .1 .05] ← 生成"喜欢"时主要看"love" [.05 .05 .8 .05] ← 生成"学习"时主要看"learning"

3.2 Luong Attention (乘性注意力)

2015年 Luong 等人简化了注意力计算：

3.3 Self-Attention (自注意力)

Bahdanau 的注意力是"解码器看编码器"(cross-attention)。Self-Attention 是序列看自己——每个位置都和其他所有位置交互。

Self-Attention: 序列内部的注意力 输入: "The cat sat on the mat" ↑ "sat" 关注: "The": 0.05 "cat": 0.45 ← "sat"的主语是"cat" "sat": 0.30 "on": 0.10 "the": 0.05 "mat": 0.05 Self-Attention 让每个词都能"看到"句子中的所有其他词 → 捕获长距离依赖(不像 RNN 要逐步传播)

3.4 Q, K, V 框架

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V

Query(查询)：当前位置想要什么信息。

Key(键)：每个位置能提供什么信息。

Value(值)：每个位置的实际内容。

注意力分数 = Query 和 Key 的相似度 → 用这个分数对 Value 加权求和。

四、技术细节

💡 通俗类比：多头注意力(Multi-Head)就像"多角度观察"——不是只从一个角度看问题(单头注意力)，而是从多个角度同时看(多头)。有的头关注语法关系，有的关注语义关系，有的关注位置关系。
⚠️ 常见误区：多头注意力的"头数"不是越多越好。头数太多会让每个头的维度(d_k = d_model / h)太小，表达能力不足。实践中，h=8 或 h=16 是常见选择，d_k=64 是经验值。
🔬 深入探讨：注意力的 O(n²) 复杂度是其最大限制——序列长度翻倍，计算量翻四倍。这就是为什么 GPT-4 的上下文窗口(32K→128K)扩展如此困难。FlashAttention、线性注意力等技术都在试图突破这个限制。

五、使用场景

💡 通俗类比：注意力机制已经从NLP扩展到CV(ViT)、语音(Whisper)、蛋白质(AlphaFold)——几乎任何需要"从大量信息中选择相关内容"的场景都可以用注意力。
⚠️ 常见误区：注意力不是"万能的"——在时序预测(时间序列)中，简单的线性模型有时能击败 Transformer。注意力擅长"长距离依赖"，但如果数据本质上只有"短距离依赖"，用注意力就是"杀鸡用牛刀"。
🔬 深入探讨：Vision Transformer(ViT)证明了"注意力不需要CNN的归纳偏置"——直接把图像切成16×16的 patch，当作"词"来处理。这打破了"视觉任务必须用CNN"的共识，也引发了"归纳偏置到底有多重要"的讨论。
  • 机器翻译：Bahdanau 的原始应用
  • 文本摘要：关注原文中最关键的句子
  • 图像描述：在图像不同区域间分配注意力
  • 语音识别：关注音频的不同时间帧
  • 所有 Transformer 任务：Self-Attention 是 Transformer 的核心

六、相似技术对比

💡 通俗类比：注意力 vs 卷积 vs 循环就像三种"看世界"的方式——卷积"局部扫描"(只看附近)，循环"顺序阅读"(一个接一个)，注意力"全局鸟瞰"(同时看所有)。各有优劣，不存在"最好"。
⚠️ 常见误区：不要以为注意力"完全取代"了卷积和循环。ConvMixer、Mamba 等混合架构证明了"结合"往往比"纯粹"更好。技术选择不是"非此即彼"，而是"取长补短"。
🔬 深入探讨：注意力和核方法(Kernel Methods)有深刻联系——softmax(QK^T)V 可以看作一种"注意力核"加权求和。这个联系启发了线性注意力(Linear Attention)——用核近似替代 softmax，把复杂度从 O(n²) 降到 O(n)。

七、关联技术

💡 通俗类比：Transformer 就是"纯注意力"架构——完全用注意力替代了 RNN 和 CNN。"Attention Is All You Need"这个标题就是字面意思——注意力真的够用了。
⚠️ 常见误区：Transformer 的位置编码是必需的！没有位置编码，自注意力是"排列不变"的——"我爱你"和"你爱我"的表示完全相同。位置编码注入了顺序信息。
🔬 深入探讨：注意力机制的"可解释性"是其一大优势——注意力权重可以直接可视化，让我们知道模型"在看哪里"。但近年研究表明，注意力权重不完全等于"重要性"——高注意力不等于高贡献。"可解释的"和"正确的解释"是两回事。
  • Transformer(模型14)：纯 Self-Attention 架构，完全取代 RNN
  • ViT(模型19)：把 Self-Attention 用于图像
  • BERT(模型15)：双向 Self-Attention 预训练
  • Flash Attention：优化注意力计算的 GPU 效率

八、实际效果

⚠️ 常见误区：注意力机制的计算量是 O(n²d)——对于1024长度的序列，注意力矩阵有100万个元素。这就是为什么长上下文(如100K tokens)需要特殊优化——朴素实现完全不可行。

九、注意力机制的数学推导

💡 数学直觉：注意力的本质是"软寻址"——不是从固定位置读取信息，而是根据查询内容动态地从所有位置加权读取。这就像在图书馆找书：硬寻址是记住"第三排第五本"，软寻址是"找关于深度学习的书"，然后根据相关性从每本书中提取信息。

9.1 加性注意力 vs 乘性注意力

注意力函数主要有两种形式：加性注意力(Additive Attention)使用一个单层前馈网络计算对齐分数：score(sᵢ, hⱼ) = v^T tanh(W₁sᵢ + W₂hⱼ)。乘性注意力(Dot-Product Attention)直接用点积计算：score(sᵢ, hⱼ) = sᵢ^T W hⱼ 或简单的 sᵢ^T hⱼ。

理论上两种方式表达能力相近，但乘性注意力在计算上更高效(可以用矩阵乘法并行化)。Luong Attention 主要使用乘性注意力，Bahdanau Attention 使用加性注意力。Transformer 选择了缩放点积注意力(Scaled Dot-Product)，兼顾了效率和稳定性。

9.2 缩放因子 √dₖ 的推导

假设 Q 和 K 的每个元素都是独立同分布的，均值为 0，方差为 1。则点积 Q·K^T 的均值为 0，方差为 dₖ。当 dₖ 较大时，点积值的方差很大，导致 softmax 的输入值过大或过小——梯度趋近于零(softmax 饱和区)。

除以 √dₖ 后，方差归一化为 1，使 softmax 工作在梯度较大的线性区。这是一个简单但关键的设计——没有这个缩放，Transformer 的训练会非常困难。

Attention(Q,K,V) = softmax(QK^T / √dₖ) · V
🔬 数学证明：设 Qᵢ, Kⱼ 的各分量独立同分布，均值 0，方差 1。则 Qᵢ·Kⱼ = Σₖ QᵢₖKⱼₖ，每个乘积的方差为 1，dₖ 个乘积之和的方差为 dₖ。因此需要除以 √dₖ 使方差归一化为 1。当使用掩码时(因果注意力)，被掩码的位置不参与 softmax，有效序列长度变短，但缩放因子不变。

9.3 Multi-Head Attention 的参数量分析

Multi-Head Attention 的参数量为 4 × d_model × d_model(不含偏置)：Q、K、V 各一个投影矩阵(d_model × d_model)，输出一个投影矩阵(d_model × d_model)。当 d_model = 512 时，参数量约为 1M。8 个头并不增加参数量——只是将同一个大矩阵分成 8 个小矩阵。

十、注意力机制的高级变体

10.1 稀疏注意力 (Sparse Attention)

标准注意力的 O(n²) 复杂度限制了处理长序列的能力。稀疏注意力通过限制每个 token 只关注部分位置来降低复杂度。Longformer 使用滑动窗口注意力(局部)+全局注意力(特殊 token)的组合，将复杂度降到 O(n)。BigBird 证明了随机注意力+窗口注意力+全局注意力的组合在理论上是图灵完备的。

10.2 Flash Attention

Flash Attention (2022) 是一种 IO 感知的注意力算法，不是近似方法，而是精确计算。核心思想是利用 GPU 的内存层次结构(HBM vs SRAM)，通过分块(tiling)计算注意力矩阵，减少对高带宽内存(HBM)的读写次数。Flash Attention 使 Transformer 训练速度提升 2-4 倍，内存使用减少 5-20 倍。

⚠️ 重要进展：Flash Attention 已成为现代 Transformer 训练的标配。PyTorch 2.0+ 的 F.scaled_dot_product_attention 已内置 Flash Attention。建议所有新项目都使用 Flash Attention。

10.3 分组查询注意力 (GQA)

Grouped Query Attention (GQA) 是 Multi-Head Attention 和 Multi-Query Attention 的折中。标准 MHA 每个头有独立的 Q、K、V 投影，MQA 所有头共享 K、V。GQA 将头分成 g 组，每组共享 K、V。LLaMA 2 70B、Mistral 等模型使用 GQA，在推理效率和模型质量之间取得了良好平衡。

💡 工程价值：GQA 的主要优势在推理阶段——KV Cache 的内存占用从 O(n·h·d) 降到 O(n·g·d)，其中 g << h。对于 70B 参数模型，GQA 可以将 KV Cache 内存减少 4-8 倍，使得在消费级 GPU 上运行更大模型成为可能。

12.0 注意力机制的代码实现

# PyTorch Scaled Dot-Product Attention
import torch, torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, mask=None):
    d_k = Q.size(-1)
    scores = torch.matmul(Q, K.transpose(-2, -1)) / (d_k ** 0.5)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)
    attn = F.softmax(scores, dim=-1)
    return torch.matmul(attn, V), attn

class MultiHeadAttention(torch.nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.d_k = d_model // num_heads
        self.h = num_heads
        self.W_q = torch.nn.Linear(d_model, d_model)
        self.W_k = torch.nn.Linear(d_model, d_model)
        self.W_v = torch.nn.Linear(d_model, d_model)
        self.W_o = torch.nn.Linear(d_model, d_model)
    def forward(self, Q, K, V, mask=None):
        B = Q.size(0)
        Q = self.W_q(Q).view(B, -1, self.h, self.d_k).transpose(1, 2)
        K = self.W_k(K).view(B, -1, self.h, self.d_k).transpose(1, 2)
        V = self.W_v(V).view(B, -1, self.h, self.d_k).transpose(1, 2)
        out, _ = scaled_dot_product_attention(Q, K, V, mask)
        out = out.transpose(1, 2).contiguous().view(B, -1, self.h * self.d_k)
        return self.W_o(out)
# PyTorch 2.0+ 内置高效注意力
import torch.nn as nn
mha = nn.MultiheadAttention(embed_dim=512, num_heads=8, batch_first=True)
attn_output, attn_weights = mha(query, key, value)
out = F.scaled_dot_product_attention(Q, K, V, attn_mask=mask)

  3. 位置编码在Transformer中是必需的，因为Self-Attention本身是排列不变的(permutation invariant)。请从数学上证明：如果不加位置编码，Self-Attention对输入序列的排列是不变的。正弦位置编码为什么能编码相对位置信息？
  4. Transformer中有两种注意力：编码器Self-Attention和解码器Cross-Attention。请解释两者在mask和Q/K/V来源上的区别。因果掩码(Causal Mask)为什么对自回归语言模型至关重要？
  5. Transformer的自注意力复杂度是O(n²·d)，空间复杂度是O(n²)。当序列长度n很大时，这成为瓶颈。Flash Attention如何通过IO-aware算法在不改变数学结果的前提下降低内存占用？
✅ 参考答案与详解
  1. 核心原理: 假设Q和K的每个分量独立且均值为0方差为1，则(QKᵀ)ᵢⱼ = Σₖqᵢₖkₖⱼ，其方差为dₖ。当dₖ=64时方差为64，值可达±√64=±8。此时softmax输入值落在两端饱和区，梯度趋近0。缩放因子1/√dₖ使QKᵀ的方差为1，softmax输入值在[-3,3]区间内，梯度不会消失。这是Vaswani等人(2017)论文中的推导，确保训练稳定。
    关键要点: (1)QKᵀ方差为dₖ；(2)大dₖ导致softmax饱和；(3)梯度消失；(4)缩放使方差=1；(5)确保训练稳定。
    实际案例: dₖ=64时未缩放的QKᵀ值可能±16以上，softmax输出趋近one-hot(梯度≈0)。缩放后值在±4以内，softmax仍可微分。
  2. 核心原理: Multi-Head Attention参数量：每个头有WᵢQ, WᵢK, WᵢV三个(d_model × dₖ)矩阵和一个WO(d_model × d_model)矩阵。总参数h×3×d_model×dₖ + d_model² = 8×3×512×64 + 512² = 1,048,576。单头(dₖ=512)参数量也为3×512×512+512²=1,048,576(参数相同)。但单头注意力捕获的关系模式有限。多头优势：(1)不同头捕获不同子空间的依赖；(2)一些头关注局部模式，一些关注全局模式；(3)类似CNN的多通道。
    关键要点: (1)参数量相同但表达能力更强；(2)不同头捕获不同依赖；(3)局部和全局模式并行；(4)类似CNN的多通道；(5)增强表达能力。
    实际案例: 可视化显示：BERT的一些头关注前驱词(局部语法)，一些头关注语义关联词，一些头关注句子末尾(全局结构)。
  3. 核心原理: Self-Attention计算A = softmax(QKᵀ/√d)V。如果对输入X做排列PX，则Q=PQ', K=PK', V=PV'，所以A' = softmax(Q'PᵀPK'ᵀ/√d)V' = A(PX = PX)，即输出也被相同排列。所以Self-Attention是排列不变的——它无法区分序列顺序。正弦位置编码P(pos,2i)=sin(pos/10000^(2i/d))能编码相对位置：PE(pos+k)可表示为PE(pos)的线性变换，使模型可学习相对位置关系。
    关键要点: (1)Self-Attention排列不变；(2)PE使模型感知顺序；(3)正弦编码支持外推到更长序列；(4)可学习PE在训练长度内效果更好；(5)RoPE用旋转编码更优。
    实际案例: BERT使用可学习位置编码。GPT系列也使用可学习PE。LLaMA使用RoPE(旋转位置编码)，可外推到训练时未见过的长度。
  4. 核心原理: 编码器Self-Attention中Q=K=V都来自编码器输出(全局双向注意力)。解码器Cross-Attention中Q来自解码器隐状态，K=V来自编码器输出(解码器查询编码器信息)。因果掩码是一个下三角矩阵，Mᵢⱼ=0当ii的答案，模型只需"抄答案"而不学习预测。
    关键要点: (1)Enc Self-Attn双向全局；(2)Dec Cross-Attn查询编码器；(3)Causal Mask阻止偷看未来；(4)训练推理一致性；(5)只看到过去信息。
    实际案例: GPT系列使用Causal Mask在训练时看到前面的词预测下一个词。T5的编码器不使用Mask(双向)，解码器使用Causal Mask。
  5. 核心原理: 标准Self-Attention需存储完整的注意力矩阵S=QKᵀ(N×N，N=序列长度)到HBM，然后再计算P=softmax(S)和O=PV，两次HBM读写。Flash Attention通过分块(tiling)和在线softmax，将S分小块计算，每块在SRAM中完成全部运算后只输出O的分块结果(不存储完整的N×N矩阵)。数学结果与标准Attention完全一致（得益于softmax的缩放不变性），但内存从O(n²)降到O(n)。这使Transformer可处理的序列长度大幅增加。
    关键要点: (1)标准Attention需存储N×N矩阵；(2)分块在SRAM中计算；(3)在线softmax避免存储中间结果；(4)数学结果完全一致；(5)内存从O(n²)降到O(n)。
    实际案例: Flash Attention使GPT-4可处理32K上下文，Flash Attention-2进一步优化了工作分配和IO，速度提升2-4倍。Mamba等模型也借鉴了类似思想。

import torch, torch.nn as nn, math

class ScaledDotProductAttention(nn.Module):
    def forward(self, Q, K, V, mask=None):
        d_k = Q.size(-1)
        scores = Q @ K.transpose(-2,-1) / math.sqrt(d_k)  # (B, H, T, T)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attn = torch.softmax(scores, dim=-1)
        return attn @ V, attn

class MultiHeadAttention(nn.Module):
    def __init__super().__init__()
        self.d_k = d_model // n_heads
        self.n_heads = n_heads
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
        self.attn = ScaledDotProductAttention()

    def forward(self, Q, K, V, mask=None):
        B = Q.size(0)
        # 投影并分头
        q = self.W_q(Q).view(B, -1, self.n_heads, self.d_k).transpose(1, 2)
        k = self.W_k(K).view(B, -1, self.n_heads, self.d_k).transpose(1, 2)
        v = self.W_v(V).view(B, -1, self.n_heads, self.d_k).transpose(1, 2)
        out, weights = self.attn(q, k, v, mask)
        out = out.transpose(1,2).contiguous().view(B, -1, self.n_heads * self.d_k)
        return self.W_o(out), weights

# 验证排列不变性
Q = torch.randn(1, 5, 64)
K = torch.randn(1, 5, 64)
V = torch.randn(1, 5, 64)
attn = ScaledDotProductAttention()
out1, _ = attn(Q, K, V)
# 排列后
perm = [2,0,4,1,3]
out2, _ = attn(Q[:,perm], K[:,perm], V[:,perm])
print("排列不变:", torch.allclose(out1[:,perm], out2, atol=1e-5))  # True

关键知识点: (1)缩放因子1/√dₖ防止softmax饱和；(2)多头并行捕获不同子空间依赖；(3)位置编码打破排列不变性；(4)Causal Mask保证自回归性质；(5)Flash Attention通过分块降低内存。
