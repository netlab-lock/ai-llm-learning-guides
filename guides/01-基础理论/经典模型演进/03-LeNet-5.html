🔢 03 · LeNet-5 — 卷积神经网络的开山之作

1998 · Yann LeCun · 手写数字识别 · 所有 CNN 的鼻祖

一、背景与动机

1990年代，美国邮政系统每天需要分拣数百万封信件，手写邮编的自动识别是一个巨大的商业需求。传统方法用手工设计的特征+分类器，准确率低且泛化差。

Yann LeCun 在 AT&T 贝尔实验室提出了 LeNet-5（1998年论文《Gradient-based learning applied to document recognition》），用端到端学习的方式直接从像素识别手写数字。LeNet-5 的核心创新是：卷积层(Convolutional Layer)——用小的可学习滤波器在图像上滑动，自动提取局部特征。

💡 核心贡献：LeNet-5 定义了 CNN 的经典范式：卷积→池化→卷积→池化→全连接→输出。这个结构影响了此后所有的 CNN 模型（AlexNet、VGG、ResNet），直到 Transformer 出现。
🔬 历史背景：LeNet-5 是 LeNet 系列的第五个版本。LeNet-1(1989)→LeNet-4(1993)→LeNet-5(1998)，逐步加深并改进。它被部署在 ATM 机上用于读取支票上的数字，是最早商业化的深度学习应用之一。但此后十年，因为计算能力限制和 SVM 的兴起，CNN 研究陷入低谷——直到 2012 年 AlexNet 的爆发。

二、名字由来

💡 通俗类比：LeNet 的"Le"来自 Yann LeCun 的姓。就像"特斯拉"汽车以 Nikola Tesla 命名一样，很多深度学习模型以发明者命名——了解命名就是了解学术传承。
⚠️ 常见误区：LeNet-5 的"5"不是指5层，而是第5个版本。LeCun 在1989-1998年间迭代了多个版本(LeNet-1到LeNet-5)。技术迭代不是一步到位的，即使是大师也需要反复试错。
🔬 深入探讨：LeNet-5 的完整架构是：输入(32×32)→C1(6@28×28)→S2(6@14×14)→C3(16@10×10)→S4(16@5×5)→C5(120@1×1)→F6(84)→Output(10)。这个"卷积-池化-全连接"的三明治结构影响了此后20年的CNN设计。

LeNet：Le(LeCun 的姓) + Net(网络)。以发明者 Yann LeCun 命名。

-5：表示这是该系列的第 5 个版本。前四个版本逐步迭代改进架构。

Yann LeCun 后来成为 Meta（Facebook）的首席 AI 科学家，2018 年获图灵奖。

三、核心原理

⚠️ 常见误区：卷积核的值不是预设的，是训练学出来的！很多人以为 Sobel、Laplacian 等手工卷积核就是 CNN 的卷积核——不是的。CNN 的卷积核从随机值开始，通过反向传播自动学习。
🔬 深入探讨：LeNet-5 的 C3 层没有用全连接的卷积——某些输入特征图不连接到某些输出特征图。这个设计减少了参数量，同时迫使不同特征图学习不同的特征。这种"稀疏连接"思想后来演变为分组卷积(Group Convolution)和深度可分离卷积(Depthwise Separable Convolution)。

3.1 卷积操作 (Convolution)

卷积是 CNN 的核心操作。一个 卷积核(kernel/filter)（如 5×5 的小矩阵）在输入图像上滑动，每到一个位置做元素乘法+求和，生成一个输出值。

卷积的三大关键特性：

特性含义效果
局部连接每个输出只看输入的局部区域捕获局部空间模式（边缘、纹理）
参数共享同一个卷积核用于所有位置参数量从 亿级 降到 百级
平移等变性物体移动→特征图也移动同一个物体不管在哪都能识别

3.2 池化操作 (Pooling)

池化层对特征图做下采样，缩小空间尺寸，提取最显著的特征：

池化的作用：① 减少计算量(尺寸减半→后续计算减 75%)；② 提供平移不变性(物体微移不影响最大值)；③ 扩大感受野(后续层能"看到"更大的区域)。

💡 参数效率：如果用全连接层直接处理 32×32 图像到 10 类，需要 32×32×10 = 10,240 个参数(仅输出层)。而 C1 层用 6 个 5×5 卷积核只要 156 个参数，就能提取 6 种不同的局部特征——这就是参数共享的威力。

四、技术细节

💡 通俗类比：参数共享就像"一把钥匙开所有门"——同一个卷积核在图像的所有位置使用相同的参数。这比每个位置用不同参数(全连接)高效得多，也是 CNN 参数量远少于 MLP 的原因。
⚠️ 常见误区：LeNet-5 时代用的激活函数是 Tanh，不是 ReLU。ReLU 要到2011年才被 Nair & Hinton 证明在深度网络中优于 Tanh。不要用现代知识"倒推"历史——每个时代有其最优选择。
🔬 深入探讨：LeNet-5 的参数量约6万(60K)，而现代 ResNet-50 有2500万，GPT-3 有1750亿。从6万到1750亿，参数量增长了约300万倍。但核心思想(分层特征提取、参数共享、非线性激活)一脉相承。

4.1 卷积输出尺寸公式

output_size = (input_size - kernel_size + 2×padding) / stride + 1

LeNet-5 中：C1 层 (32-5+0)/1+1 = 28；S2 层 (28-2)/2+1 = 14

4.2 感受野 (Receptive Field)

深层神经元"看到"的输入区域逐渐增大：

层感受野大小含义
C15×5看到局部边缘和角点
C314×14看到局部形状和纹理
C532×32看到整个输入图像

4.3 特征层级 (Feature Hierarchy)

CNN 的特征层级学习: 浅层(C1): 边缘、线条、角点 ↓ 中层(C3): 纹理、局部形状(弧线、交叉) ↓ 深层(C5): 高级语义(数字的整体结构) ↓ 全连接(F6): 全局判别特征 ↓ 输出: 分类决策 (0-9) 这种从低级到高级的自动特征学习 是 CNN 相比手工特征的最大优势

4.4 激活函数

原始 LeNet-5 使用 tanh 或自定义的 Sigmoid 变体作为激活函数。现代 CNN 统一使用 ReLU(2011年后)。

4.5 输出层：RBF 欧式距离

原始 LeNet-5 的输出层使用 RBF(Radial Basis Function)——计算输出向量与每个数字模板的欧式距离，距离最小的类别胜出。现代实现通常用 Softmax + 交叉熵替代。

五、使用场景

💡 通俗类比：LeNet 之于 CNN，就像莱特兄弟的飞机之于现代航空——原始但革命性。它证明了"让机器自己学特征"比"人工设计特征"更有效，这个范式转变影响了整个AI领域。
⚠️ 常见误区：不要以为 LeNet 已经"过时"就没用了。在嵌入式设备和IoT场景中，LeNet 级别的小模型仍然实用。不是所有问题都需要 ResNet-152——用合适的模型解决合适的问题才是工程智慧。
🔬 深入探讨：LeNet 最初部署在银行系统中读取支票上的手写数字。这是深度学习的第一个大规模商业应用(1990年代末)。有趣的是，30年后的今天，"AI+金融"仍然是最大的应用市场之一。
  • 手写数字识别：MNIST 数据集上达到 ~99.2% 准确率
  • 支票/邮编自动识别：ATM 机上的实际部署
  • 教学入门：理解 CNN 基本概念的最简单模型
  • 特征提取器原型：所有现代 CNN（VGG、ResNet、EfficientNet）的结构设计都源于 LeNet 的 Conv→Pool→FC 范式
  • 1D 卷积应用：将卷积思想扩展到时间序列、音频等 1D 数据

六、相似技术对比

💡 通俗类比：CNN vs MLP 处理图像，就像"看地图"vs"读坐标列表"——CNN 保留空间结构(看地图)，MLP 把一切展平(读坐标)。哪个更直观？当然是看地图。
⚠️ 常见误区：不要以为"更深=更好"是永恒真理。LeNet 只有5层就解决了手写数字识别。模型复杂度应该匹配问题复杂度——杀鸡用牛刀反而可能过拟合。
🔬 深入探讨：LeNet 的"卷积+池化+全连接"结构启发了 VGGNet(更深)、GoogLeNet(更宽)、ResNet(残差)。每个后继者都在 LeNet 的基础上做了一个关键改进。理解 LeNet 就理解了 CNN 的"设计空间"。

七、关联技术

💡 通俗类比：LeNet 到 AlexNet 的距离，就像黑白电视到彩色电视——核心原理相同(电子束扫描)，但规模和效果完全不同。AlexNet 用了同样的卷积+池化结构，但多了 GPU、ReLU、Dropout。
⚠️ 常见误区：很多人以为 CNN 的成功是"自然而然"的。实际上，在 LeNet(1998)和 AlexNet(2012)之间的14年里，SVM 和手工特征(HOG, SIFT)才是主流。深度学习的崛起需要三个条件同时满足：大数据、大算力、好算法。
🔬 深入探讨：池化层(Pooling)正在被现代架构逐渐淘汰。All-Convolutional Network(2015)用步长卷积替代池化，Vision Transformer(2020)完全不用池化。但池化的"不变性"思想——对小位移不敏感——仍然通过其他方式被保留。
  • AlexNet (模型05)：将 LeNet 的思想放大到 ImageNet，用 ReLU+Dropout+GPU 引爆深度学习
  • VGGNet (模型06)：用更小的 3×3 卷积核和更多层继承了 LeNet 的简洁设计哲学
  • ResNet (模型13)：用残差连接解决深层 CNN 训练困难
  • ViT (模型19)：Transformer 替代 CNN，但 patch embedding 的思想可追溯到卷积的局部感受野
  • 池化的衰落：现代网络(GPT、ViT)趋向用 stride convolution 或 patch embedding 替代池化

八、实际效果

⚠️ 常见误区：MNIST 是"太容易"的基准——现代模型在这个数据集上都接近饱和。不要用 MNIST 的成绩来证明一个模型"好"。真正的挑战在 ImageNet(1000类, 224×224)和更复杂的任务上。
🔬 深入探讨：LeNet 的成功不仅在于技术，还在于它找到了正确的应用场景(手写数字识别)。很多优秀的研究之所以默默无闻，不是因为技术不好，而是因为没有找到合适的应用场景。"正确的问题"比"正确的答案"更重要。

九、PyTorch 实现与代码实战

9.1 LeNet-5 完整实现

import torch
import torch.nn as nn
import torch.optim as optim
from torchvision import datasets, transforms
from torch.utils.data import DataLoader

class LeNet5(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 6, kernel_size=5, padding=2),
            nn.Tanh(),
            nn.AvgPool2d(2, 2),
            nn.Conv2d(6, 16, kernel_size=5),
            nn.Tanh(),
            nn.AvgPool2d(2, 2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(16 * 6 * 6, 120),
            nn.Tanh(),
            nn.Linear(120, 84),
            nn.Tanh(),
            nn.Linear(84, num_classes),
        )

    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

transform = transforms.Compose([
    transforms.Resize(32),
    transforms.ToTensor(),
    transforms.Normalize((0.5,), (0.5,))
])
train_data = datasets.MNIST('./data', train=True, download=True, transform=transform)
train_loader = DataLoader(train_data, batch_size=64, shuffle=True)

model = LeNet5()
optimizer = optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.CrossEntropyLoss()

9.2 卷积操作可视化

import torch.nn.functional as F

def visualize_conv(input_tensor, kernel):
    h_in, w_in = input_tensor.shape
    kh, kw = kernel.shape
    h_out = h_in - kh + 1
    w_out = w_in - kw + 1
    output = torch.zeros(h_out, w_out)

    for i in range(h_out):
        for j in range(w_out):
            patch = input_tensor[i:i+kh, j:j+kw]
            output[i, j] = (patch * kernel).sum()
    return output

    x = image.unsqueeze(0)
    print(f"input size: {x.shape}")  # [1, 1, 32, 32]

    for i, layer in enumerate(model.features):
        x = layer(x)

    return x
