⚡ 14 · Transformer — 一切的转折点

2017 · Vaswani 等 (Google) · "Attention Is All You Need" · BERT/GPT/LLaMA 的共同祖先

一、背景与动机

2017年之前，序列建模的主力是 RNN/LSTM。问题：① 串行计算无法并行→训练慢 ② 长距离依赖仍然困难 ③ Attention 虽好但仍依赖 RNN 骨架。

Google Brain 团队的革命性想法：完全抛弃 RNN 和卷积，只用注意力机制来建模序列。论文标题 "Attention Is All You Need" 成了深度学习最著名的口号之一。

Transformer 改变了一切：机器翻译超越所有方法，然后 BERT、GPT、ViT、DALL-E、ChatGPT、LLaMA...全部基于 Transformer。它是当代 AI 的基础架构。

💡 一句话总结：Transformer = Self-Attention(全局交互) + 位置编码(位置信息) + 残差连接(梯度流) + 层归一化(稳定训练)。它让序列中的每个位置都能直接"看到"所有其他位置。

二、名字由来

💡 通俗类比："Transformer"是"变换器"的意思——它把输入序列"变换"为输出序列。但这个名字现在代表的已经不只是一个模型，而是一个"时代"——2017年以后的AI几乎都建立在 Transformer 之上。
⚠️ 常见误区："Attention Is All You Need"这个标题很"嚣张"，但 Transformer 确实用注意力替代了 RNN 和 CNN。不过"all you need"在2024年已经不完全正确——状态空间模型(SSM/Mamba)正在挑战这个论断。
🔬 深入探讨：Transformer 论文的8位作者(Ashish Vaswani等)全部来自 Google。但讽刺的是，Google 在商业化上被 OpenAI(GPT系列)超越。"发明技术"和"利用技术"是两种不同的能力——技术民主化比技术发明更难。

Transformer：Transform(变换) + -er(执行者)。它"变换"了输入序列的表示——通过注意力机制重新编码每个位置的上下文信息。也暗示了它"变革性"(transformative)的影响。

三、核心原理

💡 通俗类比：Transformer 的编码器像"全面分析文章的含义"(双向理解)，解码器像"逐字写文章但回头看前文"(自回归生成)。GPT 只用解码器(只写不读)，BERT 只用编码器(只读不写)，T5 两者都用(又读又写)。
⚠️ 常见误区：Transformer 的"并行"是指训练时的并行——所有 token 同时计算。但推理时仍然是"自回归"的——一次生成一个 token，无法并行。训练并行 ≠ 推理并行。

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V

3.3 多头注意力 (Multi-Head Attention)

3.4 位置编码 (Positional Encoding)

Self-Attention 是排列不变的——打乱词序结果不变。需要注入位置信息。

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

每个位置有一个唯一的 d_model 维向量。正弦/余弦交替，不同频率编码不同尺度的位置信息。

🔬 为什么用正弦位置编码？

① 每个位置唯一；② 相对位置可以通过线性变换获得(PE(pos+k) 可用 PE(pos) 的线性组合表示)；③ 可以外推到训练时没见过的长度。

后续改进：RoPE(旋转位置编码，LLaMA 用)、ALiBi(Attention with Linear Biases)。

3.5 前馈网络 (FFN)

FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
d_model=512 → d_ff=2048 → d_model=512

两层 MLP，中间用 ReLU/GELU 激活。对每个位置独立计算(位置间不交互)。

四、技术细节

💡 通俗类比：位置编码就像"给每个座位编号"——没有编号，你只知道"有人"，不知道"谁坐在哪里"。正弦位置编码用不同频率的正弦函数给每个位置一个唯一"编号"——频率低的编码"粗位置"，频率高的编码"细位置"。
⚠️ 常见误区：正弦位置编码不是唯一选择——可学习位置编码(BERT/GPT用)、旋转位置编码(RoPE, LLaMA用)、相对位置编码(T5用)各有利弊。没有"最好的"位置编码——选择取决于具体场景。
🔬 深入探讨：Feed-Forward Network(FFN)在 Transformer 中占据了约2/3的参数量，但长期被忽视。最近的研究发现 FFN 层存储了模型的"知识"——通过分析 FFN 的权重可以提取事实性知识。"注意力是推理，FFN 是记忆"是一个有用的直觉。

五、使用场景

💡 通俗类比：Transformer 的通用性就像"万能插座"——文本(GPT/BERT)、图像(ViT)、音频(Whisper)、代码(Copilot)、蛋白质(AlphaFold2)……几乎所有AI突破都建立在 Transformer 上。
⚠️ 常见误区：Transformer 不是所有场景的最佳选择——在短序列、小数据、实时性要求高的场景中，CNN 或 RNN 可能更合适。"最强"不等于"最合适"。
🔬 深入探讨：Transformer 的计算瓶颈在注意力层——O(n²)的复杂度在长序列上不可接受。这催生了一个庞大的"高效注意力"研究领域：FlashAttention(硬件优化)、线性注意力(算法优化)、稀疏注意力(结构优化)。效率优化是 Transformer 实用化的关键。
  • NLP 全部任务：翻译、摘要、问答、对话、文本生成
  • 计算机视觉：ViT、Swin Transformer、DETR
  • 语音：Whisper(OpenAI语音识别)
  • 多模态：GPT-4V、LLaVA(图文理解)
  • 科学计算：AlphaFold(蛋白质结构预测)
  • 代码生成：GitHub Copilot、CodeLlama

六、相似技术对比

💡 通俗类比：Encoder-Only(BERT)像"阅读理解"——读懂全文后回答问题；Decoder-Only(GPT)像"写作文"——一个字一个字往下写；Encoder-Decoder(T5)像"翻译"——先读懂再重写。
⚠️ 常见误区：GPT 是"Decoder-Only"不意味着它"不如"Encoder-Decoder。在足够大的规模下，Decoder-Only 模型展现出了强大的"涌现能力"(emergent abilities)——小模型不具备的能力在大模型中突然出现。
🔬 深入探讨：为什么 Decoder-Only 架构在LLM中"赢"了？一个假说是：因果注意力掩码(Causal Attention Mask)让每个位置只能看到之前的 token——这天然适合"预测下一个token"的训练目标(CLMLM)。简单而有效，符合"少即是多"的设计哲学。

七、关联技术

💡 通俗类比：从 Transformer 到 GPT/BERT 的关系就像"发明了发动机"→"造出了汽车"。Transformer 是"发动机"(通用架构)，GPT/BERT 是"汽车"(具体应用)。没有发动机造不了汽车，但发动机本身不是汽车。
⚠️ 常见误区：Transformer 论文本身并没有使用"预训练"——它只是一个翻译模型。"大规模预训练"是后来 GPT/BERT 引入的。Transformer 的贡献是架构，预训练的贡献是训练范式——两者结合才有了今天的LLM。
🔬 深入探讨：Transformer 的"注意力 is all you need"论断在2024年正被挑战。Mamba(2023)用选择性状态空间模型替代注意力，在长序列上更高效。但纯 SSM 模型在"回忆精确信息"上不如 Transformer——最终可能是"混合架构"获胜。
  • BERT(模型15)：Encoder-only Transformer，双向预训练
  • GPT(模型16)：Decoder-only Transformer，自回归生成
  • ViT(模型19)：Transformer 用于视觉
  • Flash Attention：优化注意力的 GPU IO 效率
  • MoE(模型23)：用稀疏专家替代 FFN

三、核心原理

💡 通俗类比：Transformer 的编码器像"全面分析文章的含义"(双向理解)，解码器像"逐字写文章但回头看前文"(自回归生成)。GPT 只用解码器(只写不读)，BERT 只用编码器(只读不写)，T5 两者都用(又读又写)。

3.4 位置编码 (Positional Encoding)

Self-Attention 是排列不变的——打乱词序结果不变。需要注入位置信息。

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

每个位置有一个唯一的 d_model 维向量。正弦/余弦交替，不同频率编码不同尺度的位置信息。

🔬 为什么用正弦位置编码？

① 每个位置唯一；② 相对位置可以通过线性变换获得(PE(pos+k) 可用 PE(pos) 的线性组合表示)；③ 可以外推到训练时没见过的长度。

后续改进：RoPE(旋转位置编码，LLaMA 用)、ALiBi(Attention with Linear Biases)。

四、技术细节

💡 通俗类比：位置编码就像"给每个座位编号"——没有编号，你只知道"有人"，不知道"谁坐在哪里"。正弦位置编码用不同频率的正弦函数给每个位置一个唯一"编号"——频率低的编码"粗位置"，频率高的编码"细位置"。

九、Transformer 的数学基础

💡 数学直觉：自注意力可以理解为一种"软字典查找"——Query 是查询，Key 是字典的键，Value 是字典的值。每个 Query 与所有 Key 计算相似度，然后用相似度作为权重对 Value 做加权求和。

9.1 自注意力的计算图

自注意力的完整计算流程：输入 X ∈ ℝ^{n×d}，通过三个线性变换得到 Q=XW_Q，K=XW_K，V=XW_V。然后计算注意力分数 A = softmax(QK^T/√d_k)，最后输出 O = AV。整个过程可以用一个公式表示：Attention(X) = softmax(XW_QW_K^TX^T/√d_k) · XW_V。

从信息论角度，自注意力实现了"信息的动态路由"——每个位置根据内容相关性(而非位置距离)从其他位置获取信息。这解决了 RNN 中信息必须"经过所有中间步骤"的瓶颈。

9.2 位置编码的数学性质

正弦位置编码 PE(pos, 2i) = sin(pos/10000^{2i/d}) 的设计有几个精妙之处：首先，任意两个位置的编码的点积只与相对距离有关；其次，编码的每个维度是不同频率的正弦波，从低频到高频；最后，编码可以通过线性变换从位置 pos 的编码得到位置 pos+k 的编码。

可学习的位置编码(Learned Positional Embedding)直接将位置索引映射为向量，参数在训练中学习。实验表明，可学习编码和正弦编码在翻译任务上效果相当，但正弦编码的泛化能力更好(可以处理训练时未见过的长度)。

🔬 深入探讨：RoPE (Rotary Position Embedding) 是一种更先进的位置编码方式，被 LLaMA、PaLM 等现代大模型采用。RoPE 将位置信息编码为旋转矩阵，使得两个 token 的注意力分数自然地依赖于它们的相对位置。数学上，RoPE 将 Q 和 K 的每对相邻维度视为二维向量，按位置角度旋转。这种设计天然支持相对位置编码，且可以通过线性插值外推到更长的序列。

9.3 Feed-Forward Network 的作用

FFN 通常被忽视，但它在 Transformer 中扮演着"记忆"角色。研究表明，FFN 的第一层权重矩阵存储了"键"(key)，第二层存储了"值"(value)——FFN 本质上是一个"记忆层"，存储了预训练数据中的事实知识。

FFN 的维度通常是隐藏维度的 4 倍(d_ff = 4 × d_model)，这意味着 FFN 的参数量占 Transformer 总参数量的约 2/3。GeLU 激活函数比 ReLU 更平滑，在实践中效果更好。SwiGLU 被 LLaMA 等模型采用，进一步提升了性能。

十、Transformer 的工程优化

10.1 KV Cache 与推理优化

自回归生成时，每生成一个新 token 都需要计算所有之前 token 的 Key 和 Value。KV Cache 将已计算的 K 和 V 缓存起来，避免重复计算。对于 n 个 token 的序列，KV Cache 的内存占用为 O(n × L × h × d)。

KV Cache 工作原理: 无 Cache (每步重新计算): Step 1: Q₁K₁V₁ → O₁ 计算量: 1 Step 2: Q₂(K₁K₂)(V₁V₂) → O₂ 计算量: 2 Step 3: Q₃(K₁K₂K₃)(V₁V₂V₃) → O₃ 计算量: 3 总计: 1+2+3+...+n = O(n²) 有 Cache (只计算新 token): Step 1: Q₁K₁V₁ → O₁, Cache=[K₁,V₁] 计算量: 1 Step 2: Q₂K₂V₂ → O₂, Cache=[K₁K₂,V₁V₂] 计算量: 1 Step 3: Q₃K₃V₃ → O₃, Cache=[K₁K₂K₃,...] 计算量: 1 总计: n × O(1) = O(n) — 显著加速！
⚠️ 显存瓶颈：KV Cache 是大模型推理的主要显存瓶颈。对于 70B 模型，batch_size=32、seq_len=4096 时，KV Cache 可能需要 100+ GB 显存。解决方案包括：GQA/MQA 减少头数、量化 KV Cache、PagedAttention (vLLM) 按需分配显存。

10.2 混合精度训练

现代 Transformer 训练通常使用混合精度(Mixed Precision)：前向传播和反向传播使用 FP16/BF16 计算，参数更新使用 FP32。这可以将训练速度提升 2-3 倍，显存减少约 40%。BF16 比 FP16 的动态范围更大(指数位更多)，更适合大模型训练。

10.3 并行训练策略

大模型训练需要多种并行策略的组合：数据并行(DP)将数据分到多个 GPU；模型并行(TP)将模型的每一层分到多个 GPU；流水线并行(PP)将模型的不同层分到不同 GPU；ZeRO 优化将优化器状态、梯度、参数分片到多个 GPU。

💡 工程实践：DeepSpeed 和 Megatron-LM 是最常用的大模型训练框架。DeepSpeed 的 ZeRO-3 可以将每个 GPU 的显存占用降低到 1/N(N 是 GPU 数量)。FSDP 是 PyTorch 原生的类似实现。

# 完整 Transformer Encoder Layer
import torch, torch.nn as nn, math

class TransformerEncoderLayer(nn.Module):
    def __init__(self, d_model, nhead, d_ff, dropout=0.1):
        super().__init__()
        self.self_attn = nn.MultiheadAttention(d_model, nhead, batch_first=True)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff), nn.ReLU(), nn.Dropout(dropout), nn.Linear(d_ff, d_model))
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.drop1 = nn.Dropout(dropout)
        self.drop2 = nn.Dropout(dropout)
    def forward(self, x, mask=None):
        attn_out, _ = self.self_attn(x, x, x, attn_mask=mask)
        x = self.norm1(x + self.drop1(attn_out))
        x = self.norm2(x + self.drop2(self.ffn(x)))
        return x
# 正弦位置编码
class PositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=5000):
        super().__init__()
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len).unsqueeze(1).float()
        div_term = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model))
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        self.register_buffer('pe', pe.unsqueeze(0))
    def forward(self, x):
        return x + self.pe[:, :x.size(1)]
# KV Cache 推理加速
class CachedDecoder(nn.Module):
    def forward(self, x, kv_cache=None):
        if kv_cache is not None:
            k_new = self.W_k(x)
            v_new = self.W_v(x)
            k = torch.cat([kv_cache[0], k_new], dim=1)
            v = torch.cat([kv_cache[1], v_new], dim=1)

            k = self.W_k(x)
            v = self.W_v(x)

📝 练习与思考

🎯 思考题
  1. Transformer完全抛弃了RNN的递推结构，仅用Self-Attention实现序列建模。请从计算复杂度和梯度路径长度两个角度，分析Transformer相比RNN的优势和劣势。
  2. 位置编码有多种方案：正弦编码(原始Transformer)、可学习编码(BERT/GPT)、旋转位置编码RoPE(LLaMA)、ALiBi。请解释RoPE的核心思想——为什么旋转位置编码能同时编码绝对位置和相对位置信息？
  3. Transformer有三种架构变体：Encoder-only(BERT)、Decoder-only(GPT)、Encoder-Decoder(T5)。请解释三者在注意力mask上的区别和最适应的任务类型。
  4. Layer Normalization在Transformer中通常放在残差连接之后(Post-LN)或之前(Pre-LN)。请分析两种放置方式对梯度传播和训练稳定性的影响。为什么GPT-2使用Pre-LN？
  5. Feed-Forward Network(FFN)层占据了Transformer约2/3的参数量。请从知识存储的角度分析FFN的功能——GPT-3的FFN层中存储了什么？知识编辑(如ROME)是如何修改这些知识的？

✅ 参考答案与详解
  1. 核心原理: RNN计算复杂度O(n·d²)但序列操作(n步)无法并行，且梯度路径长度为O(n)——输入x₁的梯度需经n步才传到xₙ。Transformer自注意力复杂度O(n²·d)但路径长度O(1)——任意两位置直接相连。优势：(1)完全并行训练速度5-10倍；(2)任意位置直接路径长度1，解决长程依赖。劣势：(1)自注意力内存O(n²)限制长序列；(2)位置关系需显式编码；(3)在极长序列(>100K)上Mamba等线性模型更优。
    关键要点: (1)RNN串行O(n·d²)；(2)Transformer并行O(n²·d)；(3)RNN梯度路径O(n)；(4)Transformer路径O(1)；(5)长序列各有优劣。
    实际案例: Transformer在8个P100 GPU上训练3.5天完成WMT翻译，而LSTM需要数周。但Mamba在100万token时Transformer内存爆炸而Mamba可处理。
  2. 核心原理: RoPE(Rotary Position Embedding)的核心思想是将位置编码为旋转矩阵：对d维向量划分为d/2对，每对用角度mθᵢ旋转(m=位置, θᵢ=频率)。关键性质：⟨Rₘq, Rₙk⟩ = ⟨R_{ₘ-ₙ}q, k⟩——两个位置编码的内积只依赖于相对位置(m-n)，使注意力天然包含相对位置信息。相比正弦编码需通过交互学相对位置，RoPE通过数学结构隐式编码了相对位置，在长文本外推上表现远优于可学习PE。
    关键要点: (1)位置编码为旋转矩阵；(2)内积只依赖相对位置；(3)隐式编码无需学习；(4)长文本外推更优；(5)LLaMA/Qwen2等主流LLM均采用RoPE。
    实际案例: LLaMA-2支持32K上下文，Qwen2支持128K——RoPE的长文本外推能力是关键技术。RoPE外推配合YaRN方法可扩展到100万token。
  3. 核心原理: Encoder-only(BERT)：双向自注意力(无mask)，全部位置互相可见。适合"NLP理解"任务：文本分类、NER、问答。Decoder-only(GPT)：因果Causal Mask，每位置只看到自己和过去。适合"文本生成"任务：语言模型、故事生成。Encoder-Decoder(T5)：编码器双向+解码器因果+跨注意力(解码器Q查询编码器KV)。适合"条件生成"任务：翻译、摘要。三者mask策略决定了信息流向和适用任务。
    关键要点: (1)Enc双向适合理解；(2)Dec因果适合生成；(3)Enc-Dec条件生成；(4)mask决定信息流向；(5)范式选择依赖任务。
    实际案例: BERT在GLUE基准上80.5分，GPT-4(Decoder-only)在翻译/摘要方面也超越了T5(Encoder-Decoder)。
  4. 核心原理: Post-LN(原始Transformer)：x' = LN(Sublayer(x) + x)。训练初期 LN 输入的方差可能很大，导致梯度不稳定。Pre-LN(GPT-2)：x' = Sublayer(LN(x)) + x。LN在子层前应用，输入方差稳定，且残差分支不受LN影响，梯度直接传回。Pre-LN训练更稳定但可能限制深层网络的表达能力(因为每层的"残差修正"起点不同)。2020年Xiong等人的论文证明Pre-LN训练更稳定成为后续大模型的标准。
    关键要点: (1)Post-LN原始方案；(2)Pre-LN更稳定；(3)LN在子层前输入方差稳定；(4)残差不受LN影响；(5)GPT-2/BERT新版本采用Pre-LN。
    实际案例: GPT-2使用Pre-LN成功训练1.5B参数模型。最新研究Chameleon使用Sandwich-LN进一步优化。
  5. 核心原理: FFN层结构为W₂·GELU(W₁x)，其中W₁∈ℝ^{d×4d}, W₂∈ℝ^{4d×d}。MIT的Meng等人(ROME)发现FFN的W₂的行向量充当"keys"，W₁的列向量充当"values"——FFN累积地存储了(key,value)对知识。例如W₂中"Paris"这一行会与W₁中"capital of France"对应的列相乘输出"Paris"。知识编辑(ROME/MEMIT)通过修改W₁的特定列向量来纠正/插入知识，而无需重新训练。
    关键要点: (1)FFN存储(key,value)知识对；(2)W₂行=keys,W₁列=values；(3)记忆矩阵的key-value视角；(4)ROME/MEMIT实现知识编辑；(5)大模型知识主要在FFN中。
    实际案例: ROME可以修改GPT-3的"日本首都=东京"为"日本首都=江户"，准确验证了FFN知识存储理论。

import torch, torch.nn as nn, math

class PositionalEncoding(nn.Module):
    def __init__super().__init__()
        pe = torch.zeros(max_len, d_model)
        pos = torch.arange(max_len).unsqueeze(1).float()
        div = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000) / d_model))
        pe[:, 0::2] = torch.sin(pos * div)
        pe[:, 1::2] = torch.cos(pos * div)
        self.register_buffer('pe', pe.unsqueeze(0))
    def forward(self, x):
        return x + self.pe[:, :x.size(1)]

class TransformerBlock(nn.Module):
    def __init__super().__init__def forward(self, x, mask=None):
        x = x + self.drop(self.attn(self.norm1(x), self.norm1(x), self.norm1(x), attn_mask=mask)[0])
        x = x + self.drop(self.ffn(self.norm2(x)))
        return x

关键知识点: (1)自注意力+位置编码是核心；(2)Post-LN vs Pre-LN影响训练稳定性；(3)RoPE是现代LLM标配；(4)FFN存储知识；(5)三种架构变体各有适用场景。
