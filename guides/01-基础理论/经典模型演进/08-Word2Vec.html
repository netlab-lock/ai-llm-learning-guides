📝 08 · Word2Vec — 词嵌入革命

2013 · Tomas Mikolov 等 (Google) · "king - man + woman = queen" · NLP 向量化时代

一、背景与动机

在 Word2Vec 之前，NLP 中词语用独热编码(One-Hot)表示：每个词是一个长向量，只有对应位置是 1。问题：① 维度爆炸(10万词→10万维) ② 任意两个词的距离相同(正交) ③ 无法表达语义关系("好"和"优秀"的距离等于"好"和"桌子")。

Mikolov 等人的突破：用一个浅层神经网络，把词映射到低维稠密向量空间(如300维)，使得语义相近的词向量也相近。这就是 Word2Vec。

💡 一句话总结：Word2Vec 让计算机第一次"理解"了词义——king 和 queen 的向量方向一致(都是皇室)，car 和 automobile 几乎重合(同义词)。

二、名字由来

💡 通俗类比：Word2Vec 就是"Word to Vector"——把单词变成向量。就像把一个人的"性格"用几个数字表示(内向0.7, 幽默0.3, 勤奋0.9)——虽然简化了，但捕捉了核心特征。
⚠️ 常见误区：Word2Vec 不是一种"算法"，而是一个工具包名。它包含两种模型：CBOW(从上下文预测中心词)和 Skip-gram(从中心词预测上下文)。两者目标相反，效果类似。
🔬 深入探讨：Word2Vec 的词向量有一个神奇的性质：向量运算能捕捉语义关系。"king - man + woman ≈ queen"是最著名的例子。这不是巧合——训练目标隐式地学到了这些关系，虽然从未被显式教导。

Word(词) + 2(to 的谐音) + Vec(Vector 向量) = "词到向量"。这是程序员常见的命名风格(类似 py2exe, img2txt)。

Embedding(嵌入)：数学术语，指把一个空间映射到另一个空间。词嵌入 = 把词嵌入到向量空间。

三、核心原理

💡 通俗类比：Skip-gram 就像"填空题"——给你一个词(如"苹果")，让你预测它周围可能出现什么词("吃"、"红色"、"手机")。经常出现在相似上下文中的词，会被映射到相近的向量。
⚠️ 常见误区：Word2Vec 的"上下文窗口"不是越大越好。太大的窗口会让词向量捕获"主题"信息(如"苹果"和"水果"更近)，太小会捕获"语法"信息(如"苹果"和"吃"更近)。窗口大小决定了你学到什么样的语义。
🔬 深入探讨：负采样(Negative Sampling)是 Word2Vec 训练高效的关键——不计算整个词表的 softmax(词表可能有百万词)，而是随机采样几个"负例"。这把训练复杂度从 O(V) 降到 O(K)——V是词表大小，K是负样本数(通常5-20)。

3.1 分布式假说 (Distributional Hypothesis)

"一个词的含义由它的上下文决定"——J.R. Firth (1957): "You shall know a word by the company it keeps."

如果两个词经常出现在相似的上下文中，它们的语义就相似。

3.2 CBOW 模型

CBOW (Continuous Bag of Words): 用上下文词预测中心词 上下文窗口(窗口大小=2): "the" "cat" [___] "on" "the" ↑ 预测 "sat" 网络结构: 上下文词 → 查词嵌入表 → 平均 → 隐藏层 → Softmax → 预测中心词 (one-hot) (300维) (300维) (300×V) (V维) 目标: 让预测的词和真实词的概率分布接近 损失: 交叉熵 Loss = -log P(center|context)

3.3 Skip-gram 模型

Skip-gram: 用中心词预测上下文词 (CBOW 的逆) 中心词: "sat" 预测: "the", "cat", "on", "the" 中心词 → 查词嵌入表 → 隐藏层 → Softmax → 预测上下文词 (one-hot) (300维) (300×V) (V维) 优势: 训练数据更充足(每个词对都是一个训练样本) 对低频词效果更好

3.4 训练技巧

Softmax 的分母需要遍历整个词表(10万+)，计算量巨大。两个关键优化：

技巧原理效果
负采样(Negative Sampling)不计算完整 Softmax，只区分正样本和随机负样本训练速度 ×1000
层次 Softmax用二叉树代替平坦 Softmax复杂度 O(V) → O(log V)
负采样损失:
J = log σ(v'ₒ · vᵤ) + Σ E[log σ(-v'ₙ · vᵤ)]
正样本得分↑ + 负样本得分↓
其中 vᵤ 是中心词向量, v'ₒ 是上下文词向量

四、技术细节

💡 通俗类比：训练 Word2Vec 就像训练一个"猜词游戏"AI——给它看大量文本，让它反复练习"根据上下文猜词"。练得越多，它对词与词之间关系的理解就越深。
⚠️ 常见误区：Word2Vec 的词向量是"静态"的——"苹果"只有一个向量，无论它是"吃苹果"还是"苹果手机"中的苹果。这种"一词一义"的局限性后来被 ELMo(上下文相关)和 BERT(双向上下文)解决了。
🔬 深入探讨：Word2Vec 的训练目标(预测上下文)和后来 BERT 的训练目标(Masked LM)有异曲同工之妙——都在利用"上下文"学习词义。区别在于 Word2Vec 是"浅层"(一个线性层)，BERT 是"深层"(12/24层 Transformer)。

4.1 预训练词向量的特性

Word2Vec 向量的神奇代数: king - man + woman ≈ queen Paris - France + Italy ≈ Rome walking - walked + swam ≈ swimming 原理: 词向量空间中存在线性关系 方向1: 性别 (man→woman, king→queen) 方向2: 国家→首都 (France→Paris, Italy→Rome) 方向3: 时态 (walk→walked, go→went)

五、使用场景

💡 通俗类比：Word2Vec 之于 NLP，就像"拼音"之于中文学习——它是最基础的"表征"，让机器第一次"理解"了词与词之间的关系。没有词向量，就没有后来的一切 NLP 进展。
⚠️ 常见误区：Word2Vec 在2024年已很少直接使用(被BERT等取代)，但"词嵌入"的思想是所有现代 NLP 模型的基础。Embedding 层——把离散的token映射为连续向量——就是 Word2Vec 思想的直接继承。
🔬 深入探讨：Word2Vec 的成功启发了"万物皆可 Embedding"——用户 Embedding(推荐系统)、图节点 Embedding(Graph Neural Networks)、甚至蛋白质序列 Embedding。把离散实体映射到连续向量空间的思想，已经超越了 NLP 的范畴。
  • NLP 基础设施：几乎所有 NLP 任务的第一步——把文本变成向量
  • 推荐系统：把 Item2Vec(类似思想)用于商品/用户嵌入
  • 图嵌入：DeepWalk、Node2Vec 把 Word2Vec 思想用于图节点
  • 搜索引擎：查询和文档的语义匹配

六、相似技术对比

💡 通俗类比：Word2Vec vs GloVe vs FastText 就像三种不同的"学外语"方式——Word2Vec 通过"读大量文章"学(预测上下文)，GloVe 通过"查词频统计"学(共现矩阵分解)，FastText 通过"拆字根"学(子词信息)。
⚠️ 常见误区：不要以为"越新的越好"。在某些任务中(如词类比)，Word2Vec 的效果并不比 BERT 差。"够用"就好——用大炮打蚊子是浪费资源。
🔬 深入探讨：GloVe 和 Word2Vec 在数学上有等价性——Levy & Goldberg(2014)证明了 Skip-gram 隐式地在分解一个PMI矩阵。这意味着两种看似不同的方法其实在做同一件事——科学中"殊途同归"的现象比比皆是。

七、关联技术

💡 通俗类比：Word2Vec → ELMo → BERT 的演进就像"照片"→"视频"→"3D全息"——从静态词向量，到上下文相关的动态词向量，再到双向深层上下文表示。每一步都是对"理解"的加深。
⚠️ 常见误区：Word2Vec 的"king - man + woman ≈ queen"虽然很酷，但也暴露了词向量的偏见——训练数据中的性别、种族偏见会被编码进向量。这不是 Word2Vec 的bug，而是数据的"镜子"。
🔬 深入探讨：Word2Vec 的另一个意外发现是：词向量可以跨语言对齐。通过少量双语词典，可以把中文词向量和英文词向量映射到同一空间——这启发了多语言BERT(mBERT)和XLM-R的跨语言能力。
  • Transformer(模型14)：Transformer 的 Token Embedding 层就是词嵌入
  • BERT(模型15)：上下文相关的动态词嵌入，解决了 Word2Vec 一词多义问题
  • 位置编码：Transformer 需要额外的位置编码，因为嵌入层本身不包含位置信息
  • Skip-thought / Doc2Vec：将 Word2Vec 扩展到句子/文档级别

八、实际效果

💡 通俗类比：Word2Vec 的300维向量就像"300个特征维度"描述一个词——每个维度可能对应某种隐含的语义特征(虽然我们不知道每个维度具体代表什么)。
⚠️ 常见误区：Word2Vec 在词类比任务上表现很好(如 king-man+woman=queen)，但这不等于它"理解"了语义。它只是学到了统计共现模式——"理解"和"统计"之间的界限是 AI 领域最深刻的哲学问题之一。
🔬 深入探讨：Word2Vec 训练的词向量维度(通常100-300)是经验选择，没有理论最优解。太小捕获不了足够的语义信息，太大则包含噪声且计算昂贵。这个"维度选择"问题在所有 Embedding 场景中都存在。

一、背景与动机的技术原理与数学推导： 从分布式表示原理看，Word2Vec通过浅层神经网络将词映射到低维稠密向量空间。 推导：Skip-gram的目标是最大化上下文词的条件概率max Σlog P(context|word)。 优化后，语义相似的词在向量空间中距离更近，因为它们共享相似的上下文分布。 这一数学本质上，Word2Vec隐式分解了词-上下文共现矩阵的PMI（逐点互信息）， 即w·c ≈ PMI(w,c) - log k，与GloVe的显式矩阵分解殊途同归。

  • Transformer(模型14)：Transformer 的 Token Embedding 层就是词嵌入
  • BERT(模型15)：上下文相关的动态词嵌入，解决了 Word2Vec 一词多义问题
  • 位置编码：Transformer 需要额外的位置编码，因为嵌入层本身不包含位置信息
  • Skip-thought / Doc2Vec：将 Word2Vec 扩展到句子/文档级别

三、核心原理

💡 通俗类比：Skip-gram 就像"填空题"——给你一个词(如"苹果")，让你预测它周围可能出现什么词("吃"、"红色"、"手机")。经常出现在相似上下文中的词，会被映射到相近的向量。
⚠️ 常见误区：Word2Vec 的"上下文窗口"不是越大越好。太大的窗口会让词向量捕获"主题"信息(如"苹果"和"水果"更近)，太小会捕获"语法"信息(如"苹果"和"吃"更近)。窗口大小决定了你学到什么样的语义。
🔬 深入探讨：负采样(Negative Sampling)是 Word2Vec 训练高效的关键——不计算整个词表的 softmax(词表可能有百万词)，而是随机采样几个"负例"。这把训练复杂度从 O(V) 降到 O(K)——V是词表大小，K是负样本数(通常5-20)。

3.1 分布式假说 (Distributional Hypothesis)

"一个词的含义由它的上下文决定"——J.R. Firth (1957): "You shall know a word by the company it keeps."

如果两个词经常出现在相似的上下文中，它们的语义就相似。

3.2 CBOW 模型

CBOW (Continuous Bag of Words): 用上下文词预测中心词 上下文窗口(窗口大小=2): "the" "cat" [___] "on" "the" ↑ 预测 "sat" 网络结构: 上下文词 → 查词嵌入表 → 平均 → 隐藏层 → Softmax → 预测中心词 (one-hot) (300维) (300维) (300×V) (V维) 目标: 让预测的词和真实词的概率分布接近 损失: 交叉熵 Loss = -log P(center|context)

3.3 Skip-gram 模型

Skip-gram: 用中心词预测上下文词 (CBOW 的逆) 中心词: "sat" 预测: "the", "cat", "on", "the" 中心词 → 查词嵌入表 → 隐藏层 → Softmax → 预测上下文词 (one-hot) (300维) (300×V) (V维) 优势: 训练数据更充足(每个词对都是一个训练样本) 对低频词效果更好

3.4 训练技巧

Softmax 的分母需要遍历整个词表(10万+)，计算量巨大。两个关键优化：

技巧原理效果
负采样(Negative Sampling)不计算完整 Softmax，只区分正样本和随机负样本训练速度 ×1000
层次 Softmax用二叉树代替平坦 Softmax复杂度 O(V) → O(log V)
负采样损失:
J = log σ(v'ₒ · vᵤ) + Σ E[log σ(-v'ₙ · vᵤ)]
正样本得分↑ + 负样本得分↓
其中 vᵤ 是中心词向量, v'ₒ 是上下文词向量

四、技术细节

💡 通俗类比：训练 Word2Vec 就像训练一个"猜词游戏"AI——给它看大量文本，让它反复练习"根据上下文猜词"。练得越多，它对词与词之间关系的理解就越深。
⚠️ 常见误区：Word2Vec 的词向量是"静态"的——"苹果"只有一个向量，无论它是"吃苹果"还是"苹果手机"中的苹果。这种"一词一义"的局限性后来被 ELMo(上下文相关)和 BERT(双向上下文)解决了。
🔬 深入探讨：Word2Vec 的训练目标(预测上下文)和后来 BERT 的训练目标(Masked LM)有异曲同工之妙——都在利用"上下文"学习词义。区别在于 Word2Vec 是"浅层"(一个线性层)，BERT 是"深层"(12/24层 Transformer)。

4.1 预训练词向量的特性

Word2Vec 向量的神奇代数: king - man + woman ≈ queen Paris - France + Italy ≈ Rome walking - walked + swam ≈ swimming 原理: 词向量空间中存在线性关系 方向1: 性别 (man→woman, king→queen) 方向2: 国家→首都 (France→Paris, Italy→Rome) 方向3: 时态 (walk→walked, go→went)

9.1 Skip-gram 模型实现

import torch
import torch.nn as nn

class SkipGram(nn.Module):
    def __init__(self, vocab_size, embed_dim):
        super().__init__()
        self.center_embed = nn.Embedding(vocab_size, embed_dim)
        self.context_embed = nn.Embedding(vocab_size, embed_dim)

    def forward(self, center, context, neg_samples):
        center_emb = self.center_embed(center)
        context_emb = self.context_embed(context)
        neg_emb = self.context_embed(neg_samples)

        pos_score = torch.sum(center_emb * context_emb, dim=1)
        pos_loss = -torch.log(torch.sigmoid(pos_score) + 1e-10)

        neg_score = torch.bmm(neg_emb, center_emb.unsqueeze(2)).squeeze()
        neg_loss = -torch.log(torch.sigmoid(-neg_score) + 1e-10).sum(dim=1)

        return (pos_loss + neg_loss).mean()

9.2 数据预处理与训练

from collections import Counter
import numpy as np

def build_vocab(corpus, min_count=5):
    word_counts = Counter()
    for sentence in corpus:
        word_counts.update(sentence)
    vocab = {w: i for i, (w, c) in enumerate(
        word_counts.items() if c >= min_count
    )}
    return vocab

def generate_skipgram_pairs(corpus, vocab, window_size=5):
    pairs = []
    for sentence in corpus:
        indices = [vocab[w] for w in sentence if w in vocab]
        for i, center in enumerate(indices):
            for j in range(max(0, i - window_size),
                         min(len(indices), i + window_size + 1)):
                if i != j:
                    pairs.append((center, indices[j]))
    return pairs

# 负采样：按词频^0.75 采样
word_freqs = np.array([count ** 0.75 for count in word_counts.values()])
word_freqs /= word_freqs.sum()
neg_sampler = torch.tensor(word_freqs)

9.3 词向量类比实验

def word_analogy(model, word2idx, idx2word, a, b, c, top_k=5):
    # 例如: king - man + woman = queen
    vec_a = model.center_embed.weight[word2idx[a]]
    vec_b = model.center_embed.weight[word2idx[b]]
    vec_c = model.center_embed.weight[word2idx[c]]
    target = vec_a - vec_b + vec_c

    all_vecs = model.center_embed.weight.detach()
    target_norm = target / target.norm()
    all_norm = all_vecs / all_vecs.norm(dim=1, keepdim=True)
    similarities = (all_norm @ target_norm).detach().numpy()

    exclude = {word2idx[a], word2idx[b], word2idx[c]}
    for idx in exclude:
        similarities[idx] = -1

    top_indices = similarities.argsort()[-top_k:][::-1]
    for idx in top_indices:
        print(f"  {idx2word[idx]}: {similarities[idx]:.4f}")
