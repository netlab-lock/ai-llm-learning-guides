🌊 20 · Diffusion Models — 去噪生成的革命

2020 · Ho 等 (Google Brain) · 加噪→去噪 · DALL-E/Stable Diffusion/Sora 的基础

一、背景与动机

GAN 生成质量好但训练不稳定、模式崩塌。VAE 训练稳定但生成模糊。2020年，DDPM(Denoising Diffusion Probabilistic Models) 用一个极其简单的思想超越了两者：先给图像加噪直到变成纯噪声，再学习如何从噪声中恢复图像。

Diffusion 模型在图像生成质量上全面超越 GAN，成为 DALL-E 2、Stable Diffusion、Midjourney、Sora(视频生成) 的技术基础。

💡 核心比喻：想象一张照片逐渐被雪覆盖(加噪)，直到完全看不到原图。Diffusion 模型学习"扫雪"——从纯白噪声中一步步恢复出清晰图像。

二、名字由来

💡 通俗类比："Diffusion"是"扩散"的意思——就像墨水在水中扩散(从有序到无序)。Diffusion Model 的过程是：先把图像"扩散"成纯噪声(前向过程)，然后学会"反扩散"从噪声恢复图像(逆向过程)。
⚠️ 常见误区：Diffusion Model 不是"一种"模型，而是一类模型的统称——DDPM、Score-based、SDE-based等都属于 Diffusion Model。不同变体的数学表述不同，但核心思想相同：学习去噪。
🔬 深入探讨：Diffusion Model 的数学基础是"随机微分方程"(SDE)——前向过程是"加噪SDE"，逆向过程是"去噪SDE"。Song et al.(2020)的 Score SDE 统一了 DDPM 和 Score-based 模型——看似不同的方法其实是同一个数学框架的不同视角。

Diffusion(扩散)：物理学术语——分子从高浓度区域向低浓度区域扩散，最终均匀分布。类比：图像信息"扩散"到噪声中，逐步消失。

Denoising(去噪)：反过来——从噪声中恢复信号。

三、核心原理

💡 通俗类比：Diffusion Model 的训练就像"学修图"——给你一张"损坏"的照片(加了噪声)，让你修复它。练了上百万次后，你就能从任何"损坏程度"的照片中恢复出清晰图像。最终，你甚至能从纯噪声"修复"出一张全新的照片。
⚠️ 常见误区：Diffusion 的生成过程很慢——需要几十到几千步去噪。这是它最大的缺点。DDIM、DPM-Solver 等加速方法可以把步数从1000减少到20-50步，但仍然比 GAN 慢(一步生成)。
🔬 深入探讨：为什么 Diffusion Model 在图像质量上超越了 GAN？一个解释是：GAN 的训练是"不稳定的对抗"，容易模式崩溃；Diffusion 的训练是"稳定的回归"(预测噪声)，每步都有明确的监督信号。"简单而稳定的训练"胜过"复杂而不稳定的训练"。

四、技术细节

💡 通俗类比：Classifier-Free Guidance(无分类器引导)就像"调节旋钮"——生成时可以调节"多大程度上遵循文本描述"。guidance scale 越高，生成越"忠实"于文本但也越不多样；越低则越多样但可能偏离文本。
⚠️ 常见误区：Diffusion Model 的"噪声调度"(noise schedule)不是随意选的——它决定了"每一步加多少噪声"。线性调度、余弦调度、调度策略的选择对生成质量影响很大。"调度"是最容易被忽视但最重要的设计选择之一。
🔬 深入探讨：Latent Diffusion Model(Stable Diffusion 的基础)不在像素空间做扩散，而在"潜在空间"做——先用 VAE 把图像压缩到低维空间，再在低维空间做扩散。这把计算量降低了几个数量级——让高分辨率图像生成变得实用。

五、使用场景

💡 通俗类比：Stable Diffusion、DALL-E 2、Midjourney 都是 Diffusion Model——"文字描述→图像生成"是它们最知名的应用。但 Diffusion Model 不仅能生成图像，还能做图像编辑、超分辨率、3D生成、视频生成等。
⚠️ 常见误区：Diffusion Model 生成的图像可能包含偏见(训练数据的反映)、版权问题(可能"记忆"训练图像)、深度伪造风险。技术是中性的，但使用需要谨慎——这是 AI 伦理的重要议题。
🔬 深入探讨：Diffusion Model 正在挑战 GAN 在视频生成、3D生成、音频生成等领域的地位。Sora(OpenAI, 2024)用 Diffusion Transformer(DiT)生成高质量视频——这可能是"世界模拟器"(World Simulator)的早期版本。
  • 文生图：DALL-E 2/3、Stable Diffusion、Midjourney
  • 图生图：风格迁移、图像编辑、修复(inpainting)
  • 文生视频：Sora(OpenAI)、Runway Gen-2
  • 音频生成：AudioLDM
  • 分子设计：药物分子生成

六、相似技术对比

💡 通俗类比：Diffusion vs GAN 就像"慢工出细活"vs"快速出活"——Diffusion 慢但质量高、多样性好；GAN 快但训练不稳定、容易模式崩溃。在"质量优先"的场景中，Diffusion 胜出。
⚠️ 常见误区：VAE 生成的图像模糊(因为它优化的是"平均")，GAN 生成的图像清晰但多样性差(因为对抗训练)，Diffusion 两者兼得(质量高且多样)。但 Diffusion 的推理速度最慢——速度、质量、多样性三者不能兼得。
🔬 深入探讨：Diffusion Model 的"去噪"视角和物理学中的"退火"(Annealing)过程有深刻的类比——从高温(噪声)到低温(清晰图像)的过程就像金属退火中的"缓慢冷却"。这种物理类比不仅是比喻，还有严格的数学对应。

七、关联技术

💡 通俗类比：Stable Diffusion = Latent Diffusion + CLIP + U-Net。CLIP 理解文本描述，VAE 压缩图像到潜在空间，U-Net 在潜在空间中做去噪。三个组件各司其职——"模块化设计"让创新可以独立进行。
⚠️ 常见误区：Diffusion Model 的"文本条件"不完美——它经常误解空间关系("左边的猫"vs"右边的狗")、数量("3只猫"可能生成2只或4只)。这是当前所有文生图模型的通病——"语言理解"和"视觉生成"的对齐还不够好。
🔬 深入探讨：Diffusion Transformer(DiT)用 Transformer 替代了 U-Net 作为去噪网络——这把 Diffusion Model 和 LLM 统一到了同一个架构下。Sora(OpenAI, 2024)就是基于 DiT——"用 Transformer 生成视频"可能成为下一个"用 Transformer 处理一切"的范式。
  • GAN(模型09)：Diffusion 的前身和竞争对手
  • ViT(模型19)：DiT 用 Transformer 替代 U-Net 做去噪网络
  • CLIP：文本条件引导的核心组件
  • VAE：Latent Diffusion 用 VAE 做潜空间压缩

⚠️ 常见误区：Diffusion 不是"去噪 1000 次"那么简单

初学者常误解 Diffusion 的几个关键点：

  • 不是逐步去除"真实噪声"：反向过程每步预测的是"这一步的噪声分量"，而不是"从纯噪声中一步恢复原图"
  • 步数可以减少：DDIM、DPM-Solver 等方法证明 20-50 步就能得到接近 1000 步的质量。Consistency Model 甚至只需 1-2 步
  • 不是所有 Diffusion 都在像素空间：Stable Diffusion 的核心创新是在 VAE 的潜空间做扩散（64×64×4 而非 512×512×3），速度快 48 倍
  • CFG scale 不是越大越好：Classifier-Free Guidance 的 scale 过大（如 >20）会导致图像过饱和、细节失真
🔬 深入探讨：Diffusion 的数学本质 — 为什么是高斯噪声？

DDPM 选择高斯噪声不是随意的，有深刻的数学原因：

数学性质含义为什么重要
高斯分布的闭合性高斯 + 高斯 = 高斯前向过程可以一步到位（不需要逐步加噪）
中心极限定理多次加噪后趋近高斯无论原图分布如何，最终都是标准高斯
KL 散度可解析高斯间的 KL 有闭合解训练损失可以简化为 MSE
去噪 = 评分匹配预测噪声 ≈ 估计数据分布的梯度连接了 Diffusion 和 Score-based 模型

如果用其他噪声分布（如泊松噪声），前向过程的闭合性被破坏，数学推导变得极其复杂。高斯噪声是"唯一让整个理论优雅运转"的选择。

九、Diffusion 模型的数学原理深入

9.1 前向过程的马尔可夫链

DDPM 的前向过程是一个马尔可夫链——每一步只依赖前一步的状态。数学上，前向过程定义为一系列高斯分布的转移：

q(xₜ | xₜ₋₁) = N(xₜ; √(1-βₜ)·xₜ₋₁, βₜ·I)

其中 βₜ 是噪声调度（noise schedule），控制每步添加多少噪声。通过递推，可以直接从 x₀ 采样任意时间步的 xₜ：

xₜ = √ᾱₜ · x₀ + √(1-ᾱₜ) · ε, 其中 ᾱₜ = ∏ᵢ₌₁ᵗ (1-βᵢ)

这个"重参数化技巧"是 DDPM 的关键——它允许在训练时直接跳到任意时间步，而不需要逐步加噪。当 T 足够大（如1000）且 βₜ 的累积足够大时，xₜ 近似标准高斯分布 N(0, I)。

9.2 噪声调度的设计选择

噪声调度 βₜ 的选择对生成质量有重要影响。不同的调度策略会导致不同的训练动态和最终性能：

调度类型公式特点适用场景
线性调度βₜ 从 10⁻⁴ 线性增到 0.02简单，DDPM 原始使用通用，但中间步噪声不够平滑
余弦调度ᾱₜ = cos²(πt/2T)更平滑，中间步信号保留更多高质量图像生成
指数调度βₜ = exp(s·t/T)两端变化快，中间慢特定任务优化
学习调度βₜ 作为可学习参数自适应最优调度研究前沿

9.3 Score Matching 的视角

Diffusion Model 与 Score-based Model 有深刻的数学联系。Score 函数定义为对数概率密度的梯度：∇ₓ log p(x)。Song et al. (2020) 证明了预测噪声 εₜ 等价于估计 Score 函数（相差一个缩放因子）。这意味着 DDPM 和 Score-based Model 其实在做同一件事，只是数学表述不同。

从 SDE（随机微分方程）的角度，前向过程可以写成连续时间的 SDE：dx = f(x,t)dt + g(t)dw，其中 f 是漂移项，g 是扩散项，w 是维纳过程。逆向过程对应的 SDE 只需将漂移项翻转并添加 Score 函数项。

💡 数学直觉：Score 函数 ∇ₓ log p(x) 指向数据密度增长最快的方向。在去噪过程中，预测噪声 εₜ 告诉我们"哪个方向是噪声"，取反后就得到"哪个方向是数据"——这就是为什么"去噪 = 生成"的数学本质。

10.1 Diffusion 模型的条件控制技术

在实际的文生图应用中，用户不仅需要生成高质量图像，还需要精确控制生成内容。以下是主要的条件控制技术：

  • 文本条件（Text Conditioning）：通过 CLIP 或 T5 等文本编码器将文本描述编码为向量，然后通过交叉注意力注入到 U-Net 的每一层。Stable Diffusion 使用 CLIP 文本编码器，而 Imagen 使用 T5-XXL。不同的文本编码器对不同类型描述的理解能力不同。
  • 图像条件（Image Conditioning）：ControlNet 通过在 U-Net 旁添加一个"旁路网络"来注入空间条件（如边缘图、深度图、姿态图）。旁路网络的结构与 U-Net 的编码器相同，输出通过零卷积与主 U-Net 合并。
  • 风格条件：IP-Adapter 通过解耦的交叉注意力机制注入参考图像的风格，无需重新训练基础模型。
  • 区域条件：Attend-and-Excite、Prompt-to-Prompt 等技术可以控制图像中不同区域的内容。

import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
    elapsed = (time.perf_counter() - start) / iters
    print(f"Latency: {elapsed*1000:.2f}ms")
    return elapsed

🎯 思考题
  1. Diffusion前向过程q(xₜ|xₜ₋₁)=N(√(1-βₜ)xₜ₋₁, βₜI)。请推导q(xₜ|x₀)=N(√ᾱₜx₀,(1-ᾱₜ)I)。
  2. 为什么从预测均值μ变成预测噪声ε效果更好？两者有什么关系？
  3. Classifier-Free Guidance公式：ε̂=ε(xₜ,∅)+s·(ε(xₜ,c)-ε(xₜ,∅))。为什么能增强条件生成？s>1的trade-off？
  4. 为什么原始DDPM需要数百采样步？DDIM如何加速？
  5. Latent Diffusion为什么在latent space操作？计算量降低多少倍？

✅ 参考答案与详解
  1. 核心原理: 递推展开xₜ=√(1-βₜ)xₜ₋₁+√βₜε，反复代入得xₜ=√ᾱₜx₀+√(1-ᾱₜ)ε(利用独立高斯可加性)。所以q(xₜ|x₀)=N(√ᾱₜx₀,(1-ᾱₜ)I)。关键是可以采样任意t，加速训练。
  2. 核心原理: μ_θ与ε_θ的关系：μ_θ=(1/√αₜ)(xₜ-(βₜ/√(1-ᾱₜ))ε_θ)。预测ε优势：(1)ε~N(0,I)目标稳定；(2)损失权重天然合理；(3)避免不同t量纲差异。Ho et al 2020发现ε目标显著优于μ目标。
  3. 核心原理: CFG以无条件为基线+差项放大条件信号。s=1=普通预测；s>1增强贴合度但可能过饱和/多样性降低。s=7.5是Stable Diffusion典型值。"guidance scale"是重要的超参数。
  4. 核心原理: DDPM马尔可夫每步变化小需多步。DDIM用非马尔可夫前向对应确定性ODE——轨迹直可大跨跳10-50步即够。DPM-Solver高阶ODE解法进一步加速。前提：ODE轨迹平滑。
  5. 核心原理: 像素空间256×256×3=196K维，latent 32×32×4=4K维，降48倍。VAE编解码+cross-attention注入条件(text→CLIP→K,V)。开源Stable Diffusion生态极大推动应用。

import torch, torch.nn as nn

def q_sample(x0, t, alpha_cumprod):
    eps = torch.randn_like(x0)
    alpha_t = alpha_cumprod[t][:, None, None, None]
    return torch.sqrt(alpha_t) * x0 + torch.sqrt(1 - alpha_t) * eps, eps

def p_sample(model, x, t, alpha, alpha_cumprod):
    eps_pred = model(x, t)
    beta_t = 1 - alpha[t]
    coeff = beta_t / torch.sqrt(1 - alpha_cumprod[t])
    mean = (x - coeff[:,None,None,None]*eps_pred) / torch.sqrt(alpha[t])[:,None,None,None]
    return mean + torch.sqrt(beta_t)[:,None,None,None] * torch.randn_like(x)
