💬 16 · GPT & GPT-2 — 自回归生成的开端

2018/2019 · Alec Radford 等 (OpenAI) · Decoder-only Transformer · ChatGPT 的直系祖先

一、背景与动机

BERT 用 Encoder 做双向理解，但 OpenAI 选择了另一条路：Decoder-only + 自回归(Autoregressive)——用 Transformer 的解码器，从左到右逐词生成文本。GPT-1 证明了预训练+微调的有效性，GPT-2 证明了足够大的语言模型不需要微调，直接做 Zero-shot 就能完成任务。

💡 路线之争：BERT(Encoder, 双向, 理解) vs GPT(Decoder, 单向, 生成)。2019-2020年 BERT 风头更盛，但 GPT 的自回归路线后来证明是通往 AGI 的更优路径——因为生成能力自然包含了理解能力。

二、名字由来

💡 通俗类比：GPT(Generative Pre-trained Transformer)——"生成式预训练 Transformer"。每个词都很重要："生成式"(能写文章)、"预训练"(先通识学习)、"Transformer"(用注意力架构)。
⚠️ 常见误区：GPT-1(2018)只有1.17亿参数——和今天的 ChatGPT 完全不是一个量级。GPT 的"进化"是从"小模型+微调"到"大模型+提示"(GPT-3)的过程。不要把不同版本的 GPT 混为一谈。
🔬 深入探讨：OpenAI 选择"Decoder-Only"架构(不选 Encoder-Decoder)是一个关键的战略决策——它牺牲了双向理解能力，但获得了更强的生成能力和更好的扩展性。历史证明这个选择是正确的——生成能力比理解能力更"值钱"。

GPT：Generative(生成式) + Pre-trained(预训练) + Transformer。

GPT-2：第二代。OpenAI 最初因为"太危险"而不愿公开发布(后来还是发布了)。

三、核心原理

💡 通俗类比：自回归生成就像"接龙游戏"——给定"今天天气"，预测下一个词"很"；给定"今天天气很"，预测下一个词"好"。每一步都基于之前所有词来预测下一个词。
⚠️ 常见误区：GPT 的"自回归"意味着生成是有顺序的——每个 token 依赖之前所有 token。这限制了并行性(生成时不能并行)但保证了"因果性"(不会看到未来的信息)。这是设计选择，不是缺陷。

四、技术细节

💡 通俗类比：GPT-2 的"不要发布"决定就像"核武器发明后的犹豫"——OpenAI 担心模型被滥用(生成假新闻)。这是AI伦理的重要里程碑——技术能力首次超过了社会准备程度。
⚠️ 常见误区：GPT-2 的"太大太危险"在今天看来有点可笑(它只有15亿参数)。但这个决定开创了"负责任发布"的先例——后来的 GPT-3、GPT-4 都采用了渐进式发布策略。
🔬 深入探讨：从 GPT-1(117M)到 GPT-2(1.5B)到 GPT-3(175B)，参数量增长了1500倍。但性能提升不是线性的——GPT-3 展现了"涌现能力"(emergent abilities)，如少样本学习、思维链推理。这些能力在小模型中完全不存在——"量变引起质变"。

4.1 BPE 分词

Byte Pair Encoding：统计语料中频繁出现的字符对，合并为新 token。介于字符级和词级之间，兼顾稀有词处理和序列长度。

4.2 In-Context Learning (上下文学习)

GPT-2 的 Few-shot 示例: Prompt: "Translate English to French: sea otter => loutre de mer peppermint => menthe poivrée plush giraffe =>" GPT-2 输出: "girafe en peluche" 没有梯度更新！纯粹通过上下文模式学习 这是 GPT-3 Few-shot 的前身

五、使用场景

💡 通俗类比：GPT 系列就像"通用写手"——能写代码、写诗、写邮件、翻译、摘要、问答……它不是每个任务的专家，但"什么都能做一点"。这种"通用性"是它最大的优势。
⚠️ 常见误区：GPT 不等于 ChatGPT！ChatGPT 是经过 RLHF(人类反馈强化学习)对齐后的版本——更安全、更有帮助、更诚实。原始 GPT 可能会生成有害内容。"对齐"是 GPT 实用化的关键步骤。
🔬 深入探讨：GPT-3 的"few-shot learning"改变了AI应用的范式——不再需要为每个任务收集数据、训练模型，只需要写一个好的 prompt。这种"提示工程"(Prompt Engineering)成为了新的"编程方式"——用自然语言"编程"AI。
  • 文本生成：故事续写、文章生成
  • Zero-shot 任务：翻译、摘要、问答(通过 prompt)
  • 代码生成：GPT-2 的代码版本(CodeGPT)用于代码补全
  • 对话系统：早期对话模型的基础

六、相似技术对比

💡 通俗类比：GPT vs BERT 就像"自动续写"vs"阅读理解"——GPT 看到前文续写后文(只能看左边)，BERT 同时看前后文理解含义。两者是"互补"的，不是"竞争"的。
⚠️ 常见误区：GPT-3 的175B参数 vs BERT-Large 的340M参数——参数量差距约500倍。但 BERT 在某些任务(如 NER)上仍然更好。"更大"不等于"在所有任务上更好"——任务特性决定了哪个模型更合适。
🔬 深入探讨：GPT 和 BERT 的区别不仅在架构(解码器 vs 编码器)，更在"世界观"——GPT 把一切视为"续写"(生成范式)，BERT 把一切视为"理解"(判别范式)。ChatGPT 证明了"一切皆可生成"的范式更加强大和通用。

七、关联技术

💡 通俗类比：GPT-3 到 ChatGPT 的跨越主要靠"对齐"(Alignment)——RLHF 就像"家长教育"——教会模型"什么该说，什么不该说"。能力(预训练)已经够了，关键是学会"正确使用能力"。
⚠️ 常见误区：Prompt Engineering 不是"万能的"。GPT-3 有时给出荒谬的答案(幻觉)，有时拒绝合理请求(过度谨慎)。"对齐税"(Alignment Tax)——对齐会降低模型的某些能力——是一个真实的trade-off。
🔬 深入探讨：GPT 的"通用性"引发了一个深刻的问题：如果一个模型什么都能做一点，那专业模型还有存在的必要吗？答案是"看场景"——在高风险场景(医疗、法律)中，专业模型的可靠性和可解释性更重要。

九、自回归模型的数学原理

💡 数学直觉：自回归语言模型的目标是学习联合概率分布 P(x₁, x₂, ..., xₙ) 的分解。利用链式法则，它可以分解为条件概率的乘积：P(x₁)·P(x₂|x₁)·P(x₃|x₁,x₂)···P(xₙ|x₁,...,xₙ₋₁)。GPT 就是用 Transformer 解码器来参数化这些条件概率。

GPT-1 在 Penn Treebank 上的困惑度约 35.76，GPT-2 在 WebText 上约 18.34(更大更好的数据)。GPT-3 在多个基准上进一步降低。困惑度与模型质量强相关，但不完全等同于下游任务性能。

9.2 Scaling Laws 的数学形式

Kaplan 等人(2020)发现了语言模型的 Scaling Laws：L(N) ∝ N^{-α_N}，L(D) ∝ D^{-α_D}，L(C) ∝ C^{-α_C}，其中 L 是损失，N 是参数量，D 是数据量，C 是计算量。α_N ≈ 0.076，α_D ≈ 0.095，α_C ≈ 0.050。这意味着模型性能可以随资源投入的增加而可预测地提升。

Chinchilla (2022) 修正了 Scaling Laws：给定固定计算预算，模型参数量和数据量应按 1:20 的比例增长。即 1B 参数的模型需要约 20B token 的训练数据。这改变了 GPT-3 时代的"大模型小数据"范式。

🔬 深入探讨：Scaling Laws 的发现是大语言模型时代最重要的经验规律之一。它表明：(1) 模型性能不会饱和，而会随规模持续提升；(2) 性能提升是可预测的——可以在小规模实验上预测大规模结果；(3) 存在最优的资源分配策略。这些发现直接指导了 GPT-4、Claude、Gemini 等模型的训练策略。

9.3 Temperature 与采样策略

Temperature Sampling 通过调整 softmax 的温度参数 T 来控制生成的多样性：P(xᵢ) = exp(logit_i/T) / Σⱼ exp(logit_j/T)。T→0 时退化为贪心解码(最确定)，T→∞ 时退化为均匀分布(最随机)。T=1 是原始分布。实践中 T 通常取 0.7-1.0。

Top-k Sampling 只从概率最高的 k 个词中采样。Top-p (Nucleus) Sampling 从累积概率超过 p 的最小词集中采样。Top-p 的优势是自适应——当模型很确定时(少数词概率很高)，自动减少候选词；当模型不确定时，自动增加候选词。

不同采样策略的对比: 贪心解码 (T=0): 始终选择概率最高的词 → 输出确定但单调，容易重复 Temperature (T=0.7): 调整概率分布的"锐度" → 平衡质量和多样性，最常用 Top-k (k=50): 只从概率最高的50个词中采样 → 避免低概率词，但候选数固定 Top-p (p=0.9): 从累积概率>0.9的词中采样 → 自适应候选数，效果最佳 组合策略: Temperature=0.7 + Top-p=0.9 → 实践中最常用的组合

十、GPT 系列的演进与影响

10.1 从 GPT-1 到 GPT-3 的范式转变

GPT-1 (2018) 证明了"预训练+微调"范式的有效性。GPT-2 (2019) 证明了足够大的语言模型可以做"零样本"任务——不需要微调，只用提示词就能完成翻译、问答等任务。GPT-3 (2020) 将规模推到 175B 参数，证明了"In-Context Learning"——模型可以从几个示例中"学习"新任务，而不需要更新参数。

这个演进过程揭示了一个深刻的趋势：随着模型规模的增大，"显式训练"(微调)逐渐被"隐式学习"(In-Context Learning)所取代。GPT-3 的 Few-shot 学习能力在很多任务上接近甚至超过了微调的小模型。

⚠️ 范式革命：GPT-3 的出现标志着 AI 范式从"训练一个模型做一件事"转变为"用一个模型做所有事"。这降低了 AI 应用的门槛(不需要为每个任务收集数据和训练模型)，但也带来了新的挑战(模型的可靠性、可控性、安全性)。

10.2 Pre-Norm 与训练稳定性

GPT 系列使用 Pre-Norm Transformer：x + Sublayer(LayerNorm(x))。与原始 Transformer 的 Post-Norm 相比，Pre-Norm 的梯度流动更稳定，训练不容易发散。这对于大规模训练(数百 GPU、数周训练时间)至关重要——训练中断的代价极高。

然而 Pre-Norm 也有缺点：模型的表达能力可能略低于 Post-Norm，因为 LayerNorm 在子层之前会"压缩"特征的范围。DeepNorm (2022) 提出了改进的 Post-Norm 初始化方法，结合了 Post-Norm 的表达能力和 Pre-Norm 的训练稳定性。

10.3 GPT-2 的 Zero-shot 能力

GPT-2 的论文标题"Language Models are Unsupervised Multitask Learners"揭示了一个重要发现：在大规模语料上训练的语言模型隐式地学会了各种任务(翻译、摘要、问答)。模型不需要显式地被训练做这些任务——它从语言数据的统计规律中"涌现"了这些能力。

这种"涌现"能力的数学解释是：语言模型的目标函数(预测下一个词)隐式地包含了各种子任务。例如，预测"The capital of France is ___"需要知道地理知识；预测翻译对中的下一个词需要理解两种语言。

💡 核心洞察：GPT 系列的成功证明了一个简单而深刻的原则——用足够大的模型在足够多的数据上训练一个简单的目标函数(预测下一个词)，模型会"涌现"出各种复杂的能力。这个原则指导了后续所有大语言模型的开发。

# GPT 因果 Transformer Block
import torch, torch.nn as nn

class GPTBlock(nn.Module):
    def __init__(self, d_model, nhead, d_ff):
        super().__init__()
        self.ln1 = nn.LayerNorm(d_model)
        self.attn = nn.MultiheadAttention(d_model, nhead, batch_first=True)
        self.ln2 = nn.LayerNorm(d_model)
        self.ffn = nn.Sequential(nn.Linear(d_model, d_ff), nn.GELU(), nn.Linear(d_ff, d_model))
    def forward(self, x):
        seq_len = x.size(1)
        mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool().to(x.device)
        x = x + self.attn(self.ln1(x), self.ln1(x), self.ln1(x), attn_mask=mask)[0]
        x = x + self.ffn(self.ln2(x))
        return x

import numpy as np
def chinchilla_optimal(compute_budget):
    N_opt = (compute_budget / 6) ** 0.5

✅ 参考答案与详解
  1. 核心原理: 自回归分解P(x₁,...,xₜ)=∏P(xₜ|x_{关键要点:①分解捕获语言统计规律 ②预测需要语法语义语用知识 ③数据包含世界信息 ④最大化似然=压缩数据 ⑤涌现来自压缩
  2. 核心原理: GPT-2的关键洞察是：所有NLP任务都可以形式化为"给定上下文预测下一个token"。翻译："English: {text} French:"后预测法语。摘要："Article: {text} Summary:"后预测摘要。问答："Q: {question} A:"后预测答案。LLM预训练看到的prompt模式使其学会了这种"格式→响应"的映射，无需微调参数。
    关键要点:①语言模型是通用接口 ②任务编码在prompt中 ③格式→响应映射 ④无需微调多样性数据 ⑤提示工程激活能力
  3. 核心原理: T→0时采样变成greedy decoding(每步选概率最高的词)，输出确定但可能重复无趣。T→∞时输出趋近均匀随机采样(完全随机从词表取词)。Top-p=0.9意味着：每步只从累积概率前90%的最小词集合中采样(Top-p=0.9通常保留几十个词)。Top-p既避免低概率的"胡话"又保持多样性。
    关键要点:①T→0变greedy ②T→∞变随机 ③Top-p保留高概率词集合 ④平衡多样性与质量 ⑤不同任务不同T设置
  4. 核心原理: ICL可从前向传播实现"梯度下降"的角度理解：Transformer的注意力机制在prompt中检索相关信息(类似attention-based梯度)，FFN层应用学到的变换(类似MLP-based梯度)。Levi等人的证明表明单层Transformer的前向传播可以模拟一步梯度下降。ICL相当于通过前向传播(而非反向传播)执行了元学习的"内环优化"。
    关键要点:①注意力实现元学习梯度 ②前向传播模拟梯度下降 ③无需参数更新 ④提示=任务编码 ⑤Xie等人证明ICL与贝叶斯推断等价
  5. 核心原理: Kaplan Scaling Laws指出：测试损失L ∝ N^{-α}·C^{-β}(N=参数量,C=计算量)。但Chinchilla发现过去的大模型"训练不足"——同等计算量下，较小模型+更多数据+更多步数效果更好。计算最优的Chinchilla法则：D=20D(N=参数量, D=token数)。GPT-3训练了300B token但按Chinchilla应训约1.4T token，远未达最优。
    关键要点:①L∝N^{-α}C^{-β} ②Chinchilla=D=20N ③GPT-3训练不足 ④小模型+大数据更优 ⑤现在是混合范式

import torch
def temperature_sampling(logits, temperature=1.0):
    scaled = logits / temperature
    probs = torch.softmax(scaled, dim=-1)
    return torch.multinomial(probs, num_samples=1).item()

def top_p_sampling(logits, p=0.9):
    sorted_logits, sorted_idx = torch.sort(logits, descending=True)
    cumulative_probs = torch.softmax(sorted_logits, dim=-1).cumsum(dim=-1)
    # 移除累积概率超过p的词
    mask = cumulative_probs - torch.softmax(sorted_logits, dim=-1) > p
    sorted_logits[mask] = -float('inf')
    probs = torch.softmax(sorted_logits, dim=-1)
    idx = torch.multinomial(probs, 1).item()
    return sorted_idx[idx].item()

# 使用示例
logits = torch.tensor([2.0, 1.0, 0.5, 0.1, -1.0])
print(f"T=0.5:", temperature_sampling(logits, 0.5))  # 更确定性
print(f"T=2.0:", temperature_sampling(logits, 2.0))  # 更多样
print(f"Top-p=0.9:", top_p_sampling(logits, 0.9))

关键知识点: (1)自回归分解捕获语言规律 (2)所有任务可编码为prompt (3)温度控制多样性 (4)Top-p避免低概率词 (5)ICL通过前向传播实现元学习。
