2020 · Alexey Dosovitskiy 等 (Google) · 图像切成Patch当序列处理 · 打破CNN垄断

一、背景与动机

2020年之前，CNN 是视觉领域的绝对王者。虽然 Transformer 在 NLP 大获成功，但直接用于图像有困难：图像像素太多(224×224=50176个像素)，Self-Attention 的 O(n²) 复杂度无法承受。

ViT 的解决方案极其简单：把图像切成 16×16 的 patch，每个 patch 当作一个"词"(token)，然后直接用标准 Transformer Encoder 处理。224×224 图像变成 196 个 token——NLP 模型轻松处理的序列长度。

💡 核心创新：不是设计新的视觉架构，而是把图像"翻译"成 Transformer 能理解的格式。简单粗暴，但在大数据预训练下效果惊人。

二、名字由来

💡 通俗类比：ViT(Vision Transformer)就是"用 Transformer 处理图像"——把图像切成16×16的 patch，每个 patch 当作一个"词"，然后用标准 Transformer 处理。"万物皆可序列化"是它的核心思想。
⚠️ 常见误区：ViT 不是"第一个用 Transformer 做视觉的"——之前有 DETR(目标检测)等。但 ViT 是第一个证明"纯 Transformer 不需要 CNN 也能在图像分类上达到 SOTA"的——前提是数据足够多。
🔬 深入探讨：ViT 论文的核心发现：在大数据(ImageNet-21K或JFT-300M)上预训练后，ViT 超越了 CNN；但在小数据(ImageNet-1K)上，ViT 不如 CNN。这说明 Transformer 缺少 CNN 的"归纳偏置"(局部性、平移不变性)——需要更多数据来"学到"这些偏置。

ViT：Vision(视觉) + Transformer。

Patch(补丁/块)：把图像切成不重叠的小块，类似拼图的碎片。

三、核心原理

💡 通俗类比：Patch Embedding 就像"把照片切成拼图块"——224×224的图像被切成196个16×16的 patch，每个 patch 被线性投影为一个向量(类似"词向量")。然后196个"词"进入标准 Transformer。
⚠️ 常见误区：ViT 的 [CLS] token 和 BERT 的 [CLS] token 作用相同——聚合整个序列的信息用于分类。但 ViT 的位置编码是可学习的1D编码(不是2D)——它不知道 patch 的空间位置关系，需要自己学。

3.3 数据需求

ViT 在 ImageNet(1.3M图像)上从零训练不如同等大小的 CNN(因为没有 CNN 的归纳偏置：局部性+平移不变性)。但在 JFT-300M(3亿图像)上预训练后，ViT 超越了所有 CNN！

🔬 归纳偏置 vs 数据：

CNN 有两个硬编码的先验知识(归纳偏置)：① 卷积核只看局部(局部性) ② 同一卷积核用于所有位置(平移不变性)。这些先验在小数据时帮助很大。

ViT 没有这些先验——它需要从数据中学习这些模式。数据量小时学不好，但数据量足够大时，它能学到比硬编码更灵活的模式。

四、技术细节

💡 通俗类比：ViT-B/16 的"B"是 Base(86M参数)，"16"是 patch 大小(16×16)。ViT-L/16 是 Large(307M参数)。patch 越小，序列越长，计算量越大——ViT-B/32(32×32 patch)比 ViT-B/16(16×16 patch)快4倍但精度稍低。
⚠️ 常见误区：ViT 在 ImageNet-1K(130万图像)上从零训练时不如 ResNet——它需要 ImageNet-21K(1400万)或 JFT-300M(3亿)预训练。数据饥渴是纯 Transformer 在视觉中的主要限制。
🔬 深入探讨：ViT 之后出现了大量改进：DeiT(数据高效训练)、Swin Transformer(层次化+窗口注意力)、BEiT(视觉自监督预训练)。Swin Transformer 引入了"局部窗口注意力"——在小窗口内做注意力(保留局部性)，跨窗口做信息交换(保留全局性)。

五、使用场景

💡 通俗类比：ViT 的意义在于"统一了视觉和语言的架构"——同一个 Transformer 可以处理文本和图像。这直接催生了多模态模型(GPT-4V、Gemini)——"一个模型理解所有模态"。
⚠️ 常见误区：ViT 不是"在所有视觉任务上都比 CNN 好"。在小数据集、实时检测、移动端部署等场景中，CNN(如 EfficientNet、MobileNet)仍然更实用。"更强"不等于"更合适"。
🔬 深入探讨：ViT 的成功引发了"归纳偏置有多重要"的讨论。CNN 有明确的归纳偏置(局部性、平移不变性)，ViT 没有——需要从数据中学。结论是：大数据下归纳偏置不重要(可以自己学)，小数据下归纳偏置很重要(省数据)。
  • 图像分类：ImageNet SOTA(大规模预训练后)
  • 目标检测：DETR 用 Transformer 做检测
  • 语义分割：SegFormer、SETR
  • 多模态：CLIP(图文对比学习)、DALL-E(文生图)
  • 视频理解：ViViT 把视频切成时空 patch

六、相似技术对比

💡 通俗类比：ViT vs CNN 就像"全局阅读"vs"局部扫描"——ViT 同时看所有 patch(全局注意力)，CNN 逐局部扫描(卷积)。哪种更好取决于数据量和任务。
⚠️ 常见误区：Swin Transformer 不是"更好的 ViT"——它引入了 CNN 的"层次化"和"局部性"。这说明"纯 Transformer"可能不是最优解——"混合架构"(结合 Transformer 和 CNN 的优势)可能是更好的方向。
🔬 深入探讨：ViT 的 patch 大小(16×16)是一个关键超参数——patch 越小，分辨率越高，但计算量按平方增长。最近的"分层 ViT"(如 PVT、MiT)在不同层使用不同分辨率——浅层用小 patch(高分辨率)，深层用大 patch(低分辨率)。

七、关联技术

💡 通俗类比：ViT 和 CLIP 的关系就像"眼睛"和"理解"——ViT 提供视觉特征(CLIP 的视觉编码器)，CLIP 把视觉特征和文本特征对齐到同一空间。这让"用文字搜图片"成为可能。
⚠️ 常见误区：ViT 的预训练数据(JFT-300M)是非公开的——这让其他研究者难以复现。DeiT(2021)通过更好的数据增强和训练策略，只用 ImageNet-1K 就达到了可比的性能——"数据不够，技巧来凑"。
🔬 深入探讨：ViT 的成功直接催生了"多模态基础模型"——DALL-E(图像生成)、GPT-4V(图像理解)、Flamingo(视觉语言模型)都基于 ViT 的视觉编码器。"统一架构"的梦想正在成为现实。
  • Transformer(模型14)：ViT 的架构基础
  • CLIP：图文对比学习，ViT 做图像编码器
  • DALL-E / Stable Diffusion：ViT 在生成模型中的应用
  • Swin Transformer：引入局部窗口注意力，提高数据效率

八、实际效果

💡 通俗类比：ViT 在 ImageNet 上的 top-1 准确率约88%(大规模预训练后)，与最好的 CNN 持平或略优。但这不是最重要的——最重要的是它证明了"Transformer 也可以做视觉"——打开了新世界的大门。
⚠️ 常见误区：ViT 的计算量(O(n²)，n是patch数)在高分辨率图像上很昂贵——448×448图像有784个patch，注意力矩阵有784×784≈60万元素。处理视频(每帧784个patch × 帧数)更是天文数字。效率仍然是 ViT 的瓶颈。
🔬 深入探讨：ViT 论文的Figure 1(展示不同层注意力的可视化)揭示了一个有趣的现象——底层注意力是局部的(类似CNN)，高层注意力是全局的。这说明 Transformer 自己学会了"先看局部再看全局"——不需要人为设计层次结构。

九、技术深度：注意力可视化与理解

9.1 ViT 注意力图的层次化特征

ViT 论文中最令人印象深刻的发现之一是注意力图的层次化模式。研究者可视化了不同层的 [CLS] token 对其他 patch 的注意力权重，发现了一个清晰的规律：

  • 底层（Layer 1-4）：注意力主要集中在局部区域——与当前 patch 相邻的 patch 获得更高权重。这与 CNN 的卷积核行为非常相似。
  • 中层（Layer 5-8）：注意力开始扩散——模型开始关注更大范围的区域，捕获中等尺度的特征（如物体的部件）。
  • 高层（Layer 9-12）：注意力几乎全局均匀——模型关注整张图像的所有区域，进行全局推理和分类决策。

这个发现的意义在于：Transformer 自己学会了从局部到全局的层次化特征提取——不需要像 CNN 那样人为设计层次结构。这意味着给定足够的数据，Transformer 可以自动发现最优的特征提取策略。

9.2 Patch Size 对性能的影响

Patch size 是 ViT 中最重要的超参数之一。它直接影响序列长度、计算量和模型性能：

Patch Size序列长度(224²输入)注意力计算量ImageNet Top-1适用场景
8×8784784² = 614K最高高精度、不计成本
14×14256256² = 65K高平衡精度与效率
16×16196196² = 38K良好ViT 标准配置
32×324949² = 2.4K较低移动端、实时推理

Patch 越小，序列越长，计算量按平方增长（Self-Attention 的 O(n²) 复杂度）。但更小的 patch 保留了更多空间细节，通常能获得更高的精度。实际应用中需要在精度和效率之间权衡。

9.3 DeiT：数据高效的 ViT 训练

ViT 原始论文依赖 JFT-300M（3亿图像，非公开）预训练，这限制了研究社区的复现。Facebook 的 DeiT（Data-efficient Image Transformers, 2021） 通过以下技巧，仅用 ImageNet-1K（130万图像）就达到了可比性能：

  • 强数据增强：RandAugment、Mixup、CutMix、Random Erasing 等多种增强技术叠加使用。
  • 正则化：Stochastic Depth（随机丢弃层）、Label Smoothing（标签平滑）、Repeated Augmentation（重复增强）。
  • 知识蒸馏：引入一个 CNN 教师模型（RegNet），通过蒸馏 token 将 CNN 的知识传递给 ViT。
  • 训练策略：更长的训练周期（300 epoch → 1000 epoch）、更精细的学习率调度。

9.4 ViT 的位置编码深入分析

ViT 使用可学习的 1D 位置编码——每个 patch 位置对应一个可学习的向量。令人惊讶的是，即使只用 1D 编码，模型也能学到 2D 空间结构。通过计算不同位置编码之间的余弦相似度，研究者发现：

  • 空间上相邻的 patch，其位置编码向量的相似度更高。
  • 同一行/列的 patch，其位置编码也表现出较高的相似度。
  • 模型自动学到了"行列结构"——尽管没有显式提供 2D 信息。

这说明 Transformer 有很强的自适应学习能力——即使没有先验知识，也能从数据中发现结构。但这也意味着需要更多数据来学习这些模式，这正是 ViT 在小数据集上不如 CNN 的原因。从数学角度看，位置编码的原理是将离散的位置索引映射到连续的向量空间，使得模型可以通过向量运算来编码位置关系。

💡 进阶推荐阅读：论文 "An Image is Worth 16x16 Words" (Dosovitskiy et al., 2020)。重点关注 Figure 5（注意力可视化）和附录中的消融实验——它们揭示了 ViT 的核心设计选择为什么有效。论文的数学推导表明，Self-Attention 的计算复杂度与 patch 数量的平方成正比，这是 ViT 效率的根本限制。理解这一工作机制有助于选择合适的 patch size 和模型规模。

10.1 ViT 在下游任务中的微调策略

ViT 在 ImageNet 上预训练后，可以通过微调适应各种下游视觉任务。微调策略的选择对最终性能有显著影响：

  • 全量微调（Full Fine-tuning）：更新所有参数。这是最直接的方法，在数据量充足时效果最好。但需要注意学习率设置——通常比预训练学习率小 10-100 倍（如 1e-4 → 1e-5），以避免破坏预训练学到的特征。
  • 线性探测（Linear Probing）：冻结 ViT 主干，只训练最后的分类头。这种方法快速且不易过拟合，适合数据量小的场景。但性能通常比全量微调低 2-5 个百分点。
  • LoRA 微调：在注意力层的 Q、K、V 矩阵上添加低秩适配器。只训练适配器参数（约为主干参数的 0.1-1%），效果接近全量微调但显存需求大幅降低。
  • Adapter：在每个 Transformer 层后插入小型 Adapter 模块。冻结主干参数，只训练 Adapter。与 LoRA 类似但架构不同。
微调方法可训练参数显存需求ImageNet 精度适用场景
全量微调100%高最高数据充足、计算充裕
Linear Probing~0.1%极低较低快速评估、数据极少
LoRA~0.5%低接近全量资源受限、多任务
Adapter~1%低接近全量多任务持续学习

微调时还需要注意分辨率调整：ViT 预训练通常使用 224×224 或 384×384 分辨率。微调时如果使用更高分辨率（如 512×512），需要对位置编码进行插值。BiCubic 插值是最常用的方法，但可能损失一些精度。最近的 NTK-aware 插值方法效果更好，可以在不额外训练的情况下支持更高分辨率。

🔬 深入探讨：ViT 的迁移学习为什么有效？

ViT 在大规模数据上预训练后，其不同层学到了不同层次的视觉特征。底层学到了通用的纹理和边缘检测器（类似于 Gabor 滤波器），中层学到了形状和纹理组合，高层学到了语义概念（如"狗的耳朵"、"车轮"）。微调时，底层特征几乎不需要改变（它们是通用的），中高层特征需要适应新任务的类别。这就是为什么低学习率微调效果好的原因——它保护了底层的通用特征。

# 使用 PyTorch 和 timm 加载预训练 ViT 模型进行推理
import torch, timm
from PIL import Image
from torchvision import transforms

model = timm.create_model("vit_base_patch16_224", pretrained=True)
model.eval()

transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.5, 0.5, 0.5], std=[0.5, 0.5, 0.5]),
])

image = Image.open("cat.jpg")
input_tensor = transform(image).unsqueeze(0)
with torch.no_grad():
    output = model(input_tensor)
    predicted_class = output.argmax(dim=1).item()
print(f"Predicted class: {predicted_class}")
# ImageNet class 281 = "tabby cat"

10.2 ViT 在多模态模型中的角色

ViT 的最大影响不仅在于图像分类，更在于它成为了多模态 AI 的视觉基础。CLIP 使用 ViT 作为视觉编码器，与文本编码器通过对比学习对齐。DALL-E 2 使用 CLIP 的 ViT 编码器生成图像嵌入，GPT-4V 将 ViT 编码的视觉特征注入语言模型中。ViT 的真正价值在于它统一了视觉和语言的表示空间。

# 一、背景与动机 性能基准测试
import torch
import time

def benchmark(model, input_data, warmup=10, iters=100):
    for _ in range(warmup):
        model(input_data)
    torch.cuda.synchronize()
    start = time.perf_counter()
    for _ in range(iters):
        model(input_data)
    torch.cuda.synchronize()
    elapsed = (time.perf_counter() - start) / iters
    print(f"Latency: {elapsed*1000:.2f}ms")
    return elapsed

import torch, time
def benchmark(model, x, n=100):
    for _ in range(10): model(x)
    torch.cuda.synchronize()
    t0 = time.perf_counter()
    for _ in range(n): model(x)
    torch.cuda.synchronize()
    print(f"Latency: {(time.perf_counter()-t0)/n*1000:.2f}ms")

✅ 参考答案与详解
  1. 核心原理: CNN有三大归纳偏置：平移不变性(卷积核共享)、局部性(局部连接)、层次化(池化)。ViT几乎没有归纳偏置——全局注意力学习所有关系，需要更多数据来"发现"这些先验。CNN靠归纳偏置在小数据上泛化，但这也限制了其在大数据上的上限。ViT大数据下更优因为它直接从数据学习最优关系表示。
    关键要点:①CNN三偏置②ViT几乎无偏置③样本效率vs上限折中④CNN小数据泛化好⑤大数据ViT上限高
  2. 核心原理: Patch Embedding是nn.Conv2d(3, d_model, kernel_size=patch_size, stride=patch_size)。更大Patch：更短序列(计算量↓)、更粗粒度(细粒度特征丢失)、参数量不变。小Patch(4×4)计算量剧增因为序列长度O(n²/d²)。DeiT用14×14平衡效果与效率。
    关键要点:①等价于stride=ks卷积 ②Patch大序列短短 ③细粒度vs计算量折中 ④小Patch计算量O(1/p²) ⑤14/16常用
  3. 核心原理: CNN的归纳偏置(平移不变/局部/池化)在小数据上提供强有力的先验，减少过拟合。但这些偏置也成为上限——CNN无法学到超越这些偏置的新模式。ViT没有偏置，大数据下可以学到最优特征组合。JFT-300M上ViT-L/16超过了所有CNN模型。
    关键要点:①CNN偏置帮助小数据 ②偏置成为上限 ③ViT大数据学最优模式 ④JFT上ViT超越CNN ⑤迁移能力ViT更强
  4. 核心原理: DeiT蒸馏：CNN教师(如RegNetY-16GF)的软标签提供类别间关系(如猫更像老虎而非汽车的概率)，这传递了CNN学到的空间先验。硬标签只提供one-hot分类信息。此外DeiT引入 distillation token（与CLS token并行）通过蒸馏损失使ViT学习教师的行为。仅12个epoch DeiT在ImageNet上达85.2%。
    关键要点:①软标签传递类别关系 ②CNN空间先验蒸馏 ③distillation token ④硬标签信息量小 ⑤12 epoch 85.2%
  5. 核心原理: ViT浅层注意力：各位置均匀关注邻居(类似CNN的局部感受野)。深层注意力：出现明显的远近分化——local head关注空间邻居(捕获细节)，global head关注语义相关区域(如狗头→狗身)。这种分工是网络自组织形成的。可视化显示约20%的head为global head。
    关键要点:①浅层类似CNN局部 ②深层局部/全局分工 ③local head捕获细节 ④global head语义关联 ⑤自组织形成无显式设计

import torch, torch.nn as nn

class PatchEmbed(nn.Module):
    def __init__super().__init__()
        self.proj = nn.Conv2d(in_ch, embed_dim, patch_size, patch_size)  # (B,3,224,224)→(B,768,14,14)
        self.n_patches = (img_size // patch_size) ** 2  # 196
    def forward(self, x):
        x = self.proj(x).flatten(2).transpose(1,2)  # (B, 768, 196)→(B, 196, 768)
        return x

class ViTBlock(nn.Module):
    def __init__super().__init__()
        self.norm1 = nn.LayerNorm(dim)
        self.attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)
        self.norm2 = nn.LayerNorm(dim)
        self.mlp = nn.Sequential(nn.Linear(dim, dim*mlp_ratio), nn.GELU(), nn.Linear(dim*mlp_ratio, dim))
    def forward(self, x):
        x = x + self.attn(self.norm1(x), self.norm1(x), self.norm1(x))[0]
        x = x + self.mlp(self.norm2(x))
        return x
