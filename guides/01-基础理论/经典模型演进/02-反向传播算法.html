⚙️ 02 · 反向传播算法 (Backpropagation)

1986 · 让神经网络能够"学习"的核心引擎 · Rumelhart, Hinton, Williams

一、背景与动机

感知机有简单的学习规则(更新权重)，但只能训练单层网络。MLP 有多个隐藏层，那隐藏层的权重怎么更新？输出层的误差如何传递到前面的层？

1986年，David Rumelhart、Geoffrey Hinton 和 Ronald Williams 在《Nature》上发表论文，将反向传播(Backpropagation, BP)算法应用于神经网络训练，解决了这个问题。BP 的核心思想是：链式法则(Chain Rule)——把输出误差逐层向后传递，计算每个权重对误差的贡献。

💡 一句话总结：前向传播算预测，反向传播算梯度。梯度告诉你每个权重该往哪个方向调、调多少。

二、名字由来

💡 通俗类比："反向"就像你打碎了花瓶后回溯"谁碰的→谁推的→谁放的"——从结果往回追溯原因。反向传播就是从"预测错多少"往回追溯"每个权重该负多少责任"。
⚠️ 常见误区：反向传播不是一种"优化器"——它只是计算梯度的方法。SGD、Adam 才是优化器。反向传播告诉你"往哪走"，优化器决定"走多快"。两者经常被混淆。
🔬 深入探讨：反向传播的本质是链式法则(Chain Rule)的高效实现。1986年 Rumelhart 等人的贡献不是发明了链式法则，而是展示了如何在多层网络中系统地应用它。这个"系统地应用"才是关键创新。

Backpropagation：back(反向/向后) + propagation(传播)。即误差信号从输出层反向传播回输入层。这个算法最早可以追溯到 1970年 Linnainmaa 的自动微分工作，但 Rumelhart/Hinton 将其用于神经网络训练并证明了其有效性。

中文有时也叫"误差反向传播"或"BP算法"。在深度学习框架(PyTorch/TensorFlow)中，就是 loss.backward() 背后做的事情。

三、核心原理

💡 通俗类比：前向传播像"做菜"——把食材(Input)一步步加工成成品(Output)。反向传播像"品尝反馈"——"太咸了"(Loss)→"盐放多了"(Weight3偏大)→"调料勺太大"(Weight2偏大)→"手劲太大"(Weight1偏大)。每一步都在追溯原因。
⚠️ 常见误区：计算图中的"局部梯度"不是最终梯度！很多初学者把 ∂L/∂z 和 ∂L/∂W 搞混。记住：最终梯度 = 上游梯度 × 本地梯度。漏乘上游梯度是最常见的实现bug。
🔬 深入探讨：反向传播的计算复杂度和前向传播几乎相同(O(n))，这是它高效的原因。但如果用数值微分(有限差分)，每个参数需要一次前向传播，总共需要 O(n²) 次。这就是为什么"自动微分"取代了"数值微分"。

3.2 链式法则详解

假设一个简单的两层网络：

z₁ = w₁·x + b₁ (线性变换 1)
h = σ(z₁) (激活函数)
z₂ = w₂·h + b₂ (线性变换 2)
ŷ = σ(z₂) (输出激活)
L = -(y·log(ŷ) + (1-y)·log(1-ŷ)) (交叉熵损失)

目标：求 ∂L/∂w₁（损失对第一层权重的梯度）

∂L/∂w₁ = ∂L/∂ŷ · ∂ŷ/∂z₂ · ∂z₂/∂h · ∂h/∂z₁ · ∂z₁/∂w₁

每一项都可以单独计算：

偏导数项计算公式含义
∂L/∂ŷ-(y/ŷ - (1-y)/(1-ŷ))损失对预测的梯度
∂ŷ/∂z₂ŷ(1-ŷ)Sigmoid 的导数
∂z₂/∂hw₂线性层的梯度
∂h/∂z₁h(1-h)隐藏层 Sigmoid 导数
∂z₁/∂w₁x输入值

3.4 计算图与自动微分

现代深度学习框架(PyTorch、TensorFlow)使用自动微分(Automatic Differentiation, AD)来实现反向传播。你不需要手写梯度公式——框架会自动追踪前向计算构建计算图，然后自动反向求梯度。

# PyTorch 示例
import torch
import torch.nn as nn

# 定义网络
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 10)
)

# 前向传播
x = torch.randn(32, 784)  # batch_size=32
y = torch.randint(0, 10, (32,))
ŷ = model(x)
loss = nn.CrossEntropyLoss()(ŷ, y)

# 反向传播 (一行代码搞定！)
loss.backward()  # 自动计算所有参数的梯度

# 更新权重
with torch.no_grad():
    for p in model.parameters():
        p -= 0.01 * p.grad  # 梯度下降

四、技术细节

🔬 深入探讨：Adam 优化器的 β₁=0.9, β₂=0.95 不是随意选的。β₁ 控制一阶矩(动量)的衰减，相当于平均最近约10步的梯度；β₂ 控制二阶矩(方差)的衰减，相当于平均约20步。这些超参数的最优值与损失曲面的"曲率"有关。

4.1 梯度消失与梯度爆炸

反向传播中，梯度要经过多次乘法。如果每层的梯度因子 < 1，深层的梯度会指数级缩小(梯度消失)；如果 > 1，则指数级增大(梯度爆炸)。

⚠️ 梯度消失的直接影响：这就是为什么在 2012 年之前，超过 5-6 层的网络很难训练。ReLU 的引入(2011)和 ResNet 的残差连接(2015)是解决这个问题的两个关键里程碑。

4.2 随机梯度下降 (SGD) 与 Mini-batch

方法每次更新用的数据优点缺点
批梯度下降(BGD)全部数据梯度精确慢，内存大

4.3 优化器演进

优化器年份核心思想公式
SGD经典沿梯度方向走θ = θ - η·∇L
SGD+Momentum1964加入"动量"加速收敛v = βv + η∇L; θ = θ - v
AdaGrad2011自适应学习率(稀疏特征学更快)θ = θ - η/√(Σg²+ε) · g
RMSProp2012指数移动平均更新学习率v = βv + (1-β)g²; θ = θ - η/√v · g
Adam2014Momentum + RMSPropm=β₁m+(1-β₁)g; v=β₂v+(1-β₂)g²
AdamW2017Adam + 解耦权重衰减Adam梯度 - λθ (L2正则解耦)
💡 实践建议：2024年以后，LLM训练普遍使用 AdamW。学习率通常用 cosine schedule 从峰值逐渐降低。典型超参：lr=1e-4~3e-4, β₁=0.9, β₂=0.95, weight_decay=0.1。

4.4 反向传播的时间复杂度

BP 的计算量大约是前向传播的 2 倍(需要计算梯度的矩阵乘法)。对于一个 L 层、每层 N 个神经元的网络：

前向: O(L · N²)
反向: O(L · N²)
总训练: O(L · N²)

这也是为什么 GPU 对深度学习如此重要——矩阵乘法是高度可并行的。

五、使用场景

💡 通俗类比：反向传播是深度学习的"通用发动机"——无论你训练 CNN、RNN、Transformer 还是 GAN，底层都是反向传播在计算梯度。就像内燃机可以用在汽车、轮船、发电机上。
⚠️ 常见误区：反向传播要求所有操作都是可微的。但 argmax、sort、round 等操作不可微！这就是为什么 Gumbel-Softmax、Straight-Through Estimator 等技巧被发明——让"近似可微"替代"不可微"。
🔬 深入探讨：现代 LLM 训练中，反向传播面临一个新挑战：序列太长导致激活值(中间结果)占用大量显存。梯度检查点(Gradient Checkpointing)用"时间换空间"——丢弃部分前向结果，需要时重新计算。
  • 所有神经网络训练：CNN、RNN、Transformer、GAN、Diffusion——全部使用反向传播
  • 迁移学习：冻结前面层的梯度，只反向传播更新后面层
  • 对抗样本生成：用梯度对输入做 FGSM 攻击
  • 神经网络可解释性：GradCAM 利用梯度可视化 CNN 关注的区域
  • 元学习(Meta-Learning)：MAML 用二阶导数(梯度的梯度)做快速适应

六、相似技术对比

💡 通俗类比：自动微分(AD)像"精确会计"——追踪每一分钱的流向；数值微分像"粗略估计"——直接算差值；符号微分像"代数推导"——用公式求解。AD 是三者中精度和效率的最佳平衡。
⚠️ 常见误区：不要把"自动微分"和"符号微分"搞混。符号微分会生成巨大的表达式(表达式膨胀)，而自动微分直接计算数值结果，不会膨胀。PyTorch 和 TensorFlow 都用自动微分，不是符号微分。
🔬 深入探讨：前向模式 AD 和反向模式 AD 是对偶的。前向模式适合"输入少、输出多"(如计算 Jacobian 的列)，反向模式适合"输入多、输出少"(如 loss 是标量)。深度学习用反向模式，因为参数(输入)远多于 loss(输出)。
对比项反向传播数值微分符号微分
计算方式链式法则逐层求导(f(x+ε)-f(x))/ε解析求导公式
精度精确(浮点精度)近似(截断误差)精确(符号表达式)
效率O(n) 同前向传播O(n) 每个参数一次表达式爆炸
内存需存储中间激活值低高(表达式树)
应用深度学习标准梯度检查符号计算软件

七、关联技术

💡 通俗类比：反向传播和优化器的关系就像"导航"和"驾驶"——反向传播告诉你方向(梯度)，优化器决定怎么开车(SGD是手动挡，Adam是自动挡)。两者缺一不可。
⚠️ 常见误区：很多人以为"用 Adam 就不用调参了"。Adam 只是自适应调整学习率，但 weight_decay、warmup、gradient clipping 等仍然需要手动调。"自适应"不等于"全自动"。
🔬 深入探讨：反向传播在分布式训练中有特殊挑战——每个 GPU 需要同步梯度。AllReduce 通信是瓶颈。梯度压缩(如1-bit Adam)、异步更新等技术都在尝试减少这个通信开销。
  • 计算图：PyTorch 的动态图(DAG)是反向传播的实现基础
  • 梯度裁剪(Gradient Clipping)：防止梯度爆炸，RNN 训练必备
  • 混合精度训练：FP16 前向 + FP32 梯度累积，节省显存
  • 梯度累积：模拟大 batch 的效果，小显存也能训大模型
  • 分布式训练：DDP 中梯度通过 AllReduce 同步

八、实际效果

💡 通俗类比：看优化器的效果就像比较不同导航 app——同样的目的地(最优 loss)，不同的路线(优化轨迹)。有的走高速(SGD+动量)，有的走小路(Adam)，最终可能到达不同的"终点"(局部最优)。
⚠️ 常见误区：Adam 收敛快不等于最终效果好。在很多 NLP 任务中，SGD+动量的最终泛化性能优于 Adam。快和好是两个维度——短跑选手不一定能跑马拉松。
🔬 深入探讨：为什么 LLM 几乎都用 AdamW 而不是 SGD？因为语言模型的损失曲面非常复杂(高维、多鞍点)，Adam 的自适应学习率能更好地导航这种曲面。而 SGD 在简单曲面上的泛化更好——"简单"问题用"简单"方法反而更优。

九、PyTorch 自动微分与反向传播

9.1 PyTorch 计算图与 autograd

import torch

# PyTorch 自动构建计算图并自动反向传播
x = torch.tensor([2.0], requires_grad=True)
w1 = torch.tensor([3.0], requires_grad=True)
w2 = torch.tensor([4.0], requires_grad=True)

# 前向传播：y = w1 * w2 * x
y = w1 * w2 * x   # y = 24
loss = (y - 20) ** 2  # L = 16

# 反向传播：自动计算所有梯度
loss.backward()
print(f"dL/dw1 = {w1.grad}")  # 32
print(f"dL/dw2 = {w2.grad}")  # 24

9.2 手动实现反向传播

import numpy as np

def manual_backprop():
    # 前向传播
    x = np.array([1.0, 2.0])
    W1 = np.array([[0.1, 0.3], [0.2, 0.4]])
    b1 = np.array([0.01, 0.02])
    z1 = W1 @ x + b1           # 线性变换
    a1 = np.maximum(0, z1)     # ReLU 激活

    W2 = np.array([[0.5, 0.6]])
    b2 = np.array([0.03])
    z2 = W2 @ a1 + b2
    y_hat = 1 / (1 + np.exp(-z2))  # Sigmoid

    # 反向传播
    y_true = 1.0
    dz2 = y_hat - y_true
    dW2 = dz2 * a1
    db2 = dz2
    da1 = W2.T * dz2
    dz1 = da1 * (z1 > 0)
    dW1 = np.outer(dz1, x)
    db1 = dz1
    return dW1, db1, dW2, db2

9.3 梯度检验实现

def gradient_check(model, x, y, epsilon=1e-5):
    # 解析梯度（反向传播计算）
    pred = model(x)
    loss = criterion(pred, y)
    loss.backward()
    analytic_grads = [p.grad.clone() for p in model.parameters()]

    # 数值梯度（有限差分近似）
    for p, ag in zip(model.parameters(), analytic_grads):
        numerical_grad = torch.zeros_like(p.grad)
        for idx in range(p.numel()):
            old_val = p.data.view(-1)[idx].item()
            p.data.view(-1)[idx] = old_val + epsilon
            loss_plus = criterion(model(x), y).item()
            p.data.view(-1)[idx] = old_val - epsilon
            loss_minus = criterion(model(x), y).item()
            p.data.view(-1)[idx] = old_val
            numerical_grad.view(-1)[idx] = (loss_plus - loss_minus) / (2 * epsilon)
        diff = torch.norm(ag - numerical_grad) / (torch.norm(ag) + torch.norm(numerical_grad))
        print(f"relative error: {diff.item():.2e}")

十、反向传播的历史演进

反向传播算法的演进历程： 1969: Minsky 证明单层感知机局限 | 1974: Werbos 博士论文首次提出 BP 概念（未被注意） | 1986: Rumelhart, Hinton, Williams 独立发现 BP 并发表于 Nature | ← 这是真正让 BP 广为人知的里程碑 | 1989: LeCun 将 BP 应用于卷积网络(LeNet) | 2010: GPU 加速反向传播（深度学习爆发前夜） | 2012: AlexNet + BP + GPU = ImageNet 冠军（深度学习元年） | 2015: 自动微分框架兴起（Theano → TensorFlow → PyTorch） | 2017+: 混合精度训练、梯度累积、分布式 BP（大规模训练）

  2. 为什么Sigmoid激活函数会导致梯度消失？请从数学角度证明Sigmoid的导数最大值是0.25，并说明为什么深层网络中梯度会指数级衰减。
  3. 对比SGD、SGD+Momentum、Adam三种优化器的核心思想和适用场景。为什么LLM训练几乎都用AdamW而不是SGD？
  4. 什么是梯度检查(Gradient Check)？为什么在实现自定义反向传播时需要它？请描述其实现原理和误差容忍度的合理范围。
  5. 反向传播和自动微分(Automatic Differentiation)是什么关系？为什么说反向传播只是自动微分的一种特殊模式（反向模式）？
✅ 参考答案与详解
  1. 核心原理: 链式法则的完整应用。前向传播：z₁ = w₁·x = 0.5×2 = 1.0, a = σ(1.0) = 0.731, y = w₂·a = 0.3×0.731 = 0.219。损失L = ½(0.219-0.8)² = 0.169。反向传播：∂L/∂y = (y-0.8) = -0.581, ∂y/∂w₂ = a = 0.731, ∂L/∂w₂ = -0.581×0.731 = -0.425, ∂y/∂a = w₂ = 0.3, ∂a/∂z₁ = σ'(z₁) = a(1-a) = 0.731×0.269 = 0.197, ∂z₁/∂w₁ = x = 2。因此∂L/∂w₁ = -0.581×0.3×0.197×2 = -0.0686。
    关键要点: (1)前向传播计算每层输出；(2)反向传播从输出层向输入层逐层计算梯度；(3)每步梯度 = 上游梯度 × 本地梯度；(4)Sigmoid导数用σ(1-σ)计算；(5)最终梯度用于更新权重w₁ = w₁ - η·∂L/∂w₁。
    实际案例: 这就是PyTorch的loss.backward()背后做的事——自动计算每一步的梯度。手动计算帮助理解框架中的.loglik.backward()、误差反向传播等概念的本质。
  2. 核心原理: Sigmoid函数σ(z) = 1/(1+e⁻ᶻ)，其导数为σ'(z) = σ(z)(1-σ(z))。令t=σ(z)∈(0,1)，则σ'=t(1-t)，这是开口向下的抛物线，最大值在t=0.5处，值为0.25。这意味着Sigmoid的梯度最大只有0.25。在N层网络中，如果每层的梯度因子都约为0.25，则最终梯度为0.25ᴺ。5层网络梯度约0.001，10层约0.000001——这就是"梯度消失"。深层网络的底层几乎无法更新权重。
    关键要点: (1)Sigmoid导数最大值严格为0.25；(2)梯度连乘导致指数衰减；(3)|z|越大导数越小(饱和区)；(4)解决方案：ReLU、残差连接、BatchNorm；(5)这也是为什么2012年前>5层网络很难训练。
    实际案例: 1994年Hochreiter的博士论文首次系统论述了梯度消失问题，导致神经网络研究停滞近20年。直到2011年AlexNet使用ReLU+GPU才重新激活深度学习。今天LLM有数百层但仍能训练，靠的是ReLU/GELU+残差连接+LayerNorm的组合拳。
  3. 核心原理: SGD每次更新只沿当前梯度方向(θ = θ - η∇L)，收敛慢且易震荡。SGD+Momentum加入"惯性"v = βv + ∇L，沿累积方向前进，减少震荡。Adam结合了动量(一阶矩m)和自适应学习率(二阶矩v)，并做了偏差修正。LLM用AdamW因为：(1)语言模型的损失曲面复杂(高维多鞍点)，Adam的自适应学习率能更好地导航；(2)解耦的weight_decay提供更好的泛化；(3)对超参数不敏感，不需要过多调参。SGD在简单曲面上泛化更好但需要精细调参。
    关键要点: (1)SGD简单但收敛慢；(2)Momentum加速收敛减少震荡；(3)Adam自适应调整每个参数的学习率；(4)AdamW解耦权重衰减；(5)复杂优化问题首选AdamW。
    实际案例: BERT训练使用AdamW(lr=1e-4, β₁=0.9, β₂=0.999, weight_decay=0.01)，预热10000步后线性衰减。GPT-3使用AdamW(lr=6e-5, β₁=0.9, β₂=0.95)。实践证明AdamW在NLP领域的收敛速度和稳定性远优于SGD。
  4. 核心原理: 梯度检查通过数值微分(有限差分)来验证解析梯度(反向传播计算)的正确性。原理：∂f/∂x ≈ (f(x+ε) - f(x-ε)) / (2ε)，ε通常取1e-5到1e-7。对每个参数分别计算数值梯度，与反向传播计算的解析梯度比较，相对误差应<1e-4。如果误差过大，说明反向传播实现有bug。
    关键要点: (1)数值微分是梯度的"黄金标准"但极慢(O(n)次前向)；(2)解析梯度(反向传播)只需一次反向遍历；(3)梯度检查只用于验证，不用于训练；(4)相对误差 = |a-b| / (|a|+|b|)；(5)误差容忍度：双精度<1e-7，单精度<1e-4。
    实际案例: 在实现自定义RNN或Transformer层时，建议先用小网络做梯度检查确认正确后再训练。PyTorch的torch.autograd.gradcheck()就是自动的梯度检查实现。Andrew Ng的深度学习课程强烈推荐此方法来调试网络。
  5. 核心原理: 自动微分(AD)是计算导数的通用技术，分为前向模式和反向模式。反向传播就是反向模式AD：对于函数f: ℝⁿ → ℝ，先计算前向值，再沿计算图反向传播梯度。前向模式适合"输入少输出多"的情况（输入维度d，一次前向可算所有输出对某一输入的导数），反向模式适合"输入多输出少"的情况（一次反向可得标量对所有参数的导数）。深度学习中参数(输入)远多于损失(输出)，所以用反向模式。
    关键要点: (1)AD是通用求导技术；(2)反向传播=反向模式AD；(3)前向模式vs反向模式的对偶性；(4)PyTorch的torch.autograd就是AD系统；(5)AD链式法则的应用方式。
    实际案例: JAX同时支持前向模式(jacfwd)和反向模式(jacrev)。对于损失函数L(θ)（参数百万级，输出标量），反向模式只需一次计算即可得所有梯度，而前向模式需要百万次计算。这就是为什么深度学习几乎只用反向模式。

请实现一个完整的Mini-two-layer网络训练流程，包含：

  1. 使用NumPy手动实现前向传播和反向传播
  2. 实现SGD、Momentum、Adam三种优化器
  3. 在合成数据集上比较三种优化器的收敛速度
  4. 实现梯度检查函数验证反向传播的正确性
💡 参考答案与详解

解题思路: 构建一个二分类问题，比较不同优化器的收敛行为。关键是正确实现反向传播和高效率的优化器更新规则。

import numpy as np
import matplotlib.pyplot as plt

# ===== 1. 生成合成数据(双月形) =====
np.random.seed(42)
N = 200
X = np.zeros((2*N, 2))
y = np.zeros((2*N, 1))
X[:N] = np.random.randn(N, 2) * 0.5 + np.array([0, 0])
y[:N] = 0
X[N:] = np.random.randn(N, 2) * 0.5 + np.array([1, 1])
y[N:] = 1

# ===== 2. 网络定义 =====
def forward(X, W1, b1, W2, b2):
    z1 = X @ W1 + b1
    a1 = np.tanh(z1)  # tanh激活
    z2 = a1 @ W2 + b2
    a2 = 1 / (1 + np.exp(-z2))  # sigmoid
    return z1, a1, z2, a2

def backward(X, y, z1, a1, a2, W2):
    m = len(X)
    dz2 = a2 - y
    dW2 = a1.T @ dz2 / m
    db2 = np.sum(dz2, axis=0, keepdims=True) / m
    da1 = dz2 @ W2.T
    dz1 = da1 * (1 - np.tanh(z1)**2)
    dW1 = X.T @ dz1 / m
    db1 = np.sum(dz1, axis=0, keepdims=True) / m
    return dW1, db1, dW2, db2

# ===== 3. 三种优化器 =====
def optimize(params, grads, optimizer_state, optimizer_name, lr=0.01):
    if optimizer_name == "sgd":
        for p, g in zip(params, grads):
            p -= lr * g
    elif optimizer_name == "momentum":
        beta = 0.9
        for i, (p, g) in enumerate(zip(params, grads)):
            v = optimizer_state['v'][i] = beta * optimizer_state['v'][i] + lr * g
            p -= v
    elif optimizer_name == "adam":
        beta1, beta2, eps = 0.9, 0.999, 1e-8
        optimizer_state['t'] += 1
        t = optimizer_state['t']
        for i, (p, g) in enumerate(zip(params, grads)):
            m = optimizer_state['m'][i] = beta1 * optimizer_state['m'][i] + (1-beta1) * g
            v = optimizer_state['v'][i] = beta2 * optimizer_state['v'][i] + (1-beta2) * g**2
            m_hat = m / (1 - beta1**t)
            v_hat = v / (1 - beta2**t)
            p -= lr * m_hat / (np.sqrt(v_hat) + eps)

# ===== 4. 训练比较 =====
opt_names = ["sgd", "momentum", "adam"]
all_losses = {}
for name in opt_names:
    np.random.seed(0)
    W1 = np.random.randn(2, 8) * 0.1; b1 = np.zeros((1,8))
    W2 = np.random.randn(8, 1) * 0.1; b2 = np.zeros((1,1))
    params = [W1, b1, W2, b2]
    state = {'v': [np.zeros_like(p) for p in params],
             'm': [np.zeros_like(p) for p in params],
             't': 0}
    losses = []
    for _ in range(2000):
        z1, a1, z2, a2 = forward(X, *params)
        loss = -np.mean(y * np.log(a2+1e-8) + (1-y) * np.log(1-a2+1e-8))
        losses.append(loss)
        grads = backward(X, y, z1, a1, a2, W2)
        optimize(params, grads, state, name, lr=0.1)
    all_losses[name] = losses

# ===== 5. 梯度检查 =====
def grad_check(params, eps=1e-5):
    for i, p in enumerate(params):
        numerical = np.zeros_like(p)
        total = p.size
        for idx in range(min(total, 10)):  # 只检查前10个元素
            orig = p.flat[idx]
            p.flat[idx] = orig + eps
            _,_,_,a2_plus = forward(X, *params)
            loss_plus = -np.mean(y*np.log(a2_plus+1e-8) + (1-y)*np.log(1-a2_plus+1e-8))
            p.flat[idx] = orig - eps
            _,_,_,a2_minus = forward(X, *params)
            loss_minus = -np.mean(y*np.log(a2_minus+1e-8) + (1-y)*np.log(1-a2_minus+1e-8))
            p.flat[idx] = orig
            numerical.flat[idx] = (loss_plus - loss_minus) / (2*eps)
        # 与解析梯度比较
        _,_,_,a2 = forward(X, *params)
        grads = backward(X, y, *(_ for _ in [None]*0), W2=W2)

关键知识点: (1)tanh导数=1-tanh²(z)；(2)Adam的偏差修正在训练早期尤为重要；(3)梯度检查只需验证少量参数；(4)Momentum的动量系数通常取0.9；(5)不同优化器的学习率范围不同(SGD通常需要更大的lr)。
