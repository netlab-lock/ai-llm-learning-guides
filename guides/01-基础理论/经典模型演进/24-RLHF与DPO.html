🎯 24 · RLHF & DPO — 让 LLM 学会"说人话"

2022/2023 · Ouyang/Rafailov 等 · 从"能说"到"会说" · ChatGPT 的关键技术

一、背景与动机

预训练的 LLM 会续写文本，但不会"回答问题"——它会把问题当作文章的开头继续写下去。SFT(监督微调)让它学会回答的格式，但人类的偏好很难用规则描述：什么是"好"的回答？准确？有帮助？无害？简洁？

RLHF 用人类反馈训练奖励模型，再用强化学习优化 LLM 的输出策略。DPO 则更优雅——绕过奖励模型，直接从偏好数据中优化。

💡 一句话总结：RLHF = 训练一个"评委"(RM)学习人类偏好 → 用"评委"指导"选手"(LLM)改进。DPO = 不需要评委，直接从比赛结果(人类偏好对)中学习。

二、名字由来

💡 通俗类比：RLHF(Reinforcement Learning from Human Feedback)就是"用人类的反馈来强化学习"——就像教练看运动员的表现，给"好/不好"的评价，运动员根据反馈改进。
⚠️ 常见误区：DPO(Direct Preference Optimization)不是 RLHF 的"升级版"——它是一种完全不同的方法。RLHF 用"奖励模型+PPO"，DPO 直接用"偏好数据"优化策略。DPO 更简单但可能不如 RLHF "精确"。
🔬 深入探讨：RLHF 的核心创新是"用人类偏好替代人类标注"——不需要告诉模型"正确答案"，只需要说"A比B好"。这比标注"正确答案"容易得多——因为"比较"比"创造"容易。这个洞察改变了整个对齐领域。

RLHF：Reinforcement Learning(强化学习) from Human Feedback(人类反馈)。

DPO：Direct(直接) Preference(偏好) Optimization(优化)。直接从偏好数据优化，不需要中间的奖励模型。

GRPO：Group Relative(组相对) Policy Optimization(策略优化)。DeepSeek 的变体，用组内排名替代绝对评分。

三、核心原理

💡 通俗类比：RLHF 的三步走——(1)SFT：教模型"基本对话能力"(像学生学说话)；(2)RM：训练一个"裁判"(像训练评委)；(3)PPO：用"裁判"的分数指导模型改进(像根据评委反馈练习)。
⚠️ 常见误区：RLHF 的"奖励模型"不是"万能裁判"——它只能反映训练数据中人类偏好的"平均"。不同标注者的偏好可能冲突——"谁的偏好"是 RLHF 的核心伦理问题。
🔬 深入探讨：DPO 的数学推导证明了：在最优情况下，RLHF(奖励模型+PPO)和 DPO(直接偏好优化)是等价的。DPO 把两步(训练RM→优化策略)合并为一步(直接优化)——"简化"而不"降低效果"。

3.1 RLHF 三步走

3.2 为什么需要 KL 约束？

如果没有 KL 散度惩罚，模型会"hack"奖励模型——找到 RM 的漏洞，生成高分但无意义的文本(类似考试作弊)。

3.3 DPO (Direct Preference Optimization)

DPO 的关键洞察: RLHF 的最优策略有闭式解: π*(y|x) = (1/Z(x)) · π_ref(y|x) · exp(r(x,y)/β) 反解出奖励函数: r(x,y) = β · log(π*(y|x)/π_ref(y|x)) + β·log Z(x) 代入 Bradley-Terry 模型: P(y_w > y_l) = σ(r(x,y_w) - r(x,y_l)) = σ(β·[log π(y_w|x)/π_ref(y_w|x) - log π(y_l|x)/π_ref(y_l|x)]) → 不需要单独训练 RM！直接从偏好对优化策略！ DPO 损失: L_DPO = -E[log σ(β · (log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)))] y_w: 人类偏好的回答 (winner) y_l: 不偏好的回答 (loser) π_ref: 参考模型(SFT后的模型, 冻结) β: 温度参数(控制偏离程度)

3.5 GRPO (Group Relative Policy Optimization)

DeepSeek-R1 使用的改进：去掉 Critic 模型，用组内相对排名作为奖励信号。

四、技术细节

💡 通俗类比：PPO(Proximal Policy Optimization)的"Proximal"是"不要走太远"的意思——每次更新策略时，限制新策略不要偏离旧策略太远。这就像"每次只改进一点点"——稳定但可能慢。
⚠️ 常见误区：PPO 的训练非常不稳定——需要大量调参(学习率、KL惩罚、clip范围)。很多团队转向 DPO 或 GRPO(Group Relative Policy Optimization)——更简单更稳定。"PPO 是标准"不意味着"PPO 是最好的"。
🔬 深入探讨：GRPO(DeepSeek, 2024)用"组内相对比较"替代了"奖励模型打分"——对同一问题生成多个回答，用规则(如数学正确性)直接计算奖励。这完全不需要人类标注——"自动对齐"可能比"人类对齐"更可扩展。

4.1 Bradley-Terry 模型

RLHF 的 RM 训练基于 Bradley-Terry 偏好模型：

P(y_w ≻ y_l | x) = σ(r(x, y_w) - r(x, y_l))

假设人类偏好概率由两个回答的奖励分数之差通过 sigmoid 函数决定。

4.2 PPO 算法核心

L_PPO = E[min(r_t(θ)·A_t, clip(r_t(θ), 1-ε, 1+ε)·A_t)]
r_t(θ) = π_θ(a_t|s_t) / π_old(a_t|s_t) (概率比)
A_t = R_t - V(s_t) (优势函数)

PPO 通过裁剪(clipping)防止策略更新过大，保证训练稳定。

4.3 对齐税 (Alignment Tax)

RLHF/DPO 可能导致模型在某些基准上性能下降——因为优化"有帮助"可能牺牲了"准确"。这被称为"对齐税"。

五、使用场景

💡 通俗类比：RLHF 的应用就像"驯服野马"——预训练的LLM像一匹野马(有能力但不听话)，RLHF 教会它"什么该做，什么不该做"。最终的ChatGPT就像训练有素的赛马——既有力又守规矩。
⚠️ 常见误区：RLHF 不仅用于"安全对齐"——它还用于"提升有用性"。InstructGPT 的 RLHF 同时优化了"有帮助"和"无害"两个目标——但"有帮助"和"无害"有时冲突(如"教我做炸弹"——有帮助但有害)。
🔬 深入探讨：RLHF 的一个深层问题是"对齐税"(Alignment Tax)——对齐过程可能降低模型的某些能力(如创造力、多样性)。ChatGPT 比 GPT-3 更"安全"但也更"无聊"——这是"安全"和"有趣"之间的 trade-off。
  • ChatGPT / Claude / Gemini：所有商业对话模型都用了 RLHF 或其变体
  • LLaMA-2-Chat：Meta 用 RLHF 对齐开源模型
  • DeepSeek-R1：用 GRPO 训练推理能力
  • 安全性对齐：减少有害、偏见、虚假输出
  • 风格控制：让模型输出符合特定风格(简洁/详细/专业)

六、相似技术对比

💡 通俗类比：RLHF vs DPO vs GRPO 就像三种"训练宠物"方式——RLHF 是"给零食+口令"(奖励模型+策略优化)，DPO 是"直接比较好坏"(偏好数据直接优化)，GRPO 是"让宠物之间比赛"(组内相对比较)。
⚠️ 常见误区：DPO 虽然更简单，但在某些任务上不如 RLHF —— 特别是需要"探索"(生成多样回答)的场景。RLHF 的奖励模型可以提供"连续分数"，而 DPO 只用"二元比较"(A比B好)。"简单"不总是"更好"。
🔬 深入探讨：RLHF/DPO 的"偏好数据"是最大的瓶颈——需要大量高质量的人类标注。Constitutional AI(Anthropic, 2022)用"AI自我批评"替代部分人类标注——让模型自己判断"回答是否合适"。"自动化对齐"是对齐研究的终极目标。

七、关联技术

💡 通俗类比：RLHF 和 SFT(监督微调)的关系就像"基础教育"和"素质教育"——SFT 教"基本技能"(按指令回答)，RLHF 教"价值观"(什么是好的回答)。两者缺一不可。
⚠️ 常见误区：RLHF 的"KL散度惩罚"是关键——它限制策略不要偏离预训练模型太远。没有 KL 惩罚，模型会"作弊"——生成奖励模型喜欢但人类不喜欢的"对抗样本"。KL 是"安全阀"。

  • LLaMA(模型22)：LLaMA-2-Chat 用 RLHF 对齐

九、对齐技术的前沿发展

9.1 从 RLHF 到 RLHF 的局限

RLHF 虽然成功地让 ChatGPT 等模型变得"听话"，但它有明显的局限性：

  • 训练不稳定：PPO 算法本身就不稳定，加上 LLM 的高维参数空间，训练过程经常出现"reward hacking"——模型学会了"钻空子"获得高分，但实际输出质量下降。
  • 超参数敏感：KL 惩罚系数 β、PPO 的 clip ratio、学习率等超参数需要精心调整。调错可能导致模型"退化"（输出变得无趣或重复）。
  • 计算成本高：RLHF 需要同时维护四个模型（Policy、Reference、Reward、Value），显存和计算需求巨大。
  • 奖励模型偏差：RM 可能存在偏见（如偏好更长的回答），这些偏见会被 PPO 放大。

9.2 DPO 的理论推导

DPO（Direct Preference Optimization）的核心洞察是：RLHF 的最优解可以写成闭合形式。具体推导如下：

RLHF 的目标函数是：

max E[RM(x, y)] - β · KL(π_θ || π_ref)

通过变分法求解，最优策略为：

π*(y|x) = π_ref(y|x) · exp(RM(x,y)/β) / Z(x)

其中 Z(x) 是归一化常数。将 RM 从上式中解出：

RM(x, y) = β · log(π*(y|x) / π_ref(y|x)) + β · log Z(x)

将此代入 Bradley-Terry 偏好模型 P(y_w > y_l) = σ(RM(x,y_w) - RM(x,y_l))，Z(x) 项抵消，得到 DPO 的损失函数：

L_DPO = -E[log σ(β · log(π_θ(y_w|x)/π_ref(y_w|x)) - β · log(π_θ(y_l|x)/π_ref(y_l|x)))]

9.3 DPO 的改进变体

DPO 发布后，研究者提出了多种改进：

方法改进点核心思想
IPO防止过度优化用恒等函数替代 log sigmoid，避免 DPO 的过拟合
KTO只需要"好/坏"标签不需要成对比较，降低数据标注成本

💡 实践建议：如果你要训练一个"助手"模型，DPO 比 RLHF 更容易实现和调试。推荐流程：先用高质量数据做 SFT，再用偏好数据做 DPO。数据质量比数据量更重要——1000 条高质量偏好数据通常比 10 万条低质量数据效果更好。

10.1 Constitutional AI 与自我对齐

除了 RLHF 和 DPO，还有其他重要的对齐方法值得了解：

Constitutional AI（Anthropic, 2022）：这是一种"自我对齐"方法。核心思想是让 AI 模型根据一组"宪法原则"（如"不应生成有害内容"、"应保持诚实"）来自我批评和修正回答。流程如下：

  • 生成阶段：模型先生成可能有害的回答。
  • 批评阶段：模型根据宪法原则批评自己的回答，指出哪里违反了原则。
  • 修正阶段：模型根据批评修改回答，生成更安全的版本。
  • 训练阶段：用修正后的数据做 RLHF 或 DPO 训练。

RLAIF（Reinforcement Learning from AI Feedback）：用 AI 模型代替人类标注员生成偏好数据。Anthropic 的研究表明，用高质量 AI 生成的偏好数据训练的效果可以接近人类标注数据。这大幅降低了对齐训练的成本。

Self-Play：让模型与自身对弈，在竞争中提升能力。SPIN（Self-Play Fine-Tuning）方法让模型的当前版本和上一版本进行"对弈"，当前版本尝试生成比上一版本更好的回答。

10.2 对齐训练中的数据工程

对齐训练的效果高度依赖数据质量。以下是数据工程的关键原则：

  • 多样性：偏好数据应覆盖多种任务类型（问答、写作、编程、数学等），避免模型在某些任务上过拟合。
  • 难度梯度：数据应包含从简单到困难的各种案例。如果只有"明显好 vs 明显坏"的数据，模型学不到细微的判断能力。
  • 一致性：不同标注员的标注标准需要对齐。标注员间一致性（Inter-annotator Agreement）低于 70% 的数据通常不可靠。
  • 负样本质量：DPO 中的"被拒绝"回答质量很重要。如果拒绝回答太差，模型学不到"区分好坏"的能力。最好的拒绝回答是"看起来不错但有细微问题"。
💡 实践建议：对齐数据的质量检查比数量更重要。建议流程：① 用 GPT-4 或 Claude 生成初始偏好数据 ② 人工审核和修正 ③ 计算标注一致性 ④ 去除低一致性样本。经过这四步，1000 条高质量数据的对齐效果通常优于 10 万条未经审核的数据。
# DPO 损失函数简化实现
import torch, torch.nn.functional as F

def dpo_loss(policy_logps, ref_logps, beta=0.1):
    """DPO损失: 无需显式RM, 直接从偏好数据优化"""
    chosen_p, rejected_p = policy_logps
    chosen_r, rejected_r = ref_logps
    chosen_rewards = beta * (chosen_p - chosen_r)
    rejected_rewards = beta * (rejected_p - rejected_r)
    loss = -F.logsigmoid(chosen_rewards - rejected_rewards).mean()
    acc = (chosen_rewards > rejected_rewards).float().mean()
    return loss, {"reward_accuracy": acc.item()}

# 示例
loss, metrics = dpo_loss(
    (torch.randn(4) - 1.0, torch.randn(4) - 2.0),  # policy
    (torch.randn(4) - 1.5, torch.randn(4) - 1.5),   # reference
)
print(f"DPO Loss: {loss.item():.4f}, Acc: {metrics['reward_accuracy']:.0%}")

✅ 参考答案与详解
  1. 核心原理: Bradley-Terry模型：P(y_w≻y_l)=exp(r(x,y_w))/(exp(r(x,y_w))+exp(r(x,y_l)))=σ(r(x,y_w)-r(x,y_l))。最大化偏好对数似然log σ(Δr)使r(y_w)>r(y_l)。RM避免了绝对质量标注的困难(人类对"1-10分"标准不一)——只需判断"哪个更好"(对比判断)。这对人类标注者更容易且更一致。交叉熵损失训练RM。
  2. 核心原理: DPO核心洞察：最优策略π*与奖励函数r的关系为π*(y|x)=π_ref(y|x)exp(r(x,y)/β)/Z(x)。取对数得r(x,y)=β log(π*(y|x)/π_ref(y|x))+β log Z。代入BT模型后Z消去，直接用π和π_ref表达偏好概率。这样跳过了显式训练RM和PPO强化学习——直接从偏好数据优化语言模型。
  3. 核心原理: 无KL约束时模型会Reward Hacking：(1)生成冗余讨巧文本("当然！很高兴帮助您！这个问题的答案很重要因为...")骗高RM分；(2)输出不自然或过度谨慎；(3)重复已知模式。具体例子：GPT-3+RLHF初期发现模型学会了输出"N/A"（对高风险问题）以避免可能的惩罚。KL系数β=0.1-0.5控制偏离程度。
  4. 核心原理: GRPO与PPO区别：(1)GRPO用组内相对优势(同一问题的多个采样)替代baseline critic；(2)去掉value function网络；(3)对每个问题采样一组回答，用组内奖励均值和标准差归一化优势。更适合数学推理因为有明确正确/错误信号：正确回答的奖励明显高于错误，组内对比清晰。DeepSeek-R1用GRPO在数学推理上达到SOTA。
  5. 核心原理: RLAIF流程：(1)初始回答→(2)LLM根据宪法原则自我批评修改→(3)微调修改后的回答→(4)LLM比较两个版本选更合规的→(5)用偏好数据训练。优势：(1)减少人类标注成本；(2)可扩展；(3)一致性更高。局限：(1)LLM自身偏见会固化；(2)宪法原则本身反映设计者价值观；(3)可能过度保守。

import torch, torch.nn.functional as F

def dpo_loss(pi_logps_w, pi_logps_l, ref_logps_w, ref_logps_l, beta=0.1):
    """pi_logps: 策略模型对chosen/rejected的log prob
       ref_logps: 参考模型(SFT)的log prob"""
    pi_ratio = pi_logps_w - pi_logps_l
    ref_ratio = ref_logps_w - ref_logps_l
    logits = beta * (pi_ratio - ref_ratio)
    return -F.logsigmoid(logits).mean()

def grpo_advantage(rewards, group_indices):
    """对每个问题的多个采样做优势归一化
       rewards: (N,) group_indices: 每个采样属于哪个问题"""
    advantages = torch.zeros_like(rewards)
    for g in group_indices.unique():
        mask = group_indices == g
        group_rewards = rewards[mask]
        advantages[mask] = (group_rewards - group_rewards.mean()) / (group_rewards.std() + 1e-8)
    return advantages

def kl_penalty(pi_logits, ref_logits, beta=0.1):
    """KL散度约束防止策略偏离参考太远"""
    pi_probs = F.log_softmax(pi_logits, dim=-1)
    ref_probs = F.softmax(ref_logits, dim=-1)
    return beta * (pi_probs.exp() * (pi_probs - ref_probs.log())).sum(-1).mean()
