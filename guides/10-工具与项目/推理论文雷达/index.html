🚀 快速开始

通俗类比：就像组装宜家家具——先看说明书再动手，比瞎摸索快10倍。快速开始就是你的'说明书'，跳过这步后面全是返工。
常见误区：直接clone就跑，不看README的依赖版本要求。Python 3.9和3.11的包可能完全不兼容，浪费半天排环境问题。
深入探讨：典型环境配置：Python≥3.9，transformers≥4.35，torch≥2.0。首次运行建议先用--dry-run参数，避免下载数百MB模型权重到错误路径。pip install -r requirements.txt 通常需要5-10分钟。

常见误区：把所有优化技术混为一谈。量化是压缩模型参数，KV Cache是缓存计算结果，投机解码是预测生成——目标和原理完全不同。
深入探讨：核心分类维度：①模型压缩(量化INT8/INT4、剪枝、蒸馏)②推理加速(FlashAttention、PagedAttention)③采样优化(Speculative Decoding、Medusa)④调度优化(Continuous Batching)⑤硬件适配(vLLM、TensorRT-LLM)。

🔄 数据流

通俗类比：数据流就像快递物流——从揽收到分拣、运输、派送，每个环节出错都会丢件。理解数据流向，才能在出问题时快速定位'卡在哪一站'。
常见误区：忽略数据转换的中间格式。PDF→Markdown→JSON→AST→Graph，每个箭头都有损耗，不了解中间格式就无法debug解析错误。
深入探讨：典型Pipeline：PDF输入→PyMuPDF提取文本→正则清洗→LLM提取结构化信息→JSON存储→AST解析依赖→Neo4j导入图谱。每步延迟：PDF提取~2秒/篇，LLM调用~5-15秒/篇，AST解析~0.5秒/篇。

📖 功能说明

通俗类比：功能说明就像家电说明书——你用洗衣机只按'开始'，但不知道'快洗15分钟'和'标准洗45分钟'的区别，就永远洗不干净真丝衬衫。
常见误区：只看功能名称就猜测用法，不看参数说明。比如--top-k和--top-p都是采样参数，但一个控制数量一个控制概率，用错效果天差地别。
深入探讨：核心功能参数：--batch-size控制并发(默认8，显存不足设为1)、--max-tokens限制输出长度(默认2048)、--temperature控制随机性(0.0=确定性，1.0=高随机)、--top-p核采样阈值(默认0.9)。

🧠 知识图谱

通俗类比：知识图谱就像蜘蛛网——每根丝连接两个节点，整张网捕获所有关系。论文和技术就是节点，引用和依赖就是丝线。
常见误区：用LLM去构建图谱会很慢且不稳定。纯AST解析提取代码依赖关系更快更准，速度可提升10倍以上，且结果可复现。
深入探讨：AST解析原理：遍历Python代码的抽象语法树，提取import语句、函数调用、类继承关系。节点类型：Paper、Technique、Model、Metric。关系类型：CITES、IMPLEMENTS、OUTPERFORMS。典型论文可提取15-30个节点和40-80条边。
    Paper_A ──CITES──▶ Paper_B
       │                  │
  IMPLEMENTS         IMPLEMENTS
       ▼                  ▼
  FlashAttn ◀─COMPARES── PagedAttn
节点类型属性示例数量级
Papertitle, year, authors100-500篇
Techniquename, category, speedup50-200个
Modelname, params, latency20-100个
Metricname, unit, benchmark10-30个
# 构建图谱（纯 AST，不需要 LLM）

graphify cluster-only .

# 查询图谱
graphify query "FlashAttention 被哪些论文改进了？"
graphify path "PagedAttention" "vLLM"
