## 提示词 4: MoE 训练与路由策略

```

输出路径: D:\学习\AI-LLM技术\MoE训练与路由\

请生成以下文件 (每个文件 14-21KB):

01-MoE训练总览.html
- MoE 在 LLM 中的复兴: 从 Shazeer (2017) 到 Mixtral/DeepSeek-V3
- MoE vs Dense 模型的训练差异: 更少的计算量但更大的参数量
- MoE 训练的核心挑战: 负载均衡、路由坍塌、通信开销、训练不稳定
- MoE 训练的 Scaling Law: MoE 的计算效率优势量化

- MoE 训练流程的 ASCII 总览图

02-路由机制详解.html
- Token Choice Routing: 每个 token 选择 Top-K 个专家
  - 原理: Gate(x) = softmax(W_g · x), 选择得分最高的 K 个专家
  - Top-1 vs Top-2 的影响分析
- Expert Choice Routing: 每个专家选择 Top-K 个 token (Zhou et al. 2022)
  - 优势: 天然负载均衡、无需要辅助损失
  - 劣势: 可能丢弃部分 token
- Soft MoE: 所有专家都参与，但权重不同 (Puigcerver et al. 2024)
- 路由器的实现细节: capacity factor, auxiliary loss
- 路由决策的可视化方法
- 各路由机制的对比表: 均衡性、训练稳定性、推理效率、适用场景
- 路由机制的 ASCII 对比图

03-负载均衡与路由坍塌.html
- 路由坍塌 (Routing Collapse): 为什么少数专家会被过度激活
  - 正反馈循环: 被选多的专家训练更好 → 被选更多
  - 后果: 大部分专家闲置，MoE 退化为 Dense
- 辅助损失 (Auxiliary Loss):
  - Switch Transformer 的 load balancing loss
  - 公式: L_aux = α · N · Σ(f_i · p_i)
  - α 的选择: 过大损害模型质量，过小无法均衡
- DeepSeek-V3 的 Auxiliary-Loss-Free 路由:
  - 核心思想: 用 bias term 替代辅助损失
  - 原理: 给每个专家维护一个 bias，根据负载动态调整
  - 优势: 不需要调 α，不损害模型质量
- Z-loss: 训练稳定性的辅助损失
- 负载均衡的量化指标: CV (Coefficient of Variation)
- 各方法的对比表
- 路由坍塌的 ASCII 示意图

04-MoE训练的通信优化.html
- All-to-All 通信: MoE 的核心通信原语
  - 原理: 每个 GPU 将 token 发送到对应的专家 GPU
  - 通信量分析: 2 × (N-1)/N × tokens_per_gpu × hidden_dim
- Expert Parallelism (EP): 专家分布在不同 GPU 上
  - EP 与 TP/PP 的组合策略
- Token dispatching 策略:
  - 全部 All-to-All vs 分层 dispatch
  - DeepSeek-V3 的限制节点内 All-to-All
- 通信-计算重叠: 在计算 Expert FFN 的同时进行下一批 token 的 All-to-All
- Group-Limited Expert Selection: 限制每个 token 只能选择部分节点上的专家
- MoE 通信优化的 ASCII 时序图
- EP 与 TP/DP 组合的通信量对比

05-MoE-Upcycling.html
- 什么是 MoE Upcycling: 从预训练好的 Dense 模型初始化 MoE
- Upcycling vs 从头训练 MoE 的对比
- Upcycling 方法:
  - 复制 FFN 层为多个专家 (Mixtral 的方法)
  - 添加噪声初始化 (Jiang et al. 2024)
  - 渐进式 Upcycling: 逐步增加专家数量
- Upcycling 后的训练: 需要多少 token 才能恢复/超越原始 Dense
- Upcycling 的 Scaling: 8x7B vs 16x7B vs 64x7B
- Upcycling 的实际案例: Mixtral, DBRX
- Upcycling pipeline 的 ASCII 流程图

06-MoE推理特殊挑战.html
- 专家缓存 (Expert Caching): 当专家放不下显存时的 offloading 策略
- Expert Pruning: 剪枝掉不重要的专家
- 专家量化: 不同专家可以用不同精度
- Token 丢弃 (Token Dropping): 超过 capacity 时丢弃 token
- Speculative MoE: 用小模型预测路由
- 批处理中的不均匀激活: 不同 token 走不同专家导致的 batch 分裂
- MoE 推理 vs Dense 推理的延迟/吞吐对比
- MoE 推理优化的 ASCII 架构图

07-MoE的Scaling-Law.html
- MoE 的 compute-optimal 策略: 专家数量 vs 每专家大小 vs 总计算量
- MuSR (Mixture of Experts Scaling Rules): MoE 的 Scaling Law 研究
- Chinchilla for MoE: 如何在 MoE 下确定最优数据量
- 专家数量的 Scaling: 增加专家数量的收益递减规律
- MoE vs Dense 的 Scaling Law 对比
- 实际案例: DeepSeek-V3 的 MoE 设计选择 (256 专家, Top-8)
- MoE Scaling Law 的 ASCII 曲线示意

index.html — 总导航页，包含学习路线图和各文件链接
```

```
- 长上下文技术分类的 ASCII 树状图

02-位置编码外推.html
- 位置编码回顾: RoPE 的基本原理 (旋转矩阵、复数形式)
- 为什么 RoPE 不能直接外推: 高频分量的周期性问题
- Position Interpolation (PI): Chen et al. 2023
  - 原理: 将位置索引线性缩放到训练范围内
  - 公式: θ' = θ · L_train / L_target
  - 优缺点: 简单有效，但损失分辨率
- NTK-aware Scaling: Reddit "bloc97" 方法
  - 原理: 修改 RoPE 的 base frequency
  - 公式: θ' = θ · (α)^(d/(d-2))
  - 高频保真 vs 低频外推的权衡
- YaRN (Yet another RoPE extensioN): Peng et al. 2023
  - 原理: NTK-aware + 注意力缩放 + 温度因子
  - 三种方案: NTK-by-parts, Dynamic NTK, YaRN
- Code Llama 的方法: 修改 RoPE base + 长文本继续训练
- LongRoPE: 非均匀搜索每个维度的缩放因子
- 各方法的对比表: 支持倍数、训练需求、性能保持
- 位置编码外推的 ASCII 数学示意

03-ALiBi与无位置编码方案.html
- ALiBi (Attention with Linear Biases): Press et al. 2022
  - 原理: 不用位置编码，给注意力分数加线性偏置
  - 公式: attention(Q,K) = QK^T - m · |i-j|
  - 优势: 天然外推、训练效率高
  - 劣势: 对超长上下文的精度不如 RoPE 系列
- 无位置编码方案 (NoPE): 去掉位置编码后模型能否工作
- 位置编码的分类: 绝对位置编码 vs 相对位置编码 vs 无位置编码
- 各方案的外推能力对比
- 主流模型的位置编码选择: Llama (RoPE), BLOOM (ALiBi), GPT (learned)
- 位置编码方案的 ASCII 对比图

04-渐进式长度训练.html
- 为什么需要渐进式训练: 直接长上下文训练的不稳定性
- 渐进式长度训练策略:
  - 短→长分阶段训练 (Llama 3.1 的方法: 8K→128K)
  - 每阶段的训练数据量分配
  - 位置编码缩放因子的动态调整
- 长上下文数据构造:
  - 拼接短文档 vs 真实长文档
  - 长上下文的数据质量: 真实长文档比拼接的重要
  - 书籍、论文、代码库作为长上下文数据源
- 继续预训练 (Continual Pre-training) for 长上下文:
  - 学习率选择: 通常比初始预训练低
  - 训练数据: 长文档 + 短文档的混合
- Llama 3.1 的 128K 上下文训练流程详解
- 渐进式训练的 ASCII 时序图

05-长上下文训练的显存优化.html
- 长上下文训练的显存瓶颈: KV 存储、注意力矩阵、激活值
- Sequence Parallelism (SP): Megatron-LM 的实现
  - 将序列维度分割到多个 GPU
  - Ring Attention: 跨 GPU 的分布式注意力
  - Ulysses (DeepSpeed): All-to-All 实现序列并行
- 激活重计算 (Activation Checkpointing) 的特殊考虑
- 梯度累积 (Gradient Accumulation) 策略
- Flash Attention 对长上下文训练的影响
- 各方案的显存节省与通信开销对比
- Sequence Parallelism 的 ASCII 示意图

06-长上下文评测方法.html
- Needle-in-a-Haystack 测试: 原理与局限
- RULER benchmark: 多维长上下文能力评测
- LongBench: 中英双语长上下文评测
- 长上下文的能力分解:
  - 单点检索 (Single retrieval)
  - 多点检索 (Multi-key retrieval)
  - 聚合推理 (Aggregation)
  - 排序推理 (Sorting)
- 位置偏差分析: 信息在不同位置的检索准确率
- "Lost in the Middle" 问题: 中间位置信息容易丢失
- 评测结果的解读方法
- Needle-in-a-Haystack 测试的 ASCII 热力图示意

index.html — 总导航页，包含学习路线图和各文件链接
```

```
- State Space Models (SSM) 回顾: S4 → Mamba → Mamba-2
- Mamba-2 的关键发现: SSM 和注意力是同一个框架的两个特例

  - 矩阵形式: 将 SSM 写成半可分 (semiseparable) 矩阵乘法

  - 优势: 可并行训练 + 高效推理

- Jamba (AI21, 2024): Mamba + Attention 的交替层
  - 架构: Mamba 层处理长序列，Attention 层处理精确检索
  - 混合比例: 不同层使用不同的 Mamba/Attention 比例
  - MoE 版本: Jamba 1.5
- Zamba (Zyphra, 2024): 共享 Attention 层 + Mamba 主体
- Griffin (Google, 2024): Gated Linear Recurrence + Local Attention
- StripedHyena (Together AI, 2024): 长卷积 + 短卷积 + 注意力
- 混合架构的设计选择: 哪些层用 Attention，哪些用 SSM
- 混合架构的性能对比表
- 混合架构的 ASCII 架构图

  - WKV (Weighted Key-Value) 机制
  - 时间混合 (Time Mixing) 和通道混合 (Channel Mixing)
  - 递推形式: O(1) 推理复杂度
- RWKV-5 (Eagle): 多头机制、改进的注意力
- RWKV-6 (Finch): 数据依赖的线性注意力、动态衰减
- RWKV-7: 最新进展
- RWKV vs Transformer 的训练效率对比
- RWKV 在长序列推理中的优势
- RWKV 的 ASCII 递推展开图

06-其他新型注意力方案.html

  - Mistral 的实现: 固定窗口 + 滚动缓冲
  - Longformer 的 dilated sliding window

07-注意力方案选型指南.html
```

```
输出路径: D:\学习\AI-LLM技术\知识蒸馏\

请生成以下文件 (每个文件 14-21KB):

01-知识蒸馏总览.html
- 知识蒸馏的定义: 将大模型 (teacher) 的知识转移到小模型 (student)
- 知识蒸馏在 LLM 时代的复兴: 为什么比以前更重要

- LLM 蒸馏在 LLM pipeline 中的位置
- 知识蒸馏分类的 ASCII 树状图

02-经典蒸馏方法回顾.html
- Hinton 蒸馏 (2015): 软标签 + 温度缩放
  - 公式: L = α·L_CE(y, p_s) + (1-α)·T²·KL(p_t^T, p_s^T)
  - 温度 T 的作用: 平滑概率分布，传递 dark knowledge
- Feature-based 蒸馏: 匹配中间层表示
- DistilBERT: NLP 领域的经典蒸馏案例
- TinyBERT: 注意力矩阵蒸馏 + 隐藏层蒸馏
- 经典方法在 LLM 上的局限: 无法访问 logits (API-only)
- 经典蒸馏流程的 ASCII 图

03-黑盒蒸馏 (API蒸馏).html
- 黑盒蒸馏的场景: 只能访问 teacher 的文本输出
- Chain-of-Thought 蒸馏:
  - Orca 1/2 的方法: 用 GPT-4 生成详细推理过程
  - 训练数据格式: 问题 + 详细推理链 + 答案
  - 为什么有效: 小模型学到的不只是答案，还有推理过程
- Lima 方法: 用少量高质量数据对齐
- Evol-Instruct 在蒸馏中的应用
- WizardLM: 复杂度进化的指令数据

- 黑盒蒸馏 pipeline 的 ASCII 流程图

04-白盒蒸馏 (Logit蒸馏).html
- 白盒蒸馏的场景: 可以访问 teacher 的 logits
- 序列级蒸馏 (Sequence-level KD):
  - 用 teacher 的分布逐 token 蒸馏
  - 公式: L_KD = Σ_t KL(p_teacher(·|x, y_<t), p_student(·|x, y_<t))
  - KL 散度 vs 交叉熵的选择
- On-policy vs Off-policy 蒸馏:
  - Off-policy: 用 teacher 生成的数据训练 (分布不匹配)
  - On-policy: student 自己生成，teacher 打分 (分布匹配)
  - On-policy 蒸馏的效果通常更好
- Self-Imitation 蒸馏: student 从自己的好样本中学习
- GKD (Generalized Knowledge Distillation, Agarwal et al. 2024)

- 渐进式蒸馏: 逐步减小模型 (teacher1 → teacher2 → student)
- Layer-wise 蒸馏: 逐层匹配表示
- Attention Transfer: 匹配注意力矩阵
- 能力特化蒸馏: 只蒸馏特定能力 (数学/代码/推理)
- 蒸馏 + 量化组合: 先蒸馏再量化，或蒸馏时考虑量化
- 蒸馏 + LoRA: 在蒸馏过程中使用 LoRA 减少训练成本
- 各方法的适用场景对比表
```

```
02-权重平均方法.html
- Simple Averaging: 直接平均多个模型的权重
  - 公式: θ_merged = (1/N) · Σ θ_i
  - 为什么有效: loss landscape 的平坦区域
- SLERP (Spherical Linear Interpolation):
  - 原理: 在高维球面上插值 (而非直线)
  - 公式: θ_merged = sin((1-t)·α)/sin(α) · θ_1 + sin(t·α)/sin(α) · θ_2
  - 优势: 保持权重的范数和方向关系
- Weighted Averaging: 根据各模型质量加权
- 温度缩放平均: 调整 softmax 温度再平均

03-Task-Arithmetic.html
- Task Vector 的概念 (Ilharco et al. 2023):
  - 定义: τ = θ_finetuned - θ_pretrained
  - 直觉: task vector 编码了微调带来的"知识增量"
- 任务加法: θ_merged = θ_pre + Σ λ_i · τ_i
- 任务减法: 去除特定能力 θ_merged = θ - λ · τ_undesired
- Negation (任务否定): 去除有害能力
- Task Arithmetic 的 Scaling: λ 的选择与影响

04-DARE与TIES-Merging.html
- DARE (Drop And REscale, Yu et al. 2024):
  - 问题: 简单合并时参数干扰严重
  - 方法: 随机丢弃大部分 delta 参数 → 保留重要参数 → 重新缩放
  - 公式: δ'_i = m_i · δ_i / (1-p), m_i ~ Bernoulli(1-p)
  - 直觉: 大部分微调参数是冗余的，只保留关键参数
  - 效果: 显著减少合并时的干扰
- TIES-Merging (Yadav et al. 2024):
  - 三步流程: Trim (修剪小变化) → Elect Sign (投票选方向) → Disjoint Merge (合并)
  - Trim: 去掉绝对值小于阈值的参数
  - Elect Sign: 对每个参数位置，投票决定正方向或负方向
  - Disjoint Merge: 只合并方向一致的参数

  - 关键发现: 多个不同超参数的模型平均后比单个最优更好
```
