# 学习指南生成提示词 — P0 (最关键)

> 每个提示词用于生成该方向的 HTML 学习指南
> 输出格式: 多个 HTML 文件 + index.html，放在指定目录
> 风格: 中文，暗色主题，14-21KB/文件，含 ASCII 图/公式/对比表/术语表/练习

---

## 共用框架要求

每个提示词生成的指南必须遵循以下 8 维度结构:

1. **背景** — 这个技术是在什么问题/历史背景下诞生的？为什么需要它？
2. **名字由来** — 术语的来源、命名逻辑、首次出现的论文/项目
3. **原理** — 核心算法/机制的数学原理，要能教会一个有深度学习基础的人
4. **技术细节** — 实现层面的关键设计，代码级的伪代码或示意
5. **使用场景** — 什么时候用、怎么用、在哪些系统中出现
6. **相似技术对比** — 同类方案的横向对比表（性能、精度、复杂度、适用场景）
7. **关联技术** — 与上下游技术的关系，形成知识网络
8. **实际效果** — benchmark 数据、真实案例、部署经验

每个文件还需包含:
- ASCII 架构图或流程图 (至少 1 处)
- 数学公式 (LaTeX 格式，用 MathJax 渲染)
- 对比表格 (至少 1 处)
- 术语速查表 (文件末尾)
- 练习题 (2-3 道，含答案)
- 参考文献链接
