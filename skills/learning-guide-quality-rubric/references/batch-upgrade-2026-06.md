# 2026-06 批量升级实战：333文件A→S/A+流水线

## 背景
09-厂商与前沿目录333个HTML，初始A级(402-442分)。
92%文件有通用模板内容（同一tip复制到所有h2节，2105个问题点）。

## 4阶段执行流水线

### Phase 1: 文件级修复（Python脚本，1分钟）
- 修复练习题: 269个文件（基于h2标题生成针对性问题）
- 修复学习路径: 54个文件
- 修复向前串联: 149个文件
- 策略: 提取文件的h2标题，生成引用实际章节名的练习题
- 效果: +10~15分/文件

### Phase 2c: 通用内容替换（Python脚本，2分钟）
- 检测24种通用tip模式 + 5种warn + 3种deep + 2种h3模式
- 为每个h2节生成唯一tip/warn/deep
- 使用hashlib种子确保同一文件同一section总是生成相同内容
- 厂商上下文字典（17国产+5国际，含co/tech/focus/model字段）
- 20+主题类比库（注意力/MoE/推理/训练/量化/长上下文/多模态/Agent/部署/对齐等）
- 效果: 消除全部2105个通用模板问题，+5~15分/文件

### Phase 3: 结构性补充（Python脚本，1分钟）
- 为h3=0文件添加h3子节（31个文件，主要是国际LLM系列）
- 为deep_kw不足文件添加深度关键词段落（20个文件）
- 补充缺失练习题（35个）和学习路径（54个）
- 效果: +3~8分/文件

### 结果
- S级: 161个 (48.3%)

- A级: 25个 (7.5%)
- S+A+比例: 92%
- 分数范围: 429-450（初始402-442）

## 关键Pitfalls

### 1. 位置偏移Bug（最常见！）
循环中修改content后，后续regex match的位置全部错位。
**解法**: 收集所有(start, end, new_text)替换，按start逆序应用。

### 2. 子agent读写整个HTML的Token消耗
6个12-22KB HTML = 100K+ input tokens → 子agent超时中断。
**解法**: 用Python脚本直接处理，不经子agent。或子agent只输出JSON配置。

### 3. 评分函数一致性
评分脚本硬编码exercises=1会给所有文件假的+15分bonus。
**解法**: 动态检测`思考与练习|思考题|练习题`和`学习路径|学习路线`。

### 4. 通用内容检测模式覆盖不足
只检测3-5个模式会遗漏大量通用内容。需要24+模式完整覆盖。
最大的重复源: "本节概念是理解现代LLM系统的重要一环"出现231次。

### 5. f-string中的HTML实体花括号
`f'<div>{x}</div>'` 安全，但 `f'P(y_{{<t}})'` 会报错。
**解法**: 用字符串拼接或SafeDict.format_map()。
