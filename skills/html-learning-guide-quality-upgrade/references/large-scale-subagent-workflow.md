# 大规模子agent驱动工作流 (200+文件)

## 适用场景
- 200+个HTML文件需要提升到S级
- 每个文件需要理解主题并生成相关内容
- 不能用纯脚本批量处理（内容需要主题相关）

## 执行策略

### Phase 1: 分析与规划
1. 运行分析脚本获取所有文件的当前分数和缺失项
2. 按目录分组，统计每组文件数和平均分数
3. 按分数从高到低排序（接近S级的先处理，ROI最高）
4. 规划批次：每批3-4个文件，3个子agent并发

### Phase 2: 批量处理
```
Round 1: 最接近S级的目录（3个子agent × 3-4文件 = 9-12文件）
Round 2: 继续处理 + 重试429失败的批次
Round 3-15: 按目录推进
Round 16+: 精修接近S级的文件
```

### Phase 3: 质量审计
1. 分数验证（全部≥450）
2. 模板化tips检测和修复
3. 重复段落检测和修复
4. 死链检测和修复
5. CSS风格一致性检查
6. 内容深度抽查（读取3-5个文件的实际内容）

### Phase 4: 精修
- 对因修复导致分数下降的文件进行精修
- D1瓶颈诊断（chars≥8000但总分<450）
- 补充code_blocks/h3/deep_keywords

## 子agent任务模板

```
context: |
  项目路径: /path/to/project
  分析脚本: python3 /path/to/analyze_html.py <file_path>
  
  Style [A/B] CSS类: class="[box ]tip" class="[box ]warn" class="[box ]deep" class="ascii"
  
  文件列表:
  1. /path/to/file1.html (当前分数)
  2. /path/to/file2.html (当前分数)
  3. /path/to/file3.html (当前分数)
  
  操作步骤(每个文件):
  1. 运行分析脚本查看缺失项
  2. 读取文件全文理解主题
  3. 根据缺失项补充内容(保留已有内容)
  4. 写回文件
  5. 验证分数≥450

goal: 增强N个HTML学习指南到≥450分。每个补充中文技术内容(≥8000字)、
      tip≥3/warn≥2/deep≥2、ASCII图≥3、表格≥3。CSS用Style [A/B]。
toolsets: [terminal, file]
```

## 429限流处理
- 并发3个子agent是MiMo API的上限
- 遇到429：等待30-60秒后重试
- 连续3次429：降并发到2个
- 401≠429：401是key过期，需要换key

## 效率数据（2026-06-29实测）
- 217个文件，总计约15轮子agent调用
- 每轮3个子agent，每个处理3-4个文件
- 每轮耗时约10-30分钟
- 总耗时约6-8小时
- 429限流是主要瓶颈（约占30%的等待时间）
