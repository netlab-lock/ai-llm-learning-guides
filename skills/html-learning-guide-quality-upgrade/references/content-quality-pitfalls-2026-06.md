# 内容质量陷阱 — 2026-06-29 实战记录

## 背景
09-厂商与前沿目录333个HTML文件，17个国产厂商+5个国际厂商。
分数从100% S+A+降到71.2%的原因分析。

## 陷阱清单

### 1. H1标题含emoji污染模板内容
**影响**: 394处warn + 1655处deep
**示例**: `"关于🏥 百川智能 全系列系统性学习指南的一个重要误区"`
**根因**: 模板生成器直接用h1标题(含emoji)作为topic变量
**修复**: 用厂商名替换h1标题，清理emoji

### 2. 泛化表格927处
**影响**: 194个文件
**示例**: `<tr><td>核心概念</td><td>X的基本原理</td><td>理解底层机制比记忆参数更重要</td></tr>`
**检测**: `'理解底层机制比记忆参数更重要' in content`
**修复**: 删除+替换为厂商特定表格。⚠️删除前先准备替代！

### 3. 模板warn 6种模式1165处
| 模板 | 次数 |
|------|------|
| 时最容易犯的错误是"只看论文不看代码" | 204 |
| 的技术选型是"非此即彼"的 | 200 |
| 的评估结果可以直接用于生产 | 197 |
| 只需要关注技术实现就够了 | 196 |
| 的关键是算法创新 | 187 |
| 时不要陷入"技术完美主义" | 181 |

### 4. 截断warn 673处
**示例**: `"时建议采用渐进式学习策略"` (缺主语)
**修复**: 补全主语 `f'学习{mfr}技术{text}'`

### 5. 泛化tip 213处
**示例**: `"需要精准的参数调优和系统设计"` 出现191次
**修复**: 用厂商特定技术描述替换

### 6. "自动售货机/贩卖机"类比20处
**检测**: `'自动售货机' in content or '自动贩卖机' in content`
**修复**: 替换为更具体的类比

## 分数影响数据

| 操作 | S+A+ | 变化 |
|------|------|------|
| 初始 | 100% | - |
| 修复模板warn | 92.8% | -7.2% |
| 修复模板tip | 92.8% | 不变 |
| 移除泛化表格 | 64% | -28.8% |
| 添加主题表格 | 71.2% | +7.2% |

**结论**: 移除泛化表格是分数下降的最大原因。

## 正确的清理策略

1. **先诊断**: 运行全面审计，列出所有问题类型和数量
2. **先替代**: 为每个要删除的泛化内容准备主题特定的替代
3. **分批处理**: warn→tip→表格，每批验证分数
4. **长度保证**: 替换内容≥原内容100%长度
5. **多轮修复**: 删除→评分→补充→评分，直到收敛
