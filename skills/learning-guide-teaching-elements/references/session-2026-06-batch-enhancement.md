# 会话记录：批量增强教学元素的坑

## 问题背景

AI-LLM技术学习指南项目，~500个B级HTML文件需要增强教学元素(tip/warn/deep/ascii/table)。
初始批量增强使用关键词匹配模板，导致同一文件内所有h2节得到相同的通用内容。

## 发现的关键问题

### 1. 按文件路径匹配 vs 按h2标题匹配

初始脚本 `enhance_b_files.py` 的 `get_keyword_match()` 函数用文件路径+全文关键词匹配，
导致同一个文件的所有h2节都匹配到同一个类别，内容完全相同。

**修复**：用LLM API为每个文件一次性生成所有h2节的独特内容。

### 2. CSS检测误判

`.callout.tip {` 包含子串 `.tip {`，导致脚本认为CSS已存在。

**修复**：用正则 `(?<![a-zA-Z0-9_-])\.tip\s*\{` 匹配独立类定义。

### 3. 有CSS但没有教学元素

部分文件已有 `.tip {` CSS定义但没有 `class="tip"` 的教学元素。
初始检测只看CSS，漏掉了这些文件。

**修复**：同时检查CSS定义和实际元素使用。

### 4. MiMo API JSON输出问题

MiMo返回的JSON常有：非法反斜杠转义、控制字符、未转义换行。
需要三级fallback解析。

### 5. 大文件API超时

10+个h2节的文件prompt很长，MiMo响应超过300s。
**修复**：分批处理每批8节，timeout设600s。

### 6. 进程批量死亡

未捕获异常导致进程终止。
**修复**：主循环内try/except，单文件失败continue。

## 处理结果

500+文件修复完成，重复率从67-92%降到<20%。
单文件2-5分钟，全量约5-8小时（3并发含限流）。
