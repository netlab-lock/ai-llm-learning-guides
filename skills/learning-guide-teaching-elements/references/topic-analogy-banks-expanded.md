# 主题-类比映射库 (Topic Analogy Banks) — 扩展版

## 用途
为LLM学习指南的每个h2节生成唯一、贴切的tip类比。按h2标题关键词匹配最佳类比。

## 使用方法
```python
import hashlib, random

def generate_tip(h2_title, mfr, idx):
    topic = clean_h2_title(h2_title)[:25]
    analogies = find_best_analogy(h2_title)
    seed = hashlib.md5(f"{mfr}:{h2_title}:{idx}".encode()).hexdigest()[:8]
    random.seed(int(seed, 16))
    analogy, detail = random.choice(analogies)
    return f'{topic}就像{analogy}——{detail}。'
```

## 类比字典 (30+主题)

### 注意力机制
- 显微镜调焦 → 对焦越精确看到的细节越多，但视野越窄
- 老师课堂提问 → 一次只关注举手的学生(高权重token)
- 搜索引擎排序 → 根据关键词相关度(注意力权重)排序结果

### MoE架构
- 医院分诊台 → 根据症状分配专科医生
- 餐厅后厨 → 凉菜师、热菜师、面点师各司其职
- 公司部门 → HR管招聘、财务管账、技术管开发

### 推理
- 快递分拣中心 → 包裹(请求)进来按地址(类型)分拣
- 自动贩卖机 → 投币(输入)→选择(路由)→出货(输出)
- 消防站响应 → 平时待命，警报来了快速出动

### 训练
- 飞行员模拟器 → 在模拟环境中反复练习直到应对各种情况
- 运动员集训 → 系统性训练+定期考核+针对性补弱
- 高考备考 → 大量刷题→总结规律→查漏补缺

### 量化
- 货币换算 → 美元→人民币精度损失但面值等价
- 照片分辨率 → 4K→1080p→720p越来越小但主体清晰

### 长上下文
- 长篇小说 → 需要记住前面的伏笔才能理解后面
- 马拉松 → 不是冲刺速度而是持久耐力

### 多模态
- 翻译官 → 听一种语言→理解→说另一种语言
- 全科医生 → 看化验单+听描述+触诊综合判断

### Agent
- 项目经理 → 接到任务→拆解→分配给专家→汇总结果
- 管家 → 理解主人需求→安排厨师做饭+司机出行

### 部署
- 城市供水系统 → 水厂(模型)→管道(网络)→水龙头(终端)
- 物流网络 → 仓库(数据中心)→分拣中心(推理节点)→快递柜(终端)

### 对齐
- 新员工培训 → 先学规章制度再跟师傅学实际操作
- 导航校准 → GPS定位需要和实际位置对齐

### 数据
- 食材采购 → 好的食材是好菜的基础
- 矿石冶炼 → 原矿→选矿→冶炼→精炼

### 评估
- 体检报告 → 单项指标正常不代表整体健康
- 高考 → 分数反映能力但不完全还要看综合素质
